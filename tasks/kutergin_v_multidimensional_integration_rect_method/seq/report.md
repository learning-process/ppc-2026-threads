## 1. Контекст

Численное интегрирование многомерных функций является вычислительно трудоемкой
задачей, необходимой в физическом моделировании и статистике. Данная
последовательная реализация служит базовым эталоном. Она необходима для
подтверждения корректности математической логики алгоритма и используется для
расчета ускорения и эффективности в последующих параллельных реализациях (`OMP`,
`TBB`, `STL`, `ALL`).

## 2. Постановка задачи

- **Входные данные:**
  - Список границ интегрирования для каждой размерности $[a_i, b_i]$.
  - Количество шагов разбиения $n_i$ для каждой размерности.
  - Математическая функция $f(x_1, x_2, \dots, x_N)$, заданная через
    `std::function`.
- **Выходные данные:** Приближенное значение гиперобъема (результат интеграла)
  типа `double`.
- **Ограничения и крайние случаи:**
  - Количество размерностей $d \ge 1$.
  - Количество шагов по каждой оси $n_i > 0$.
  - Алгоритм должен корректно обрабатывать интервалы нулевой длины ($a_i =
    b_i$).

## 3. Базовый алгоритм

Реализована многошаговая схема метода средних прямоугольников. Пространство
интегрирования представляется в виде сетки из элементарных гиперпрямоугольников.
Значение функции вычисляется в геометрическом центре каждого элемента.

Для поддержки произвольной размерности $N$ без использования глубокой рекурсии
(которая может привести к переполнению стека) применена **линеаризация
итерационного пространства**:

1. Вычисляется общее число итераций как произведение шагов по всем осям: $Total
   = \prod n_i$.
2. Выполняется один цикл от $0$ до $Total$.
3. Внутри цикла многомерный индекс $(i_1, i_2, \dots, i_N)$ восстанавливается из
   линейного индекса $k$ с помощью операций деления и взятия остатка.

- **Асимптотика по времени:** $O(Total \cdot N)$, где $Total$ — общее число
  ячеек сетки, $N$ — размерность.
- **Асимптотика по памяти:** $O(N)$ для хранения текущих координат и шагов.

## 4. Детали реализации

Файлы: `seq/include/rect_method_seq.hpp`, `seq/src/rect_method_seq.cpp`

- **`ValidationImpl()`**: Проверяет согласованность размеров векторов входных
  данных и гарантирует, что количество шагов по всем осям положительно.
- **`PreProcessingImpl()`**: Копирует входные данные в локальную структуру
  `local_input_` и инициализирует переменную результата `res_` нулем.
- **`RunImpl()`**: Выполняет основной вычислительный цикл. Рассчитывает шаги
  $h$, элементарный гиперобъем $dV$ и аккумулирует значения функции в центрах
  ячеек.
- **`PostProcessingImpl()`**: Передает накопленный итоговый результат в выходной
  буфер фреймворка.

**Листинг ключевого фрагмента (`RunImpl`):**

```cpp
size_t dims = local_input_.limits.size();  // число размерностей пространства
std::vector<double> coords(dims);          // создание вектора координат размером dims
size_t total_iterations = 1;
std::vector<double> h(dims);

double d_v = 1.0;
for (size_t i = 0; i < dims; i++) {
  total_iterations *= local_input_.n_steps[i];
  h[i] = (local_input_.limits[i].second - local_input_.limits[i].first) / local_input_.n_steps[i];
  d_v *= h[i];
}

double total_sum = 0.0;
std::vector<int> current_indices(dims, 0);

for (size_t i = 0; i < total_iterations; ++i) {
  // Вычисление координат центра текущей ячейки
  for (size_t dm = 0; dm < dims; ++dm) {
    coords[dm] = local_input_.limits[dm].first + ((current_indices[dm] + 0.5) * h[dm]);
  }
  total_sum += local_input_.func(coords);
  // Обновление многомерного индекса
  for (int dm = static_cast<int>(dims) - 1; dm >= 0; --dm) {
    current_indices[dm]++;
    if (current_indices[dm] < local_input_.n_steps[dm]) {
      break;
    }
    current_indices[dm] = 0;
  }
}

res_ = total_sum * d_v;
```

## 5. Проверка корректности

Корректность последовательной реализации проверялась путем сравнения полученного
численного результата с аналитическим решением определенных интегралов
(вычисленных по формуле Ньютона-Лейбница). Поскольку метод средних
прямоугольников является точным для констант и линейных функций, ожидалось
минимальное расхождение результатов.

**Методика тестирования:**

- Сравнение вещественных чисел производилось с абсолютной погрешностью $\epsilon
  = 10^{-4}$.
- Тестовое покрытие включает размерности от 1D до 5D.
- Проверялись различные свойства подынтегральных функций: симметрия,
  константность и линейная зависимость.

**Характерные примеры входа и ожидаемого выхода:**

1. **1D Линейная функция ($f(x) = x$):**
    - **Вход:** Границы $[0.0, 1.0]$, шаги $n=100$.
    - **Математический расчет:** $\int_{0}^{1} x \,dx = \left[ \frac{x^2}{2}
      \right]_{0}^{1} = 0.5$.
    - **Ожидаемый выход:** `0.5`.
    - *Суть теста:* Проверка базовой логики и точности на одной размерности.

2. **2D Линейная функция ($f(x, y) = x + y$):**
    - **Вход:** Область $[0.0, 1.0] \times [0.0, 1.0]$, шаги $n=\{50, 50\}$.
    - **Математический расчет:** $\int_{0}^{1}\int_{0}^{1} (x+y) \,dx\,dy =
      1.0$.
    - **Ожидаемый выход:** `1.0`.
    - *Суть теста:* Проверка работы алгоритма при вложенности измерений.

3. **3D Константа ($f(x, y, z) = 1.0$):**
    - **Вход:** Параллелепипед $[0.0, 2.0] \times [0.0, 3.0] \times [0.0, 1.0]$,
      шаги $n=\{20, 20, 20\}$.
    - **Математический расчет:** Объем фигуры $V = 2 \times 3 \times 1 = 6.0$.
    - **Ожидаемый выход:** `6.0`.
    - *Суть теста:* Проверка корректности вычисления многомерного гиперобъема
      ($dV$).

4. **4D Линейная функция ($f = x_1 + x_2 + x_3 + x_4$):**
    - **Вход:** Гиперкуб $[0.0, 1.0]^4$, шаги $n=\{5, 5, 5, 5\}$.
    - **Математический расчет:**
      $\int_{0}^{1}\int_{0}^{1}\int_{0}^{1}\int_{0}^{1} (x_1+x_2+x_3+x_4)
      \,dx_1\,dx_2\,dx_3\,dx_4 = 4 \times 0.5 = 2.0$.
    - **Ожидаемый выход:** `2.0`.
    - *Суть теста:* Проверка корректности итеративного обхода для четырехмерного
      пространства на грубой сетке.

5. **5D Сумма координат ($f = \sum_{i=1}^{5} x_i$):**
    - **Вход:** Границы $\{[0,2], [0,3], [0,1], [0,1], [0,1]\}$, шаги $n=\{3, 3,
      3, 3, 3\}$.
    - **Ожидаемый выход:** `24.0`.
    - *Суть теста:* Проверка устойчивости алгоритма на высокой размерности с
      малым количеством шагов.

Во всех сценариях последовательная версия последовательная версия успешно
проходит валидацию.

## 8. Выводы

Разработанная последовательная версия корректно выполняет численное
интегрирование в пространствах произвольной размерности методом средних
прямоугольников. Алгоритм успешно обрабатывает различные конфигурации входных
данных, включая краевые случаи (интервалы нулевой длины) и высокую размерность
(до 5D включительно).

**Почему эта версия используется как эталон для ускорения:**

1. **Отсутствие накладных расходов:** Данная версия представляет собой «чистую»
   вычислительную логику без затрат на управление пулом потоков, синхронизацию
   кэшей или упаковку/рассылку сообщений MPI. Она фиксирует реальную
   производительность выполнения математических операций и вызовов
   `std::function` на одном физическом ядре.
2. **Гарантия математической точности:** Реализация прошла расширенный набор
   функциональных тестов, подтвердив свою устойчивость к накоплению ошибок
   округления. Результаты этой версии используются как стандарт для проверки
   корректности параллельных реализаций. Любое отклонение в результатах OMP,
   TBB, STL или ALL будет свидетельствовать о наличии гонок данных (data race)
   или ошибках декомпозиции, а не о проблемах в самом методе интегрирования.
3. **База для метрик:** Поскольку во всех последующих реализациях используется
   идентичное вычислительное ядро, время SEQ-версии в режиме `task_run`
   принимается за $T_1$. Это позволяет применить закон Амдала для точного
   расчета ускорения ($S = T_1 / T_p$) и эффективности ($E = S / p$), выявляя
   чистую выгоду от использования конкретных технологий параллельного
   программирования.
