## 1. Introduction

Данная работа посвящена реализации базового последовательного алгоритма поразрядной сортировки (Radix Sort) для чисел
с плавающей точкой двойной точности (`double`) с последующим объединением блоков с помощью четно-нечетного слияния
Бэтчера (Odd-Even Batcher Merge). Ожидаемый результат — создание корректной последовательной версии (SEQ), которая
станет эталоном для проверки правильности и оценки ускорения в будущих параллельных реализациях.

## 2. Problem Statement

**Задача:** Отсортировать массив элементов типа `double`.
**Ограничения:**

- Данные типа `double` нельзя напрямую сортировать классической поразрядной сортировкой из-за стандарта IEEE 754.
- Необходимо реализовать логику четно-нечетного слияния Бэтчера, требующую, чтобы количество сливаемых блоков было
  степенью двойки (в рамках подготовки к параллельным версиям).
- Алгоритм должен выполняться строго в одном потоке.

## 3. Baseline Algorithm (Sequential)

Последовательный алгоритм состоит из двух этапов:

1. **Локальная сортировка (LSD Radix Sort):** Массив чисел `double` побитово преобразуется в `uint64_t`. Знаковый
   бит инвертируется (для сохранения естественного порядка отрицательных и положительных чисел). Применяется
   8-проходная (побайтовая) сортировка подсчетом. Затем числа конвертируются обратно.
2. **Слияние Бэтчера:** Массив логически делится на блоки (для симуляции слияния), которые попарно сливаются
   итеративным алгоритмом.

## 4. Execution Scheme (Sequential)

**Структура выполнения:**

- **Подготовка данных:** Если размер массива не является степенью двойки, он дополняется максимальными значениями
  (padding), чтобы алгоритм Бэтчера работал корректно.
- **Сортировка:** Весь массив сортируется единым проходом функции `RadixSortDouble`.
- **Очистка:** Добавленные фиктивные элементы удаляются (через `resize` массива до исходного размера) перед
  выдачей результата.

## 5. Implementation Details

- **Преобразование типов:** Использована функция `std::memcpy` для безопасного (strict-aliasing safe)
  преобразования `double` в `uint64_t` и обратно.
- **Сортировка подсчетом:** Реализована с выделением промежуточных буферов. Для каждого из 8 байт (проходов)
  создается локальный массив счетчиков `count` размером 256 элементов.
- **Оптимизация памяти:** Вектор `temp` выделяется один раз до начала циклов по байтам, что позволяет избежать
  аллокаций внутри горячего цикла сортировки.

## 7. Results and Discussion
