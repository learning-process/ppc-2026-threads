## 1. Introduction

Данная работа посвящена реализации параллельного алгоритма поразрядной сортировки (Radix Sort) для чисел
с плавающей точкой двойной точности (`double`) с последующим объединением блоков с помощью четно-нечетного слияния
Бэтчера (Odd-Even Batcher Merge). В качестве технологии распараллеливания используется библиотека Intel Threading
Building Blocks (oneTBB), ориентированная на задачи (task-based parallelism).

## 2. Problem Statement

**Задача:** Отсортировать массив элементов типа `double`.
**Ограничения:**

- Данные типа `double` нельзя напрямую сортировать классической поразрядной сортировкой из-за стандарта IEEE 754.
- Необходимо использовать четно-нечетное слияние Бэтчера, требующее, чтобы количество сливаемых блоков было
  степенью двойки.
- Требуется использовать примитивы библиотеки TBB для максимальной балансировки нагрузки между ядрами.

## 3. Baseline Algorithm (Sequential)

Последовательный алгоритм состоит из двух этапов:

1. **Локальная сортировка (LSD Radix Sort):** Массив чисел `double` побитово преобразуется в `uint64_t`. Знаковый
   бит инвертируется (для сохранения естественного порядка отрицательных и положительных чисел). Применяется
   8-проходная (побайтовая) сортировка подсчетом. Затем числа конвертируются обратно.
2. **Слияние Бэтчера:** Массив делится на логические блоки, которые попарно сливаются итеративным алгоритмом.

## 4. Parallelization Scheme (oneTBB)

**Тасковая (задачная) архитектура:**

- **Управление ареной:** Используется `tbb::task_arena` для явного контроля над максимальным количеством потоков,
  выделенных под задачу.
- **Распределение задач (tbb::parallel_for):** Цикл локальных сортировок и циклы на этапах слияния Бэтчера
  разбиваются на итерационные пространства с помощью `tbb::blocked_range<int>`. TBB автоматически делит этот
  диапазон на подзадачи (tasks).
- **Балансировка (Work-stealing):** Планировщик TBB динамически перераспределяет задачи (слияние отдельных
  пар блоков) между простаивающими рабочими потоками, сводя к минимуму время простоя ядер.

## 5. Implementation Details

- **Захват контекста лямбдами:** Логика вычислений передается в TBB-алгоритмы в виде лямбда-выражений. Для этого
  строго контролируется захват переменных (сырые указатели на данные `raw_data` и размеры блоков `size`
  копируются по значению для потокобезопасности).
- **Интеграция с GCC 14:** Использование TBB позволяет элегантно избавиться от сложной логики ручной синхронизации.
  Все `tbb::parallel_for` выполняются синхронно относительно вызывающего потока, что обеспечивает естественный
  барьер перед переходом к следующей итерации сети Бэтчера.

## 6. Experimental Setup

- **Hardware:** 13th Gen Intel(R) Core(TM) i5-13420H, 12 логических ядер (потоков)
- **OS:** Ubuntu Linux (Docker container)
- **Toolchain:** GNU C++ Compiler (g++-14), CMake 3.28, OpenMPI, oneTBB. Build Type: Release (-O3)
- **Environment:** `PPC_NUM_THREADS` варьировался для тестирования.
- **Data:** Массивы генерировались внутренним механизмом `ppc_perf_tests`.

## 7. Results and Discussion

### 7.1 Correctness

Корректность TBB-реализации подтверждена прохождением набора функциональных тестов (`ppc_func_tests`). Алгоритм
работает детерминированно благодаря строгой изоляции обрабатываемых диапазонов `blocked_range` в сети Бэтчера.

### 7.2 Performance

Время измерялось метрикой `task_run` (чистое время работы алгоритма).
Базовое время последовательной версии (SEQ) на 1 потоке: **0.01237 s**.

| Mode | Count | Time, s | Speedup | Efficiency |
|------|-------|---------|---------|------------|
| seq  | 1     | 0.01237 | 1.00    | 100.0%     |
| omp  | 4     | 0.00831 | 1.48    | 37.0%      |
| tbb  | 4     | 0.00764 | 1.61    | 40.2%      |
| tbb  | 8     | 0.00664 | 1.86    | 23.2%      |
| stl  | 4     | 0.00995 | 1.24    | 31.0%      |

**Анализ результатов:**
Технология TBB показала абсолютное лидерство среди одноузловых решений (ускорение 1.61x на 4 потоках и 1.86x
на 8 потоках). Интеллектуальный планировщик `task_arena` с механизмом кражи задач (work-stealing) блестяще
справился с балансировкой большого количества мелких независимых слияний на последних этапах алгоритма Бэтчера,
где ручное разделение (как в STL или OMP) приводит к простоям потоков.

## 8. Conclusions

Реализация с использованием Intel Threading Building Blocks признана наиболее эффективной для распараллеливания
сети слияния Бэтчера в рамках одного вычислительного узла. TBB предоставляет удобные высокоуровневые
абстракции (алгоритмы поверх пула потоков), которые минимизируют накладные расходы на малых объемах данных и
отлично масштабируются.
