## 4. Схема распараллеливания

Распараллеливание реализовано с помощью алгоритма `tbb::parallel_for`.

- **Примитив:** Выбран `tbb::parallel_for`, так как итерации внешнего цикла по этапам (`stage`)
  и по блокам строк (`i`) независимы.
- **Диапазон:** Используется `tbb::parallel_for(0, num_blocks, ...)`, который разбивает итерации от
  0 до `num_blocks-1` на чанки.
- **Разбиение (Partitioner):** Используется `tbb::auto_partitioner` (по умолчанию), который адаптивно
  подбирает размер чанков для балансировки нагрузки.
- **`grainsize`:** Явно не задавался, используется значение по умолчанию, подходящее для большинства задач.
- **Конкуренция:** Ограничивается с помощью `tbb::global_control::max_allowed_parallelism`, устанавливая
  желаемое число рабочих потоков. Фактическое число потоков управляется раннером курса.

### Листинг ключевого фрагмента (ядро распараллеливания)

```cpp
void FoxAlgorithmImpl(...) {
  // ...
  tbb::parallel_for(0, num_blocks, [&](int stage) {
    for (int i = 0; i < num_blocks; ++i) {
      int broadcast_block = (i + stage) % num_blocks;
      for (int j = 0; j < num_blocks; ++j) {
        MultiplyBlock(a, b, result, i * block_size, j * block_size, block_size,
                      broadcast_block * block_size, j * block_size);
      }
    }
  });
}```

Внешний цикл по `stage` параллелится. Для каждого этапа все блоки `(i, j)` обрабатываются последовательно. Такая схема выбрана потому, 
что умножения блоков внутри одного этапа независимы и в TBB нет необходимости в примитивах синхронизации, т.к. каждый поток работает со 
своей уникальной парой блоков.

## 5. Детали реализации

*   **Файлы:** `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`
*   **Класс:** `YakimovIMultOfDenseMatricesFoxAlgorithmTBB`
*   **Изменения относительно SEQ:** Логика предобработки и постобработки аналогична версии OpenMP. В `RunImpl` вызов `FoxAlgorithmImpl` 
использует `tbb::parallel_for`.
*   **Особенности:** В отличие от версии OpenMP, здесь нет директивы `atomic`. Гонки данных отсутствуют, так как каждый результирующий 
блок `(i, j)` на каждом этапе `stage` обновляется только одним потоком.
