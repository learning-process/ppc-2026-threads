Базовый алгоритм полностью совпадает с последовательной версией: используется блочный алгоритм Фокса для квадратных матриц и тройной цикл для неквадратных.

## 4. Схема распараллеливания
Основная идея заключается в распараллеливании итераций по этапам (`stage`) и блокам строк (`i`).

*   **Параллелизуемая область:** Циклы по `stage` и `i` являются независимыми, так как их перестановка не влияет на конечный результат. Для их распараллеливания используется директива `#pragma omp parallel for collapse(2)`.
*   **Переменные:**
    *   `shared(a, b, result, num_blocks, block_size)`: Все матрицы и параметры задач являются общими для всех потоков.
    *   `private(stage, i, broadcast_block, j)`: Переменные цикла и промежуточные вычисления являются локальными для каждого потока.
*   **Синхронизация:** Ключевым моментом является операция накопления результата в `result(...) += sum`. Несколько потоков могут одновременно пытаться обновить один и тот же элемент `result`. Для обеспечения атомарности этой операции используется `#pragma omp atomic`.
*   **Выбор schedule:** Директива `collapse(2)` объединяет два цикла в один, итерации которого статически распределяются между потоками. Это подходит, так как вычислительная нагрузка между итерациями циклов `stage` и `i` примерно одинакова.
*   **Барьеры:** Неявный барьер в конце `parallel for` гарантирует, что все потоки завершат все этапы, прежде чем функция завершится.

### Листинг ключевого фрагмента (ядро распараллеливания)

```cpp
void FoxAlgorithmImpl(...) {
  // ...
  #pragma omp parallel for collapse(2) default(none) shared(a, b, result, num_blocks, block_size)
  for (int stage = 0; stage < num_blocks; ++stage) {
    for (int i = 0; i < num_blocks; ++i) {
      int broadcast_block = (i + stage) % num_blocks;
      for (int j = 0; j < num_blocks; ++j) {
        MultiplyBlock(a, b, result, i * block_size, j * block_size, block_size,
                      broadcast_block * block_size, j * block_size);
      }
    }
  }
}

void MultiplyBlock(...) {
  // ...
  #pragma omp atomic
  result(row_start + i, col_start + j) += sum;
}```

Директива `parallel for` распределяет итерации между потоками. Директива `atomic` внутри `MultiplyBlock` гарантирует, что накопление в результирующую матрицу будет происходить без гонок данных.

## 9. Выводы

Реализация на OpenMP демонстрирует значительное ускорение, особенно при использовании количества потоков, не превышающего число физических ядер (6). `collapse(2)` эффективно распределяет работу, а `atomic` предотвращает гонки данных, но вносит свой вклад в издержки на больших потоках. OpenMP является хорошим выбором для данной задачи благодаря простоте реализации и хорошей масштабируемости в пределах одного узла.
