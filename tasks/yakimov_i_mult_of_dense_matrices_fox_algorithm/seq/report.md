## 1. Контекст

Данная реализация служит эталонным последовательным алгоритмом для задачи умножения плотных матриц.
Основная цель — предоставить корректную реализацию для последующего сравнения производительности
с параллельными версиями (OpenMP, TBB, STL, гибридная - all). Также эта версия используется для
валидации корректности всех остальных реализаций.

## 2. Постановка задачи

- **Входные данные:** Две матрицы A и B, хранящиеся в текстовых файлах. Первая строка файла
  содержит размеры матрицы (строки и столбцы), последующие строки содержат элементы матрицы.
- **Выходные данные:** Сумма всех элементов результирующей матрицы C = A * B в виде числа двойной точности.
- **Ограничения:**
  - Количество столбцов матрицы A должно равняться количеству строк матрицы B для выполнения умножения
  (`A.cols == B.rows`).
  - Алгоритм умеет обрабатывать как квадратные, так и неквадратные матрицы.
  - Для неквадратных матриц используется стандартный алгоритм тройного вложенного цикла.
- **Крайние случаи:** Матрицы с нулевыми размерами, единичные матрицы, матрицы с нулевыми элементами.

## 3. Базовый алгоритм

Для квадратных матриц реализован **Алгоритм Фокса**:

1. Матрицы A и B делятся на блоки размера `block_size x block_size`.
2. Результирующая матрица C инициализируется нулями.
3. Для каждого этапа `stage` от `0` до `num_blocks - 1`:
    - Вычисляется смещение `broadcast_block = (i + stage) % num_blocks`.
    - Для каждого блока `(i, j)` выполняется умножение блока матрицы A (`i`, `broadcast_block`) на блок
  матрицы B (`broadcast_block`,
    - `j`) с накоплением в результирующем блоке `(i, j)`.

- **Асимптотика по времени:** O(n^3) для стандартного умножения. Для блочного алгоритма Фокса также O(n^3),
  так как общее количество операций не меняется.
- **Асимптотика по памяти:** O(n^2) для хранения трех матриц.
- **Инварианты:** Результат `C` всегда равен `A * B` в пределах погрешности вычислений с плавающей точкой.
  Блочное представление не нарушает математического результата.

## 4. Детали реализации

- **Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`
- **Класс:** `YakimovIMultOfDenseMatricesFoxAlgorithmSEQ`
- **Методы:**
  - `ValidationImpl()`: Проверяет, что входной идентификатор задачи > 0, а выходное значение еще не
  было установлено.
  - `PreProcessingImpl()`: Считывает матрицы A и B из файлов `A_{id}.txt` и `B_{id}.txt`, проверяет возможность
  умножения (`A.cols == B.rows`). Для квадратных матриц вычисляет оптимальный размер блока (начиная с 64 и удваивая,
  но не более 256 и размера матрицы).
  - `RunImpl()`: Вызывает функцию `FoxAlgorithmImpl`, которая реализует алгоритм Фокса для квадратных матриц,
  делящихся на блоки без остатка, или стандартный тройной цикл для всех остальных случаев.
  - `PostProcessingImpl()`: Вычисляет сумму всех элементов результирующей матрицы и сохраняет её в выходные данные.

### Листинг ключевого фрагмента (последовательное умножение блока)

```cpp
void MultiplyBlock(const DenseMatrix &a, const DenseMatrix &b, DenseMatrix &result,
                   int row_start, int col_start, int block_size,
                   int a_row_offset, int b_col_offset) {
  for (int i = 0; i < block_size; ++i) {
    for (int j = 0; j < block_size; ++j) {
      double sum = 0.0;
      for (int k = 0; k < block_size; ++k) {
        sum += a(row_start + i, a_row_offset + k) * b(b_col_offset + k, col_start + j);
      }
      result(row_start + i, col_start + j) += sum;
    }
  }
}```

Данный код выполняет умножение двух блоков матриц. Внешние два цикла (`i` и `j`) перебирают элементы результирующего блока, а внутренний 
цикл (`k`) выполняет скалярное произведение строки блока A и столбца блока B. Результат накапливается в соответствующем элементе матрицы 
`result`. Этот фрагмент является ядром всего вычисления и в последовательной версии выполняется без какой-либо параллелизации.
