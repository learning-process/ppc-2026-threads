## 4. Схема распараллеливания

Работа распределяется между потоками путем статического разбиения этапов алгоритма Фокса.

- **Разбиение данных:** Итерации внешнего цикла по этапам (`stage`) независимы. Общее количество
  этапов (`num_blocks`) статически распределяется между потоками.
- **Работа потока:** Каждый поток получает непрерывный диапазон этапов и обрабатывает их последовательно,
  выполняя все необходимые умножения блоков для этих этапов.
- **Локальные результаты:** Отсутствуют, так как потоки накапливают результат непосредственно в общую
  матрицу `result`. Это безопасно, потому что на разных этапах потоки работают с разными результирующими блоками.
- **Синхронизация:** `std::thread::join()` используется в главном потоке для ожидания завершения всех
  созданных потоков. Никаких мьютексов или атомарных операторов не требуется, так как запись в результирующую
  матрицу происходит без конфликтов (каждая пара `(i, j)` на каждом этапе обрабатывается только одним потоком).
- **Управление потоками:** Потоки создаются в конструкторе, и каждому присваивается функция `ProcessStage` с
  его диапазоном этапов.

### Листинг ключевого фрагмента (создание и запуск потоков)

```cpp
void LaunchThreads(..., int num_blocks, int block_size, unsigned int num_threads) {
  std::vector<std::thread> threads;
  int stages_per_thread = num_blocks / static_cast<int>(num_threads);
  int remaining_stages = num_blocks % static_cast<int>(num_threads);
  int stage_start = 0;

  for (unsigned int thread_index = 0U; thread_index < num_threads; ++thread_index) {
    int stages_for_this_thread = stages_per_thread + (thread_index < remaining_stages ? 1 : 0);
    int stage_end = stage_start + stages_for_this_thread;
    threads.emplace_back([&a, &b, &result, stage_start, stage_end, num_blocks, block_size]() {
      for (int stage = stage_start; stage < stage_end; ++stage) {
        ProcessStage(a, b, result, stage, num_blocks, block_size);
      }
    });
    stage_start = stage_end;
  }
  for (std::thread &thread : threads) { thread.join(); }
}```

Этот код создает пул потоков, статически распределяя между ними этапы алгоритма. Каждый поток независимо обрабатывает свой диапазон 
этапов.

## 9. Выводы

Реализация на `std::thread` полностью работоспособна и демонстрирует хорошее ускорение. Однако она требует больше ручного кода для 
разбиения работы и управления потоками по сравнению с OpenMP и TBB. Это делает её более подверженной ошибкам и менее гибкой в условиях 
неравномерной нагрузки. Там, где возможно, предпочтительнее использовать более высокоуровневые абстракции.
