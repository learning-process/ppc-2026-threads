## 1. Контекст

Задача обнаружения границ (рёбер) на изображении является одной из базовых задач компьютерного зрения.
Оператор Собеля — классический градиентный метод, вычисляющий приближённую производную яркости изображения
в двух направлениях: горизонтальном и вертикальном. Результат позволяет выделить резкие перепады яркости,
соответствующие границам объектов. Последовательная реализация служит эталоном корректности и базой для
измерения ускорения параллельных версий.

## 2. Постановка задачи

**Входные данные**: контейнер `InType = ImgContainer`, содержащий:

- `width` — ширина изображения в пикселях;
- `height` — высота изображения в пикселях;
- `pixels` — вектор значений пикселей в диапазоне `[0, 255]`.

Входное изображение может быть полутоновым (`grayscale`) или цветным. Цветные изображения форматов
`.png`, `.jpg`, `.bmp` автоматически преобразуются в полутоновые по формуле:

```bash
gray = 0.299 * R + 0.587 * G + 0.114 * B
```

Поддерживается также текстовый формат `.txt`: первая строка содержит размеры `width height`,
далее следуют значения пикселей.

**Выходные данные**: `OutType = std::vector<int>` — вектор магнитуд градиента в диапазоне `[0, 255]`,
размером `width * height`.

**Ограничения**: `width >= 1`, `height >= 1`, все значения пикселей принадлежат `[0, 255]`,
функция непрерывно определена для всей области изображения.

## 3. Базовый алгоритм

Алгоритм применяет оператор Собеля к каждому внутреннему пикселю изображения:

1. Для каждого пикселя `(row, col)`, где `1 <= row < height-1` и `1 <= col < width-1`,
   вычисляется свёртка с двумя ядрами `3×3`:

```bash
Gx = [[-1, 0, 1],      Gy = [[-1, -2, -1],
      [-2, 0, 2],             [ 0,  0,  0],
      [-1, 0, 1]]             [ 1,  2,  1]]
```

2. По полученным значениям вычисляется магнитуда градиента:

```bash
magnitude = sqrt(Gx^2 + Gy^2)
```

3. Результат ограничивается диапазоном `[0, 255]` и записывается в выходной вектор.

4. Граничные пиксели (первая и последняя строки, первый и последний столбцы) остаются нулевыми,
   так как для них невозможно применить полное ядро `3×3`.

**Временная сложность**: O(width × height). **Память**: O(width × height).

## 4. Детали реализации

Реализация расположена в файлах `seq/include/ops_seq.hpp` и `seq/src/ops_seq.cpp`.
Класс `ShakirovaESobelEdgeDetectionSEQ` наследуется от `BaseTask`.

`ValidationImpl` делегирует проверку входных данных методу `GetInput().IsValid()`,
который проверяет корректность размеров и вектора пикселей.

`PreProcessingImpl` сохраняет ширину, высоту и вектор пикселей в поля класса,
а также инициализирует выходной вектор нулями нужного размера.

Основная логика вычислений реализована в `RunImpl`. Ядра Собеля заданы как
`std::array<std::array<int, 3>, 3>` и хранятся локально, что исключает лишние аллокации.
Обход пикселей выполняется в лексикографическом порядке (строка за строкой).
Для каждого внутреннего пикселя вычисляются `gx` и `gy`, затем — магнитуда с `std::clamp`.

`PostProcessingImpl` не выполняет дополнительных действий и возвращает `true`.

Эталонная реализация метода `RunImpl`:

```cpp
bool RunImpl() {
  const std::array<std::array<int, 3>, 3> k_gx = {{{-1, 0, 1}, {-2, 0, 2}, {-1, 0, 1}}};
  const std::array<std::array<int, 3>, 3> k_gy = {{{-1, -2, -1}, {0, 0, 0}, {1, 2, 1}}};

  auto &out = GetOutput();

  for (int row = 1; row < height_ - 1; ++row) {
    for (int col = 1; col < width_ - 1; ++col) {
      int gx = 0;
      int gy = 0;

      for (int ky = -1; ky <= 1; ++ky) {
        for (int kx = -1; kx <= 1; ++kx) {
          const int pixel = input_[((row + ky) * width_) + (col + kx)];
          const auto ky_idx = static_cast<size_t>(ky) + 1;
          const auto kx_idx = static_cast<size_t>(kx) + 1;
          gx += pixel * k_gx.at(ky_idx).at(kx_idx);
          gy += pixel * k_gy.at(ky_idx).at(kx_idx);
        }
      }

      const int magnitude =
          static_cast<int>(std::sqrt(static_cast<double>((gx * gx) + (gy * gy))));
      out[(row * width_) + col] = std::clamp(magnitude, 0, 255);
    }
  }

  return true;
}
```

Основным преимуществом реализации является отсутствие динамических аллокаций внутри цикла
и использование `std::array` с фиксированным размером для ядер свёртки, что повышает
производительность кэша и предсказуемость вычислений.

Производительность измерялась в режимах `pipeline` и `task_run` на изображении достаточно
большого размера. Тесты запускались при различном числе MPI-процессов (`n = 2`, `n = 4`, `n = 6`).

| Число процессов | Режим     | Время (сек) |
| --------------- | --------- | ----------- |
| 2               | pipeline  | 0.1058      |
| 2               | task_run  | 0.0814      |

| 4               | task_run  | 0.1519      |

Наилучшее время достигается при `n = 2`, что соответствует ожидаемому поведению
для последовательной реализации: увеличение числа MPI-процессов не ускоряет вычисления,
а лишь добавляет накладные расходы на синхронизацию.

## 8. Выводы

Реализованная SEQ-версия оператора Собеля корректно выделяет рёбра на полутоновых
изображениях произвольного размера и формата. Все 6 функциональных тестов проходят
успешно при различных конфигурациях запуска. Последовательная реализация демонстрирует
линейную зависимость времени выполнения от числа пикселей и служит эталоном корректности
для последующих параллельных реализаций. Основное ограничение — отсутствие распараллеливания,
вследствие чего многоядерность архитектуры не используется.
