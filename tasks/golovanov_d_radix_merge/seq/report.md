## 1. Контекст

В данной работе реализована и исследована параллельная сортировка больших массивов чисел с плавающей запятой двойной точности (`double`). Последовательная версия служит эталонной реализацией, на которой основываются и с которой сравниваются все последующие параллельные версии (OpenMP, TBB, `std::thread`, MPI+гибрид). Она задаёт структуру пайплайна (Validation → PreProcessing → Run → PostProcessing) и обеспечивает базовую метрику производительности.

## 2. Постановка задачи

**Вход:** Вектор `std::vector<double>` произвольной длины `n ≥ 0`.

**Генерация данных (тесты производительности):**
Массив формируется симметрично: от `-count` до `+count`, где `count = 500 000`. Итоговый массив имеет `n = 1 000 001` элемент:
```
[-500000.0, -499999.0, ..., -1.0, 0.0, 1.0, ..., 499999.0, 500000.0]
```

**Выход:** Отсортированный по неубыванию вектор той же длины.

**Критерий корректности:** `std::ranges::is_sorted(output) == true`.

**Ограничения и крайние случаи:** `n ≥ 0`. При `n = 0` возвращается пустой вектор. При `n = 1` сортировка не производится.

## 3. Базовый алгоритм

Последовательный алгоритм включает следующие этапы:

1. **Преобразование чисел с плавающей точкой**  
   Число `double` интерпретируется как 64-битное целое (`uint64_t`). Для обеспечения правильного порядка отрицательных чисел выполняется преобразование:  
   `ToSortable(bits) = (bits & sign_mask) ? ~bits : (bits ^ sign_mask)`.

2. **LSD-поразрядная сортировка (Radix sort)**  
   - Выполняется 8 проходов (по одному на каждый байт 64-битного числа).  
   - На каждом проходе подсчитывается гистограмма (256 корзин), вычисляются префиксные суммы и выполняется перестановка элементов во временный буфер.  
   - После завершения всех проходов выполняется обратное преобразование `FromSortable`.

Асимптотика по времени: **O(k·n)** = **O(8·n)**, где k — число байт (константа).  
Асимптотика по памяти: **O(n)** для временного буфера.  

Инвариант: после каждого байтового прохода массив частично упорядочен по младшим байтам, а после восьмого прохода полностью отсортирован в порядке возрастания.

## 4. Детали реализации

**Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

В `ValidationImpl` проверяется неотрицательность входа (при `n = 0` задача считается корректной).  
В `PreProcessingImpl` происходит генерация тестового массива (симметричный диапазон).  
В `RunImpl` вызывается функция `RadixSort::Sort(arr)`, содержащая основную логику алгоритма.  
В `PostProcessingImpl` выполняется финальная проверка (в тестах производительности проверяется `is_sorted`).

Ключевой фрагмент реализации — цикл по байтам:

```cpp
// Файл: radix_sort.cpp
for (int byte = 0; byte < kBytes; ++byte) {
    std::array<size_t, kRadix> count{};

    // Подсчёт гистограммы
    for (size_t i = 0; i < n; ++i) {
        auto b = static_cast<size_t>((data[i] >> (byte * 8)) & kByteMask);
        count.at(b)++;
    }

    // Префиксные суммы (переход от counts к позициям)
    size_t sum = 0;
    for (auto &c : count) {
        size_t tmp = c;
        c = sum;
        sum += tmp;
    }

    // Перестановка элементов
    for (size_t i = 0; i < n; ++i) {
        auto b = static_cast<size_t>((data[i] >> (byte * 8)) & kByteMask);
        buffer[count.at(b)++] = data[i];
    }

    data.swap(buffer);
}
```

Данный фрагмент демонстрирует ядро LSD radix sort — последовательное выполнение восьми байтовых проходов.
