# Подробное описание oneTBB-реализации

## 1. Назначение TBB-версии

oneTBB-реализация предназначена для параллельного умножения разреженных матриц
с элементами типа `double`, представленных в формате CCS. Она сохраняет общую
модель данных последовательной версии, но распределяет вычисление столбцов
результата между рабочими задачами oneTBB.

Вычисляется произведение `C = A * B`. Входные и выходная матрицы используют
массивы `values`, `row_indices` и `col_ptrs`.

## 2. Формат CCS

В формате CCS ненулевые элементы хранятся по столбцам. Для столбца `j` диапазон
его элементов задается так:

```text
col_ptrs[j] .. col_ptrs[j + 1]
```

Массив `values` содержит ненулевые значения, а `row_indices` хранит номера
строк этих значений. Такой формат соответствует выбранной схеме вычислений,
потому что результат строится по столбцам.

## 3. Проверка входных данных

Перед вычислением проверяется корректность структуры обеих матриц:

- размер `col_ptrs` должен быть равен `cols + 1`;
- массивы `values` и `row_indices` должны иметь одинаковую длину;
- первый элемент `col_ptrs` должен быть равен нулю;
- последний элемент `col_ptrs` должен совпадать с числом ненулевых элементов;
- значения в `col_ptrs` должны быть неубывающими;
- индексы строк должны находиться в диапазоне `[0, rows)`.

Также проверяется условие совместимости размеров для умножения:
`A.cols == B.rows`.

## 4. Базовый алгоритм

Для каждого столбца матрицы `B` вычисляется соответствующий столбец матрицы
`C`. Временный плотный аккумулятор длины `A.rows` хранит значения текущего
столбца результата до переноса в CCS.

Если в столбце `j` матрицы `B` найден ненулевой элемент `B(k, j)`, то столбец
`k` матрицы `A` добавляет вклад:

```text
C(i, j) += A(i, k) * B(k, j)
```

После обработки столбца ненулевые значения аккумулятора записываются в
`values` и `row_indices`, а массив `col_ptrs` фиксирует границы столбцов.

## 5. Схема oneTBB-параллелизации

Работа делится по диапазону столбцов результата. Число частей выбирается как
минимум из количества потоков `PPC_NUM_THREADS` и числа столбцов матрицы `B`.
Каждая часть содержит непрерывный диапазон столбцов.

Для запуска параллельной работы используется `tbb::parallel_for`. Каждая
рабочая задача получает номер части и вычисляет только свой диапазон столбцов.
Внутри части создается локальная матрица CCS, куда записываются результаты
только этой части.

## 6. Независимость блоков и отсутствие гонок

Входные матрицы `A` и `B` являются общими для рабочих задач, но только
читаются. Массив границ частей также только читается после построения.

Каждая задача oneTBB записывает результат в отдельный элемент массива
частичных матриц. Общие массивы итоговой матрицы `values`, `row_indices` и
`col_ptrs` не изменяются во время параллельного участка. Поэтому гонки данных
при формировании частичных результатов отсутствуют.

## 7. Детерминированный порядок результата

После завершения `tbb::parallel_for` частичные матрицы объединяются
последовательно. Объединение выполняется в порядке возрастания номеров частей,
то есть в порядке столбцов результата.

Благодаря этому итоговые массивы `values`, `row_indices` и `col_ptrs`
получаются детерминированными и не зависят от порядка планирования задач
oneTBB.

## 8. Производительные режимы

Производительные тесты используют два режима инфраструктуры курса:

- `pipeline` — измеряет полный цикл задачи: проверку, подготовку, запуск и
  завершающий этап;
- `task_run` — измеряет основной вычислительный этап после подготовки данных.

Точные таблицы времени не приводятся, так как результат зависит от локального
окружения, параметров сборки и числа потоков.

## 9. Проверка корректности

Функциональные тесты сравнивают результат с заранее заданным CCS-представлением.
Проверяются размеры матрицы, массив `col_ptrs`, индексы `row_indices` и
значения `values`. Для элементов типа `double` используется допустимая
погрешность.

Набор тестов включает квадратные и прямоугольные матрицы, диагональные матрицы,
умножение на единичную матрицу и случай нулевого результата.

## 10. Команды запуска

Команды для локальной проверки TBB-реализации:

```powershell
cmake --build build --config Release --parallel
$env:PPC_NUM_THREADS="4"
.\build\bin\ppc_func_tests.exe --gtest_filter="*safaryan_a_sparse_matrix_mult_crs_tbb*"
.\build\bin\ppc_perf_tests.exe --gtest_filter="*safaryan_a_sparse_matrix_mult_crs_tbb*"
```

## 11. Выводы

oneTBB-версия распараллеливает вычисление по независимым диапазонам столбцов и
сохраняет CCS-формат результата. Локальные частичные матрицы исключают гонки
данных, а последовательное объединение сохраняет детерминированный порядок
элементов. Реализация подходит для сравнения с SEQ, OMP, STL и ALL версиями.
