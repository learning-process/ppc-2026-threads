## 1. Введение

Умножение плотных матриц — одна из фундаментальных операций линейной алгебры, применяемая в научных
расчётах, машинном обучении, компьютерной графике и многих других областях. Классический алгоритм
умножения двух матриц размера \(N \times N\) имеет вычислительную сложность \(O(N^3)\), что при больших
\(N\) приводит к значительным временным затратам. Для ускорения вычислений разработаны различные
блочные алгоритмы, один из которых — алгоритм Кэннона. Изначально он был предложен для параллельных
вычислительных систем с топологией решётки, но может быть реализован и как последовательный, сохраняя
блочную структуру. Данная работа посвящена реализации и анализу последовательной версии алгоритма
Кэннона для умножения квадратных плотных матриц с элементами типа `double`.

## 2. Постановка задачи

**Входные данные**:

- `block_dim` — размер блока (положительное целое число);
- `matrix_a`, `matrix_b` — квадратные матрицы размера \(N \times N\), заданные как
  `std::vector<std::vector<double>>`.

**Условия корректности**:

- `matrix_a` и `matrix_b` не пусты;
- `matrix_a` и `matrix_b` квадратные, имеют одинаковый размер \(N\);
- \(N\) делится на `block_dim` без остатка.

**Выходные данные**:

- `matrix_c` — результат умножения `matrix_a * matrix_b` (матрица \(N \times N\)).

Требуется реализовать последовательный алгоритм Кэннона с блочным разбиением, обеспечивающий
корректное умножение для любых допустимых входных данных.

## 3. Базовый (последовательный) алгоритм

В последовательной версии полностью реализована блочная схема Кэннона без какой-либо параллелизации.
Алгоритм состоит из следующих этапов:

1. **Разбиение**  
   Исходные матрицы \(A\) и \(B\) делятся на блоки размера `block_dim × block_dim`.
   Количество блоков по каждому измерению:
   \[
   \text{blocks} = \frac{N}{\text{block\_dim}}.
   \]

2. **Начальный циклический сдвиг**  
   Для каждого блока \((i, j)\) (где \(i,j = 0 \dots \text{blocks}-1\)) вычисляется сдвиг:
   \[
   \text{shift} = (i + j) \bmod \text{blocks}.
   \]
   - Блок \(A`[i][j]`\) заполняется элементами из \(A\) с индексами строк
     от \(i \cdot \text{block\_dim}\) до \((i+1)\cdot \text{block\_dim}-1\) и столбцов
     от \(\text{shift} \cdot \text{block\_dim}\) до \((\text{shift}+1)\cdot \text{block\_dim}-1\).
   - Блок \(B`[i][j]`\) заполняется элементами из \(B\) с индексами строк
     от \(\text{shift} \cdot \text{block\_dim}\) до \((\text{shift}+1)\cdot \text{block\_dim}-1\) и столбцов
     от \(j \cdot \text{block\_dim}\) до \((j+1)\cdot \text{block\_dim}-1\).

3. **Основной цикл (повторяется `blocks` раз)**  
   Для каждого шага `step = 0 … blocks-1`:
   - Для всех пар блоков \((i, j)\) выполняется умножение блоков:
     \[
     C`[i][j]` \mathrel{+}= A`[i][j]` \times B`[i][j]`,
     \]
     где умножение блоков производится стандартным тройным вложенным циклом (все три цикла по
     `block_dim`).
   - Если `step < blocks-1`, выполняется циклический сдвиг:
     - Каждая строка блоков \(A\) сдвигается влево на одну позицию (первый блок строки перемещается
       в конец).
     - Каждый столбец блоков \(B\) сдвигается вверх на одну позицию (первый блок столбца
       перемещается вниз).

4. **Сборка результата**  
   Из накопленных блоков \(C`[i][j]`\) формируется итоговая матрица \(C\) размера \(N \times N\):
   элемент на позиции \((i \cdot \text{block\_dim} + b_i,\; j \cdot \text{block\_dim} + b_j)\) берётся
   из соответствующего блока.

## 4. Схема параллелизации (отсутствует)

Последовательная версия не использует никаких средств параллельного программирования. Все циклы
выполняются в одном потоке, синхронизация не требуется. Это позволяет использовать данную реализацию
как эталон для оценки ускорения, достигаемого параллельными версиями, и для проверки корректности.

### 5.2 Класс и основные методы

Класс `RemizovKDenseMatrixMultiplicationCannonAlgorithm` (унаследован от `BaseTask`) реализует
требуемый интерфейс:

- `ValidationImpl()` – проверяет, что `block_dim > 0`, матрицы не пусты, они квадратные, одного
  размера, и размер матрицы кратен `block_dim`.
- `PreProcessingImpl()` – очищает выходной вектор.
- `RunImpl()` – выполняет все этапы алгоритма: разбиение на блоки, начальный сдвиг, основной цикл,
  сборку результата.
- `PostProcessingImpl()` – всегда возвращает `true`.

### 5.3 Вспомогательные статические методы

- `MultiplyBlock(a, b, c, block_size)` – последовательное умножение двух блоков (тройной цикл).
- `InitializeBlocks(...)` – заполнение блоков A и B с учётом начального сдвига.
- `RunCannonCycle(...)` – основной цикл: умножение блоков и циклические сдвиги.
- `ShiftBlocksLeft(...)` – циклический сдвиг строк блоков A влево.
- `ShiftBlocksUp(...)` – циклический сдвиг столбцов блоков B вверх.
- `AssembleOutput(...)` – сборка итоговой матрицы из блоков C.

### 5.4 Представление блоков в памяти

Для хранения блоков используется четырёхмерный вектор:

```cpp
std::vector<std::vector<std::vector<std::vector<double>>>>
```cpp

Индексация: `blocks`[i][j]`[bi][bj]`, где `i, j` – индексы блока, `bi, bj` – индексы внутри блока.
Такое представление интуитивно понятно и соответствует блочной логике, но приводит к фрагментации
памяти. В последовательной версии это не критично.

**Анализ производительности**:

- Последовательная версия выполняет примерно \(N^3 = 1024^3 \approx 1.07 \times 10^9\) операций

- Основное время тратится на тройные циклы внутри `MultiplyBlock`. Общее количество вызовов

  типа double (~8 МБ) на каждую матрицу. Это умещается в кэш L3 (обычно 12 МБ у i5-1155G7), что
  способствует достаточно высокой производительности.

## Приложение (фрагмент кода)

```cpp
// Последовательное умножение двух блоков
void RemizovKDenseMatrixMultiplicationCannonAlgorithm::MultiplyBlock(
    const std::vector<std::vector<double>> &a,
    const std::vector<std::vector<double>> &b,
    std::vector<std::vector<double>> &c,
    int block_size) {
  for (int i = 0; i < block_size; ++i) {
    for (int j = 0; j < block_size; ++j) {
      double accumulator = 0.0;
      for (int k = 0; k < block_size; ++k) {
        accumulator += a[i][k] * b[k][j];
      }
      c`[i][j]` += accumulator;
    }
  }
}
```cpp
