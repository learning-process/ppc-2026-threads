## 1. Контекст

Последовательная версия алгоритма поразрядной сортировки служит отправной точкой проекта и базисом для всех последующих
параллельных реализаций (OpenMP, TBB, STL, MPI+STL). Разработка корректного и стабильного последовательного решения
необходима для генерации эталонных выходных данных, валидации граничных условий и вычисления точных метрик ускорения и
эффективности параллелизма на последующих этапах работы.

## 2. Постановка задачи

- **Входные данные:** Объект `std::vector<int>` размера $N$.

- **Выходные данные:** Объект `std::vector<int>` размера $N$, содержащий те же элементы, переставленные в порядке
  неубывания.

- **Ограничения и крайние случаи:**
  - Размер массива $N \ge 1$. В соответствии с логикой `ValidationImpl`, пустой входной вектор (`GetInput().empty()`)
      считается некорректным и блокируется на этапе валидации.
  - Массив может содержать произвольные комбинации знаковых данных: только положительные числа, только отрицательные
      числа, а также смешанные наборы данных.
  - Алгоритм поразрядной сортировки должен корректно обрабатывать знаковый бит (старший байт типа `int`), чтобы
      отрицательные числа корректно располагались перед положительными.

## 3. Базовый алгоритм

В качестве вычислительного ядра последовательной версии используется поразрядная сортировка (Radix Sort) по байтам.
Внутри каждого разряда (байта) применяется распределяющая сортировка подсчетом (Counting Sort).

### **Пошаговое описание алгоритма:**

1. Выполняется итеративный проход по всем байтам 32-битного числа `int` (всего 4 итерации, от младшего байта к
   старшему).
2. На каждой итерации для текущего байта выделяется массив счетчиков `count` размером 256 элементов.
3. Подсчитывается частота появления каждого значения байта среди элементов исходного массива. При обработке самого
   старшего (знакового) байта применяется битовая маска `^= 0x80` для инверсии знакового бита, что позволяет корректно
   упорядочить отрицательные числа.
4. Формируются префиксные суммы в массиве `count` для определения финальных позиций элементов.
5. Производится устойчивое копирование элементов во временный буфер `result` при чтении исходного массива в обратном
   направлении (с использованием `std::ranges::reverse_view`).
6. Текущий массив перезаписывается отсортированным состоянием из буфера `result`.

### **Сложность алгоритма:**

- **Асимптотика по времени:** $O(K \cdot N)$, где $N$ — количество элементов в массиве, а $K = \text{sizeof(int)} = 4$ —
  фиксированное число байт. Таким образом, временная сложность линейна: $O(N)$.
- **Асимптотика по памяти:** $O(N + B)$, где $N$ — память под вспомогательный буфер `result` на каждой итерации, а
  $B = 256$ — фиксированный массив счетчиков `count`. Итоговая пространственная сложность: $O(N)$.

### **Инварианты и критерий корректности:**

- _Инвариант устойчивости:_ Элементы с одинаковыми значениями текущего байта сохраняют свой относительный порядок.
- _Инвариант поразрядного продвижения:_ После $i$-й итерации ($i \in [1, 4]$) подмассив гарантированно отсортирован по
  младшим $i$ байтам.
- _Критерий корректности:_ Для любого индекса $j \in [1, N-1]$ выполняется условие:
  `output_data[j - 1] <= output_data[j]`.

## 4. Детали реализации

Логика последовательной версии находится в файлах: `seq/include/ops_seq.hpp` и `seq/src/ops_seq.cpp`.

Класс `LevonychevIRadixBatcherSortSEQ` наследуется от компонента `BaseTask` и переопределяет методы жизненного цикла
конвейера:

- `ValidationImpl()`: Проверяет входной вектор на пустоту. Возвращает `false`, если вход пуст, предотвращая дальнейшие
  вычисления.
- `PreProcessingImpl()`: На данном этапе последовательной версии дополнительная модификация структуры данных не
  требуется, метод тривиально возвращает `true`.
- `RunImpl()`: Выделяет выходной вектор, дублируя входной (`GetOutput() = GetInput()`), и последовательно запускает 4
  итерации метода `CountingSort` по каждому байту данных.
- `PostProcessingImpl()`: Тривиально возвращает `true`.

Основной вычислительный фрагмент метода распределения по байтам:

```cpp
void LevonychevIRadixBatcherSortSEQ::CountingSort(InType &arr, size_t byte_index) {
  const size_t byte = 256;
  std::vector<int> count(byte, 0);
  OutType result(arr.size());

  bool is_last_byte = (byte_index == (sizeof(int) - 1ULL));
  for (auto number : arr) {
    int value_of_byte = (number >> (byte_index * 8ULL)) & 0xFF;

    if (is_last_byte) {
      value_of_byte ^= 0x80;
    }

    ++count[value_of_byte];
  }

  for (size_t i = 1ULL; i < byte; ++i) {
    count[i] += count[i - 1];
  }

  for (int &val : std::ranges::reverse_view(arr)) {
    int value_of_byte = (val >> (byte_index * 8ULL)) & 0xFF;

    if (is_last_byte) {
      value_of_byte ^= 0x80;
    }

    result[--count[value_of_byte]] = val;
  }
  arr = result;
}
```

_Пояснение к коду:_ Смещение `byte_index * 8ULL` выделяет нужный байт числа. Условие `is_last_byte` определяет проход по
старшему знаковому байту. Операция `value_of_byte ^= 0x80` смещает диапазон значений байта $[-128, 127]$ в беззнаковый
диапазон $[0, 255]$. Это транслирует отрицательные значения в начало индексов массива `count`, гарантируя корректность
сквозной сортировки знаковых целых чисел.

## 5. Проверка корректности

Корректность последовательной реализации подтверждается набором функциональных тестов, определенных в файле
`tests/functional/main.cpp`. Валидация выходящего массива осуществляется `CheckTestOutputData`, выполняющим
последовательную проверку выполнения инварианта неубывания элементов.

**Функциональные тесты покрывают характерные сценарии и знаковые комбинации:**

1. **Размер $N = 1$ (`one_element`):** `input_data_ = {5}`. Тест проверяет граничное условие минимального размера
   массива.
2. **Только положительные числа (`only_positive`):** `input_data_ = {170, 45, 75, 90, 2, 24, 802, 66}`. Проверяется
   базовая логика устойчивого поразрядного распределения без участия знакового бита.
3. **Только отрицательные числа (`only_negative`):** `input_data_ = {-170, -45, -75, -90, -2, -24, -802, -66}`.
   Проверяется корректность работы маски инверсии знакового бита `^= 0x80`.
4. **Смешанные данные (`mixed`):** `input_data_ = {-170, 45, 75, -90, 2, 24, -802, 66}`. Проверка распределения знаковых
   и беззнаковых диапазонов совместно.

**Характерные примеры входа и ожидаемого выхода на основе тестовых сценариев:**

1. _Только положительные числа (`only_positive`):_
    - Вход: `[170, 45, 75, 90, 2, 24, 802, 66]`
    - Выход: `[2, 24, 45, 66, 75, 90, 170, 802]`
2. _Только отрицательные числа (`only_negative`):_
    - Вход: `[-170, -45, -75, -90, -2, -24, -802, -66]`
    - Выход: `[-802, -170, -90, -75, -66, -45, -24, -2]`
3. _Смешанные данные (`mixed`):_
    - Вход: `[-170, 45, 75, -90, 2, 24, -802, 66]`
    - Выход: `[-802, -170, -90, 2, 24, 45, 66, 75]`

Во всех случаях последовательная версия успешно проходит автоматическую валидацию на соответствие инварианту
упорядоченности.
