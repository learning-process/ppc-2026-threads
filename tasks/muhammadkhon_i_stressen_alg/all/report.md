## 1. Контекст

В версии `ALL` используется гибридная схема `MPI + OMP`. На уровне MPI
между процессами распределяются семь подзадач верхнего уровня алгоритма Штрассена,
а внутри каждого rank-а локальные вычисления выполняются через OpenMP.

## 2. Постановка задачи

Та же постановка, что и в `SEQ`: `C = A * B`, допуск `1e-9`. Дополнительно —
двухуровневое распараллеливание: межпроцессный уровень через MPI,
внутрипроцессный через OpenMP.

## 3. Базовый алгоритм

Алгоритм остаётся тем же: padding, разбиение на квадранты, `M1..M7`, сборка.
`kCutoff = 64`. Рекурсия через `std::function impl`

## 4. Межпроцессная схема

MPI используется на верхнем уровне алгоритма:

- `rank 0` рассылает padded-матрицы через `MPI_Bcast`;
- подзадачи `M1..M7` распределяются по rank-ам по схеме round-robin;
- каждый процесс считает только свои подзадачи и накапливает вклад в локальный буфер;
- частичные результаты собираются через `MPI_Reduce(MPI_SUM)`;
- итоговый результат рассылается через `MPI_Bcast`.

## 5. Внутрипроцессная схема

Внутри каждого MPI-процесса используется OpenMP:

- базовое блочное умножение параллелится через `#pragma omp parallel for`;
- верхний уровень `StrassenOmpLocal(...)` параллелится через `#pragma omp task`.

## 6. Детали реализации

Файлы:

- `all/include/ops_all.hpp`
- `all/src/ops_all.cpp`

Ключевые моменты:

- `ValidationImpl()` проверяет вход только на `rank 0`;
- `PreProcessingImpl()` подготавливает данные на `rank 0`, остальные ждут;
- `RunImpl()` делает `MPI_Bcast`, локальные OMP-вычисления, `MPI_Reduce`, `MPI_Bcast`;
- размеры рассылаются через `std::array<uint64_t, 4>` — без C-массивов.
