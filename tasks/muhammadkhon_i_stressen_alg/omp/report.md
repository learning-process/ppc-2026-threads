## 1. Контекст

После того как `SEQ` стала нормально работать, я перенёс задачу на OpenMP.
Для Штрассена это естественный шаг: семь произведений `M1..M7` независимы
и удобно ложатся на `omp task`.

## 2. Постановка задачи

Та же постановка, что и в `SEQ`: вычислить `C = A * B`.
Корректность оценивается относительно эталонного обычного умножения, допуск `1e-9`.

## 3. Базовый алгоритм

Математическая часть не менялась. `kCutoff = 64`, `kBlockSize = 64`.
Внутренняя последовательная часть оставлена как есть — параллелится только верхний уровень.

## 4. Схема распараллеливания

Граница параллелизации — верхний уровень вычисления `M1..M7`.

Как это устроено:

- создаётся `#pragma omp parallel` область;
- внутри — `#pragma omp single nowait`, чтобы только один поток раздал задачи;
- каждое произведение `M1..M7` запускается отдельной `#pragma omp task`;
- `#pragma omp taskwait` дожидается всех семи перед сборкой квадрантов.

По данным:

- `m1..m7` объявлены как `shared`, каждая задача пишет в свой буфер;
- гонок нет — буферы не делятся между задачами;
- `reduction`, `atomic`, `critical` в горячем участке не понадобились;
- `default(none)` заставил явно перечислять переменные — это помогло
  увидеть потенциальные ошибки на этапе компиляции.

## 5. Детали реализации

Файлы:

- `omp/include/ops_omp.hpp`
