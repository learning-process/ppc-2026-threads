## 2. Постановка задачи

Нужно вычислить произведение двух матриц `A` и `B` размера `n x n`:

```text
C = A * B
```

Вход: `a_rows`, `a_cols_b_rows`, `b_cols`, векторы `a` и `b`.
Выход: вектор `c` размера `a_rows * b_cols`.

Ограничения:

- все размеры больше нуля;
- если максимальная из трёх размерностей не степень двойки, матрицы дополняются нулями;
- критерий корректности: совпадение с эталонным умножением с допуском `1e-9`.

## 3. Базовый алгоритм

Алгоритм Штрассена с порогом `kCutoff = 64`: при `n <= 64` используется
блочное умножение (`kBlockSize = 64`), при `n > 64` — рекурсивное разбиение.

Шаги:

1. Padding до степени двойки.
2. Разбиение на квадранты `A11..A22`, `B11..B22`.
3. Вычисление `M1..M7`.
4. Сборка `C11..C22` из промежуточных произведений.
5. Обрезка результата до исходного размера.

Рекурсия реализована через `std::function impl` внутри обёртки `StrassenSeq`,
что позволило избежать предупреждений clang-tidy о рекурсивных свободных функциях.

Асимптотика: `O(n^2.81)` по времени; памяти больше, чем у наивного умножения,
из-за промежуточных буферов.

## 4. Детали реализации

Файлы:

- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

Пайплайн:

- `ValidationImpl()` — проверяет размеры входных векторов;
- `PreProcessingImpl()` — выделяет и заполняет нулями padded-матрицы;
- `RunImpl()` — запускает `StrassenSeq`, копирует результат;
- `PostProcessingImpl()` — ничего не делает, возвращает `true`.
