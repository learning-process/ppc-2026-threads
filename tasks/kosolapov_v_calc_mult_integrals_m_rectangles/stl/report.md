## 1. Контекст

Реализация на `std::thread` вручную разбивает сетку на блоки и запускает потоки.
Это демонстрирует базовые принципы ручного управления потоками.

## 2. Постановка задачи

Аналогично SEQ.

## 3. Базовый алгоритм

Тот же двойной цикл, но сетка разбивается на непрерывные диапазоны одномерного индекса `idx = i*steps + j`.
Каждый поток обрабатывает свой диапазон и сохраняет локальную сумму; затем главный поток суммирует результаты.

## 4. Схема распараллеливания

- Общее число элементов `total = steps*steps`.
- Число потоков `num_threads` (задаётся через `PPC_NUM_THREADS`).
- Каждый поток получает диапазон `[start, end)`, где размер диапазона ≈ total/num_threads.
- Локальные суммы хранятся в векторе `std::vector<double> local_sums(num_threads)`.
- Потоки запускаются через `std::async` или `std::thread` (в примере использован `std::async`).
- После завершения всех потоков суммы складываются.

## 5. Детали реализации

Файлы: `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`.

Ключевой фрагмент (упрощённо):

    unsigned int num_threads = std::thread::hardware_concurrency();
    size_t total = steps * steps;
    size_t chunk_size = (total + num_threads - 1) / num_threads;
    std::vector<std::future<double>> futures;

    for (unsigned int t = 0; t < num_threads; ++t) {
        size_t start = t * chunk_size;
        size_t end = std::min(start + chunk_size, total);
        futures.push_back(std::async(std::launch::async, [=]() {
            double local = 0.0;
            for (size_t idx = start; idx < end; ++idx) {
                int i = idx / steps;
                int j = idx % steps;
                double x = a + (i+0.5)*hx;
                double y = c + (j+0.5)*hy;
                local += CallFunction(func_id, x, y);
            }
            return local;
        }));
    }
    double sum = 0.0;
    for (auto& f : futures) sum += f.get();

## 6. Проверка корректности

Корректность проверяется сравнением с аналитическими значениями интегралов (погрешность ≤ 0.05 при `steps ≥ 20`):

- Для `func_id=1`: точное значение 2/3 ≈ 0.6667.
- Для `func_id=2`: точное значение 2.
- Для `func_id=3`: приближённое 2.230.
- Для `func_id=4`: точное значение 0.

Функциональные тесты из `tests/functional/main.cpp` прогоняют все реализации с шагами 20 и 40 для каждого `func_id`.
Результаты сравниваются с ожидаемыми значениями через `CheckTestOutputData`.

## 9. Выводы

Ручное управление потоками даёт базовое ускорение, но уступает библиотечным решениям из-за overhead
и неоптимального балансирования. Использовать `std::thread` целесообразно только при специфических
требованиях к управлению потоками.
