## 1. Контекст

Версия на oneTBB использует `parallel_reduce` для суммирования значений по всем ячейкам сетки.
Это позволяет библиотеке автоматически балансировать нагрузку.

## 2. Постановка задачи

Аналогично SEQ.

## 3. Базовый алгоритм

Алгоритм тот же, но для обхода сетки используется одномерный индекс `idx = i*steps + j`,
а для суммирования – `parallel_reduce` с инициализацией 0 и оператором сложения.

## 4. Схема распараллеливания

- Диапазон: `blocked_range<size_t>(0, total)`, где `total = steps*steps`.
- Метод: `parallel_reduce` с функцией тела, вычисляющей локальную сумму.
- `grainsize` не задан явно (используется значение по умолчанию).
- Планировщик TBB сам разбивает диапазон на подзадачи.
- Конкуренция ограничивается через `global_control` (не используется, полагаемся на `PPC_NUM_THREADS`).

## 5. Детали реализации

Файлы: `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`.

Ключевой фрагмент:

    double hx = (b - a) / steps;
    double hy = (d - c) / steps;
    size_t total = static_cast<size_t>(steps) * steps;

    double sum = tbb::parallel_reduce(
        tbb::blocked_range<size_t>(0, total), 0.0,
        [&](const tbb::blocked_range<size_t>& range, double local_sum) {
            for (size_t idx = range.begin(); idx != range.end(); ++idx) {
                int i = static_cast<int>(idx / steps);
                int j = static_cast<int>(idx % steps);
                double x = a + ((i + 0.5) * hx);
                double y = c + ((j + 0.5) * hy);
                local_sum += CallFunction(func_id, x, y);
            }
            return local_sum;
        },
        std::plus<>()
    );
    return sum * hx * hy;

## 6. Проверка корректности

Корректность проверяется сравнением с аналитическими значениями интегралов (погрешность ≤ 0.05 при `steps ≥ 20`):

- Для `func_id=1`: точное значение 2/3 ≈ 0.6667.
- Для `func_id=2`: точное значение 2.
- Для `func_id=3`: приближённое 2.230.
- Для `func_id=4`: точное значение 0.

Функциональные тесты из `tests/functional/main.cpp` прогоняют все реализации с шагами 20 и 40 для каждого `func_id`.
Результаты сравниваются с ожидаемыми значениями через `CheckTestOutputData`.

TBB показывает более низкую эффективность по сравнению с OpenMP из-за overhead на
разбиение диапазонов и копирование лямбд.
Также видны большие наклодные расходы, которые на малом количестве потоков замедляют прогамму.
 Так что данная реализация является гораздо менее эффективной.

## 9. Выводы

TBB – удобная библиотека для задач с динамической нагрузкой, но для регулярной сетки она не превосходит OpenMP.
 Её преимущество проявилось бы на более сложных, нерегулярных разбиениях. В данной реализации TBB малоэффективно.
