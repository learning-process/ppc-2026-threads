## 1. Контекст

Гибридная версия сочетает MPI для межпроцессного распараллеливания и многопоточность (STL) внутри каждого MPI-процесса.
Цель – эффективное использование кластеров с общей памятью в узлах.

## 2. Постановка задачи

Аналогично SEQ, но данные (сетка) распределяются по MPI-процессам по строкам (по i).
Каждый процесс обрабатывает свою группу строк, используя многопоточность для ускорения.

## 3. Базовый алгоритм

1. MPI инициализация.
2. Определение ранга и числа процессов.
3. Распределение строк: каждый процесс получает `rows_per_proc` или `rows_per_proc+1` строк.
4. Внутри процесса – многопоточное вычисление суммы по своим строкам и всем столбцам.
5. Локальная сумма умножается на `hx*hy` и отсылается процессу 0 через `MPI_Reduce`.
6. Процесс 0 выводит результат, `MPI_Bcast` рассылает его всем (для единообразия).

## 4. Межпроцессная схема

- Коммуникатор: `MPI_COMM_WORLD`.
- Распределение строк: целочисленное деление steps на size.
- `MPI_Reduce` с оператором `MPI_SUM` для сбора глобальной суммы.
- `MPI_Bcast` для рассылки результата.
- `MPI_Barrier` для синхронизации (необязательно, но оставлено для надёжности).

## 5. Внутрипроцессная схема

Внутри каждого MPI-процесса используется `ComputeLocalSum`, который может задействовать OpenMP, TBB или `std::thread`
(в примере реализован через `std::async`). В данной работе используется вариант с `std::async` для простоты.

## 6. Детали реализации

Файлы: `all/include/ops_all.hpp`, `all/src/ops_all.cpp`.

Ключевые фрагменты:

    int rank, size;
    MPI_Comm_rank(MPI_COMM_WORLD, &rank);
    MPI_Comm_size(MPI_COMM_WORLD, &size);
    int rows_per_proc = steps / size;
    int remainder = steps % size;
    int start_i = rank * rows_per_proc + std::min(rank, remainder);
    int end_i = start_i + rows_per_proc + (rank < remainder ? 1 : 0);
    double local_sum = ComputeLocalSum(func_id, steps, a, c, hx, hy, start_i, end_i);
    double global_sum = 0.0;
    MPI_Reduce(&local_sum, &global_sum, 1, MPI_DOUBLE, MPI_SUM, 0, MPI_COMM_WORLD);
    MPI_Bcast(&global_sum, 1, MPI_DOUBLE, 0, MPI_COMM_WORLD);
    MPI_Barrier(MPI_COMM_WORLD);
    return global_sum * hx * hy;

Функция `ComputeLocalSum` разбивает свои строки на потоки через `std::async`.

## 7. Проверка корректности

Корректность проверяется сравнением с аналитическими значениями интегралов (погрешность ≤ 0.05 при `steps ≥ 20`):

- Для `func_id=1`: точное значение 2/3 ≈ 0.6667.
- Для `func_id=2`: точное значение 2.
- Для `func_id=3`: приближённое 2.230.
- Для `func_id=4`: точное значение 0.

Функциональные тесты из `tests/functional/main.cpp` прогоняют все реализации с шагами 20 и 40 для каждого `func_id`.
Результаты сравниваются с ожидаемыми значениями через `CheckTestOutputData`.

## 10. Выводы

Гибридная схема полезна для больших задач, не помещающихся в память одного узла.
 На одном узле она не даёт выигрыша по сравнению с чисто потоковыми решениями, но демонстрирует
  работоспособность MPI+threads. Эффективность падает при увеличении числа процессов из-за коммуникаций.
