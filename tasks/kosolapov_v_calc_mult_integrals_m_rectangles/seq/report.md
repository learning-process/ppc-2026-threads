## 1. Контекст

Последовательная версия служит эталоном корректности и базой для сравнения ускорения всех параллельных реализаций
 (OMP, TBB, STL, ALL). Задача заключается в численном интегрировании нескольких функций двух
 переменных методом средних прямоугольников на заданных областях.

## 2. Постановка задачи

**Входные данные**:  

- `steps` – количество разбиений по каждому измерению (целое положительное число);  
- `func_id` – идентификатор интегрируемой функции (1..4).

**Выходные данные**:  

- приближённое значение двойного интеграла `double`.

**Ограничения**:  

- `steps > 0`;  
- `func_id ∈ {1,2,3,4}`;  
- функции определены и непрерывны на своих прямоугольных областях.

**Крайние случаи**:  

- минимальное `steps = 1` (один прямоугольник);  
- большие `steps` (проверка сходимости и производительности).

**Интегрируемые функции и области**:

| func_id | Функция       | Область [xmin,xmax]×[ymin,ymax] | Точное значение (приближённое) |
|---------|---------------|---------------------------------|--------------------------------|
| 1       | x² + y²       | [0,1]×[0,1]                     | 2/3 ≈ 0.6666667                |
| 2       | sin(x)·cos(y) | [0,π]×[0,π/2]                   | 2.0                            |
| 3       | exp(-(x²+y²)) | [-1,1]×[-1,1]                   | ≈ 2.230                        |
| 4       | sin(x+y)      | [0,π]×[0,π]                     | 0.0                            |

## 3. Базовый алгоритм

Метод средних прямоугольников для двойного интеграла:

1. Выбрать количество разбиений `steps` по x и y.
2. Вычислить шаги `hx = (b-a)/steps`, `hy = (d-c)/steps`.
3. Для каждой ячейки `(i,j)` вычислить координаты центра:  
   `x = a + (i+0.5)*hx`, `y = c + (j+0.5)*hy`.
4. Прибавить значение функции в центре к общей сумме.
5. Умножить сумму на `hx * hy`.

**Асимптотика по времени**: O(steps²) – число итераций квадратично зависит от разбиения.  
**Асимптотика по памяти**: O(1) – хранятся только скаляры.  
**Инварианты**: результат сходится к точному интегралу при `steps → ∞`; последовательный расчёт детерминирован.

## 4. Детали реализации

**Файлы**:  

- `seq/include/ops_seq.hpp` – объявление класса `KosolapovVCalcMultIntegralsMRectanglesSEQ`.  
- `seq/src/ops_seq.cpp` – реализация.

**Методы**:

- `ValidationImpl()` – проверяет `steps > 0` и `func_id ∈ [1,4]`.
- `PreProcessingImpl()` – не требует действий (возвращает `true`).
- `RunImpl()` – извлекает входные параметры, получает границы через `GetBounds()`, вызывает `RectanglesIntegral()`,
 сохраняет результат.
- `PostProcessingImpl()` – возвращает `true`.

**Особенности**:  

- Вызов `CallFunction(func_id, x, y)` через switch dispatching.  
- Четыре статические функции `Function1..4` – ядра вычислений.  
- Границы и шаги вычисляются в `RectanglesIntegral()`.

## 5. Проверка корректности

Корректность проверяется сравнением с аналитическими значениями интегралов (погрешность ≤ 0.05 при `steps ≥ 20`):

- Для `func_id=1`: точное значение 2/3 ≈ 0.6667.
- Для `func_id=2`: точное значение 2.
- Для `func_id=3`: приближённое 2.230.
- Для `func_id=4`: точное значение 0.

Функциональные тесты из `tests/functional/main.cpp` прогоняют все реализации с шагами 20 и 40 для каждого `func_id`.
Результаты сравниваются с ожидаемыми значениями через `CheckTestOutputData`.
