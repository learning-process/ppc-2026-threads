## 1. Контекст

OpenMP-версия реализует параллельное вычисление двойного интеграла методом прямоугольников с использованием директив
 `parallel for` и `reduction`. Распараллеливание внешнего цикла по `i` (строки сетки) обеспечивает независимость
  итераций, что даёт хорошую масштабируемость на многоядерных системах.

## 2. Постановка задачи

Входные и выходные данные полностью совпадают с последовательной версией (SEQ).
 Базовый алгоритм – метод средних прямоугольников.
 Отличие только в способе организации вычислений: внешний цикл выполняется параллельно в несколько потоков.

## 3. Базовый алгоритм

Алгоритм идентичен SEQ, но внешний цикл по `i` распараллеливается.
Каждый поток обрабатывает свою долю строк (непрерывный блок итераций).
Промежуточные суммы накапливаются в локальных переменных каждого потока,
а затем редуцируются в общую переменную `result` с помощью оператора `+`.

## 4. Схема распараллеливания

- **Область параллелизации**: цикл `for (int i = 0; i < steps; i++)`.
- **Переменные**:
  - `shared`: `steps`, `a`, `hx`, `c`, `hy`, `func_id` – общие для всех потоков (только чтение).
  - `private`: `i`, `x`, `j`, `y` – каждый поток имеет свою копию.
  - `reduction(+: result)`: локальная копия `result` для каждого потока, в конце все суммируются.
- **Директива**: `#pragma omp parallel for reduction(+: result) default(none) shared(...)`.
- **Расписание**: по умолчанию `schedule(static)` – итерации делятся на непрерывные блоки примерно равного размера.
- **Число потоков**: задаётся переменной окружения `OMP_NUM_THREADS` (или `PPC_NUM_THREADS`).
- **Синхронизация**: неявный барьер в конце параллельной области; редукция выполняется автоматически.

## 5. Детали реализации

**Файлы**:  

- `omp/include/ops_omp.hpp` – объявление класса.  
- `omp/src/ops_omp.cpp` – реализация.

**Ключевой фрагмент кода**:

    double hx = (b - a) / steps;
    double hy = (d - c) / steps;
    double result = 0.0;
    #pragma omp parallel for reduction(+ : result) default(none) shared(steps, a, hx, c, hy, func_id)
    for (int i = 0; i < steps; i++) {
        double x = a + ((i + 0.5) * hx);
        for (int j = 0; j < steps; j++) {
            double y = c + ((j + 0.5) * hy);
            result += CallFunction(func_id, x, y);
        }
    }
    result *= (hx * hy);

**Особенности**:

- Явное указание `default(none)` заставляет перечислить все переменные, что повышает безопасность.
- Редукция переменной `result` исключает гонки данных.
- Вложенный цикл по `j` выполняется последовательно внутри каждого потока.

## 6. Проверка корректности

Корректность проверяется сравнением с аналитическими значениями интегралов (погрешность ≤ 0.05 при `steps ≥ 20`):

- Для `func_id=1`: точное значение 2/3 ≈ 0.6667.
- Для `func_id=2`: точное значение 2.
- Для `func_id=3`: приближённое 2.230.
- Для `func_id=4`: точное значение 0.

Функциональные тесты из `tests/functional/main.cpp` прогоняют все реализации с шагами 20 и 40 для каждого `func_id`.
Результаты сравниваются с ожидаемыми значениями через `CheckTestOutputData`.
