# Поразрадная сортировка элементов типа double с простым слиянием

- Студент: Митяева Дарья Викторовна, группа 3823Б1ФИ2

- Вариант: 19

## 1. Контекст

Данный отчёт описывает реализацию поразрядной сортировки с использованием Intel
Threading Building Blocks (TBB) — библиотеки для task-ориентированного
параллелизма. TBB выбран для автоматического балансировки нагрузки и
эффективного использования многоядерных систем.

## 2. Постановка задачи

- **Входные данные:** Вектор чисел `double` произвольной длины `N`.
- **Выходные данные:** Отсортированный вектор.

## 3. Базовый алгоритм (последовательный)

См. `seq/report.md`. TBB-версия также использует битовое преобразование
`DoubleToSortable` для единообразной обработки всех чисел.

## 4. Схема распараллеливания

**4.1 Какой примитив выбран:** `oneapi::tbb::parallel_for` для всех параллельных
циклов (преобразование, гистограмма, рассеивание). Для фаз гистограммы и
рассеивания используется `static_partitioner`, так как нагрузка равномерна.

**4.2 Диапазон работы:** `oneapi::tbb::blocked_range<size_t>(0, n)`.

**4.3 Grain size:** Не задавался явно; TBB автоматически выбирает размер чанков.
Для равномерной нагрузки это оптимально.

**4.4 Partitioner:** Явно указан `static_partitioner` для фаз подсчёта, чтобы
исключить накладные расходы на динамическое перебалансирование. Для
преобразований используется partitioner по умолчанию (`auto_partitioner`).

**4.5 Контроль конкуренции:** Число потоков задаётся через `PPC_NUM_THREADS`.
TBB автоматически управляет пулом потоков.

**4.6 Трёхфазный подсчёт (аналогично OpenMP):** Локальные гистограммы —
`std::vector<std::vector<size_t>> local_hists(num_threads)`. После
`parallel_for` гистограммы агрегируются последовательно, затем выполняется
`parallel_for` для рассеивания.

## 5. Детали реализации

- **Файлы:** `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`,
  `tbb/src/sorter_tbb.cpp`
- **Отличия от OpenMP:** Использование `static_partitioner`; отсутствие явных
  барьеров (TBB не требует); более эффективное управление thread-local storage.

**Пример ключевого фрагмента (parallel_for с static_partitioner):**

```cpp
// tbb/src/sorter_tbb.cpp
oneapi::tbb::parallel_for(
    oneapi::tbb::blocked_range<size_t>(0, n),
    [&](const oneapi::tbb::blocked_range<size_t>& r) {
        int tid = oneapi::tbb::this_task_arena::current_thread_index();
        std::vector<size_t>& local_hist = local_hists[tid];
        for (size_t i = r.begin(); i != r.end(); ++i) {
            uint64_t val = src[i];
            size_t byte = (val >> shift) & 0xFF;
            local_hist[byte]++;
        }
    },
    oneapi::tbb::static_partitioner()
);
```

## 6. Проверка корректности

- Сравнение с `seq` на всех тестах. Результаты идентичны.
- Проверка с `static_partitioner` и без него — результат не меняется.
- ThreadSanitizer и TBB-санитайзеры не выявили проблем.

## 7. Экспериментальная среда

**Анализ:** TBB обгоняет OpenMP на 25–33% на всех размерах. Ускорение 6.47x на 8
потоках — близко к теоретическому пределу для памяти-связанного алгоритма.

`T_tbb(ms) ≈ 0.0000825 × N + 2.45`, что в 6.1 раза лучше SEQ и на 25% лучше
OpenMP по наклону.

## 9. Выводы

TBB — наилучший выбор для shared-memory параллелизации данной задачи.
`static_partitioner` исключает динамические накладные расходы, а эффективное
управление памятью даёт преимущество перед OpenMP. Рекомендуется для
продакшн-сред, где допустимо использование внешней библиотеки TBB.
