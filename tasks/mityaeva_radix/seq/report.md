## 1. Контекст

Данный отчёт описывает последовательную (базовую) версию поразрядной сортировки
(LSD radix sort) для чисел двойной точности. Эта версия служит эталоном
корректности и производительности для сравнения со всеми параллельными
реализациями (OMP, TBB, STL, ALL). Вычислительное ядро реализует классический
алгоритм поразрядной сортировки, обрабатывая байты IEEE 754-представления числа.

## 2. Постановка задачи

- **Входные данные:** Вектор чисел `double` произвольной длины `N`.
- **Выходные данные:** Вектор, отсортированный в порядке неубывания.
- **Ограничения:** Вектор не пуст. Алгоритм должен корректно разделять и
  сортировать отрицательные и положительные числа из-за особенностей
  представления IEEE 754 (сначала сортируются отрицательные числа в порядке
  убывания, затем неотрицательные в порядке возрастания).

## 3. Базовый алгоритм

1. **Разделение:** Исходный вектор разделяется на два: `negatives`
   (отрицательные числа) и `non_negatives` (ноль и положительные числа).
2. **Сортировка отрицательных чисел:** К вектору `negatives` применяется
   поразрядная сортировка LSD, но байты интерпретируются для сортировки в
   порядке **убывания** (из-за инвертированного представления отрицательных
   чисел в IEEE 754). Используется стабильная сортировка подсчётом для каждого
   байта.
3. **Сортировка неотрицательных чисел:** К вектору `non_negatives` применяется
   стандартная поразрядная сортировка LSD (по возрастанию).
4. **Слияние:** Отсортированный массив `negatives` (уже в правильном порядке)
   копируется в результирующий вектор, затем дописывается отсортированный
   `non_negatives`.

- **Сложность по времени:** O(n), так как выполняется фиксированное число
  проходов (8 байт × 2 группы = 16 проходов сортировки подсчётом). Константа
  зависит от размера алфавита (256).
- **Сложность по памяти:** O(n) для вспомогательного буфера, используемого в
  сортировке подсчётом.
- **Инварианты:** После каждого прохода сортировки подсчётом элементы остаются
  стабильно отсортированными по обработанным младшим байтам. Результат слияния
  `negatives` и `non_negatives` даёт глобально отсортированный массив.

## 4. Детали реализации

- **Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`
- **Валидация (`ValidationImpl`):** Проверяет, что размер входных данных не
  равен нулю.
- **Предобработка (`PreProcessingImpl`):** Копирует входные данные во внутренний
  буфер.
- **Запуск (`RunImpl`):** Вызывает функцию `SorterSeq::LSDSortDouble`.
- **Постобработка (`PostProcessingImpl`):** Копирует отсортированный результат в
  выходной буфер.

**Пример ключевого фрагмента (разделение и слияние):**

```cpp
// seq/src/sorter_seq.cpp
std::vector<double> negatives, non_negatives;
for (double val : data) {
    if (val < 0) negatives.push_back(val);
    else non_negatives.push_back(val);
}
CountingSortDesc(negatives, 0, NEG_SHIFT);
CountingSortAsc(non_negatives, 0, POS_SHIFT);
result.insert(result.end(), negatives.begin(), negatives.end());
result.insert(result.end(), non_negatives.begin(), non_negatives.end());
```

## 5. Проверка корректности

Корректность проверена сравнением результата с эталонной функцией
`std::ranges::is_sorted` для:

- Пустого массива (отсекается валидацией).
- Массивов с одним элементом.
- Массивов со случайными значениями (положительными, отрицательными, нулями).
- Массивов, уже отсортированных в разном порядке.
- Массивов с дублирующимися значениями.

Граничные случаи (положительный/отрицательный ноль, `+inf`, `-inf`) отрабатывают
без ошибок.
