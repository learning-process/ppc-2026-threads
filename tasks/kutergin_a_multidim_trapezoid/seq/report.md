## 1. Контекст

Численное интегрирование многомерных функций является вычислительно трудоемкой задачей, необходимой
в физическом моделировании, квантовой химии и статистике. Данная последовательная реализация
реализует классический многомерный метод трапеций и служит базовым математическим эталоном. Она
необходима для строгого подтверждения корректности вычислительной логики алгоритма и используется в
качестве отправной точки ($T_1$) для расчета ускорения и эффективности в параллельных и гибридных
реализациях (OMP, TBB, STL, ALL).

## 2. Постановка задачи

- **Входные данные:**
  - Список пар границ интегрирования для каждой размерности std::vector<std::pair<double,
    double>> borders.
  - Общее количество шагов разбиения int n (одинаковое для всех осей).
  - Математическая функция $f(x_1, x_2, \dots, x_N)$, заданная через std::function.
- **Выходные данные:** Приближенное значение интеграла (гиперобъема) типа double.
- **Ограничения и крайние случаи**
  - Количество размерностей пространства $d \ge 1$.
  - Число интервалов разбиения по каждой оси $n > 0$.
  - Все границы интегрирования должны быть конечными числами, при этом строго $a_i < b_i$.
  - Значения подынтегральной функции во всех узлах сетки должны принимать конечные значения
    (std::isfinite).

## 3. Базовый алгоритм

Реализована составная многомерная формула метода трапеций. В отличие от одномерного случая, где
граничные узлы имеют вес $0.5$, в $N$-мерном пространстве вес узла определяется как $0.5^k$, где
$k$ — количество координат узла, лежащих на границах гиперкуба интегрирования.

Для обхода многомерной сетки произвольной размерности $N$ без использования глубокой рекурсии
применен итеративный подход с функцией генерации многомерного индекса:

1. Вычисляется шаг интегрирования $h_i$ по каждому измерению и элементарный объем ячейки
$cell\_volume = \prod h_i$.
2. Заводится вектор индексов idx размерности $N$. На каждом шаге координаты узла восстанавливаются
по формуле $x_i = a_i + idx_i \cdot h_i$.
3. Функция NextIndex эмулирует работу многомерного счетчика (аналог системы счисления с основанием
$n+1$), последовательно инкрементируя индексы от младшего измерения к старшему.

- Асимптотика по времени: $O((n+1)^N \cdot N)$, где $n$ — число шагов, $N$ — размерность
 пространства.
- Асимптотика по памяти: $O(N)$ для хранения текущих индексов, шагов и координат расчетного узла.

## 4. Детали реализации

Файлы: seq/include/ops_seq.hpp, seq/src/ops_seq.cpp

- **ValidationImpl():** Проверяет валидность указателя на функцию, положительность шагов n,
непустоту вектора borders, а также вызывает вспомогательную функцию ValidateBorders() для проверки
конечности границ и условия $a_i < b_i$.
- **PreProcessingImpl():** Обнуляет выходной буфер перед началом вычислений.
- **RunImpl():** Содержит основной цикл генерации узлов. Рассчитывает веса weight для граничных
точек, вычисляет значения функций, проверяет их на std::isfinite и аккумулирует взвешенную сумму. В
конце масштабирует результат на cell_volume.
- **PostProcessingImpl():** Финализирует операцию, проверяя полученный результат на корректность
(std::isfinite).

Листинг ключевого фрагмента (RunImpl):

```cpp
const auto &[func, borders, n] = GetInput();
const int dim = static_cast<int>(borders.size());

std::vector<double> h(dim);
double cell_volume = 1.0;

for (int i = 0; i < dim; ++i) {
  const double left = borders[i].first;
  const double right = borders[i].second;
  h[i] = (right - left) / n;
  cell_volume *= h[i];
}

const int max_index = n;
std::vector<int> idx(dim, 0);
std::vector<double> point(dim);
double sum = 0.0;

while (true) {
  double weight = 1.0;

  for (int i = 0; i < dim; ++i) {
    point[i] = (borders[i].first + (idx[i] * h[i]));

    if ((idx[i] == 0) || (idx[i] == n)) {
      weight *= 0.5;
    }
  }

  double f_val = func(point);
  if (!std::isfinite(f_val)) {
    return false;
  }

  sum += weight * f_val;

  if (!NextIndex(idx, dim, max_index)) {
    break;
  }
}

GetOutput() = sum * cell_volume;

```

## 5. Проверка корректности

Валидация последовательного кода производилась с помощью разработанных юнит-тестов на базе
фреймворка Google Test. Численные результаты сравнивались с точными аналитическими значениями
интегралов, вычисленных по формуле Ньютона-Лейбница.

**Методика тестирования:**

- Замеры близости значений выполнялись с абсолютной погрешностью $\epsilon = 10^{-3}$ (заданный
kTolerance в тестах).
- Тестовая матрица включает пространства от 1D до 3D с разной плотностью разбиения (число шагов
варьируется от 60 до 250).

**Характерные сценарии тестирования:**

- 1D Константа (Const_1D_unit): Интегрирование $f(x)=1$ на интервале $[0, 1]$, 120 шагов.
Проверка базовой разметки весов. Ожидаемый результат: 1.0.
- 2D Сумма (Linear_2D_sum): Интегрирование $f(x, y) = x + y$ на области $[0, 1] \times [0, 1]$,
100 шагов. Тестирование корректности вычисления многомерных весов на ребрах и углах. Ожидаемый
результат: 1.0.
- 1D Тригонометрия (Sin_0_pi): Интегрирование функции $f(x) = \sin(x)$ на интервале $[0, \pi]$,
250 шагов. Проверка сходимости на нелинейной гладкой функции. Ожидаемый результат: 2.0.
- 2D Экспонента (Exp_2D): Интегрирование $f(x, y) = e^{x+y}$ на гиперкубе $[0, 1]^2$, 200
шагов. Проверка накопления погрешности на быстрорастущих функциях. Ожидаемый результат:
$\approx 2.9512$.

**Значение этой версии как эталона:**
    - Чистая производительность: Данный код лишен накладных расходов на создание потоков (OpenMP/
    TBB), копирование памяти между узлами (MPI) или внутреннюю синхронизацию примитивов. Время
    выполнения этой версии представляет собой идеальный базис $T_1$.
    - Критерий отсутствия Race Condition: Математические результаты SEQ-версии принимаются за
    абсолютно верные. Любые расхождения результатов в последующих реализациях (OMP, TBB, STL, ALL)
    будут однозначно свидетельствовать об ошибках синхронизации памяти или некорректном разделении
    диапазонов данных между потоками/процессами.
