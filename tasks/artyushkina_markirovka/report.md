## 1. Введение

Задача маркировки связных компонент (Connected Component Labeling, CCL) является одной из фундаментальных задач компьютерного зрения и обработки изображений. Она заключается в идентификации всех связных областей (объектов) на бинарном изображении и присвоении каждой области уникальной метки.

В данной работе представлены реализации алгоритма CCL с использованием 8-связности для пяти технологий: последовательная (SEQ), OpenMP (OMP), Intel TBB (TBB), стандартная библиотека потоков C++ (STL) и MPI-совместимая (ALL). Цель — сравнить производительность и корректность различных подходов к распараллеливанию.

## 2. Постановка задачи

**Входные данные**: вектор `std::vector<uint8_t>`, где:
- `[0]` — количество строк `rows` (высота изображения)
- `[1]` — количество столбцов `cols` (ширина изображения)
- `[2..]` — значения пикселей: `0` (объект, чёрный) или `255` (фон, белый)

**Выходные данные**: вектор `std::vector<uint8_t>`, где:
- `[0]` — `rows`
- `[1]` — `cols`
- `[2..]` — метки компонент (`0` — фон, `1..N` — уникальные метки объектов)

**Ограничения**: `rows >= 1`, `cols >= 1`. Используется **8-связность** — пиксели считаются связными, если они соприкасаются по стороне или углу.

**Критерий корректности**: побайтовое совпадение выходных данных параллельной версии с результатом SEQ версии.

## 3. Базовый алгоритм (последовательный)

SEQ реализация использует обход в ширину (BFS). Для каждого непомеченного пикселя-объекта запускается BFS, который рекурсивно помечает всю связную компоненту уникальным номером.

```
для каждого пикселя (i, j):
    если пиксель является объектом и не помечен:
        current_label++
        очередь.push((i, j))
        метки[i][j] = current_label
        
        пока очередь не пуста:
            (x, y) = очередь.pop()
            для каждого из 8 соседей (dx, dy):
                если сосед является объектом и не помечен:
                    метки[x+dx][y+dy] = current_label
                    очередь.push((x+dx, y+dy))
```

**Временная сложность**: O(rows * cols). **Затраты памяти**: O(rows * cols).

## 4. Схема распараллеливания

### 4.1 Общая схема для OMP, STL, TBB

Параллельные реализации используют двухпроходный алгоритм с системой непересекающихся множеств (DSU).

**Первый проход (маркировка)**: Каждому пикселю-объекту присваивается временная метка. Для каждого пикселя анализируются 8 соседей. Если соседние пиксели имеют разные метки, они объединяются через операцию Union.

**Второй проход (разрешение эквивалентностей)**: Временные метки заменяются на корневые метки из DSU, затем выполняется перенумерация от 1 до N.

### 4.2 Специфика TBB

Использует `tbb::parallel_for` для параллельной обработки строк и `tbb::spin_mutex` для защиты операций Union. Этапы обработки: инициализация, объединение по горизонтали, по вертикали, по диагонали, разрешение корней, перенумерация.

### 4.3 Специфика ALL

Реализация предназначена для MPI. Использует 4-связность (только соседи сверху, слева и по диагоналям вверх).

void MarkingComponentsTBB::MergeHorizontalPairsTbb() {
  tbb::parallel_for(0, rows_, [this](int y_coord) {
    for (int x_coord = 0; x_coord < cols_ - 1; ++x_coord) {
      int idx = (y_coord * cols_) + x_coord;
      if (labels_[idx] != 0 && labels_[idx + 1] != 0) {
        UnionLabels(labels_[idx], labels_[idx + 1]);
      }
    }
  });
}
