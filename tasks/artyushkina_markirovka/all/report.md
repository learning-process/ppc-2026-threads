## 1. Контекст

Исходная задача — маркировка связных компонент на бинарном изображении, где чёрные
пиксели (значение 0) соответствуют объектам, а белые (значение 255) — фону.

ALL-реализация предназначена для MPI-тестов и использует двухпроходный алгоритм с
системой непересекающихся множеств (DSU). В отличие от других реализаций, ALL
использует 4-связность (только соседи сверху, слева и по диагоналям вверх) вместо
8-связности.

## 2. Постановка задачи

**Входные данные**: вектор `std::vector<uint8_t>`, где:

- `[0]` — количество строк `rows`
- `[1]` — количество столбцов `cols`
- `[2..]` — значения пикселей: `0` (объект) или `255` (фон)

**Выходные данные**: вектор `std::vector<uint8_t>`, где:

- `[0]` — `rows`
- `[1]` — `cols`
- `[2..]` — метки компонент (`0` — фон, `1..N` — уникальные метки объектов)

## 3. Базовый алгоритм

ALL-реализация использует классический двухпроходный алгоритм маркировки.

**Первый проход (FirstPass):** Каждому пикселю-объекту присваивается временная метка.
Анализируются 4 соседа: верхний-левый, верхний, верхний-правый, левый. Если найдены
соседи с метками, выбирается минимальная метка, и различные метки объединяются.

**Второй проход (SecondPass):** Временные метки заменяются на корневые из DSU,
затем выполняется перенумерация от 1 до N.

## 4. Детали реализации

**Файлы реализации:**

- `all/include/ops_all.hpp` — объявление класса `MarkingComponentsALL`
- `all/src/ops_all.cpp` — реализация двухпроходного алгоритма

**Ключевые методы:**

- `FindRoot(std::vector<int> &parent, int label)` — поиск корня (статический)
- `UnionLabels(std::vector<int> &parent, int label1, int label2)` — объединение
- `FirstPass()` — первый проход алгоритма
- `SecondPass()` — второй проход алгоритма

**Соседи для первого прохода (4-связность):**

```cpp
std::vector<NeighborOffsetAll> GetFirstPassNeighbors() {
    std::vector<NeighborOffsetAll> neighbors(4);
    neighbors[0] = {.di = -1, .dj = -1,
        .check_i_min = 1, .check_i_max = 0,
        .check_j_min = 1, .check_j_max = 0};  // верхний-левый
    neighbors[1] = {.di = -1, .dj = 0,
        .check_i_min = 1, .check_i_max = 0,
        .check_j_min = 0, .check_j_max = 0};  // верхний
    neighbors[2] = {.di = -1, .dj = 1,
        .check_i_min = 1, .check_i_max = 0,
        .check_j_min = 0, .check_j_max = 1};  // верхний-правый
    neighbors[3] = {.di = 0, .dj = -1,
        .check_i_min = 0, .check_i_max = 0,
        .check_j_min = 1, .check_j_max = 0};  // левый
    return neighbors;
}
```

**FindRoot с сжатием пути:**

```cpp
int MarkingComponentsALL::FindRoot(std::vector<int> &parent, int label) {
    int root = label;
    while (parent[static_cast<size_t>(root)] != root) {
        root = parent[static_cast<size_t>(root)];
    }
    int current = label;
    while (current != root) {
        int next = parent[static_cast<size_t>(current)];
        parent[static_cast<size_t>(current)] = root;
        current = next;
    }
    return root;
}
```

**UnionLabels:**

```cpp
void MarkingComponentsALL::UnionLabels(std::vector<int> &parent,
    int label1, int label2) {
    int root1 = FindRoot(parent, label1);
    int root2 = FindRoot(parent, label2);
    if (root1 != root2) {
        if (root1 < root2) {
            parent[static_cast<size_t>(root2)] = root1;
        } else {
            parent[static_cast<size_t>(root1)] = root2;
        }
    }
}
```

## 8. Выводы

ALL-реализация корректно решает задачу маркировки компонент с использованием
4-связности. Статические методы `FindRoot` и `UnionLabels` делают код удобным.

**Основные отличия от других версий:**

- Использует 4-связность вместо 8
- Предназначена для MPI (требует mpirun)
- Классический двухпроходный алгоритм
