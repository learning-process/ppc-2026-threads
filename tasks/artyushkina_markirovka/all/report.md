## 1. Контекст

Исходная задача — маркировка связных компонент на бинарном изображении, где чёрные пиксели (значение 0) соответствуют объектам, а белые (значение 255) — фону.

ALL-реализация предназначена для MPI-тестов и использует двухпроходный алгоритм с системой непересекающихся множеств (DSU). В отличие от других реализаций, ALL использует 4-связность (только соседи сверху, слева и по диагоналям вверх) вместо 8-связности.

## 2. Постановка задачи

**Входные данные**: вектор `std::vector<uint8_t>`, где:
- `[0]` — количество строк `rows`
- `[1]` — количество столбцов `cols`
- `[2..]` — значения пикселей: `0` (объект) или `255` (фон)

**Выходные данные**: вектор `std::vector<uint8_t>`, где:
- `[0]` — `rows`
- `[1]` — `cols`
- `[2..]` — метки компонент (`0` — фон, `1..N` — уникальные метки объектов)

## 3. Базовый алгоритм

ALL-реализация использует классический двухпроходный алгоритм маркировки.

**Первый проход (FirstPass):** Каждому пикселю-объекту присваивается временная метка. Анализируются 4 соседа: верхний-левый, верхний, верхний-правый, левый. Если найдены соседи с метками, выбирается минимальная метка, и различные метки объединяются через Union.

**Второй проход (SecondPass):** Временные метки заменяются на корневые из DSU, затем выполняется перенумерация от 1 до N.

## 4. Детали реализации

**Файлы реализации:**
- `all/include/ops_all.hpp` — объявление класса `MarkingComponentsALL`
- `all/src/ops_all.cpp` — реализация двухпроходного алгоритма

**Ключевые методы:**
- `FindRoot(std::vector<int> &parent, int label)` — поиск корня с сжатием пути (статический)
- `UnionLabels(std::vector<int> &parent, int label1, int label2)` — объединение множеств (статический)
- `FirstPass()` — первый проход алгоритма
- `SecondPass()` — второй проход алгоритма

**Соседи для первого прохода (4-связность):**
```cpp
std::vector<NeighborOffsetAll> GetFirstPassNeighbors() {
  std::vector<NeighborOffsetAll> neighbors(4);
  neighbors[0] = {.di = -1, .dj = -1, ...};  // верхний-левый
  neighbors[1] = {.di = -1, .dj = 0, ...};   // верхний
  neighbors[2] = {.di = -1, .dj = 1, ...};   // верхний-правый
  neighbors[3] = {.di = 0, .dj = -1, ...};   // левый
  return neighbors;
}
```

**Первый проход:**
```cpp
void MarkingComponentsALL::FirstPass() {
  int next_label = 1;
  auto neighbors = GetFirstPassNeighbors();
  
  for (int i = 0; i < rows_; ++i) {
    for (int j = 0; j < cols_; ++j) {
      if (input[idx] == 255) continue;
      
      int min_label = next_label;
      bool has_neighbors = false;
      FindMinLabelFromNeighbors(i, j, rows_, cols_, labels_, neighbors, min_label, has_neighbors);
      
      if (!has_neighbors) {
        AssignNewLabel(i, j, next_label, labels_, equivalent_labels_);
      } else {
        AssignExistingLabel(i, j, min_label, labels_);
        UnionNeighborLabels(i, j, rows_, cols_, labels_, equivalent_labels_, neighbors, min_label);
      }
    }
  }
}
```

**Второй проход:**
```cpp
void MarkingComponentsALL::SecondPass() {
  // замена меток на корневые
  for (int i = 0; i < rows_; ++i) {
    for (int j = 0; j < cols_; ++j) {
      if (labels_[i][j] != 0) {
        labels_[i][j] = FindRoot(equivalent_labels_, labels_[i][j]);
      }
    }
  }
  
  // перенумерация от 1 до N
  std::vector<int> remap(label_count, 0);
  int current_label = 1;
  for (int i = 1; i < label_count; ++i) {
    if (equivalent_labels_[i] == i) {
      remap[i] = current_label++;
    }
  }
  
  for (int i = 0; i < rows_; ++i) {
    for (int j = 0; j < cols_; ++j) {
      if (labels_[i][j] != 0) {
        labels_[i][j] = remap[labels_[i][j]];
      }
    }
  }
}
```

**Важное замечание:** ALL-реализация использует 4-связность, в то время как все остальные версии используют 8-связность. Это означает, что результаты ALL могут отличаться на изображениях, где объекты соединены только по диагонали. Для теста 6 (диагональная связность) 4-связность даёт две отдельные компоненты, что и ожидается в выходных данных.

## 8. Выводы

ALL-реализация корректно решает задачу маркировки компонент с использованием 4-связности. Статические методы `FindRoot` и `UnionLabels` делают код удобным для использования. Реализация предназначена для MPI-тестов, но в текущей конфигурации запускается с одним процессом.

**Основные отличия от других версий:**
1. Использует 4-связность вместо 8
2. Предназначена для MPI (требует mpirun)
3. Классический двухпроходный алгоритм без параллелизма

**Рекомендации:**
- Расширить до 8-связности для единообразия с другими версиями
- Добавить параллельную обработку через MPI при работе с несколькими процессами
- Рассмотреть возможность внутрипроцессного распараллеливания через STL-потоки
