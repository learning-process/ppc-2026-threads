## 1. Контекст
Алгоритм Штрассена — это метод быстрого умножения квадратных матриц, который имеет асимптотическую сложность O(n^log₂7) ≈ O(n^2.807) вместо O(n³) для стандартного алгоритма. Алгоритм применяется для **плотных матриц** (dense matrices), где большинство элементов ненулевые. Последовательная версия необходима как эталонная (baseline) реализация для измерения эффективности параллельных версий (OMP, STL, TBB) и для проверки корректности результатов.

## 2. Постановка задачи

**Входные данные:**
- Две квадратные матрицы A и B размером n×n
- Матрицы представлены как одномерные векторы `std::vector<double>` размером n²
- Размер матрицы n (целое положительное число)

**Границы допустимости для InType:**
- 1 ≤ n ≤ INT_MAX/2 (ограничение на переполнение при вычислении n²)
- a.size() == n² и b.size() == n² (строгое соответствие размеров)
- Элементы матриц: произвольные конечные double значения
- Матрицы представлены в row-major порядке: A[i,j] = a[i*n + j]

**Выходные данные:**
- Матрица C = A × B размером n×n (вектор размером n²)

**Ограничения и инварианты:**
- n > 0
- Размеры векторов A и B должны быть равны n²
- Алгоритм оптимален для плотных матриц с высокой заполненностью

**Крайние случаи:**
- Матрицы 1×1
- Размеры, не являющиеся степенями двойки (требуется паддинг)
- Малые матрицы (где наивное умножение эффективнее)

## 3. Базовый алгоритм

**Пошаговое описание:**
1. **Валидация:** Проверить n > 0 и соответствие размеров векторов
2. **Паддинг:** Дополнить матрицы нулями до ближайшей степени двойки (padded_n)
3. **Рекурсивное разбиение:** Разделить каждую матрицу на 4 подматрицы размером n/2 × n/2
4. **Вычисление 7 произведений Штрассена:**
   - M1 = (A11 + A22) × (B11 + B22)
   - M2 = (A21 + A22) × B11
   - M3 = A11 × (B12 - B22)
   - M4 = A22 × (B21 - B11)
   - M5 = (A11 + A12) × B22
   - M6 = (A21 - A11) × (B11 + B12)
   - M7 = (A12 - A22) × (B21 + B22)
5. **Сборка результата:**
   - C11 = M1 + M4 - M5 + M7
   - C12 = M3 + M5
   - C21 = M2 + M4
   - C22 = M1 + M3 - M2 + M6
6. **Главная структура данных:**
Итеративный стек узлов рекурсии для предотвращения stack overflow:
```cpp
struct StrassenNode {
    std::vector<double> a, b;           // Подматрицы текущего уровня
    int n;                              // Размер текущих подматриц
    int result_slot;                    // Индекс слота для результата
    bool expanded;                      // Флаг развертывания узла
    std::array<int, 7> child_slots;     // Слоты для 7 произведений M1-M7
};
```
7. **Базовый случай:** Для n ≤ 64 используется наивное умножение O(n³)
8. **Unpadding:** Извлечение результирующей матрицы исходного размера

**Реализация базового алгоритма:**
```cpp
std::vector<double> LazarevaATestTaskSEQ::Strassen(const std::vector<double>& a, 
                                                   const std::vector<double>& b, int n) {
    if (n <= 64) {
        return NaiveMult(a, b, n);  // Базовый случай O(n³)
    }
    
    // Разбиение на 4 подматрицы n/2 × n/2
    std::vector<double> a11, a12, a21, a22, b11, b12, b21, b22;
    Split(a, n, a11, a12, a21, a22);
    Split(b, n, b11, b12, b21, b22);
    
    // 7 произведений Штрассена
    auto m1 = Strassen(Add(a11, a22), Add(b11, b22), n/2);  // (A11+A22)(B11+B22)
    auto m2 = Strassen(Add(a21, a22), b11, n/2);            // (A21+A22)B11
    auto m3 = Strassen(a11, Sub(b12, b22), n/2);            // A11(B12-B22)
    auto m4 = Strassen(a22, Sub(b21, b11), n/2);            // A22(B21-B11)
    auto m5 = Strassen(Add(a11, a12), b22, n/2);            // (A11+A12)B22
    auto m6 = Strassen(Sub(a21, a11), Add(b11, b12), n/2);  // (A21-A11)(B11+B12)
    auto m7 = Strassen(Sub(a12, a22), Add(b21, b22), n/2);  // (A12-A22)(B21+B22)
    
    // Сборка результата
    auto c11 = Add(Sub(Add(m1, m4), m5), m7);  // M1 + M4 - M5 + M7
    auto c12 = Add(m3, m5);                    // M3 + M5
    auto c21 = Add(m2, m4);                    // M2 + M4  
    auto c22 = Add(Sub(Add(m1, m3), m2), m6);  // M1 + M3 - M2 + M6
    
    return Merge(c11, c12, c21, c22, n/2);
}
```

**Асимптотика по времени:** O(n^log₂7) ≈ O(n^2.807)

**Асимптотика по памяти:** O(n²) для хранения промежуточных результатов

**Инварианты и критерий корректности:**
- На каждом уровне рекурсии размер матрицы является степенью двойки
- Результат совпадает с классическим умножением матриц с точностью ε = 1e-5
- Итеративная реализация предотвращает переполнение стека вызовов
- Детерминированность: одинаковые входы всегда дают одинаковые выходы


## 4. Детали реализации

**Файлы:** 
- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

**ValidationImpl:**
- Проверяет n > 0 через условие `if (n <= 0) return false`
- Проверяет соответствие размеров через `std::cmp_equal(GetInput().a.size(), expected)`
- Вычисляет ожидаемый размер как `static_cast<size_t>(n) * static_cast<size_t>(n)`
- Использует `std::cmp_equal` для безопасного сравнения знаковых/беззнаковых типов

**PreProcessingImpl:**
- Сохраняет исходный размер `n_` и вычисляет `padded_n_` через `NextPowerOfTwo(n_)`
- Вызывает `PadMatrix()` для дополнения входных матриц нулями до размера `padded_n_ × padded_n_`
- Инициализирует результирующий вектор `result_` размером `padded_n²` нулями
- Функция `NextPowerOfTwo()` находит минимальную степень двойки ≥ n через битовые сдвиги

**RunImpl:**
- Вызывает итеративную реализацию `Strassen()` с падированными матрицами
- Использует явный стек (`call_stack`) и структуру `StrassenNode` для избежания рекурсии
- Каждый узел содержит: матрицы a, b, размер n, индекс слота результата, флаг expansion
- Для n ≤ 64 переключается на `NaiveMult()` (ikj-порядок для оптимизации кэша)

**PostProcessingImpl:**
- Вызывает `UnpadMatrix()` для извлечения результата размером `n × n` из падированного
- Копирует элементы из левого верхнего угла падированной матрицы
- Записывает финальный результат в `GetOutput()`

**Обработанные исключительные и граничные случаи:**
- Размеры, не являющиеся степенями двойки (автоматический паддинг через `PadMatrix`)
- Малые матрицы (threshold = 64 для переключения на наивное умножение)
- Предотвращение переполнения стека через итеративную реализацию с explicit stack
- Безопасное приведение типов через `static_cast<ptrdiff_t>` и `static_cast<size_t>`
- Корректная обработка индексации при split/merge операциях

**Комментарий о самом дорогом фрагменте:**
Наиболее затратными операциями являются вычисление 7 произведений M1-M7 на верхних уровнях рекурсии, которые требуют множественных аллокаций промежуточных матриц размером n/2 × n/2. Дополнительные накладные расходы создают 18 операций сложения/вычитания подматриц на каждом уровне рекурсии (каждая O(n²)) и Split/Merge операции с плохой локальностью памяти. При n ≤ 64 наивное умножение ikj-порядка работает быстрее благодаря лучшей cache locality и отсутствию overhead на управление рекурсией и временными объектами.

## 8. Выводы
Данная последовательная версия используется как эталон для ускорения по следующим причинам:

**Корректность и надежность:** Реализация точно следует алгоритму Штрассена с корректной обработкой всех граничных случаев включая паддинг произвольных размеров до степеней двойки, автоматическое переключение на наивное умножение для малых подматриц, и безопасную работу с индексами через явные приведения типов.

**Оптимальная настройка порога:** Threshold = 64 элемента обеспечивает оптимальный баланс между асимптотическим преимуществом алгоритма Штрассена O(n^2.807) и практической эффективностью наивного алгоритма O(n³) на малых подзадачах с хорошей локальностью кэша.

**Архитектурная стабильность:** Итеративная реализация с явным стеком call_stack и структурой StrassenNode предотвращает stack overflow при больших матрицах, обеспечивает предсказуемое потребление памяти O(n²) и детерминированный порядок выполнения операций.

**Базовая линия для параллелизма:** Позволяет измерить реальное ускорение (speedup) параллельных версий OMP/STL/TBB относительно одного потока, выявить bottlenecks в алгоритме и оценить эффективность распараллеливания различных фаз (split, compute, merge).

**Асимптотическое преимущество:** Для больших n > 256 алгоритм демонстрирует существенное превосходство над O(n³), что делает его привлекательной целью для параллелизации с потенциалом значительного общего ускорения.

**Детерминированность и воспроизводимость:** Результаты строго детерминированы и совпадают с эталонным наивным умножением в пределах машинной точности (ε = 1e-5), что критично для валидации параллельных версий и обеспечения корректности в различных вычислительных средах.

**Область применения:** Реализация оптимизирована для плотных матриц с высокой заполненностью. Для разреженных матриц использование полного хранения приводит к неоправданным затратам памяти O(n²) и избыточным вычислениям над нулевыми элементами.

Данная реализация служит reference implementation для всех параллельных технологий и гарантирует корректность через строгую валидацию входных данных и результатов.
