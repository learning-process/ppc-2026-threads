## 1. Контекст

Задача повышения контраста полутоновых изображений решает проблему плохой различимости деталей
из-за узкого диапазона яркости пикселей.
Линейная растяжка гистограммы — простейший метод улучшения,
который отображает исходный диапазон значений [min, max] на полный доступный диапазон [0, 255].
Последовательная версия (SEQ) служит эталоном корректности и базой для измерения ускорения параллельных реализаций.

## 2. Постановка задачи

### Входные данные

- Вектор `uint8_t` произвольной длины \( N \) — пиксели полутонового изображения (значения от 0 до 255).

### Выходные данные

- Вектор `uint8_t` той же длины — обработанное изображение после линейной растяжки.

### Ограничения

- Размер изображения \( N >= 1 \).
- Значения пикселей — целые беззнаковые 8 бит.

### Крайние случаи

| Случай                           | Вход            | Выход           |
| -------------------------------- | --------------- | --------------- |
| Все пиксели равны                | `[128,128,128]` | `[128,128,128]` |
| Минимальный диапазон (разница 1) | `[100,101]`     | `[0,255]`       |
| Максимальный диапазон            | `[0,255]`       | `[0,255]`       |
| Один пиксель                     | `[42]`          | `[42]`          |
| Все нули                         | `[0,0,0]`       | `[0,0,0]`       |
| Все 255                          | `[255,255,255]` | `[255,255,255]` |

## 3. Базовый алгоритм

### Пошаговое описание

1. Найти минимальное (`min_pixel`) и максимальное (`max_pixel`) значение яркости среди всех пикселей входного изображения.
2. Если `min_pixel == max_pixel` (изображение однородное), вернуть копию входных данных (растяжка невозможна).
3. Вычислить коэффициент масштабирования:

```cpp
float scale = 255.0F / static_cast<float>(max_pixel - min_pixel);
```

4. Для каждого пикселя `x_i` вычислить новый:

```cpp
output[i] = static_cast<uint8_t>(std::round(static_cast<float>(input[i] - min_pixel) * scale));
```

5. Вернуть результирующий вектор.

### Асимптотика

- **Время:** O(N) — один линейный проход для поиска min/max + один проход для преобразования.
- **Память:** O(N) — выходной вектор того же размера, что и входной.

### Инварианты и критерий корректности

- Инвариант: сохраняется порядок пикселей (монотонное преобразование).
- Критерий: выходной диапазон всегда [0, 255], если только вход не был константным.

## 4. Детали реализации

### Файлы

- `seq/include/ops_seq.hpp` — объявление класса `PylaevaSIncContrastImgByLshSEQ`.
- `seq/src/ops_seq.cpp` — реализация методов.

### Методы задачи

| Метод                  | Назначение                                                                                                   |
| ---------------------- | ------------------------------------------------------------------------------------------------------------ |
| `ValidationImpl()`     | Проверяет, что входные данные не пусты (`!GetInput().empty()`). Возвращает `false` при пустом векторе.       |
| `PreProcessingImpl()`  | Выделяет память под выходной вектор того же размера, что и входной.                                          |
| `RunImpl()`            | Основной алгоритм: поиск min/max, проверка на константность, вычисление scale, преобразование с округлением. |
| `PostProcessingImpl()` | Ничего не делает (возвращает `true`).                                                                        |

### Обработка особых случаев

- Пустой вход → `ValidationImpl()` вернёт `false`, задача не выполняется.
- Константное изображение → `RunImpl()` копирует вход в выход без преобразования.
- Округление выполняется через `std::round` для минимизации ошибок.
