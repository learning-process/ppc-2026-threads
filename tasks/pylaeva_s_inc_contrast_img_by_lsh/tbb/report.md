## 2. Постановка задачи

Полная постановка задачи приведена в отчёте для SEQ-версии [seq/report.md](../seq/report.md).  
Кратко:

- **Вход:** вектор `uint8_t` длины N (пиксели полутонового изображения).
- **Выход:** вектор `uint8_t` той же длины после линейного растяжения диапазона [min, max] → [0, 255].

OpenMP-реализация должна давать **идентичный** SEQ результат при любом количестве потоков.

## 3. Базовый алгоритм

Последовательная версия (SEQ):

1. Найти min и max среди всех пикселей (один линейный проход).
2. Если min == max → вернуть копию входных данных.
3. Вычислить `scale = 255.0 / (max - min)`.
4. Для каждого пикселя: `out[i] = round((input[i] - min) * scale)`.
5. Вернуть выходной вектор.

Сложность: O(N) времени, O(N) памяти.

## 4. Схема распараллеливания

```cpp
  tbb::parallel_for(tbb::blocked_range<size_t>(0, size), [&](const auto &r) {
    for (auto i = r.begin(); i != r.end(); ++i) {
      output[i] = static_cast<uint8_t>(std::round((input[i] - min_pixel) * scale));
    }
  }, tbb::static_partitioner());
```

### Выбранные примитивы

- **Для поиска min/max:** `tbb::parallel_reduce` — параллельная редукция с накоплением промежуточных результатов.
- **Для преобразования пикселей:** `tbb::parallel_for` — независимое применение операции к каждому элементу.
- **Для копирования при константном изображении:** `tbb::parallel_for` с `simple_partitioner`.

### Диапазон работы

Используется `tbb::blocked_range<size_t>(start, end, grainsize)`. Диапазон представляет собой полуинтервал [begin, end). TBB автоматически рекурсивно разбивает его на более мелкие поддиапазоны до достижения размера grainsize.

### Размер grainsize

- **parallel_reduce (поиск min/max):** grainsize не указан явно. Используется значение по умолчанию, которое TBB подбирает эмпирически, обычно около 1–8 итераций на задачу. Это обеспечивает хороший баланс между параллелизмом и накладными расходами на создание задач.
- **parallel_for (преобразование):** также используется grainsize по умолчанию. Для массива из 10 миллионов элементов TBB создаст сотни или тысячи задач, что обеспечивает эффективную балансировку нагрузки.
- **parallel_for (копирование при константном изображении):** используется `tbb::simple_partitioner()`, который разбивает диапазон на максимально мелкие задачи (grainsize ≈ 1). Это оправдано, так как операция копирования очень лёгкая, и требуется максимальное распараллеливание.

### partitioner

| Примитив                        | partitioner                       | Обоснование                                                                                                                      |
| ------------------------------- | --------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| `parallel_reduce`               | `auto_partitioner` (по умолчанию) | Адаптивно подстраивается под загрузку системы                                                                                    |
| `parallel_for` (преобразование) | `static_partitioner`              | Диапазон известен, итерации одинаковой сложности. static_partitioner минимизирует накладные расходы на динамическое планирование |
| `parallel_for` (копирование)    | `simple_partitioner`              | Максимальное измельчение для равномерной загрузки при очень лёгких операциях                                                     |

## 5. Детали реализации

### Файлы

- `tbb/include/ops_tbb.hpp` — заголовок класса `PylaevaSIncContrastImgByLshTBB`.
- `tbb/src/ops_tbb.cpp` — реализация с использованием TBB.

### Локальные аккумуляторы и объединение результатов

Для `parallel_reduce` определена структура `MinMax` как `std::pair<uint8_t, uint8_t>`. Локальный аккумулятор `init` инициализируется первым элементом массива. Тело редукции для каждого поддиапазона обновляет локальные min и max. Функция объединения (combine) принимает два результата и возвращает пару из минимального и максимального значений.

### Работа с памятью

Кэш-зависимости не являются проблемой, так как доступ к массиву линейный, а TBB автоматически распределяет поддиапазоны так, чтобы соседние индексы с высокой вероятностью обрабатывались одним потоком. Это минимизирует ложное разделение кэша (false sharing). Для данной задачи не требуется использования `cache_aligned_allocator` или `scalable_allocator`, так как каждый поток работает с непрерывными блоками памяти, и конфликтов кэш-линий не возникает.

### Особые случаи

- Пустой вход → `ValidationImpl()` возвращает `false`.
- Константное изображение (min == max) → выполняется `parallel_for` с `simple_partitioner` для параллельного копирования входных данных в выходной массив.
- Диапазон начинается с индекса 1 в `parallel_reduce`, так как элемент с индексом 0 используется для инициализации.
