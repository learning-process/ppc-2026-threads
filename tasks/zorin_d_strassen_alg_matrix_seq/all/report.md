## 1. Контекст

В версии `ALL` используется гибридная схема `MPI + OMP`.
На уровне MPI между процессами распределяются независимые подзадачи
верхнего уровня алгоритма Штрассена, а внутри каждого rank-а локальная
часть вычисления выполняется через OpenMP.

## 2. Постановка задачи

Нужно вычислить произведение двух квадратных матриц `double` алгоритмом
Штрассена. Базовая математическая логика совпадает с `SEQ`, но в `ALL`
добавляется двухуровневое распараллеливание:

- межпроцессный уровень через MPI;
- внутрипроцессный уровень через OpenMP.

Подробный общий контекст дан в
[report.md](/C:/Users/N1lr/ppc-2026-threads/tasks/zorin_d_strassen_alg_matrix_seq/report.md).

## 3. Базовый алгоритм

Алгоритм остаётся тем же, что и в остальных версиях:

- если размер матрицы не является степенью двойки, выполняется padding нулями;
- матрицы разбиваются на четыре квадранта;
- вычисляются семь произведений `M1..M7`;
- из них собираются блоки `C11`, `C12`, `C21`, `C22`;
- после вычисления padding удаляется.

Для малых размеров используется блочное умножение.
В этой версии выбран порог `kCutoff = 128`.

## 4. Межпроцессная схема

MPI используется на верхнем уровне алгоритма.

Что делает код:

- `rank 0` получает исходный вход и рассылает размер матрицы всем процессам;
- затем через `MPI_Bcast` всем rank-ам отправляются подготовленные матрицы
  `A` и `B` после padding;
- верхние подзадачи `M1..M7` распределяются по rank-ам по схеме
  `task_id = rank, rank + size, ...`;
- каждый процесс считает только свои подзадачи;
- частичные вклады в итоговую матрицу собираются через `MPI_Reduce`;
- готовый результат из `rank 0` рассылается через `MPI_Bcast`.

## 5. Внутрипроцессная схема

Внутри каждого MPI-процесса используется OpenMP.

OpenMP применяется в двух местах:

- в блочном умножении малых подматриц;
- в локальной рекурсивной части `StrassenOmpLocal(...)`, где ветви
  распараллеливаются через `omp task`.

То есть внутри одного rank-а работа строится так:

- процесс получает одну или несколько подзадач `M1..M7`;
- для каждой такой подзадачи локально запускается OMP-ветка;
- после этого локальный вклад добавляется в соответствующие квадранты
  результирующей матрицы.

## 6. Детали реализации

Файлы:

- `tasks/zorin_d_strassen_alg_matrix_seq/all/include/ops_all.hpp`
- `tasks/zorin_d_strassen_alg_matrix_seq/all/src/ops_all.cpp`

Ключевые моменты реализации:

- `ValidationImpl()` проверяет вход на `rank 0`;
- остальные rank-ы получают данные после MPI-рассылки;
- `PreProcessingImpl()` подготавливает выходной буфер;
- `RunImpl()` делает padding, MPI-рассылку, локальные OMP-вычисления,
  `MPI_Reduce` и финальный `MPI_Bcast`.

Такое устройство делает код устойчивым для запуска в нескольких процессах:
корневой процесс получает исходные данные, а остальные rank-ы работают уже
с подготовленными матрицами.
