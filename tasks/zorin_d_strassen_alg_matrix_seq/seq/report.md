## 2. Постановка задачи

Нужно вычислить произведение двух квадратных матриц `A` и `B`
размера `n x n`:

```text
C = A * B
```

Вход:

- размер `n`;
- матрица `A` из `n * n` элементов типа `double`;
- матрица `B` из `n * n` элементов типа `double`.

Выход:

- матрица `C` того же размера.

Ограничения и крайние случаи:

- `n > 0`;
- обе входные матрицы должны иметь ровно `n * n` элементов;
- если `n` не является степенью двойки, матрицы дополняются нулями
  до ближайшей степени двойки;
- после вычисления результат обрезается обратно до исходного размера.

## 3. Базовый алгоритм

За основу взят алгоритм Штрассена. На каждом шаге матрицы делятся
на четыре блока, затем вместо восьми обычных блочных произведений
считаются семь промежуточных матриц `M1..M7`, из которых потом
собирается результат.

В этой реализации я оставил два важных практических решения.

1. Алгоритм реализован не через глубокую рекурсию функций,
   а через собственный стек `Frame`. Так проще контролировать память
   и состояние вычислений.
2. Для маленьких блоков используется обычное блочное умножение.
   В коде это видно по `kCutoff = 64`. Без этого на малых размерах
   версия получалась только медленнее из-за лишних промежуточных буферов.

Асимптотика:

- по времени: лучше классического `O(n^3)`,
  теоретически около `O(n^2.81)`;
- по памяти: больше, чем у обычного умножения, потому что создаются
  временные буферы для промежуточных блоков.

Критерий корректности здесь простой: результат должен совпадать
с эталонным обычным умножением матриц с точностью `1e-9`.

## 4. Детали реализации

Файлы:

- `tasks/zorin_d_strassen_alg_matrix_seq/seq/include/ops_seq.hpp`
- `tasks/zorin_d_strassen_alg_matrix_seq/seq/src/ops_seq.cpp`

Что делает пайплайн задачи:

- `ValidationImpl()` проверяет, что `n > 0`, размеры входных векторов
  корректны и выходной буфер ещё пуст;
- `PreProcessingImpl()` заранее выделяет память под результат;
- `RunImpl()` делает padding до ближайшей степени двойки,
  запускает `StrassenIter(...)` и копирует результат обратно;
- `PostProcessingImpl()` в этой версии ничего дополнительно не делает.

По структуре данных:

- матрицы хранятся в `std::vector<double>`;
- индекс элемента считается как `row * n + col`;
- внутри итеративного Штрассена используется структура `Frame`,
  где лежат текущие подматрицы, промежуточные `M1..M7`
  и стадия вычисления.

Сначала я делал вариант с более прямолинейным копированием подматриц,
но он быстро начал раздуваться по памяти. Поэтому рабочая версия
получилась уже через аккуратное разбиение и повторное использование
буферов.
