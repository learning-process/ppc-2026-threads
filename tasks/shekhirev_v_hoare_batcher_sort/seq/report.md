## 1. Введение

Последовательная версия алгоритма (SEQ) является базовой. Она необходима для проверки корректности
математической логики и служит эталоном производительности. Именно относительно времени выполнения SEQ-версии
рассчитывается ускорение (speedup) и эффективность будущих параллельных реализаций. На данном этапе
фиксируется алгоритм без накладных расходов на создание и синхронизацию потоков.

## 2. Постановка задачи

- Вход: Неотсортированный массив `std::vector<int>`.
- Выход: Тот же массив, отсортированный по неубыванию.
- Ограничения: Алгоритм слияния Бэтчера исторически требует, чтобы размер массива был строго степенью двойки.
  Для работы с произвольными размерами требуется механизм выравнивания данных.

## 3. Базовый алгоритм

Для решения задачи используется гибридный подход:

1. **Дополнение (Padding):** Размер исходного массива увеличивается до ближайшей степени двойки ($2^k$). Новые
   (фиктивные) элементы заполняются максимальным значением `INT_MAX`.
2. **Сортировка Хоара (QuickSort):** Расширенный массив делится ровно пополам. Каждая половина сортируется
   независимо алгоритмом Хоара.
3. **Слияние Бэтчера:** Две отсортированные половины сливаются в один единый отсортированный массив с помощью
   четно-нечетного слияния Бэтчера.
4. **Усечение:** Поскольку фиктивные элементы `INT_MAX` при сортировке гарантированно оказываются в конце
   массива, на финальном этапе массив просто усекается до исходного размера.

Особенность реализации — отказ от рекурсии в пользу итеративных проходов с явной эмуляцией стека
(использование `std::vector` в качестве стека состояний). Это исключает риск переполнения аппаратного стека
(Stack Overflow) на больших объемах данных.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.

- `ValidationImpl`: Всегда возвращает `true`, так как алгоритм безопасен для любых входных данных, включая
  пустые массивы.
- `PreProcessingImpl`: Вычисляет ближайшую степень двойки `p2` для текущего размера массива. Входные данные
  копируются в выходной буфер, который затем расширяется до `p2` с заполнением недостающих ячеек значением
  `INT_MAX`.
- `RunImpl`: Содержит основную логику. Выполняет ранний возврат для массивов из 0 или 1 элемента. Вызывает
  `HoareSort` для левой и правой половин, затем объединяет их через `BatcherMerge`.
- `PostProcessingImpl`: Возвращает массиву его исходный (дополненный) размер с помощью `resize()`, отсекая
  хвост из `INT_MAX`.

Ключевой фрагмент итеративной сортировки Хоара:

```cpp
// File: seq/src/ops_seq.cpp
void ShekhirevHoareBatcherSortSEQ::HoareSort(std::vector<int> &arr, int start_left, int start_right) {
  std::vector<std::pair<int, int>> stk;
  stk.reserve(32);
  stk.emplace_back(start_left, start_right);

  while (!stk.empty()) {
    auto [left, right] = stk.back();
    stk.pop_back();

    if (left >= right) continue;

    int partition_index = HoarePartition(arr, left, right);
    if (partition_index < right) {
      stk.emplace_back(partition_index, right);
    }
    if (left < partition_index - 1) {
      stk.emplace_back(left, partition_index - 1);
    }
  }
}
```

Итеративный подход в `BatcherMerge` также реализован через стек состояний (`std::tuple<int, int, int, int>`),
что позволяет избежать разрастания рекурсивного дерева вызовов.

## 5. Проверка корректности

Проверка осуществляется с помощью стандартной функции `std::ranges::is_sorted` внутри тестового каркаса (файл
`tests/functional/main.cpp`).

Функциональные тесты покрывают массивы случайных чисел в диапазоне `[-1000, 1000]`. Проверяются следующие
размеры (`size`):

- `0, 1` — граничные вырожденные случаи.
- `8, 128, 256` — идеальные случаи (степени двойки, не требующие padding-а).
- `13, 200, 500` — произвольные размеры (проверка логики дополнения до степени двойки и последующего
  усечения).

Все тесты успешно пройдены, что подтверждает корректность работы алгоритма, правильную работу со смещениями
индексов и логику обработки фиктивных `INT_MAX` элементов.

## 8. Выводы

В ходе работы была реализована надежная последовательная версия гибридной сортировки. Переход от рекурсии к
итеративному подходу с явным использованием `std::vector` в качестве стека позволил сделать алгоритм
устойчивым к переполнению памяти на больших объемах данных.

Искусственное дополнение размера массива до степени двойки (посредством `INT_MAX`) оказалось простым и
элегантным решением, которое позволяет использовать классическую сеть Бэтчера без усложнения логики
ветвлениями для нечетных размерностей. На этапе пост-обработки эти лишние элементы отбрасываются за $O(1)$.

Алгоритм имеет четкое разделение данных: две независимые половины сортируются в `HoareSort`, а слияние в
`BatcherMerge` оперирует предсказуемыми шагами. Это говорит о том, что выбранный подход обладает высоким
потенциалом для эффективного распараллеливания на последующих этапах лабораторной работы.
