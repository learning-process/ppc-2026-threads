## 1. Контекст

Вычисление многомерных интегралов на регулярных сетках высокой плотности требует огромных вычислительных мощностей,
зачастую превосходящих ресурсы одного физического сервера. Гибридная технология (ALL), совмещающая MPI и OpenMP,
позволяет объединить ресурсы множества вычислительных узлов (процессов с изолированной памятью) и одновременно
задействовать многопоточность внутри каждого узла (общая память), полностью загружая аппаратный потенциал современных
суперкомпьютеров.

---

## 2. Постановка задачи

Вычислительное ядро производит расчет кратного интеграла в рамках многопроцессорной/многопоточной конфигурации:

* **Макро-декомпозиция (MPI):** Отрезок интегрирования по первому измерению (`limits[0]`) делится на крупные сегменты
между `num_procs` процессами.
* **Микро-декомпозиция (OpenMP):** Локальный сегмент процесса разделяется на `num_omp_threads` частей для параллельного
обхода внутри ядер CPU.
* **Глобальный сбор:** Редукция финального результата со всех распределенных процессов осуществляется на корневом
процессе (`rank == 0`).

---

## 3. Базовый алгоритм

Алгоритм использует двухступенчатую схему геометрического декомпозирования данных по **внешней оси интегрирования**:

1. **Распределение MPI:** Каждый MPI-процесс определяет свой номер (`rank`) и общее число процессов (`num_procs`). На
основе этого вычисляются глобальные границы подзадачи `mpi_left` и `mpi_right`.
2. **Локальное выравнивание:** Границы процессов сглаживаются функцией `CustomRound` для предотвращения дублирования или
 пропуска граничных точек расчетной сетки.
3. **Параллелизм OpenMP:** Внутри локального отрезка процесса порождается пул OpenMP-потоков. Поток `tid` рассчитывает
свои микро-границы `t_left` и `t_right`, которые повторно выравниваются по сетке.
4. **Вычисление и двухэтапная редукция:**

* *Этап 1:* Потоки одного процесса суммируют локальные вклады в `local_res` через директиву `#pragma omp
reduction(+:local_res)`.
* *Этап 2:* Все MPI-процессы передают свои локальные суммы коллективной операции `MPI_Reduce`, складывающей их в
переменную `global_res` на процессе 0.

## 4. Детали реализации

* **Файлы:** `all/include/ops_all.hpp`, `all/src/ops_all.cpp`

Ключевой фрагмент гибридного вычислительного ядра в методе `RunImpl()`:

```cpp
// Шаг 1: Определение конфигурации распределенной памяти
int rank = 0, num_procs = 1;
MPI_Comm_rank(MPI_COMM_WORLD, &rank);
MPI_Comm_size(MPI_COMM_WORLD, &num_procs);

// Шаг 2: Расчет макро-границ для текущего MPI-процесса
double total_range = limits[0].second - limits[0].first;
double mpi_delta = total_range / num_procs;
double mpi_left = limits[0].first + rank * mpi_delta;
double mpi_right = limits[0].first + (rank + 1) * mpi_delta;

if (rank == 0) mpi_left = limits[0].first;
if (rank == num_procs - 1) mpi_right = limits[0].second;

mpi_left = CustomRound(mpi_left, actual_step[0]);
mpi_right = CustomRound(mpi_right, actual_step[0]);

// Шаг 3: Внутрипроцессное распараллеливание средствами OpenMP
double local_res = 0.0;
double omp_delta = (mpi_right - mpi_left) / num_omp_threads;

#pragma omp parallel num_threads(num_omp_threads) default(none) \
    shared(limits, simpson_factor, actual_step, omp_delta, mpi_left, mpi_right, function, num_omp_threads) \
    reduction(+ : local_res)
{
  int tid = omp_get_thread_num();
  double t_left = mpi_left + tid * omp_delta;
  double t_right = mpi_left + (tid + 1) * omp_delta;

  if (tid == 0) t_left = mpi_left;
  if (tid == num_omp_threads - 1) t_right = mpi_right;

  t_left = CustomRound(t_left, actual_step[0]);
  t_right = CustomRound(t_right, actual_step[0]);

  if (t_right > t_left) {
    local_res += OuntNtIntegral(t_left, t_right, simpson_factor, limits, actual_step, function);
  }
}

// Шаг 4: Межпроцессная глобальная синхронизация
double global_res = 0.0;
MPI_Reduce(&local_res, &global_res, 1, MPI_DOUBLE, MPI_SUM, 0, MPI_COMM_WORLD);

if (rank == 0) {
  I_res_ = global_res;
}

```

### Разбор механизмов взаимодействия

* `MPI_Reduce`: Коллективная блокирующая операция. Выполняет межпроцессную редукцию (суммирование `MPI_SUM`) значений
типа `MPI_DOUBLE`. Процессы отправляют локальные `local_res`, а процесс `0` собирает итоговую сумму. На остальных рангах
 `I_res_` остается равным нулю.
* `default(none) shared(...)`: Гарантирует изоляцию контекста. Локальные переменные потоков (`tid`, `t_left`,
`t_right`) создаются внутри стека параллельной области OpenMP, исключая интерференцию между потоками внутри одного
процесса.

## 5. Проверка корректности

Корректность гибридного решения верифицируется через тесты `Google Test`, запускаемые в MPI-окружении (например, через
`mpiexec -n <P>`):

* Проверяется строгое математическое совпадение результатов с последовательной версией при различных комбинациях
конфигураций `ranks × threads` (например, 2 процесса по 4 потока, 4 процесса по 3 потока и т.д.).
* Подтверждается корректность работы сквозной фильтрации `t_right > t_left` для исключения дегенеративных (пустых)
интервалов.
