## 1. Контекст

Вычисление многомерных интегралов является одной из самых ресурсоемких задач численного анализа. Рост размерности
приводит к экспоненциальному увеличению количества узлов интегрирования (так называемое «проклятие размерности»).
Последовательная версия алгоритма (SEQ) обеспечивает корректное вычисление интеграла по детерминированной сетке и
выступает в качестве эталона (baseline) для оценки корректности работы и измерения ускорения будущих параллельных
версий.

---

## 2. Постановка задачи

* **Входные данные (`InType`):**
* `double h`: базовый шаг интегрирования.
* `std::vector<std::pair<double, double>> limits`: границы интегрирования для каждого измерения (размерность интеграла
определяется как `limits.size()`).
* `std::function<double(const std::vector<double>&)> function`: подынтегральная функция.

* **Выходные данные (`OutType`):**
* `double I_res`: вычисленное значение многомерного интеграла.

* **Ограничения и крайние случаи:**
* Вектор границ не должен быть пустым.
* Объект функции должен быть валидным.
* Шаг `h` должен удовлетворять условию $h \le 0.01$ для обеспечения базовой точности.

---

## 3. Базовый алгоритм

Алгоритм реализует составной метод Симпсона (метод парабол) для каждого измерения. Для исключения накладных расходов на
глубокую рекурсию при высоких размерностях, обход узлов сетки переведен на итеративную схему с явным использованием
структуры данных «стек».

* **Коррекция сетки:** Для каждого измерения $i$ шаг $h$ адаптируется под актуальный размер отрезка так, чтобы число
элементарных интервалов (`quan_steps`) было строго четным. Формируется итоговый множитель метода Симпсона:

$$\text{simpson\_factor} = \prod_{i=0}^{d-1} \frac{\text{actual\_step}_i}{3}$$

* **Весовые коэффициенты:** На каждом шаге по оси узел получает вес `dim_weight` равный `1.0` (для границ), `4.0` (для
нечетных индексов внутри отрезка) или `2.0` (для четных индексов). Итоговый вес точки равен произведению её весов по
всем осям.
* **Асимптотика по времени:** $\mathcal{O}(\prod_{i=0}^{d-1} K_i)$, где $K_i$ — число шагов по $i$-му измерению. В
худшем случае при одинаковом числе шагов $K$ асимптотика составляет $\mathcal{O}(K^d)$.
* **Асимптотика по памяти:** $\mathcal{O}(d \cdot K)$ в пике из-за хранения промежуточных состояний векторов координат
в объектах `std::stack`.

## 4. Детали реализации

* **Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

В классе `VinyaikinaEMultidimIntegrSimpsonSEQ` переопределены четыре ключевых метода жизненного цикла задачи:

* `ValidationImpl()`: Проверяет непустоту вектора лимитов, валидность указателя на функцию и ограничение на
максимальный размер шага ($h \le 0.01$).
* `PreProcessingImpl()`: Обнуляет аккумулятор результата `I_res` перед вычислением.
* `RunImpl()`: Вычисляет скорректированные четные шаги для каждого измерения, считает `simpson_factor` и запускает
итеративную функцию `count()`.
* `PostProcessingImpl()`: Переносит сохраненный результат `I_res` в структуру вывода `GetOutput()`.

Основной вычислительный узел (`count`) реализован следующим образом:

```cpp
// File: seq/src/ops_seq.cpp
double count(double simpson_factor, std::vector<std::pair<double, double>> &limits, std::vector<double>& actual_step, 
std::function<double(const std::vector<double> &)> function) {
  std::stack<std::pair<std::vector<double>, double>> stack; 
  stack.emplace(std::vector<double>(), 1.0);
  
  double i_res = 0.0;
  while (!stack.empty()) {
    std::vector<double> point = stack.top().first; 
    double weight = stack.top().second;
    stack.pop();

    if (point.size() == limits.size()) {
      i_res += function(point) * weight * simpson_factor;
      continue;
    }

    int dim = point.size();
    double step = actual_step[dim] / 1.0;
    int steps_count = static_cast<int>((limits[dim].second - limits[dim].first) / step);
    
    for (int i = 0; i <= steps_count; ++i) {
        double x = limits[dim].first + i * step;
        double dim_weight = (i == 0 || i == steps_count) ? 1.0 : ((i % 2 != 0) ? 4.0 : 2.0);

        point.push_back(x);
        stack.emplace(point, weight * dim_weight); 
        point.pop_back();
    }
  }
  return i_res;
}

```

## 5. Проверка корректности

Корректность алгоритма верифицируется в `tests/functional/main.cpp` с помощью фреймворка `Google Test`. Набор тестов
включает:

1. Интегрирование константных и линейных функций на малых размерностях ($d=1, d=2$).
2. Вычисление кратных интегралов для тригонометрических функций ($\sin(x)\cos(y)$).
3. Сравнение полученного значения с аналитически вычисленным значением первообразной с учетом погрешности метода
Симпсона.
