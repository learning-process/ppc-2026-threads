## 1. Контекст

Технология Intel oneTBB реализует параллелизм на основе задач (Task-based parallelism). В отличие от жесткого связывания
 итераций с потоками операционной системы, TBB использует планировщик с кражей задач (Work-stealing scheduler). Это
 позволяет эффективно балансировать нагрузку на многоядерных и многопоточных CPU, таких как Intel Xeon, предотвращая
 простои ядер при неоднородной плотности вычислений.

---

## 2. Постановка задачи

Вычислительное ядро решает задачу численного интегрирования функции заданной размерности. Архитектура параллельной
структуры адаптирована под TBB:

* **Интервал разбиения:** Вектор логических задач распределяется по диапазону `tbb::blocked_range<int>(0,
num_threads)`.
* **Сборка данных:** Операция редукции промежуточных сумм от каждого выделенного TBB-воркера возложена на
функциональный объект `std::plus<double>()`.

---

## 3. Базовый алгоритм

Для обеспечения высокой производительности и минимизации накладных расходов на частые аллокации динамической памяти
(внутренние векторы и стеки `std::stack`) применено **крупноблочное параллельное декомпозирование**:

1. Весь интервал по первому измерению (`limits[0]`) делится на `num_threads` виртуальных участков шириной `delta`.
2. Создается одномерный диапазон `tbb::blocked_range<int>` от `0` до `num_threads` со стандартным размером блока (
`grainsize = 1`), что указывает TBB распределять эти крупные задачи между потоками пула.
3. Внутри тела редукции для текущего индекса подзадачи `i` рассчитываются локальные границы `left_border` и
`right_border` с прецизионным округлением `CustomRound` для стыковки узлов расчетной сетки.
4. Воркep аккумулирует значения в `local_res`, а TBB автоматически сливает их по завершении параллельного участка.

## 4. Детали реализации

* **Файлы:** `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`

Синтаксис параллельной редукции в `RunImpl()` использует перегрузку `tbb::parallel_reduce` с лямбда-выражениями,
захватывающими контекст задачи по ссылке:

```cpp
tbb::blocked_range<int> range(0, num_threads);

I_res_ = tbb::parallel_reduce(range,
  0.0, // Инициализация аккумулятора
  [&](const tbb::blocked_range<int> &r, double local_res) {
    for (int i = r.begin(); i != r.end(); ++i) {
      double left_border = limits[0].first;
      double right_border = limits[0].second;

      // Логика изоляции потоков по границам первого измерения
      if (i != 0) {
        left_border = CustomRound(limits[0].first + (delta * i), actual_step[0]);
      }
      if (i != num_threads - 1) {
        right_border = CustomRound(limits[0].second - (delta * (num_threads - i - 1)), actual_step[0]);
      }

      // Вызов вычислительного DFS ядра
      local_res += OuntNtIntegral(left_border, right_border, simpson_factor, limits, actual_step, function);
    }
    return local_res;
  },
  std::plus<double>() // Слияние результатов (Combine)
);

```

### Разбор механизмов oneTBB

* `tbb::blocked_range<int>`: Автоматически управляет разбиением итерационного пространства. Поскольку количество
подзадач равно числу потоков, TBB эффективно распределяет их по физическим ядрам.
* **Лямбда-тело (Reduce):** Принимает текущий поддиапазон `r` и локальный накопитель `local_res`. Это потокобезопасная
область, так как каждый поток оперирует своей копией `local_res`.
* `std::plus<double>()`: Стандартный редуктор. Когда два поддиапазона завершают работу, TBB вызывает этот оператор для
сложения их результатов в иерархическом порядке (дерево редукции).

## 5. Проверка корректности

Тестирование параллельного кода осуществляется через фреймворк `Google Test`:

* Проверяется математическая эквивалентность результатов последовательной, OpenMP и TBB версий программы на одинаковых
подынтегральных функциях.
* Изучается поведение алгоритма при граничных значениях параметров, когда `num_threads` превышает реальное число ядер
(проверка стабильности планировщика задач TBB).

Применение Intel oneTBB позволило безопасно реализовать многопоточное вычисление кратных интегралов. Использование

TBB.
