## 1. Контекст
Последовательная версия служит эталонной реализацией, на основе которой строится
распараллеливание. Она позволяет:
1. Отработать базовую логику сортировки Шелла и слияния.
2. Получить baseline производительности для расчёта ускорения (`S`) и
   эффективности (`E`) параллельных версий.

## 2. Постановка задачи
- **Вход:** вектор целых чисел типа `int` размера $N$;
- **Выход:** упорядоченный по возрастанию вектор целых чисел размера $N$;
- **Критерий корректности:** выходной вектор $result$ считается отсортированным
  корректно, если  
  $$i < j \implies result_i \leq result_j \quad \forall i,j.$$

## 3. Базовый алгоритм
Алгоритм состоит из трёх этапов:
1. **Разделение:** исходный массив делится на две половины (левую и правую).
2. **Сортировка Шелла:** каждая половина сортируется по отдельности с
   использованием последовательной сортировки Шелла (убывающие промежутки,
   начиная с `gap = size/2`).
3. **Слияние:** две отсортированные половины объединяются в один отсортированный
   массив стандартным двухиндексным слиянием.

**Асимптотика:**  
- Время: в среднем $O(N \log^2 N)$ для сортировки Шелла (зависит от выбора
  промежутков) плюс $O(N)$ на слияние.  
- Память: $O(N)$ – требуется дополнительный вектор для хранения результата
  слияния (в данной реализации слияние производится в новый вектор).

## 4. Детали реализации
- **Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.
- **Основные методы:**
  - `ShellSort`: сортировка Шелла с начальным промежутком `size/2` и последующим
    делением на 2.
  - `MergeSortedVectors`: классическое двухпутевое слияние в новый вектор.
  - `RunImpl`: делит входные данные на две половины, вызывает `ShellSort` для
    каждой, затем выполняет слияние.
- **Особенности:** не используется дополнительная буферизация, результат слияния
  сохраняется в `output_data_`.

## 5. Проверка корректности
Корректность проверена на 8 функциональных тестах (см. `func_tests.cpp`),
включающих:
- уже отсортированный массив;
- обратный порядок;
- один элемент;
- все одинаковые элементы;
- случайный набор с дубликатами;
- два элемента;
- отрицательные и положительные числа;
- смешанные данные. Результат сравнивался с эталонной сортировкой `std::sort`.
