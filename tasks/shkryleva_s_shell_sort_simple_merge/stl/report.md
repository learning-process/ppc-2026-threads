## 1. Контекст
Реализация на нативных потоках C++ (std::thread) позволяет получить максимальный
контроль над созданием и синхронизацией потоков. Это полезно для сравнения
производительности с более высокоуровневыми подходами (OpenMP, TBB).

## 2. Постановка задачи
Сортировка массива `int` размера $10^6$. Базовая линия – SEQ (250 мс).

## 3. Базовый алгоритм
Алгоритм аналогичен OMP: разбиение на блоки, параллельная сортировка Шеллом
каждого блока, затем иерархическое слияние (параллельное на каждом уровне).

## 4. Схема распараллеливания
- **Декомпозиция:** массив делится на `num_threads` блоков.
- **Параллельная сортировка:** создаются потоки `std::thread`, каждый выполняет
  `ShellSort` на своём блоке. После запуска всех потоков вызывается `join()`.
- **Иерархическое слияние:** в цикле, пока число блоков больше 1, создаются
  потоки для слияния соседних пар блоков. Каждый поток сливает свою пару во
  временный буфер. Размер блока удваивается на каждой итерации.
- **Синхронизация:** явные вызовы `join()` после каждого уровня слияния.

## 5. Детали реализации
- **Файлы:** `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`.
- **Функции:** `ShellSort`, `Merge`, `SortSegments`, `HierarchicalMerge`.
- **Особенности:** буфер для слияния создаётся внутри каждого потока.
  Параллельное слияние позволяет эффективно использовать все ядра на каждом
  этапе.

## 6. Проверка корректности
Тесты из `func_tests.cpp` (8 наборов) пройдены. Результаты идентичны SEQ.

## 7. Экспериментальная среда
