# Подробное описание STL-реализации

## 1. Назначение STL-версии

STL-реализация предназначена для параллельного умножения разреженных матриц с
элементами типа `double`, представленных в формате CCS. В отличие от версий на
OpenMP и oneTBB, здесь потоки создаются вручную с помощью `std::thread`.

Вычисляется произведение `C = A * B`. Входные матрицы и результат используют
массивы `values`, `row_indices` и `col_ptrs`.

## 2. Формат CCS

CCS хранит ненулевые элементы по столбцам. Для столбца `j` диапазон элементов
задается границами:

```text
col_ptrs[j] .. col_ptrs[j + 1]
```

Массив `values` содержит ненулевые значения, а `row_indices` хранит номера
строк этих значений. Такой формат согласуется с алгоритмом, который строит
результирующую матрицу по столбцам.

## 3. Проверка входных данных

Перед вычислением проверяется корректность CCS-представления обеих матриц:

- размер `col_ptrs` должен быть равен `cols + 1`;
- массивы `values` и `row_indices` должны иметь одинаковую длину;
- первый элемент `col_ptrs` должен быть равен нулю;
- последний элемент `col_ptrs` должен совпадать с числом ненулевых элементов;
- значения в `col_ptrs` должны быть неубывающими;
- индексы строк должны находиться в диапазоне `[0, rows)`.

Также проверяется условие `A.cols == B.rows`, без которого произведение матриц
не определено.

## 4. Базовый алгоритм

Для каждого столбца матрицы `B` строится соответствующий столбец результата.
Временный плотный аккумулятор длины `A.rows` хранит значения текущего столбца
до записи в CCS.

Если в столбце `j` матрицы `B` найден ненулевой элемент `B(k, j)`, то столбец
`k` матрицы `A` добавляет вклад:

```text
C(i, j) += A(i, k) * B(k, j)
```

После обработки столбца ненулевые значения аккумулятора переносятся в
`values` и `row_indices`, а массив `col_ptrs` фиксирует границы столбцов.

## 5. Схема распараллеливания через std::thread

Работа делится по непрерывным диапазонам столбцов результата. Число диапазонов
выбирается как минимум из значения `PPC_NUM_THREADS` и числа столбцов матрицы
`B`. Каждый диапазон содержит независимый набор столбцов.

Для каждого диапазона создается отдельный объект `std::thread`. Поток получает
номер диапазона и вычисляет только свои столбцы. Результат записывается в
локальную частичную матрицу CCS, соответствующую этому диапазону.

## 6. Запуск и join потоков

Все потоки сначала создаются в цикле и добавляются в контейнер потоков. После
этого отдельным циклом вызывается `join` для каждого потока. Такой порядок
важен: если вызывать `join` сразу после создания потока, вычисления фактически
станут последовательными.

Вызов `join` после запуска всех потоков гарантирует, что основной поток
дождется завершения всех рабочих потоков перед объединением частичных
результатов.

## 7. Локальные буферы и отсутствие гонок

Входные матрицы `A` и `B` являются общими для потоков, но только читаются.
Каждый поток использует собственный временный аккумулятор и собственную
частичную матрицу CCS.

Общие массивы итоговой матрицы `values`, `row_indices` и `col_ptrs` не
изменяются во время работы потоков. Поэтому одновременной записи в один и тот
же контейнер не происходит, и гонки данных исключаются.

## 8. Детерминированное объединение

После завершения всех потоков частичные результаты объединяются
последовательно. Объединение выполняется в порядке возрастания номеров
диапазонов, то есть в порядке столбцов результата.

Такой подход сохраняет детерминированный порядок элементов в массивах
`values`, `row_indices` и `col_ptrs`, независимо от того, в каком порядке
завершились рабочие потоки.

## 9. Производительные режимы

Производительные тесты используют два режима инфраструктуры курса:

- `pipeline` — измеряет полный жизненный цикл задачи;
- `task_run` — измеряет основной вычислительный этап после подготовки данных.

Точные таблицы времени не приводятся, потому что результаты зависят от
локального окружения, параметров сборки и значения `PPC_NUM_THREADS`.

## 10. Проверка корректности

Функциональные тесты сравнивают результат с заранее заданным CCS-представлением.
Проверяются размеры матрицы, массив `col_ptrs`, массив `row_indices` и значения
`values`. Для чисел типа `double` используется допустимая погрешность.

Набор тестов включает квадратные, прямоугольные и диагональные матрицы,
умножение на единичную матрицу и случай, когда результат не содержит ненулевых
элементов.

## 11. Команды запуска

Команды для локальной проверки STL-реализации:

```powershell
cmake --build build --config Release --parallel
$env:PPC_NUM_THREADS="4"
.\build\bin\ppc_func_tests.exe --gtest_filter="*safaryan_a_sparse_matrix_mult_crs*"
.\build\bin\ppc_perf_tests.exe --gtest_filter="*safaryan_a_sparse_matrix_mult_crs*"
```

## 12. Выводы

STL-версия показывает ручную организацию параллельного вычисления через
`std::thread`. Разбиение по столбцам делает задачи независимыми, локальные
буферы предотвращают гонки данных, а последовательное объединение сохраняет
детерминированный CCS-результат.
