# Подробное описание последовательной реализации

## 1. Назначение SEQ-версии

Последовательная версия является базовой реализацией алгоритма умножения
разреженных матриц. Она не использует параллельные технологии и выполняет все
этапы вычисления в одном потоке. Эта версия нужна для проверки корректности
общего подхода и для сравнения с последующими параллельными вариантами.

Задача: вычислить произведение `C = A * B`, где `A` и `B` — разреженные
матрицы с элементами типа `double`, сохраненные в формате CCS.

## 2. Входные и выходные данные

Входом является пара объектов `SparseMatrixCCS`. Каждый объект описывает одну
разреженную матрицу и содержит:

- `values` — массив ненулевых значений;
- `row_indices` — номера строк для элементов из `values`;
- `col_ptrs` — границы столбцов в массивах `values` и `row_indices`;
- `rows` — количество строк;
- `cols` — количество столбцов.

Выходом является матрица `C` в таком же CCS-представлении. Ее размер
определяется как `A.rows x B.cols`.

## 3. Ограничения и крайние случаи

Умножение возможно только при условии `A.cols == B.rows`. Перед вычислением
проверяется внутренняя согласованность каждой матрицы:

- размер `col_ptrs` должен быть равен `cols + 1`;
- первый элемент `col_ptrs` должен быть равен нулю;
- последний элемент `col_ptrs` должен совпадать с числом ненулевых элементов;
- массивы `values` и `row_indices` должны иметь одинаковый размер;
- значения в `col_ptrs` должны быть неубывающими;
- каждый индекс строки должен принадлежать диапазону `[0, rows)`.

Если при умножении получается значение, близкое к нулю, оно не записывается в
результат. Это позволяет сохранить разреженное представление и не добавлять
численный шум.

## 4. Формат CCS

CCS, или Compressed Column Storage, хранит матрицу по столбцам. Для столбца
`j` диапазон ненулевых элементов задается так:

```text
col_ptrs[j] .. col_ptrs[j + 1]
```

Индекс элемента внутри этого диапазона используется одновременно для массивов
`values` и `row_indices`. Например, `values[k]` — значение ненулевого элемента,
а `row_indices[k]` — строка, в которой он расположен.

Такой формат удобен для данной реализации, потому что алгоритм строит
результирующую матрицу также по столбцам.

## 5. Последовательный алгоритм

Алгоритм проходит по столбцам матрицы `B`. Для каждого столбца создается
временный плотный аккумулятор размера `A.rows`. Аккумулятор хранит значения
текущего столбца результата до переноса в CCS-структуру.

Для текущего столбца `j` матрицы `B` алгоритм перебирает возможный индекс `k`.
Если найден ненулевой элемент `B(k, j)`, то соответствующий столбец `k`
матрицы `A` добавляет вклад в аккумулятор:

```text
C(i, j) += A(i, k) * B(k, j)
```

После обработки всех вкладов ненулевые значения аккумулятора переносятся в
`values` и `row_indices` результата. Позиция начала каждого столбца
фиксируется в `col_ptrs`.

## 6. Методы класса задачи

Класс `SafaryanATaskSEQ` реализует интерфейс учебной инфраструктуры:

- `ValidationImpl` проверяет корректность CCS-структуры обеих матриц и
  совместимость размеров для умножения;
- `PreProcessingImpl` создает пустую выходную матрицу нужного размера;
- `RunImpl` запускает последовательное умножение и записывает результат;
- `PostProcessingImpl` не выполняет дополнительных преобразований.

Разделение на эти методы соответствует общему жизненному циклу задач в
проекте и упрощает запуск функциональных и производительных тестов.

## 7. Проверка корректности

Функциональные тесты используют заранее заданные матрицы и ожидаемые ответы в
формате CCS. Проверяются следующие случаи:

- простое умножение квадратных матриц;
- умножение прямоугольных матриц совместимых размеров;
- результат без ненулевых элементов;
- умножение на единичную матрицу;
- умножение диагональных матриц.

Сравнение результата включает размеры матрицы, массив `col_ptrs`, индексы
`row_indices` и значения `values`. Для значений типа `double` используется
допустимая погрешность.

## 8. Производительные тесты

Производительный тест генерирует две случайные квадратные разреженные матрицы
в формате CCS. Для проверки результата входные и выходная матрицы переводятся
в плотное представление, после чего выполняется сравнение с эталонным плотным
умножением.

Тесты запускаются в режимах `pipeline` и `task_run`. Точные численные таблицы
не приводятся, потому что результат зависит от параметров сборки и машины.
При отсутствии точных данных окружение указывается как локальное окружение.

## 9. Команды запуска

Команды для локальной проверки SEQ-версии:

```powershell
cmake --build build --config Release --parallel
.\build\bin\ppc_func_tests.exe --gtest_filter="*safaryan_a_sparse_matrix_mult_crs*"
.\build\bin\ppc_perf_tests.exe --gtest_filter="*safaryan_a_sparse_matrix_mult_crs*"
```

## 10. Выводы

Последовательная реализация корректно выполняет умножение разреженных матриц в
формате CCS и сохраняет результат в тех же массивах `values`, `row_indices` и
`col_ptrs`. Она задает понятную базовую схему вычислений и используется как
основа для анализа параллельных реализаций.
