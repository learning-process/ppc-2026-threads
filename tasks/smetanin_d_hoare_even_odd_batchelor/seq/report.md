## 1. Контекст

Эталонная последовательная реализация сортировки массива целых чисел. По ней проверяют корректность
всех остальных версий (OMP, TBB, STL, ALL) и относительное ускорение по времени.

---

## 2. Постановка задачи

**Вход.** `InType = std::vector<int>` - массив целых чисел произвольной длины `n` (в том числе `n = 0`).

**Выход.** `OutType = std::vector<int>` той же длины - перестановка элементов входа в неубывающем порядке.

**Алгоритм.** Итеративная сортировка на базе разбиения Хоара с медианным опорным элементом;
после каждого локального разбиения на подотрезок выполняется нечётно-чётное слияние на текущем
отрезке `[l, r]`, чтобы соответствовать постановке задачи с слиянием Бэтчера.

**Ограничения и крайние случаи.**

- `n ≤ 1` - массив уже упорядочен, дополнительной работы не требуется.
- В `ValidationImpl` для SEQ допускается любой вектор (возвращается `true`); фактические ограничения
  задаёт инфраструктура тестов.

---

## 3. Базовый алгоритм

1. **Разбиение Хоара.** Опорный элемент - значение в середине отрезка `[lo, hi]`; указатели сходятся
   навстречу, после прохода получается индекс `p`, разделяющий массив на две части для рекурсивной
   обработки в модели «стек вместо рекурсии».

2. **Стек подотрезков.** Вместо явной рекурсии используется `std::stack<std::pair<int,int>>`: на стек
   кладутся границы большего и меньшего подотрезка в порядке, зависящем от их размеров.

3. **OddEvenMerge на `[l, r]`.** После каждого разбиения на текущем отрезке выполняется фазовое
   сравнение пар на расстояниях `1, 2, 4, …` (пока не охвачен весь отрезок).

**Сложность по времени.** В худшем случае для быстрой сортировки - O(n²); на практике для случайных
данных ожидается около O(n log n) с дополнительными проходами odd-even merge на каждом шаге стека.

**Сложность по памяти.** O(n) под массив данных плюс O(log n) под стек границ в типичном случае.

**Инвариант.** После успешного завершения `RunImpl` выход совпадает с результатом полной сортировки
всего массива.

## 4. Детали реализации

**Файлы:**

- `seq/include/ops_seq.hpp` - класс `SmetaninDHoarSortSEQ`.
- `seq/src/ops_seq.cpp` - функции `HoarePartition`, `OddEvenMerge`, `HoarSortBatcher` в анонимном
  namespace и методы пайплайна PPC.

**Пайплайн PPC.**

- `ValidationImpl` - всегда `true`.
- `PreProcessingImpl` - копирование входа в выход.
- `RunImpl` - при `n > 1` вызов `HoarSortBatcher(data, 0, n - 1)`.
- `PostProcessingImpl` - `true`.

**Ключевой фрагмент:**

```cpp
// File: seq/src/ops_seq.cpp
void HoarSortBatcher(std::vector<int> &arr, int lo, int hi) {
  std::stack<std::pair<int, int>> stk;
  stk.emplace(lo, hi);
  while (!stk.empty()) {
    auto [l, r] = stk.top();
    stk.pop();
    if (l >= r) {
      continue;
    }
    int p = HoarePartition(arr, l, r);
    // сначала кладём на стек большую/меньшую половину в порядке из исходника
    stk.emplace(/* подотрезки [l,p] и [p+1,r] */);
    OddEvenMerge(arr, l, r);
  }
}
```

## 5. Проверка корректности

**Тесты.** `tests/functional/main.cpp` - класс `SmetaninDRunFuncTests`; `tests/performance/main.cpp` -
класс `SmetaninDRunPerfTests`.

**Функциональные тесты.** Размеры из набора `kTestParam`: `0, 1, 2, 10, 100, 1000, 10000`; метки
`empty`, `single`, …, `big`. Ожидаемый результат - отсортированная копия входа (в MPI-режиме вход
синхронизируется через `MPI_Bcast` на все ранги).

**Критерий.** Для функциональных тестов: побайтовое (поэлементное) равенство `output_data == expected_`.
Для перф-тестов: `std::ranges::is_sorted(output_data)` на массиве размера **1 000 000** с
детерминированным содержимым (`kSize_ - i`).
