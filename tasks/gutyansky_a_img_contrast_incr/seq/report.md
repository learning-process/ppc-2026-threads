## 1. Контекст

Последовательная версия служит эталоном корректности и базой для измерения ускорения. Эталонная реализация необходима
для проверки эффективности всех параллельных вариантов и определения достижимого ускорения.

## 2. Постановка задачи

Задача состоит в линейном растяжении гистограммы яркости полутонового изображения (или любого одномерного массива байт).

**Входные данные**: вектор `uint8_t` произвольной длины (например, пиксели изображения).  
**Выходные данные**: вектор той же длины, каждый элемент преобразован по следующему алгоритму:

- Найти минимальное `min` и максимальное `max` значения во входном массиве.
- Если `max == min` (все значения одинаковы), выходной массив копирует входной.
- Иначе для каждого элемента `x`:  
  `new = (255 * (x - min)) / (max - min)` (деление целочисленное).

**Ограничения**: входной массив не пуст.  
**Крайние случаи**: все пиксели равны, массив из одного элемента, максимальный разброс (0..255).

## 3. Базовый алгоритм

1. Проверка корректности: массив не пуст.
2. Выделение памяти под выходной массив.
3. Нахождение `min` и `max` одним проходом (`std::minmax_element`).
4. Вычисление `delta = max - min`.
5. Если `delta == 0` - копирование входного массива в выходной.
6. Иначе - цикл по всем элементам, вычисление нового значения по формуле с\
   использованием `uint16_t` для предотвращения переполнения.

**Асимптотика по времени**: O(N) для поиска min/max + O(N) для преобразования. Итого: O(N).  
**Асимптотика по памяти**: O(N) для хранения выходного массива.  
**Инварианты**: выходной массив той же длины; значения лежат в диапазоне [0,255]; при `max==min` выходной массив равен входному.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

- `ValidationImpl()`: проверяет наличие элементов во входном массиве `!GetInput().empty()`.
- `PreProcessingImpl()`: выделяет память под `GetOutput()` размером `GetInput().size()`.
- `RunImpl()`: реализует описанный алгоритм.
- `PostProcessingImpl()`: всегда возвращает `true` (ничего не делает).

Ключевой фрагмент:

```cpp
const auto bounds = std::minmax_element(GetInput().begin(), GetInput().end());
uint8_t lower_bound = *bounds.first;
uint8_t upper_bound = *bounds.second;
uint8_t delta = upper_bound - lower_bound;

if (delta == 0) {
  std::copy(GetInput().begin(), GetInput().end(), GetOutput().begin());
} else {
  for (size_t i = 0; i < sz; i++) {
    auto old_value = static_cast<uint16_t>(GetInput()[i]);
    uint16_t new_value = (255 * (old_value - lower_bound)) / delta;
    GetOutput()[i] = static_cast<uint8_t>(new_value);
  }
}
```

## 5. Проверка корректности

Функциональные тесты используют 10 предопределённых наборов данных (файлы _test_1.txt_ … _test_10.txt_).
Каждый из тестовых файлов, хранящихся в директории `data/`, содержит размер массива, входные значения и ожидаемые
выходные значения. Среди функциональных тестов представлены как тесты для обычных случаев (несколько различных элементов),
так и тесты для краевых случаев (все элементы одинаковые, массив из одного элемента и т.п.).

Производительность измеряется на массиве большого размера (1e8 элементов) заполненным с использованием специального паттерна,
упрощающего валидацию результата. Тестируется ветвь алгоритма с преобразованием данных, так как тестирование копирования
входного массива в выходной не представляет особого интереса.
