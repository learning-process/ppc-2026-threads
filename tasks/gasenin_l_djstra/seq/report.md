## 1. Введение

Алгоритм Дейкстры — классический метод поиска кратчайших путей от одной вершины
до всех остальных в графах с неотрицательными весами рёбер.
В данной работе рассматривается последовательная реализация для полносвязного графа,
где вес ребра вычисляется аналитически как $|u - v|$.
Данная версия служит эталоном (baseline) для последующего сравнения с параллельными
реализациями: OMP, STL, TBB и ALL.

## 2. Постановка задачи

Дан неориентированный полносвязный граф с $n$ вершинами.

- Вес ребра между вершинами $u$ и $v$ равен $|u - v|$.
- **Входные данные:** количество вершин $n$.
- **Выходные данные:** сумма длин кратчайших путей от вершины 0 до всех остальных.
- **Ограничения:** $n > 0$. Теоретическая сумма: $\frac{n(n-1)}{2}$.

**Инварианты и крайние случаи:**

- При $n = 1$ сумма равна 0 (только вершина-источник).
- Кратчайший путь от 0 до $v$ в данном графе всегда равен $v$, что проверяется
  аналитической формулой $\frac{n(n-1)}{2}$ и используется в тестах корректности.

## 3. Последовательный алгоритм

Используется алгоритм Дейкстры со сложностью $O(n^2)$:

1. **Инициализация:** массив расстояний `dist` заполняется бесконечностью (`kInf`),
   кроме `dist[0] = 0`. Массив `visited` — `false`.
2. **Основной цикл** (повторяется $n$ раз):
   - `FindMinDist` — линейный проход, выбирает непосещённую вершину $u$ с минимальным
     `dist[u]`. Сложность — $O(n)$ за итерацию.
   - Вершина $u$ помечается посещённой (`visited[u] = true`).
   - `RelaxEdges` — для всех непосещённых $v \ne u$: если $dist[u] + |u - v| < dist[v]$,
     обновляет `dist[v]`. Сложность — $O(n)$ за итерацию.
3. **Постобработка:** суммирование элементов `dist`, результат записывается в выходные данные.

### 3.1 Ключевой фрагмент — поиск вершины минимума

Файл: `seq/src/ops_seq.cpp`, метод `FindMinDist`.

```cpp
InType GaseninLDjstraSEQ::FindMinDist(const std::vector<InType> &dist,
                                       const std::vector<bool> &visited) {
  auto n = static_cast<InType>(dist.size());
  InType min_dist = std::numeric_limits<InType>::max();
  InType u = -1;
  for (InType i = 0; i < n; ++i) {
    if (!visited[i] && dist[i] < min_dist) {
      min_dist = dist[i];
      u = i;
    }
  }
  return u;
}
```

Функция возвращает индекс непосещённой вершины с наименьшим расстоянием или `-1`,
если таких вершин нет. Это условие завершения основного цикла.

### 3.2 Ключевой фрагмент — релаксация рёбер

Файл: `seq/src/ops_seq.cpp`, метод `RelaxEdges`.

```cpp
void GaseninLDjstraSEQ::RelaxEdges(InType u, std::vector<InType> &dist,
                                    const std::vector<bool> &visited) {
  auto n = static_cast<InType>(dist.size());
  for (InType vert = 0; vert < n; ++vert) {
    if (!visited[vert] && u != vert) {
      InType weight = (u > vert) ? (u - vert) : (vert - u);
      if (dist[u] != std::numeric_limits<InType>::max() && dist[u] + weight < dist[vert]) {
        dist[vert] = dist[u] + weight;
      }
    }
  }
}
```

Вес ребра вычисляется через ветвление без `std::abs`, что эквивалентно для целых
типов. Проверка на `max()` предотвращает переполнение при сложении.

## 4. Детали реализации

- **Заголовочный файл:** `seq/include/ops_seq.hpp`.
- **Файл реализации:** `seq/src/ops_seq.cpp`.
- **Основные методы:**
  - `ValidationImpl` — проверяет $n > 0$.
  - `PreProcessingImpl` — не выделяет память (массивы создаются в `RunImpl`).
  - `RunImpl` — содержит полный цикл алгоритма и агрегацию результата.
  - `PostProcessingImpl` — пустой (результат уже записан в `RunImpl`).
- **Память:** $O(n)$ для `dist` и `visited`. Матрица смежности не создаётся;
  веса вычисляются аналитически в `RelaxEdges`.
- **Параллелизм:** отсутствует. Все операции выполняются в одном потоке, что
  гарантирует детерминированность результата и служит чистым baseline.

### 6.2 Производительность

Определения столбцов таблицы:

- **Режим** — сценарий тестового фреймворка (`task_run` / `pipeline`).
- **MPI-процессов** — число запущенных рангов (для SEQ влияет только на накладные
  расходы MPI-инициализации, не на вычисление).

- **Ускорение** — $S = T_{\text{baseline}} / T$ (для SEQ всегда 1.00).
- **Эффективность** — $E = S / P$, где $P = 1$ поток (для SEQ всегда 100%).
