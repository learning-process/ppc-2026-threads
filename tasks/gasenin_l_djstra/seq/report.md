- Вариант: Поиск кратчайших путей в полносвязном графе с весами $w(u, v) = |u - v|$

## 1. Введение
Алгоритм Дейкстры является классическим методом поиска кратчайших путей от одной вершины до всех остальных в графах с неотрицательными весами ребер. В данной работе рассматривается последовательная реализация для полносвязного графа, где веса ребер вычисляются аналитически. Данная версия служит эталоном (baseline) для последующего сравнения с параллельными реализациями.

## 2. Постановка задачи
Дан неориентированный полносвязный граф с $n$ вершинами.
- Вес ребра между вершинами $u$ и $v$ равен $|u - v|$.
- **Входные данные:** количество вершин $n$.
- **Выходные данные:** сумма длин кратчайших путей от вершины 0 до всех остальных.
- **Ограничения:** $n > 0$. Сумма для такого типа графа вычисляется по формуле $\frac{n(n-1)}{2}$.

## 3. Последовательный алгоритм
Используется последовательный алгоритм Дейкстры со сложностью $O(V^2)$:
1. Инициализация: массив расстояний `dist` заполняется бесконечностью, кроме `dist[0] = 0`. Массив `visited` заполняется `false`.
2. В цикле по всем вершинам:
   - Выбирается непосещенная вершина $u$ с минимальным значением $dist[u]$.
   - Вершина $u$ помечается как посещенная.
   - Для всех остальных непосещенных вершин $v$ выполняется релаксация: если $dist[u] + |u - v| < dist[v]$, то $dist[v]$ обновляется.
3. По завершении цикла вычисляется общая сумма значений в массиве `dist`.


## 4. Implementation Details
- **Заголовочный файл:** `ops_seq.hpp`.
- **Файл реализации:** `ops_seq.cpp`.
- **Основные компоненты:**
  - `FindMinDist`: вспомогательный метод для линейного поиска вершины с минимальным расстоянием.
  - `RelaxEdges`: метод для обновления путей, использующий формулу `(u > vert) ? (u - vert) : (vert - u)` для весов.
  - `RunImpl`: управляет жизненным циклом алгоритма и агрегацией результата.
- **Память:** используется $O(n)$ памяти для хранения расстояний и статуса посещения, матрица смежности не создается.

### 6.1 Корректность
Корректность алгоритма подтверждена функциональными тестами в `main.cpp` для различных размеров графа (3, 5, 7 вершин). Итоговая сумма во всех случаях совпала с теоретической величиной $\frac{n(n-1)}{2}$.

**Анализ логов при запуске на разном количестве процессов:**
Несмотря на запуск тестов через `mpiexec` с параметрами `-n 2`, `-n 4` и `-n 6`, время работы последовательной версии остается стабильным (в районе 0.016с). Это подтверждает, что реализация `SEQ` корректно игнорирует доступные ресурсы параллелизма и выполняет вычисления строго в одном потоке, не тратя время на межпроцессорное взаимодействие. Небольшие колебания времени (в пределах миллисекунд) обусловлены системными шумами и накладными расходами измерительного фреймворка.
