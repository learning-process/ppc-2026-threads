## 1. Контекст

Умножение разреженных матриц — одна из ключевых операций в вычислительной линейной алгебре, широко применяемая в научных
расчётах, машинном обучении, компьютерной графике и других областях. При работе с разреженными матрицами (большинство
элементов — нули) использование плотного представления приводит к неэффективному расходу памяти и вычислительных
ресурсов. Формат CRS (Compressed Row Storage, строковый формат хранения) позволяет хранить только ненулевые элементы,
что даёт существенную экономию памяти и времени.

Последовательная версия служит эталоном корректности и производительности (baseline) для сравнения с параллельными
реализациями на базе OpenMP, oneTBB, `std::thread` и гибридной MPI+threads версии. Все дальнейшие замеры ускорения будут
нормироваться на время выполнения именно этой последовательной реализации.

## 2. Постановка задачи

### Входные данные

- Две матрицы A и B в формате CRS с комплексными элементами типа `std::complex<double>`
- Каждая матрица описывается структурой `CRSMatrix`:
    - `rows` — количество строк
    - `cols` — количество столбцов
    - `values` — вектор ненулевых элементов (последовательная запись по строкам)
    - `index_col` — вектор номеров столбцов для каждого элемента из `values`
    - `index_row` — вектор указателей начала каждой строки в `values` (длина `rows+1`)

### Выходные данные

- Матрица C = A × B в том же формате CRS

### Ограничения

- `A.cols` должно быть равно `B.rows`
- Элементы матриц — комплексные числа (`std::complex<double>`)
- Нулевыми считаются элементы, чей модуль не превышает `1e-12`

### Крайние случаи

- Пустые матрицы (все элементы нулевые)
- Одна из матриц нулевая
- Умножение на единичную матрицу
- Матрицы размером 1×1
- Неквадратные матрицы (при согласованных размерностях)

## 3. Базовый алгоритм

### Описание

Алгоритм классического умножения матриц `C = A × B` в CRS-формате:

1. Для каждой строки `i` матрицы A (от `0` до `A.rows-1`):
    - Создать ассоциативный контейнер (`unordered_map`) для накопления сумм по столбцам результата
    - Для каждого ненулевого элемента `A[i][k]` (где `k` — номер столбца в A):
        - Найти строку `k` в матрице B (все ненулевые элементы `B[k][j]`)
        - Для каждого элемента `B[k][j]` добавить произведение `A[i][k]` × `B[k][j]` к накопленной сумме для столбца j
    - Отсортировать накопленные элементы по возрастанию номеров столбцов
    - Добавить в результирующую матрицу только те элементы, чей модуль превышает `1e-12`
    - Обновить `index_row`

### Асимптотика

- Время: `O(nnz(A) × avg_nnz_per_row(B))` в лучшем случае, где nnz — количество ненулевых элементов. В худшем случае
  (плотные матрицы) — `O(n³)`.
- Память: `O(nnz(A) + nnz(B) + nnz(C))` дополнительной памяти для временного хранения.

### Инварианты

- После завершения алгоритма `res.index_row[i]` указывает на начало i-й строки в `values` и `index_col`
- `res.index_row.size() == res.rows + 1`
- Все элементы с модулем `≤ 1e-12` отфильтрованы
- В каждой строке столбцы отсортированы по возрастанию

### Критерий корректности

Результат считается корректным, если для всех `i`, `j`:

- Если `C[i][j]` (рассчитанный по плотному алгоритму) имеет модуль `≤ 1e-12`, то в CRS-представлении этот элемент
  отсутствует
- Иначе в CRS-представлении есть элемент `(i, j)` с комплексным значением, отличающимся от эталонного не более чем на
  `1e-9` по модулю

## 4. Детали реализации

### Файлы

- `seq/include/ops_seq.hpp` — объявление класса PosternakACRSMulComplexMatrixSEQ
- `seq/src/ops_seq.cpp` — реализация методов

### Методы класса

| Метод                | Назначение                                                                                                |
| -------------------- | --------------------------------------------------------------------------------------------------------- |
| ValidationImpl()     | Проверяет корректность входных данных: валидность матриц, согласованность размерностей (A.cols == B.rows) |
| PreProcessingImpl()  | Инициализирует выходную матрицу: задаёт размерность, очищает контейнеры, резервирует память под index_row |
| RunImpl()            | Выполняет основное вычисление — умножение матриц в CRS-формате                                            |
| PostProcessingImpl() | Проверяет валидность выходной матрицы                                                                     |

### Особенности реализации

1. Накопление сумм: Используется `std::unordered_map<int, std::complex<double>>` для временного хранения сумм по
   столбцам в текущей строке. Это даёт амортизированное O(1) на вставку/обновление.

2. Сортировка результатов: После накопления всех произведений для строки элементы сортируются с помощью
   `std::ranges::sort` по номеру столбца для соблюдения формата CRS.

3. Фильтрация нулей: Элементы с модулем ≤ 1e-12 не добавляются в результирующую матрицу.

4. Корректность формата: После обработки всех строк в index_row добавляется финальный указатель на конец вектора values.

### Обработка исключительных случаев

### Ключевой фрагмент алгоритма умножения

```cpp
// Файл: seq/src/ops_seq.cpp — ядро умножения в RunImpl()
for (int row = 0; row < res.rows; ++row) {
  std::unordered_map<int, std::complex<double>> row_sum;

  for (int idx_a = a.index_row[row]; idx_a < a.index_row[row + 1]; ++idx_a) {
    const int col_a = a.index_col[idx_a];
    const auto &val_a = a.values[idx_a];

    for (int idx_b = b.index_row[col_a]; idx_b < b.index_row[col_a + 1]; ++idx_b) {
      const int col_b = b.index_col[idx_b];
      const auto &val_b = b.values[idx_b];
      row_sum[col_b] += val_a * val_b;
    }
  }

  res.index_row.push_back(static_cast<int>(res.values.size()));

  std::vector<std::pair<int, std::complex<double>>> sorted_elements(row_sum.begin(), row_sum.end());
  std::ranges::sort(sorted_elements, [](const auto &a, const auto &b) { return a.first < b.first; });

  for (const auto &[col_idx, value] : sorted_elements) {
    if (std::abs(value) > 1e-12) {
      res.values.push_back(value);
      res.index_col.push_back(col_idx);
    }
  }
}
res.index_row.push_back(static_cast<int>(res.values.size()));
```

## 5. Проверка корректности

### Функциональные тесты

Тесты расположены в `tests/functional/main.cpp`. Используются следующие сценарии:

| ID  | Описание               | Размерность     | Особенность                    |
| --- | ---------------------- | --------------- | ------------------------------ |
| 0   | Базовый тест           | 2×3 × 3×2 = 2×2 | Целые числа                    |
| 1   | Комплексные числа      | 1×1 × 1×1 = 1×1 | (1+i) × (1-i) = 2+0i           |
| 2   | Умножение на единичную | 3×3 × 3×3 = 3×3 | Результат равен A              |
| 3   | Пустые матрицы         | 2×2 × 2×2 = 2×2 | Все элементы нулевые           |
| 4   | Умножение на нулевую   | 2×2 × 2×2 = 2×2 | Результат нулевой              |
| 5   | Комплексные            | 4×3 × 3×4 = 4×4 | Смешанные комплексные значения |

### Реализация проверки

Функция `MatricesEqual` сравнивает две CRS-матрицы с учётом погрешности `1e-9`. Проверяются:

- Размерности `(rows, cols)`
- Количество ненулевых элементов
- Значения элементов (с точностью до `1e-9`)
- Индексы столбцов
- Указатели начала строк (`index_row`)

### Пример проверки

Тест ID=1 (комплексные числа 1×1):

- Вход: `A = [1+i], B = [1-i]`
- Ожидаемый результат: `C = (1+i)(1-i) = 1 - i² = 1 + 1 = 2 + 0i`
- Фактический результат: `values = [2+0i], index_col = [0], index_row = [0, 1]` — соответствует ожиданию
