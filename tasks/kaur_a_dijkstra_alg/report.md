## 1. Введение

Реализован алгоритм Дейкстры для графа, представляющего собой цепочку вершин,
где каждое ребро имеет вес 1. Задача выбрана как классическая задача на графах
с естественным параллелизмом при релаксации рёбер. Реализованы и сравнены пять
технологий: последовательная (SEQ), OpenMP (OMP), oneTBB (TBB), std::thread (STL)
и гибридная MPI + потоки (ALL). Результат — сумма кратчайших расстояний от
стартовой вершины 0 до всех достижимых вершин.

## 2. Единая постановка задачи

- **Вход**: количество вершин N (int).
- **Выход**: сумма кратчайших расстояний от вершины 0 до всех вершин.
- **Граф**: вершины 0..N-1, рёбра (i → i+1) с весом 1.
- **Ограничения**: N > 0.
- **Критерий корректности**: сумма расстояний должна равняться N*(N-1)/2.

## 3. Единая методика эксперимента

- **CPU**: Intel Core i7-12700H (14 ядер, 20 потоков)
- **RAM**: 32 GB DDR5
- **OS**: Ubuntu 22.04 (контейнер Docker)
- **Compiler**: GCC 14.2.0
- **Build type**: Release
- **Переменные окружения**: PPC_NUM_THREADS для OMP/STL/TBB,
  PPC_NUM_PROC + PPC_NUM_THREADS для ALL
- **Входные данные**: N = 200 (perf), функциональные тесты: 3, 5, 7.
- **Speedup**: T_seq / T_parallel
- **Efficiency**: Speedup / num_workers
- **Число повторов**: не менее 5, агрегация медианой.

## 4. Сводка корректности

Все backend-ы сравнивались с SEQ на функциональных тестах (N=3,5,7) и
показывали ожидаемую сумму (3, 10, 21). Тесты производительности для N=200
также дают корректный результат 19900.

## 6. Интерпретация различий

- **SEQ** — эталон. Сложность O(N²) из-за линейного поиска минимума.
- **OMP** — параллельный поиск минимума и релаксация рёбер. Ускорение
  ограничено накладными расходами на синхронизацию при малых N.
- **TBB** — использование tbb::parallel_for на соседях вершины. Эффективен
  при большом количестве соседей, в цепочке выигрыш мал.
- **STL** — ручное управление потоками с барьерами. Показывает результаты,
  близкие к OMP, но требует осторожного обращения с condition_variable.
- **ALL** — MPI распределяет вершины между процессами, внутри каждого
  работает OpenMP или std::thread. Коммуникационные издержки окупаются
  только при очень больших N.
