## 2. Постановка задачи

### Входные данные

Дан неструктурированный массив вещественных чисел:

$$A \in \mathbb{R}^{N}  $$

Массив передается в виде структуры `std::vector<double>`.

### Выходные данные

Результирующий массив:

$$B \in \mathbb{R}^{N}  $$

где для любых индексов $i < j$ выполняется условие $B[i] \le B[j]$. Массив сохраняется в тот же вектор (или возвращается
как результат выполнения задачи).

### Ограничения

Выполнение алгоритма возможно только при условии наличия элементов в массиве. Проверка осуществляется на этапе
валидации:

$$N > 0  $$

При пустом массиве вычисление не выполняется.

### Граничные случаи

Алгоритм корректно обрабатывает следующие ситуации:

- наличие отрицательных и положительных чисел;
- наличие нулей;
- уже отсортированные массивы (в прямом и обратном порядке);
- массивы, состоящие из одинаковых элементов.

## 3. Базовый алгоритм

Алгоритм использует поразрядную сортировку с основанием системы счисления 256 (просиходит сортировка по байтам).

Поскольку тип `double` занимает 8 байт (64 бита), требуется ровно 8 проходов. Для выполнения побитовых операций
вещественные числа безопасно преобразуются в беззнаковое целое число `uint64_t` с помощью `std::bit_cast`.

Для каждого байта (от младшего к старшему):

1. Вычисляется частота встречаемости каждого значения байта (от 0 до 255).
2. Вычисляются префиксные суммы для определения стартовых позиций каждого «корзины» (bucket).
3. Элементы исходного массива распределяются во временный массив `temp` согласно вычисленным позициям.

### Коррекция отрицательных чисел (Специфика IEEE 754)

Стандарт IEEE 754 устроен так, что при интерпретации `double` как беззнакового целого числа, положительные числа
сохраняют правильный порядок сортировки. Однако отрицательные числа получают инвертированный порядок и оказываются
больше положительных (из-за установленного старшего знакового бита).

Для исправления этого после 8 проходов выполняется фаза слияния:

1. Массив разделяется на два подмассива: `negative` и `positive`.
2. Порядок элементов в подмассиве `negative` обращается (reverse).
3. Подмассивы сливаются в итоговый массив в порядке: сначала отрицательные, затем положительные.

### Асимптотика

#### Временная сложность

$$O(P \times N) = O(N)  $$

где:

- $N$ — количество элементов массива;
- $P$ — количество проходов (константа, $P = 8$ для `double`). Финальное разделение и слияние также выполняется за
  $O(N)$.

#### Пространственная сложность

$$O(N + R) = O(N)  $$

Дополнительная память расходуется на временный массив `temp` размера $N$, массивы для отрицательных и положительных
чисел, а также массив счетчиков размера $R = 256$.

## 4. Детали реализации

### Структура файлов

- `frolova_s_radix_sort_double/common/include/common.hpp` — общие структуры и типы;
- `frolova_s_radix_sort_double/seq/include/ops_seq.hpp` — объявление sequential backend;
- `frolova_s_radix_sort_double/seq/src/ops_seq.cpp` — реализация этапов выполнения задачи.

### ValidationImpl

Функция проверяет наличие элементов во входном массиве.

```cpp
bool FrolovaSRadixSortDoubleSEQ::ValidationImpl() {
  return !GetInput().empty();
}

```

### PreProcessingImpl

Не требует дополнительных действий в последовательной версии, возвращает `true`.

### RunImpl

Основной этап вычислений. Выполняет 8 итераций подсчета и распределения с использованием `std::bit_cast<uint64_t>(val)`
для доступа к битам вещественного числа. В конце выполняется логика распределения на `negative` и `positive`, разворот
вектора `negative` с помощью `std::ranges::reverse` и финальное слияние.

### PostProcessingImpl

Возвращает статус успешного завершения постобработки.
