## 1. Контекст

Параллелизм реализован **вручную**: создаются потоки `std::thread`, работа делится по индексам, в конце вызывается `join`. Число потоков берётся из `PPC_NUM_THREADS` (функция `GetNumThreads()`).

## 2. Постановка задачи

- **Вход:** кортеж `(b_size, A, B)` — размер блока `b_size` и две квадратные матрицы `A`, `B` размера `n×n` (`common/include/common.hpp`).
- **Выход:** матрица `C = A·B` размера `n×n`.
- **Ограничения:** `b_size > 0`; `n` кратно `b_size`; матрицы квадратные (проверяется в `ValidationImpl`).
- **Эталон для сравнения:** SEQ, режим `task_run`, один поток: **\(T_{seq} = 0.0499\) с** (матрицы `512×512`, блок `32×32`).

## 3. Базовый алгоритм

**Блочный алгоритм Кэннона:**

1. `grid_sz = n / b_size`.
2. **Раскладка** `DistributeData` — начальное размещение блоков `A` и `B` в сетке.
3. **Цикл шагов** `step = 0 … grid_sz-1`: для каждой пары блоков `(i,j)` — `BlockMultiplyAccumulate`.
4. Сдвиги `RotateBlocksA` / `RotateBlocksB` между шагами.
5. **Сборка** `CollectResult` в итоговую матрицу.

Сложность по времени: \(O(n^3)\). По памяти: \(O(n^2)\).

## 4. Схема распараллеливания

Вспомогательная функция `ParallelFor(work_size, func)`:

- создаёт `num_threads` потоков (не больше `work_size`);
- поток с номером `t` обрабатывает индексы `t, t+num_threads, t+2·num_threads, …`;
- после цикла создания — **отдельный цикл `join`** для всех потоков.

Так параллелятся:

- раскладка блоков (по индексу строки сетки `i`);
- умножение на шаге (по `i`, внутри цикл по `j`);
- сдвиги и сборка.

Запись идёт в разные блоки — **mutex не используется**.

```cpp
// File: stl/src/ops_stl.cpp
for (int thread_id = 0; thread_id < num_threads; ++thread_id) {
  threads.emplace_back([&, thread_id]() {
    for (int index = thread_id; index < work_size; index += num_threads) {
      func(index);
    }
  });
}
for (auto &thread : threads) {
  thread.join();
}
```
