## 1. Контекст

Та же схема Кэннона, что в SEQ, но работа распределяется библиотекой **oneTBB** (`tbb::parallel_for`).
Потоки создаёт и балансирует сама библиотека; число рабочих потоков ограничивается через `PPC_NUM_THREADS`.

## 2. Постановка задачи

- **Вход:** кортеж `(b_size, A, B)` — размер блока `b_size` и две квадратные матрицы `A`, `B`
  размера `n×n` (`common/include/common.hpp`).
- **Выход:** матрица `C = A·B` размера `n×n`.
- **Ограничения:** `b_size > 0`; `n` кратно `b_size`; матрицы квадратные (проверяется в `ValidationImpl`).
- **Эталон для сравнения:** SEQ, режим `task_run`, один поток: **\(T_{seq} = 0.0499\) с**
  (матрицы `512×512`, блок `32×32`).

## 3. Базовый алгоритм

**Блочный алгоритм Кэннона:**

1. `grid_sz = n / b_size`.
2. **Раскладка** блоков `DistributeBlocks` со сдвигом `(i+j) mod grid_sz`.
3. **Шаги** `step = 0 … grid_sz-1`: умножение всех пар блоков `MultiplyBlocks`.
4. Сдвиг блоков `RotateAll` (кроме последнего шага).
5. **Сборка** `AssembleResult` в матрицу `C`.

Сложность по времени: \(O(n^3)\). По памяти: \(O(n^2)\).

## 4. Схема распараллеливания

- Раскладка и сборка блоков — `parallel_for` по двумерному диапазону `blocked_range2d` (индексы `i`, `j` сетки).
- Умножение на шаге Кэннона — тот же `parallel_for` по `(i,j)`.
- Сдвиг блоков — два `parallel_for` по размеру сетки (строки `A`, столбцы `B`).

Каждая пара `(i,j)` обрабатывается независимо, гонок данных нет.

Пример:

```cpp
// File: tbb/src/ops_tbb.cpp
tbb::parallel_for(tbb::blocked_range2d<int>(0, grid_sz, 0, grid_sz),
                  [&](const tbb::blocked_range2d<int> &r) {
  for (int i = r.rows().begin(); i != r.rows().end(); ++i) {
    for (int j = r.cols().begin(); j != r.cols().end(); ++j) {
      MultiplyBlocks(bl_a[i][j], bl_b[i][j], bl_c[i][j], b_size);
    }
  }
});
```
