## 1. Контекст

Реализовано последовательное умножение квадратных матриц по **блочной схеме Кэннона**:
матрица делится на сетку блоков размера `b_size`, на каждом из `grid_sz` шагов выполняется
умножение и накопление в блоках результата, между шагами блоки `A` и `B` циклически сдвигаются.
Эта версия задаёт **эталон корректности** и базовое время \(T_{seq}\) для параллельных реализаций
(OMP, TBB, STL, ALL).

## 2. Постановка задачи

- **Вход:** кортеж `(b_size, A, B)` — размер блока и две матрицы `n×n` (`common/include/common.hpp`).
- **Выход:** матрица `C = A·B`.
- **Ограничения:** `b_size > 0`, матрицы квадратные, `n` кратно `b_size` (проверка в `ValidationImpl`).

## 3. Базовый алгоритм

1. `grid_sz = n / b_size`.
2. **Раскладка** `DistributeData`: для блока `(i,j)` сдвиг столбцов `A` — `(i+j) mod grid_sz`,
   сдвиг строк `B` — тот же индекс.
3. **Цикл шагов** `step = 0 … grid_sz-1`: для всех `(i,j)` — `BlockMultiplyAccumulate`;
   затем `RotateBlocksA` / `RotateBlocksB` (кроме последнего шага).
4. **Сборка** `CollectResult` в матрицу `n×n`.

- **Сложность по времени:** \(O(n^3)\).
- **Сложность по памяти:** \(O(n^2)\) на сетку блоков и результат.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.

Конвейер задачи: `ValidationImpl` → `PreProcessingImpl` → `RunImpl` → `PostProcessingImpl`.

**Листинг основного цикла Кэннона:**

```cpp
// File: seq/src/ops_seq.cpp
for (int step = 0; step < grid_sz; ++step) {
  for (int i = 0; i < grid_sz; ++i) {
    for (int j = 0; j < grid_sz; ++j) {
      BlockMultiplyAccumulate(bl_a[i][j], bl_b[i][j], bl_c[i][j], b_size);
    }
  }
  if (step < grid_sz - 1) {
    RotateBlocksA(bl_a, grid_sz);
    RotateBlocksB(bl_b, grid_sz);
  }
}
```

_Пояснение:_ внешний цикл по шагам алгоритма Кэннона; внутри — полный проход по сетке блоков.
Узкое место — тройной цикл в `BlockMultiplyAccumulate` (порядок циклов `i–k–j` для локальности).
