## 1. Введение

Сортировка данных является одной из фундаментальных задач в области информатики.
Поразрядная сортировка (Radix Sort) позволяет достичь линейной сложности относительно количества элементов
при фиксированном диапазоне значений (в отличие от алгоритмов, основанных на сравнениях,
таких как QuickSort или MergeSort).

Данная работа направлена на реализацию последовательной версии алгоритма для работы с 32-битными
целыми числами со знаком.

## 2. Постановка задачи

Необходимо реализовать последовательный алгоритм поразрядной сортировки (LSD — Least Significant Digit)
для вектора целых чисел типа `int32_t`.

- **Входные данные**: Неотсортированный вектор `std::vector<int32_t>`.
- **Выходные данные**: Тот же вектор, элементы которого расположены в порядке неубывания.
- **Ограничения**: Алгоритм должен корректно обрабатывать отрицательные числа.

## 3. Описание алгоритма (последовательный)

Реализованный алгоритм использует поразрядную сортировку по основанию 10:

1. **Нормализация**: Так как классическая поразрядная сортировка работает с неотрицательными числами,
   в начале работы находится минимальный элемент. Если он меньше нуля, весь массив сдвигается
   в положительную область на величину $|min\_val|$.
2. **Сортировка по разрядам**: Цикл проходит по всем разрядам (единицы, десятки, сотни и т.д.)
   максимального числа. На каждом шаге используется стабильная сортировка подсчетом (Counting Sort).
3. **Возврат к исходному диапазону**: После завершения всех проходов выполняется обратный сдвиг значений,
   если на первом этапе массив был нормализован.

## 4. Схема распараллеливания

В данной версии (SEQ) распараллеливание не предусмотрено. Все вычисления выполняются последовательно
на одном ядре процессора. Данные обрабатываются одним потоком, что является эталонным значением
для вычисления ускорения в будущих параллельных версиях.

## 5. Детали реализации

- **Структура кода**:
  - `VotincevDRadixMergeSortSEQ`: Класс, реализующий интерфейс задачи.
  - `SortByDigit`: Метод, выполняющий сортировку подсчетом для конкретного десятичного разряда.
  - `RunImpl`: Основной метод, координирующий нормализацию, цикл по разрядам и возврат к исходному диапазону.
- **Особенности**: Для подсчета частот цифр (0-9) используется `std::array<int32_t, 10>`,
  расположенный на стеке, что ускоряет работу по сравнению с динамическим выделением памяти.
- **Память**: Алгоритм требует $O(n)$ дополнительной памяти для создания временного массива.
- **Оценка сложности**: Алгоритм работает за $O(d \cdot (n + k))$, где $d$ — количество разрядов,
  $n$ — размер массива, $k$ — основание системы счисления (10).

## Приложение

```cpp


// Если массив пустой - сортировать нечего
bool VotincevDRadixMergeSortSEQ::ValidationImpl() {
  return !GetInput().empty();
}

// Препроцессинга нет для сортировки
// (для изображений могло бы быть что-то вроде перевода в ч/б)
bool VotincevDRadixMergeSortSEQ::PreProcessingImpl() {
  return true;
}

// Постобработка не проводится
bool VotincevDRadixMergeSortSEQ::PostProcessingImpl() {
  return true;
}


bool VotincevDRadixMergeSortSEQ::RunImpl() {
  std::vector<int32_t> working_array = GetInput();

  auto [min_it, max_it] = std::ranges::minmax_element(working_array);
  int32_t min_val = *min_it;
  int32_t max_val = *max_it;

  // Сдвиг в положительную область для корректной работы с отрицательными числами
  if (min_val < 0) {
    for (auto &num : working_array) {
      num -= min_val;
    }
    max_val -= min_val;
  }

  // Цикл по разрядам (LSD)
  for (int64_t exp = 1; static_cast<int64_t>(max_val) / exp > 0; exp *= 10) {
    SortByDigit(working_array, static_cast<int32_t>(exp));
  }

  // Возврат к исходному диапазону
  if (min_val < 0) {
    for (auto &num : working_array) {
      num += min_val;
    }
  }

  GetOutput() = std::move(working_array);
  return true;
}
