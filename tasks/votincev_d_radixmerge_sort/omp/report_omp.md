## 1. Введение

Сортировка данных является одной из фундаментальных задач в области информатики. В отличие от алгоритмов,
основанных на сравнениях, поразрядная сортировка (Radix Sort) обладает линейной сложностью относительно
количества элементов. Применение технологии OpenMP позволяет ускорить процесс обработки больших массивов
данных за счет распределения вычислений между несколькими ядрами процессора.

Данная работа посвящена реализации и анализу эффективности параллельной версии поразрядной сортировки
с использованием OpenMP.

## 2. Постановка задачи

Необходимо реализовать параллельный алгоритм поразрядной сортировки для вектора 32-битных целых чисел.

- **Входные данные**: Неотсортированный вектор `std::vector<int32_t>`.
- **Выходные данные**: Отсортированный вектор.
- **Ограничения**: Корректная обработка отрицательных чисел и обеспечение стабильности сортировки внутри потоков.
(стабилен за счет использования вспомогательного буфера + обход с конца)

## 3. Описание алгоритма (последовательный)

Последовательный алгоритм служит базой для локальной сортировки в потоках. Он основан на десятичной
поразрядной сортировке (LSD):

1. Определение диапазона значений для нормализации (сдвиг отрицательных чисел).
2. Поразрядный обход данных с использованием сортировки подсчетом (Counting Sort).
3. Восстановление исходного диапазона значений.

## 4. Схема распараллеливания

Реализованная схема распараллеливания использует принцип "разделяй и властвуй":

1. **Разделение**: Входной массив делится на $N$ равных частей (где $N$ — количество потоков OpenMP).
2. **Локальная сортировка**: Каждый поток независимо выполняет полную поразрядную сортировку своего
   участка данных (`LocalRadixSort`).
3. **Параллельное слияние**: Отсортированные блоки объединяются парами в итеративном цикле. Используется
   барьерная синхронизация (`#pragma omp barrier`), чтобы гарантировать завершение этапов слияния перед
   переходом к следующему уровню "дерева" слияния.

## 5. Детали реализации

Параллельная версия реализована с использованием директив OpenMP для распределения блоков массива между потоками и последующего древовидного слияния.

- **Директива параллельной области:**
Использована конструкция `#pragma omp parallel default(none) shared(n, working_array, temp_buffer)`.
Явное указание `default(none)` исключает случайное разделение переменных и гарантирует строгий контроль над памятью.
- **Shared-переменные:**

1) исходный/выходной массив `working_array`
2) временный буфер для слияния `temp_buffer`
3) общий размер массива `n`.
- **Private-переменные:**

1) идентификатор потока `tid`
2) общее число запущенных потоков `n_threads`
3) границы ответственности потока `l` и `r`
4) переменные `items` и `rem`
Эти переменные создаются внутри параллельного блока.

- **Reduction (Редукция):**
Для нахождения минимального элемента вектора используется параллельный цикл с явным указанием редукции:

```cpp
#pragma omp parallel for reduction(min : min_val) default(none) shared(n, input)
for (int32_t i = 0; i < n; ++i) {
    min_val = std::min(input[i], min_val);
}
* **Политика планирования (Schedule):** Распределение элементов массива по потокам выполнено вручную на основе статического деления индексов (`int32_t items = n / n_threads`).
(то есть schedule не указывался, эквивалент schedule(static))
Это оптимально, так как:
итерации циклов нормализации выполняют строго фиксированное и одинаковое количество арифметических операций.
Следовательно, равномерная статическая блочная схема распределения итераций минимизирует накладные расходы.

Использование OpenMP позволило сократить время сортировки 1 млн элементов с 2.414с до 1.300с. Наблюдается
снижение эффективности при увеличении числа потоков, что объясняется накладными расходами на
синхронизацию (барьеры) и необходимостью последовательного слияния крупных блоков данных на финальных этапах.

## Приложение

```cpp


// Если массив пустой - сортировать нечего
bool VotincevDRadixMergeSortOMP::ValidationImpl() {
  return !GetInput().empty();
}

// Препроцессинга нет для сортировки
// (для изображений могло бы быть что-то вроде перевода в ч/б)
bool VotincevDRadixMergeSortOMP::PreProcessingImpl() {
  return true;
}

// Постобработка не проводится
bool VotincevDRadixMergeSortOMP::PostProcessingImpl() {
  return true;
}


bool VotincevDRadixMergeSortOMP::RunImpl() {
  const auto &input = GetInput();
  auto n = static_cast<int32_t>(input.size());

  std::vector<uint32_t> working_array(static_cast<size_t>(n));
  int32_t min_val = input[0];

#pragma omp parallel for reduction(min : min_val) default(none) shared(n, input)
  for (int32_t i = 0; i < n; ++i) {
    min_val = std::min(input[i], min_val);
  }

#pragma omp parallel for default(none) shared(n, working_array, input, min_val)
  for (int32_t i = 0; i < n; ++i) {
    working_array[static_cast<size_t>(i)] = static_cast<uint32_t>(input[i]) - static_cast<uint32_t>(min_val);
  }

  std::vector<uint32_t> temp_buffer(static_cast<size_t>(n));

// статическое разделение массива
#pragma omp parallel default(none) shared(n, working_array, temp_buffer)
  {
    int tid = omp_get_thread_num();
    int n_threads = omp_get_num_threads();

    int32_t items = n / n_threads;
    int32_t rem = n % n_threads;
    int32_t l = (tid * items) + std::min(tid, rem);
    int32_t r = l + items + (tid < rem ? 1 : 0);

    if (l < r) {
      LocalRadixSort(working_array.data() + l, working_array.data() + r);
    }
// древовидное слияние
    for (int32_t step = 1; step < n_threads; step *= 2) {
#pragma omp barrier
      if ((tid % (2 * step) == 0) && (tid + step < n_threads)) {
        int32_t m = ((tid + step) * items) + std::min(tid + step, rem);
        int32_t next_tid = std::min(tid + (2 * step), n_threads);
        int32_t next_r = (next_tid * items) + std::min(next_tid, rem);

        Merge(working_array.data(), temp_buffer.data(), l, m, next_r);
        std::copy(temp_buffer.data() + l, temp_buffer.data() + next_r, working_array.data() + l);
      }
    }
  }

  std::vector<int32_t> result(static_cast<size_t>(n));
#pragma omp parallel for default(none) shared(n, result, working_array, min_val)
  for (int32_t i = 0; i < n; ++i) {
    result[static_cast<size_t>(i)] =
        static_cast<int32_t>(working_array[static_cast<size_t>(i)] + static_cast<uint32_t>(min_val));
  }

  GetOutput() = std::move(result);
  return true;
}

Пояснение:
Память контролируется с помощью default(none).
Локальные переменные tid, l, r внутри паралелльного блока - они private (для каждого потока своя переменная).
#pragma omp barrier:
внутри цикла step объявлен, гарантирует, что все потоки завершат локальную сортировку и предыдущий этап слияния.
(до того, как "сосед" будет объединять результаты)
