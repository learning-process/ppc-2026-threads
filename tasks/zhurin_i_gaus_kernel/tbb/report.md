## Введение

**Цель работы:** реализация параллельной фильтрации изображений с использованием библиотеки Intel oneTBB.

**Актуальность:** библиотека TBB предоставляет высокоуровневые средства автоматического распараллеливания
 вычислений и балансировки нагрузки между потоками, что позволяет эффективно использовать
 многоядерные процессоры при обработке изображений.

## Постановка задачи

Разработать программу, выполняющую свёртку целочисленного полутонового изображения с ядром Гаусса 3×3:

```cpp
static constexpr std::array<std::array<int, 3>, 3> kKernel = {{
    {1, 2, 1},
    {2, 4, 2},
    {1, 2, 1}
}};
static constexpr int kShift = 4;   // деление на 16
```

**Входные данные (`InType`):**

- ширина `width > 0`,
- высота `height > 0`,
- количество вертикальных полос `parts` (`1 ≤ parts ≤ width`),
- матрица пикселей `image[height][width]`.

**Выходные данные (`OutType`):**

- матрица того же размера, каждый пиксель — результат свёртки, поделённый на 16 (сдвиг вправо на 4 бита).

**Ограничения:**

- границы изображения обрабатываются нулевым дополнением (padding);
- выполняется проверка корректности размеров изображения и значения `parts`.

---

## Описание алгоритма

Алгоритм аналогичен последовательной версии, однако управление параллелизмом делегируется библиотеке Intel TBB.

Разбиение диапазонов выполняется с использованием tbb::blocked_range.

Балансировка нагрузки между worker-потоками осуществляется
автоматически средствами TBB runtime.

### Основные этапы

1. Создание расширенной матрицы `padded` размером `(height + 2) × (width + 2)` с нулевыми границами.
2. Копирование исходного изображения в центральную область `padded`.
3. Разбиение изображения на вертикальные полосы:
   - `base_width = width / parts`,
   - `remainder = width % parts`.
4. Параллельная обработка полос с использованием `tbb::parallel_for`.
5. Для каждого пикселя вычисляется свёртка с ядром Гаусса.
6. Результат нормализуется побитовым сдвигом:

```cpp
result = sum >> kShift;
```

Каждая полоса обрабатывается независимо, поэтому гонки данных отсутствуют.

## Детали реализации

**Файлы:**

- `tbb/include/ops_tbb.hpp`
- `tbb/src/ops_tbb.cpp`

### Ключевой вызов TBB

```cpp
tbb::parallel_for(tbb::blocked_range<int>(0, np),
    [&](const tbb::blocked_range<int> &range) {
        for (int part = range.begin(); part < range.end(); ++part) {
            ...
        }
});
```

### Особенности реализации

- используется библиотека `oneTBB`;
- применяется автоматическое распределение задач между потоками;
- свёртка развёрнута вручную для повышения производительности;
- распараллеливание выполняется по вертикальным полосам изображения;
- используется `tbb::blocked_range` для балансировки нагрузки.

### Валидация входных данных

Проверяются:

- положительные размеры изображения;
- корректность числа полос;
- соответствие размеров матрицы значениям `width` и `height`.
