## Постановка задачи

Разработать программу, выполняющую свёртку целочисленного полутонового изображения с ядром Гаусса 3×3:

```cpp
static constexpr std::array<std::array<int, 3>, 3> kKernel = {{
    {1, 2, 1},
    {2, 4, 2},
    {1, 2, 1}
}};
static constexpr int kShift = 4;   // деление на 16
```

**Входные данные (`InType`):**

- ширина `width > 0`,
- высота `height > 0`,
- количество вертикальных полос `parts` (`1 ≤ parts ≤ width`),
- матрица пикселей `image[height][width]`.

**Выходные данные (`OutType`):**

- матрица того же размера, каждый пиксель — результат свёртки, поделённый на 16 (сдвиг вправо на 4 бита).

**Ограничения:**

- границы изображения обрабатываются нулевым дополнением (padding);
- выполняется проверка корректности размеров изображения и значения `parts`.

---

## Описание алгоритма

Алгоритм идентичен последовательной версии, однако каждая вертикальная полоса обрабатывается в отдельном потоке.

Каждый поток обрабатывает собственную вертикальную полосу изображения.

После запуска всех потоков выполняется вызов join(),
обеспечивающий ожидание завершения вычислений.

### Основные этапы

1. Создание расширенной матрицы `padded` размером `(height + 2) × (width + 2)` с нулевыми границами.
2. Копирование исходного изображения в центральную область `padded`.
3. Разбиение изображения на вертикальные полосы:
   - `base_width = width / parts`,
   - `remainder = width % parts`.
4. Создание потоков для обработки полос изображения.
5. Выполнение свёртки для каждого пикселя.
6. Нормализация результата побитовым сдвигом:

```cpp
result = sum >> kShift;
```

7. Ожидание завершения всех потоков с помощью `join()`.

Каждый поток работает только со своей областью результирующей матрицы, поэтому гонки данных отсутствуют.

## Детали реализации

**Файлы:**

- `stl/include/ops_stl.hpp`
- `stl/src/ops_stl.cpp`

### Ключевой фрагмент кода

```cpp
std::vector<std::thread> threads;

for (int part = 0; part < num_parts_; ++part) {
    threads.emplace_back([&, part]() {
        /* обработка полосы */
    });
}

for (auto &t : threads) {
    t.join();
}
```

### Особенности реализации

- используется стандартная библиотека потоков C++;
- управление потоками выполняется вручную;
- каждая вертикальная полоса обрабатывается отдельным потоком;
- свёртка развёрнута вручную для повышения производительности;
- применяется нулевой padding.

### Валидация входных данных

Проверяются:

- положительные размеры изображения;
- корректность числа полос;
- соответствие размеров матрицы значениям `width` и `height`.
