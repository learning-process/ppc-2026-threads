## 1. Контекст

Последовательная версия фиксирует базовый алгоритм составной формулы средних прямоугольников для функции нескольких
переменных. От её корректности и времени напрямую зависят все последующие сравнения: именно SEQ выступает источником
эталонных значений интеграла и знаменателем в ускорениях и эффективности параллельных реализаций.

## 2. Постановка задачи

- **Вход** — кортеж `(f, bounds, n)`:
  - `f` — функция `std::function<double(const std::vector<double>&)>`, принимающая точку размерности `dim` и
    возвращающая значение в этой точке;
  - `bounds` — вектор пар `(a_i, b_i)` для каждой координаты, задающих границы интегрирования прямоугольной области;
  - `n` — число шагов разбиения, одинаковое по каждой координате.
- **Выход**: приближённое значение интеграла `I ≈ V * Σ f(x_cell)`, где `V` — объём элементарной ячейки.
- **Ограничения**: `n > 0`, `bounds` непуст, для каждой пары `a_i < b_i`, оба значения конечны (`std::isfinite`),
  `f` задана (не `nullptr` в обёртке `std::function`).
- **Краевые случаи**:
  - `dim = 1` — обычный одномерный интеграл;
  - произвольное `dim ≥ 1` (тесты охватывают `dim ∈ {1, 2, 3}`);
  - функции с разным числом операций (константа, линейная сумма, сумма квадратов, произведение).

## 3. Базовый алгоритм

Алгоритм составной формулы средних прямоугольников для `dim`-мерного интеграла:

1. Для каждой координаты `i` вычисляется шаг `h_i = (b_i - a_i) / n`.
2. Объём элементарной ячейки `V = ∏ h_i`.
3. Полное число ячеек `total = n^dim`. Каждой ячейке ставится в соответствие линейный индекс `cell ∈ [0, total)`.
4. По индексу `cell` восстанавливается `dim`-мерный мультииндекс `(k_0, ..., k_{dim-1})` базой `n`,
   а затем координата средней точки ячейки `x_i = a_i + (k_i + 0.5) * h_i`.
5. Суммируется `sum += f(x_cell)` по всем ячейкам.
6. Итог: `I = V * sum`.

**Сложность:**

- По времени: `O(n^dim * dim)` — основное вычисление, плюс одна вычислительно-дорогая операция (вызов `f`) на ячейку.
- По памяти: `O(dim)` — единственный буфер для текущей точки.

**Инварианты:**

- Каждая из `n^dim` ячеек обходится ровно один раз; средняя точка вычисляется детерминированно по `cell`.
- Порядок суммирования фиксирован, поэтому результат полностью воспроизводим.

## 4. Детали реализации

Файлы: [seq/include/ops_seq.hpp](include/ops_seq.hpp), [seq/src/ops_seq.cpp](src/ops_seq.cpp).

Класс `NazyrovAMultidimIntegralRectangleSeq` наследует `BaseTask = ppc::task::Task<InType, OutType>` и переопределяет
четыре стадии конвейера:

- `ValidationImpl` — `func != nullptr`, `n > 0`, `bounds` непуст, для каждой пары `a_i < b_i` и оба значения конечны.
- `PreProcessingImpl` — тривиальна: `true`. Расчёт шага и числа ячеек перенесён в `RunImpl`, чтобы не дублировать
  его в подготовке.
- `RunImpl` — последовательный обход всех ячеек и суммирование значений функции (см. листинг).
- `PostProcessingImpl` — тривиальна: `true`. Итог уже записан в `GetOutput()` внутри `RunImpl`.

Ключевой фрагмент — внутренний цикл по ячейкам и декодирование мультииндекса:

```cpp
// File: seq/src/ops_seq.cpp
double sum = 0.0;
std::vector<double> point(static_cast<std::size_t>(dim));

for (std::int64_t cell = 0; cell < total; ++cell) {
  std::int64_t tmp = cell;
  for (int i = dim - 1; i >= 0; --i) {
    const int ki = static_cast<int>(tmp % n);
    tmp /= n;
    const double coordinate = bounds[i].first + ((ki + 0.5) * h[i]);
    point[i] = coordinate;
  }
  sum += func(point);
}

GetOutput() = sum * cell_vol;
```

Раскодирование индекса делается «справа налево» по основанию `n`: это даёт устойчивый порядок обхода, который позже
позволяет параллельным версиям разбивать диапазон `[0, total)` на непересекающиеся куски без потерь по точкам.
Умножение на `cell_vol` вынесено наружу цикла, что устраняет один умножение на ячейку и заметно уменьшает влияние
накладных расходов `std::function` на общую сходимость к памяти.

## 5. Проверка корректности

Эталонные значения зашиты в [tests/functional/main.cpp](../tests/functional/main.cpp), общая функция `MakeTestDef`
формирует пары «функция → ожидаемый интеграл»:

- `f(x) = 1` на `[0, 1]^d`, ожидаемое значение `1`;
- `f(x) = Σ x_i` на `[0, 1]^d`, ожидаемое значение `d/2`;
- `f(x) = Σ x_i^2` на `[0, 1]^d`, ожидаемое значение `d/3`;
- `f(x) = ∏ x_i` на `[1, 2]^d`, ожидаемое значение `(3/2)^d`.

Сравнение выполняется по абсолютной ошибке с допуском `kTol = 1e-4`. Набор `kTestParams` покрывает 10 пар
`(func_id, dim, n)`, включая `dim = 1, 2, 3` и `n ∈ {50, 100, 200}` — это охватывает как «гладкие» сходимости
(`f = 1`), так и более дорогие функции с накоплением ошибки округления (`f = Σ x_i^2` на `n = 200, dim = 3`).

## 8. Выводы

Последовательная реализация даёт чистый, детерминированный эталон: один цикл по `n^dim` ячейкам, декодирование
мультииндекса по основанию `n`, единый порядок суммирования. Время `task_run = 1.184 с` принято за базу для расчёта
ускорений и эффективности в отчётах OMP, TBB, STL и ALL. На этом размере задачи (`27 M` ячеек) основным «узким местом»
является вызов подынтегральной функции через `std::function`, что делает задачу хорошим кандидатом для распараллеливания
по диапазону ячеек: работа равномерно распределяется по индексу, между итерациями нет зависимостей.
