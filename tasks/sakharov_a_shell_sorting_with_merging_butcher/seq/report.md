## 1. Контекст

Последовательная версия используется как базовая реализация задачи и как эталон для сравнения параллельных backend-ов.
Она выполняет сортировку в одном потоке и задает baseline для расчета ускорения OpenMP, oneTBB, STL и ALL.

## 2. Постановка задачи

На вход поступает массив целых чисел `std::vector<int>`. Нужно вернуть новый массив того же типа, содержащий те же
элементы в порядке неубывания. Корректными входами считаются пустой массив, массив из одного элемента, отрицательные
значения и повторяющиеся элементы.

## 3. Базовый алгоритм

В последовательной версии входной массив делится на две части. Каждая часть сортируется алгоритмом Шелла, после чего
выполняется четно-нечетное слияние Бэтчера: отдельно сливаются элементы с четными и нечетными индексами, затем соседние
элементы дополнительно сравниваются и при необходимости меняются местами. После слияния результат дополнительно проходит
финальную сортировку Шелла, чтобы гарантировать полностью упорядоченный выход для всех размеров входа.

## 4. Детали реализации

Файлы реализации:

- `seq/include/ops_seq.hpp`;
- `seq/src/ops_seq.cpp`.

Класс `SakharovAShellButcherSEQ` наследуется от `ppc::task::BaseTask` и возвращает тип задачи
`ppc::task::TypeOfTask::kSEQ`. Основная работа выполняется в `RunImpl`.

Минимальный листинг из `seq/src/ops_seq.cpp`:

```cpp
bool SakharovAShellButcherSEQ::RunImpl() {
  const auto &input = GetInput();

  if (input.empty()) {
    GetOutput().clear();
    return true;
  }

  GetOutput() = ShellSortWithButcherMerge(input);
  return true;
}
```

Фрагмент показывает общий поток выполнения: пустой вход обрабатывается отдельно, для остальных случаев вызывается
функция сортировки с четно-нечетным слиянием. В версии SEQ нет потоков, блокировок и межпроцессного обмена.

## 5. Проверка корректности

Корректность проверяется общими функциональными тестами из `tests/functional/main.cpp`. Для каждого теста ожидаемый
результат строится с помощью `std::ranges::sort`, после чего сравнивается с результатом `SakharovAShellButcherSEQ`.

Проверяются пять групп входов: четный размер, нечетный размер с отрицательными числами, повторяющиеся значения, один
элемент и пустой массив. По результатам локального запуска все 5 функциональных тестов SEQ прошли успешно.
