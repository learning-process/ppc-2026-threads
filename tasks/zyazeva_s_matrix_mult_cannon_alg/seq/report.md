## 1. Контекст

Задача — произведение двух квадратных матриц `A` и `B`
размера `N×N`.

Последовательная версия служит вычислительным эталоном,
относительно которого рассчитывается ускорение
всех параллельных реализаций.

## 2. Постановка задачи

**Входные данные**
(`InType = std::tuple<size_t, std::vector<double>, std::vector<double>>`):

- `sz` — сторона квадратной матрицы (тип `size_t`);
- `m1` — матрица `A`, хранимая построчно
  в одномерном `std::vector<double>`
  размером `sz * sz`;
- `m2` — матрица `B`, аналогично.

**Выходные данные** (`OutType = std::vector<double>`):

результирующая матрица `C = A·B`,
хранимая построчно,
размер `sz * sz`.

**Ограничения:**

- `sz > 0` —
  нулевой размер считается недопустимым;
- `m1.size() == sz * sz`
  и `m2.size() == sz * sz` —
  несоответствие размеров блокируется
  в `ValidationImpl`;
- элементы имеют тип `double`,
  переполнение не рассматривается.

**Крайние случаи:**

- матрица `1×1` —
  корректно обрабатывается основным циклом;
- передача пустых векторов
  (`sz > 0`, но `size == 0`) —
  отклоняется валидацией.

## 3. Базовый алгоритм

Используется классический тройной цикл: для каждой позиции `(i, j)`
результирующей матрицы вычисляется скалярное произведение
`i`-й строки `A` и `j`-го столбца `B`.

```text
для row от 0 до sz - 1:
  для col от 0 до sz - 1:
    accumulated = 0

    для inner от 0 до sz - 1:
      accumulated +=
          A[row * sz + inner] *
          B[inner * sz + col]

    C[row * sz + col] = accumulated
````

Асимптотика по времени:

```text
O(N³)
```

Асимптотика по памяти:

```text
O(N²)
```

Дополнительная память требуется
только для хранения результирующей матрицы.

Инвариант алгоритма:

после завершения `RunImpl` каждый элемент `C[i][j]` равен точному скалярному произведению
соответствующих строки и столбца.

## 4. Детали реализации

**Файлы:**

- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

Ключевой фрагмент `RunImpl`:

```cpp
// File: seq/src/ops_seq.cpp
std::vector<double> res_m(sz * sz, 0.0);

for (size_t row_idx = 0; row_idx < sz; ++row_idx) {
  for (size_t col_idx = 0; col_idx < sz; ++col_idx) {
    double accumulated = 0.0;

    for (size_t inner_idx = 0; inner_idx < sz; ++inner_idx) {
      accumulated +=
          m1[(row_idx * sz) + inner_idx] *
          m2[(inner_idx * sz) + col_idx];
    }

    res_m[(row_idx * sz) + col_idx] = accumulated;
  }
}
```

**Этапы пайплайна:**

- `ValidationImpl` —
  проверяет `sz > 0`,
  `m1.size() == sz * sz`,
  `m2.size() == sz * sz`;
- `PreProcessingImpl` —
  сбрасывает выходной вектор
  в пустое состояние;
- `RunImpl` —
  выполняет основной цикл
  и записывает результат
  в `GetOutput()`;
- `PostProcessingImpl` —
  ничего не делает
  и возвращает `true`.

## 5. Проверка корректности

Корректность SEQ-версии проверяется
функциональными тестами
из `tests/functional/main.cpp`.

Результат сравнивается с вычисленным вручную
или другим верифицированным эталоном.

Минимальный набор тестов включает:

- единичные матрицы:
  `I·A = A`,
  `A·I = A`;
- нулевые матрицы:
  `0·A = 0`.

Дополнительно проверяются:

- матрица `1×1` — произведение двух скаляров;
- матрица `2×2` — результат сверяется аналитически;
- матрица произвольного размера — сравнение с контрольным вычислением.

## 8. Выводы

SEQ-версия реализует классический тройной цикл без дополнительных оптимизаций, что делает её предсказуемым
и понятным вычислительным эталоном.

Сложность `O(N³)` и последовательный, детерминированный порядок суммирования
исключают нестабильность результата.

Именно поэтому данная реализация используется как baseline: отсутствие специфичных для конкретной технологии
оптимизаций позволяет корректно оценивать прирост производительности параллельных версий.
