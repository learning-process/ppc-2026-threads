## 1. Контекст

Гибридная реализация объединяет два уровня параллелизма: межпроцессный (MPI) и внутрипроцессный (OpenMP). Такой подход
позволяет эффективно использовать ресурсы кластерных систем, где узлы объединены сетью, а внутри каждого узла находятся
многоядерные процессоры с общей памятью.

## 2. Постановка задачи

Краткая постановка: решение СЛАУ $Ax = b$ методом сопряженных градиентов. Сравнение производится с последовательным
эталоном `seq` ($T \approx 0.128$ с). Основная цель — оценить накладные расходы на пересылки данных между MPI-рангами
при сохранении многопоточности внутри каждого ранга.

## 3. Базовый алгоритм

Алгоритм следует математической схеме метода CG. В гибридной версии данные (векторы $r, p, x$) распределяются между
процессами. Каждый ранг отвечает за вычисление своего блока данных, а глобальные скалярные величины и векторы
направлений синхронизируются через вызовы MPI.

## 4. Межпроцессная схема (MPI)

Реализована схема **Master-Worker**:

- **Rank 0**: Читает входные данные, выполняет валидацию и распределяет их через `MPI_Bcast`.
- **Распределение нагрузки**: Каждому процессу назначается диапазон строк (блок) матрицы и соответствующих элементов
  векторов. Индексы вычисляются функцией `start` / `end` с учетом остатка от деления $N$ на количество процессов.
- **Коммуникации**:
    - `MPI_Allreduce`: Используется для вычисления глобальных скалярных произведений (сумма локальных частей от каждого
      ранга).
    - `MPI_Allgatherv`: Используется в конце каждой итерации для полной синхронизации вектора направлений $p$ между
      всеми процессами.
    - `MPI_Gatherv`: Финальный сбор вектора решения на Rank 0.

## 5. Внутрипроцессная схема (OpenMP)

Внутри каждого MPI-процесса вычисления распараллеливаются с помощью OpenMP. Директивы `#pragma omp parallel for`
применяются к локальным диапазонам строк, за которые отвечает данный ранг. Это позволяет задействовать все ядра
процессора, выделенные на один процесс.

## 6. Детали реализации

Файлы: `all/include/ops_all.hpp`, `all/src/ops_all.cpp`.

**Листинг ключевого фрагмента (Гибридное скалярное произведение):**

```cpp
// File: all/src/ops_all.cpp
double local_sum = 0.0;
#pragma omp parallel for default(none) shared(start, end, v1, v2) reduction(+ : local_sum)
for (int i = start; i < end; ++i) {
    local_sum += v1[i] * v2[i];
}
double global_sum = local_sum;
if (is_mpi) {
    MPI_Allreduce(&local_sum, &global_sum, 1, MPI_DOUBLE, MPI_SUM, MPI_COMM_WORLD);
}
return global_sum;
```

_Пояснение:_ Сначала потоки OpenMP вычисляют частичную сумму в рамках своего процесса, затем `MPI_Allreduce` объединяет
эти суммы в глобальное значение, доступное всем процессам.

## 10. Выводы

Гибридная схема оправдана при решении задач сверхбольшой размерности, не помещающихся в память одного узла. Для данной
задачи ($3000 \times 3000$) на одном физическом процессоре оптимальным является минимальное количество MPI-процессов при
максимальном количестве потоков OpenMP, так как внутрипроцессная синхронизация через общую память значительно быстрее
межпроцессного обмена сообщениями.
