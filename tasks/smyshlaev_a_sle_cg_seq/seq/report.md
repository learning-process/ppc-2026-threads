## 1. Контекст

Метод сопряженных градиентов это эффективный итерационный алгоритм решения систем линейных алгебраических уравнений
(СЛАУ) вида $Ax = b$. Цель данной реализации — создать строгий, алгоритмически корректный последовательный эталон,
который будет использоваться для проверки точности вычислений и оценки ускорения параллельных версий (OMP, TBB, STL,
ALL).

## 2. Постановка задачи

**Входные данные:**

- Квадратная матрица $A$ размером $N \times N$.
- Вектор свободных членов $b$ размером $N$.

**Выходные данные:**

- Вектор неизвестных $x$ размером $N$, являющийся решением системы.

**Ограничения и краевые случаи:**

- Матрица $A$ и вектор $b$ не должны быть пустыми ($N > 0$).
- Размеры должны быть согласованы: количество строк и столбцов $A$ равно $N$, размер $b$ равен $N$.
- Алгоритм прерывается досрочно, если норма невязки становится меньше заданного $\epsilon = 10^{-9}$ или скалярное
  произведение стремится к нулю.

## 3. Базовый алгоритм

Алгоритм начинает работу с начального приближения $x_0 = 0$. На каждом шаге $k$ алгоритм вычисляет вектор направления
$p_k$ и обновляет решение $x_{k+1} = x_k + \alpha_k p_k$, минимизируя квадратичный функционал.

**Пошаговое описание итерации:**

1. Вычисляется умножение матрицы на вектор: $Ap = A \times p$.
2. Вычисляется шаг $\alpha = \frac{(r, r)}{(p, Ap)}$.
3. Обновляется текущее решение: $x = x + \alpha p$.
4. Обновляется вектор невязки: $r = r - \alpha Ap$.
5. Вычисляется коэффициент $\beta = \frac{(r_{new}, r_{new})}{(r_{old}, r_{old})}$.
6. Обновляется вектор направления: $p = r + \beta p$.

**Асимптотика:**

- По времени: $O(K \cdot N^2)$, где $N$ — размерность матрицы, а $K$ — количество итераций (максимум $2N$). Внутри
  каждой итерации самое тяжелое вычисление — умножение матрицы на вектор $O(N^2)$.
- По памяти: $O(N^2)$ для хранения матрицы $A$ и $O(N)$ для векторов $x, r, p, Ap$.

**Инварианты корректности:** На каждой итерации вектор невязки должен строго соответствовать формуле $r_k = b - Ax_k$.

## 4. Детали реализации

Файлы реализации: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.

Реализация разделена на четыре этапа жизненного цикла задачи (`BaseTask`):

- `ValidationImpl`: Строго проверяет совпадение размерностей матрицы $A$ и вектора $b$.
- `PreProcessingImpl`: Выполняет оптимизацию локальности кэша процессора. Двумерная матрица
  `std::vector<std::vector<double>>` «выпрямляется» в одномерный массив `flat_A_`. Это критически важно для
  производительности $O(N^2)$ операций умножения, чтобы избежать промахов кэша.
- `RunImpl`: Выполняет основной итерационный процесс сопряженных градиентов.
- `PostProcessingImpl`: Возвращает готовый вектор результата.

**Листинг ключевого фрагмента (Итерационный процесс из `RunImpl`):**

```cpp
// File: seq/src/ops_seq.cpp
for (int iter = 0; iter < max_iterations; ++iter) {
  ComputeAp(flat_A_, p, ap, n);

  double p_ap = ComputeDotProduct(p, ap);
  if (std::abs(p_ap) < 1e-15) break; // Защита от деления на ноль

  double alpha = rs_old / p_ap;
  double rs_new = UpdateResultAndResidual(result, r, p, ap, alpha);

  if (std::sqrt(rs_new) < epsilon) break; // Достигнута требуемая точность

  double beta = rs_new / rs_old;
  UpdateP(p, r, beta);

  rs_old = rs_new;
}
```

_Пояснение к коду:_ Вычислительное ядро разбито на вспомогательные функции (например, `ComputeAp`, `ComputeDotProduct`).
Использование «выпрямленной» `flat_A_` внутри `ComputeAp` обеспечивает последовательное чтение памяти. В цикле
предусмотрены явные точки раннего выхода (`break`), если достигнута машинная точность, что предотвращает лишние
итерации.

## 5. Проверка корректности

Корректность алгоритма проверялась через фреймворк функциональных тестов (`tests/functional/main.cpp`). В качестве
эталонных кейсов использовались:

1. `Simple2x2`: Простая матрица $2 \times 2$ с очевидным целочисленным ответом.
2. `Medium3x3`: Система $3 \times 3$ с диагональным преобладанием.
3. `IdentityMatrix4x4`: Единичная матрица (решение должно в точности совпадать с вектором $b$).
4. `Harder2x2`: Матрица с дробными корнями. Сравнение вещественных чисел производилось с допуском $\epsilon = 10^{-6}$,
   чтобы исключить ошибки накопления при операциях с плавающей точкой.
