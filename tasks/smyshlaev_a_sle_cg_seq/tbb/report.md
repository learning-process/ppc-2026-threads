## 1. Контекст

Данный отчет описывает реализацию метода сопряженных градиентов с использованием библиотеки TBB. В отличие от
потоково-ориентированного подхода, TBB использует концепцию задач, что позволяет более гибко распределять нагрузку на
современных многоядерных процессорах с неоднородной архитектурой.

## 2. Постановка задачи

Краткая постановка: нахождение вектора $x$ для системы $Ax = b$. Последовательная версия из `seq/report.md` принимается
за эталон с временем выполнения $T_{seq} \approx 0.128$ с.

## 3. Базовый алгоритм

Алгоритм основан на итерационном процессе поиска минимума квадратичной формы. Основные операции включают умножение
разреженной или плотной матрицы на вектор, вычисление скалярных произведений и обновление векторов. В данной реализации
используется плотное представление матрицы, оптимизированное под кэш-линии.

## 4. Схема распараллеливания

Распараллеливание реализовано через высокоуровневые примитивы TBB:

- `tbb::parallel_for`: Применяется для умножения матрицы на вектор (`ComputeAp`) и обновления векторов (`UpdateP`).
  Позволяет автоматически разбивать диапазон строк на подзадачи.
- `tbb::parallel_reduce`: Используется для вычисления скалярных произведений векторов и суммы квадратов невязки. Это
  позволяет избежать явных блокировок (mutex) при накоплении суммы.
- `tbb::blocked_range<size_t>`: Задает диапазон итераций.
- **Grainsize**: Использован автоматический выбор размера порции данных (partitioner по умолчанию), что позволяет
  библиотеке TBB эффективно балансировать нагрузку.

**Контроль конкуренции:** Число работающих потоков (workers) ограничивается внешней инфраструктурой курса через
настройки runtime TBB, соответствующие значению `PPC_NUM_THREADS`.

## 5. Детали реализации

Файлы: `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`. В реализации критически важные функции вынесены в анонимный
namespace для инкапсуляции логики.

**Листинг ключевого фрагмента (Параллельная редукция):**

```cpp
// File: tbb/src/ops_tbb.cpp
double rs_new = tbb::parallel_reduce(
    tbb::blocked_range<size_t>(0, result.size()), 0.0,
    [&](const tbb::blocked_range<size_t> &range, double init) {
        for (size_t i = range.begin(); i != range.end(); ++i) {
            result[i] += alpha * p[i];
            r[i] -= alpha * ap[i];
            init += r[i] * r[i];
        }
        return init;
    }, std::plus<double>());
```

_Пояснение:_ Здесь `parallel_reduce` одновременно обновляет векторы `result` и `r` (операция записи в независимые
области памяти) и накапливает локальную сумму квадратов невязки в переменную `init`. Финальное объединение локальных
сумм происходит через функтор `std::plus<double>()`.

связано с механизмом **work-stealing**, который позволяет потокам на производительных P-ядрах забирать задачи у более
медленных E-ядер, обеспечивая более равномерную загрузку процессора i7-1255U.
