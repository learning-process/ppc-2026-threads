## 1. Контекст

В данном разделе рассматривается последовательная реализация поразрядной сортировки (LSD Radix Sort) целых
чисел. Эта версия необходима в качестве эталона корректности и базовой точки отсчета времени выполнения для
последующего вычисления ускорения и эффективности параллельных реализаций (OMP, TBB, STL, ALL).

## 2. Постановка задачи

Задача состоит в сортировке массива 32-битных знаковых целых чисел по неубыванию.

- **Входные данные (`InType`)**: Вектор целых чисел `std::vector<int>`.
- **Выходные данные (`OutType`)**: Вектор целых чисел `std::vector<int>`, отсортированный по неубыванию.
- **Ограничения**: Входной вектор не должен быть пустым.
- **Краевые случаи**: Наличие отрицательных чисел, массив из одинаковых элементов, уже отсортированный массив
  или массив, отсортированный в обратном порядке.

## 3. Базовый алгоритм

Используется поразрядная сортировка (LSD Radix Sort), выполняющая проходы от младшего разряда к старшему. В
качестве разряда выбран 1 байт (8 бит), что дает 256 корзин (`kBuckets = 256`). Для каждого байта применяется
устойчивая сортировка подсчетом (Counting Sort).

Поскольку поразрядная сортировка напрямую не работает с отрицательными числами, перед сортировкой алгоритм
находит минимальный элемент массива `min_val`. Если `min_val < 0`, вычисляется смещение `offset = -min_val`.
Во время распределения по корзинам это смещение прибавляется к каждому числу, делая все обрабатываемые
значения неотрицательными.

- **Асимптотика по времени**: $O(d \cdot (N + K))$, где $d=4$ (количество байт в 32-битном числе), $N$ —
  количество элементов, $K=256$ — количество корзин. Итоговая временная сложность составляет $O(N)$.
- **Асимптотика по памяти**: $O(N + K)$ для дополнительного буфера `output` размером $N$ и массива счетчиков
  размером $K$.
- **Инварианты и критерий корректности**: Размер выходного массива должен быть равен размеру входного массива.
  Выходной массив строго отсортирован по неубыванию ($a_i \le a_{i+1}$).

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

Пайплайн задачи реализован через переопределение методов базового класса:

- `ValidationImpl()`: Проверяет, что входной вектор не пуст (`!GetInput().empty()`). Блокирует запуск на
  пустых данных.
- `PreProcessingImpl()`: Копирует входные данные в выходной буфер (`GetOutput() = GetInput()`).
- `RunImpl()`: Вызывает основную логику сортировки `RadixSort(GetOutput())`.
- `PostProcessingImpl()`: Возвращает результат проверки массива на отсортированность с помощью
  `std::ranges::is_sorted(GetOutput())`.

Фрагмент основной логики (обработка проходов по разрядам):

```cpp
// File: seq/src/ops_seq.cpp
  int offset = -min_val;
  for (int exp = 1; (max_val + offset) / exp > 0 || (min_val + offset) / exp > 0; exp <<= kBitsPerDigit) {
    CountingSort(data, exp, offset);
  }
```

Этот код гарантирует, что количество проходов сортировки подсчетом будет минимально необходимым для покрытия
диапазона значений в массиве с учетом смещения для отрицательных чисел.

## 5. Проверка корректности

- **Как сравнивается результат**: Во время тестирования результат работы `RunImpl` проверяется вызовом
  `PostProcessingImpl`, а также сравнивается с эталонной функцией стандартной библиотеки `std::sort` в рамках
  общего тестового конвейера (`tests/functional/main.cpp`).
- **Тесты**: Использованы детерминированные тесты на случайных массивах, массивах, отсортированных в прямом и
  обратном порядке, массивах из одинаковых элементов, а также на массивах, состоящих только из положительных
  или только из отрицательных чисел.
- **Характерный пример**:
  - Вход: `[-5, 10, 0, -5, 3]`
  - Ожидаемый выход: `[-5, -5, 0, 3, 10]`
