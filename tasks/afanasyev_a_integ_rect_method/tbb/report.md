## 1. Контекст

Переносится вычисление трёхмерного интеграла методом прямоугольников из последовательной версии в TBB

## 2. Постановка задачи

Вычисление трёхмерного интеграла функции exp(-(x^2 + y^2 + z^2)) по единичному кубу методом прямоугольников

## 3. Базовый алгоритм (SEQ)

1 - Инициализация
   1.1 Получаем n из входных данных
   1.2 Вычисляем шаг сетки h = 1.0 / n
   1.3 Создаем массив индексов idx[k_dim] = {0, 0, 0}
   1.4 Создаем массив координат x[k_dim] = {0, 0, 0}
   1.5 Обнуляем сумму sum = 0.0
2 - Обход всех узлов сетки
   2.1 Для текущего набора индексов вычисляем координаты центров ячеек x[dim] = (idx[dim] + 0.5) × h
   2.2 Вычисляем значение подынтегральной функции exp(-(x^2 + y^2 + z^2))
   2.3 Значение добавляем к сумме
   2.4 Инкрементируем индексы с переносом разряда
     2.4.1 Увеличивается idx[0]
     2.4.2 Если idx[0] == n, сброс в 0 и увеличение idx[1]
     2.4.3 Если idx[1] == n, сброс в 0 и увеличение idx[2]
     2.4.4 Если idx[2] == n, цикл завершается
3 - Получение финального результата
   3.1 Объём ячейки volume = h^3
   3.2 Результат result = sum × volume

## 4. Схема распараллеливания (TBB)

Вся итеративная часть распараллеливается с использованием библиотеки Intel TBB. В отличие от SEQ и
STL, в TBB используются высокоуровневые абстракции:

- tbb::parallel_reduce — автоматическое параллельное выполнение с редукцией
- tbb::blocked_range — автоматическое разбиение диапазона на блоки
- линеаризация 3D: линейный индекс задаётся как `i*n*n + j*n + k`
- автоматическая балансировка — TBB динамически распределяет блоки между потоками

Все n^3 итерации представляются как один непрерывный диапазон: каждый поток обрабатывает свой блок,
возвращает локальную сумму, а TBB суммирует результаты.

## 5. Детали реализации

tbb/include/ops_tbb.hpp — объявление класса AfanasyevAIntegRectMethodTBB
tbb/src/ops_tbb.cpp — реализация

```cpp
const int64_t total_points = static_cast<int64_t>(n) * n * n;

const double sum = tbb::parallel_reduce(
    tbb::blocked_range<int64_t>(0, total_points), 0.0,
    [&](const tbb::blocked_range<int64_t> &range, double local_sum) {
        std::array<double, 3> x{};
        const int64_t plane = static_cast<int64_t>(n) * n;
        for (int64_t index = range.begin(); index != range.end(); ++index) {
            const int i = static_cast<int>(index / plane);
            const int j = static_cast<int>((index / n) % n);
            const int k = static_cast<int>(index % n);

            x[0] = (static_cast<double>(i) + 0.5) * h;
            x[1] = (static_cast<double>(j) + 0.5) * h;
            x[2] = (static_cast<double>(k) + 0.5) * h;

            local_sum += ExampleIntegrand(x);
        }
        return local_sum;
    },
    std::plus<>()
);
```

Устранены сразу несколько рисков гонки:
1 - Гонка sum — устранена через parallel_reduce с автоматической редукцией
2 - Гонка x — `std::array<double, 3> x` создаётся внутри лямбды, приватная для каждого потока
3 - Гонка индексов — i, j, k вычисляются из локального индекса, приватны для каждой итерации
4 - Автоматическая синхронизация — TBB управляет синхронизацией без явных блокировок

Результат идентичен последовательной версии, включая граничные случаи: проверка n > 0 в
