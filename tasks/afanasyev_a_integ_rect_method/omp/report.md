## 1. Контекст

Переносится вычисление трёхмерного интеграла методом прямоугольников из последовательной версии в OpenMP

## 2. Постановка задачи

Вычисление трёхмерного интеграла функции exp(-(x^2 + y^2 + z^2)) по единичному кубу методом прямоугольников

## 3. Базовый алгоритм (SEQ)

1 - Инициализация
   1.1 Получаем n из входных данных
   1.2 Вычисляем шаг сетки h = 1.0 / n
   1.3 Создаем массив индексов idx[k_dim] = {0, 0, 0}
   1.4 Создаем массив координат x[k_dim] = {0, 0, 0}
   1.5 Обнуляем сумму sum = 0.0
2 - Обход всех узлов сетки
   2.1 Для текущего набора индексов вычисляем координаты центров ячеек x[dim] = (idx[dim] + 0.5) * h
   2.2 Вычисляем значение подынтегральной функции exp(-(x^2 + y^2 + z^2))
   2.3 Значение добавляем к сумме
   2.4 Инкрементируем индексы с переносом разряда
     2.4.1 Увеличивается idx[0]
     2.4.2 Если idx[0] == n, сброс в 0 и увеличение idx[1]
     2.4.3 Если idx[1] == n, сброс в 0 и увеличение idx[2]
     2.4.4 Если idx[2] == n, цикл завершается
3 - Получение финального результата
   3.1 Объём ячейки volume = h^3
   3.2 Результат result = sum * volume

## 4. Схема распараллеливания (OMP)

Вся итеративная часть распараллеливается. В отличие от SEQ, в OMP используются явные вложенные циклы for (i) for (j) for (k) для естественного распараллеливания, для синхронизции используюися: reduction(+ : sum) - автоматически создаёт приватные копии sum в каждом потоке и выполняет финальное сложение и collapse - объединяет тройной цикл в один логический
n, h, k_dim - shared
i, j, k - private
sum - reduction

## 5. Детали реализации

omp/include/ops_omp.hpp — объявление класса AfanasyevAIntegRectMethodOMP
omp/src/ops_omp.cpp — реализация

```cpp
#pragma omp parallel for collapse(3) reduction(+ : sum) default(none) shared(n, h, k_dim)
for (int i = 0; i < n; ++i) {
  for (int j = 0; j < n; ++j) {
    for (int k = 0; k < n; ++k) {
      std::vector<double> x = {(static_cast<double>(i) + 0.5) * h,
                               (static_cast<double>(j) + 0.5) * h,
                               (static_cast<double>(k) + 0.5) * h};
      sum += ExampleIntegrand(x);
    }
  }
}
```

Устранены сразу несколько рисков гонки:
1 - Гонка sum — устранена reduction(+)
2 - Гонка x — каждый поток создаёт собственный std::vector<double> x, гарантированно private
3 - Гонка индексов** — i, j, k приватны благодаря parallel for
