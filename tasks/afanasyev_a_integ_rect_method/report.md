## 1. Введение

Вычисление многомерных интегралов — одна из фундаментальных задач вычислительной математики. Метод
прямоугольников — простейший подход к численному интегрированию, но при переходе к трём измерениям
объём вычислений растёт кубически. Это делает задачу удобным примером для оценки эффективности
разных технологий параллельного программирования. Независимые вычисления в каждой ячейке позволяют
наглядно увидеть влияние накладных расходов на синхронизацию, распределение работы и управление
памятью.

## 2. Постановка задачи

Подынтегральная функция - exp(-(x^2 + y^2 + z^2))
Область интегрирования - Единичный куб [0, 1] × [0, 1] × [0, 1]
Метод - Прямоугольников (точка в центре ячейки)
Вход - Целое число n > 0 — количество разбиений по каждому измерению
Выход - Приближённое значение интеграла
Объём вычислений - n^3 итераций
Критерий корректности - Совпадение результата с последовательной версией

## 3. Единая методика эксперимента

CPU - Apple Silicon M1 pro
RAM - 16 ГБ
OS - macOS Tahoe 26.0.1
Компилятор - Clang
Размер задачи - 8 000 000 итераций, n = 200
Команды запуска:

Обычный запуск — `.\ppc_perf_tests --gtest_filter=*afanasyev_a_integ_rect_method*`

Запуск с потоками — `mpirun -n N .\ppc_perf_tests --gtest_filter=*afanasyev_a_integ_rect_method*`

(подставьте число процессов вместо N).
Формулы - Ускорение = Time(seq) / Time(parallel) Эффективность = Ускорение / кол-во потоков
Измерения - Приводятся медианные показатели для режимов task_run и task_pipeline

## 4. Сводка корректности

Все реализации (SEQ, OMP, TBB, STL, ALL) прошли тесты корректности фреймворка ppc_func_tests. На
граничных случаях n=0, n=1 алгоритм не зависает и выдаёт корректное приближение интеграла.
Отклонений в выходных результатах не зафиксировано.

## 6. Интерпретация различий

SEQ — установил хороший базовый результат (0.0217 с). Тройной вложенный цикл доминирует по времени.
Компилятор не смог полностью векторизовать последовательную версию.

OMP — на 1 потоке показала сильный результат благодаря collapse(3), который позволил компилятору
агрессивно оптимизировать и векторизовать тройной цикл. Уже на 2 потоках начинается деградация из‑за
накладных расходов на синхронизацию и распределение итераций. К 8 потокам время выросло до 0.0356 с
(в 5.3 раза хуже, чем на 1 потоке).

TBB — единственная технология с монотонным ускорением при росте числа потоков (от 0.0308 с на 1 до
0.0115 с на 8). Это достигается линеаризацией 3D-пространства, parallel_reduce с редукцией и
work-stealing планировщиком.

STL — худшие результаты: ручное разбиение по первому измерению и создание std::vector на итерацию,
накладные расходы на потоки и синхронизацию перекрыли выигрыш уже при 2 потоках.

ALL (STL + TBB) — наилучшая стабильность: время почти не меняется с ростом числа потоков за счёт
двухуровневой декомпозиции, ограничения избыточного распараллеливания и умеренных накладных расходов.

## 7. Заключение

Наилучшей параллельной версией для трёхмерного интеграла на одной машине при n = 200 считаю ALL:
на малом числе потоков OMP сильно быстрее, но с ростом числа потоков ALL демонстрирует стабильность и
конкурентную скорость.
