## 2. Постановка задачи

Вычисление трёхмерного интеграла функции exp(-(x^2 + y^2 + z^2)) по единичному кубу методом прямоугольников

## 3. Базовый алгоритм (SEQ)

1 - Инициализация
   1.1 Получаем n из входных данных
   1.2 Вычисляем шаг сетки h = 1.0 / n
   1.3 Создаем массив индексов idx[k_dim] = {0, 0, 0}
   1.4 Создаем массив координат x[k_dim] = {0, 0, 0}
   1.5 Обнуляем сумму sum = 0.0
2 - Обход всех узлов сетки
   2.1 Для текущего набора индексов вычисляем координаты центров ячеек x[dim] = (idx[dim] + 0.5) × h
   2.2 Вычисляем значение подынтегральной функции exp(-(x^2 + y^2 + z^2))
   2.3 Значение добавляем к сумме
   2.4 Инкрементируем индексы с переносом разряда
     2.4.1 Увеличивается idx[0]
     2.4.2 Если idx[0] == n, сброс в 0 и увеличение idx[1]
     2.4.3 Если idx[1] == n, сброс в 0 и увеличение idx[2]
     2.4.4 Если idx[2] == n, цикл завершается
3 - Получение финального результата
   3.1 Объём ячейки volume = h^3
   3.2 Результат result = sum × volume

## 4. Схема распараллеливания (STL)

Вся итеративная часть распараллеливается. В отличие от SEQ, в STL используется явное разделение работы
между потоками. Для синхронизации используются:

- ручное разделение итераций по первому измерению между потоками
- std::thread — создание пула потоков
- std::ref — передача ссылок на partial_sums в потоки
- std::accumulate — финальное суммирование частичных результатов

Каждый поток получает диапазон итераций по первому измерению, затем выполняет вложенные циклы по
остальным измерениям, накапливает сумму в локальной переменной и записывает результат в свой элемент
массива partial_sums.

```cpp
void ComputePartialSum(int start, int end, int n, int k_dim, double h, double &local_sum) {
  std::vector<int> idx(k_dim, 0);
  std::vector<double> x(k_dim, 0.0);
  local_sum = 0.0;

  for (int i0 = start; i0 < end; ++i0) {
    idx[0] = i0;
    for (int i1 = 0; i1 < n; ++i1) {
      idx[1] = i1;
      for (int i2 = 0; i2 < n; ++i2) {
        idx[2] = i2;
        for (int dim = 0; dim < k_dim; ++dim) {
          x[dim] = (static_cast<double>(idx[dim]) + 0.5) * h;

      }
    }
  }
}
```

Устранены сразу несколько рисков гонки:
1 - Гонка sum — устранена через массив partial_sums, каждый поток записывает в свой уникальный элемент
2 - Гонка idx — каждый поток создаёт собственный `std::vector<int> idx`
3 - Гонка x — каждый поток создаёт собственный `std::vector<double> x`
4 - Гонка local_sum — локальная переменная, приватная для каждого потока
