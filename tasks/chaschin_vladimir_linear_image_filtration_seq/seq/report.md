    Горизонтальное разбиение означает,
    что независимой единицей работы является строка изображения.Каждый проход записывает разные строки выходного буфера,
    поэтому синхронизация между рабочими потоками внутри одного прохода не требуется;
барьер нужен только между горизонтальным и вертикальным проходами,
    так как второй проход читает временный массив целиком.

    ##2. Постановка задачи

        **Вход : **кортеж `(image, width, height)`,
    где `image` — вектор яркостей пикселей типа `float`, `width` — ширина изображения, `height` — высота изображения
                                                                                           .

                                                                                               **Выход
    : **вектор `float` размера `width *height` со сглаженным изображением.

          **Ограничения и корректность входа : **

                                               -изображение не должно быть пустым;

- ожидается согласованность размеров: `image.size() == width *height`;
- для корректного обращения к соседям в текущей реализации практически предполагаются `width >= 2` и `height >= 2`;
- пиксели хранятся в row -
    major порядке:элемент `(x, y)` находится по индексу `y *width + x`.

                                                                    ##3. Базовый алгоритм

                                                                    1. Выделить временный
                                                                    массив `temp` размера `width *height`.2. Для каждой
                                                                    строки выполнить горизонтальную
          свертку:-левый
             край: `(2 * p[x] + p[x + 1]) / 3`;
- правый край: `(p[x - 1] + 2 * p[x]) / 3`;
- внутренний пиксель: `(p[x - 1] + 2 * p[x] + p[x + 1]) / 4`.3. Для каждой строки результирующего массива выполнить
                                                          вертикальную свертку по `temp
                   `:-верхний
                край: `(2 * temp[y] + temp[y + 1]) / 3`;
- нижний край: `(temp[y - 1] + 2 * temp[y]) / 3`;
- внутренний пиксель: `(temp[y - 1] + 2 * temp[y] + temp[y + 1]) / 4`.

1. Вернуть заполненный выходной массив.

**Сложность:** `O(width * height)`, так как каждый пиксель обрабатывается константное число раз.  
**Память:** `O(width * height)` для временного массива `temp` и `O(width * height)` для выходного массива.

## 4. Детали реализации

Последовательная версия выполняет оба прохода обычными вложенными циклами по строкам и столбцам.
Функция `HorizontalFilterAt` вычисляет значение временного буфера для одного пикселя горизонтального прохода,
а `VerticalFilterAt` вычисляет итоговое значение по временным данным.
Разделение на две небольшие inline-функции делает обработку границ
одинаковой для всех пикселей и упрощает сравнение с параллельными версиями.

Порядок обхода строк сохраняет локальность доступа к памяти в горизонтальном проходе.
В вертикальном проходе результат также записывается построчно,
а чтение соседних строк из `temp` остается регулярным и предсказуемым для кэша.

## 5. Проверка корректности

Корректность проверяется функциональными тестами Google Test.
Тесты строят детерминированные квадратные изображения размеров
`4`, `8`, `16`, `32` и `60/64`, вычисляют эталон последовательной двухпроходной
сверткой и сравнивают каждый элемент результата с допустимой погрешностью `1e-5`.
Производительные тесты дополнительно используют изображение `512 x 512` и
сравнивают результат с тем же эталонным алгоритмом с погрешностью `1e-4`.

## 6. Экспериментальная среда и методика

Измерения предполагается выполнять через штатный исполняемый файл `ppc_perf_tests` в конфигурации Release.
Для параллельных режимов число рабочих потоков задается настройками фреймворка PPC и/или переменными окружения,
после чего фиксируется время выполнения чистой стадии `RunImpl()` на детерминированном изображении `512 x 512`.

**Определения метрик:**

- `Time` — среднее время выполнения алгоритма в секундах;
- `Speedup = T_seq / T_p`, где `T_seq` — время последовательной версии, `T_p` — время параллельной версии;
- `Efficiency = Speedup / Workers`.
