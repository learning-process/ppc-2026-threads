## 1. Назначение реализации

Последовательная версия задает базовую реализацию умножения двух разреженных
комплексных матриц в формате CRS. Все остальные версии используют ту же
математическую схему, но по-разному организуют распараллеливание.

---

## 2. Проверка входных данных

Корректность входа проверяется функцией `ValidateMatrix`.
Для каждой матрицы контролируются:

- неотрицательные размеры `rows` и `cols`;
- размер массива `row_ptr`, равный `rows + 1`;
- совпадение размеров `values` и `col_index`;
- условия `row_ptr.front() == 0` и `row_ptr.back() == nnz`;
- монотонность массива `row_ptr`;
- допустимость индексов столбцов в `col_index`.

Дополнительно в `ValidationImpl` проверяется совместимость размеров:
число столбцов матрицы `A` должно совпадать с числом строк матрицы `B`.

---

## 3. Основная идея алгоритма

Вычисление выполняется построчно. Для строки `i` матрицы `A` функция `ProcessRow`
просматривает все ее ненулевые элементы `A[i,j]`. Для каждого такого элемента
берется строка `j` матрицы `B`, после чего значения `A[i,j] * B[j,k]`
накапливаются по столбцам `k` результата.

Для накопления используются три временные структуры:

- `row_vals` — массив комплексных частичных сумм по столбцам;
- `row_mark` — массив меток, показывающий, встречался ли столбец в текущей строке;
- `used_cols` — список реально затронутых столбцов.

Если столбец `k` встречается в строке впервые, в `row_mark[k]` записывается номер
строки, значение помещается в `row_vals[k]`, а индекс `k` добавляется в `used_cols`.
При повторном попадании в тот же столбец сумма просто увеличивается.

После завершения строки список `used_cols` сортируется функцией `std::ranges::sort`,
и только точные ненулевые значения записываются в итоговую матрицу `c_`.

---

## 4. Структура реализации

Реализация находится в `seq/src/ops_seq.cpp`.

Основные функции:

- `ValidationImpl` — проверка входных матриц;
- `PreProcessingImpl` — копирование `A` и `B` во внутренние поля `a_` и `b_`,
  инициализация `c_`;
- `ProcessRow` — вычисление одной строки результата;
- `RunImpl` — последовательный проход по всем строкам матрицы `A`;
- `PostProcessingImpl` — передача результата наружу.

В `RunImpl` временные массивы `row_vals`, `row_mark` и `used_cols`
создаются один раз и переиспользуются для всех строк.

---

## 5. Особенности реализации

- Результат формируется сразу в итоговую матрицу `c_`, без промежуточных
  построчных контейнеров.
- Фильтрация нулей выполняется через точное сравнение
  с `std::complex<double>(0.0, 0.0)`.
- В конце работы `c_.row_ptr[m]` получает итоговое число ненулевых элементов.

---

## 6. Вывод

SEQ-реализация является опорной версией задачи. Она проста по структуре,
полностью определяет логику умножения `CRS x CRS -> CRS`
и используется как эталон для сравнения с параллельными реализациями.

---

## Приложение (фрагмент кода)

```cpp
for (int ak = a_start; ak < a_end; ++ak) {
  const int j = a_.col_index[ak];
  const auto a_ij = a_.values[ak];

  const int b_start = b_.row_ptr[j];
  const int b_end = b_.row_ptr[j + 1];

  for (int bk = b_start; bk < b_end; ++bk) {
    const int k = b_.col_index[bk];
    const auto b_jk = b_.values[bk];

    if (row_mark[k] != i) {
      row_mark[k] = i;
      row_vals[k] = a_ij * b_jk;
      used_cols.push_back(k);
    } else {
      row_vals[k] += a_ij * b_jk;
    }
  }
}
```
