## 1. Назначение реализации

ALL-версия сочетает два уровня параллелизма:

- MPI распределяет строки матрицы `A` между процессами;
- OpenMP распараллеливает локальное умножение внутри каждого процесса.

Это гибридная реализация для систем с распределенной и общей памятью.

---

## 2. Распределение строк между MPI-процессами

Разбиение матрицы `A` выполняется не просто по числу строк.
Для этого используется функция `BuildRowBounds`, которая учитывает накопленное
число ненулевых элементов по строкам.

Идея такая:

- вычисляется общее число ненулевых элементов в матрице `A`;
- для каждого процесса выбирается граница по накопленному `nnz`;
- если матрица пустая или процесс один, используется обычное равномерное деление строк.

По построенным границам затем формируются:

- `BuildCountsFromBounds` - число строк на процесс;
- `BuildDisplacements` - смещения;
- `BuildNNZCounts` - число ненулевых элементов локальной части.

---

## 3. Передача данных через MPI

### Передача матрицы `A`

Функция `ScatterRows` передает каждому процессу только его локальный набор строк
матрицы `A`. Для этого отдельно рассылаются:

- длины строк;
- массив `col_index`;
- массив `values`.

Комплексные значения предварительно преобразуются функцией `PackComplexValues`
в массив `double`, где для каждого элемента подряд записываются вещественная
и мнимая части. На принимающей стороне используется `UnpackComplexValues`.

### Передача матрицы `B`

Матрица `B` нужна на каждом процессе целиком, поэтому она рассылается
функцией `BroadcastMatrix`.

Сначала передаются размеры и число ненулевых элементов, затем:

- `col_index`;
- `row_ptr`;
- упакованный массив комплексных значений.

---

## 4. Локальное умножение внутри процесса

Локальная работа вынесена в функцию `MultiplyLocalOMP`.

Внутри нее:

- создается массив `rows_data` типа `LocalRowData`;
- выбирается число потоков как `min(GetNumThreads(), a.rows)`;
- запускается параллельная область OpenMP;
- для каждой строки используется та же маркерная схема, что и в OMP-версии.

Для накопления применяются:

- `row_vals` - массив частичных сумм;
- `row_mark` - массив меток;
- `used_cols` - список использованных столбцов.

Вспомогательные функции:

- `AccumulateRowProducts` - накопление произведений для строки;
- `CollectRowValues` - сортировка `used_cols` и перенос точных ненулевых значений
  в структуру `LocalRowData`.

После локальной параллельной части внутри процесса строится локальный CRS-результат.

---

## 5. Сборка итогового результата

После вычисления локальных матриц результата используется функция `GatherMatrix`.

Она:

- собирает с процессов число локальных ненулевых элементов;
- собирает длины строк;
- собирает `col_index`;
- собирает упакованные комплексные значения.

На процессе `0` по этим данным строится итоговая матрица `global`.

В `RunImpl` после `GatherMatrix` поведение зависит от режима тестирования:

- в `kPerf` на нерутовых процессах результат упрощается до корректной пустой структуры
  нужного размера, после чего выполняется `MPI_Barrier`;
- в остальных режимах итоговая матрица `c_` рассылается всем процессам
  через `BroadcastMatrix`.

---

## 6. Особенности реализации

- Реализация действительно использует MPI для распределения данных,
  а не просто синхронизацию процессов.
- Матрица `A` делится по строкам, а матрица `B` передается всем целиком.
- Локальная вычислительная часть построена на OpenMP и логически близка
  к OMP-версии.
- Для MPI-передачи комплексных значений применяется явная упаковка в `double`.

---

## 7. Вывод

ALL-реализация является полноценной гибридной схемой `MPI + OMP`.
Она сочетает межпроцессное разбиение строк матрицы `A` с локальным
многопоточным умножением и отдельной сборкой результата на процессе `0`.

---

## Приложение (фрагмент кода)

```cpp
BroadcastMatrix(b_, rank);

std::vector<int> row_bounds(static_cast<std::size_t>(size) + 1ULL, 0);
std::vector<int> nnz_counts(static_cast<std::size_t>(size), 0);
if (rank == 0) {
  row_bounds = BuildRowBounds(a_, size);
  nnz_counts = BuildNNZCounts(a_, row_bounds);
}
MPI_Bcast(row_bounds.data(), size + 1, MPI_INT, 0, MPI_COMM_WORLD);
MPI_Bcast(nnz_counts.data(), size, MPI_INT, 0, MPI_COMM_WORLD);

const MatrixCRS local_a = ScatterRows(a_, row_bounds, nnz_counts, rank, size);
const MatrixCRS local_c = MultiplyLocalOMP(local_a, b_);

GatherMatrix(local_c, c_, row_bounds, rank, size, a_.rows);

if (GetStateOfTesting() == ppc::task::StateOfTesting::kPerf) {
  if (rank != 0) {
    c_.rows = a_.rows;
    c_.cols = b_.cols;
    c_.values.clear();
    c_.col_index.clear();
    c_.row_ptr.assign(static_cast<std::size_t>(c_.rows) + 1ULL, 0);
  }
  MPI_Barrier(MPI_COMM_WORLD);
  return true;
}

BroadcastMatrix(c_, rank);
MPI_Barrier(MPI_COMM_WORLD);
```
