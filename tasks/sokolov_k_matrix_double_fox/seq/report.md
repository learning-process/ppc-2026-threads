## 1. Контекст

Последовательная версия алгоритма Фокса служит эталоном для проверки корректности параллельных
реализаций и базой для расчета ускорения. Алгоритм Фокса - это блочный метод умножения матриц,
который обеспечивает хорошую локальность обращений к памяти за счет работы с блоками
фиксированного размера.

## 2. Постановка задачи

**Входные данные:** целое число n - размер квадратных матриц A и B.

**Ограничения:** n > 0. Матрицы заполняются фиксированными значениями: A\[i\]\[j\] = 1.5,
B\[i\]\[j\] = 2.0.

**Выходные данные:** целое число n при успешной проверке результата, иначе -1.

**Критерий корректности:** каждый элемент результирующей матрицы C = A *B должен равняться
3.0* n с точностью 1e-9.

**Крайние случаи:**

- n = 1: матрица 1x1, один блок размером 1x1
- n - простое число: размер блока будет равен 1, количество блоков равно n
- n - составное число: размер блока выбирается как наибольший делитель,
  не превосходящий sqrt(n)

## 3. Базовый алгоритм

Алгоритм состоит из нескольких этапов:

### 3.1. Выбор размера блока

Выбирается наибольший делитель числа n, не превосходящий sqrt(n). Это обеспечивает квадратные блоки одинакового размера.

```text
ChooseBlockSize(n):
    div = floor(sqrt(n))
    while div >= 1:
        if n % div == 0:
            return div
        div -= 1
    return 1
```

### 3.2. Декомпозиция матриц

Исходные матрицы переводятся из строчно-упорядоченного представления в блочно-упорядоченное:

```text
DecomposeToBlocks(flat, blocks, n, bs, q):
    for bi in [0, q):
        for bj in [0, q):
            block_off = (bi * q + bj) * bs * bs
            for i in [0, bs):
                for j in [0, bs):
                    blocks[block_off + i*bs + j] = flat[(bi*bs + i)*n + bj*bs + j]
```

### 3.3. Шаг Фокса

На каждом шаге step от 0 до q-1 для каждой блочной строки i выполняется умножение блоков:

```text
FoxStep(A, B, C, bs, q, step):
    for i in [0, q):
        k = (i + step) % q
        for j in [0, q):
            MultiplyBlocks(A[i][k], B[k][j], C[i][j], bs)
```

### 3.4. Умножение блоков

Умножение двух блоков размера bs x bs выполняется по схеме ikj для улучшения cache-локальности:

```text
MultiplyBlocks(A_block, B_block, C_block, bs):
    for i in [0, bs):
        for k in [0, bs):
            val = A_block[i][k]
            for j in [0, bs):
                C_block[i][j] += val * B_block[k][j]
```

### 3.5. Сборка результата

После всех шагов блочный массив переводится обратно в строчно-упорядоченную матрицу.

**Асимптотика по времени:** O(n^3) - стандартная сложность умножения матриц.

**Асимптотика по памяти:** O(n^2) - требуется хранить три матрицы размером n x n.

**Инварианты:**

- Размер блока всегда делит n нацело
- Количество блоков q = n / block_size
- После каждого шага Фокса частичный результат корректен

## 4. Детали реализации

**Файлы:**

- seq/include/ops_seq.hpp - объявление класса SokolovKMatrixDoubleFoxSEQ
- seq/src/ops_seq.cpp - реализация методов

**ValidationImpl:** проверяет, что входное значение n > 0.

**PreProcessingImpl:** инициализирует матрицы A и B константными значениями 1.5 и 2.0,
выполняет декомпозицию в блочное представление.

**RunImpl:** выполняет q шагов алгоритма Фокса, на каждом шаге перемножая соответствующие
блоки.

**PostProcessingImpl:** собирает результат из блочного представления, проверяет корректность
каждого элемента результирующей матрицы.

**Обработка граничных случаев:**

- При n = 1 создается один блок 1x1
- При простом n размер блока равен 1, что корректно обрабатывается
- Проверка на переполнение не требуется, так как используется тип double

## 5. Проверка корректности

**Метод проверки:** при заполнении матриц константами A = 1.5 и B = 2.0 каждый элемент
результата должен равняться sum(1.5 *2.0) по n элементам = 3.0* n.

**Функциональные тесты:** реализовано 12 тест-кейсов для матриц размером 1x1, 2x2, 3x3, 4x4,
5x5, 6x6, 7x7, 8x8, 9x9, 10x10, 50x50, 100x100.

**Характерные примеры:**

- n = 1: ожидаемый результат C\[0\]\[0\] = 3.0
- n = 4: каждый элемент C\[i\]\[j\] = 12.0
- n = 100: каждый элемент C\[i\]\[j\] = 300.0

Все тесты проходят успешно, что подтверждает корректность базовой реализации.
