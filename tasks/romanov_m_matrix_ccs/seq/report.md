## 1. Контекст
Умножение матриц является базовой математической операцией во многих
дисциплинах, таких как
численное моделирование, машинное обучение и обработка изображений. В реальности
матрицы больших
размеров в инженерных задачах часто являются разреженными (подавляющее
большинство их
элементов равно нулю). Если использовать стандартные структуры данных, то
возникает проблема с
избыточным использованием ресурсов и лишними арифметическими операциями.

Последовательная версия (SEQ) реализует алгоритм вычисления специально для таких
матриц с
использованием формата хранения compressed column storage (CCS).

## 2. Постановка задачи
Задача состоит в реализации операции умножения двух разреженных вещественных
матриц $C = A \times B$. Обе входные матрицы ($A$ и $B$), а также выходная
результирующая матрица ($C$) представлены в формате сжатого хранения по столбцам
(CCS).

**Входные данные:**
Пара матриц разреженного формата std::pair<MatrixCCS, MatrixCCS>, где каждая
структура MatrixCCS содержит:
1. rows_num: количество строк матрицы.

2. cols_num: количество столбцов матрицы.

3. nnz: количество ненулевых элементов (Number of Non-Zeroes).

4. col_ptrs: вектор смещений столбцов размера cols_num + 1, где col_ptrs[j]
указывает на индекс начала элементов $j$-го столбца в массивах индексов и
значений.

5. row_inds: вектор индексов строк размера nnz, содержащий номера строк для
каждого ненулевого элемента.

6. vals: вектор вещественных значений размера nnz, хранящий непосредственно
значения элементов.

**Выходные данные:**
Результирующая разреженная матрица MatrixCCS

**Ограничения и крайние случаи:**
1. Операция умножения матриц определена только тогда, когда количество столбцов
левой матрицы $A$ в точности совпадает с количеством строк правой матрицы $B$.

2. Размерности матриц должны быть строго положительными.

3. Векторы смещений столбцов col_ptrs должны иметь корректный размер, равный
числу столбцов плюс один.

4. Элементы результирующей матрицы, абсолютное значение которых после накопления
не превышает заданную точность, признаются численным нулем и не должны
включаться в финальную сжатую структуру данных матрицы $C$.

## 3. Базовый алгоритм
Формат CCS оптимизирован для последовательного извлечения элементов по столбцам.
В связи с этим
классический алгоритм умножения матриц «строка на столбец» для данного формата
неэффективен,
так как требует произвольного доступа к строкам левой матрицы. Вместо этого
применяется алгоритм,
основанный на линейной комбинации столбцов.

**Пошаговое описание:**
1. Алгоритм итерируется по столбцам $j$ правой матрицы от 0 до n-1 столбца
матрицы.
2. Для каждого столбца $j$ фиксируется индекс начала элементов в сжатой
структуре
3. Инициализируется временный плотный вектор-аккумулятор `accumulator` размера
`A.\text{rows\_num}` нулевыми значениями. Дополнительно используются
вспомогательный вектор
`active_rows` для фиксации затронутых строк и логический массив масок
`row_mask`.
4. Выполняется проход по всем ненулевым элементам $j$-го столбца матрицы `B`.
Для каждого
элемента извлекается индекс его строки `k` и значение `v_b`.
5. Индекс строки $k$ указывает на номер столбца левой матрицы $A$, который
необходимо извлечь. Производится итерация по всем ненулевым элементам $k$-го
столбца матрицы $A$. Значение элемента $A_{i,k}$ умножается на $v\_b$ и
прибавляется к accumulator[i].
6. Если строка $i$ затрагивается впервые (проверяется по row_mask[i]), индекс
$i$ заносится в вектор active_rows, а маска row_mask[i] инвертируется в true.
Это избавляет от необходимости сканировать весь аккумулятор.
7. После завершения прохода по элементам столбца $B$, вектор active_rows
сортируется по возрастанию для соблюдения требований формата CCS к
упорядоченности индексов строк внутри столбца.
8. Осуществляется обход по active_rows: элементы из accumulator с абсолютным
значением $> 10^{-12}$ переносятся в итоговые массивы c.vals и c.row_inds.
Вектор аккумулятора и маски для данных строк сбрасываются в исходное состояние.

**Асимптотика по времени:** $O(B.\text{cols\_num} \times
(\text{avg\_nnz\_B\_col} \times \text{avg\_nnz\_A\_col} + \text{active\_rows}
\log(\text{active\_rows})))$. В худшем случае для плотных матриц алгоритм
эквивалентен плотному умножению $O(L \times M \times N)$. Для высокоразреженных
матриц сложность близка к линейной относительно числа ненулевых операций.

**Асимптотика по памяти:** $O(A.\text{rows\_num})$ дополнительной памяти под
векторы аккумулятора, масок и активных строк на один такт работы, что
пренебрежимо мало по сравнению с памятью под сами матрицы.


## 4. Детали реализации
Файлы: seq/include/ops_seq.hpp, seq/src/ops_seq.cpp

**ValidationImpl():** проверяет корректность входной структуры данных. Выполняет
верификацию соответствия $A.\text{cols\_num} == B.\text{rows\_num}$, валидирует
положительность измерений и размеры векторов смещений col_ptrs, защищая
программу от некорректного обращения к памяти.

**PreProcessingImpl():** подготавливает выходные структуры данных,
гарантированно очищая результирующие вектора vals и row_inds перед началом
нового расчета.

**RunImpl():** содержит основное тело вычислительного цикла алгоритма Густавсона
с накоплением данных в векторе-аккумуляторе.

**PostProcessingImpl():** возвращает true, так как всё завершается
непосредственно на этапе вычислений.
```cpp
for (size_t j = 0; j < b.cols_num; ++j) {
  c.col_ptrs[j] = c.vals.size();

  for (size_t kb = b.col_ptrs[j]; kb < b.col_ptrs[j + 1]; ++kb) {
    size_t k = b.row_inds[kb];
    double v_b = b.vals[kb];

    for (size_t ka = a.col_ptrs[k]; ka < a.col_ptrs[k + 1]; ++ka) {
      size_t i = a.row_inds[ka];
      if (!row_mask[i]) {
        row_mask[i] = true;
        active_rows.push_back(i);
      }
      accumulator[i] += a.vals[ka] * v_b;
    }
  }

  std::ranges::sort(active_rows);

  for (size_t row_idx : active_rows) {
    if (std::abs(accumulator[row_idx]) > 1e-12) {
      c.vals.push_back(accumulator[row_idx]);
      c.row_inds.push_back(row_idx);
    }
    accumulator[row_idx] = 0.0;
    row_mask[row_idx] = false;
  }
  active_rows.clear();
}
```

## 5. Проверка корректности
**Функциональные тесты:**
1. Тест базового перемножения квадратных матриц 2х2

2. Тест на крайний случай — умножение на нулевую матрицу

3. Тест перемножения неквадратных прямоугольных матриц

Все функциональные тесты успешно выполняются как на последовательном backend
(RomanovMMatrixCCSSeq), так и на параллельных модулях вычислений, гарантируя
стопроцентную воспроизводимость и математическую тождественность результатов.
