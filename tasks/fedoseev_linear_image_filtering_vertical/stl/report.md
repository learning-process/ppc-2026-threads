## 4. Схема распараллеливания

**Разбиение данных:** изображение делится по строкам
на непрерывные горизонтальные полосы.
Для `num_threads` потоков: `rows_per_thread = h / num_threads`,
остаток распределяется между первыми потоками.

**Локальные результаты:** каждый поток пишет непосредственно
в выходной массив, так как диапазоны строк не пересекаются
– синхронизация не требуется.

**Запуск и ожидание:** создаются объекты `std::thread`,
каждый выполняет фильтрацию своей полосы.
Затем главный поток вызывает `join()` для всех.

## 5. Детали реализации

**Файлы:** [`stl/include/ops_stl.hpp`](include/ops_stl.hpp),
[`stl/src/ops_stl.cpp`](src/ops_stl.cpp)

**Код запуска потоков:**

```cpp
// File: stl/src/ops_stl.cpp
std::vector<std::thread> threads;
for (int t = 0; t < num_threads; ++t) {
    int start_row = ...;
    int end_row = ...;
    threads.emplace_back([&, start_row, end_row]() {
        for (int row = start_row; row < end_row; ++row) {
            for (int col = 0; col < w; ++col) {
                // вычисление пикселя с kernel.at(ky+1).at(kx+1)
            }
        }
    });
}
for (auto& t : threads) t.join();
```

**Определение числа потоков:**
`int num_threads = ppc::util::GetNumThreads();` (читает `PPC_NUM_THREADS`).

## 9. Выводы

STL показывает высокую производительность благодаря минимальным
накладным расходам и оптимальному разбиению по строкам
(хорошая локальность кэша). Ручное управление потоками
даёт полный контроль, но требует аккуратности.
Для данной задачи STL оказалась наиболее эффективной реализацией.
