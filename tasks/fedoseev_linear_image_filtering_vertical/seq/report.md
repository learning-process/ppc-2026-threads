## 1. Контекст

Последовательная реализация служит эталоном корректности и базой для оценки
ускорения параллельных версий. Задача – применить сглаживающий фильтр
к изображению в оттенках серого.

## 2. Постановка задачи

- **Вход**: изображение `width` × `height` пикселей, вектор `data` целых чисел (0–255).
- **Выход**: обработанное изображение тех же размеров.
- **Ограничения**: минимальный размер 3×3.
- **Алгоритм**: свёртка с ядром Гаусса 3×3 (сумма весов 16):

```text
[1 2 1]
[2 4 2]
[1 2 1]
```

- **Краевые условия**: выход за границы – ближайший допустимый пиксель (clamp).

## 3. Базовый алгоритм

```cpp
// Псевдокод
for (int row = 0; row < h; ++row)
for (int col = 0; col < w; ++col) {
  int sum = 0;
  for (int ky = -1; ky <= 1; ++ky)
    for (int kx = -1; kx <= 1; ++kx) {
      int px = clamp(col + kx, 0, w-1);
      int py = clamp(row + ky, 0, h-1);
      sum += src[py*w + px] * kernel.at(ky+1).at(kx+1);
    }
  dst[row*w + col] = sum / 16;
}
```

- **Сложность по времени**: O(w×h×9) ~ O(N)
- **Сложность по памяти**: O(w×h) + константа.

## 4. Детали реализации

**Файлы:** [`seq/include/ops_seq.hpp`](include/ops_seq.hpp),
[`seq/src/ops_seq.cpp`](src/ops_seq.cpp)

Класс `LinearImageFilteringVerticalSeq` наследует `BaseTask` и переопределяет
четыре метода жизненного цикла:

- [`ValidationImpl()`](src/ops_seq.cpp#L24) – проверяет, что ширина и высота ≥ 3
и размер данных соответствует `width*height`. Возвращает `false` при нарушении.
- [`PreProcessingImpl()`](src/ops_seq.cpp#L28) – инициализирует выходное
изображение нулями, копируя размеры из входного.
- [`RunImpl()`](src/ops_seq.cpp#L94) – выполняет свёртку, используя
вспомогательную лямбду `get_pixel` для безопасного доступа к пикселям.
- [`PostProcessingImpl()`](src/ops_seq.cpp#L101) – всегда возвращает `true`.

**Ключевой фрагмент – цикл свёртки:**

```cpp
// File: seq/src/ops_seq.cpp
const std::array<std::array<int, 3>, 3> kernel = {{{{1,2,1}},{{2,4,2}},{{1,2,1}}}};
const int kernel_sum = 16;

auto get_pixel = [&](int col, int row) -> int {
  col = std::clamp(col, 0, w-1);
  row = std::clamp(row, 0, h-1);
  return src[(static_cast<size_t>(row) * static_cast<size_t>(w)) +
             static_cast<size_t>(col)];
};

for (int row = 0; row < h; ++row) {
  for (int col = 0; col < w; ++col) {
      int sum = 0;
      for (int ky = -1; ky <= 1; ++ky)
          for (int kx = -1; kx <= 1; ++kx)
              sum += get_pixel(col + kx, row + ky) *
                     kernel.at(ky+1).at(kx+1);
      dst[row * w + col] = sum / kernel_sum;
  }
}
```

## 5. Проверка корректности

Функциональные тесты (`ppc_func_tests`) прогоняются для всех типов изображений
(const, grad, rand, check) и размеров 3,5,7,10,16. Результат сравнивается
с эталонной функцией `ReferenceFilter`. Все тесты пройдены.
