## 1. Контекст

OpenMP-реализация использует директивы распараллеливания циклов для ускорения
свёртки на многоядерном CPU. За основу взят последовательный код.

## 2. Постановка задачи

См. `seq/report.md`.

## 3. Базовый алгоритм

Последовательная свёртка (см. секцию 3 отчёта SEQ).

## 4. Схема распараллеливания

**Распараллеливаемая область:** внешний цикл по строкам
(`for (int row = 0; row < h; ++row)`).

**Директива:**

```cpp
#pragma omp parallel for schedule(static) default(none) \
    shared(w, h, src, dst, kernel, kernel_sum)
```

**Переменные:** все внешние переменные – `shared`, переменная `row` – частная
(`private` по умолчанию). `default(none)` требует явного указания атрибутов,
что предотвращает случайные гонки.

**Схема распределения:** `schedule(static)` – строки разбиваются
на непрерывные блоки примерно поровну между потоками. Минимизирует
накладные расходы, так как стоимость каждой итерации одинакова.

**Барьер:** неявный барьер в конце цикла.

**Число потоков** задаётся через `PPC_NUM_THREADS` и передаётся
в `omp_set_num_threads` внутри фреймворка.

## 5. Детали реализации

**Файлы:** [`omp/include/ops_omp.hpp`](include/ops_omp.hpp),
[`omp/src/ops_omp.cpp`](src/ops_omp.cpp)

**Изменения относительно SEQ:** добавлена `#pragma omp parallel for`
перед циклом по строкам. Никаких дополнительных синхронизаций не требуется,
так как каждый поток пишет в непересекающиеся строки выходного массива.

**Код параллельной секции:**

```cpp
// File: omp/src/ops_omp.cpp
#pragma omp parallel for schedule(static) default(none) \
    shared(w, h, src, dst, kernel, kernel_sum)
for (int row = 0; row < h; ++row) {
    for (int col = 0; col < w; ++col) {
        // вычисление пикселя (аналогично SEQ)
        // используется kernel.at(ky+1).at(kx+1)
    }
}
```

## 8. Результаты

**Эффективность (на 2048×2048):** 186% → 96% → 54% → 25%.

## 9. Выводы

OpenMP даёт стабильное ускорение до 2.66× на 4 потоках (1024×1024).
Дальнейшее увеличение потоков снижает эффективность из-за накладных
расходов на синхронизацию и ограничений памяти.
Реализация проста и эффективна для данной задачи.
