## 4. Схема распараллеливания

**Примитив:** `tbb::parallel_for` по диапазону столбцов с блоками.

**Диапазон:** `tbb::blocked_range<int>(0, w, block_width)`,
где `block_width = 64`.

**Размер зёрна (`grainsize`)**: задаётся неявно через `block_width`.
Экспериментально подобран для уменьшения накладных расходов.

**Partitioner:** используется умолчательный `auto_partitioner`
(рекомендован для статической загрузки).

**Контроль конкуренции:** через `PPC_NUM_THREADS` (передаётся
в `global_control::max_allowed_parallelism` внутри фреймворка).

**Время (мс):**

**Ускорение:**

| Размер | 1 поток | 2 потока | 4 потока | 8 потоков |

- При одном потоке TBB медленнее SEQ из-за накладных расходов на планировщик.
- Масштабирование улучшается с ростом размера задачи
  (на 1024 ускорение 1.97×, на 2048 – 1.78×).

## 9. Выводы

TBB демонстрирует хорошее масштабирование на больших изображениях,
но уступает OpenMP и STL в эффективности из-за более тяжёлого планировщика
и разбиения по столбцам (менее кэш-дружественное). Однако TBB обеспечивает
потокобезопасность и автоматическую балансировку, что может быть
преимуществом для более сложных нерегулярных задач.
