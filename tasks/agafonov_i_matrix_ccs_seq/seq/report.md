## 1. Контекст

Данный модуль содержит последовательную реализацию алгоритма перемножения разреженных матриц,
хранящихся в сжатом столбцовом формате (Compressed Column Storage, CCS).
Последовательная версия выполняет роль эталона корректности результатов для валидации
вычислений в гетерогенных и многопоточных конфигурациях. Кроме того, данная реализация
фиксирует базовую линию производительности (baseline T_1), относительно которой в последующих этапах
работы будут вычисляться коэффициенты параллельного ускорения (Speedup) и относительной
эффективности использования вычислительных ресурсов (Efficiency).

## 2. Постановка задачи

Необходимо вычислить произведение двух матриц $A$ и $B$. Обе матрицы хранятся в формате CCS,
который включает три массива: vals(ненулевые значения),
row_inds(индексы строк) и col_ptrs(индексы столбцов).
Входные данные: Матрицы A и B.
Выходные данные: Матрица C = A x B в формате CCS.
Ограничения: Число столбцов левой матрицы должно строго совпадать с числом строк правой матрицы.

Каждая матрица в рамках данной задачи кодируется тремя одномерными массивами, что минимизирует
накладные расходы на хранение нулевых элементов:
1.vals (double): Непрерывный массив, содержащий значения всех ненулевых элементов матрицы,
упорядоченных строго по столбцам.
2.row_inds (int): Массив индексов строк, хранящий индекс строки для каждого соответствующего
элемента из массива значений vals.
3.col_ptrs (int): Массив указателей  на начало каждого столбца. Размерность массива
строго равна числу столбцов плюс единица (cols + 1).

## 3. Базовый алгоритм

Классическое умножение матриц «строка на столбец» (сложность O(N^3)) абсолютно неэффективно для формата
CCS, так как этот формат не предоставляет быстрого доступа к строкам левой матрицы. Вместо этого
применен алгоритм, основанный на линейной комбинации столбцов.

Вычислительное ядро последовательно итерируется по столбцам правой матрицы B. Для текущего столбца j
извлекаются все его ненулевые элементы. Если элемент имеет значение v_b и находится в строке k,
алгоритм находит соответствующий k-й столбец в левой матрице A. Все ненулевые элементы этого столбца
матрицы A перемножаются на константу v_b, а полученные частичные произведения динамически
аккумулируются в плотном векторе-накопителе. По завершении обработки всех ненулевых элементов
столбца j, накопленный вектор фильтруется (отсекаются значения, близкие к машинному нулю),
индексируется и упаковывается в массивы результирующей матрицы C.

Временная сложность: Оценивается как O(cols_B *nz_col_B* nz_col_A), где nz_col — среднее
количество ненулевых элементов в одном столбце. В худшем случае (для плотных матриц) сложность
составляет O(M *N* K), однако на разреженных структурах алгоритм работает кратно быстрее.
Пространственная сложность: Для вычислений требуется O(rows_A) дополнительной памяти под
плотный вектор накопления частичных сумм и массивы масок строк. Общая память структуры данных
пропорциональна фактическому числу ненулевых элементов O(nz_A + nz_B + nz_C).
Инварианты структуры: Алгоритм гарантирует сохранение строгого порядка возрастания индексов
строк внутри каждого сформированного столбца результирующей матрицы C.

## 4. Детали реализации

Файлы: seq/include/ops_seq.hpp, seq/src/ops_seq.cpp
ValidationImpl: Проверяет совместимость размерностей (A.cols_num == B.rows_num) и целостность структуры массивов col_ptrs
PreProcessingImpl: Очищает буферы результирующей матрицы перед началом вычислений
RunImpl: Выполняет основную логику. Для эффективного поиска задействованы вспомогательные массивы: accumulator(сумма),
row_mask(отметка задействованных строк) и active_rows(список строк с ненулевыми результатами)
PostProcessingImpl: Возвращает статус успешного завершения

## 5. Проверка корректности

Корректность алгоритма подтверждена прохождением набора функциональных тестов (`AgafonovIFuncTests`).
Проверены следующие граничные случаи:
Базовое перемножение 2x2
Умножение на единичную матрицу (`Identity_Check`)
Умножение матриц, дающих нулевой результат (`Zero_Result`)
Перемножение прямоугольных матриц разных размерностей (`Rectangular_Check`)

Характерные примеры входа и ожидаемого выхода

**Пример 1: Плотные матрицы в формате CCS**
Вход А: строка 0: [1, 2], строка 1: [3, 4]
Вход B: строка 0: [5, 6], строка 1: [7, 8]
Ожидаемый выход C: строка 0: [19, 22], строка 1: [43, 50]

**Пример 2: Матрицы с непересекающимися элементами**
Вход А: строка 0: [0, 2], строка 1: [0, 0]
Вход B: строка 0: [3, 0], строка 1: [0, 0]
Ожидаемый выход C: строка 0: [0, 0], строка 1: [0, 0]

**Пример 3: Диагональные разреженные матрицы**
Вход А: строка 0: [2, 0], строка 1: [0, 3]
Вход B: строка 0: [4, 0], строка 1: [0, 5]
Ожидаемый выход C: строка 0: [8, 0], строка 1: [0, 15]
