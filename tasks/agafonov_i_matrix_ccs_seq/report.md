## 1. Введение

Умножение разреженных матриц в формате CCS (Compressed Column Storage) это вычислительно сложная задача с
ярко выраженной нерегулярной структурой доступа к памяти. Ненулевые элементы распределены по столбцам хаотично,
что создает естественный дисбаланс нагрузки при параллельной обработке. Данная задача идеально подходит для
комплексного сравнения различных моделей параллелизма: от ручного статического распределения потоков до динамического
планирования задач и гибридных межпроцессных вычислений на уровне кластера

## 2. Единая постановка задачи

Необходимо вычислить произведение двух разреженных матриц C = A * B. Формат хранения CCS подразумевает упаковку
данных в три массива: значения(vals), индексы строк(row_inds) и указатели начала столбцов (col_ptrs).
Вход: Матрицы A и B (совместимых размерностей: число столбцов A равно числу строк B).
Выход: Матрица C в корректном формате CCS с отсортированными индексами строк.
Критерий корректности: Покомпонентное совпадение результирующей матрицы со значениями, полученными в
базовой последовательной версии алгоритма (SEQ)

## 3. Единая методика эксперимента

Все замеры производительности проводились в идентичном окружении:

- CPU: AMD Ryzen 5 5600
- RAM: 16 GB
- OS: WSL2 (Ubuntu)
- Compiler: g++-14
- Build type: Release
- Размер тестовых данных: Матрицы 1000x1000, плотность (density) = 0.01.

Метрики рассчитывались по классическим формулам:

- Ускорение (Speedup): S = T_seq/T_par
- Эффективность (Efficiency): E = S/W, где W общее число потоков

В гибридной версии (ALL) конфигурация задавалась как 2 процесса x 2 потока (W=4), для остальных
технологий использовались 4 независимых потока. Для стабилизации замеров результаты агрегировались тестовым
каркасом с выделением медианного значения после серии прогревочных запусков.

## 6. Интерпретация различий

SEQ — что показывает baseline.
OMP — сильные и слабые стороны именно на этой задаче.
TBB — роль grain size и runtime.
STL — цена ручного управления потоками.
ALL — цена коммуникации и выигрыш гибридности.

Разброс в эффективности между технологиями напрямую объясняется спецификой разреженных матриц:
SEQ (Эталон): Реализует чистое вычислительное ядро без накладных расходов ОС на инициализацию многопоточности,
синхронизацию или выделение разделяемой памяти. Служит надежным якорем масштабируемости ($T_1$).
Разница времени между режимами обусловлена дополнительным оверхедом внутренней обвязки тестового каркаса в режиме `task_run`

TBB (Лидер): Показал наилучшую эффективность (57%). Механизм кражи задач (Work Stealing) и автоматический
разбиватель диапазонов (auto_partitioner) идеально справились с хаотичным распределением ненулевых элементов,
сглаживая нагрузку между ядрами

OMP: Показал достойный результат (ускорение 2.11х). Однако статический планировщик `schedule(static)`
жестко закрепляет столбцы за потоками, что при неравномерной плотности матрицы приводит к небольшому простою
быстрых потоков на неявном барьере в конце параллельной секции

ALL: Гибридная версия показала ускорение 1.93х. Снижение метрик относительно чистого TBB ожидаемо: на
задачах малого объема (1000x1000) накладные расходы на упаковку данных, сетевые вызовы `MPI_Gatherv` и
широковещательную рассылку `MPI_Bcast` съедают часть выигрыша от распараллеливания

STL (Аутсайдер): Ручное распараллеливание через `std::thread` со статическим разбиением (Static Chunking)
провалилось (ускорение 1.01х). Быстрые потоки, обработавшие пустые столбцы, полностью блокируются на `join()`,
простаивая в ожидании самого загруженного воркера. Плюс высок системный overhead на клонирование потоков ОС

## 8. Заключение

Для вычислений на единичном узле с общей памятью (Shared Memory) технология Intel oneTBB является
наиболее оптимальным решением для работы с разреженными структурами данных (CCS/CRS) благодаря встроенному
динамическому балансировщику. Использование ручного управления потоками (std::thread) для таких задач
нецелесообразно. Гибридная технология (MPI+TBB) уступает на малых размерностях из-за цены синхронизации,
однако остается единственным архитектурно верным решением при масштабировании задачи за пределы одного
вычислительного узла на кластерный уровень.

Физическая структура нерегулярного доступа к памяти в CCS
Принципиальное отличие формата CCS заключается в том, что итерация по столбцам имеет фиксированные границы,
но глубина вложенного обхода определяется динамически массивом смещений.
Это порождает сильный дисбаланс потоков при статическом разбиении.
Структура векторов матрицы 3х3
[7.0  0.0  2.0]
[0.0  0.0  0.0]
[0.0  5.0  9.0]
Вектор значений (vals): [7.0, 5.0, 2.0, 9.0 ]
Вектор строк (row_inds): [0, 2, 0,2]
Вектор смещений (col_ptrs): [0, 1, 2, 4]
