## 1. Контекст

В данной версии задачи используется библиотека oneTBB для распараллеливания сортировки Шелла.

TBB позволяет реализовать задачно-ориентированную модель параллелизма, в которой
runtime самостоятельно распределяет задачи между worker-потоками и балансирует нагрузку.

Основная идея распараллеливания состоит в независимой сортировке нескольких блоков
массива с последующим последовательным слиянием результатов.

## 2. Постановка задачи

Требуется реализовать многопоточную сортировку массива целых чисел средствами oneTBB.
Входные, выходные данные и корректность результата аналогичны последовательной версии.

## 3. Базовый алгоритм

Структура алгоритма та же, что и в *OMP* и *STL*:
предварительное вычисление границ блоков, параллельная сортировка, последовательное слияние.

Отличие TBB от статических схем в том, что свободные ядра могут подхватывать незавершённые
задачи из очередей других ядер. Это полезно при неравномерной нагрузке,
хотя для данной задачи блоки примерно одинакового размера.

## 4. Схема распараллеливания

Для распараллеливания используется `tbb::parallel_for(...)`,
так как каждая часть массива сортируется независимо, задача естественно
декомпозируется на набор независимых подзадач.

**Используется диапазон:** `tbb::blocked_range<size_t>(0, num_threads)`.
Каждая итерация соответствует обработке одного блока массива `block_id`.

Для блока вычисляются начало `start = block_id * block_size` и конец
`end = std::min(start + block_size, n)`.

Явный *grainsize* не задаётся. В данной реализации диапазон уже содержит небольшое количество
крупных задач: `num_threads`. Каждая задача сортирует достаточно большой блок массива,
поэтому дополнительное дробление не требуется.

Используется *partitioner* по умолчанию, так как блоки имеют близкий размер и
вычислительная нагрузка примерно одинакова. Значит, дополнительные стратегии
балансировки не требуются.

Каждая задача использует собственный локальный буфер `std::vector<int> local`.
После сортировки данные перемещаются в `blocks[block_id]`. Это уменьшает вероятность
гонок данных и снижает конкуренцию между потоками.

## 5. Детали реализации

Файлы:

- `tbb/include/ops_tbb.hpp`
- `tbb/src/ops_tbb.cpp`

### **Изменения относительно SEQ**

- массив разбивается на несколько блоков;
- сортировка блоков выполняется параллельно через parallel_for;
- используется контейнер `std::vector<std::vector<int>> blocks`
    для хранения локальных результатов.

### **Слияние**

После завершения параллельной сортировки выполняется последовательное слияние
`std::merge(...)`, которое остаётся последовательным участком программы
и является основным ограничением масштабируемости.

Наибольший выигрыш достигается при использовании 2–4 потоков.

При дальнейшем увеличении уровня параллелизма ускорение практически перестаёт
расти, а при 8 работниках наблюдается увеличение времени выполнения относительно 4.

Основными причинами снижения эффективности является последовательный этап слияния,
накладные расходы параллелизации, выделение верменных буферов для каждого блока.

Эффективность уменьшается с ростом числа рабочих потоков.

## 9. Выводы

oneTBB позволяет удобно реализовать параллельную обработку независимых блоков массива через parallel_for.
Ускорение достигается за счёт одновременной сортировки частей массива, однако масштабируемость
ограничивается последовательным слиянием, затратами на планирование задач, дополнительными аллокациями памяти.

Практически оптимальный результат для данной реализации достигается при 2–4 работниках.
Дальнейшее увеличение степени параллелизма не даёт существенного выигрыша и приводит к росту overhead runtime.

## Приложение

```bash
  tbb::parallel_for(tbb::blocked_range<size_t>(0, num_threads), [&](const tbb::blocked_range<size_t> &r) {
    for (size_t block_id = r.begin(); block_id < r.end(); ++block_id) {
      size_t start = block_id * block_size;
      size_t end = std::min(start + block_size, n);

      if (start >= n) {
        continue;
      }

      std::vector<int> local;
      local.reserve(end - start);

      for (size_t i = start; i < end; ++i) {
        local.push_back(input[i]);
      }

      ShellSort(local);

      blocks[block_id] = std::move(local);
    }
  });
```
