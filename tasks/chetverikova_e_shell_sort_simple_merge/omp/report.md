## 1. Контекст

В данном отчёте рассматривается реализация алгоритма сортировки Шелла с использованием
OpenMP. Цель — ускорить последовательную версию за счёт параллельной сортировки независимых
фрагментов массива. После локальной сортировки выполняется последовательное слияние отсортированных блоков.

## 2. Постановка задачи

Требуется адаптировать последовательный алгоритм для многопоточной среды.

Корректность результата определяется аналогично последовательной версии:

- массив отсортирован;
- все элементы сохранены;
- размер результата совпадает с размером входных данных.

Подробное описание baseline-версии приведено в `seq/report.md`.

## 3. Базовый алгоритм

Алгоритм состоит из двух фаз:

1. **Параллельная сортировка**: массив делится на N блоков по числу потоков.
   Каждый поток сортирует свой блок с помощью `ShellSort` независимо.
   Границы блоков вычисляются, равномерно распределяя остаток.

2. **Последовательное слияние**: после сортировки блоки сливаются с помощью
   `std::inplace_merge`. Эта фаза выполняется в одном потоке, так как зависимость
   по данным не даёт её распараллелить.

## 4. Схема распараллеливания

### Параллельная область

Распараллеливается цикл сортировки независимых частей массива:

`#pragma omp parallel for default(none) \`
`shared(input, output, ind_parts, local_buffers, counts_parts) \`
`schedule(static)`

Каждый поток:

- получает собственный диапазон элементов;
- копирует его во временный буфер;
- выполняет локальный Shell Sort;
- сохраняет результат в local_buffers[i].

Shared-переменные: input, output, ind_parts, local_buffers, counts_parts.
Private-переменные: left, right, temp, индекс цикла i.

### Синхронизация

Гонок данных нет — каждый поток пишет в свой независимый диапазон. Неявный барьер в конце
`parallel for` гарантирует, что все блоки отсортированы до начала слияния.

Используется `schedule(static)`, так как все части массива имеют близкий размер и
static уменьшает накладные расходы планировщика.

### Изменения относительно SEQ

- массив делится на несколько частей по числу потоков;
- каждая часть сортируется параллельно;
- результаты временно сохраняются в local_buffers.

### Слияние

После завершения параллельной сортировки выполняется последовательное слияние `std::merge(...)`,
которое остаётся последовательным участком программы и является основным ограничением масштабируемости.

## Приложение

```bash
#pragma omp parallel for default(none) shared(input, output, ind_parts, local_buffers, counts_parts) schedule(static)
  for (size_t i = 0; i < counts_parts; ++i) {
    auto left = static_cast<std::ptrdiff_t>(ind_parts[i]);
    auto right = static_cast<std::ptrdiff_t>(ind_parts[i + 1]);

    std::vector<int> temp(input.begin() + left, input.begin() + right);
    ShellSort(temp);
    local_buffers[i] = std::move(temp);
  }
```
