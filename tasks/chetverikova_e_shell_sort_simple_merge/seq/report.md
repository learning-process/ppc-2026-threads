## 1. Контекст

Рассматривается задача сортировки с объединением двух подходов: сортировки Шелла и простого слияния.
Массив делится на две части, которые сортируются с помощью **алгоритма Шелла**,
а после данные части объединяются **простым слиянием**.

Последовательная реализация используется как:

- эталон корректности,
- база для сравнения ускорений в параллельных версиях.

## 2. Постановка задачи

Требуется отсортировать вектор целых чисел произвольного размера по неубыванию.

Входные данные: Вектор целых чисел произвольной длины (включая пустой вектор).
Выходные данные: Вектор тех же целых чисел, отсортированный в порядке неубывания (по возрастанию).

### Ограничения

Тип данных: `std::vector<int>`.
Размер массива больше 0.
Допускаются:

- отрицательные значения;
- повторяющиеся элементы;
- уже отсортированные массивы;
- массивы, отсортированные в обратном порядке.

### Крайние случаи

Пустой входной вектор: пустой выходной вектор.
Вектор из одного элемента: тот же вектор (уже отсортирован).
Вектор из двух одинаковых элементов: порядок сохраняется (устойчивость слияния).

### Корректный результат

Результат считается корректным, если:

- выходной массив содержит те же элементы, что и входной;
- массив упорядочен по неубыванию;
- размер выходного массива совпадает с размером входного.

## 3. Базовый алгоритм

### Пошаговое описание

1. Исходный массив делится на две части:
    - левую;
    - правую.
2. Для каждой части независимо выполняется сортировка Шелла:
    - выбирается шаг gap = n / 2;
    - выполняется модифицированная сортировка вставками для элементов, расположенных через gap;
    - шаг последовательно уменьшается вдвое;
    - процесс продолжается до gap = 1.
3. После сортировки обеих частей выполняется простое слияние методом `std::ranges::merge`:
    - сравниваются текущие элементы двух массивов;
    - меньший элемент помещается в результирующий массив;
    - после исчерпания одного массива добавляются оставшиеся элементы второго.
4. Полученный массив является итоговым результатом сортировки.

### Асимптотика

Shell Sort зависит от последовательности шагов:

- средняя сложность: O(n log² n)
- худший случай: O(n²)
Слияние O(n).

Общая сложность: O(n²) в худшем случае.
Дополнительная память: O(n) — нужны временные копии левой и правой частей.

### Инварианты и критерий корректности

После завершения каждого прохода Shell Sort элементы, расположенные на расстоянии gap,
упорядочены относительно друг друга.
После выполнения merge результирующий массив полностью отсортирован.

## Приложение

```bash
ShellSort(std::vector<int> &data) {
  if (data.empty()) {
    return;
  }

  size_t n = data.size();
  for (size_t gap = n / 2; gap > 0; gap /= 2) {
    for (size_t i = gap; i < n; i++) {
      int temp = data[i];
      size_t j = i;

      while (j >= gap && data[j - gap] > temp) {
        data[j] = data[j - gap];
        j -= gap;
      }

      data[j] = temp;
    }
  }
}
```
