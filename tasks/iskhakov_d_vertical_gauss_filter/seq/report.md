## 2. Постановка задачи

* **На вход** код получает матрицу изображения в оттенках серого, а также её размеры (строки и столбцы).
* **На выход** же идут данные полученной матрицы после расчетов, а также её размеры по вертикали и горизонтали
* **Ограничениями** является проверка, что у матрицы размеры по вертикали и/или горизонтали не могут быть равны
или меньше 0, а также их перемножение должно строго соответствовать размерам входной матрицы, дабы предотвратить
некорректный подсчет и исключения

```cpp
if (in.width <= 0 || in.height <= 0) return false;
if (in.data.size() != static_cast<size_t>(in.width) * static_cast<size_t>(in.height)) return false;
return true;
```

* К **крайним случаям** относится корректный подсчет граничных пикселей, который осуществляется путем взятия
зеркального пикселя и использования его для расчетов

```cpp
if (col < 0) col = -col - 1;
else if (col >= width) col = (2 * width) - col - 1;

if (row < 0) row = -row - 1;
else if (row >= height) row = (2 * height) - row - 1;

return res[(row * width) + col];
```

## 3. Базовый алгоритм

Пошаговое описание.

1. Перед началом выполнения, программа проверяет данные на корректность в блоке ValidationImpl, а конкретно,
что число столбцов и/или строк не должно быть <= 0 и что размер матрицы должен быть их произведением
2. На вход программе поступает матрица и её размеры, число строк и столбцов в блоке RunImpl, после чего присутствует
логика деления на полосы (bands) с kNumBands=1, что в последовательном режиме означает единственную полосу — всё
изображение, но служит заготовкой для параллельных версий
3. Подсчёт самих значений идёт в двойном цикле for, который проходится по всем пикселям матрицы и высчитывает новые
значения, отталкиваясь от строгих данных:

```cpp
const int kDivConst = 16;
const std::array<std::array<int, 3>, 3> kGaussKernel = {{{1, 2, 1}, {2, 4, 2}, {1, 2, 1}}};
```

Сам подсчёт идёт с обращением к окружающим его пикселям, а в случае ухода за границу матрицы используется метод,
который отвечает за зеркальный подсчёт пикселей (для отрицательных индексов

`col = -col - 1`,

для индексов дальше последнего в строке

`col >= width → col = 2*width - col - 1; `, аналогично с высотой).

4. Результат записывается в заранее созданный вектор, после чего отправляется вместе с количеством строк и столбцов
обратно в код тестов для проверки корректности.


* **Асимптотика по времени:** O(N), где N — число пикселей.
* **Асимптотика по памяти:** O(N), для одновременного хранения получаемой матрицы и новой, вычисленной матрицы, общим
размером 2N элементов.

* **Инварианты и критерий корректности:**
1. Сами значения пикселей находятся строго в диапазоне [0, 255], т.е. вес ядра равен 16, благодаря чему выход за
пределы невозможен.
2. Каждый полученный пиксель зависит сугубо от окна 3 на 3.


## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

**ValidationImpl** - выполняет проверку входных параметров, на размер строки и столбца больше 0, а также на то, что
их перемножение будет выдавать размер самой матрицы
**RunImpl** - выполняет основную реализацию, а именно подсчет новой матрицы и отправка итогового результата обратно
в тестовый файл для проверки корректности

Также имеется функция **IskhakovDGetPixelMirrorSeq**, отвечающая за обработку граничных случаев (обращение к пикселям
за границей матрицы):

```cpp
uint8_t IskhakovDGetPixelMirrorSeq(const std::vector<uint8_t> &res, int col, int row, int width, int height) {
  if (col < 0) {
    col = -col - 1;
  } else if (col >= width) {
    col = (2 * width) - col - 1;
  }
  if (row < 0) {
    row = -row - 1;
  } else if (row >= height) {
    row = (2 * height) - row - 1;
  }
  return res[(row * width) + col];
}
```

остальные функции как **PostProcessingImpl** и **PreProcessingImpl** никакой задачи не выполняют


## 5. Проверка корректности

Были использованы функциональные тесты, ориентированные на обнаружение исключений (Негативные тесты), а конкретно


1. На матрицу равную 0 размеру, в котором ожидается выход в условии проверки числа строк и столбцов

```cpp
Matrix zero_sizes{.width = 0, .height = 0, .data = {}};
Matrix zero_width{.width = 0, .height = 5, .data = std::vector<uint8_t>(5)};
Matrix zero_height{.width = 5, .height = 0, .data = std::vector<uint8_t>(5)};
```

2. На несоответствие данных, а именно матрицы не равный размеру строки умноженному на размер столбца

```cpp
Matrix size_mismatch{.width = 3, .height = 3, .data = {1, 2, 3}};
```

3. На размер строки или столбца меньше 0

```cpp
Matrix negative_width{.width = -1, .height = 5, .data = std::vector<uint8_t>(5)};
Matrix negative_height{.width = 5, .height = -1, .data = std::vector<uint8_t>(5)};
```


Также имеются тесты на корректное выполнение (Позитивные тесты), например

```cpp
const Matrix kInput2x2 = MakeMatrix(2, 2, {1, 2, 3, 4});
const Matrix kExpected2x2 = MakeMatrix(2, 2, {1, 2, 2, 3});

const Matrix kInput1x1 = MakeMatrix(1, 1, {100});
const Matrix kExpected1x1 = MakeMatrix(1, 1, {100});
```

в котором отправляются корректные данные и ожидается уже просчитанный заранее результат

Данная версия реализации (SEQ) успешно проходит все функциональные тесты
