# Умножение разреженных матриц в формате CCS

- Студент: Сафарян Григор
- Группа: 3823Б1ПР5
- Технология: SEQ
- Вариант: 5
- Тип элементов: `double`
- Формат хранения: CCS, столбцовое сжатое хранение

## Краткое описание

В лабораторной работе реализовано умножение двух разреженных матриц,
представленных в формате CCS. Матрицы хранят только ненулевые элементы в
массивах `values`, `row_indices` и `col_ptrs`.

Имя директории содержит фрагмент `crs`, но в тексте отчета и в реализации
используется формат CCS. Переименование папок не выполнялось, чтобы сохранить
совместимость со структурой проекта.

## Расположение реализации

Основные файлы SEQ-версии:

- `common/include/common.hpp` — описание структуры `SparseMatrixCCS` и типов;
- `seq/include/ops_seq.hpp` — объявление класса `SafaryanATaskSEQ`;
- `seq/src/ops_seq.cpp` — последовательный алгоритм умножения;
- `tests/functional/main.cpp` — функциональные тесты;
- `tests/performance/main.cpp` — производительные тесты.

## Проверка

Функциональные тесты проверяют фиксированные случаи умножения: квадратные,
прямоугольные, диагональные матрицы, умножение на единичную матрицу и нулевой
результат. Производительный тест использует случайные разреженные матрицы и
сравнивает результат с эталонным плотным умножением.

## Команды запуска

Команды для локального запуска SEQ-тестов:

```powershell
cmake --build build --config Release --parallel
.\build\bin\ppc_func_tests.exe --gtest_filter="*safaryan_a_sparse_matrix_mult_crs_seq*"
.\build\bin\ppc_perf_tests.exe --gtest_filter="*safaryan_a_sparse_matrix_mult_crs_seq*"
```

## Итог

SEQ-реализация выполняет умножение разреженных матриц с элементами `double` в
формате CCS, проходит функциональные проверки и служит базовой версией для
сравнения с параллельными реализациями.
