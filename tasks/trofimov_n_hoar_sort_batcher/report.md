## 1. Введение

Цель работы — реализовать и сравнить версии сортировки Хоара + слияния Батчера
для технологий SEQ, OMP, TBB, STL и гибридной ALL.

## 2. Единая постановка задачи

Дано: `std::vector<int>` произвольной длины.
Требуется: вернуть массив, отсортированный по неубыванию.

Критерии корректности:

- верная работа для `size = 0` и `size = 1`;
- поддержка отрицательных значений и дубликатов;
- в ALL-версии — глобально отсортированный результат после MPI-обмена.

## 3. Единая методика эксперимента

- Сравнение выполняется при одинаковом тестовом наборе (`tests/functional`, `tests/performance`).
- Базовая версия для speedup: `seq`.
- Метрики: время (s), ускорение `S = T_seq / T_x`, эффективность `E = S / workers`.
- В режимах OMP/TBB/STL workers = число потоков, в ALL — `ranks × threads`.

## 4. Сводка корректности

Все реализации сравниваются с эталонной последовательной логикой:

- функциональные тесты покрывают пустой, единичный, уже отсортированный,
  обратно отсортированный массивы, случаи с дубликатами и отрицательными числами;
- дополнительно проверяется отсортированность (`std::ranges::is_sorted`).

## 6. Интерпретация

- На приведённых измерениях лучший результат показывает STL-режим.
- ALL включает накладные расходы MPI (`scatter/gather/bcast`), что на данном размере
  входа может ограничивать итоговое ускорение.
- Для других размеров и `ranks × threads` конфигураций лидер может отличаться.
