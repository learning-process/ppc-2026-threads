## Постановка задачи

Реализовать поразрядную сортировку (radix sort) для массива целых чисел (int) с использованием четно-нечетного слияния Бэтчера для параллельного объединения отсортированных блоков. Задача выполнена в пяти технологиях: SEQ, OMP, TBB, STL, ALL.

### Входные данные
- Массив целых чисел `std::vector<int>` произвольного размера, включая отрицательные числа.

### Выходные данные
- Тот же массив, отсортированный по возрастанию.

---

## Описание алгоритма

### Поразрядная сортировка (Radix Sort LSD)

Сортировка выполняется побайтово (4 прохода для 32-битного int). На каждом проходе применяется устойчивая сортировка подсчётом по текущему байту. Для корректной обработки знаковых чисел старший байт обрабатывается с XOR 0x80, что переставляет отрицательные числа перед положительными.

### Слияние Бэтчера (Odd-Even Merge)

После параллельной сортировки блоков применяется сеть слияния Бэтчера: входные массивы разбиваются на чётные и нечётные элементы, каждая половина сортируется/сливается рекурсивно, затем соседние пары сравниваются и при необходимости переставляются.

## Окружение

- **ОС:** Windows 11 / Ubuntu 24 (CI)
- **Компилятор:** MSVC (локально), GCC 14 / Clang 21 (CI)
- **Тип сборки:** Release (-O3)
- **Число потоков:** задаётся через `PPC_NUM_THREADS`

## Сборка и запуск

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --parallel

# Функциональные тесты
./build/bin/ppc_func_tests --gtest_filter="*solonin_v_radix_sort_batcher*"

# Тесты производительности
./build/bin/ppc_perf_tests --gtest_filter="*solonin_v_radix_sort_batcher*"
```

Все пять реализаций дают идентичные результаты, проверенные функциональными тестами. Параллельные версии (OMP, TBB, STL, ALL) демонстрируют ускорение на больших массивах за счёт параллельной сортировки независимых блоков с последующим слиянием Бэтчера.
