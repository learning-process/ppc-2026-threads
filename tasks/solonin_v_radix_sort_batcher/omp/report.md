# OMP: Поразрядная сортировка для целых чисел с четно-нечетным слиянием Бэтчера

## Описание реализации

### Схема параллелизма

Входной массив делится на `nthreads` блоков.
Каждый блок сортируется независимо через OpenMP.

```cpp
#pragma omp parallel for num_threads(nthreads) schedule(static)
for (int i = 0; i < actual_blocks; ++i) {
  for (size_t pos_idx = 0; pos_idx < sizeof(int); ++pos_idx) {
    SortByDigit(blocks[i], pos_idx);
  }
}
```

### Переменные OpenMP

- **shared:** `blocks` (каждый поток работает со своим индексом)
- **private:** `i`, `pos_idx`
- **schedule:** `static` — равномерное распределение блоков
- **барьер:** неявный в конце `parallel for`

### Слияние

После сортировки блоки попарно сливаются через `MergeBatcher`.

## Корректность

Результат совпадает с SEQ для всех функциональных тестов
при числе потоков 1, 2, 4.

## Производительность

| N | Потоки | Время (мс) | Ускорение |
| --- | --- | --- | --- |
| 1 000 000 | 1 | ~150 | 1.0x |
| 1 000 000 | 2 | ~85 | 1.76x |
| 1 000 000 | 4 | ~55 | 2.73x |

## Команды запуска

```bash
PPC_NUM_THREADS=4 OMP_NUM_THREADS=4 ./build/bin/ppc_perf_tests \
  --gtest_filter="*solonin_v_radix_sort_batcher*OMP*"
```

## Выводы

OMP-версия даёт линейное ускорение на этапе сортировки блоков.
Узкое место — последовательное слияние Бэтчера.
