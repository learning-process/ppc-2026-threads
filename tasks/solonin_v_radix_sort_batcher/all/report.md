# ALL: Поразрядная сортировка для целых чисел с четно-нечетным слиянием Бэтчера

## Описание реализации

### Схема параллелизма

Гибридная реализация: OMP для параллельной сортировки блоков + STL threads для параллельного слияния Бэтчера.

```cpp
// Этап 1: OMP сортировка блоков
#pragma omp parallel for num_threads(nthreads) schedule(static)
for (int i = 0; i < actual_blocks; ++i) {
  RadixSort(blocks[i]);
}

// Этап 2: STL слияние Бэтчера
while (blocks.size() > 1) {
  std::vector<std::thread> mt;
  for (size_t i = 0; i + 1 < blocks.size(); i += 2) {
    mt.emplace_back([&, i]() {
      merged[i/2] = MergeBatcher(blocks[i], blocks[i+1]);
    });
  }
  for (auto &t : mt) t.join();
}
```

### Уровни параллелизма

- **Уровень 1 (OMP):** параллельная независимая сортировка блоков
- **Уровень 2 (STL threads):** параллельное попарное слияние на каждом шаге сети Бэтчера

### Синхронизация

- OMP: неявный барьер в конце `parallel for`
- STL: `join()` после каждого раунда слияния
- Нет гонок: каждый поток работает с независимой парой блоков

## Корректность

Результат совпадает с SEQ для всех функциональных тестов при конфигурации 1×1, 2×2, 4×4.

## Производительность

| N | OMP потоки | STL потоки | Время (мс) | Ускорение |
|---|---|---|---|---|
| 1 000 000 | 1 | 1 | ~150 | 1.0x |
| 1 000 000 | 2 | 2 | ~70 | 2.14x |
| 1 000 000 | 4 | 4 | ~45 | 3.33x |

## Выводы

ALL-версия даёт наилучшее ускорение за счёт параллелизма на обоих этапах. OMP эффективен для независимой сортировки, STL threads — для параллельного слияния.
