## 2. Постановка задачи

Требуется вычислить:

```text
C = A * B
```

где `A` и `B` - квадратные матрицы размера `n × n` с элементами `double`.

**Формат входа:**

- `input[0]` - размер `n`;
- далее матрицы `A` и `B` в row-major порядке.

**Формат выхода:**

- матрица `C` размера `n × n`.

**Ограничения:**

- `n > 0`, `input.size() == 1 + 2 * n * n`;
- при необходимости - padding до степени двойки и обрезка результата;
- критерий корректности: совпадение с наивным умножением, допуск `1e-7`.

## 3. Описание алгоритма (базового/последовательного)

Алгоритм Штрассена реализован итеративно через стек `Frame`.

**Шаги:**

1. Если `n <= kThreshold` (`kThreshold = 64`) - обычное умножение.
2. Иначе деление на квадранты, вычисление `M1..M7`, сборка `C11..C22`.
3. Padding до степени двойки перед вычислением, обрезка после.

**Асимптотика:** около `O(n^2.81)` по времени; память выше, чем у наивного
умножения, из-за временных буферов.

## 4. Схема распараллеливания

Параллелизация не используется. Все вычисления выполняются
в одном потоке. Эта версия служит эталоном корректности
и отправной точкой для метрик ускорения.

## 5. Детали реализации

**Файлы:**

- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

**Пайплайн:**

- `ValidationImpl()` - проверка формата входа;
- `PreProcessingImpl()` - выделение выхода;
- `RunImpl()` - padding, итеративный Штрассен, копирование результата;
- `PostProcessingImpl()` - проверка размера выхода.

**Структура данных:**

- матрицы в `std::vector<double>`, индекс `row * n + col`;
- структура `Frame` хранит подматрицы, `M1..M7` и стадию вычисления.

**Крайние случаи:** `n <= 64`, нестепень двойки (padding), пустой вход.
