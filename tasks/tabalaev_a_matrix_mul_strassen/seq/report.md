## 1. Контекст

Задача умножения плотных матриц — фундаментальная операция линейной алгебры,
используемая во многих вычислительных задачах. Классический алгоритм имеет
сложность $O(N^3)$, но алгоритм Штрассена снижает асимптотику до $O(N^{2.81})$
ценой более сложного управления памятью.

Последовательная реализация задаёт **эталон корректности** и **эталон времени** для
всех остальных версий, её время  SEQ ($T_{\mathrm{seq}}$) используется как
знаменатель при вычислении ускорения $S = T_{\mathrm{seq}} / T_p$

## 2. Постановка задачи

**Вход.**  
Структура `InType = MatrixData`, определённая в `common/include/common.hpp`:

- `a_rows` — количество строк первой матрицы.
- `a_cols_b_rows` — количество столбцов первой матрицы (и строк второй).
- `b_cols` — количество столбцов второй матрицы.
- `a` — вектор размера `a_rows * a_cols_b_rows` с элементами первой матрицы.
- `b` — вектор размера `a_cols_b_rows * b_cols` с элементами второй матрицы.

**Выход.**  
`OutType = std::vector<double>` — результирующая матрица размера `a_rows × b_cols`.

**Ограничения (проверяются в `ValidationImpl`).**

- Все размерности `a_rows`, `a_cols_b_rows`, `b_cols` строго больше нуля.
- Количество столбцов первой матрицы равно количеству строк второй (`a_cols_b_rows`).
- Реальные размеры векторов `a` и `b` точно соответствуют переданным размерностям
  (иначе вход некорректен).

**Крайние случаи.**  
Матрицы могут быть произвольных размеров (не квадратными, не степенями двойки).
Алгоритм Штрассена требует квадратных матриц размера $2^k \times 2^k$, поэтому в
реализации предусмотрено **выравнивание нулями (padding)** до ближайшей степени
двойки, покрывающей максимальное измерение.

## 3. Базовый алгоритм

Используется **алгоритм Штрассена** в итеративной форме (без рекурсии,
с явным стеком состояний). Пошаговое описание:

1. **Выравнивание:** исходные матрицы дополняются нулями до размера $n \times n$,
   где $n = 2^k$ — наименьшая степень двойки, не меньшая максимального из
   `a_rows`, `a_cols_b_rows`, `b_cols`.
2. **Базовый случай:** если текущий размер блока $n \le 32$, выполняется
   классическое кубическое умножение `BaseMultiply` (тройной цикл) — это
   предотвращает чрезмерные накладные расходы для малых блоков.
3. **Разбиение:** каждая матрица делится на 4 блока размера $n/2 \times n/2$:
   $$A = \begin{pmatrix} A_{11} & A_{12} \\ A_{21} & A_{22} \end{pmatrix},\quad
     B = \begin{pmatrix} B_{11} & B_{12} \\ B_{21} & B_{22} \end{pmatrix}.$$
4. **Вычисление 7 промежуточных матриц** (вместо 8 при стандартном разбиении):
   - $P_1 = (A_{11} + A_{22})(B_{11} + B_{22})$
   - $P_2 = (A_{21} + A_{22})B_{11}$
   - $P_3 = A_{11}(B_{12} - B_{22})$
   - $P_4 = A_{22}(B_{21} - B_{11})$
   - $P_5 = (A_{11} + A_{12})B_{22}$
   - $P_6 = (A_{21} - A_{11})(B_{11} + B_{12})$
   - $P_7 = (A_{12} - A_{22})(B_{21} + B_{22})$
5. **Сборка результата** (через сложения и вычитания матриц $P_i$):
   - $C_{11} = P_1 + P_4 - P_5 + P_7$
   - $C_{12} = P_3 + P_5$
   - $C_{21} = P_2 + P_4$
   - $C_{22} = P_1 - P_2 + P_3 + P_6$
6. **Обрезка (cropping):** после завершения умножения выровненных матриц из
   результата извлекается подматрица исходных размеров `a_rows × b_cols`.

**Асимптотика по времени:** $O(N^{\log_2 7}) \approx O(N^{2.81})$, где $N$ — размер
выровненной матрицы.

**Асимптотика по памяти:** $O(N^2)$ для хранения промежуточных блоков на каждом
уровне декомпозиции.

**Критерий корректности:** для любых допустимых входных матриц $A$ и $B$ результат
алгоритма Штрассена должен совпадать с результатом классического умножения с
погрешностью $\epsilon = 10^{-8}$

## 4. Детали реализации

**Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.

Класс `TabalaevAMatrixMulStrassenSEQ` наследуется от `BaseTask` и переопределяет
четыре виртуальных метода:

- **`ValidationImpl`**  
  Проверяет корректность входных данных: положительные размерности,
  согласованность размеров матриц, соответствие размеров векторов.
  При нарушении любого условия возвращает `false`, что блокирует дальнейшее
  выполнение задачи.

- **`PreProcessingImpl`**  
  1. Сохраняет исходные размеры (`a_rows_`, `a_cols_b_rows_`, `b_cols_`).  
  2. Вычисляет `max_dim = max(a_rows_, a_cols_b_rows_, b_cols_)` и находит
     `padded_n_` — наименьшую степень двойки, не меньшую `max_dim`.  
  3. Выделяет векторы `padded_a_` и `padded_b_` размера `padded_n_ * padded_n_`,
     заполняет их нулями.  
  4. Копирует элементы исходных матриц в соответствующие позиции (верхний левый
     угол выровненных матриц).  

- **`RunImpl`**  
  1. Вызывает `StrassenMultiply(padded_a_, padded_b_, padded_n_)`, который возвращает
     результат умножения выровненных матриц.  
  2. Извлекает из него значимую часть размера `a_rows_ × b_cols_` и записывает
     в `GetOutput()`.  

- **`PostProcessingImpl`**  
  Никаких действий не требуется (результат уже сохранён в `GetOutput()`).

**Обработанные исключительные и граничные случаи:**

- Нулевые размерности: отсекаются в `ValidationImpl`.
- Несогласованные размеры: отсекаются в `ValidationImpl`.
- Неквадратные матрицы и размеры, не являющиеся степенями двойки: корректно
  обрабатываются через padding и последующее обрезание.
- Переход на базовое умножение при $n \le 32$ отсекает излишние накладные расходы
  для малых блоков.

**Ключевая особенность:**  
Отказ от системной рекурсии. Алгоритм реализован **итеративно с явным стеком
фреймов** (`std::stack<StrassenFrame>`). Каждый фрейм хранит подматрицы, размер
и стадию (0 — разбиение/умножение, 1 — сборка).

```cpp
// File: seq/src/ops_seq.cpp
std::vector<double> TabalaevAMatrixMulStrassenSEQ::StrassenMultiply(
    const std::vector<double> &mat_a, const std::vector<double> &mat_b, size_t n) {
  std::stack<StrassenFrame> frames;
  std::stack<std::vector<double>> results;
  frames.push({mat_a, mat_b, n, 0});
  while (!frames.empty()) {
    StrassenFrame current = std::move(frames.top());
    frames.pop();
    if (current.stage == 0) {
      if (current.n <= 32) {
        results.push(BaseMultiply(current.mat_a, current.mat_b, current.n));
      } else {
        PushStrassenSubtasks(frames, current.mat_a, current.mat_b, current.n);
      }
    } else {
      results.push(CombineStrassenResults(results, current.n));
    }
  }
  return results.top();
}
```

## 5. Проверка корректности

**Метод сравнения.**  
Результат алгоритма Штрассена сравнивается с результатом классического кубического
умножения (отдельная эталонная реализация). Сравнение покомпонентное, с допуском
`constexpr double kEpsilon = 1e-8`.

**Функциональные тесты.**  
В `tests/functional/main.cpp` определён набор из 6 параметризованных тестов,
покрывающих все критические ветвления алгоритма:

| Размер       | Тип                       | Что проверяет                                             |
|--------------|---------------------------|-----------------------------------------------------------|
| 3×3          | SmallPadded               | Fallback (n ≤ 32) на неквадратной матрице                 |
| 4×4          | SmallPowerOfTwo_4x4       | Fallback на точной степени двойки                         |
| 15×15        | MediumPadded              | Padding до 16×16, базовый случай после выравнивания       |
| 16×16        | MediumPowerOfTwo_16x16    | Работа без padding, проверка разбиения                    |
| 255×255      | LargePadded               | Padding до 256×256, полное разбиение на 7 подзадач        |
| 256×256      | LargePowerOfTwo_256x256   | Полное разбиение на 7 подзадач и сборка                   |

Все 6 параметризованных тестов проходят успешно, расхождений не наблюдается.
