## 1. Введение

В данной работе исследованы несколько реализаций алгоритма поразрядной
сортировки (LSD Radix Sort) для массива вещественных чисел типа `double`
с последующим итеративным слиянием.

Рассматривались реализации:

- Без использования технологий параллелизма (последовательная версия,
  технология **SEQ**);
- С использованием библиотеки OpenMP (технология **OMP**);
- С использованием стандартной библиотеки `std::thread`
  (технология **STL**);
- С использованием библиотеки Intel OneTBB (технология **TBB**);
- С использованием MPI для параллелизма на уровне процессов
  совместно с OpenMP (технология **ALL**).

Данная задача подходит для параллелизма, поскольку сортировка большого
массива данных (20 млн элементов) требует значительных вычислительных
ресурсов.

---

## 2. Единая постановка задачи

Дан массив `A` размера `n` с элементами типа `double`.
Необходимо отсортировать массив по возрастанию.

### Условия корректности

- Проверяется, что массив не пуст;
- Результат должен быть отсортирован по неубыванию;
- Погрешность: **1e-14**;
- Сравнение с SEQ.

  реализации к времени сравниваемой реализации;
- **Efficiency (эффективность)** = speedup / workers.

## 6. Интерпретация результатов

### Сравнение технологий

| Технология | Макс. ускорение | Эфф-ть (12 workers) |
| --- | --- | --- |
| SEQ | 1.00x | 1.00 |
| OMP | 1.99x | 0.17 |
| STL | 2.03x | 0.17 |
| TBB | 1.50x | 0.13 |
| ALL | 1.70x | 0.14 |

### Ключевые наблюдения

1. **Суперлинейное ускорение** (OMP, STL):
   - Ускорение >1.0 на 1 потоке объясняется лучшим использованием
     кэш-памяти,
   - Преобразование double → uint64_t выполняется параллельно
     даже на 1 потоке.

2. **Memory-bound характер задачи**:
   - Увеличение числа потоков не даёт прироста производительности,
   - Ограничение — пропускная способность памяти.

3. **STL — лучший результат** (2.03x):
   - Минимальные накладные расходы,
   - Ручное распределение работы.

4. **TBB — хорошая масштабируемость, но низкое ускорение**:
   - Ускорение растёт от 0.95x до 1.50x,
   - Накладные расходы на библиотеку.

5. **ALL — компромиссный вариант**:
   - Использование 2 процессов даёт выигрыш (1.70x),
   - Добавление потоков не улучшает результат.

## 8. Заключение

Реализованы и протестированы последовательная, а также параллельные
реализации алгоритма поразрядной сортировки массива вещественных чисел
типа `double` с использованием технологий SEQ, OMP, STL, TBB и ALL.

### Результаты экспериментов

- Подтверждена корректность всех реализаций;
- Вычислено время работы алгоритмов;
- Проведено сравнение с последовательной реализацией.

### Основные результаты

| Технология | Максимальное ускорение |
| --- | --- |
| OMP | 1.99x |
| STL | 2.03x (лучший) |
| TBB | 1.50x |
| ALL | 1.70x |

### Выводы

1. **STL** показала наибольшее ускорение (2.03x) благодаря минимальным
   накладным расходам и ручному распределению работы.

2. **OMP** показала сопоставимый результат (1.99x), что подтверждает
   эффективность OpenMP для данной задачи.

3. **TBB** продемонстрировала хорошую масштабируемость (ускорение
   растёт от 0.95x до 1.50x), но уступила в абсолютном ускорении
   из-за накладных расходов библиотеки.

4. **ALL** (MPI+OpenMP) показала результат 1.70x, что подтверждает
   эффективность использования двух процессов, но накладные расходы
   на коммуникации ограничивают дальнейшее масштабирование.

5. Все параллельные реализации ограничены пропускной способностью
   памяти (memory-bound), поэтому увеличение числа потоков свыше 2
   не даёт значительного прироста производительности.

### Рекомендации

- Для сортировки больших массивов на системах с общей памятью
  рекомендуется использовать STL или OpenMP.
- Для распределённых систем (кластеры) целесообразно применять
  гибридный подход MPI+OpenMP.
- Для maximum compatibility и автоматической балансировки
  нагрузки подходит Intel TBB.
