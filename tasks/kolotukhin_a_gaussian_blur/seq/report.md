## 1. Контекст

Данный отчёт описывает последовательную (`SEQ`) реализацию фильтра размытия по Гауссу с ядраом свертки 3×3.
Эта версия служит эталоном корректности и базовой точкой для расчёта ускорения и эффективности параллельных версий.
Все данные, алгоритмы и результаты последующих отчётов сравниваются с данной версией.

## 2. Постановка задачи

### Формат данных

Типы данных в задаче описаны [здесь](../common/include/common.hpp) и представляют следующие типы:

- **Input:** кортеж `(pixel_data, img_width, img_height)`, где `pixel_data` – вектор `uint8_t`
(оттенки серого) размера `img_width × img_height`.
- **Output:** вектор `uint8_t` такого же размера, содержащий результирующее изображение в оттенках серого.

### Ядро свёртки (3x3)

```text
1 2 1
2 4 2 / 16
1 2 1
```

### Ограничения

- Входное изображение должно быть непустым (`img_width × img_height > 0`).
Проверка происходит на этапе чтения изображения при формировании теста (`SetUp()` в
[тестах производительности](../tests/performance/main.cpp)).
- Размер входного вектора пикселей должен строго соответствовать произведению
полученных `img_width` и `img_height`. Проверка проходит в `ValidationImpl()`.
- Входное изображение для обработки должно быть в оттенках серого (`uint8_t`).
- Для пикселей на границах используется отражение – координаты за пределами изображения приводятся к ближайшим
допустимым координатам.

### Критерий корректности

Поэлементное совпадение результата с эталонным выходом для заданного входа.

## 3. Базовый алгоритм

Для каждого пикселя `(col, row)` исходного изображения:

1. Инициализируется аккумулятор `acc = 0`.
2. Происходит обход окрестности 3x3 с центром в `(col, row)`.
3. Для каждого соседа `(col+dx, row+dy)`:
   - Функция `GetPixel` возвращает значение пикселя с учётом ограничения координат границами изображения (clamp).
   - Значение пикселя умножается на соответствующий вес из ядра Гаусса.
   - Результат добавляется в аккумулятор.
4. Результирующее значение пикселя вычисляется как `acc / 16` (средневзвешенное по ядру) и записывается в выходной
вектор.

**Асимптотическая сложность:**

- По времени: `O(W × H × 9)` = `O(N)`.
- По памяти: `O(N)` для хранения одного изображения, `O(2N)` с учётом входного и выходного буферов.

**Инварианты и гарантии:**

- Размер выходного вектора строго равен размеру входного.
- Позиция каждого пикселя в выходном массиве соответствует его позиции во входном.
- Обработка границ гарантирует корректное применение ядра для всех пикселей.

## 4. Детали реализации

**Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

**Структура класса `KolotukhinAGaussianBlurSEQ`:**

- **`ValidationImpl()`** проверяет, что размер входного вектора пикселей в точности равен `img_width × img_height`.
- **`PreProcessingImpl()`** выделяет память для выходного вектора `output` размером `img_width × img_height`, заполняя
его нулями. Это предотвращает доступ к неинициализированной памяти.
- **`RunImpl()`** содержит основную вычислительную логику – вложенный цикл (row, col, окрестность 3×3).
- **`PostProcessingImpl()`** – пустая операция, так как финальный результат уже находится в `GetOutput()`.

### Основной цикл обработки

К каждому пикселю `(row, col)` входного изображения (в оттенках серого) применяются 9 операций от ядра.
Результат нормируется и записывается в выходное изображение.

```cpp
// Фрагменты файла: seq/src/ops_seq.cpp
const static std::array<std::array<int, 3>, 3> kKernel = {{{{1, 2, 1}}, {{2, 4, 2}}, {{1, 2, 1}}}};
const static int kSum = 16;

for (int row = 0; row < img_height; row++) {
  for (int col = 0; col < img_width; col++) {
    int acc = 0;
    for (int dy = -1; dy <= 1; dy++) {
      for (int dx = -1; dx <= 1; dx++) {
        std::uint8_t pixel = GetPixel(pixel_data, img_width, img_height, col + dx, row + dy);
        acc += kKernel.at(1 + dy).at(1 + dx) * static_cast<int>(pixel);
      }
    }
    output[(static_cast<std::size_t>(row) * static_cast<std::size_t>(img_width)) + static_cast<std::size_t>(col)] =
            static_cast<std::uint8_t>(acc / kSum);
  }
}
```

**Обработка граничных случаев:**

Для пикселя (`pos_x`, `pos_y`) применяются ограничения от размеров  исходного изображения (`img_height`, `img_width`).

```cpp
// Фрагмент файла: seq/src/ops_seq.cpp
std::uint8_t KolotukhinAGaussinBlureSEQ::GetPixel(const std::vector<std::uint8_t> &pixel_data, int img_width, 
                                                  int img_height, int pos_x, int pos_y) {
  std::size_t x = static_cast<std::size_t>(std::max(0, std::min(pos_x, img_width  - 1)));
  std::size_t y = static_cast<std::size_t>(std::max(0, std::min(pos_y, img_height - 1)));
  return pixel_data[(y * static_cast<std::size_t>(img_width)) + x];
}
```

Для пикселя на границе, например `pos_x = -1`, функция корректно возвращает значение ближайшего действительного
пикселя. Это позволяет применять ядро 3×3 ко всем пикселям без ветвления в основной логике.

### 1. Результат выполнения теста `test_corner`

Проверяется работа ядра свертки для яркого (`255`) углового пикселя и корректность обработки граничных пикселей.

```text
 255  0   0      143  47   0
  0   0   0  ->   47  15   0
  0   0   0       0    0   0
```

Следующая схема описывает ситуацию, если бы оригинальное изображение было расширено на число пикселей
равное радиусу ядра.

```text
    [-1] [0] [1] [2] [3]
[-1]255  255  0   0   0
[0] 255 |255  0   0|  0     143  47  0
[1]  0  | 0   0   0|  0 ->   47  15  0
[2]  0  | 0   0   0|  0      0   0   0
[3]  0    0   0   0   0
```

В данном примере ядро применяется только внутри выделенного участка.
Внешние значения интенсивностей можно считать «мнимыми»,
поскольку в текущей реализации для вычисления интенсивности пикселя на границе
**копируется** значение ближайшего существующего пикселя. Исходное изображение **не расширяется**.

_Применяем машинное округление при вычислениях: при делении целочисленных значений любая дробная часть
отбрасывается без какого-либо округления._

Для данного примера должны быть проведены вычисления для `[row, col]`:

1. Позиция в исходном изображении `[0,0]`:

```text
255 * 1 + 255 * 2 + 0 * 1 + 
255 * 2 + 255 * 4 + 0 * 2 +
 0  * 1 +  0  * 2 + 0 * 1 =
 = 2295

 2295 / 16 = 143.4375 -> 143 
```

2. Позиция в исходном изображении `[0,1]`:

```text
255 * 1 + 0 * 2 + 0 * 1 + 
255 * 2 + 0 * 4 + 0 * 2 +
 0  * 1 + 0 * 2 + 0 * 1 =
 = 765

 765 / 16 = 47.8125 -> 47 
```

3. Позиция в исходном изображении `[0,2]`:

```text
Все интенсивности = 0 -> Новое значение = 0
```

4. Позиция в исходном изображении `[1,0]`:

```text
255 * 1 + 255 * 2 + 0 * 1 + 
 0  * 2 +  0  * 4 + 0 * 2 +
 0  * 1 +  0  * 2 + 0 * 1 =
 = 765

 765 / 16 = 47.8125 -> 47 
```

5. Позиция в исходном изображении `[1,1]`:

```text
255 * 1 + 0 * 2 + 0 * 1 + 
 0  * 2 + 0 * 4 + 0 * 2 +
 0  * 1 + 0 * 2 + 0 * 1 =
 = 255

 255 / 16 = 15.9375 -> 15 
```

6. Позиция в исходном изображении `[1,2]`, `[2,0]`, `[2,1]`, `[2,2]`:

```text
Все интенсивности = 0 -> Новое значение = 0
```
