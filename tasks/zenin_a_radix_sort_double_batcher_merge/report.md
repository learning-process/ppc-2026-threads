## 1. Introduction

Данная лабораторная работа посвящена реализации параллельного алгоритма поразрядной сортировки (LSD Radix Sort)
для чисел с плавающей точкой двойной точности (`double`) с последующим объединением отсортированных блоков
с помощью алгоритма чётно-нечётного слияния Бэтчера. Целью работы является создание корректно работающих
версий: последовательной (SEQ) и параллельных (OMP, TBB, STL) и анализ производительности.

## 2. Problem Statement

**Задача:** Отсортировать массив элементов типа `double` с использованием поразрядной сортировки
и чётно-нечётного слияния Бэтчера.

**Ограничения:**

- Алгоритм чётно-нечётного слияния Бэтчера требует, чтобы количество сливаемых блоков было
  степенью двойки.

- Данные типа `double` нельзя напрямую сортировать классической поразрядной сортировкой из-за
  особенностей представления чисел с плавающей точкой в памяти (стандарт IEEE 754).

## 3. Baseline Algorithm (Sequential)

Последовательный алгоритм состоит из двух этапов:

1. **Преобразование и сортировка (LSD Radix Sort):** Массив чисел `double` побитово преобразуется
   в массив беззнаковых целых чисел `uint64_t`. Для сохранения правильного порядка выполняется
   битовая манипуляция: для отрицательных чисел инвертируются все биты, для положительных —
   только знаковый бит. Затем применяется 8-проходная побайтовая сортировка подсчётом.
   По завершении числа конвертируются обратно в `double`.
2. **Слияние Бэтчера:** Массив дополняется до ближайшей степени двойки значениями
   `std::numeric_limits<double>::max()`, делится на две половины, каждая сортируется отдельно,
   затем половины сливаются через итеративную сеть чётно-нечётного слияния Бэтчера.

## 4. Parallelization Scheme

**OMP:** Массив дополняется до степени двойки и делится на две половины. Каждая половина
сортируется параллельно через `#pragma omp parallel sections`. Слияние Бэтчера выполняется
с параллельным первым проходом через `#pragma omp parallel for`.

**TBB:** Две половины массива сортируются параллельно через `tbb::parallel_invoke`.
Первый проход слияния Бэтчера распараллелен через `tbb::parallel_for`.

**STL:** Одна половина массива сортируется в отдельном `std::thread`, вторая — в основном потоке.
Первый проход слияния Бэтчера распараллелен через пул `std::thread` с числом потоков равным
`ppc::util::GetNumThreads()`.

## 5. Implementation Details

- **Обработка чисел IEEE 754:** Для применения целочисленной поразрядной сортировки к типу `double`
  выполняется побитовое копирование элементов в тип `uint64_t`. Если число отрицательное —
  инвертируются все биты, если положительное — только старший (знаковый) бит. Это гарантирует
  сохранение естественного порядка при целочисленном сравнении.
- **Дополнение массива (Padding):** Алгоритм Бэтчера требует степени двойки. Массив динамически
  расширяется фиктивными элементами (`std::numeric_limits<double>::max()`) до ближайшей степени
  двойки, с последующим усечением до оригинального размера.
- **Специфика STL:** Для предотвращения создания избыточного числа потоков при большом массиве
  параллельный проход слияния ограничен числом `ppc::util::GetNumThreads()`, что исключает
  деградацию производительности.
- **Специфика ALL:** Реализован механизм одновременной инициализации MPI, OMP, TBB и STL
  для валидации отсутствия конфликтов между их внутренними планировщиками.

## 7. Results and Discussion

### 7.1 Correctness

Корректность всех реализаций подтверждена прохождением 48 функциональных тестов (по 12 на каждую
технологию) через Google Test. Проверены случаи: пустой массив, одиночный элемент, отрицательные
числа, положительные числа, смешанные знаки, дубликаты, уже отсортированный массив, нули и -0.0.

Успешно разработаны и протестированы четыре версии алгоритма: последовательная (SEQ) и три
параллельные (OMP, TBB, STL). Алгоритм корректно обрабатывает особенности представления `double`
по стандарту IEEE 754. TBB-версия оказалась наиболее эффективной благодаря встроенному
планировщику задач. Дальнейший рост ускорения ожидается при увеличении объёма входных данных,
когда полезная работа начнёт доминировать над накладными расходами на параллелизацию.
