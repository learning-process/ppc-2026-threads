# Поиск кратчайших путей из одной вершины — TBB

- Студент: Паньков Андрей Владимирович
- Группа: 3823Б1ПР3
- Технология: Intel oneTBB
- Вариант: 1

## 1. Назначение

TBB-версия использует `tbb::parallel_for` для параллельного построения списка смежности. Планировщик oneTBB сам
разбивает диапазон вершин на блоки и распределяет их между рабочими потоками.

## 2. Схема распараллеливания

Как и в других потоковых версиях, перед параллельным участком выполняются сортировка ребер и построение массива
смещений. Затем диапазон `[0, n)` передается в `tbb::blocked_range`.

Каждая итерация обрабатывает одну вершину:

- берет диапазон ребер из `offsets[from]` и `offsets[from + 1]`;
- резервирует память под список исходящих ребер;
- добавляет пары `(to, weight)` в `adjacency[from]`.

## 3. Детали реализации

Запись в `adjacency` не требует синхронизации, потому что разные итерации работают с разными вершинами. После
параллельной подготовки графа выполняется общий последовательный алгоритм Дейкстры.

Использование TBB удобно тем, что код не создает потоки вручную и не задает размер блоков явно.

## 4. Проверка корректности

TBB-реализация использует общий набор функциональных тестов. Выходной вектор расстояний должен совпадать с эталоном
для базового графа, графа с недостижимой вершиной и графа со стартом из ненулевой вершины.

## 5. Результаты

Локальный performance-тест на графе из `3000` вершин показал около `0.000022` секунд для `task_run` и `0.000542`
секунд для `task_pipeline`.

## 6. Вывод

TBB хорошо подходит для задач с динамической балансировкой, но в данной реализации объем параллельной части мал.
Поэтому планировщик oneTBB не успевает дать заметное преимущество над последовательной версией на тестовом графе.
