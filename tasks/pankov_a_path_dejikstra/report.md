# Сводный отчет: поиск кратчайших путей из одной вершины

- Студент: Паньков Андрей Владимирович
- Группа: 3823Б1ПР3
- Вариант: 1
- Технологии: SEQ, OMP, STL, TBB, ALL

## 1. Введение

В работе рассматривается задача поиска кратчайших расстояний от одной стартовой вершины до всех остальных вершин
ориентированного взвешенного графа. Для графов с неотрицательными весами используется алгоритм Дейкстры.

Реализованы последовательная версия, потоковые варианты с использованием OpenMP, `std::thread` и Intel oneTBB, а
также ALL-версия на MPI. Отчетная ветка добавляет только Markdown-документацию, поэтому может быть объединена
независимо от ветки с исходным кодом ALL; после merge обоих pull request в задаче будут и реализация, и полный набор
отчетов.

## 2. Постановка задачи

**Входные данные:**

- `n` — количество вершин графа.
- `source` — номер стартовой вершины.
- `edges` — список ориентированных ребер вида `(from, to, weight)`.

**Выходные данные:**

- Вектор расстояний от `source` до каждой вершины.
- Для недостижимых вершин используется значение `kInfinity`.

**Ограничения:**

- `n > 0`.
- `source < n`.
- Концы каждого ребра должны принадлежать диапазону `[0, n)`.
- Вес ребра должен быть неотрицательным.

## 3. Базовый алгоритм

Во всех реализациях для вычисления расстояний применяется одинаковое ядро алгоритма Дейкстры:

1. Массив расстояний инициализируется значением `kInfinity`.
2. Для стартовой вершины расстояние устанавливается равным нулю.
3. В приоритетной очереди хранится пара `(distance, vertex)`.
4. На каждой итерации извлекается вершина с минимальной текущей оценкой расстояния.
5. Для всех исходящих ребер выполняется релаксация.
6. Устаревшие записи в очереди пропускаются сравнением с актуальным значением расстояния.

Сложность вычислительной части составляет `O((V + E) log V)` при использовании приоритетной очереди.

## 4. Схема распараллеливания

Последовательная версия строит список смежности прямым проходом по списку ребер.

Параллельные реализации ускоряют этап предварительной обработки графа. Список ребер сортируется по начальной вершине,
после чего строится массив смещений для каждой вершины. Затем разные диапазоны вершин обрабатываются параллельно:

- OMP использует `#pragma omp parallel for`.
- STL делит вершины на блоки и запускает `std::thread`.
- TBB использует `tbb::parallel_for` с `tbb::blocked_range`.
- ALL передает метаданные графа и упакованный список ребер всем MPI-процессам через `MPI_Bcast`, строит одинаковый
  список смежности на каждом процессе, вычисляет расстояния на rank 0 и рассылает итоговый вектор всем участникам.

Основной цикл алгоритма Дейкстры остается последовательным, так как корректный выбор очередной вершины с минимальным
расстоянием является глобальной зависимостью алгоритма.

## 5. Проверка корректности

Функциональные тесты покрывают три сценария:

- базовый связный граф;
- граф с недостижимой вершиной;
- запуск алгоритма из вершины, отличной от нулевой.

Для каждого теста результат сравнивается с заранее рассчитанным вектором кратчайших расстояний. Все реализации
используют общий тип входных и выходных данных, что позволяет запускать один и тот же набор тестов для разных
технологий.

Для ALL-версии та же проверка выполняется под `mpirun`: валидность входных данных синхронизируется между процессами,
а результат после вычисления рассылается всем rank, чтобы поведение pipeline оставалось единым для тестового
фреймворка.

## 6. Тесты производительности

В performance-тестах используется разреженный граф из `3000` вершин. Для каждой вершины `i` добавляются ребра
`i -> i + 1` с весом `1` и `i -> i + 2` с весом `2`. Ожидаемое расстояние до вершины `i` равно `i`.

Такой граф удобен для проверки корректности и воспроизводимости результата: кратчайший путь известен без отдельного
эталонного алгоритма.

## 8. Анализ результатов

На выбранном размере графа вычислительная часть выполняется очень быстро, поэтому накладные расходы параллельной
предобработки становятся заметнее полезной работы. Из-за этого последовательная версия показывает лучшее время.

Параллельные реализации полезны как демонстрация построения структуры графа разными технологиями. ALL-версия добавляет
стоимость MPI-коммуникаций, поэтому на локальном разреженном графе она также не дает выигрыша. Для получения заметного
ускорения требуется либо более тяжелая предобработка, либо другой алгоритмический подход к параллельному поиску
кратчайших путей.

## 9. Выводы

Все реализации используют один и тот же корректный алгоритм Дейкстры и отличаются способом подготовки и распространения
данных графа. Функциональные тесты подтверждают совпадение результатов для всех технологий. Для данной задачи основное
ограничение производительности связано с последовательной природой выбора минимальной вершины в алгоритме Дейкстры.
