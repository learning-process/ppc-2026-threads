# Поиск кратчайших путей из одной вершины — SEQ

- Технология: SEQ
- Вариант: 1

## 1. Назначение

Последовательная реализация служит базовой версией алгоритма Дейкстры. Она задает эталон корректности для потоковых
реализаций и используется как базовая линия при анализе производительности.

## 2. Входные и выходные данные

Вход представлен структурой `GraphInput`, содержащей количество вершин, стартовую вершину и список ориентированных
ребер. Выходом является вектор расстояний от стартовой вершины до всех остальных вершин.

Недостижимые вершины сохраняют значение `kInfinity`.

## 3. Алгоритм

1. В `ValidationImpl` проверяется размер графа, корректность стартовой вершины, границы ребер и неотрицательность весов.
2. В `PreProcessingImpl` строится список смежности.
3. В `RunImpl` запускается алгоритм Дейкстры с приоритетной очередью.
4. В `PostProcessingImpl` проверяется размер выходного вектора.

Для каждой вершины в очереди хранится текущая оценка расстояния. Если извлеченная оценка устарела, она пропускается.
Это позволяет не выполнять операцию decrease-key и использовать стандартную `std::priority_queue`.

## 4. Сложность

- Время: `O((V + E) log V)`.
- Память: `O(V + E)` для списка смежности, массива расстояний и приоритетной очереди.

## 5. Проверка корректности

Тесты проверяют связный граф, граф с недостижимой вершиной и запуск из ненулевой вершины. Для каждого случая результат
сравнивается с заранее известным вектором расстояний.

## 6. Результаты

В performance-тесте используется разреженный граф из `3000` вершин. Локальный запуск показал время порядка
`0.000012` секунд для режима `task_run` и `0.000063` секунд для режима `task_pipeline`.

## 7. Вывод

SEQ-версия является наиболее простой и предсказуемой. На малом тестовом графе она быстрее потоковых вариантов, так как
не несет накладных расходов на создание или планирование параллельных задач.
