## 1. Назначение

OpenMP-версия использует параллелизм на этапе построения списка смежности. Основной алгоритм Дейкстры остается тем же,
что и в последовательной версии.

## 2. Схема распараллеливания

Перед параллельным участком список ребер сортируется по начальной вершине. Затем строится массив `offsets`, задающий
диапазон ребер для каждой вершины.

После этого цикл по вершинам выполняется через:

```cpp
#pragma omp parallel for default(none) schedule(static)
```

Каждый поток обрабатывает свой набор вершин и заполняет только соответствующие элементы списка смежности. Общая запись
в одну и ту же ячейку отсутствует, поэтому дополнительные блокировки не требуются.

## 3. Детали реализации

- Количество потоков берется из `ppc::util::GetNumThreads()`.
- Для каждой вершины заранее вызывается `reserve`, чтобы уменьшить число перевыделений памяти.
- После построения графа запускается общий последовательный `DijkstraSeq`.

## 4. Проверка корректности

OMP-реализация запускается на том же наборе функциональных тестов, что и SEQ. Результат должен полностью совпадать с
эталонным вектором расстояний.

## 5. Результаты

На локальном performance-тесте для графа из `3000` вершин время составило примерно `0.000018` секунд для `task_run` и
`0.000259` секунд для `task_pipeline`.

## 6. Вывод

OpenMP упрощает распараллеливание подготовки данных, но для данного размера графа накладные расходы превышают выигрыш.
Основная вычислительная часть алгоритма остается последовательной из-за глобального выбора минимальной вершины.
