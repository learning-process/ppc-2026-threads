# Поиск кратчайших путей из одной вершины — STL

- Вариант: 1

## 1. Назначение

STL-реализация показывает ручное распараллеливание предобработки графа с помощью `std::thread`. Она использует тот же
формат входных данных и тот же алгоритм Дейкстры, что и последовательная версия.

## 2. Схема распараллеливания

Сначала ребра сортируются по начальной вершине, после чего вычисляются смещения `offsets`. Далее вершины делятся на
почти равные блоки:

```cpp
chunk = (n + num_threads - 1) / num_threads
```

Для каждого непустого блока создается поток. Поток заполняет списки смежности только для вершин своего диапазона.
После запуска все потоки синхронизируются через `join()`.

## 3. Детали реализации

- Количество потоков берется из `ppc::util::GetNumThreads()`.
- Вектор потоков заранее резервируется.
- Параллельная запись безопасна, так как разные потоки работают с разными индексами `adjacency`.
- После построения списка смежности вызывается общее ядро `DijkstraSeq`.

## 4. Проверка корректности

Корректность проверяется сравнением результата с заранее рассчитанными кратчайшими расстояниями. Тесты покрывают
разные стартовые вершины и недостижимые вершины.

## 5. Результаты

Локальный performance-тест на разреженном графе из `3000` вершин показал около `0.000020` секунд для `task_run` и
`0.000452` секунд для `task_pipeline`.

## 6. Вывод

STL-версия дает полный контроль над созданием потоков и разбиением работы, но требует больше ручного кода, чем OpenMP
или TBB. На малом графе стоимость создания и синхронизации потоков заметно влияет на итоговое время.
