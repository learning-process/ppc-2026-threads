## 1. Введение

В данном отчёте представлена параллельная реализация алгоритма Грэхема для построения выпуклой оболочки
множества точек на плоскости с использованием параллельных алгоритмов стандартной библиотеки C++ (STL).
Выпуклая оболочка – это наименьшее выпуклое множество, которое содержит все заданные точки. Применение
политик выполнения std::execution::par позволяет ускорить наиболее трудоёмкий этап алгоритма (сортировку)
за счёт распараллеливания на многопроцессорных системах.

## 2. Постановка задачи

**Входные данные:** Множество точек на плоскости в виде вектора структур Point с координатами типа double
(x, y).

**Выходные данные:** Выпуклая оболочка - последовательность точек в порядке обхода против часовой стрелки,
начиная с самой нижней левой точки.

**Ограничения:**

- Количество точек: от 0 до любого целого.
- Координаты: произвольные вещественные числа.
- Дубликаты точек: допускаются.
- Коллинеарные точки: обрабатываются (в оболочку попадают только крайние).

## 3. Базовый алгоритм

Алгоритм Грэхема в последовательной версии состоит из трёх основных этапов:

1. **Выбор опорной точки** - находится точка с минимальной координатой y (при равенстве - с минимальной x).
2. **Сортировка по полярному углу** - все остальные точки сортируются относительно опорной точки
с использованием векторного произведения.
3. **Проход Грэхема** - построение выпуклой оболочки с использованием стека.

## 4. Схема распараллеливания

### 4.1 Выбор опорной точки

Поиск опорной точки выполняется последовательным линейным проходом, так как параллельные алгоритмы
для поиска минимума с пользовательским компаратором не поддерживаются стандартной библиотекой.

```text
Point FindLowestLeftmost(const std::vector<Point>& points) {
  Point p0 = points[0];
  for (size_t i = 1; i < points.size(); ++i) {
    if (points[i].y < p0.y || (points[i].y == p0.y && points[i].x < p0.x)) {
      p0 = points[i];
    }
  }
  return p0;
}
```

### 4.2 Сортировка по полярному углу

Для сортировки используется параллельная версия алгоритма std::execution::par:

```cpp
std::sort(std::execution::par, sorted.begin(), sorted.end(), PointComparator(p0));
```

**Компаратор для сравнения точек по полярному углу:**

```cpp
class PointComparator {
 public:
  explicit PointComparator(const Point& p0) : p0_(p0) {}
  
  bool operator()(const Point& a, const Point& b) const {
    double o = ((a.x - p0_.x) * (b.y - p0_.y)) - ((a.y - p0_.y) * (b.x - p0_.x));
    if (o != 0.0) return o > 0;
    double dx1 = a.x - p0_.x, dy1 = a.y - p0_.y;
    double dx2 = b.x - p0_.x, dy2 = b.y - p0_.y;
    return (dx1 * dx1 + dy1 * dy1) < (dx2 * dx2 + dy2 * dy2);
  }
 private:
  Point p0_;
};
```

### 4.3 Проход Грэхема

Проход Грэхема остаётся последовательным из-за зависимости от предыдущих точек:

```text
std::vector<Point> stack;
stack.push_back(p0);
stack.push_back(sorted[0]);

for (size_t i = 1; i < sorted.size(); ++i) {
  while (stack.size() >= 2 && Orient(stack[stack.size() - 2], stack[stack.size() - 1], sorted[i]) <= 0.0) {
    stack.pop_back();
  }
  stack.push_back(sorted[i]);
}
```
