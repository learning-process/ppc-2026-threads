## 1. Введение

В данном отчёте представлена параллельная реализация алгоритма Грэхема для построения выпуклой оболочки
множества точек на плоскости с использованием библиотеки Intel TBB (Threading Building Blocks). Выпуклая
оболочка - это наименьшее выпуклое множество, которое содержит все заданные точки. TBB предоставляет
высокоуровневые алгоритмы для параллельного программирования, позволяющие эффективно использовать многоядерные
процессоры без низкоуровневого управления потоками.

## 2. Постановка задачи

**Входные данные:** Множество точек на плоскости в виде вектора структур Point с координатами типа double
(x, y).

**Выходные данные:** Выпуклая оболочка - последовательность точек в порядке обхода против часовой стрелки,
начиная с самой нижней левой точки.

**Ограничения:**

- Количество точек: от 0 до любого целого.
- Координаты: произвольные вещественные числа.
- Дубликаты точек: допускаются.
- Коллинеарные точки: обрабатываются (в оболочку попадают только крайние).

## 3. Базовый алгоритм

Алгоритм Грэхема в последовательной версии состоит из трёх основных этапов:

1. **Выбор опорной точки** – находится точка с минимальной координатой y (при равенстве – с минимальной x).
2. **Сортировка по полярному углу** – все остальные точки сортируются относительно опорной точки
с использованием векторного произведения.
3. **Проход Грэхема** – построение выпуклой оболочки с использованием стека.

## 4. Схема распараллеливания

### 4.1 Параллельный поиск опорной точки (parallel_reduce)

Для поиска точки с минимальными координатами используется алгоритм параллельной редукции. Диапазон точек
разбивается на блоки, каждый поток обрабатывает свой блок, находя локальный минимум, затем результаты
объединяются.

```cpp
Point FindLowestLeftmost(const std::vector<Point>& points) {
  return tbb::parallel_reduce(
    // 1. Разбиение диапазона на блоки
    tbb::blocked_range<size_t>(0, points.size()),
    
    // 2. Начальное значение
    points[0],
    
    // 3. Тело редукции: поиск локального минимума в блоке
    [&](const tbb::blocked_range<size_t>& r, Point init) {
      for (size_t i = r.begin(); i < r.end(); ++i) {
        if (points[i].y < init.y || (points[i].y == init.y && points[i].x < init.x)) {
          init = points[i];
        }
      }
      return init;
    },
    
    // 4. Функция объединения результатов
    [](const Point& a, const Point& b) {
      if (a.y < b.y || (a.y == b.y && a.x < b.x)) return a;
      return b;
    }
  );
}
```

**Декомпозиция данных:**

- blocked_range<size_t>(0, points.size()) – разбивает диапазон индексов на непрерывные блоки.
- Размер блока выбирается автоматически TBB с учётом количества потоков и гранулярности.

**Функция объединения (редукция):**

- Выполняется над результатами всех потоков.
- Использует бинарный оператор, возвращающий точку с меньшими координатами.

### 4.2 Параллельная сортировка (parallel_sort)

После выбора опорной точки выполняется параллельная сортировка остальных точек по полярному углу.

```text
tbb::parallel_sort(sorted.begin(), sorted.end(), PointComparator(p0));
```

**Особенности реализации:**

- parallel_sort использует параллельную версию алгоритма быстрой сортировки.
- Разбиение данных происходит рекурсивно с использованием задач TBB.
- При достижении определённого размера подмассива сортировка переключается на последовательную.

**Компаратор для сравнения точек по полярному углу:**

```cpp
class PointComparator {
 public:
  explicit PointComparator(const Point& p0) : p0_(p0) {}
  
  bool operator()(const Point& a, const Point& b) const {
    double o = ((a.x - p0_.x) * (b.y - p0_.y)) - ((a.y - p0_.y) * (b.x - p0_.x));
    if (o != 0.0) return o > 0;
    double dx1 = a.x - p0_.x, dy1 = a.y - p0_.y;
    double dx2 = b.x - p0_.x, dy2 = b.y - p0_.y;
    return (dx1 * dx1 + dy1 * dy1) < (dx2 * dx2 + dy2 * dy2);
  }
 private:
  Point p0_;
};
```
