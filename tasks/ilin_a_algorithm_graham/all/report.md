## 1. Введение

В данном отчёте представлена гибридная реализация алгоритма Грэхема для построения выпуклой оболочки
множества точек на плоскости с использованием MPI (для межпроцессного взаимодействия) и Intel TBB
(для внутрипроцессного распараллеливания). Такой подход позволяет эффективно использовать как
распределённые вычислительные ресурсы, так и многоядерные процессоры внутри каждого узла.

## 2. Постановка задачи

**Входные данные:** Множество точек на плоскости в виде вектора структур Point с координатами типа double (x, y).

**Выходные данные:** Выпуклая оболочка - последовательность точек в порядке обхода против часовой стрелки,
начиная с самой нижней левой точки.

**Ограничения:**

- Количество точек: от 0 до любого целого.
- Координаты: произвольные вещественные числа.
- Дубликаты точек: допускаются.
- Коллинеарные точки: обрабатываются.

## 3. Базовый алгоритм

Алгоритм Грэхема в последовательной версии состоит из трёх основных этапов: выбор опорной точки,
сортировка по полярному углу и сканирование Грэхема.

## 4. Схема распараллеливания

Гибридная схема сочетает MPI для распределения данных между процессами и TBB для параллельной обработки
внутри каждого процесса.

### 4.1 Распределение данных (MPI)

Каждый MPI-процесс получает свою часть точек. Данные распределяются статически: каждый процесс обрабатывает
свой локальный набор точек.

```text
int rank, size;
MPI_Comm_rank(MPI_COMM_WORLD, &rank);
MPI_Comm_size(MPI_COMM_WORLD, &size);
```

### 4.2 Параллельный поиск глобальной опорной точки

**Локальный поиск (TBB parallel_reduce):** каждый процесс находит опорную точку в своей части данных.

```cpp
Point local_p0 = tbb::parallel_reduce(
    tbb::blocked_range<size_t>(0, points_.size()), points_[0],
    [&](const tbb::blocked_range<size_t>& r, Point init) {
        for (size_t i = r.begin(); i < r.end(); ++i) {
            if (points_[i].y < init.y || (points_[i].y == init.y && points_[i].x < init.x)) {
                init = points_[i];
            }
        }
        return init;
    },
    [](const Point& a, const Point& b) {
        return (a.y < b.y || (a.y == b.y && a.x < b.x)) ? a : b;
    }
);
```

**Глобальная редукция (MPI_Allreduce):** результаты всех процессов объединяются.

```cpp
std::array<double, 2> local_min = {local_p0.y, local_p0.x};
std::array<double, 2> global_min = {};
MPI_Allreduce(local_min.data(), global_min.data(), 2, MPI_DOUBLE, MPI_MIN, MPI_COMM_WORLD);
Point global_p0{global_min[1], global_min[0]};
```

### 4.3 Локальная сортировка (TBB parallel_sort)

Каждый процесс сортирует свои точки относительно глобальной опорной точки.

```text
tbb::parallel_sort(sorted.begin(), sorted.end(), PointComparator(global_p0));
```

### 4.4 Сбор данных со всех процессов (MPI_Allgatherv)

После локальной сортировки данные собираются на всех процессах.

```text
int local_count = sorted.size();
std::vector<int> counts(size), displs(size);
MPI_Allgather(&local_count, 1, MPI_INT, counts.data(), 1, MPI_INT, MPI_COMM_WORLD);

int total_count = 0;
for (int i = 0; i < size; ++i) {
    displs[i] = total_count;
    total_count += counts[i];
}

// Подготовка буфера для передачи координат
std::vector<double> send_buffer(local_count * 2);
for (int i = 0; i < local_count; ++i) {
    send_buffer[i * 2] = sorted[i].x;
    send_buffer[i * 2 + 1] = sorted[i].y;
}

// Сбор данных
std::vector<double> recv_buffer(total_count * 2);
MPI_Allgatherv(send_buffer.data(), local_count * 2, MPI_DOUBLE,
               recv_buffer.data(), counts.data(), displs.data(), MPI_DOUBLE,
               MPI_COMM_WORLD);
```

### 4.5 Глобальная сортировка и построение оболочки

После сбора данных выполняется повторная сортировка всех точек и сканирование Грэхема.

```text
std::vector<Point> global_sorted;
for (int i = 0; i < total_count; ++i) {
    global_sorted.push_back({recv_buffer[i * 2], recv_buffer[i * 2 + 1]});
}
tbb::parallel_sort(global_sorted.begin(), global_sorted.end(), PointComparator(global_p0));

std::vector<Point> hull;
GrahamScan(global_sorted, global_p0, hull);
```
