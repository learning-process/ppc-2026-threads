## 1. Введение

В данном отчёте представлена параллельная реализация алгоритма Грэхема для построения выпуклой оболочки
множества точек на плоскости с использованием технологии OpenMP. Выпуклая оболочка - это наименьшее выпуклое
множество, которое содержит все заданные точки или фигуру. Применение OpenMP позволяет ускорить наиболее
трудоёмкие этапы алгоритма за счёт распараллеливания на многопроцессорных системах.

## 2. Постановка задачи

**Входные данные:** Множество точек на плоскости в виде вектора структур Point с координатами типа double (x, y).

**Выходные данные:** Выпуклая оболочка - последовательность точек в порядке обхода против часовой стрелки,
начиная с самой нижней левой точки.

**Ограничения:**

- Количество точек: от 0 до любого целого.
- Координаты: произвольные вещественные числа.
- Дубликаты точек: допускаются.
- Коллинеарные точки: обрабатываются (в оболочку попадают только крайние).

## 3. Базовый алгоритм

Алгоритм Грэхема в последовательной версии состоит из трёх основных этапов: выбор опорной точки,
сортировка по полярному углу и сканирование Грэхема. В параллельной версии этапы, допускающие
распараллеливание, были адаптированы с использованием OpenMP.

### 3.1 Выбор опорной точки

Находится точка с наименьшей координатой y (самая нижняя). При равенстве y выбирается точка с наименьшей x.
Эта точка гарантированно принадлежит выпуклой оболочке.

В параллельной реализации используется OpenMP для ускорения поиска:

```text
Point FindLowestLeftmost(const std::vector<Point>& points) {
  Point p0 = points[0];
  #pragma omp parallel default(none) shared(points, p0)
  {
    Point local_p0 = p0;
    #pragma omp for nowait
    for (size_t i = 1; i < points.size(); ++i) {
      if (points[i].y < local_p0.y || (points[i].y == local_p0.y && points[i].x < local_p0.x)) {
        local_p0 = points[i];
      }
    }
    #pragma omp critical
    {
      if (local_p0.y < p0.y || (local_p0.y == p0.y && local_p0.x < p0.x)) {
        p0 = local_p0;
      }
    }
  }
  return p0;
}
```

**Схема распараллеливания поиска:**

- Каждый поток обрабатывает свою часть массива точек.
- Найденный локальный минимум сохраняется в local_p0.
- Критическая секция обновляет глобальный минимум.

### 3.2 Сортировка по полярному углу

Все остальные точки сортируются по полярному углу относительно опорной точки. Для сравнения углов
используется векторное произведение.

В OMP-версии для сортировки используется стандартная функция std::ranges::sort (последовательная),
так как параллельная сортировка в OpenMP реализуется нестандартными средствами.

### 3.3 Сканирование Грэхема

Используется стек для построения оболочки. Данный этап является последовательным из-за зависимости
между итерациями.

## 4. Схема распараллеливания

### 4.1 Выбор опорной точки

В последовательной версии опорная точка ищется простым линейным проходом:

```text
Point p0 = points[0];
for (size_t i = 1; i < points.size(); ++i) {
    if (points[i].y < p0.y || (points[i].y == p0.y && points[i].x < p0.x)) {
        p0 = points[i];
    }
}
```

Для распараллеливания используется OpenMP с разделением массива на части:

```text
Point FindLowestLeftmost(const std::vector<Point>& points) {
    Point p0 = points[0];
    
    #pragma omp parallel default(none) shared(points, p0)
    {
        Point local_p0 = p0;
        
        #pragma omp for nowait
        for (size_t i = 1; i < points.size(); ++i) {
            if (points[i].y < local_p0.y || (points[i].y == local_p0.y && points[i].x < local_p0.x)) {
                local_p0 = points[i];
            }
        }
        
        #pragma omp critical
        {
            if (local_p0.y < p0.y || (local_p0.y == p0.y && local_p0.x < p0.x)) {
                p0 = local_p0;
            }
        }
    }
    return p0;
}
```

**Схема работы:**

1. Главный поток создаёт команду параллельных потоков.
2. Каждый поток получает свою часть массива точек (разбиение происходит автоматически).
3. Каждый поток ищет локальный минимум в своей части.
4. После завершения цикла потоки входят в критическую секцию для обновления глобального минимума.
5. Критическая секция гарантирует, что обновление происходит без гонок данных.

### 4.2 Сортировка по полярному углу

```text
std::ranges::sort(sorted, PointComparator(p0));
```

### 4.3 Проход Грэхема

```text
std::vector<Point> stack;
stack.push_back(p0);
stack.push_back(sorted[0]);

for (size_t i = 1; i < sorted.size(); ++i) {
    while (stack.size() >= 2 && Orient(stack[stack.size() - 2], stack[stack.size() - 1], sorted[i]) <= 0.0) {
        stack.pop_back();
    }
    stack.push_back(sorted[i]);
}
```

**Узкие места:**

- Сортировка остаётся последовательной и занимает большую часть времени.
- Сканирование Грэхема имеет последовательную природу.
- Эффективность падает при увеличении числа потоков из-за накладных расходов на синхронизацию.
