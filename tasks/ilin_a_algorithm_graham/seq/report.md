## 1. Введение

В данном отчёте представлена последовательная реализация алгоритма Грэхема.
Этот алгоритм используется для построения выпуклой оболочки множества точек на плоскости.
Выпуклая оболочка – это наименьшее выпуклое множество, которое содержит все заданные точки или фигуру.
Задача является фундаментальной в вычислительной геометрии и находит применение в робототехнике.

## 2. Постановка задачи

**Входные данные:** Множество точек на плоскости в виде вектора структур Point с координатами типа double (x, y).

**Выходные данные:** Выпуклая оболочка - последовательность точек в порядке обхода против часовой стрелки,
начиная с самой нижней левой точки.

**Ограничения:**

- Количество точек: от 0 до любого целого.
- Координаты: произвольные вещественные числа.
- Дубликаты точек: допускаются.
- Коллинеарные точки: обрабатываются (в оболочку попадают только крайние).

## 3. Базовый алгоритм

Алгоритм Грэхема состоит из трёх основных этапов.

### 3.1 Выбор опорной точки

Находится точка с наименьшей координатой y (самая нижняя). При равенстве y выбирается точка с наименьшей x.
Эта точка гарантированно принадлежит выпуклой оболочке.

```text
Point FindLowestLeftmost(const std::vector<Point>& points) {
  Point p0 = points[0];
  for (size_t i = 1; i < points.size(); ++i) {
    if (points[i].y < p0.y || (points[i].y == p0.y && points[i].x < p0.x)) {
      p0 = points[i];
    }
  }
  return p0;
}
```

### 3.2 Сортировка по полярному углу

Все остальные точки сортируются по полярному углу относительно опорной точки. Для сравнения углов
используется векторное произведение:

```text
double Orient(const Point& p, const Point& q, const Point& r) {
  return ((q.x - p.x) * (r.y - p.y)) - ((q.y - p.y) * (r.x - p.x));
}
```

- Положительное значение приводит к левому повороту (меньший угол).
- Отрицательное значение приводит к правому повороту (больший угол).
- Ноль означает, что точки коллинеарны (сортируются по расстоянию).

Компаратор для сортировки:

```text
class PointComparator {
 public:
  explicit PointComparator(const Point& p0) : p0_(p0) {}
  
  bool operator()(const Point& a, const Point& b) const {
    double o = Orient(p0_, a, b);
    if (o != 0.0) return o > 0;
    return DistanceSq(p0_, a) < DistanceSq(p0_, b);
  }
 private:
  Point p0_;
};
```

### 3.3 Проход Грэхема

Используется стек для построения оболочки:

```text
std::vector<Point> stack;
stack.push_back(p0);
stack.push_back(sorted[0]);

for (size_t i = 1; i < sorted.size(); ++i) {
  while (stack.size() >= 2) {
    Point p = stack[stack.size() - 2];
    Point q = stack[stack.size() - 1];
    if (Orient(p, q, sorted[i]) <= 0.0) {
      stack.pop_back();
    } else {
      break;
    }
  }
  stack.push_back(sorted[i]);
}
```

**Принцип работы:**

1. В стек помещаются опорная точка и первая отсортированная точка.
2. Для каждой следующей точки проверяется ориентация двух верхних точек стека и текущей точки.
3. Если ориентация ≤ 0 (правый поворот или коллинеарность), верхняя точка удаляется из стека.
4. В противном случае текущая точка добавляется в стек.
