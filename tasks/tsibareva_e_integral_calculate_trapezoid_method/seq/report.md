## 1. Контекст

Исходная задача заключается в численном вычислении кратных интегралов методом трапеций. Последовательная версия служит
эталоном корректности и базой для измерения ускорения параллельных реализаций. Метод трапеций является классическим
подходом к численному интегрированию, и допускает естественное распараллеливание за счёт независимости вычислений в
узлах сетки.

В частности, если рассмотреть задачу численного интегрирования в пространстве произвольной размерности, можно заметить:
каждое из "оснований" трапеции в определённом измерении имеет коэффициент вхождения в сумму, кратный двум. Например, в
двумерном пространстве граничные значения войдут в сумму по одному разу, а все значения, имеющие координату x в
интервале между ними - по два. Следовательно, можно прийти к выводу: задача приблизительного вычисления интеграла
методом трапеций (а на самом деле, любым аналогичным методом) может быть сведена к суммированию значений с весовыми
коэффициентами.

## 2. Постановка задачи

Входные данные: составная структура `Integral` содержит векторы нижних (`lo`) и верхних (`hi`) пределов интегрирования,
вектор количества шагов разбиения (`steps`), интегрируемую функцию (`f`) и размерность пространства (`dim`).

Выходные данные: приближённое значение кратного интеграла типа `double`.

Примерный подход к решению: вычислить кратный интеграл n-ой степени по прямоугольному параллелепипеду, ограниченному
lo[i] и hi[i] - пределами интегрирования каждого измерения. Следовательно, должна быть использована составная формула
трапеций, обобщённая на многомерный случай: интеграл приближается суммой значений функции в узлах равномерной сетки,
умноженных на весовые коэффициенты (1 для внутренних узлов, 1/2 для граней, 1/4 для рёбер, 1/8 для вершин и так далее) и
на произведение шагов сетки, соответствующие "высотам" трапеций.

Ограничения: dim ≥ 1, lo[i] < hi[i], steps[i] > 0, а также предположено, что функция непрерывна.

## 3. Базовый алгоритм

Алгоритм может быть не только хорошо разделён на вычисление частичных сумм, что будет описано в следующих отчетах о
распараллеливании задачи, но декомпозирован на несколько этапов вычисления и предвычисления значений.

1. Сначала является вычисление шага сетки для каждого измерения: h[i] = (hi[i] - lo[i]) / steps[i]. При этом
   организуется обход всех узлов сетки в лексикографическом порядке.
2. Для каждого узла вычисляются координаты, определяется количество граничных координат, затем вычисляется вес = 0.5\*
   (boundary_count).
3. Затем значение функции умножается на вес и добавляется к сумме.
4. После обхода сумма умножается на произведение шагов.

Временная сложность: O(steps). Память: O(dim\*steps[i]) для каждого i-ого измерения или проще: O(dim).

## 4. Детали реализации

Реализация осуществлена в файлах `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`, причем класс
TsibarevaEIntegralCalculateTrapezoidMethodSEQ наследуется от BaseTask. В реализации предположено использование
неотрицательных критически не корректных значений, таких как dim < 0, steps[i] < 0 и так далее, однако набор тестов
предполагает обработку нулевых случаев. В таком случае, `ValidationImpl` может осуществлять, например, инвертирование
пределов интегрирования или подготовку значений функции, если бы функция интегрирования была вынесена в качестве четко
описанного метода. Однако реализованный подход позволяет использовать произвольную хранимую подынтегральную функцию
пространства произвольной размерности, что делает предвычисление массива значений избыточным. На основании
перечисленного, было принято решение не выделять особую функциональность `ValidationImpl`. В таком случае,
`PreProcessingImpl` обнуляет выходную переменную.

Основная логика вычислений была реализована в методе `RunImpl`, где было осуществлено:

1. Последовательное вычисление шагов без предвычисления.
2. Нелинейный обход узлов d-мерной сетки посредством функции `IterateGridPoints` - инкремента d-мерного счётчика.
   Реализация нелинейного счетчика обхода обусловлена необходимостью вычислений в произвольном пространстве, инкремент
   которого не может быть чётко определён без объёмных предвычислений индексов размерностей измерений, причём можно
   показать, что эта задача имеет временную сложность, близкую к сложности исходной задачи.
3. Вычисление весов и накопление суммы. Заметим, что была реализована специальная функция `ComputePoint`, которая
   восстанавливает координаты точки по индексам.

Заметим, что основным преимуществом, как и особенностью, реализации, является хранение подынтегральной функции как
`std::function`, что обеспечивает произвольность размерности и позволяет не определять дополнительные функции для
эмулирования вычислений.

В общем виде эталонная реализация базового алгоритма может иметь примерно следующий вид:

```cpp
bool RunImpl() {
  int dim = GetInput().dim;

  std::vector<double> h(dim);
  for (int i = 0; i < dim; ++i) {
    h[i] = (GetInput().hi[i] - GetInput().lo[i]) / GetInput().steps[i];
  }

  std::vector<int> indexes(dim, 0);
  double sum = 0.0;

  while (true) {
    std::vector<double> point = ComputePoint(indexes, h, dim);

    int boundary_count = 0;
    for (int i = 0; i < dim; ++i) {
      if (indexes[i] == 0 || indexes[i] == GetInput().steps[i]) boundary_count++;
    }
    double weight = (boundary_count == 0) ? 1.0 : std::pow(0.5, boundary_count);
    sum += weight * GetInput().f(point);

    if (!IterateGridPoints(indexes, dim)) break;
  }

  double res_h = 1.0;
  for (int i = 0; i < dim; ++i) res_h *= h[i];
  GetOutput() = sum * res_h;
  return true;
}
```

## 5. Проверка корректности

Корректность SEQ проверялась на функциональных тестах, включающих: двумерные случаи, трёхмерные случаи, постоянные
функции, нелинейные функции, граничные области, вырожденные области. Примеры функционального тестирования, областей
интегрирования, хранимых функций можно видеть в таблице:

| Тест       | Размерность | Функция | Область     | Шаги  | Ожидание |
| ---------- | ----------- | ------- | ----------- | ----- | -------- |
| Simple2D   | 2           | x²+y²   | [0,1]²      | 100²  | 0.66667  |
| Constant2D | 2           | 5       | [0,2]×[0,3] | 50×50 | 30.0     |
| Simple3D   | 3           | x+y+z   | [0,1]³      | 50³   | 1.5      |
| Constant3D | 3           | 3       | [0,2]³      | 40³   | 24.0     |

А также по результатам тестирования может быть сделан вывод, что расхождения вычисленных значений с аналитическими
значениями не превышают 1e-4.
