## 1. Контекст

Последовательная версия алгоритма вычисления многомерного интеграла методом прямоугольников служит эталоном
корректности для всех параллельных реализаций. Она определяет базовую производительность, относительно которой
измеряется ускорение параллельных версий.

## 2. Постановка задачи

**Входные данные:**

- Подынтегральная функция f(x1, x2, ..., xn) — функция n переменных
- Нижние границы интегрирования a_i для каждой размерности i = 1..n
- Верхние границы интегрирования b_i для каждой размерности i = 1..n
- Число шагов разбиения N

**Выходные данные:**

- Приближенное значение многомерного интеграла

**Ограничения:**

- Размерность `n >= 1`
- Число шагов `N > 0`
- Для каждой размерности должно выполняться `a_i < b_i`
- Функция должна быть определена на всей области интегрирования

**Крайние случаи:**

- Одномерный интеграл
- Минимальное число шагов
- Отрицательные границы интегрирования
- Постоянная функция

## 3. Базовый алгоритм

Метод прямоугольников (метод средних точек) аппроксимирует интеграл суммой значений функции в средних точках
элементарных прямоугольников, умноженной на объем этих прямоугольников.

**Пошаговое описание:**

1. Для каждой размерности i вычисляется шаг разбиения: h_i = (b_i - a_i) / N
2. Вычисляется объем элементарного n-мерного прямоугольника: V = h_1 *h_2* ... * h_n
3. Для каждой размерности предвычисляются координаты средних точек: x_{i,j} = a_i + (j + 0.5) * h_i,
   где j = 0..N-1
4. Перебираются все N^n комбинаций индексов (j_1, j_2, ..., j_n), где каждый j_i принадлежит [0, N-1]
5. Для каждой комбинации индексов формируется точка (x_{1,j_1}, x_{2,j_2}, ..., x_{n,j_n}) и вычисляется
   значение функции
6. Все значения функции суммируются
7. Результат: I = V * сумма f(x)

**Асимптотика по времени:** O(N^n) — экспоненциальная зависимость от размерности

**Асимптотика по памяти:** O(n * N) — для хранения предвычисленных координат средних точек

**Инварианты:**

- Общее число вычислений функции равно N^n
- Каждая точка сетки обрабатывается ровно один раз
- Порядок обхода точек не влияет на результат

**Критерий корректности:**

Для функций, интеграл которых известен аналитически, относительная погрешность должна уменьшаться при увеличении N.
Для постоянной функции f(x) = c результат должен быть точным: I = c * произведение (b_i - a_i).

## 4. Детали реализации

**Файлы:**

- `seq/include/ops_seq.hpp` — объявление класса `BortsovaAIntegralsRectangleSEQ`
- `seq/src/ops_seq.cpp` — реализация методов

**Класс:** `BortsovaAIntegralsRectangleSEQ : public BaseTask`

**Основные методы:**

### ValidationImpl()

Проверяет корректность входных данных:

- Размерность должна быть >= 1
- Число шагов должно быть > 0
- Для каждой размерности нижняя граница должна быть строго меньше верхней
- Функция не должна быть `nullptr`

При нарушении любого условия возвращает `false`, что блокирует выполнение задачи.

### PreProcessingImpl()

Выполняет предвычисления:

1. Вычисляет шаги разбиения для каждой размерности
2. Вычисляет объем элементарного прямоугольника
3. Предвычисляет координаты средних точек для всех размерностей и сохраняет их в векторе `midpoints_`

### RunImpl()

Основной вычислительный цикл:

1. Инициализирует сумму нулем
2. Перебирает все `N^n` точек сетки через единый линейный индекс
3. Для каждого линейного индекса вычисляет многомерные индексы через операции деления и остатка
4. Формирует точку из предвычисленных координат
5. Вычисляет значение функции и добавляет к сумме
6. Умножает сумму на объем элементарного прямоугольника

### PostProcessingImpl()

Сохраняет результат в выходной буфер задачи.

**Обработка граничных случаев:**

- При `N = 1` вычисляется значение функции только в центре области
- При `n = 1` алгоритм корректно работает как одномерное интегрирование
- Отрицательные границы обрабатываются корректно благодаря общей формуле

Реализовано 10 тестовых случаев в `tests/functional/main.cpp`, покрывающих:

Все тесты используют относительную погрешность `1e-3`. Детальное описание тестов см. в корневом `report.md`.
