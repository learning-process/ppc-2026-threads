## 1. Контекст
Ручная std::thread-версия нужна для демонстрации низкоуровневого контроля над распараллеливанием: явного разбиения данных, управления жизненным циклом потоков и синхронизацией.

## 2. Постановка задачи
Умножение квадратных матриц A×B = C с использованием блочного алгоритма Фокса. 
Baseline: последовательная версия seq/ops_seq.cpp.

## 3. Базовый алгоритм
Последовательная версия реализует блочный алгоритм:
1. Разбивает матрицы на блоки размером sqrt(n)
2. В трёх вложенных циклах по блокам перемножает соответствующие блоки
3. Внутренняя функция Matmul() выполняет стандартный тройной вложенный цикл для перемножения блоков с накоплением результата в выходной матрице

Асимптотика: O(n³) по времени, O(n²) по памяти.

## 4. Схема распараллеливания
### Разбиение данных на диапазоны
Используется статическое распределение блоков между потоками:
```
std::vector\<std::pair\<int, int>> blocks = Blocks(count_block);
blocks_per_thread = blocks.size() / num_threads;
```
Каждый поток получает непрерывный диапазон блоков [start_idx, end_idx)
Последний поток получает оставшиеся блоки (при некратном делении)

### Вычисления каждого потока
Для каждого назначенного блока (ic, jc) поток:
1. Вычисляет границы по строкам (istart, iend) и столбцам (jstart, jend)
2. Во внутреннем цикле по kc перебирает все блоки по K
3. Вызывает Matmul() для перемножения текущего блока A и B с накоплением в GetOutput()

### Локальные результаты
Нет локальных буферов — потоки пишут непосредственно в глобальную GetOutput().
Потому что:
1. Разные потоки работают с разными блоками выходной матрицы (разные (ic, jc))
2. Анализ: блоки (ic, jc) не пересекаются по строкам и столбцам, следовательно каждая ячейка C[i][j] обновляется только одним потоком
3. Гонок данных нет, следовательно не требуются atomic, mutex или critical

### Join
join() выполняется после завершения цикла создания всех потоков. Это позволяет потокам работать параллельно, а не последовательно

## 5. Детали реализации
### Файлы
stl/include/ops_stl.hpp — объявление класса с методом Blocks() для генерации списка блоков
stl/src/ops_stl.cpp — реализация с ручным управлением потоками

### Карта поток -> диапазон блоков
blocks = [(0,0), (0,1), ..., (count_block-1, count_block-1)]
num_threads = 4
blocks_per_thread = total_blocks / 4

Поток 0: blocks[0 ... blocks_per_thread-1]
Поток 1: blocks[blocks_per_thread ... 2*blocks_per_thread-1]
...
Поток 3: blocks[3*blocks_per_thread ... end] (остаток)

### Объединение результатов
Объединение не требуется, так как потоки пишут непосредственно в результирующую матрицу в непересекающиеся области.

### Определение числа потоков
```
auto num_threads = static_cast<size_t>(ppc::util::GetNumThreads());
if (num_threads == 0) {
    num_threads = 4;  // значение по умолчанию
}
```

## 6. Проверка корректности
Результат STL-версии сравнивается с ожидаемым результатом в функциональных тестах аналогичных для последовательной версии с точностью 0.0001

### Проверка на отсутствие гонок
- Различные блоки (ic, jc) соответствуют непересекающимся диапазонам строк и столбцов выходной матрицы
- Каждый поток обрабатывает уникальный набор блоков
- Обновления GetOutput()[i*n+j] для разных потоков никогда не попадают в одну ячейку
- Чтение из входных матриц a и b — только для чтения, безопасно
Итог: синхронизация не требуется.
