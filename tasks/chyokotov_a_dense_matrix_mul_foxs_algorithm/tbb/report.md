## 1. Контекст

TBB-версия выбрана как высокоуровневая альтернатива OpenMP для задач с регулярными вычислениями,
обеспечивающая автоматическую балансировку нагрузки, масштабируемость на многоядерных системах и
более гибкое управление задачами без явной работы с потоками.

## 2. Постановка задачи

Умножение квадратных матриц A×B = C с использованием блочного алгоритма Фокса.
Baseline: последовательная версия seq/ops_seq.cpp.

## 3. Базовый алгоритм

Последовательная версия реализует блочный алгоритм:

1. Разбивает матрицы на блоки размером sqrt(n)
2. В трёх вложенных циклах по блокам перемножает соответствующие блоки
3. Внутренняя функция Matmul() выполняет стандартный тройной вложенный цикл для перемножения блоков
с накоплением результата в выходной матрице

Асимптотика: O(n^3) по времени, O(n^2) по памяти.

## 4. Схема распараллеливания

### Примитив

Выбранный примитив - tbb::parallel_for на двух уровнях:

- В RunImpl() — для параллельного выполнения по блокам (blocked_range2d)
- В Matmul() — для параллельного выполнения по строкам (parallel_for по i)

### Диапазон работы - поблочный

Двумерный диапазон блоков (по строкам и столбцам) разбивается на прямоугольные поддиапазоны, каждый
из которых обрабатывается отдельным потоком, такое разбиение сохраняет локальность данных — один
поток обрабатывает непрерывную группу блоков, что минимизирует промахи кэша.
Для Grain size и Partitioner используется автоматическое разбиение TBB и auto_partitioner.

### Ограничение конкуренции

В текущей реализации отсутствует, так как TBB сама управляет числом потоков через глобальный
планировщик.

## 5. Детали реализации

### Относительно последовательной версии

- Подключение заголовочного файла
- Добавление TBB директив в функции Matmul(), RunImpl()

### Локальные аккумуляторы и объединение результатов

- Внешний цикл по блокам обеспечивает, что одна ячейка output обрабатывается только одним потоком
за раз.
- Внутренний parallel_for по i даёт каждому потоку разные строки (разные i), поэтому записи в
output идут в разные области памяти.

TBB показывает отличную балансировку. Благодаря автоматическому разбиению на задачи и краже работы

(overhead) при 2 потоках минимальны

## 9. Выводы

TBB удобен для неоднородных и динамических задач, автоматически балансируя нагрузку. Но для мелких
или тривиально параллельных задач overhead TBB может оказаться выше, чем польза.
