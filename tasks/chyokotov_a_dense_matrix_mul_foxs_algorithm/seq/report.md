## 2. Постановка задачи
### Входные данные:
InType = std::pair\<std::vector\<double>, std::vector\<double>> — две матрицы A и B, хранящиеся в 
виде одномерных векторов построчно.

### Выходные данные:
OutType = std::vector\<double> — матрица C = A × B (тоже построчно)

### Ограничения:
- Матрицы квадратные (размер n × n)
- Размер вычисляется как sqrt(a.size())
- Элементы — double

### Крайние случаи:
- Пустая матрица (n = 0) — корректно обрабатывается
- Матрица 1×1
- Произвольные размеры, не кратные размеру блока

## 3. Базовый алгоритм
1. Вычисляется размер блока block_size = sqrt(n)
2. Вычисляется количество блоков count_block = ceil(n / block_size)
3. Тройной вложенный цикл по блокам ic, jc, kc:
4. Определяются границы блоков по строкам i, j, k
5. Вызывается Matmul() для перемножения текущих блоков, внутри Matmul() — стандартный тройной 
вложенный цикл

Асимптотика по времени: O(n^3) - классическое умножение матриц
Асимптотика по памяти: O(n^2) - хранение трёх матриц (A, B, C)
Критерий корректности: результат совпадает с прямым умножением с точностью до 0.0001

## 4. Детали реализации
- ValidationImpl() - Проверяет, что размеры матриц A и B совпадают (first.size() == second.size())
- PreProcessingImpl() - Очищает и выделяет память под выходную матрицу (resize(n*n, 0.0))
- RunImpl() - Основной алгоритм блочного умножения. Вычисляет блоки.
- Вспомогательные методы:
CalculateBlockSize() — sqrt(n)
CountBlock() — (n + size - 1) / size
Matmul() — перемножение блоков с аккумуляцией в GetOutput()

### Обработанные исключительные случаи:
n = 0 -> ранний return true
Неполные блоки на границах — через std::min(iend, n)
