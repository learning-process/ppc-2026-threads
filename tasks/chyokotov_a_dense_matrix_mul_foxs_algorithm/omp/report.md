## 1. Контекст
Из последовательной версии (SEQ) в OpenMP переносится блочный алгоритм умножения плотных матриц с целью ускорения вычислений за счёт параллельного выполнения итераций на нескольких потоках.

## 2. Постановка задачи
Реализовать параллельное умножение квадратных матриц A × B = C с использованием OpenMP
Baseline: последовательная реализация в seq/ops_seq.cpp, служащая эталоном для проверки корректности и измерения ускорения

## 3. Базовый алгоритм
Последовательная версия реализует блочный алгоритм:
1. Разбивает матрицы на блоки размером sqrt(n)
2. В трёх вложенных циклах по блокам перемножает соответствующие блоки
3. Внутренняя функция Matmul() выполняет стандартный тройной вложенный цикл для перемножения блоков с накоплением результата в выходной матрице

Асимптотика: O(n³) по времени, O(n²) по памяти.

## 4. Схема распараллеливания
Параллелизация происходит для внутренних циклов Matmul()

### Переменные: shared / private
- Shared:
a, b, n, istart, iend, jstart, jend, kstart, kend  -  Чтение
GetOutput() -  Записывание, требуется синхронизация
- Private:
i, j, k, sum  -  Каждый поток имеет свою копию

### atomic
Так как разные потоки могут одновременно обновлять одну и ту же ячейку GetOutput()[index] при обработке разных блоков, то необходимо гарантировать что операция выполняется как неделимая

### Shedule 
Выбран статический shedule по умолчанию (благодаря collapse(2)), так как нет необходимости в динамическом распределении, так как вычислительная нагрузка однородна

## 5. Детали реализации
### Относительно последовательной версии
- Подключение заголовочного файла
- Добавление OMP директив в функции Matmul()

### Устранённые риски гонок
- Одновременная запись в GetOutput() из разных потоков разрешена #pragma omp atomic
- Чтение/запись sum между потоками sum - автоматически private
- Доступ к общим неизменяемым данным (a, b)	shared - безопасно, так как используется только чтение

## 6. Проверка корректности
Результат OpenMP-версии сравнивается с ожидаемым результатом в функциональных тестах аналогичных для последовательной версии с точностью 0.0001

### Характерные случаи
Характерные случаи
- Корректность atomic: при одновременном обновлении одной ячейки разными блоками результат не теряется
- Границы блоков: неполные блоки на границах матрицы корректно обрабатываются через std::min()
- Пустая матрица: ранний выход без ошибок
