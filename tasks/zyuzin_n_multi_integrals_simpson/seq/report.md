Последовательная реализация является эталонной версией алгоритма. Она не использует никаких средств
параллелизма и служит двум целям: проверке математической корректности формулы и базой отсчёта для

## 2. Постановка задачи

### Входные данные

Структура `SimpsonInput` из `common/include/common.hpp`:

- `lower_bounds` — вектор нижних пределов интегрирования по каждому измерению;
- `upper_bounds` — вектор верхних пределов интегрирования по каждому измерению;
- `n_steps` — вектор числа шагов по каждому измерению (должны быть чётными и положительными);
- `func` — подынтегральная функция типа `std::function<double(const std::vector<double>&)>`.

### Выходные данные

Значение интеграла типа `double`.

### Ограничения

- Размеры всех трёх векторов должны совпадать и быть ненулевыми;
- `lower_bounds[i] <= upper_bounds[i]` для каждого измерения;
- `n_steps[i] > 0` и `n_steps[i] % 2 == 0` (требование метода Симпсона);
- `func` должна быть непустой (проверяется через `static_cast<bool>(input.func)`).

## 3. Алгоритм

Реализован составной метод Симпсона на декартовом произведении равномерных 1D сеток.

### Шаг сетки

```cpp
h[dim] = (upper_bounds[dim] - lower_bounds[dim]) / n_steps[dim];
```

### Общее число узлов

```cpp
total_points = ∏(n_steps[dim] + 1)
```

### Линейная нумерация и декодирование

Все узлы нумеруются одним счётчиком `point_idx` от 0 до `total_points - 1`. Многомерный
индекс восстанавливается через смешанную систему счисления:

```cpp
indices[dim] = temp % (n_steps[dim] + 1);
temp        /= (n_steps[dim] + 1);
```

### Вес Симпсона

Тензорное произведение одномерных весов. Одномерный вес для узла с индексом `index` на сетке
из `n` шагов:

```cpp
double GetSimpsonWeight(int index, int n) {
  if (index == 0 || index == n) return 1.0;
  if (index % 2 == 1)          return 4.0;
  return 2.0;
}
```

### Накопление суммы и финальное масштабирование

```cpp
sum += weight * input.func(point);
// ...
factor = ∏(h[dim] / 3.0);
result = sum * factor;
```

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

### Методы класса `ZyuzinNSimpsonSEQ`

`ValidationImpl()` — проверяет совпадение размеров векторов, непустоту области интегрирования,
допустимость пределов и чётность числа шагов.

`PreProcessingImpl()` — сбрасывает `result_` в 0.0 перед каждым запуском.

`RunImpl()` — вызывает `ComputeSimpsonMultiDim()` и сохраняет результат в `result_`.

`PostProcessingImpl()` — копирует `result_` в `GetOutput()`.

`ComputeSimpsonMultiDim()` — основная функция: единственный цикл по `point_idx` от 0 до
`total_points - 1`, на каждой итерации:

1. Выделяет `std::vector<int> indices(num_dims)` и `std::vector<double> point(num_dims)`;
2. Декодирует `point_idx` в индексы и координаты;
3. Вычисляет тензорный вес Симпсона;
4. Накапливает `sum += weight * func(point)`.

`GetSimpsonWeight(int index, int n)` — статическая вспомогательная функция.

## 5. Проверка корректности

Функциональные тесты (12 случаев, допуск `1e-3`):

| ID | Название       | Функция                          | Область              | Ожидание  |
|----|----------------|----------------------------------|----------------------|-----------|
| 0  | 1d_linear      | x                                | [0,1]                | 0.5       |
| 1  | 1d_quadratic   | x^2                              | [0,1]                | 1/3       |
| 2  | 2d_sum         | x+y                              | [0,1]^2              | 1.0       |
| 3  | 2d_product     | x·y                              | [0,1]^2              | 0.25      |
| 4  | 2d_sum_squares | x^2+y^2                          | [0,1]^2              | 2/3       |
| 5  | 2d_constant    | 1                                | [0,2]×[0,3]          | 6.0       |
| 6  | 3d_constant    | 1                                | [0,1]^3              | 1.0       |
| 7  | 3d_sum         | x+y+z                            | [0,1]^3              | 1.5       |
| 8  | 1d_sin         | sin(x)                           | [0,π]                | 2.0       |
| 9  | 2d_sin_cos     | sin(x)·cos(y)                    | [0,π/2]^2            | 1.0       |
| 10 | 2d_exp         | exp(x+y)                         | [0,1]^2              | (e−1)^2   |
| 11 | 3d_product     | x·y·z                            | [0,1]^3              | 1/8       |

Performance-тест: `sin(x)·cos(y)·exp(z)` на `[0,π]×[0,π/2]×[0,1]`,
`n_steps = {180, 180, 120}`, ~4M узлов, ожидание `2·(e−1) ≈ 3.4366`.
