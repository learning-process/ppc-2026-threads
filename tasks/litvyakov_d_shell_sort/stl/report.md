## 1. Введение

В данном отчёте рассматривается реализация алгоритма с использованием `std::thread`.
Это позволяет оценить накладные расходы ручного управления потоками
в сравнении со специализированными фреймворками.

---

## 2. Постановка задачи

Требуется распределить блоки массива между потоками средствами стандартной библиотеки,
обеспечив корректность и отсутствие гонок данных.

---

## 3. Описание алгоритма

Структура та же, что в OMP и TBB:

1. Вычисление числа блоков и их границ.
2. Создание пула потоков, каждый сортирует свой блок.
3. Ожидание завершения всех потоков (`.join()`).
4. Последовательное слияние блоков в главном потоке.

---

## 4. Схема распараллеливания

Создаётся вектор потоков `std::vector<std::thread> thread_pool(parts_count)`. Каждому
потоку передаётся лямбда с захватом ссылки на вектор и границ своего диапазона.

Конфликтов записи нет — диапазоны `[l, r)` у разных потоков не пересекаются. Барьер
синхронизации — цикл `join()` в главном потоке. Мьютексы не используются.

Нагрузка распределяется статически при создании потоков.

## 8. Выводы

При равномерно разделённых данных `std::thread` работает не хуже специализированных
фреймворков. Если балансировка не требуется, использование стандартной библиотеки
полностью оправдано.

## Приложение

```cpp
  std::vector<std::thread> thread_pool(parts_count);

  for (std::size_t i = 0; i < parts_count; ++i) {
    const std::size_t l = bounds[i];
    const std::size_t r = bounds[i + 1];
    thread_pool[i] = std::thread([&vec, l, r]() {
      BaseShellSort(vec.begin() + static_cast<std::ptrdiff_t>(l),
                    vec.begin() + static_cast<std::ptrdiff_t>(r));
    });
  }

  for (auto &t : thread_pool) {
    t.join();
  }
```

Первый цикл запускает потоки, второй дожидается завершения каждого.
