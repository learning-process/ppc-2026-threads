## 1. Контекст

Использование стандартной библиотеки C++ (`std::thread`) позволяет реализовать параллелизм на низком уровне,
предоставляя полный контроль над жизненным циклом потоков. Этот подход полезен для понимания базовых
механизмов многопоточности и оценки накладных расходов на создание и синхронизацию потоков в сравнении
с высокоуровневыми библиотеками, такими как TBB или OpenMP.

## 2. Постановка задачи

Необходимо реализовать параллельный алгоритм фильтрации Гаусса, распределяя нагрузку между объектами
`std::thread`. Задача требует ручного разделения данных и балансировки нагрузки между потоками.

## 3. Базовый алгоритм

Алгоритм применяет ядро свертки 3x3. Математическое описание и детали последовательной реализации приведены
в `seq/report.md`. В STL-версии используется декомпозиция изображения по горизонтальным полосам (строкам).

## 4. Схема распараллеливания

Реализована схема статического распределения строк изображения:

1. Определяется количество доступных потоков через системный вызов или утилиту `ppc::util::GetNumThreads()`.
2. Общее количество строк изображения `height` делится на количество потоков.
3. Вычисляется `rows_per_thread` и остаток `remainder` для обработки случаев, когда `height` не кратно
   числу потоков.
4. Создается вектор объектов `std::thread`.
5. Каждому потоку назначается диапазон строк `[row_start, row_end)`.
6. Потоки вызывают метод `ProcessRows`, который последовательно обрабатывает каждый пиксель в назначенных
   строках.
7. После запуска всех потоков основной поток выполняет `join()`, ожидая завершения вычислений.
8. Такая схема минимизирует вероятность ложного разделения кэша, так как потоки работают с непрерывными
   участками памяти.

## 5. Детали реализации

Основные методы:

```cpp
void VdovinAGaussBlockSTL::ProcessRows(int row_start, int row_end) {
  for (int py = row_start; py < row_end; py++) {
    for (int px = 0; px < width_; px++) {
      ApplyGaussianToPixel(py, px);
    }
  }
}

bool VdovinAGaussBlockSTL::RunImpl() {
  if (input_image_.empty() || output_image_.empty()) {
    return false;
  }

  int num_threads = ppc::util::GetNumThreads();
  int rows_per_thread = height_ / num_threads;
  int remainder = height_ % num_threads;

  std::vector<std::thread> threads;
  threads.reserve(num_threads);

  int offset = 0;
  for (int ti = 0; ti < num_threads; ti++) {
    int count = rows_per_thread + (ti < remainder ? 1 : 0);
    threads.emplace_back(&VdovinAGaussBlockSTL::ProcessRows, this, offset, offset + count);
    offset += count;
  }

  for (auto &th : threads) {
    th.join();
  }

  return true;
}
```

## 9. Выводы

Прямое использование `std::thread` показало результаты, сопоставимые с TBB. Основным преимуществом является
отсутствие зависимости от сторонних библиотек. Однако ручное управление пулом потоков и распределением данных
требует большего объема кода и более тщательного тестирования на наличие ошибок синхронизации.
