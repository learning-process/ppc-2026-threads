## 1. Контекст

В данной работе исследуется распараллеливание алгоритма фильтрации Гаусса с использованием технологии OpenMP.
OpenMP является стандартом де-факто для программирования систем с общей памятью, позволяя эффективно
использовать многоядерные архитектуры современных процессоров без значительного усложнения исходного кода
за счет использования прагм компилятора.

## 2. Постановка задачи

Требуется реализовать параллельную версию фильтра Гаусса 3x3, используя блочное разбиение изображения.
Основная цель — сократить время обработки изображения большого разрешения за счет распределения нагрузки
между несколькими потоками. Последовательная реализация используется как эталон для расчета ускорения.

## 3. Базовый алгоритм

Алгоритм последовательно применяет ядро 3x3 к каждому пикселю. Подробное описание базового алгоритма и
математической модели приведено в `seq/report.md`. Ключевой особенностью является использование функции
`std::clamp` для обработки границ и фиксированного размера блока для оптимизации кэша.

## 4. Схема распараллеливания

Для распараллеливания изображения используется стратегия декомпозиции по блокам:

1. Изображение разбивается на сетку блоков фиксированного размера (32x32 пикселя).
2. Вычисляется общее количество блоков `total_blocks`.
3. С помощью директивы `#pragma omp parallel for` итерации цикла по блокам распределяются между доступными
   потоками.
4. Использование `schedule(static)` обеспечивает равномерное распределение блоков, что оптимально в данной
   задаче, так как вычислительная сложность обработки каждого блока идентична.
5. Каждый поток независимо вычисляет координаты своего блока и вызывает функцию `ApplyGaussianToPixel` для
   каждого пикселя внутри блока.
6. Отсутствие зависимостей по данным (результаты записываются в новый буфер) позволяет избежать
   использования блокировок или атомарных операций.

## 5. Детали реализации

Ключевой фрагмент кода `RunImpl`:

```cpp
bool VdovinAGaussBlockOMP::RunImpl() {
  if (input_image_.empty() || output_image_.empty()) {
    return false;
  }

  int num_blocks_y = (height_ + kBlockSize - 1) / kBlockSize;
  int num_blocks_x = (width_ + kBlockSize - 1) / kBlockSize;
  int total_blocks = num_blocks_y * num_blocks_x;

#pragma omp parallel for schedule(static) default(none) shared(total_blocks, num_blocks_x)
  for (int bi = 0; bi < total_blocks; bi++) {
    int by = (bi / num_blocks_x) * kBlockSize;
    int bx = (bi % num_blocks_x) * kBlockSize;
    int y_end = std::min(by + kBlockSize, height_);
    int x_end = std::min(bx + kBlockSize, width_);
    for (int py = by; py < y_end; py++) {
      for (int px = bx; px < x_end; px++) {
        ApplyGaussianToPixel(py, px);
      }
    }
  }

  return true;
}
```

Директива `default(none)` гарантирует, что все переменные явно классифицированы как `shared` или `private`,
что повышает надежность кода и исключает гонки данных.

## 6. Проверка корректности

Корректность OMP-версии проверялась путем сравнения результирующих контрольных сумм с данными, полученными
от SEQ-версии. Тесты проводились для различного количества потоков (1, 2, 4, 6) на изображениях размером
до 5000x5000 пикселей.

## 9. Выводы

Применение OpenMP позволило значительно повысить производительность фильтрации Гаусса. Основное ускорение
достигается за счет эффективного распределения независимых блоков между ядрами процессора. Наблюдается
стабильное ускорение, хотя на 6 потоках эффективность начинает снижаться из-за конкуренции за доступ к памяти.
