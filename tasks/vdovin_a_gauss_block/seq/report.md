## 1. Контекст

Линейная фильтрация изображений является одной из базовых задач компьютерного зрения, используемой для
предобработки данных, подавления шумов и выделения признаков. Фильтр Гаусса, в частности, позволяет
сглаживать изображения за счет подавления высокочастотных компонент. Последовательная реализация данного
алгоритма критически важна для установления базовой линии производительности и обеспечения эталона
корректности при разработке параллельных версий.

## 2. Постановка задачи

Задано изображение в виде трехканального массива. Требуется применить к каждому пикселю свертку с ядром
Гаусса размером 3x3.

- **Входные данные**: размеры изображения (ширина `W`, высота `H`). Изображение генерируется программно.
- **Выходные данные**: среднее значение яркости всех пикселей результирующего изображения, выступающее в
  роли контрольной суммы.
- **Ограничения**: размеры изображения должны быть не менее 3x3 пикселей.

## 3. Базовый алгоритм

Последовательный алгоритм выполняет проход по всему изображению. Для каждого пикселя вычисляется новое
значение как взвешенная сумма значений его соседей в окне 3x3.
Пошаговое описание:

1. Изображение разбивается на прямоугольные блоки для улучшения локальности кэш-памяти.
2. Выполняется цикл по блокам по вертикали и горизонтали.
3. Внутри каждого блока выполняется цикл по пикселям.
4. Для каждого пикселя и каждого цветового канала:
    - Вычисляются координаты соседних пикселей в окне 3x3.
    - Применяется функция `std::clamp` для обработки граничных условий (повтор граничного значения).
    - Каждое значение умножается на соответствующий коэффициент ядра.
    - Результаты суммируются и нормируются делением на 16.
    - Результирующее значение записывается в выходной буфер.

Асимптотика по времени: `O(W * H * K^2)`, где `K=3`.
Асимптотика по памяти: `O(W * H)` для хранения исходного и результирующего изображений.
Критерий корректности: результат совпадает с эталонным расчетом, выполненным в функциональных тестах.

## 4. Детали реализации

В коде реализованы следующие методы:

- `ValidationImpl`: проверяет, что размеры изображения не меньше 3.
- `PreProcessingImpl`: выделяет память под входной и выходной векторы, инициализирует входные данные.
- `RunImpl`: выполняет обход изображения по блокам и вызывает функцию применения фильтра.
- `PostProcessingImpl`: вычисляет среднее значение всех элементов выходного вектора для получения контрольной суммы.

Ниже представлен основной фрагмент реализации обработки пикселя:

```cpp
void VdovinAGaussBlockSEQ::ApplyGaussianToPixel(int py, int px) {
  for (int ch = 0; ch < kChannels; ch++) {
    int sum = 0;
    for (int ky = -1; ky <= 1; ky++) {
      for (int kx = -1; kx <= 1; kx++) {
        int ny = std::clamp(py + ky, 0, height_ - 1);
        int nx = std::clamp(px + kx, 0, width_ - 1);
        sum += input_image_[(((ny * width_) + nx) * kChannels) + ch] * kKernel.at(ky + 1).at(kx + 1);
      }
    }
    output_image_[(((py * width_) + px) * kChannels) + ch] = static_cast<uint8_t>(std::clamp(sum / kKernelSum, 0, 255));
  }
}
```

## 5. Проверка корректности

Корректность алгоритма проверяется с помощью `Google Test`. Генерируются изображения различных размеров,
и результаты работы `RunImpl` сравниваются с ожидаемыми значениями. Проверяются граничные случаи (3x3) и
стандартные изображения высокого разрешения.
