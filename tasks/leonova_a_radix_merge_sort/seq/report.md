## 1. Введение

Последовательная версия алгоритма поразрядной сортировки с простым слиянием (Radix Merge Sort)
служит эталоном для проверки корректности и базой для оценки ускорения параллельных реализаций.
Алгоритм сочетает поразрядную сортировку для малых подмассивов и сортировку слиянием для больших,
что позволяет эффективно обрабатывать массивы целых чисел со знаком.

## 2. Постановка задачи

- **Вход**: Массив `std::vector<int64_t>` целых чисел со знаком (64-бит).
- **Выход**: Отсортированный по возрастанию массив той же длины.
- **Ограничения**:
  - Размер массива `N ≥ 1`.
  - Алгоритм должен корректно обрабатывать как положительные, так и отрицательные числа.
  - Сортировка должна быть устойчивой в рамках каждого разряда.
  - Алгоритм должен сохранять исходные элементы (без потерь).

## 3. Базовый алгоритм

### 3.1 Общая схема

Алгоритм реализует гибридный подход:

- Для подмассивов размером **≤ 32** элементов применяется **поразрядная сортировка**.
- Для подмассивов размером **> 32** элементов применяется **рекурсивное разделение** (сортировка слиянием)
с последующим слиянием.

### 3.2 Поразрядная сортировка (Radix Sort)

Шаги поразрядной сортировки

1. Преобразование входного подмассива в беззнаковые ключи.

```cpp
uint64_t ToUnsigned(int64_t value) {
    auto unsigned_value = static_cast<uint64_t>(value);
    constexpr uint64_t kSignBitMask = 0x8000000000000000ULL;
    return unsigned_value ^ kSignBitMask;
}
```

2. Для каждого байта (`0..7`):
   - Подсчёт гистограммы частот байта (256 корзин).
   - Вычисление префиксных сумм (позиций записи).
   - Перераспределение элементов во временный массив согласно текущему байту.
   - Копирование отсортированных элементов обратно.

Сложность Radix Sort

- По времени: `O(8 * N) = O(N)` для фиксированной разрядности.
- По памяти: `O(N)` для временных массивов.

### 3.3 Сортировка слиянием (Merge Sort)

Для подмассивов больше порога используется рекурсивное разделение:

1. Рекурсивно сортируются левая и правая половины (применяется Radix Sort для малых фрагментов).
2. Выполняется слияние двух отсортированных половин через `SimpleMerge`.

Сложность Merge Sort

- По времени: `O(N log N)`.
- По памяти: `O(N)` для временного буфера при слиянии.

### 3.4 Гибридная стратегия

Используется итеративный подход с ручным стеком для избежания глубокой рекурсии:

1. Помещаем задачу `(left, right, sorted = false)` в стек.
2. Пока стек не пуст:

   - Если размер `≤ 32` → применяем `Radix Sort`.
   - Если `sorted == false` → разделяем и кладём подзадачи, затем задачу на слияние.
   - Если `sorted == true` → выполняем слияние половин.

Общая сложность

- В среднем: `O(N log N)` с меньшей константой благодаря `Radix Sort` на малых размерах.

## 4. Схема распараллеливания

Данный отчёт описывает последовательную версию. Параллелизм отсутствует.

## 5. Граничные случаи

- **Пустой массив:** отклоняется в `ValidationImpl()`.
- **Массив из одного элемента:** сортировка не требуется.
- **Массив из 32 элементов:** применяется Radix Sort.
- **Массив из 33 элементов:** применяется гибридный подход.
