## 1. Контекст

Ручная многопоточная версия позволяет тонко настроить управление потоками с помощью стандартной библиотеки `<thread>`.
Данный подход исключает непрозрачность автоматических планировщиков времени выполнения и даёт полное управление над
разбиением данных на блоки и синхронизацией.

## 2. Постановка задачи

Задано вычисление выпуклой оболочки на наборе из 500 000 генерируемых точек. Эталон `SEQ` отрабатывает за 0.1275 с.
Цель: достичь максимального ускорения, управляя потоками вручную.

## 3. Базовый алгоритм

Базовый процесс состоит из поиска опорной точки `O(n)`, сортировки по углу `O(n log n)` и сборки оболочки `O(n)`.

## 4. Схема распараллеливания

В STL-реализации параллелизм применен к `O(n)`-задачам:

1. **Распределение блоков:** Массив делится на `num_threads` равных (или почти равных) блоков (чанков). Вычисляются
   границы каждого блока (`lo`, `hi`). Каждому потоку назначается строго один блок, что предотвращает false sharing.
2. **Генерация точек:** Каждый поток независимо генерирует точки в своем индексе диапазона `[lo, hi)`.
3. **Поиск опорной точки:** Каждый поток находит локальный индекс минимума в своем блоке и записывает его в
   предвыделенный массив `local_results`. После завершения всех потоков (через `.join()`) главный поток находит
   глобальный минимум среди `local_results` (всего `num_threads` элементов).

Сортировка выполняется `std::sort` (без `std::execution::par`, чтобы корректно сравнить влияние именно ручных потоков на
`O(n)` фазы), построение стека остается последовательным.

## 5. Детали реализации

Файлы: `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`

```cpp
for (int thread_idx = 0; thread_idx < num_threads; thread_idx++) {
    int lo = thread_idx * chunk;
    int hi = (thread_idx == num_threads - 1) ? n : ((thread_idx + 1) * chunk);
    threads.emplace_back([&pts, &local_results, thread_idx, lo, hi]() {
        local_results[thread_idx] = FindLocalPivot(pts, lo, hi);
    });
}
for (auto &th : threads) {
    th.join();
}
```

Данный фрагмент иллюстрирует ручное создание пула потоков. Лямбда-функция захватывает ссылки на массивы данных (`pts`,
`local_results`) и копирует индексы границ по значению (`lo`, `hi`, `thread_idx`). Вектор `local_results` защищен от
гонок тем фактом, что каждый поток пишет исключительно по своему уникальному индексу `thread_idx`. Метод `join()` явно
дожидается окончания вычислений до начала фазы глобальной редукции (нахождения общего минимума).

## 6. Проверка корректности

## 9. Выводы

Ручное управление потоками через STL показало наилучшие результаты среди внутрипроцессорных параллельных реализаций.
Избежание универсальных накладных расходов планировщиков позволило извлечь максимальную выгоду из подготовки `O(n)`
данных. Однако, сортировка `O(n log n)` не параллелится, поэтому предел ускорения по закону Амдала достигнут уже на 2-4
