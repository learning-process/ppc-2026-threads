## 1. Контекст

Алгоритм Грэхема решает задачу построения минимальной выпуклой оболочки для заданного множества точек.
Последовательная версия служит эталонной реализацией, на которой основываются и с которой сравниваются
все последующие параллельные версии. Она задает структуру пайплайна (Validation -> PreProcessing -> Run).

## 2. Постановка задачи

**Вход:** целое число `n >= 0` — количество точек.
**Генерация данных:** `n` точек равномерно располагаются на единичной окружности
(`cos(2*pi*i/n), sin(2*pi*i/n)`), при `n > 3` добавляется центральная точка `(0, 0)`.
**Выход:** целое число — количество вершин выпуклой оболочки.
**Ограничения и крайние случаи:** `n >= 0`. При `n = 0` оболочка пустая, при `n = 1` содержит одну точку.
Центральная точка (если добавлена) не должна попасть в результирующую оболочку.

## 3. Базовый алгоритм

Последовательный алгоритм Грэхема включает следующие этапы:

1. Выбор опорной точки (с минимальной Y, а при равенстве — с минимальной X).
2. Сортировка остальных точек по полярному углу относительно опорной (против часовой стрелки).
3. Линейный проход со стеком: точки добавляются в стек, и если образуется правый поворот
   (векторное произведение <= 0), точка извлекается из стека.

Асимптотика по времени: O(n log n) из-за этапа сортировки.
Асимптотика по памяти: O(n) для хранения исходного массива и стека (результата).
Инвариант: по окончании прохода в стеке хранятся вершины минимального выпуклого многоугольника,
охватывающего все заданные точки.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

В `ValidationImpl` проверяется неотрицательность входа.
В `PreProcessingImpl` происходит генерация точек (используется структура `std::pair<double, double>` для совместимости).
В `RunImpl` реализована основная логика алгоритма, инкапсулированная в функции `FindPivot`, `AngleSort` и `BuildHull`.
В `PostProcessingImpl` возвращается размер полученной оболочки.

```cpp
for (const auto &p : pts) {
    while (hull.size() > 1 &&
           CrossProduct(hull[hull.size() - 2], hull.back(), p) <= 0.0) {
        hull.pop_back();
    }
    hull.push_back(p);
}
```

Данный фрагмент иллюстрирует финальную фазу построения оболочки. Стек `hull` корректируется
при каждом добавлении новой точки.

## 5. Проверка корректности

Результат реализации SEQ принимается за эталон. Корректность подтверждается встроенными
тестами курса `ppc_func_tests`.
В качестве тестов были проверены крайние случаи: массив из 0 и 1 точки (вернул 0 и 1),
а также коллинеарные точки на одной прямой.

центральная точка `(0,0)` отсекается алгоритмом.
