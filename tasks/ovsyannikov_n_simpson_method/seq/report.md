## 1. Контекст

Последовательная реализация метода Симпсона является фундаментом всей работы.
Она служит «золотым эталоном» (baseline) для проверки корректности всех
последующих параллельных версий и необходима для расчета ускорения (speedup).
Без строго выверенной последовательной версии невозможно подтвердить
отсутствие ошибок в параллельных вычислениях.

## 2. Постановка задачи

**Объект:** Двойной интеграл функции $f(x, y) = x + y$ на прямоугольной области
$[ax, bx] \times [ay, by]$.
**Входные данные:**

- Границы по X (`ax`, `bx`) и Y (`ay`, `by`) типа `double`.
- Количество разбиений по осям (`nx`, `ny`) типа `int`.
**Ограничения:**
- Значения `nx` и `ny` должны быть строго положительными и четными.
- Если границы по любой оси совпадают ($a = b$), результат должен быть равен 0.0.

## 3. Базовый алгоритм

Алгоритм основан на составной формуле Симпсона (методу парабол):

1. Вычисляются шаги сетки: $hx = (bx - ax) / nx$ и $hy = (by - ay) / ny$.
2. Проводится итерация по узлам сетки $(i, j)$, где $i \in [0, nx], j \in [0, ny]$.
3. Для каждого узла вычисляется коэффициент веса $w_{i,j} = C_i \times C_j$, где:
   - $C = 1$, если индекс равен $0$ или $n$;
   - $C = 4$, если индекс нечетный;
   - $C = 2$, если индекс четный (и не крайний).
4. Накапливается сумма: $S = \sum (w_{i,j} \times f(x_i, y_j))$.
5. Итоговое значение: $I = S \times (hx \times hy) / 9.0$.

**Асимптотика по времени:** $O(nx \times ny)$ — необходимо посетить каждый узел сетки.
**Асимптотика по памяти:** $O(1)$ — хранятся только текущие суммы и параметры.
**Инвариант:** Результат не должен зависеть от порядка суммирования.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.

- `ValidationImpl`: Проверяет, что $nx, ny > 0$ и они четные.
- `PreProcessingImpl`: Инициализирует локальные переменные параметров и обнуляет результат.
- `RunImpl`: Содержит вложенные циклы для обхода сетки и накопления взвешенной суммы.
- `PostProcessingImpl`: Записывает финальное вычисленное значение в поле вывода задачи.

Логика выбора коэффициентов вынесена в статический метод `GetCoeff`, что снижает
когнитивную сложность основного расчетного метода.

## 5. Проверка корректности

Результат сравнивается с аналитическим значением интеграла для $f(x, y) = x + y$:
$$I_{analyt} = \frac{(bx - ax)(by - ay)(ax + bx + ay + by)}{2}$$
Используются функциональные тесты:

1. `normal_10x10`: Стандартная сетка на области $[0,1] \times [0,1]$, ожидаемый выход `1.0`.
2. `zero_width`: Граничный случай $ax = bx$, ожидаемый выход `0.0`.
3. `negative_range`: Область с отрицательными координатами.
