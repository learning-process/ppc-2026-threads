# Вычисление многомерных интегралов по методу Симпсона — OMP

- **Student**: Овсянников Никита
- **Technology**: OMP (OpenMP)
- **Variant**: 11

## 1. Контекст

Данная версия переносит вычисления метода Симпсона на технологию OpenMP. Целью является
сокращение времени расчета за счет распределения итераций внешнего цикла между потоками
процессора с использованием модели разделяемой памяти. OpenMP выбран как наиболее
эффективный инструмент для параллелизма на уровне циклов.

## 2. Постановка задачи

Краткая постановка совпадает с базовой (SEQ). В качестве эталона для сравнения времени
используется последовательная версия со временем выполнения 0.023865 с.

## 3. Базовый алгоритм

Алгоритм реализует двумерную схему Симпсона: накопление взвешенной суммы значений функции
в узлах сетки (коэффициенты 1, 2 или 4) с последующим масштабированием результата на
величину шага сетки по осям X и Y.

## 4. Схема распараллеливания

- **Область:** Распараллеливанию подвергнут внешний цикл по индексу `i` (строки сетки).
- **Атрибуты переменных:** Использован модификатор `default(none)` для обеспечения
  максимальной безопасности. Параметры задачи (`nx_l`, `ax_l`, `hx` и т.д.) объявлены
  как `shared`.
- **Редукция:** Для накопления итоговой суммы `total_sum` применена директива
  `reduction(+:total_sum)`, что позволяет избежать состояния гонки (race condition)
  без использования явных критических секций.
- **Синхронизация:** Используется неявная барьерная синхронизация в конце
  директивы `parallel for`.

## 5. Детали реализации

Файлы: `omp/include/ops_omp.hpp`, `omp/src/ops_omp.cpp`.
Основные изменения относительно SEQ включают:

1. Локальное копирование полей класса в стек потока перед циклом.
2. Применение прагмы `#pragma omp parallel for`.
3. Использование вспомогательного метода `GetCoeff` для вычисления весов, что снизило
   когнитивную сложность метода `RunImpl`.

## 6. Проверка корректности

Версия OMP прошла полный набор функциональных тестов. Результаты идентичны
последовательной версии, что подтверждает корректность использования механизмов редукции.

## 7. Экспериментальная среда

- **CPU**: 13th Gen Intel(R) Core(TM) i5-13500H (16 логических потоков).
- **Compiler**: MSVC v143.
- **Build Type**: Release.
- **Команды запуска**: `./build/bin/Release/ppc_perf_tests.exe --gtest_filter=*OMP*`.

## 8. Результаты

Замеры на сетке 2000x2000:

| Mode | Time (s) | Speedup | Efficiency |
|------|----------|---------|------------|
| SEQ  | 0.0238   | 1.00    | 100%       |
| OMP  | 0.0052   | 4.58    | 28.6%      |

## 9. Выводы

Технология OpenMP продемонстрировала наилучшее масштабирование на системе Windows,
достигнув ускорения в 4.58 раза. Это подтверждает её эффективность для задач с
интенсивными вычислениями в циклах и минимальными зависимостями по данным.
