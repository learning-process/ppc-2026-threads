# Вычисление многомерных интегралов по методу Симпсона — STL

- **Student**: Овсянников Никита
- **Technology**: STL (Standard Template Library Threads)
- **Variant**: 11

## 1. Контекст

Данная реализация использует стандартные механизмы многопоточности C++11 (`std::thread`).
Целью является реализация параллелизма без использования сторонних библиотек (OpenMP/TBB),
что обеспечивает максимальную переносимость кода и позволяет вручную контролировать
жизненный цикл потоков и распределение вычислительной нагрузки.

## 2. Постановка задачи

Краткая постановка совпадает с базовой (SEQ). В качестве эталона для сравнения времени
используется последовательная версия со временем выполнения 0.023865 с.

## 3. Базовый алгоритм

Алгоритм реализует двумерную схему Симпсона: накопление взвешенной суммы значений функции
в узлах сетки с последующим масштабированием результата на величину шага интегрирования.

## 4. Схема распараллеливания

- **Декомпозиция:** Использована ручная декомпозиция данных. Общее количество строк сетки
  ($nx + 1$) делится на равные блоки (chunks) между доступными аппаратными потоками.
- **Потоки:** Создается вектор объектов `std::thread`. Количество потоков определяется
  через `ppc::util::GetNumThreads()`.
- **Синхронизация и редукция:** С целью исключения состояния гонки (race condition) и отказа
  от дорогостоящих мьютексов, каждый поток записывает свой результат в выделенную ячейку
  вектора `partial_results`.
- **Завершение:** Основной поток вызывает `join()` для всех созданных потоков, после чего
  выполняет финальную редукцию через `std::accumulate`.

## 5. Детали реализации

Файлы: `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`.
Основные технические решения:

1. Расчет `chunk_size` и `extra` итераций для равномерного распределения нагрузки.
2. Использование лямбда-функции `worker` для выполнения расчетов внутри потоков.
3. Применение `static_cast` и `std::cmp_less` для обеспечения совместимости с типами и
   удовлетворения требований статического анализатора (Clang-Tidy).

## 6. Проверка корректности

Корректность STL-версии подтверждена функциональными тестами, включая проверку точности
на больших сетках и обработку краевых случаев (нулевая площадь интегрирования).

## 7. Экспериментальная среда

- **CPU**: 13th Gen Intel(R) Core(TM) i5-13500H (16 логических потоков).
- **Compiler**: MSVC v143.
- **Build Type**: Release.
- **Команды запуска**: `./build/bin/Release/ppc_perf_tests.exe --gtest_filter=*STL*`.

## 9. Выводы

Ручное управление потоками через `std::thread` позволило достичь ускорения в 2.11 раза.
Технология эффективна, однако требует более сложного кода для управления диапазонами
итераций и ручной сборки результатов по сравнению с высокоуровневыми директивами OpenMP.
