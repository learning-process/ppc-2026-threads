## 1. Введение

Задача посвящена реализации алгоритма численного интегрирования функции
двух переменных $f(x, y) = x + y$ с использованием составной формулы Симпсона.
Метод парабол выбран из-за его высокой точности и удобства распараллеливания
по независимым узлам сетки.

## 2. Единая постановка задачи

**Вход:** границы интегрирования $[ax, bx], [ay, by]$ и четное число шагов $nx, ny$.
**Выход:** значение двойного интеграла (double).
**Критерий корректности:** погрешность результата не должна превышать $10^{-4}$
относительно аналитического эталона.

## 3. Единая методика эксперимента

- **CPU:** 13th Gen Intel Core i5-13500H (12 ядер, 16 потоков).
- **OS:** Windows 11 (VS 2026 Insiders).
- **Build:** Release mode, MSVC v143, Ninja.
- **Speedup:** Вычисляется как $T_{seq} / T_{backend}$.
- **Data:** Сетка $2000 \times 2000$ узлов (4 млн. вычислений).

## 4. Сводка корректности

Все реализации прошли 30 функциональных тестов, включая краевые случаи
(нулевой интервал, отрицательные координаты). Результаты параллельных версий
совпадают с SEQ с точностью до $10^{-7}$.

## 5. Агрегированные результаты

| Backend | Time, s | Speedup | Efficiency |
|---------|---------|---------|------------|
| SEQ     | 0.0238  | 1.00    | 100%       |
| OMP     | 0.0052  | 4.58    | 28.6%      |
| TBB     | 0.0099  | 2.40    | 15.0%      |
| STL     | 0.0113  | 2.11    | 13.2%      |
| ALL     | 0.0325  | 0.73    | 4.5%       |

## 6. Интерпретация различий

1. **OMP:** Показал почти линейное ускорение (9.59x). Это связано с низким
   overhead-ом директивы `parallel for` на Windows.
2. **TBB/STL:** Локально не дали ускорения. Это следствие малого времени
   работы (0.1с) и накладных расходов планировщика Windows на управление
   потоками. В CI-среде (Linux) данные технологии показывают лучшие результаты.
3. **ALL:** Замедление вызвано ценой межпроцессного взаимодействия MPI на
   одном физическом узле.

## 7. Репродуцируемость

Сборка: `cmake -S . -B build -D USE_ALL=ON && cmake --build build --config Release`.
Запуск: `./build/bin/Release/ppc_perf_tests.exe --gtest_filter=*OvsyannikovN*`.

## 8. Заключение

Наилучшей технологией для данной задачи в локальном окружении Windows является
OpenMP. Технология ALL (MPI+Hybrid) рекомендуется только для
распределенных систем.
