# Вычисление многомерных интегралов по методу Симпсона — TBB

- **Student**: Овсянников Никита
- **Technology**: TBB (Intel Threading Building Blocks)
- **Variant**: 11

## 1. Контекст

Данная реализация переносит вычисления метода Симпсона на библиотеку Intel oneTBB.
Целью является использование высокоуровневых абстракций и планировщика задач
(Task Scheduler) для эффективного распределения нагрузки между ядрами процессора.

## 2. Постановка задачи

Краткая постановка совпадает с базовой (SEQ). В качестве эталона для сравнения
времени используется последовательная версия со временем выполнения 0.023865 с.

## 3. Базовый алгоритм

Алгоритм реализует двумерную схему Симпсона: расчет взвешенной суммы значений
функции в узлах сетки с последующим масштабированием результата на шаг сетки.

## 4. Схема распараллеливания

- **Примитив:** Использован алгоритм `tbb::parallel_reduce`. Выбор обусловлен тем,
  что задача сводится к редукции (суммированию) значений. Это позволяет избежать
  использования явных мьютексов и минимизировать накладные расходы.
- **Диапазон:** Применяется `tbb::blocked_range<int>` для внешнего цикла по строкам.
- **Редукция:** Локальные суммы потоков объединяются с помощью функтора
  `std::plus<>()` (прозрачный функтор C++14).
- **Декомпозиция:** TBB автоматически разбивает диапазон итераций на части,
  балансируя нагрузку между потоками.

## 5. Детали реализации

Файлы: `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`.
Реализация использует лямбда-выражение для описания тела цикла. Для безопасности
используются локальные копии параметров задачи (`nx_l`, `ax_l`, `hx` и т.д.),
захватываемые лямбдой. Коэффициенты Симпсона рассчитываются вспомогательным
методом `GetCoeff`.

## 6. Проверка корректности

Корректность TBB-версии подтверждена функциональными тестами. Результаты
полностью совпадают с аналитическим эталоном $f(x,y)=x+y$ и SEQ версией.

## 7. Экспериментальная среда

- **CPU**: 13th Gen Intel(R) Core(TM) i5-13500H (16 логических потоков).
- **Compiler**: MSVC v143.
- **Build Type**: Release.
- **Команды запуска**: `./build/bin/Release/ppc_perf_tests.exe --gtest_filter=*TBB*`.

## 8. Результаты

Замеры на сетке 2000x2000 узлов:

| Mode | Time (s) | Speedup | Efficiency |
|------|----------|---------|------------|
| SEQ  | 0.0238   | 1.00    | 100%       |
| TBB  | 0.0099   | 2.40    | 15.0%      |

## 9. Выводы

TBB-реализация показала стабильное ускорение в 2.4 раза. Несмотря на более высокие
накладные расходы планировщика задач по сравнению с OpenMP для данной размерности,
технология TBB обеспечивает лучшую переносимость и гибкость управления ресурсами.
