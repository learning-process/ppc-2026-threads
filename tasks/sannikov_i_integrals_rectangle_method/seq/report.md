## 1. Контекст

Данная реализация является эталонной последовательной версией алгоритма
вычисления многомерных определённых интегралов методом средних прямоугольников.
Она служит базовой линией для проверки корректности и расчёта ускорения
в параллельных версиях.

## 2. Постановка задачи

На вход подаётся подынтегральная функция `std::function<double(const std::vector<double>&)>`,
вектор пар границ интегрирования `std::vector<std::pair<double, double>>` для каждого
измерения и число разбиений `int n`. Необходимо вычислить приближённое значение
многомерного интеграла.

## 3. Базовый алгоритм

1. **Построение сетки:** Для каждого измерения вычисляется шаг
   `h[i] = (right - left) / n` и объём элементарной ячейки `cell_v` как
   произведение всех шагов.
2. **Обход ячеек:** Все `n^dim` ячеек перебираются с помощью многомерного
   индекса, инкрементируемого функцией `NextIndex` (счётчик в системе счисления
   с основанием `n`).
3. **Вычисление суммы:** Для каждой ячейки вычисляется координата центра
   `x[i] = left + (idx[i] + 0.5) * h[i]`, значение функции прибавляется к сумме.
4. **Результат:** Интеграл равен `sum * cell_v`.

Асимптотика по времени: $O(dim \cdot n^{dim})$ (перебор всех ячеек с вычислением
координат). Асимптотика по памяти: $O(dim)$ на вспомогательные массивы.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

```cpp
  std::vector<int> idx(dim, 0);
  std::vector<double> x(dim);
  double sum = 0.0;

  while (true) {
    for (std::size_t i = 0; i < dim; ++i) {
      const double left_border = borders[i].first;
      x[i] = left_border + ((static_cast<double>(idx[i]) + 0.5) * h[i]);
    }

    const double fx = func(x);
    if (!std::isfinite(fx)) {
      return false;
    }

    sum += fx;

    if (!NextIndex(idx, dim, n)) {
      break;
    }
  }

  GetOutput() = sum * cell_v;
```

Фрагмент демонстрирует основной вычислительный цикл. `ValidationImpl` проверяет
конечность границ, что левая граница строго меньше правой, наличие функции и
положительность `n`. `PreProcessingImpl` обнуляет результат. `RunImpl` производит
основную вычислительную работу.

## 5. Проверка корректности

Корректность алгоритма подтверждается функциональными тестами. Тестируются
различные типы подынтегральных функций: константная, линейная, квадратичная,
тригонометрическая и экспоненциальная. Результат сравнивается с аналитическим
значением интеграла с точностью `eps = 1e-3`.

области `[0, π] × [0, π]` с `n = 10000` (10⁸ ячеек) за ~1.36 с. Алгоритм
корректно обрабатывает произвольное число измерений и валидирует входные данные.

ускорения и эффективности всех параллельных бэкендов.
