## 1. Контекст

Ручная thread-based версия на базе стандартной библиотеки C++ позволяет реализовать параллелизм без зависимостей от
внешних библиотек (OpenMP/TBB). Это дает возможность полностью контролировать жизненный цикл каждого потока и оценивать
накладные расходы на низкоуровневое управление ресурсами ОС.

## 2. Постановка задачи

Краткая постановка: маркировка связных областей. SEQ-версия ([seq/report.md](../seq/report.md)) считается эталоном
корректности.

## 3. Базовый алгоритм

Последовательная версия использует BFS. В STL-версии реализована схема со сканированием независимых горизонтальных полос
изображения с последующим слиянием через DSU.

## 4. Схема распараллеливания

- **Разбиение на диапазоны:** Изображение делится статически на N горизонтальных полос. Формула для индекса строк:
  `row_start = (tid * height) / num_threads`, `row_end = ((tid + 1) * height) / num_threads`.
- **Работа потока:** Каждый поток выполняет функцию `ScanStripe` для своей полосы.
- **Локальные результаты:** Все потоки пишут в общий массив `local_labels`, но в непересекающиеся области (строки),
  соответствующие их диапазонам. Для уникальности меток используется `label_offset = tid * max_labels_per_thread`.
- **Join:** Вызов `join()` выполняется в отдельном цикле **после** того, как все потоки были созданы и запущены. Это
  обеспечивает реальный параллелизм, в отличие от последовательного создания и ожидания.
- **Синхронизация:** Явная синхронизация (mutex/atomic) внутри потоков не требуется, так как данные (строки)
  изолированы. Барьером служит этап `join()`.

## 5. Детали реализации

- **Файлы:** `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`
- **Карта поток -> диапазон:** Прямое отображение ранга потока (`tid`) на индекс строки.
- **Объединение:** Выполняется последовательно в основном потоке через функции `MergeHorizontal` и `MergeBoundaries`
  после завершения всех рабочих потоков.

```cpp
std::vector<std::thread> threads(static_cast<size_t>(num_threads));
for (int tid = 0; tid < num_threads; ++tid) {
  const int row_start = (tid * height_) / num_threads;
  const int row_end = ((tid + 1) * height_) / num_threads;
  const int label_offset = tid * max_labels_per_thread;
  threads[static_cast<size_t>(tid)] =
      std::thread(ScanStripe, row_start, row_end, width_, label_offset, std::cref(image_), std::ref(local_labels));
}
for (auto &thr : threads) {
  thr.join();
}
```

## 6. Проверка корректности

Сравнение с SEQ на 13 функциональных тестах.

- **Крайние случаи:** Проверена работа на 1 потоке и на изображениях, где высота меньше числа потоков (пустые диапазоны
  обрабатываются корректно).
- **Гонки:** За счет статической изоляции диапазонов строк в `local_labels` и использования `label_offset` гонки данных
  исключены.
