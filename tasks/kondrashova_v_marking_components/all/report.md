## 1. Контекст
Гибридная версия (MPI + OpenMP) позволяет масштабировать решение на кластерные системы. OpenMP используется для параллелизма внутри одного вычислительного узла (процесса), а MPI — для взаимодействия между узлами.

## 2. Постановка задачи
Краткая постановка: маркировка связных областей. SEQ-версия ([seq/report.md](../seq/report.md)) — baseline.

## 3. Базовый алгоритм
Изображение делится на крупные сегменты между процессами MPI. Внутри каждого сегмента выполняется параллельное сканирование полос через OpenMP. Слияние границ между процессами выполняется через обмен граничными строками и DSU.

## 4. Схема распараллеливания
Алгоритм представляет собой сложную многоэтапную схему:
1.  **Рассылка данных:** Ранг 0 рассылает параметры изображения и сами данные всем процессам через `MPI_Bcast`.
2.  **Гибридное сканирование (MPI+OMP):** Каждый процесс определяет свой диапазон строк (`mpi_row_start`, `mpi_row_end`). Внутри этого диапазона строки делятся между потоками OpenMP через `#pragma omp parallel` с ручным расчетом индексов.
3.  **Слияние границ MPI:** Соседние процессы обмениваются метками граничных строк через `MPI_Sendrecv` и обновляют свои структуры DSU.
4.  **Глобальная синхронизация:** С помощью `MPI_Allreduce` с операцией `MPI_MAX` все процессы собирают актуальную информацию о метках.
5.  **Финальная обработка:** Ранг 0 выполняет итоговую перемаркировку и рассылает финальный результат (`global_count` и `labels_1d_`) всем процессам через `MPI_Bcast`.

## 5. Детали реализации
*   **Файлы:** `all/include/ops_all.hpp`, `all/src/ops_all.cpp`
*   **Синхронизация:** Используются барьеры `MPI_Barrier` и коллективные операции для обеспечения целостности данных.
*   **Смещение меток:** Для уникальности меток используется глобальное смещение, учитывающее ранг процесса и номер потока: `(rank_ * num_threads * max_labels_per_thread) + (tid * max_labels_per_thread)`.

```cpp
#pragma omp parallel num_threads(num_threads) default(none) shared(local_labels, width, image, num_threads) \
    firstprivate(mpi_row_start, mpi_rows, max_labels_per_thread)
{
    const int tid = omp_get_thread_num();
    const int omp_row_start = mpi_row_start + ((tid * mpi_rows) / num_threads);
    const int omp_row_end = mpi_row_start + (((tid + 1) * mpi_rows) / num_threads);
    const int label_offset = (rank_ * num_threads * max_labels_per_thread) + (tid * max_labels_per_thread);
    ScanStripe(omp_row_start, omp_row_end, width, label_offset, image, local_labels);
}
```
