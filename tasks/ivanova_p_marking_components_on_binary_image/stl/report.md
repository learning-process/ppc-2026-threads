## 1. Контекст

Версия на `std::thread` показывает ручное управление потоками и явное разбиение данных. Это позволяет сравнить
runtime-ориентированный подход с ручным управлением и понять цену `create/join`.

## 2. Постановка задачи

Та же, что и в `seq/report.md`.

## 3. Базовый алгоритм

Алгоритм повторяет SEQ, но разбивает изображение на полосы строк, обрабатывая каждую полосу в отдельном потоке.

## 4. Схема распараллеливания

Декомпозиция:

- количество потоков `num_threads = ppc::util::GetNumThreads()`,
- каждая нитка получает диапазон строк `start_row..end_row`,
- потоки создаются в цикле и собираются в вектор `threads`,
- все `join()` выполняются после создания всех потоков.

Каждый поток:

- инициализирует локальный родительский массив `local_parent`,
- ведёт локальную нумерацию меток `local_label`,
- вызывается `ProcessStripeStl` для обработки своей полосы.

Локальные результаты:

- `labels_` заполняется глобально в потоке,
- локальный `parent` хранит связи эквивалентности внутри полосы,
- после join выполняется последовательное объединение на границах и перенос локальных родителей в глобальный DSU.

Синхронизация:

- прямая синхронизация при `join()` — после создания всех потоков,
- `labels_` меняется каждым потоком только на своей полосе,
- глобальные данные `parent_` заполняются в `MergeLocalParentsStl` последовательно.

## 5. Детали реализации

Файлы: `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`.

`PreProcessingImpl` выделяет глобальный вектор `labels_`, глобальный DSU `parent_`, определяет `width_`,`height_`.

`FirstPass()`:

- создаёт `num_threads` потоков,
- для каждого потока инициализирует `local_parent` и `local_label`,
- выполняет `ProcessStripeStl` по строкам,
- вызывает `join()` после создания всех потоков.

`MergeBoundariesStl` объединяет смежные полосы последовательно.

`MergeLocalParentsStl` переносит отношения эквивалентности из локальных DSU в глобальный `parent_`.

`SecondPass()` нормализует метки через плоский вектор `new_labels`.

Ключевой технический фокус: `join` вызывается только после создания всех потоков, поэтому параллелизм действительно
реализован, а не сериализован.

## 6. Проверка корректности

Проверка через `tests/functional/main.cpp`.

Важно, что в этой реализации проверяются:

- отсутствие гонок при записи `labels_` в разные диапазоны,
- корректная обработка стыков между полосами,
- одинаковый результат для `num_threads=1` и `num_threads>1`.

## 7. Экспериментальная среда

Команды:

```bash
set PPC_NUM_THREADS=4
scripts/run_tests.py --running-type=threads --counts 1 2 4
```

## 9. Выводы

`std::thread`-версия удобна для обучения, потому что явно показывает карту поток → диапазон. При этом она менее
гибкая, чем runtime в TBB, и чувствительна к накладным расходам `create/join`.
