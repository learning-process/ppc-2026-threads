## 1. Контекст

Данный отчет описывает гибридную реализацию метода сопряженных градиентов для решения систем
линейных алгебраических уравнений (СЛАУ). Гибридная модель (ALL) объединяет две парадигмы:

- **Межпроцессный параллелизм (Distributed Memory):** использование стандарта MPI для
распределения данных и работы между изолированными узлами или процессами (рангами)
- **Внутрипроцессный параллелизм (Shared Memory):** использование OpenMP внутри каждого
MPI-процесса для загрузки всех доступных физических ядер узла

Эта версия необходима для масштабирования вычислений на вычислительные кластеры, где данные
не могут поместиться в памяти одного узла, а также для оценки накладных расходов
на сетевую коммуникацию.

## 2. Постановка задачи

Входные данные, выходные данные и критерии корректности идентичны последовательной версии:

- **Вход:** матрица коэффициентов A (развернутая по строкам, $n \times n$), вектор правой
части b (размера $n$), размерность $n$
- **Выход:** вектор решения x (размера $n$)
- **Критерий корректности:** $\sqrt{\sum_{i=0}^{n-1} r_i^2} < 10^{-8}$

**Задача распараллеливания:** реализовать блочное распределение матрицы A и векторов между
MPI-процессами, минимизировать объем пересылаемых данных на каждой итерации и использовать
OpenMP для локальных вычислений внутри каждого процесса.

## 3. Схема распараллеливания

Иерархический параллелизм реализован следующим образом:

### Межпроцессная схема (MPI)

**Декомпозиция данных:** Матрица $A$ логически делится на горизонтальные полосы (блоки строк).
Каждый MPI-процесс (ранг) получает свой непрерывный диапазон из `local_n` строк.
Функция `CalculateDistribution` вычисляет смещение (offset) и размер блока для каждого ранга,
балансируя остаток от деления. Векторы r, p, ap и x также распределены.

**Коммуникация:**

- На каждой итерации для умножения матрицы на вектор $A \cdot p$ каждому процессу требуется
полная копия вектора направления спуска $p$. Для этого используется вызов `MPI_Allgatherv`,
который собирает распределенные локальные куски `local_p` со всех рангов в глобальный вектор `p_full`
- Вычисление скалярных произведений (норма невязки, шаг $\alpha$) требует глобальной редукции.
Локальные суммы суммируются с помощью `MPI_Allreduce`

### Внутрипроцессная схема (OpenMP)

Внутри каждого ранга локальные операции (умножение своей подматрицы на `p_full`,
расчет локальных сумм скалярных произведений, обновление `local_x`, `local_r`, `local_p`)
распараллеливаются между потоками узла с помощью `#pragma omp parallel for`.

## 4. Детали реализации

Основная логика разделена на вспомогательные функции в `all/src/ops_all.cpp`,
сочетающие MPI-коммуникации и OpenMP-директивы:

- **ComputeLocalMatVec:** выполняет умножение локального блока строк матрицы $A$ на полный
вектор $p_{full}$. Вычисления распределены через OpenMP
- **GlobalDot:** вычисляет локальную сумму скалярного произведения (через OMP редукцию),
а затем синхронизирует результат между всеми процессами через `MPI_Allreduce`
- **UpdateLocalXR:** локально обновляет векторы приближения и невязки и возвращает
глобальную норму новой невязки (также через `MPI_Allreduce`)
- **RunImpl:** управляет основным циклом. Для сбора вектора $p$ предварительно подготавливаются
массивы `counts` и `displs`, после чего в цикле вызывается сборка.

## 8. Выводы

Реализация гибридного пайплайна (ALL) выявила классические особенности
коммуникационно-интенсивных алгоритмов:

**Суперлинейное ускорение (Cache Effect):** На 2 и 4 рангах наблюдается
сверхлинейное ускорение ($S > 4$). Это связано с локализацией данных:
при делении матрицы $A$ на блоки, рабочие наборы данных (working sets)
каждого процесса начинают полностью помещаться в быструю кэш-память L2/L3
процессора. Кроме того, накладные расходы на MPI-коммуникацию между
процессами на одном физическом узле (через shared memory) минимальны.

**Цена коммуникации:** При переходе к 6 процессам время выполнения возрастает
с 122 ms до 154 ms, а эффективность падает. Метод сопряженных градиентов
требует вызова барьерных коллективных операций (`MPI_Allgatherv` для сбора $p$
и `MPI_Allreduce` для скалярных произведений) на каждой из $2n$ итераций цикла.

На малых и средних размерностях задачи затраты на сериализацию, упаковку данных
и синхронизацию 6 процессов перекрывают выгоду от параллельных вычислений.

**Область применения:** Гибридная версия (MPI + OpenMP) избыточна для решения
СЛАУ на локальной машине с общей памятью (где STL или OMP версии показывают
себя стабильнее без расходов на MPI). Однако эта реализация архитектурно
готова к переносу на распределенные кластеры, где матрица $A$ слишком велика
для хранения в RAM одного вычислительного узла.
