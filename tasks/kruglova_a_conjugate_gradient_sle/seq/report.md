## 1. Контекст

Последовательная версия алгоритма решения систем линейных алгебраических уравнений (СЛАУ) методом сопряженных градиентов
является эталонной реализацией, на основе которой строится вся дальнейшая работа по распараллеливанию. Она необходима
для:

1. Фиксации и верификации базовой математической логики итерационного метода.
2. Создания базовой линии производительности (baseline), относительно которой будет рассчитываться ускорение ($S$) и
эффективность ($E$) в параллельных версиях (OMP, TBB, STL, MPI).

## 2. Постановка задачи

- **Вход:**
  - одномерный вектор вещественных чисел типа `double` размера $n \times n$, представляющий развернутую по строкам
  квадратную матрицу коэффициентов `A`;
  - одномерный вектор вещественных чисел типа `double` размера $n$, представляющий вектор правой части `b`;
  - целочисленное значение `size`, задающее размерность системы $n$.

- **Выход:** упорядоченный одномерный вектор вещественных чисел типа `double` размера $n$, содержащий вычисленные
значения неизвестных `x`.

- **Ограничения:**
  - Критерий корректности: выходной вектор $x$ считается корректным решением СЛАУ, если норма вектора невязки $r = b -
  Ax$ не превышает заданного малого порогового значения точности $\epsilon$:
  $$\|r\|_2 = \sqrt{\sum_{i=0}^{n-1} r_i^2} < \epsilon, \quad \text{где } \epsilon = 10^{-8}$$

Матрица $A$ математически предполагается симметричной и положительно определенной для гарантированной сходимости метода
сопряженных градиентов.

## 3. Базовый алгоритм

Метод сопряженных градиентов — это эффективный итерационный метод, основанный на построении последовательности
направлений спуска (градиентов), сопряженных (ортогональных) относительно матрицы системы $A$.

1. **Инициализация:** В качестве начального приближения вектора решений принимается нулевой вектор ($x_0 = 0$).
Начальный вектор невязки инициализируется значениями правой части $r_0 = b$. Начальный вектор направления спуска
совпадает с вектором невязки: $p_0 = r_0$. Вычисляется начальное скалярное произведение невязки $rs_{old} = r_0^T \cdot
r_0$.
2. **Итерационный процесс:** Выполняется последовательность шагов (не более $2n$ итераций):
    - Вычисляется произведение матрицы на текущий вектор направления: $ap = A \cdot p$.
    - Определяется длина шага вдоль направления спуска: $\alpha = \frac{rs_{old}}{p^T \cdot ap}$.
    - Обновляется вектор приближенного решения: $x = x + \alpha p$.
    - Корректируется вектор текущей невязки: $r = r - \alpha ap$.
    - Вычисляется новое скалярное произведение невязки: $rs_{new} = r^T \cdot r$.
    - Если норма невязки $\sqrt{rs_{new}}$ становится меньше порогового значения $\epsilon = 10^{-8}$, итерационный
    процесс останавливается.
    - Производится поправка направления спуска: $\beta = \frac{rs_{new}}{rs_{old}}$, после чего формируется новое
    сопряженное направление: $p = r + \beta p$. Значение $rs_{old}$ обновляется текущим значением $rs_{new}$.

**Асимптотика:**

- **По времени:** $O(k \cdot n^2)$, где $k$ — число итераций до сходимости (в худшем случае $k \le 2n$, что дает общую
сложность $O(n^3)$). Основную вычислительную сложность на каждой итерации составляет умножение плотной матрицы на
вектор, требующее $O(n^2)$ операций.
- **По памяти:** $O(n^2)$. Требуется хранение исходной матрицы системы размера $n \times n$, а также нескольких
вспомогательных векторов размера $n$ (`b`, `x`, `r`, `p`, `ap`).

**Инвариант:** на каждой $m$-й итерации вектор невязки $r_m$ ортогонален всем предыдущим направлениям спуска $p_0, p_1,
\dots, p_{m-1}$. **Критерий корректности:** достижение условия $\sqrt{r^T \cdot r} < 10^{-8}$.

## 4. Детали реализации

В рамках архитектуры `BaseTask` используются стандартные методы `ValidationImpl`, `PreProcessingImpl` и
`PostProcessingImpl` для формальной проверки и подготовки векторов. Основная логика вычислений и управления потоком
данных сосредоточена в методе `RunImpl`:

- **ValidationImpl:** метод выполняет верификацию корректности входных данных. Проверяет, что размерность системы `size`
строго больше нуля, размер одномерного массива `A` в точности равен $n \times n$, а размер вектора правой части `b`
равен $n$.
- **PreProcessingImpl:** выделяет память под результирующий вектор `GetOutput()` и инициализирует его начальным нулевым
приближением с помощью `.assign(..., 0.0)`.
- **MatrixVectorMultiply:** вспомогательная функция, инкапсулирующая самый ресурсоемким фрагмент — вычисление
произведения строки развернутой матрицы на вектор в двойном вложенном цикле.
- **RunImpl:** координирует выполнение шагов метода. Извлекает входные структуры, запускает цикл сопряженных градиентов,
отслеживает математические критерии останова (включая деление на ноль при занулении шага) и возвращает управление после
сходимости.

```cpp
bool KruglovaAConjGradSleSEQ::RunImpl() {
  const auto &a = GetInput().A;
  const auto &b = GetInput().b;
  int n = GetInput().size;
  auto &x = GetOutput();

  std::vector<double> r = b;
  std::vector<double> p = r;
  std::vector<double> ap(n, 0.0);

  double rsold = 0.0;
  for (int i = 0; i < n; ++i) {
    rsold += r[i] * r[i];
  }

  const double tolerance = 1e-8;

  for (int iter = 0; iter < n * 2; ++iter) {
    MatrixVectorMultiply(a, p, ap, n);

    double p_ap = 0.0;
    for (int i = 0; i < n; ++i) {
      p_ap += p[i] * ap[i];
    }

    if (std::abs(p_ap) < 1e-15) {
      break;
    }

    double alpha = rsold / p_ap;
    for (int i = 0; i < n; ++i) {
      x[i] += alpha * p[i];
      r[i] -= alpha * ap[i];
    }

    double rsnew = 0.0;
    for (int i = 0; i < n; ++i) {
      rsnew += r[i] * r[i];
    }

    if (std::sqrt(rsnew) < tolerance) {
      break;
    }

    for (int i = 0; i < n; ++i) {
      p[i] = r[i] + ((rsnew / rsold) * p[i]);
    }
    rsold = rsnew;
  }
  return true;
}
```
