## 1. Контекст

Задача построения выпуклых оболочек компонент связности является классической в области компьютерного зрения
и обработки изображений.
Она применяется для:

- Выделения ключевых признаков объектов
- Сжатия контурной информации
- Упрощения формы объектов для последующего анализа
- Распознавания образов

## 2. Постановка задачи

На вход подается бинарное изображение, представленное структурой `BinaryImage`.
На выходе требуется получить компоненты связности и их выпуклые оболочки.

Корректным входом считаются изображения, для которых:

- `width > 0` и `height > 0`;
- `pixels.size() == width * height`;
- пиксели имеют значения в диапазоне `[0, 255]`.

Крайние случаи, блокируемые в `ValidationImpl`:

- нулевые или отрицательные размеры изображения;
- несоответствие размера массива пикселей ожидаемому.

Корректным результатом считается:

- все компоненты связности найдены (4-связность);
- каждая точка принадлежит ровно одной компоненте;
- выпуклая оболочка минимальна по включению;
- вершины оболочки идут в порядке обхода против часовой стрелки.

## 3. Базовый алгоритм

Главная структура данных задачи - бинарное изображение и компоненты связности:

- `width`, `height` задают размеры изображения;
- `pixels` хранит значения пикселей (0 или 255 после бинаризации);
- `components` хранит найденные компоненты связности;
- `convex_hulls` хранит выпуклые оболочки для каждой компоненты.

Пошаговая схема:

1. Бинаризация изображения: пиксели со значением > 128 становятся 255, иначе 0.
2. Поиск компонент связности методом BFS (4-связность).
3. Для каждой найденной компоненты построение выпуклой оболочки алгоритмом Эндрю.
4. Сохранение компонент и их оболочек в выходную структуру.

Именно эта версия выбрана как baseline,
потому что реализует полную математическую логику без параллельных накладных расходов.

## 4. Детали реализации

Файлы:

- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

Pipeline реализации:

- `ValidationImpl` проверяет корректность размеров изображения и массива пикселей;
- `PreProcessingImpl` копирует входное изображение и выполняет бинаризацию;
- `RunImpl` последовательно находит компоненты связности и строит их оболочки;
- `PostProcessingImpl` передает готовое изображение наружу.

Обработанные граничные случаи:

- пустое изображение (все пиксели = 0);
- изображение с одной точкой;
- изображение с двумя точками;
- компоненты на границе изображения;
- компоненты сложной формы (вогнутые, с отверстиями).

## 5. Проверка корректности

Результат сравнивается с ожидаемым на наборе функциональных тестов.

Использованные функциональные тесты (5 штук):

1. **Case1 (one)** - одна точка в центре изображения 5×5:
   - Вход: пиксель (2,2) = 200, остальные = 0.
   - Ожидаемый результат: компонента из одной точки `(2,2)`.

2. **Case2 (two)** - две изолированные точки на изображении 6×6:
   - Вход: пиксели (1,1) = 255 и (4,4) = 255.
   - Ожидаемый результат: две компоненты, каждая из одной точки.

3. **Case3 (three)** - горизонтальная линия на изображении 7×3:
   - Вход: пиксели (2,1), (3,1), (4,1) = 255.
   - Ожидаемый результат: компонента из двух точек `(2,1)` и `(4,1)` (оболочка отрезка).

4. **Case4 (four)** - прямоугольная компонента 4×4 на изображении 8×8:
   - Вход: прямоугольник с углами (3,2), (6,2), (6,5), (3,5).
   - Ожидаемый результат: оболочка из 4 угловых точек.

5. **Case5 (fivee)** - ромбовидная компонента на изображении 9×9:
   - Вход: все точки, где |x-4| + |y-4| ≤ 4 (ромб).
   - Ожидаемый результат: оболочка из 4 вершин `(0,4)`, `(4,0)`, `(8,4)`, `(4,8)`.

Для сравнения результатов используется функция `NormalizeAll`, которая:

- нормализует порядок точек в оболочке (сортировка по y, затем по x);
- удаляет дубликаты;
- сортирует сами оболочки для возможности сравнения.

performance: изображение 6000×6000 со случайными точками (2 точки на строку).
