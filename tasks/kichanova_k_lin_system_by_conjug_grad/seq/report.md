## 1. Контекст

Последовательная реализация метода сопряжённых градиентов для решения СЛАУ Ax = b
с симметричной положительно определённой матрицей.
Эта версия служит эталоном корректности и baseline для расчёта ускорения всех параллельных реализаций.

## 2. Постановка задачи

**Входные данные:** структура `LinSystemData`:

- A - матрица n*n (row-major, `std::vector<double>` размера n^2)
- b - вектор правой части размера n
- n - размерность
- epsilon - требуемая точность (по умолчанию 10^(-10))

**Выходные данные:** `std::vector<double> x` размера n - приближённое решение.

**Критерий корректности:** ||b - A x||_2 < epsilon * sqrt(n)

**Ограничения:**

- A  - симметричная матрица со всеми положительными собственными значениями
- размеры A и b согласованы
- n <= доступная память / sizeof(double)

## 3. Базовый алгоритм

Метод сопряжённых градиентов — итерационный метод для симметричных матриц со всеми положительными собственными значениями.

**Пошаговое описание:**

1. Инициализация: x^(0) = 0, r^(0) = b, p^(0) = r^(0).
2. Вычислить A p^(k).
3. alpha_k = (r^(k), r^(k)) / (p^(k), A p^(k)).
4. x^(k+1) = x^(k) + alpha_k p^(k).
5. r^(k+1) = r^(k) - alpha_k A p^(k).
6. Если ||r^(k+1)||_2 < epsilon: выход.
7. beta_k = (r^(k+1), r^(k+1)) / (r^(k), r^(k)).
8. p^(k+1) = r^(k+1) + beta_k p^(k), переход к шагу 2.

**Асимптотика:**

- По времени: O(итераций * n^2), основная операция — матрично-векторное умножение
- По памяти: O(n^2) для хранения A

**Инварианты:**

- Ортогональность невязок: (r^(i), r^(j)) = 0 при i != j
- Сопряжённость направлений: (p^(i), A p^(j)) = 0 при i != j
- Монотонное убывание нормы невязки

## 4. Детали реализации

**Файлы:**  seq/include/ops_seq.hpp, seq/src/ops_seq.cpp

**Вспомогательные функции:**

- `ComputeDotProduct(a, b, n)` - скалярное произведение, O(n)
- `ComputeMatrixVectorProduct(a, v, result, n)` - умножение матрицы на вектор, O(n^2)
- `UpdateSolution(x, p, alpha, n)` - x_i += alpha * p_i, O(n)
- `UpdateResidual(r, ap, alpha, n)` - r_i -= alpha * ap_i, O(n)
- `UpdateSearchDirection(p, r, beta, n)` - p_i = r_i + beta * p_i, O(n)

**Pipeline базового класса:**

- `ValidationImpl` — проверяет n > 0, размер A равен n^2, размер b равен n
- `PreProcessingImpl` — инициализирует выходной вектор x нулями размера n
- `RunImpl` — основной цикл.
  Крайние случаи: n = 0 -> false; ||r^(0)|| < epsilon -> немедленный выход;
  |(p, Ap)| < 10^(-30) -> досрочный выход во избежание деления на ноль.
- `PostProcessingImpl` — пустая

**Ключевой фрагмент (матрично-векторное умножение):**

```cpp
void ComputeMatrixVectorProduct(const std::vector<double>& a, const std::vector<double>& v, std::vector<double>& result, int n) {
  const auto stride = static_cast<size_t>(n);
  for (int i = 0; i < n; ++i) {
    double sum = 0.0;
    const double* a_row = &a[static_cast<size_t>(i) * stride];
    for (int j = 0; j < n; ++j) {
      sum += a_row[j] * v[j];
    }
    result[i] = sum;
  }
}
```

Цикл по i - независимые строки матрицы, каждая требует n умножений и сложений.
Это основная точка распараллеливания в последующих версиях.

## 5. Проверка корректности

Корректность проверяется в tests/functional/main.cpp вычислением невязки ||b-Ax||_2 и сравнением с порогом epsilon * sqrt(n).

**Тестовый набор (12 конфигураций):**

- Единичная матрица: n = 2,3,5. Точное решение x = b
- Диагональная матрица: n = 2,4,6. Решение x_i = b_i / A_ii за 1 итерацию
- Трёхдиагональная трёхдиагональная: n = 3,5,7
- Случайная матрица: n = 4,8,10. Генерируется как M^T *M + n* I

Результат: все 12 тестов пройдены. Невязка для всех случаев < 10^(-15) при epsilon = 10^(-10).
