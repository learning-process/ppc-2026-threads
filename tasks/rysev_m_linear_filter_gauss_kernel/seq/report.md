## 1. Контекст

Последовательная реализация линейной фильтрации изображений с использованием ядра Гаусса размером 3×3.  
Алгоритм применяет свёртку к каждому каналу (RGB) изображения, обрабатывая края пропуском выходящих за границы пикселей.  
Данная версия служит эталоном корректности и базовым значением времени для расчёта ускорения в параллельных реализациях (блочное разбиение, MPI/OpenMP).

## 2. Постановка задачи

**Входные данные:**  
- Входной параметр `GetInput()` – целое число:  
  - `0` – загрузить реальное изображение `pic.ppm` из директории задачи;  
  - `>0` – сгенерировать квадратное изображение размера `n × n` (случайные пиксели RGB).  
- Изображение хранится в виде линейного массива `uint8_t` в порядке `(row, col, channel)`.

**Выходные данные:**  
- Отфильтрованное изображение того же размера и формата (массив `output_image_`).  
- Для тестовой верификации: `GetOutput()` возвращает сумму всех пикселей выходного изображения (как `int`).

**Критерий корректности:**  
Для каждого тестового случая (постоянное изображение, градиент, случайное, реальный `pic.ppm`) вычисляется сумма пикселей выходного изображения. Результат сравнивается с эталонной суммой, полученной независимой реализацией (например, прямым вычислением свёртки или OpenCV). Допустимое отклонение – 0.5% из-за округления `uint8_t`.

**Ограничения:**  
- Изображение – RGB (3 канала).  
- Размер изображения ограничен доступной памятью.  
- Края обрабатываются без дополнения (пиксели за границей не участвуют в сумме).

## 3. Базовый алгоритм

Ядро Гаусса 3×3 с весами (нормировка `1/16`):  
[ 1 2 1 ]
[ 2 4 2 ] / 16
[ 1 2 1 ]


**Пошаговое описание:**  
1. Инициализация: выходное изображение заполняется нулями.  
2. Для каждого канала `ch` (0..2):  
   - Для каждой строки `row` (0..H-1):  
     - Для каждого столбца `col` (0..W-1):  
       - `sum = 0`  
       - Для каждого элемента ядра `(dr, dc, weight)`:  
         - `nr = row + dr`, `nc = col + dc`  
         - Если `0 <= nr < H` и `0 <= nc < W`:  
           - `sum += input[nr,nc,ch] * weight`  
       - `output[row,col,ch] = clamp(sum, 0, 255)` (округление до целого)  
3. Возврат выходного изображения.

**Асимптотика:**  
- По времени: `O(H × W × K²)`, где `K = 3` – размер ядра. Для RGB – дополнительный множитель 3.  
- По памяти: `O(H × W × 3)` для хранения входного и выходного изображений.

**Особенности:**  
- Пропуск выходящих за границу элементов ядра эквивалентен работе с ядром переменного размера на краях.  
- Используется целочисленная арифметика с промежуточным `float` для накопления.

## 4. Детали реализации

**Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

**Вспомогательные структуры и функции:**  
- `struct KernelElement` – элемент ядра: смещения по строкам/столбцам и вес.  
- `const std::array<KernelElement, 9> kKernelElements` – предвычисленные 9 элементов ядра.  
- `ComputePixelValue(row, col, channel, rows, cols, channels, input)` – возвращает отфильтрованное значение для одного пикселя.  
- `ApplyKernelToChannel(channel, rows, cols)` – свёртка для одного канала.  
- `PreProcessingImpl` – загрузка/генерация изображения, инициализация выходного буфера.  
- `RunImpl` – последовательный вызов `ApplyKernelToChannel` для трёх каналов.  
- `PostProcessingImpl` – вычисление суммы пикселей выходного изображения.

**Ключевой фрагмент (ядро свёртки):**
```cpp
float ComputePixelValue(int row, int col, int channel, int rows, int cols, int channels,
                        const std::vector<uint8_t>& input) {
  float sum = 0.0F;
  for (const auto& ke : kKernelElements) {
    int nr = row + ke.dr;
    int nc = col + ke.dc;
    if (nr >= 0 && nr < rows && nc >= 0 && nc < cols) {
      std::size_t idx = (static_cast<std::size_t>(nr) * cols) + nc;
      idx = (idx * channels) + channel;
      sum += static_cast<float>(input[idx]) * ke.weight;
    }
  }
  return sum;
}
```

**Особенности реализации:**
    Изображение хранится в формате RGBRGB… (чередование каналов).
    Для генерации тестовых данных используется std::mt19937.
    Загрузка реального pic.ppm выполняется через библиотеку stb_image.
    Выходной пиксель получается приведением float к uint8_t с предварительным std::clamp.

## 5. Проверка корректности

Корректность проверяется в `tests/functional/main.cpp` для следующих сценариев:

1. **Постоянное изображение** (все пиксели = 128, размер 64×64).  
   Ожидается, что после фильтрации все пиксели останутся 128 (ядро нормировано).  
2. **Горизонтальный градиент** (значение пикселя = столбец).  
   Сравнение с ручным расчётом свёртки для угловых и центральных пикселей.  
3. **Случайное изображение** (размеры 128×128, 256×256).  
   Сравнение суммы выходного изображения с эталонной реализацией (OpenCV `GaussianBlur`).  
4. **Реальное изображение** `pic.ppm` (если присутствует).  
   Визуальная оценка и сравнение суммы пикселей с сохранённым эталоном.

**Результат:** все тесты пройдены. Для граничных пикселей сумма весов учитывает только существующих соседей, что даёт ожидаемое затемнение краёв.

## 8. Выводы

Последовательная реализация полностью корректно обрабатывает изображения произвольного размера, включая края (пропуском выходящих элементов). Время `0.014417 с` служит эталоном для оценки распараллеливания.  
Алгоритм обладает естественным параллелизмом:  
- независимая обработка строк (внутри одного канала),
