## 1. Контекст

Данный отчет описывает реализацию гибридной модели параллелизма для фильтра Гаусса. Комбинация MPI и TBB
позволяет масштабировать вычисления на кластерные системы (распределенная память) и одновременно
максимально эффективно использовать ресурсы многоядерных узлов (общая память).

## 2. Постановка задачи

Основная задача — разделить изображение на вертикальные полосы между MPI-процессами. Сложность
заключается в том, что фильтр 3x3 требует для расчетов граничных пикселей данные из соседних полос,
хранящихся в памяти других процессов. Итоговый результат должен быть собран на Master-процессе.

## 3. Базовый алгоритм

Математическая основа — свертка с ядром Гаусса. Алгоритм дополнен фазами межпроцессорного обмена:
рассылка данных (Scatter), обмен границами (Halo Exchange), локальные вычисления и сборка (Gather).

## 4. Межпроцессная схема (MPI)

- **Декомпозиция**: Изображение режется вертикально. Master (rank 0) вычисляет количество столбцов
  для каждого процесса с учетом остатка.
- **Halo Exchange**: Чтобы избежать блокировок (deadlocks), обмен граничными столбцами реализован
  через `MPI_Sendrecv`. Процессы обмениваются «теневыми» столбцами с левым и правым соседями.
- **Распределение данных**: Так как данные в памяти лежат построчно (row-major), а нарезка идет
  по столбцам, Master-процесс выполняет ручную упаковку данных в буферы перед отправкой.

## 5. Внутрипроцессная схема (TBB)

Внутри каждого MPI-процесса обработка локальной полосы (включая полученные Halo-зоны)
параллелится с помощью `tbb::parallel_for`.

- **Тайлинг**: Применяется `tbb::blocked_range2d`. Это позволяет TBB нарезать локальную полосу
  на оптимальные блоки, минимизируя промахи кэша при чтении соседних строк в окне 3x3.

## 6. Детали реализации

Файлы: `all/include/ops_all.hpp`, `all/src/ops_all.cpp`.

В коде реализована четкая логика разделения обязанностей Master и Worker.

```cpp
// Фрагмент обмена границами из all/src/ops_all.cpp
MPI_Sendrecv(send_left.data(), height, MPI_UNSIGNED_CHAR, left_proc, 1, 
             recv_right.data(), height, MPI_UNSIGNED_CHAR, right_proc, 1, 
             MPI_COMM_WORLD, MPI_STATUS_IGNORE);
MPI_Sendrecv(send_right.data(), height, MPI_UNSIGNED_CHAR, right_proc, 2, 
             recv_left.data(), height, MPI_UNSIGNED_CHAR, left_proc, 2, 
             MPI_COMM_WORLD, MPI_STATUS_IGNORE);
```

Этот механизм гарантирует, что каждый процесс получит необходимые данные от соседей для корректного
расчета пикселей на стыках полос.

## 7. Проверка корректности

Тесты проводились через `mpirun`. Проверено попиксельное совпадение с SEQ версией.
Особое внимание уделено вырожденным случаям (матрица 1x1 на 4 процессах), где большая
часть процессов получает 0 столбцов и должна корректно пропустить фазу вычислений.

## 8. Экспериментальная среда

- **Конфигурация**: Ranks (MPI процессы) × Threads (TBB потоки на ранг).
- **CPU**: Intel Core i7-10750H (6 ядер).
- **Команды запуска**:

```bash
# Сборка
cmake -S . -B build -D USE_FUNC_TESTS=ON -D CMAKE_BUILD_TYPE=Release
cmake --build build --parallel

# Запуск на 2 процессах по 4 потока (всего 8 воркеров)
export PPC_NUM_PROC=2
export PPC_NUM_THREADS=4
scripts/run_tests.py --running-type=performance
```

## 10. Выводы

Гибридная схема MPI+TBB успешно реализована. Механизм обмена Halo-зонами обеспечил корректность
результата на границах разделения. Версия оправдывает себя при работе с объемами данных,
превышающими объем памяти одного вычислительного узла.
