## 1. Контекст

Данный отчет описывает реализацию базового последовательного алгоритма фильтрации Гаусса.
Эта версия является фундаментом проекта: она определяет логику обработки данных и служит
единственным эталоном (baseline) для оценки корректности и измерения ускорения во всех
параллельных реализациях (OpenMP, TBB, STL, MPI).

## 2. Постановка задачи

**Входные данные**: структура `Matrix`, включающая ширину, высоту и одномерный массив байт
`std::vector<uint8_t> data`.
**Выходные данные**: новая структура `Matrix` с обработанным изображением того же размера.
**Ограничения**:

- Положительные размеры: `width > 0` и `height > 0`.
- Соответствие данных: размер вектора должен быть равен `width * height`.
**Обработка границ**: Зеркальное отражение. Если фильтр вылетает за границы, значение берется
из «отраженного» внутрь пикселя. Это позволяет избежать затемнения краев изображения.

## 3. Базовый алгоритм

Алгоритм реализует математическую операцию свертки. Для каждой точки изображения вычисляется
значение на основе окна 3x3.

1. Применяется ядро Гаусса:

   ```text
   {{1, 2, 1},
    {2, 4, 2},
    {1, 2, 1}}
   ```

2. Сумма весов соседних пикселей делится на 16.
3. Асимптотика по времени: $O(W \cdot H)$, так как окно фильтра фиксировано.
4. Асимптотика по памяти: $O(W \cdot H)$ для хранения результирующего кадра.
Инвариант: значения пикселей всегда остаются в диапазоне [0, 255].

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.

Реализация `RunImpl` в последовательной версии намеренно следует логике вертикального обхода.
Это сделано для того, чтобы последовательный алгоритм структурно соответствовал требованиям
задания по «вертикальному разбиению».

```cpp
// Фрагмент из seq/src/ops_seq.cpp
for (int horizontal_band = start_band; horizontal_band < end_band; ++horizontal_band) {
  for (int vertical_band = 0; vertical_band < height; ++vertical_band) {
    int sum = 0;
    // Сборка суммы 3x3 через зеркальный доступ
    sum += kGaussKernel[0][0] * KopilovDGetPixelMirrorSeq(matrix, horizontal_band - 1, 
                                                          vertical_band - 1, width, height);
    // ... остальные 8 соседей ...
    result[(vertical_band * width) + horizontal_band] = static_cast<uint8_t>(sum / kDivConst);
  }
}
```

Вспомогательная функция `KopilovDGetPixelMirrorSeq` преобразует координаты за границами
(например, -1 в 0), реализуя зеркальную логику без усложнения циклов.

## 5. Проверка корректности

Корректность подтверждается функциональными тестами из `tests/functional/main.cpp`.
Проверены сценарии:

- Матрица 1x1 (максимальное использование зеркалирования).
- Матрицы с равномерным цветом (выход должен быть идентичен входу).
- Валидация некорректных размеров в `ValidationImpl`.
Результаты сравнивались с заранее вычисленными вручную значениями.
