## 1. Контекст

Данный отчет описывает параллельную реализацию фильтра Гаусса с использованием библиотеки Intel oneTBB.
В отличие от директивного подхода OpenMP, TBB предоставляет таск-ориентированную модель, которая
позволяет более гибко управлять декомпозицией данных и динамически балансировать нагрузку между ядрами.

## 2. Постановка задачи

Цель — адаптировать алгоритм свертки под механизмы TBB, обеспечив эффективную работу с кэшем.
Результаты работы должны попиксельно совпадать с эталонной версией из `seq/report.md`.

## 3. Базовый алгоритм

Алгоритм последовательно применяет ядро Гаусса 3x3 к каждому пикселю. Операция относится к категории
memory-bound, так как вычисления минимальны, а основной объем времени тратится на чтение 9 соседних байт
и запись результата в новый массив.

## 4. Схема распараллеливания

Для этой реализации выбрана схема с двумерным разбиением:

- **Примитив**: Использован высокоуровневый алгоритм `tbb::parallel_for`.
- **Диапазон (Range)**: Вместо нарезки только по строкам применен `tbb::blocked_range2d<int>`.
- **Тайлинг**: Использование 2D-диапазона позволяет TBB делить изображение на прямоугольные блоки (тайлы).
  Это значительно улучшает локальность данных: при расчете соседних пикселей в блоке нужные строки
  с большей вероятностью уже находятся в кэше L2/L3.
- **Планировщик**: Использован `auto_partitioner` (по умолчанию). Библиотека применяет механизм
  work-stealing, перекидывая задачи с перегруженных ядер на свободные.
- **Конкуренция**: Число потоков ограничивается через `PPC_NUM_THREADS`.

## 5. Детали реализации

Файлы: `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`.

В методе `RunImpl` вызывается лямбда-функция, которая получает текущий поддиапазон `range`.
Внутренние циклы проходят только по тем строкам и столбцам, которые выделены конкретной задаче.

```cpp
// Фрагмент из tbb/src/ops_tbb.cpp
tbb::parallel_for(tbb::blocked_range2d<int>(0, height, 0, width), 
                  [&](const tbb::blocked_range2d<int> &range) {
  for (int j = range.rows().begin(); j != range.rows().end(); ++j) {
    for (int i = range.cols().begin(); i != range.cols().end(); ++i) {
      int pixel_sum = 0;
      // Применение ядра Гаусса...
      destination_image[(j * width) + i] = static_cast<uint8_t>(pixel_sum / kDivisor);
    }
  }
});
```

## 6. Проверка корректности

Тестирование проводилось через `tests/functional/main.cpp`. TBB-версия успешно прошла все
функциональные тесты, включая обработку вырожденных матриц 1x1 и 2x2. Поскольку итерации
в `parallel_for` независимы, проблем с гонкой данных не возникло.

## 7. Экспериментальная среда

- **Сборка**: Release.
- **Размер задачи**: 8192 × 8192 пикселей.

## 9. Выводы

oneTBB показал лучший результат среди всех локальных технологий. Использование `blocked_range2d`
позволило оптимально загрузить кэш процессора и скрыть накладные расходы на создание потоков.
Модель полностью оправдала себя для 2D-фильтрации.
