## 1. Контекст

Данный отчёт посвящен реализации фильтра Гаусса с использованием библиотеки Intel oneTBB. В отличие от директивного подхода OpenMP, TBB предлагает таск-ориентированную модель параллелизма, которая позволяет более эффективно нарезать двумерные данные и динамически балансировать нагрузку между ядрами процессора.

## 2. Постановка задачи

Цель — адаптировать алгоритм свертки под механизмы TBB. Основное внимание уделяется оптимизации доступа к памяти за счет блочного разбиения изображения. Результаты сравниваются с эталоном из `seq/report.md`.

## 3. Базовый алгоритм

Последовательный алгоритм выполняет свертку окна 3x3 для каждого пикселя. Операция является вычислительно легкой, но крайне требовательной к пропускной способности памяти (memory-bound), так как для вычисления одного значения требуется загрузить 9 соседних байт.

## 4. Схема распараллеливания

Для достижения максимальной производительности была выбрана следующая схема:

- **Примитив параллелизма:** Использован высокоуровневый алгоритм `tbb::parallel_for`.
- **Диапазон (Range):** Вместо одномерного разбиения по строкам применен `tbb::blocked_range2d<int>(0, height, 0, width)`.
- **Преимущество тайлинга:** Использование 2D-диапазона позволяет TBB разбивать изображение на прямоугольные «плитки» (tiles). Это значительно улучшает локальность кэша: когда поток обрабатывает блок пикселей, соседние строки этого блока с большой вероятностью уже находятся в L2/L3 кэше, что снижает штрафы за чтение памяти.
- **Балансировка нагрузки:** Использован стандартный `auto_partitioner`. Планировщик TBB применяет алгоритм *work-stealing* (кража задач), что позволяет эффективно использовать ресурсы, если некоторые ядра завершают свою работу быстрее других из-за системных прерываний.
- **Контроль конкуренции:** Число работающих потоков ограничивается через внешние настройки среды исполнения (`PPC_NUM_THREADS`), которые фреймворк курса транслирует в планировщик задач TBB.

## 7. Экспериментальная среда

- **Build type**: Release (`-O3`)
- **Compiler**: GCC 11.4.0
- **Библиотека**: Intel oneTBB (v2021+)
- **Размер задачи**: 8192 × 8192 пикселей

**Команды сборки и запуска:**

```bash
# Сборка
cmake -S . -B build -D USE_FUNC_TESTS=ON -D USE_PERF_TESTS=ON -D CMAKE_BUILD_TYPE=Release
cmake --build build --parallel

# Запуск функциональных тестов
scripts/run_tests.py --running-type=threads --counts 2 4

# Замер производительности на 4 потоках
export PPC_NUM_THREADS=4
scripts/run_tests.py --running-type=performance
```
