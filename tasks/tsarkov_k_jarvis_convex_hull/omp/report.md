## 1. Контекст

В данной работе последовательная версия алгоритма Джарвиса была адаптирована для OpenMP.

Основная идея распараллеливания заключается в параллельном поиске следующей точки выпуклой оболочки.
На каждом шаге алгоритм выполняет полный проход по множеству точек,
поэтому данный участок хорошо подходит для распределения работы между потоками.

OpenMP позволяет распараллелить вычисления с минимальными изменениями структуры последовательного алгоритма.

## 2. Постановка задачи

Необходимо реализовать OpenMP-версию алгоритма построения выпуклой оболочки множества точек на плоскости.

Полная постановка задачи, описание входных и выходных данных, а также ограничения приведены в `seq/report.md`.

SEQ-версия используется как baseline для сравнения производительности и проверки корректности результатов.

## 3. Базовый алгоритм

Последовательная версия алгоритма Джарвиса:

1. находит крайнюю левую точку.
2. последовательно выбирает следующую вершину оболочки.
3. выполняет полный проход по множеству точек на каждой итерации.
4. повторяет процесс до возврата в стартовую точку.

Наиболее затратной частью алгоритма является поиск следующей точки оболочки.

## 4. Схема распараллеливания

### Параллельная область

В OpenMP-версии распараллеливается участок поиска лучшей следующей точки оболочки.

Каждый поток обрабатывает собственный диапазон точек и вычисляет локальный лучший кандидат.

После завершения parallel region локальные результаты объединяются в общий результат.

### Используемая директива

```cpp
#pragma omp parallel for default(none) schedule(static) \
num_threads(static_cast<int>(actual_thread_count)) \
shared(unique_points, current_point_index, range_begin, \
range_end, chunk_size, actual_thread_count, local_best_states)
```

### default(none)

Используется `default(none)`, чтобы явно задавать атрибуты всех переменных.

Это уменьшает вероятность ошибок и появления скрытых shared-переменных.

### Shared-переменные

Shared-переменными являются:

- `unique_points`
- `current_point_index`
- `range_begin`
- `range_end`
- `chunk_size`
- `actual_thread_count`
- `local_best_states`

Все потоки читают общий массив точек и записывают результат только в собственную ячейку массива `local_best_states`.

### Private-переменные

Переменная цикла:

```cpp
int thread_index
```

является private автоматически.

Также внутри цикла используются локальные переменные:

- `current_thread_index`
- `thread_range_begin`
- `thread_range_end`

Каждый поток работает только со своими локальными значениями.

### Schedule

Используется:

```cpp
schedule(static)
```

Статическое распределение подходит данной задаче, так как:

- все итерации имеют примерно одинаковую стоимость;
- каждый поток выполняет одинаковый объём работы;
- отсутствует необходимость динамической балансировки.

Использование `static schedule` уменьшает overhead scheduler-а OpenMP.

### Синхронизация

В данной реализации не используются:

- `critical`
- `atomic`
- `reduction`

Гонки данных отсутствуют, так как каждый поток записывает результат
только в собственный элемент массива `local_best_states`.

После завершения `parallel for` используется неявный барьер OpenMP,
после которого выполняется объединение локальных результатов.

## 5. Детали реализации

Используемые файлы:

- `omp/include/ops_omp.hpp`
- `omp/src/ops_omp.cpp`

### Изменения относительно SEQ

По сравнению с последовательной версией были добавлены:

- OpenMP parallel region;
- разделение диапазонов между потоками;
- массив локальных лучших состояний;
- объединение локальных результатов после parallel region.

Основная логика алгоритма Джарвиса осталась неизменной.

### Устранение гонок данных

Основной риск гонок был связан с выбором следующей точки оболочки.

Для устранения конфликтов:

- каждый поток вычисляет локальный лучший результат независимо;
- запись выполняется только в собственный элемент массива;
- итоговый результат вычисляется после завершения parallel region.

## 6. Проверка корректности

Корректность OpenMP-версии проверялась сравнением результатов с SEQ-реализацией.

Использовались следующие functional tests:

- одна точка;
- две точки;
- треугольник;
- квадрат с внутренними точками;
- коллинеарные точки;
- повторяющиеся точки;
- ромб с внутренними точками.

Дополнительно тестирование выполнялось при:

- 1 потоке;
- 2 потоках;
- 4 потоках.

Во всех случаях результаты совпадали с SEQ-версией.

однако эффективность остаётся ограниченной из-за overhead параллелизации.
