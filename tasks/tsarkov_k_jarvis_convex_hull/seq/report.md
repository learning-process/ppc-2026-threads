## 1. Контекст

В данной работе реализуется последовательная версия алгоритма Джарвиса
для построения выпуклой оболочки множества точек на плоскости.

Последовательная реализация используется как baseline для сравнения всех параллельных backend-ов.
Именно результаты SEQ-версии используются при вычислении ускорения и эффективности OpenMP, TBB, STL и ALL реализаций.

Алгоритм Джарвиса последовательно обходит точки выпуклой оболочки, начиная с крайней левой точки,
и на каждом шаге выбирает следующую точку по направлению обхода.

## 2. Постановка задачи

### Входные данные

На вход подается набор точек на плоскости.

Каждая точка задается двумя целочисленными координатами:

- координата x,
- координата y.

Количество точек может быть произвольным.

### Выходные данные

Результатом работы является набор точек, образующих выпуклую оболочку множества.

Точки возвращаются в порядке обхода оболочки.

### Ограничения

Алгоритм должен корректно обрабатывать:

- одну точку;
- две точки;
- коллинеарные точки;
- повторяющиеся точки;
- внутренние точки.

Перед построением оболочки выполняется удаление дубликатов.

### Крайние случаи

В реализации отдельно обрабатываются:

- пустой набор точек;
- один элемент;
- два элемента;
- полностью коллинеарные множества.

## 3. Базовый алгоритм

### Описание алгоритма

Алгоритм Джарвиса работает следующим образом:

1. Находится крайняя левая точка множества.
2. Текущая точка добавляется в выпуклую оболочку.
3. Среди остальных точек выбирается следующая точка оболочки:
   - используется ориентация тройки точек;
   - выбирается наиболее "левая" точка относительно текущего направления.
4. Процесс повторяется до возврата в начальную точку.

Перед началом работы удаляются повторяющиеся точки.

### Основная структура данных

Основной структурой данных является:

```cpp
struct Point {
  int x;
  int y;
};
```

Точки хранятся в `std::vector<Point>`.

### Асимптотика

#### По времени

```text
O(n * h)
```

где:

- n - количество точек;
- h - количество точек выпуклой оболочки.

На каждом шаге алгоритм выполняет полный проход по множеству точек для поиска следующей вершины оболочки.

#### По памяти

```text
O(n)
```

Дополнительная память используется для хранения массива уникальных точек и результирующей оболочки.

### Инварианты и корректность

Во время работы алгоритма сохраняются следующие инварианты:

- каждая найденная точка принадлежит выпуклой оболочке;
- порядок обхода оболочки сохраняется;
- внутренняя точка не может быть выбрана следующей вершиной оболочки.

Корректным считается результат, совпадающий с геометрически правильной выпуклой оболочкой множества.

## 4. Детали реализации

### Используемые файлы

- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

### ValidationImpl

Проверяет корректность входных данных:

- отсутствие пустого входа;
- корректность размеров массивов.

### PreProcessingImpl

Выполняет:

- чтение входных данных;
- копирование точек;
- удаление дубликатов.

### RunImpl

Реализует основной алгоритм Джарвиса:

- поиск стартовой точки;
- последовательный поиск следующей вершины оболочки;
- построение результирующего контура.

### PostProcessingImpl

Записывает результат в выходной буфер.

### Обработанные граничные случаи

В реализации отдельно обработаны:

- одна точка;
- две точки;
- повторяющиеся точки;
- коллинеарные точки;
- точки внутри оболочки.
