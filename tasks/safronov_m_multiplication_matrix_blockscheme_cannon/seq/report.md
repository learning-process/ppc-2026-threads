## 1. Контекст

В работе реализована последовательная версия алгоритма Кэннона
для умножения плотных квадратных матриц типа `double`
с использованием блочного разбиения.

Последовательная реализация нужна как эталонная база.
Именно с ней сравниваются все параллельные версии,
поэтому здесь особенно важно корректно определить входные данные,
логику разбиения на блоки и способ сборки результата.

## 2. Постановка задачи

Необходимо вычислить произведение двух плотных квадратных матриц.

На вход подаются:

* размер блока `size_block`;
* квадратная матрица `A`;
* квадратная матрица `B`.

На выходе должна быть получена матрица `C = A × B`
того же размера `N × N`.

Ограничения и крайние случаи:

* `size_block > 0`;
* матрицы `A` и `B` не пустые;
* матрицы `A` и `B` квадратные;
* размеры матриц совпадают;
* `N % size_block == 0`;
* элементы матриц имеют тип `double`.

Если хотя бы одно из условий нарушается,
задача считается некорректной и не должна выполняться.

## 3. Базовый алгоритм

Алгоритм состоит из четырёх основных этапов.

1. Проверка входных данных.
   Сначала убеждаемся, что размер блока положительный,
   обе матрицы квадратные, одинакового размера
   и делятся на `size_block` без остатка.

2. Разбиение матриц на блоки.
   Исходные матрицы переводятся в четырёхмерные структуры,
   где первые две размерности задают позицию блока в сетке,
   а последние две описывают элементы внутри блока.

3. Выполнение схемы Кэннона.
   Сначала делается начальное циклическое выравнивание:
   блоки матрицы `A` в каждой строке сдвигаются влево,
   а блоки матрицы `B` в каждом столбце сдвигаются вверх.
   Затем выполняется `columns_blocks` итераций,
   на каждой из которых соответствующие блоки перемножаются
   и накапливаются в блоки результата `C`.

4. Сборка результата.
   После завершения всех итераций блоки результирующей матрицы
   копируются обратно в обычную матрицу `N × N`.

Асимптотика по времени: `O(N^3)`.

Асимптотика по памяти: `O(N^2)`.

Инварианты и критерий корректности:

* каждая операция умножения блока выполняется
  над подматрицами одинакового размера `size_block × size_block`;

* результат в блоке `C[i][j]`
  является суммой произведений соответствующих блоков `A` и `B`;

* после всех итераций каждая ячейка итоговой матрицы
  содержит корректную сумму по всему внутреннему индексу `k`.

## 4. Детали реализации

Файлы:
`seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.

Класс `SafronovMMultiplicationMatrixBlockSchemeCannon` содержит:

* `ValidationImpl()`;
* `PreProcessingImpl()`;
* `RunImpl()`;
* `PostProcessingImpl()`.

`ValidationImpl()` проверяет корректность входа:
положительность `size_block`, непустоту матриц,
их квадратность, равенство размеров
и делимость `N` на `size_block`.

`PreProcessingImpl()` очищает выходной контейнер
перед запуском вычислений.

`RunImpl()` выполняет основную логику:
вычисляет число блоков,
создаёт четырёхмерные блочные структуры,
заполняет их из исходных матриц,
запускает алгоритм Кэннона
и затем собирает итоговую матрицу результата.

`PostProcessingImpl()` дополнительных действий не требует
и просто завершает выполнение.

Отдельные вспомогательные методы
отвечают за локальные части алгоритма:

* `MultiplyingBlocks(...)`
  перемножает два блока
  и накапливает результат в блоке `C`;

* `ShiftingBlocksMatrixALeft(...)`
  выполняет циклический сдвиг блоков матрицы `A` влево;

* `ShiftingBlocksMatrixBUp(...)`
  выполняет циклический сдвиг блоков матрицы `B` вверх;

* `AlgorithmCannon(...)`
  управляет основным циклом схемы Кэннона;

* `FillingResultingMatrix(...)`
  собирает итоговую матрицу из блоков результата.

Граничные случаи,
которые обрабатываются реализацией:

* некорректный размер блока;
* пустые входные матрицы;
* несовпадение размеров матриц;
* нецелочисленное разбиение на блоки;
* работа с матрицами,
  заполненными одинаковыми значениями,
  и с дробными числами `double`.

## 5. Проверка корректности

Результат проверяется сравнением всех элементов
выходной матрицы с эталонной матрицей.
Для чисел типа `double`
используется допустимое отклонение `1e-10`.

Функциональные тесты охватывают
несколько характерных сценариев:

* матрицы `2 × 2`
  с заранее известным ответом;

* матрицы,
  заполненные одинаковыми значениями;

* матрицы `4 × 4`, `6 × 6`, `8 × 8`, `9 × 9`;

* разные размеры блока `size_block`;

* случаи с дробными значениями.

Примеры входа и ожидаемого выхода:

1. `A = [[1, 2], [3, 4]]`,
   `B = [[5, 6], [7, 8]]`,
   `size_block = 1`

   Ожидаемый результат:
   `[[19, 22], [43, 50]]`.

2. `A` и `B` размера `4 × 4`,
   заполненные значениями `1.0` и `2.0`,
   `size_block = 2`

   Ожидаемый результат:
   матрица `4 × 4`,
   заполненная значением `8.0`.

3. `A` и `B` размера `6 × 6`,
   заполненные значениями `1.5` и `0.5`,
   `size_block = 3`

   Ожидаемый результат:
   матрица `6 × 6`,
   заполненная значением `4.5`.

Основное время уходит на `MultiplyingBlocks(...)`,
потому что там выполняется тройной вложенный цикл
над элементами блока.
Именно эта часть определяет
основную стоимость всего алгоритма.

```cpp
// File: seq/src/ops_seq.cpp
void SafronovMMultiplicationMatrixBlockSchemeCannon::MultiplyingBlocks(
    std::vector<std::vector<double>> &block_a,
    std::vector<std::vector<double>> &block_b,
    std::vector<std::vector<double>> &block_c,
    int size_block) {
  for (int i = 0; i < size_block; i++) {
    for (int j = 0; j < size_block; j++) {
      for (int k = 0; k < size_block; k++) {
        block_c[i][j] += block_a[i][k] * block_b[k][j];
      }
    }
  }
}
```

## 8. Выводы

Последовательная версия алгоритма Кэннона реализована как эталонная.
Она корректно обрабатывает входные данные,
соблюдает ограничения на размеры матриц и размер блока,
а также формирует правильный результат для всех проверенных тестов.

Именно эта версия используется как база для ускорения,
потому что она задаёт однозначный порядок вычислений
и не содержит накладных расходов на синхронизацию
между потоками или процессами.
