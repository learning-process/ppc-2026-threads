## 1. Контекст

В данной реализации последовательный алгоритм Кэннона с блочным разбиением переносится на oneTBB
для ускорения вычислений за счёт параллельной обработки независимых блоков матрицы.

Базовая SEQ-версия остаётся эталоном: она определяет корректный порядок вычислений и структуру
блочного умножения, а TBB-версия сохраняет эту логику, добавляя параллельное выполнение над
блоками.

Основная идея перехода — выделить независимые участки работы (блоки матриц), которые можно
обрабатывать параллельно без изменения алгоритма.

## 2. Постановка задачи

На вход подаются:

* `size_block` — размер блока;
* матрица `A`;
* матрица `B`.

На выходе: матрица `C = A × B`.

Ограничения:

* `size_block > 0`;
* матрицы квадратные;
* размеры совпадают;
* `N % size_block == 0`.

## 3. Базовый алгоритм

Используется блочная схема алгоритма Кэннона:

1. Проверка входных данных.
2. Разбиение матриц на блоки `size_block × size_block`.
3. Формирование 4D структур блоков `A`, `B`, `C`.
4. Начальное циклическое смещение блоков.
5. Основной цикл:

   * перемножение соответствующих блоков;
   * накопление результата;
   * циклические сдвиги блоков `A` и `B`.
6. Сборка итоговой матрицы из блоков `C`.

## 4. Схема распараллеливания

В данной реализации параллелизация выполнена с помощью `TBB` через `parallel_for`.
Основная идея — разбиение вычислений по независимым блокам матрицы.

### 4.1. Используемые примитивы

В коде используется:

* `tbb::parallel_for` с `blocked_range<int>`
* `tbb::parallel_for` с `blocked_range2d<int>`

Это позволяет распараллеливать как одномерные проходы по строкам/столбцам блоков, так и двумерные обходы сетки блоков.

### 4.2. Разбиение диапазона

Для всех `parallel_for` диапазон задаётся через `blocked_range` / `blocked_range2d`:

* `blocked_range<int>(0, columns)` — используется для обработки строк или столбцов блоков
* `blocked_range2d<int>(0, columns_blocks, 0, columns_blocks)` — используется для обработки всей сетки блоков матрицы

Каждый поток получает свой поддиапазон итераций и работает с независимыми блоками.

### 4.3. Распараллеленные участки

#### 4.3.1. Инициализация блоков

```cpp
tbb::parallel_for(tbb::blocked_range2d<int>(0, columns_blocks, 0, columns_blocks),
  [&](const tbb::blocked_range2d<int> &range) {
    for (int i = range.rows().begin(); i != range.rows().end(); ++i) {
      for (int j = range.cols().begin(); j != range.cols().end(); ++j) {
        int shift = (i + j) % columns_blocks;
        for (int k = 0; k < size_block; k++) {
          for (int col = 0; col < size_block; col++) {
            matrix_blocks_a[i][j][k][col] = matrix_a[(i * size_block) + k][(shift * size_block) + col];
            matrix_blocks_b[i][j][k][col] = matrix_b[(shift * size_block) + k][(j * size_block) + col];
          }
        }
      }
    }
  });
```

Каждый поток обрабатывает отдельные пары блоков `(i, j)`, пересечений по записи нет.

#### 4.3.2. Перемножение блоков в алгоритме Кэннона

```cpp
tbb::parallel_for(tbb::blocked_range2d<int>(0, columns_blocks, 0, columns_blocks),
  [&](const tbb::blocked_range2d<int> &range) {
    for (int j = range.rows().begin(); j != range.rows().end(); ++j) {
      for (int k = range.cols().begin(); k != range.cols().end(); ++k) {
        MultiplyingBlocks(matrix_blocks_a[j][k],
                          matrix_blocks_b[j][k],
                          matrix_blocks_c[j][k],
                          size_block);
      }
    }
  });
```

Каждый поток работает со своим блоком результата `matrix_blocks_c[j][k]`, поэтому гонок записи нет.

#### 4.3.3. Циклические сдвиги блоков

```cpp
tbb::parallel_for(tbb::blocked_range<int>(0, columns),
  [&](const tbb::blocked_range<int> &range) {
    for (int i = range.begin(); i != range.end(); ++i) {
      std::vector<std::vector<double>> tmp = std::move(matrix_blocks_a[i][0]);
      for (int j = 1; j < columns; j++) {
        matrix_blocks_a[i][j - 1] = std::move(matrix_blocks_a[i][j]);
      }
      matrix_blocks_a[i][columns - 1] = std::move(tmp);
    }
  });
```

Аналогично для `matrix_blocks_b`.

Каждая строка или столбец блоков обрабатывается независимо.

#### 4.3.4. Сборка результата

```cpp
tbb::parallel_for(tbb::blocked_range2d<int>(0, columns_blocks, 0, columns_blocks),
  [&](const tbb::blocked_range2d<int> &range) {
    for (int i = range.rows().begin(); i != range.rows().end(); ++i) {
      for (int j = range.cols().begin(); j != range.cols().end(); ++j) {
        for (int k = 0; k < size_block; k++) {
          for (int col = 0; col < size_block; col++) {
            matrix_c[(i * size_block) + k][(j * size_block) + col] = matrix_blocks_c[i][j][k][col];
          }
        }
      }
    }
  });
```

Каждый поток копирует свой независимый блок в итоговую матрицу.

### 4.4. Итоговая схема

Параллелизация построена вокруг принципа:

* разбиение матрицы на блоки
* независимая обработка блоков через `parallel_for`
* отсутствие пересечений по памяти при записи
* сохранение логики алгоритма Кэннона без изменений

## 5. Детали реализации

Файлы:

* `tbb/include/ops_tbb.hpp`
* `tbb/src/ops_tbb.cpp`

Реализация TBB-версии сохраняет структуру SEQ, но переносит ключевые этапы на `tbb::parallel_for`.

### Основные изменения относительно SEQ

* Инициализация блоков матриц `A` и `B` выполняется через `parallel_for` по 2D-диапазону блоков.
* Основной шаг алгоритма Кэннона (умножение блоков) распараллелен через `blocked_range2d`.
* Циклические сдвиги блоков `A` и `B` распараллелены по строкам/столбцам блоков.
* Сборка итоговой матрицы `C` также выполняется параллельно по блокам.

### Корректность и гонки

Каждый поток работает с уникальными блоками данных:

* при умножении — отдельный `matrix_blocks_c[j][k]`
* при копировании — отдельные участки итоговой матрицы
* при сдвигах — независимые строки/столбцы блоков

Пересечений по записи между потоками нет.

## 6. Проверка корректности

Корректность проверяется сравнением результата TBB-версии с SEQ-эталоном.

Используются тесты:

* матрицы малого размера (2×2, 4×4);
* матрицы с одинаковыми значениями;
* случайные матрицы разных размеров;
* различные значения `size_block`.

Во всех тестах результаты совпадают с SEQ-версией.

Дополнительно проверяется совпадение результатов с допустимой точностью для `double`.

## 8. Результаты

Производительность измерялась на матрицах размера 1024×1024 при `size_block = 64`.

| Workers | Time (s) | Speedup | Efficiency |
| ------: | -------: | ------: | ---------: |
|       1 |     4.80 |    1.00 |       1.00 |
|       2 |     2.85 |    1.68 |       0.84 |
|       4 |     2.45 |    1.96 |       0.49 |
|       8 |     2.70 |    1.78 |       0.22 |

Ускорение достигается за счёт распараллеливания независимых операций над блоками матрицы
в рамках `tbb::parallel_for` с использованием двумерного диапазона `blocked_range2d`.

Разбиение пространства блоков позволяет равномерно распределить вычисления между worker-потоками без пересечения по памяти.

Максимальный прирост достигается при 2–4 потоках.
Дальнейшее увеличение числа workers до 8 не даёт прироста производительности:
вычислительная часть насыщается, а накладные расходы планировщика oneTBB начинают доминировать.
