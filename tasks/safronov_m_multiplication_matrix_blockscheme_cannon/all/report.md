## 1. Контекст

Данная реализация расширяет блочную схему умножения матриц алгоритма Кэннона на гибридную модель вычислений. Используется межпроцессное распределение через MPI и внутрипроцессное параллеление через TBB.

Цель — ускорить умножение плотных матриц типа double за счёт разделения работы на уровни:

* MPI отвечает за распределение блоков матриц между процессами;
* TBB ускоряет вычисление произведения внутри каждого блока.

Таким образом, нагрузка распределяется сначала по процессам, затем внутри каждого процесса по потокам.

## 2. Постановка задачи

Требуется реализовать умножение двух квадратных матриц одинакового размера с использованием блочного алгоритма Кэннона в гибридной MPI + TBB модели.

SEQ-версия выступает эталоном для сравнения корректности и производительности.

На вход подаются:

- `size_block` — размер блока;
- матрица `A`;
- матрица `B`.

На выходе: матрица `C = A × B`.

Ограничения:

- `size_block > 0`;
- матрицы квадратные;
- размеры совпадают;


## 3. Базовый алгоритм

Используется блочная схема алгоритма Кэннона:

1. Проверка входных данных.
2. Разбиение матриц на блоки `size_block × size_block`.
3. Формирование 4D структур блоков `A`, `B`, `C`.
4. Начальное циклическое смещение блоков.
5. Основной цикл:
   - перемножение соответствующих блоков;
   - накопление результата;
   - циклические сдвиги блоков `A` и `B`.
6. Сборка итоговой матрицы из блоков `C`.

## 4. Межпроцессная схема (MPI уровень)

### 4.1 Формирование процессной решётки

В начале выполнения вычисляется:

* rank процесса через `MPI_Comm_rank`
* общее число процессов через `MPI_Comm_size`

Далее вычисляется параметр:

* `q = sqrt(size)`

Используются только `q × q` процессов. Остальные процессы исключаются из вычислений через `MPI_Comm_split`:

```cpp
int rank = 0;
int size = 1;

MPI_Comm_rank(MPI_COMM_WORLD, &rank);
MPI_Comm_size(MPI_COMM_WORLD, &size);

int q = static_cast<int>(std::sqrt(size));
int active = q * q;

int color = (rank < active) ? 0 : MPI_UNDEFINED;
MPI_Comm_split(MPI_COMM_WORLD, color, rank, &comm);
```

Таким образом создаётся логическая 2D-решётка процессов размером q × q, необходимая для алгоритма Кэннона.

### 4.2 Паддинг и подготовка данных

Матрицы приводятся к размеру, кратному q, чтобы обеспечить равномерное разбиение на блоки:

```cpp
int padded_n = CalcPaddedSize(n, q);
int block_size = padded_n / q;
```

Если размер исходной матрицы не делится на q, она дополняется нулями:

```cpp
PadMatrix(A, padded_a, padded_n);
PadMatrix(B, padded_b, padded_n);
```

Это позволяет каждому процессу получить блок одинакового размера без обработки краевых случаев.

### 4.3 Распределение данных (DistributeData)

Распределение выполняется процессом с rank = 0.
Каждому процессу соответствует свой блок матриц A и B.

Индексация процессов соответствует координатам в решётке:

* row = proc / q
* col = proc % q

Для каждого процесса формируется локальный блок:

```cpp
send_a[(i * block_size) + j] = matrix_a_full[a_row][a_col];
send_b[(i * block_size) + j] = matrix_b_full[b_row][b_col];
```

Передача осуществляется через MPI_Send:

```cpp
MPI_Send(send_a.data(), block_size * block_size, MPI_DOUBLE, proc, 0, comm);
MPI_Send(send_b.data(), block_size * block_size, MPI_DOUBLE, proc, 1, comm);
```

Приём на стороне worker-процессов:

```cpp
MPI_Recv(local_a.data(), block_size * block_size, MPI_DOUBLE, 0, 0, comm, MPI_STATUS_IGNORE);
MPI_Recv(local_b.data(), block_size * block_size, MPI_DOUBLE, 0, 1, comm, MPI_STATUS_IGNORE);
```

Таким образом каждый процесс получает независимый набор данных без необходимости синхронного доступа к глобальной памяти.

### 4.4 Коммуникации в алгоритме Кэннона (CannonAlgorithm)

Основной вычислительный цикл состоит из q шагов.
На каждом шаге:

1. Выполняется локальное умножение блоков;
2. Выполняется обмен данными с соседними процессами;
3. Обновляются локальные блоки A и B.

Локальное перемножение выполняется через TBB (описано в следующем разделе), а межпроцессный обмен реализован через `MPI_Sendrecv`.

Определение соседей в решётке:

```cpp
int left  = (row * q) + ((col - 1 + q) % q);
int right = (row * q) + ((col + 1) % q);
int up    = (((row - 1 + q) % q) * q) + col;
int down  = (((row + 1) % q) * q) + col;
```

Обмен блоками:

```cpp
MPI_Sendrecv(local_a.data(), block_size * block_size, MPI_DOUBLE,
             left, 10,
             next_a.data(), block_size * block_size, MPI_DOUBLE,
             right, 10, comm, MPI_STATUS_IGNORE);

MPI_Sendrecv(local_b.data(), block_size * block_size, MPI_DOUBLE,
             up, 11,
             next_b.data(), block_size * block_size, MPI_DOUBLE,
             down, 11, comm, MPI_STATUS_IGNORE);
```

Каждый процесс после обмена получает обновлённые блоки A и B и переходит к следующему шагу вычислений.

### 4.5 Сбор результата

После завершения вычислений каждый процесс отправляет свой локальный блок результата на root-процесс:

```cpp
MPI_Send(local_c.data(), block_size * block_size, MPI_DOUBLE, 0, 20, comm);
```

Root-процесс собирает результат:

```cpp
MPI_Recv(recv_buf.data(), block_size * block_size, MPI_DOUBLE, proc, 20, comm, MPI_STATUS_IGNORE);
```

И размещает блоки в итоговой матрице:

```cpp
FillResultFromBuffer(flat_result, recv_buf, proc / q, proc % q, block_size, padded_n);
```

## 5. Внутрипроцессная схема (TBB)

Внутри каждого рабочего MPI-процесса используется oneTBB для распараллеливания умножения элементов внутри локального блока.

Основная внутрипроцессная операция реализована в функции `ParallelMultiplyBlocks`:

```cpp
void SafronovMMultiplicationMatrixBlockSchemeCannonALL::ParallelMultiplyBlocks(
    const std::vector<double> &a, const std::vector<double> &b, std::vector<double> &c, int block_size) {
  tbb::parallel_for(tbb::blocked_range2d<int>(0, block_size, 0, block_size), [&](const tbb::blocked_range2d<int> &r) {
    for (int i = r.rows().begin(); i < r.rows().end(); ++i) {
      for (int k = 0; k < block_size; ++k) {
        double temp = a[(i * block_size) + k];

        for (int j = r.cols().begin(); j < r.cols().end(); ++j) {
          c[(i * block_size) + j] += temp * b[(k * block_size) + j];
        }
      }
    }
  });
}
```

Здесь используется `tbb::parallel_for` с `blocked_range2d<int>`, который делит блок результата на двумерные участки. Каждый поток получает свою часть координат `(i, j)` и выполняет вычисления только в своём диапазоне.

Особенности этой схемы:

* вычисления идут внутри локальных буферов `local_a`, `local_b`, `local_c`;
* запись в `c` выполняется по непересекающимся диапазонам элементов;
* результат блока накапливается без обращения к общему глобальному массиву;
* TBB используется только для локального ускорения умножения, а не для межпроцессного обмена.

Таким образом, MPI отвечает за распределение блоков между процессами, а TBB ускоряет вычисление внутри одного блока.

В реализации есть несколько ключевых этапов:

## 7. Проверка корректности

Корректность реализации проверяется сравнением результата гибридной версии с эталонной SEQ-версией.

Проверяются следующие случаи:

* совпадение размера результата с исходной матрицей;
* корректность разбиения на блоки;
* корректность работы на размерах, не кратных числу процессов в решётке;
* совпадение численных значений результата после сборки;
* согласованность между данными, полученными разными MPI-процессами.

Используются тесты:

* матрицы малого размера (2×2, 4×4);
* матрицы с одинаковыми значениями;
* случайные матрицы разных размеров;
* различные значения `size_block`, удовлетворяющие ограничениям.

Во всех тестах результаты совпадают с SEQ-версией.

Для паддинга используются нули, поэтому лишние элементы не влияют на итоговую часть матрицы размера `n × n`.

## 10. Выводы

Гибридная схема оправдана, когда матрица достаточно велика и число блоков позволяет загрузить и процессы MPI, и потоки TBB.

Сильная сторона подхода — разделение ответственности: MPI управляет распределением данных, а TBB ускоряет локальные вычисления.

Ограничение появляется там, где коммуникационные расходы начинают конкурировать с вычислениями. В таком случае гибридная схема уже не даёт линейного роста и требует аккуратного выбора размера задачи и конфигурации `ranks × threads_per_rank`.
