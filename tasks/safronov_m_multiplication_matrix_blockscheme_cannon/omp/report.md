## 1. Контекст

В работе реализована OpenMP-версия алгоритма Кэннона для умножения плотных квадратных матриц с использованием блочного разбиения.

Последовательная версия используется как эталон.
OpenMP добавлен для ускорения вычислений за счёт параллельной обработки независимых блоков матрицы.

Основная идея — распараллелить работу по блокам, так как каждый блок результата вычисляется независимо.

## 2. Постановка задачи

На вход подаются:

* `size_block` — размер блока;
* матрица `A`;
* матрица `B`.

На выходе: матрица `C = A × B`.

Ограничения:

* `size_block > 0`;
* матрицы квадратные;
* размеры совпадают;
* `N % size_block == 0`.

## 3. Базовый алгоритм

Используется блочная схема алгоритма Кэннона:

1. Проверка входных данных.
2. Разбиение матриц на блоки `size_block × size_block`.
3. Формирование 4D структур блоков `A`, `B`, `C`.
4. Начальное циклическое смещение блоков.
5. Основной цикл:

   * перемножение соответствующих блоков;
   * накопление результата;
   * циклические сдвиги блоков `A` и `B`.
6. Сборка итоговой матрицы из блоков `C`.

## 4. Схема распараллеливания

### 4.1. Формирование блоков

Заполнение блоков `matrix_blocks_a` и `matrix_blocks_b` распараллелено по двум индексам `(i, j)`.

Каждая итерация обрабатывает отдельный блок матрицы, поэтому потоки не пересекаются по записи.

```cpp
#pragma omp parallel for collapse(2) default(none) \
    shared(matrix_blocks_a, matrix_blocks_b, matrix_a, matrix_b, size_block, columns_blocks)
for (int i = 0; i < columns_blocks; i++) {
  for (int j = 0; j < columns_blocks; j++) {
    int shift = (i + j) % columns_blocks;
    for (int k = 0; k < size_block; k++) {
      for (int col = 0; col < size_block; col++) {
        matrix_blocks_a[i][j][k][col] =
            matrix_a[(i * size_block) + k][(shift * size_block) + col];

        matrix_blocks_b[i][j][k][col] =
            matrix_b[(shift * size_block) + k][(j * size_block) + col];
      }
    }
  }
}
```

### 4.2. Умножение блоков

Основная вычислительная нагрузка — перемножение блоков `C[j][k]`.

```cpp
#pragma omp parallel for collapse(2) default(none) \
    shared(matrix_blocks_a, matrix_blocks_b, matrix_blocks_c, size_block, columns_blocks)
for (int j = 0; j < columns_blocks; j++) {
  for (int k = 0; k < columns_blocks; k++) {
    MultiplyingBlocks(matrix_blocks_a[j][k], matrix_blocks_b[j][k], matrix_blocks_c[j][k], size_block);
  }
}
```

Каждый поток работает со своим блоком результата `matrix_blocks_c[j][k]`, поэтому записи не пересекаются.

### 4.3. Циклические сдвиги блоков

Сдвиги выполняются отдельно по строкам (для A) и по столбцам (для B).
Каждая строка/столбец блоков обрабатывается независимо.

```cpp
#pragma omp parallel for default(none) shared(matrix_blocks_a, columns)
for (int i = 0; i < columns; i++) {
  std::vector<std::vector<double>> tmp = std::move(matrix_blocks_a[i][0]);
  for (int j = 1; j < columns; j++) {
    matrix_blocks_a[i][j - 1] = std::move(matrix_blocks_a[i][j]);
  }
  matrix_blocks_a[i][columns - 1] = std::move(tmp);
}
```

Аналогично выполняется сдвиг для matrix_blocks_b.

### 4.4. Сборка результата

Финальная сборка матрицы C также распараллелена по блокам `(i, j)`.

Каждый поток копирует свой блок в итоговую матрицу.

```cpp
#pragma omp parallel for collapse(2) default(none) \
    shared(matrix_blocks_c, matrix_c, size_block, columns_blocks)
for (int i = 0; i < columns_blocks; i++) {
  for (int j = 0; j < columns_blocks; j++) {
    for (int k = 0; k < size_block; k++) {
      for (int col = 0; col < size_block; col++) {
        matrix_c[(i * size_block) + k][(j * size_block) + col] = matrix_blocks_c[i][j][k][col];
      }
    }
  }
}
```

### 4.5. Итоговая идея распараллеливания

Вся OpenMP-версия построена вокруг одного принципа:

* разбиение матрицы на блоки;
* параллельная обработка независимых блоков;
* отсутствие пересечений по памяти при записи;
* локальное вычисление внутри каждого блока.

Это позволяет ускорить вычисления без изменения логики алгоритма Кэннона.

## 5. Детали реализации

Реализация OpenMP-версии алгоритма Кэннона построена на расширении
последовательной версии без изменения основной логики блочного умножения.

Файлы реализации:

* `omp/include/ops_omp.hpp`
* `omp/src/ops_omp.cpp`

Класс `SafronovMMultiplicationMatrixBlockSchemeCannonOMP` сохраняет структуру BaseTask и переопределяет основные этапы вычисления:

* `ValidationImpl()` — проверка входных данных;
* `PreProcessingImpl()` — очистка выходного буфера;
* `RunImpl()` — основная логика алгоритма;
* `PostProcessingImpl()` — завершение выполнения.

Относительно SEQ-версии ключевые изменения связаны с добавлением параллельных областей OpenMP в трёх местах:

1. Формирование блочных матриц `matrix_blocks_a` и `matrix_blocks_b`.
2. Основной цикл умножения блоков в алгоритме Кэннона.
3. Сборка итоговой матрицы из блоков результата.

Все изменения направлены на распараллеливание независимых итераций без изменения порядка вычислений.

## 6. Проверка корректности

Корректность реализации проверяется сравнением результата OpenMP-версии с последовательной (SEQ) реализацией.

Используется поэлементное сравнение матриц с допустимой погрешностью для `double`.

Функциональные тесты включают:

* матрицы малого размера (2×2, 4×4);
* матрицы с одинаковыми значениями;
* случайные матрицы разных размеров;
* различные значения `size_block`, удовлетворяющие ограничениям.

Во всех тестах результаты совпадают с SEQ-версией.

## 8. Результаты

Производительность измерялась на матрицах размера 1024×1024 при `size_block = 64`.

| Threads | Time (sec) | Speedup | Efficiency |
| ------: | ---------: | ------: | ---------: |
|       1 |       4.80 |    1.00 |       1.00 |
|       2 |       2.95 |    1.63 |       0.81 |
|       4 |       2.70 |    1.78 |       0.44 |
|       8 |       2.60 |    1.85 |       0.23 |

Максимальное ускорение составляет примерно **1.7–1.8×**,
после чего наблюдается снижение эффективности из-за накладных расходов на управление потоками и ограничений памяти.

Основные узкие места:

* работа с четырёхмерными структурами блоков;
* интенсивные операции копирования при формировании и сборке блоков;
* ограниченная выгода от увеличения числа потоков при фиксированном размере блоков.

## 9. Выводы

OpenMP-реализация алгоритма Кэннона показывает умеренное ускорение за счёт параллельной обработки блоков матрицы.

Наибольший выигрыш достигается при небольшом и среднем числе потоков,
где баланс между вычислениями и накладными расходами оптимален.

Дальнейшее увеличение числа потоков не даёт линейного роста производительности
из-за стоимости управления параллельными областями и работы с памятью.
