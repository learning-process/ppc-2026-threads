## 1. Контекст

Данная версия реализует параллельную обработку алгоритма Кэннона с использованием
низкоуровневых потоков `std::thread`. В отличие от TBB и OpenMP, управление
параллелизмом полностью ручное: разбиение данных, создание потоков,
распределение диапазонов и синхронизация выполняются явно.

Это позволяет детально контролировать поведение программы и измерять реальные
накладные расходы на создание и завершение потоков.

## 2. Постановка задачи

Необходимо вычислить произведение двух плотных квадратных матриц.

На вход подаются:

* `size_block` — размер блока;
* матрица `A`;
* матрица `B`.

На выходе: матрица `C = A × B`.

Ограничения:

* `size_block > 0`;
* матрицы квадратные;
* размеры совпадают;
* `N % size_block == 0`.

STL-версия должна сохранить алгоритмическую структуру SEQ, но ускорить
выполнение за счёт распределения независимых операций между потоками.

## 3. Базовый алгоритм

Используется блочная схема алгоритма Кэннона:

1. Проверка входных данных.
2. Разбиение матриц на блоки `size_block × size_block`.
3. Формирование 4D структур блоков `A`, `B`, `C`.
4. Начальное циклическое смещение блоков.
5. Основной цикл:

   * перемножение соответствующих блоков;
   * накопление результата;
   * циклические сдвиги блоков `A` и `B`.
6. Сборка итоговой матрицы из блоков `C`.

## 4. Схема распараллеливания

### 4.1 Общий подход

В STL-версии используется ручное управление потоками через `std::thread`.
Основная идея — разделение диапазонов индексов между потоками и
параллельное выполнение независимых частей вычислений.

Количество потоков определяется через:

```cpp
std::thread::hardware_concurrency()
```

Если значение недоступно, используется 1 поток.

Далее диапазон индексов разбивается на части:

* базовый размер диапазона
* остаток распределяется по первым потокам

### 4.2 Распараллеливание заполнения блоков

Формирование блоков матриц A и B выполняется параллельно:

* каждый поток обрабатывает свой диапазон строк блоков
* вычисление shift выполняется локально
* запись идёт в уникальные элементы массива блоков

Фрагмент реализации:

```cpp
threads.emplace_back(&SafronovMMultiplicationMatrixBlockSchemeCannonSTL::FillBlocks,
                     start, end, columns_blocks,
                     size_block,
                     std::ref(matrix_a),
                     std::ref(matrix_b),
                     std::ref(matrix_blocks_a),
                     std::ref(matrix_blocks_b));
```

### 4.3 Распараллеливание умножения блоков (основная нагрузка)

В функции `AlgorithmCannon` каждый поток получает свой диапазон
строк блоков `j` и выполняет умножение независимых блоков C.

Ключевая идея:

* каждый поток работает с уникальным набором `matrix_blocks_c[j][k]`
* отсутствует пересечение записей между потоками
* внутри блока используется локальная арифметика без синхронизации

```cpp
threads.emplace_back(&SafronovMMultiplicationMatrixBlockSchemeCannonSTL::CannonTask,
                     start, end, columns_blocks,
                     size_block,
                     std::ref(matrix_blocks_a),
                     std::ref(matrix_blocks_b),
                     std::ref(matrix_blocks_c));
```

### 4.4 Распараллеливание циклических сдвигов

Сдвиги блоков A и B выполняются независимо по индексам:

* `ShiftALeftTask` — каждый поток обрабатывает диапазон строк блоков A
* `ShiftBUpTask` — каждый поток обрабатывает столбцы блоков B

Каждый поток модифицирует только свой участок данных.

### 4.5 Сборка результата

Финальная сборка матрицы C также распараллелена:

* каждый поток обрабатывает свой диапазон блоков
* запись идёт в уникальные позиции итоговой матрицы

### 4.6 Синхронизация

Синхронизация реализуется через `join()`:

* каждый этап (fill, compute, shift, final merge) завершается ожиданием потоков
* это обеспечивает корректные переходы между фазами

## 5. Детали реализации

Реализация STL-версии построена на ручном управлении потоками через `std::thread`.

Основные особенности:

* диапазоны вычисляются через `base_size` и остаток `rem`
* каждый поток получает непрерывный диапазон `[start, end)`
* используется `std::thread::hardware_concurrency()`
* данные передаются по ссылке через `std::ref`

Ключевые функции:

* `FillBlocks` — формирование блоков A и B
* `CannonTask` — умножение блоков
* `ShiftALeftTask`, `ShiftBUpTask` — сдвиги блоков
* `FillingTask` — сборка результата

## 6. Проверка корректности

Корректность проверяется сравнением STL-версии с SEQ.

Проверки:

* совпадение размеров матриц
* корректность разбиения на блоки
* совпадение всех элементов результата

Дополнительно:

* проверка крайних случаев
* проверка минимального числа блоков

## 8. Результаты

Производительность измерялась на матрице 1024×1024 при `size_block = 64`.

| Threads | Time (s) | Speedup | Efficiency |
| ------: | -------: | ------: | ---------: |
|       1 |     4.80 |    1.00 |       1.00 |
|       2 |     2.60 |    1.85 |       0.92 |
|       4 |     2.40 |    2.00 |       0.50 |
|       8 |     2.50 |    1.92 |       0.24 |

Ускорение достигается за счёт разделения вычислений по независимым диапазонам блоков.
Каждый поток работает со своим участком данных без пересечений.

Основные ограничения масштабируемости связаны с overhead создания потоков и join,
а также уменьшением полезной работы на поток при росте их числа.

Наилучшая производительность наблюдается при 2–4 потоках.

## 9. Выводы

Реализация на `std::thread` даёт ускорение примерно до 2x относительно SEQ.

При этом масштабирование ограничено накладными расходами и отсутствием
автоматического планировщика задач.

STL-версия требует аккуратного ручного управления диапазонами и синхронизацией.
