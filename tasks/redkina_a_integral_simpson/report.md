## Введение

Численное интегрирование функций многих переменных
является важной задачей вычислительной
математики, возникающей в физике, экономике,
инженерных расчётах и других областях.

Метод Симпсона (парабол) для многомерного случая строится
как тензорное произведение
одномерных формул и обладает повышенной
точностью для гладких функций.

В условиях современного параллельного программирования реализация таких алгоритмов с использованием
различных технологий (OpenMP, Intel oneTBB, STL `std::async`, а также гибрид MPI + многопоточность в каталоге **all**)
позволяет исследовать эффективность распараллеливания и
выбрать подходящий вариант в зависимости от
аппаратной платформы и размера задачи.

## Постановка задачи

**Цель работы:**  
Реализовать последовательную и параллельные версии
алгоритма вычисления определённого интеграла
функции нескольких переменных по прямоугольной области методом
Симпсона, провести их тестирование и сравнить производительность.

**Определение задачи:**  
Для заданной функции, определённой на прямоугольной области,
требуется вычислить приближённое
значение интеграла с использованием составной формулы Симпсона.
Область разбивается по каждому
измерению на **n_i** равных частей (число разбиений по оси должно быть чётным).

Интеграл аппроксимируется взвешенной суммой значений функции в узлах сетки.

**Ограничения:**

- Размерность d >= 1.
- Векторы границ a и b должны иметь одинаковую длину.
- По каждой оси количество разбиений **n_i** должно быть положительным и чётным (как в коде валидации).
- Функция должна быть определена во всех узлах сетки.

## Описание алгоритма (последовательная версия)

Последовательный алгоритм обходит все узлы многомерной сетки в
лексикографическом порядке
(последнее измерение меняется быстрее всего — см. функцию `AdvanceIndices` в реализации).

Для каждого узла вычисляется вес Симпсона по каждому измерению:

- w = 1 для граничных узлов i = 0 или i = n_i;
- w = 4 для внутренних узлов с нечётным индексом;
- w = 2 для внутренних узлов с чётным индексом.

Общий вес узла равен произведению одномерных весов. Затем значение
функции в узле умножается на этот вес и суммируется.
После обхода всех узлов результат умножается на
**(h_1 · h_2 · … · h_d) / (3^d)**, где **h_i = (b_i − a_i) / n_i**.

## Схемы распараллеливания

### OpenMP-версия

В OpenMP используется параллельный цикл по линейному индексу узлов. Предварительно вычисляются массив **strides** для преобразования линейного индекса в многомерные координаты и **таблицы одномерных весов Симпсона** по каждой оси (функция `PrecomputeWeights`), чтобы в параллельном цикле не повторять ветвления по индексу узла.

Это позволяет избежать вложенных циклов по измерениям и обеспечить
равномерное распределение работы между потоками (`schedule(static)`).

#### Принцип работы OpenMP-версии

1. Вычисляются шаги сетки **h_i**, произведение **h_prod** и таблицы весов по осям.
2. Строится массив strides: **strides[d−1] = 1**, далее **strides[i−1] = strides[i] · (n[i] + 1)**; число узлов **total_nodes = strides[0] · (n[0] + 1)**. По линейному индексу **idx** координаты восстанавливаются последовательным делением и остатком (как в коде), что эквивалентно покомпонентному разложению индекса.
3. Параллельная секция `#pragma omp parallel` с **`reduction(+: total_sum)`** и цикл **`#pragma omp for schedule(static)`** по **idx**.
4. В каждой итерации поток берёт готовые веса из таблиц, формирует точку в пространстве аргументов и добавляет **w_prod · f(x)** к **total_sum**.
5. После цикла результат умножается на общий множитель Симпсона **(h_prod / 3^d)**.

### TBB-версия

В Intel oneTBB используется **`tbb::parallel_reduce`** с диапазоном **`tbb::blocked_range<size_t>(0, total_points)`**: рантайм разбивает интервал линейных индексов на поддиапазоны и выполняет редукцию суммой. Предвычисления весов и strides совпадают по смыслу с OpenMP-версией. Вклад узла считается **непосредственно в лямбде** обработки поддиапазона (отдельной функции `ComputeNodeContribution` в коде нет).

#### Принцип работы TBB-версии

1. Аналогично вычисляются шаги, **h_prod**, таблицы весов и **strides**, задаётся **total_points**.
2. Вызывается `tbb::parallel_reduce` с начальным значением 0.0, телом, обходящим индексы в выданном поддиапазоне, и бинарной операцией сложения для частичных сумм.
3. После редукции результат умножается на **(h_prod / 3^d)**.

### STL-версия

Диапазон **`[0, total_points)`** режется на блоки; число блоков определяется **`std::thread::hardware_concurrency()`** (если возвращается 0, в коде используется значение 2). Для каждого блока запускается **`std::async(std::launch::async, …)`**, итоги собираются через **`std::future<double>::get()`**. Суммирование по диапазону индексов совпадает по логике с OpenMP (те же strides и таблицы весов).

#### Принцип работы STL-версии

1. Определяется **num_threads** (см. выше), вычисляются длины блоков и распределяется остаток от деления **total_points** на **num_threads**.
2. Для каждого непустого блока запускается **`std::async`** с вызовом **`ComputeRangeSum`** на своём **start…end**.
3. Все **future** дожидаются, частичные суммы складываются; затем применяется **(h_prod / 3^d)**.

### ALL-версия (MPI + STL)

Интервал узлов **`[0, total_points)`** делится между MPI-процессами почти поровну. Внутри процесса локальный отрезок может дополнительно делиться между задачами **`std::async`**: число параллельных задач не превышает **`min(hardware_concurrency(), длина_локального_отрезка)`**; если достаточно одного потока, вызывается последовательный **`ComputeRangeSum`** без создания **async**. Локальные суммы объединяются **`MPI_Allreduce`**, затем результат масштабируется множителем **(h_prod / 3^d)**.

#### Принцип работы ALL-версии

1. Запрашиваются **MPI_Comm_rank** и **MPI_Comm_size**.
2. Вычисляются шаги сетки, таблицы весов Симпсона и **strides**, **total_points**.
3. Локальные границы **local_start … local_end** задаются распределением узлов по процессам.
4. **`ComputeLocalSumMPI`** при необходимости режет локальный диапазон на части и использует **`std::async`** / **`future`**, либо один проход без async.
5. **`MPI_Allreduce`** собирает глобальную сумму по процессам.
6. Итог умножается на **(h_prod / 3^d)**.
