## 1. Контекст

Задача — численно найти значение кратного интеграла по прямоугольнику в R^d составной квадратурой Симпсона.
Последовательная реализация задаёт эталон результата и базовое время (работа p = 1), относительно которых
сравниваются параллельные реализации в общих тестах курса.

## 2. Постановка задачи

**Вход (`InType`):** векторы границ **a**, **b** одинаковой длины **d**; по каждой оси **n_i > 0** и **чётное**
(число отрезков); пользовательская функция **f(x)**.

**Выход (`OutType`):** одно вещественное число — приближение интеграла.

**Ограничения:** **d ≥ 1**; **a_i < b_i**; размерности **a**, **b**, **n** согласованы; **f** определена во всех узлах
сетки; в **ValidationImpl** отклоняются некорректные входы.

**Крайние случаи:** минимальная сетка по оси (чётное **n_i**); проверки из **`tests/functional/main.cpp`**
(константы, полиномы, тригонометрия, экспонента, несколько размерностей).

## 3. Базовый алгоритм

По каждой оси строится **n_i + 1** узлов; всего **∏_i (n_i + 1)** узлов. Обход в лексикографическом порядке
(`AdvanceIndices`: быстрее всего меняется последняя координата). Для узла **(i_1,…,i_d)** вес по оси задаётся правилами
1/4/2 Симпсона; полный вес — произведение одномерных весов; к сумме добавляется **w_prod · f(x)**.

**Время:** **O(∏ (n_i + 1))** вызовов **f** (доминирует вычисление функции).

**Память:** **O(d)** под индексы и точку плюс входные векторы.

**Инвариант:** после полного обхода **sum** совпадает с суммой взвешенных значений **f** по всем узлам; итог умножается
на **(h_1·…·h_d)/3^d**, **h_i = (b_i − a_i)/n_i**.

## 4. Детали реализации

**Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.

Каркас задачи курса: **`ValidationImpl`** (проверка размеров, **a < b**, чётность **n_i**, наличие **func**);
**`PreProcessingImpl`** — копирование входа, обнуление результата; **`RunImpl`** — двойной цикл по индексам через
`AdvanceIndices` и **`EvaluatePoint`**; **`PostProcessingImpl`** — запись **`result_`** в выход.

Критический фрагмент (идея суммирования):

```cpp
// File: seq/src/ops_seq.cpp (фрагмент)
while (has_next) {
  EvaluatePoint(a_, h, n_, indices, func_, point, sum);
  has_next = AdvanceIndices(indices, n_);
}
result_ = (h_prod / denominator) * sum;
```

## 5. Проверка корректности

Результат **SEQ** сравнивается с аналитикой в тестах (где задана) и с другими реализациями в общем наборе
**`tests/functional/main.cpp`**: одинаковые **`InType`** для всех реализаций, допуск по точности задаётся тестами курса.
