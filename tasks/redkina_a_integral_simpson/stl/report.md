# STL-версия

Реализация: `stl/src/ops_stl.cpp`.

## Идея

Число узлов сетки **`total_points`** считается так же, как в OMP/TBB. Диапазон **`[0, total_points)`** режется на
**непересекающиеся полуинтервалы** `[start, end)`; каждый обрабатывается в отдельной асинхронной задаче. Сумма
взвешенных значений функции по своему куску считается в **`ComputeRangeSum`** (линейный индекс → индексы через
`strides`, веса из таблиц).

## Параллелизм через `std::async`

- **`num_threads = std::thread::hardware_concurrency()`**; если вернулось **0**, в коде используется запасное значение
  **2**, чтобы всё же распараллелить работу.
- Длина блока: **`total_points / num_threads`**, остаток от деления распределяется по первым блокам (каждый из первых
  `rem` блоков на один узел длиннее).
- Для каждого непустого блока вызывается **`std::async(std::launch::async, …)`** с лямбдой, возвращающей локальную
  сумму на `[start, end)`.
- **`std::future<double>::get()`** для всех задач; глобальная сумма — сумма локальных.

После сбора задач результат умножается на **(h_prod / 3^d)**.
