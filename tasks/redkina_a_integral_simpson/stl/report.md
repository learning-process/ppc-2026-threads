## 1. Контекст

Реализация на стандартной библиотеке без OpenMP/TBB: диапазон линейных индексов режется на блоки, каждый блок
обрабатывается асинхронно. Это ручное разбиение «поток работы → задача», а не обёртка в **`std::thread`** с явным
**join**; синхронизация — через **`std::future<double>::get()`** после запуска набора задач.

## 2. Постановка задачи

Входные данные: границы a и b, чётные n_i по каждой оси, функция f.
Выход: численное значение интеграла.
SEQ используется как эталон корректности и базовое время.

## 3. Базовый алгоритм

Тот же подсчёт суммы по узлам с предвычисленными весами и **strides**; функция **`ComputeRangeSum(start, end, …)`**
обходит **idx** на своём полуинтервале.

## 4. Схема распараллеливания

- **Число задач:** **`num_threads = std::thread::hardware_concurrency()`**, при **0** в коде подставляется **2**.
- **Разбиение:** **`total_points / num_threads`**, остаток распределяется по первым блокам (классическое разбиение с
  передачей лишних единиц первым индексам).
- **Запуск:** **`std::async(std::launch::async, lambda)`** на каждый непустой диапазон; **`get()`** собирает частичные
  суммы — отдельные **mutex** для суммы не нужны, каждая задача возвращает свой **double**.
- **Гонки:** отсутствуют при условии непересекающихся диапазонов **[start, end)**.

В отличие от учебного примера с **`join()`** внутри цикла создания потоков, здесь задачи стартуют пакетом, затем
выполняется серия **get()** — параллелизм не сериализуется на этапе ожидания до завершения постановки очереди задач.

## 5. Детали реализации

**Файлы:** `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`.

```cpp
// File: stl/src/ops_stl.cpp (идея)
futures.push_back(std::async(std::launch::async, [=, &a, &h, &weights, &strides, &func]() {
  return ComputeRangeSum(start, end, a, h, weights, strides, func, dim);
}));
```

## 6. Проверка корректности

Сравнение с **SEQ** и **`tests/functional/main.cpp`**. Граничные случаи — малые **total_points**, когда число создаваемых
**async** может быть меньше **`num_threads`** (цикл прерывается при исчерпании точек).

## 7. Экспериментальная среда

**Release**, **g++**. Число асинхронных задач связано с **`hardware_concurrency()`** на машине автора; для сравнения с OMP
имеет смысл выставлять **`PPC_NUM_THREADS`** там, где инфраструктура курса синхронизирует лимит потоков.

## 9. Выводы

`std::async` даёт кроссплатформенное распараллеливание без OpenMP; на измеренной конфигурации время оказалось лучшим среди
потоковых реализаций в этом сценарии.
