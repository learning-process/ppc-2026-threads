# ALL-версия (MPI + STL)

Реализация: `all/src/ops_all.cpp`.

## Два уровня параллелизма

1. **Между процессами MPI**: линейный диапазон узлов `[0, total_points)` делится между процессами почти поровну:
   первые `rem = total_points % world_size` процессов получают на один узел больше остальных.
2. **Внутри процесса**: локальный фрагмент `[local_start, local_end)` суммируется функцией **`ComputeLocalSumMPI`**,
   которая при необходимости снова режет диапазон и использует **`std::async`** / **`std::future`**, как в STL-версии.

## Детали внутреннего распараллеливания

- Число параллельных задач не больше **`min(std::thread::hardware_concurrency(), local_size)`**, где **`local_size =
  local_end − local_start`**. Если доступных «потоков» по API **0**, подставляется **2**, но итоговое число задач всё равно
  ограничивается **`local_size`**.
- Если после этого остаётся **одна** задача (**`num_threads == 1`** или короткий локальный хвост), вызывается один проход
  **`ComputeRangeSum`** без создания **`std::async`**.
- Иначе локальный интервал режется на **`num_threads`** блоков с распределением остатка по первым блокам; для каждого
  блока — **`std::async`**, затем суммирование результатов **`get()`**.

## Завершение

Локальные суммы собираются **`MPI_Allreduce`** с операцией **`MPI_SUM`**. Итог умножается на **(h_prod / 3^d)** на каждом
процессе (результат одинаковый после редукции).

Предвычисления весов Симпсона и **strides** совпадают по смыслу с OMP/STL/TBB.
