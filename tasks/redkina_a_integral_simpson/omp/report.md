## 1. Контекст

Из **SEQ** перенесена та же математика Симпсона; распараллеливается только суммирование по узлам сетки: один
**линейный индекс** **0 … total_nodes−1** вместо вложенных циклов по осям. Граница распараллеливания — тело цикла по
**idx** внутри **`#pragma omp parallel`**.

## 2. Постановка задачи

Совпадает с **`seq/report.md`**. **SEQ** — baseline для сравнения значения интеграла и для оценки ускорения по времени.

## 3. Базовый алгоритм

Суммирование взвешенных значений **f** по всем узлам; тот же множитель **(h_prod/3^d)**, что и в **SEQ**. Отличие —
итерации по **idx** распределяются между потоками OpenMP.

## 4. Схема распараллеливания

- **Область:** одна **parallel region** на весь цикл по узлам.
- **Директивы:** в коде **`#pragma omp parallel default(none) ... reduction(+: total_sum)`** и
  **`#pragma omp for schedule(static)`** по **idx** (см. `ops_omp.cpp`).
- **`default(none)`** — явный список **`shared(...)`** для всех используемых в цикле данных; локальные буферы
  **indices** и **point** объявлены внутри параллельной области (по смыслу **private** для каждого потока).
- **`reduction(+: total_sum)`** — устраняет гонку при накоплении суммы; итог редукции согласован с концом **parallel**.
- **`schedule(static)`** — равномерное статическое разбиение **idx** по потокам, предсказуемая нагрузка при однородной
  стоимости итерации.
- **Барьер:** неявный в конце **parallel** (и в конце **omp for** внутри него), прежде чем вычислять финальный **result_**
  вне параллельной области.

## 5. Детали реализации

**Файлы:** `omp/include/ops_omp.hpp`, `omp/src/ops_omp.cpp`.

Предвычисляются **таблицы весов** (`PrecomputeWeights`) и **strides** (`ComputeStrides`); в цикле нет гонок за счёт
**reduction** и независимых диапазонов **idx**.

```cpp
// File: omp/src/ops_omp.cpp (фрагмент)
#pragma omp parallel default(none) \
    shared(total_nodes, a_local, h_local, weights_local, strides_local, func_local, dim) \
    reduction(+ : total_sum)
{
  std::vector<int> indices(dim);
  std::vector<double> point(dim);
#pragma omp for schedule(static)
  for (int idx = 0; idx < total_nodes; ++idx) {
    // восстановление indices, w_prod, total_sum += w_prod * func_local(point);
  }
}
```

Число потоков OpenMP задаётся стандартно: **`OMP_NUM_THREADS`**, в курсе также **`PPC_NUM_THREADS`** (см. документацию
runner’а).

## 6. Проверка корректности

Сравнение с **SEQ** и общий прогон **`tests/functional/main.cpp`**. При **1 потоке** результат должен совпадать с **SEQ**
(в пределах типовых допусков).

## 7. Экспериментальная среда

Тип сборки **Release**, компилятор и ОС — как в корневом **`report.md`**. Перед замерами: **`export PPC_NUM_THREADS=T`**
(и **`OMP_NUM_THREADS`**) для фиксации числа потоков при воспроизведении.

## 9. Выводы

OpenMP даёт ускорение за счёт параллельного обхода узлов; **`schedule(static)`** уместен при равной стоимости итераций.
Дальнейший рост **T** ограничен памятью, кэшем и накладными расходами; точные цифры — в корневом отчёте и при заданном
**PPC_NUM_THREADS**.
