# OpenMP-версия (OMP)

Реализация: `omp/src/ops_omp.cpp`.

## Идея

Вместо вложенных циклов по измерениям используется **один линейный индекс** `idx` от `0` до `total_nodes − 1`,
где `total_nodes = strides[0] · (n[0] + 1)` — число узлов сетки. По `idx` восстанавливаются многомерные индексы узла
с помощью массива **strides** (аналогично перемешиванию осей в многомерном массиве).

## Предвычисления

- **Шаги сетки** `h[i]` и произведение **h_prod** для финального множителя Симпсона.
- **Таблицы весов** `PrecomputeWeights`: для каждого измерения массив длины `n_i + 1` с коэффициентами 1, 4 или 2 по
  правилам метода Симпсона — в горячем цикле берётся произведение готовых значений, без повторного ветвления по
  индексу.
- **Strides** `ComputeStrides`: `strides[d−1] = 1`; далее `strides[i−1] = strides[i] · (n[i] + 1)`. Тогда для `idx`
  последовательно: `indices[j] = remainder / strides[j]`, `remainder %= strides[j]` (эквивалентно покомпонентному
  восстановлению координат из линейного индекса).

## Параллелизм

- Внешняя область: `#pragma omp parallel` с `reduction(+: total_sum)`.
- Цикл `#pragma omp for schedule(static)` по `idx`: статическое распределение итераций, предсказуемая нагрузка на
  потоки.
- В теле цикла: восстановление индексов, `w_prod` из таблиц весов, вызов функции интегрируемой функции, накопление в
  `total_sum`.

После параллельной части результат умножается на **(h_prod / 3^d)**, как в последовательной версии («усреднения» здесь
нет — только общий множитель Симпсона).
