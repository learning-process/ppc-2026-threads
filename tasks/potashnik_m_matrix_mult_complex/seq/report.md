## 1. Контекст

Последовательная версия реализована как базовый эталон для
сравнения с параллельными технологиями (OMP, TBB, STL, ALL).
Алгоритм перебирает все ненулевые элементы матриц в формате
CCS и накапливает произведения в ассоциативной таблице
`std::map`. Отсутствие параллелизма позволяет получить
детерминированный и воспроизводимый эталон времени выполнения.

## 2. Постановка задачи

- **Вход:** две разреженные матрицы `A` (height × width) и `B` (height' × width'), где `width == height'`.
Формат CCS (три вектора: значения `val`, индексы строк `row_ind`, индексы столбцов `col_ptr`).
- **Выход:** разреженная матрица `C = A × B` в формате CCS.
- **Алгоритм:** перебор всех `nnz(A)` и `nnz(B)` с накоплением результатов в `std::map<(row, col), Complex>`.
- **Ограничения:** умножение определено только если `A.width == B.height`; нулевые элементы не хранятся.

## 3. Описание алгоритма

Вычислительное ядро — двойной вложенный цикл:

```cpp
for (size_t i = 0; i < matrix_left.Count(); i++) {
  size_t row_left = row_ind_left[i];
  size_t col_left = col_ptr_left[i];
  Complex left_val = val_left[i];

  for (size_t j = 0; j < matrix_right.Count(); j++) {
    size_t row_right = row_ind_right[j];
    size_t col_right = col_ptr_right[j];
    Complex right_val = val_right[j];

    if (col_left == row_right) {
      buffer[{row_left, col_right}] += left_val * right_val;
    }
  }
}
```

Асимптотика: O(`nnz_A` × `nnz_B`). После заполнения буфера
результат записывается в `CCSMatrix` путём обхода упорядоченной карты.

## 4. Детали реализации

**Файлы:**

- `seq/include/ops_seq.hpp` — объявление класса.
- `seq/src/ops_seq.cpp` — реализация алгоритма.

**Структура пайплайна:**

- `ValidationImpl` — проверка совместимости: `A.width == B.height`.
- `PreProcessingImpl` / `PostProcessingImpl` — без действий.
- `RunImpl` — извлечение данных, двойной цикл, формирование результата.

**Формат CCS (Compressed Column Storage):**

В данной реализации используется нестандартный вариант CCS: три плоских вектора `val`, `row_ind`, `col_ptr`.
Каждый индекс `i` соответствует одному ненулевому элементу.
Это упрощает перебор и вставку, но увеличивает объём при плотных матрицах.

Последовательная версия стабильна и детерминирована. Время
выполнения 149 мс при `nnz = 22000` обусловлено O(`nnz_A` ×
`nnz_B`) сложностью алгоритма и логарифмическими вставками в
`std::map`. Основное узкое место — не вычислительная нагрузка,
а структура данных `std::map` (аллокации и балансировка
красно-чёрного дерева). Эта версия служит базовым эталоном для
всех параллельных реализаций.
