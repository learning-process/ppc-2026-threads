## 1. Введение

Задача — приближенно вычислить значение кратного интеграла по d-мерному гиперпараллелепипеду,
используя составную квадратурную схему Симпсона.

## 2. Постановка задачи

Требуется вычислить приближенное значение многомерного определенного интеграла
от заданной функции на гиперпрямоугольной области.

**Входные данные:**

- Вектор `a` типа `std::vector<double>` — нижние пределы интегрирования по каждому измерению.
- Вектор `b` типа `std::vector<double>` — верхние пределы интегрирования по каждому измерению.
- Вектор `n` типа `std::vector<int>` — количество подинтервалов разбиения по каждому измерению
(каждое значение должно быть четным и положительным).
- Объект `func` типа `std::function<double(const std::vector<double>&)>` — подынтегральная функция.

**Выходные данные:**

- Значение типа `double` — приближенное значение интеграла.

**Ограничения:**

- Размерности векторов `a`, `b` и `n` должны быть одинаковыми и больше нуля.
- Для каждого измерения `i` должно выполняться `a[i] < b[i]`.
- Каждый элемент `n[i]` должен быть четным положительным числом.

## 3. Схема распараллеливания

В параллельной реализации с использованием стандартной библиотеки STL (std::thread)
применяется декомпозиция по данным с ручным управлением потоками.

Вычисляется общее количество узлов сетки `total_points`
как произведение всех элементов `dimensions_[i]` (количество точек по каждому измерению).
Каждое измерение имеет размерность `n[i] + 1`,
поэтому общее количество узлов равно `(n₁+1) × (n₂+1) × ... × (n_d+1)`.

Количество потоков определяется через функцию `ppc::util::GetNumThreads()`
или с помощью `std::thread::hardware_concurrency()`. Если полученное значение равно нулю,
используется значение по умолчанию (2 потока).
Количество потоков также ограничивается общим количеством узлов сетки.

Общий диапазон индексов от 0 до `total_points` разбивается на непрерывные блоки.
Каждый поток получает свой поддиапазон `[start_idx, end_idx)`.
Размер блока для каждого потока вычисляется как `points_per_thread = total_points / num_threads`,
а остаток распределяется между первыми потоками.

Для каждого потока создается объект `std::thread`, который вызывает метод `ComputePartialSumRange()`.
Этот метод для каждого индекса в диапазоне выполняет преобразование глобального индекса
в многомерный индекс, вычисляет вес и координаты точки, вызывает подынтегральную функцию
и накапливает частичную сумму в локальной переменной `local_sum`.
Результат сохраняется в соответствующий элемент массива `partial_sums`.

После завершения всех потоков (через вызов `join()`)
частичные суммы объединяются с помощью `std::accumulate`.

Особенностью данной реализации является гибридный подход:
если общее количество точек `total_points` меньше порогового значения (10000),
выполняется последовательная обработка через метод `ProcessSequential()`,
иначе запускается параллельная обработка через метод `ProcessParallel()`.
Это позволяет избежать накладных расходов на создание потоков для малых задач.

## 4. Детали реализации

Реализация находится в файлах `tasks\vlasova_a_simpson_method\stl\include\ops_stl.hpp`
и `tasks\vlasova_a_simpson_method\stl\src\ops_stl.cpp` и представляет собой класс
`VlasovaASimpsonMethodSTL`, унаследованный от базового класса `BaseTask`.

**Ключевые компоненты и их функции:**

1. **`ValidationImpl()`**:
    - Проверяет согласованность размерностей векторов `a`, `b` и `n`.
    - Убеждается, что нижний предел меньше верхнего по каждому измерению (`a[i] < b[i]`).
    - Проверяет, что количество шагов `n[i]` является четным и положительным.
    - Проверяет, что указатель на функцию `func` корректен.

2. **`PreProcessingImpl()`**:
    - Вычисляет шаги интегрирования `h_[i] = (b[i] - a[i]) / n[i]`.
    - Определяет размерность сетки `dimensions_[i] = n[i] + 1`.

3. **`RunImpl()`**:
    - Вычисляет общее количество точек `total_points` как произведение всех `dimensions_[i]`.
    - Если `total_points < 10000`, вызывает `ProcessSequential()` для последовательной обработки.
    - Иначе вызывает `ProcessParallel()` для параллельной обработки.
    - Вычисляет итоговый множитель `factor` как произведение `h_[i] / 3.0`.
    - Устанавливает `result_ = sum * factor`.

4. **`ProcessSequential()`**:
    - Выполняет последовательный перебор всех индексов от 0 до `total_points - 1`.
    - Для каждого индекса выполняет преобразование в многомерный индекс
     через операции деления и взятия остатка.
    - Вызывает `ComputeWeight()` и `ComputePoint()` для каждого узла.
    - Накопление результата в переменной `sum`.

5. **`ProcessParallel()`**:
    - Определяет количество потоков `num_threads` через `ppc::util::GetNumThreads()`
    или `std::thread::hardware_concurrency()`.
    - Вычисляет `points_per_thread = total_points / num_threads`
     и `remainder = total_points % num_threads`.
    - Создает вектор `std::vector<std::thread> threads`
    и вектор `std::vector<double> partial_sums(num_threads, 0.0)`.
    - Для каждого потока запускает лямбда-функцию,
    которая вызывает `ComputePartialSumRange()`.
    - После завершения всех потоков объединяет `partial_sums` через `std::accumulate`.

6. **`ComputePartialSumRange()`**:
    - Принимает начальный и конечный индексы диапазона.
    - Для каждого индекса в диапазоне выполняет преобразование
    в многомерный индекс через операции деления и взятия остатка.
    - Вызывает `ComputeWeight()` и `ComputePoint()` для каждого узла.
    - Возвращает накопленную частичную сумму через параметр `partial_sum`.

7. **`PostProcessingImpl()`**:
    - Проверяет, что результат `result` является конечным числом (`std::isfinite`).

Методы `ComputeWeight()` и `ComputePoint()` полностью идентичны последовательной версии.

**Рекомендации по использованию памяти:**

Алгоритм не требует выделения больших объемов памяти.
Основное потребление связано с хранением входных векторов (`a`, `b`, `n`)
и небольшого служебного состояния (`h_`, `dimensions_`).
Значения функции не сохраняются, а сразу добавляются в сумму.
Каждый поток использует собственные локальные векторы (`cur_index`, `cur_point`),
что позволяет избежать конфликтов при доступе к памяти.
Для хранения частичных сумм используется вектор `partial_sums` размера `num_threads`.

**Анализ результатов:**

Параллельная реализация на STL (std::thread) показала результаты хуже последовательной версии.
Время выполнения в режиме `pipeline` увеличилось с 0.4246 с до 0.5205 с (ускорение 0.82x),
а в режиме `task_run` — с 0.4191 с до 0.5061 с (ускорение 0.83x).
Фактически, параллельная версия работает медленнее последовательной примерно на 20-25%.

Это объясняется следующими факторами:

1. **Накладные расходы на создание потоков:** Создание и запуск 20 потоков с помощью `std::thread`
требует значительного времени. Для задачи с 8 миллионами итераций это время сопоставимо с самими вычислениями.
2. **Отсутствие редукции в std::thread:** Здесь используется отдельный массив `partial_sums`,
что требует дополнительного ожидания потоков и финального суммирования.
3. **Малый объем вычислений на итерацию:** Для каждой итерации выполняется относительно небольшой объем работы.
Накладные расходы на управление потоками превышают выигрыш от параллелизации.
4. **Порог переключения (10000 итераций):** Для малых задач используется последовательная версия,
но для данной задачи (8 млн итераций) используется параллельная, где накладные расходы доминируют.

## 8. Приложения

```cpp
void VlasovaASimpsonMethodSTL::ProcessParallel(double &sum) const {
  size_t dim = task_data_.a.size();

  size_t total_points = 1;
  for (size_t i = 0; i < dim; ++i) {
    total_points *= static_cast<size_t>(dimensions_[i]);
  }

  unsigned int num_threads = ppc::util::GetNumThreads();
  if (num_threads == 0) {
    num_threads = std::thread::hardware_concurrency();
  }
  if (num_threads == 0) {
    num_threads = 2;
  }

  if (static_cast<size_t>(num_threads) > total_points) {
    num_threads = static_cast<unsigned int>(total_points);
  }

  size_t points_per_thread = total_points / num_threads;
  size_t remainder = total_points % num_threads;

  std::vector<std::thread> threads;
  std::vector<double> partial_sums(num_threads, 0.0);

  size_t start_idx = 0;
  for (unsigned int thread_idx = 0; thread_idx < num_threads; ++thread_idx) {
    size_t end_idx = start_idx + points_per_thread + (thread_idx < remainder ? 1 : 0);
    threads.emplace_back([this, start_idx, end_idx, &partial_sums, thread_idx]() {
      double local_sum = 0.0;
      ComputePartialSumRange(static_cast<int>(start_idx), static_cast<int>(end_idx), local_sum);
      partial_sums[thread_idx] = local_sum;
    });
    start_idx = end_idx;
  }

  for (auto &thread : threads) {
    if (thread.joinable()) {
      thread.join();
    }
  }

  sum = std::accumulate(partial_sums.begin(), partial_sums.end(), 0.0);
}

bool VlasovaASimpsonMethodSTL::RunImpl() {
  size_t dim = task_data_.a.size();

  size_t total_points = 1;
  for (size_t i = 0; i < dim; ++i) {
    total_points *= static_cast<size_t>(dimensions_[i]);
  }

  double sum = 0.0;
  if (total_points < 10000) {
    ProcessSequential(sum);
  } else {
    ProcessParallel(sum);
  }

  double factor = 1.0;
  for (size_t i = 0; i < dim; ++i) {
    factor *= h_[i] / 3.0;
  }

  result_ = sum * factor;
  GetOutput() = result_;

  return true;
}
```
