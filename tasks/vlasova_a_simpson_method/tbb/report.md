## 1. Введение

Задача — приближенно вычислить значение кратного интеграла по d-мерному гиперпараллелепипеду,
используя составную квадратурную схему Симпсона.

## 2. Постановка задачи

Требуется вычислить приближенное значение многомерного определенного интеграла 
от заданной функции на гиперпрямоугольной области.

**Входные данные:**
- Вектор `a` типа `std::vector<double>` — нижние пределы интегрирования по каждому измерению.
- Вектор `b` типа `std::vector<double>` — верхние пределы интегрирования по каждому измерению.
- Вектор `n` типа `std::vector<int>` — количество подинтервалов разбиения по каждому измерению
 (каждое значение должно быть четным и положительным).
- Объект `func` типа `std::function<double(const std::vector<double>&)>` — подынтегральная функция.

**Выходные данные:**
- Значение типа `double` — приближенное значение интеграла.

**Ограничения:**
- Размерности векторов `a`, `b` и `n` должны быть одинаковыми и больше нуля.
- Для каждого измерения `i` должно выполняться `a[i] < b[i]`.
- Каждый элемент `n[i]` должен быть четным положительным числом.

## 3. Схема распараллеливания

В параллельной реализации с использованием Intel TBB (Threading Building Blocks)
применяется декомпозиция по данным с использованием алгоритма `parallel_reduce`.

Вычисляется общее количество узлов сетки `total_points` 
как произведение всех элементов `dimensions_[i]` (количество точек по каждому измерению).
Каждое измерение имеет размерность `n[i] + 1`, 
поэтому общее количество узлов равно `(n₁+1) × (n₂+1) × ... × (n_d+1)`.

Для распараллеливания используется шаблонная функция `tbb::parallel_reduce`, которая принимает:
- Диапазон итераций, заданный через `tbb::blocked_range<size_t>(0, total_points)`.
- Начальное значение редукции `0.0`.
- Тело редукции (лямбда-функция), которое обрабатывает поддиапазон и возвращает частичную сумму.
- Функцию объединения результатов от разных потоков: `[](double x, double y) { return x + y; }`.

TBB автоматически разбивает диапазон на поддиапазоны,
распределяет их между потоками и выполняет редукцию. В отличие от OpenMP,
где используется статическое планирование,
TBB применяет динамическое планирование с учетом гранулярности задач.

В отличие от последовательной версии, где использовался метод `Nextindex()`
для последовательного перебора, в TBB версии применяется линейная индексация. 
Глобальный индекс `idx` преобразуется в многомерный индекс по формуле: 
для каждого измерения `i` значение индекса вычисляется как `cur_index[i] = idx % dimensions_[i]`,
а затем `idx` делится на `dimensions_[i]`.
Это позволяет каждому потоку независимо определять координаты своей точки без необходимости синхронизации.

## 4. Детали реализации

Реализация находится в файлах `tasks\vlasova_a_simpson_method\tbb\include\ops_tbb.hpp` и 
`tasks\vlasova_a_simpson_method\tbb\src\ops_tbb.cpp` и представляет собой класс 
`VlasovaASimpsonMethodTBB`, унаследованный от базового класса `BaseTask`.

**Ключевые компоненты и их функции:**

1.  **`ValidationImpl()`**:
    - Проверяет согласованность размерностей векторов `a`, `b` и `n`.
    - Убеждается, что нижний предел меньше верхнего по каждому измерению (`a[i] < b[i]`).
    - Проверяет, что количество шагов `n[i]` является четным и положительным.
    - Проверяет, что указатель на функцию `func` корректен.

2.  **`PreProcessingImpl()`**:
    - Вычисляет шаги интегрирования `h_[i] = (b[i] - a[i]) / n[i]`.
    - Определяет размерность сетки `dimensions_[i] = n[i] + 1`.

3.  **`RunImpl()`**:
    - Вычисляет общее количество точек `total_points` как произведение всех `dimensions_[i]`.
    - Использует функцию `tbb::parallel_reduce` с `tbb::blocked_range<size_t>` 
    для параллельного вычисления суммы.
    - В лямбда-функции тела редукции для каждого индекса `idx` в диапазоне
    выполняет преобразование в многомерный индекс через операции деления и взятия остатка.
    - Вызывает `ComputeWeight()` и `ComputePoint()` для каждого узла.
    - Накопление результата происходит через параметр `local_sum` с последующей редукцией.
    - Функция объединения `[](double x, double y) { return x + y; }` суммирует результаты всех потоков.

4.  **`PostProcessingImpl()`**:
    - Проверяет, что результат `result` является конечным числом (`std::isfinite`).

Методы `ComputeWeight()` и `ComputePoint()` полностью идентичны последовательной версии.

**Рекомендации по использованию памяти:**

Алгоритм не требует выделения больших объемов памяти.
Основное потребление связано с хранением входных векторов (`a`, `b`, `n`) и небольшого служебного состояния
(`h_`, `dimensions_`). Значения функции не сохраняются, а сразу добавляются в сумму, что делает этот подход
экономичным с точки зрения RAM.
Каждый поток использует собственные локальные векторы (`cur_index`, `cur_point`), 
что позволяет избежать конфликтов при доступе к памяти.

### 6.2 Производительность

Измерение производительности проводилось на трехмерной задаче интегрирования

Тесты производительности были запущены в двух режимах:

- **task_run** — измерение времени только вычислительного ядра `RunImpl()`.

**Анализ результатов:**

Параллельная реализация на Intel TBB, вопреки ожиданиям, 
показала результаты хуже последовательной версии. 
Время выполнения в режиме `pipeline` увеличилось с 0.4246 с до 0.5356 с (ускорение 0.79x), 
а в режиме `task_run` — с 0.4191 с до 0.5415 с (ускорение 0.77x). 
Фактически, параллельная версия работает медленнее последовательной примерно на 25%.

Это объясняется следующими факторами:
1. **Высокие накладные расходы TBB:** Библиотека TBB создает задачи, 
разбивает диапазоны на поддиапазоны и управляет пулом потоков, 
что требует значительных вычислительных затрат.
2. **Малый объем вычислений на итерацию:** Для каждой итерации выполняется 
относительно небольшой объем работы (преобразование индексов, вычисление веса, вызов функции). 
Накладные расходы на управление задачами TBB превышают выигрыш от параллелизации.
3. **Динамическое планирование:** TBB использует динамическое планирование с кражей задач (work-stealing),
 что добавляет дополнительные накладные расходы.
4. **Преобразование индексов:** Добавляются затраты на преобразование глобального индекса в многомерный.

Для данной задачи TBB оказалась неэффективной из-за высоких накладных расходов
относительно малого объема вычислений на одну итерацию.

## 8. Приложения

```cpp
bool VlasovaASimpsonMethodTBB::RunImpl() {
  size_t dim = task_data_.a.size();

  size_t total_points = 1;
  for (size_t i = 0; i < dim; ++i) {
    total_points *= static_cast<size_t>(dimensions_[i]);
  }

  double sum = tbb::parallel_reduce(tbb::blocked_range<size_t>(0, total_points), 0.0,
                                    [this, dim](const tbb::blocked_range<size_t> &range, double local_sum) {
    std::vector<int> cur_index(dim, 0);
    std::vector<double> cur_point;
    double local_weight = 0.0;

    for (size_t idx = range.begin(); idx != range.end(); ++idx) {
      size_t temp_idx = idx;
      for (size_t i = 0; i < dim; ++i) {
        cur_index[i] = static_cast<int>(temp_idx % static_cast<int>(dimensions_[i]));
        temp_idx /= static_cast<int>(dimensions_[i]);
      }

      ComputeWeight(cur_index, local_weight);
      ComputePoint(cur_index, cur_point);
      local_sum += local_weight * task_data_.func(cur_point);
    }
    return local_sum;
  }, [](double x, double y) { return x + y; });

  double factor = 1.0;
  for (size_t i = 0; i < dim; ++i) {
    factor *= h_[i] / 3.0;
  }

  result_ = sum * factor;
  GetOutput() = result_;

  return true;
}
```
