## 2. Постановка задачи

Требуется вычислить приближенное значение многомерного определенного интеграла от заданной функции на гиперпрямоугольной области.

**Входные данные:**
- Вектор `a` типа `std::vector<double>` — нижние пределы интегрирования по каждому измерению.
- Вектор `b` типа `std::vector<double>` — верхние пределы интегрирования по каждому измерению.
- Вектор `n` типа `std::vector<int>` — количество подинтервалов разбиения по каждому измерению (каждое значение должно быть четным и положительным).
- Объект `func` типа `std::function<double(const std::vector<double>&)>` — подынтегральная функция.

**Выходные данные:**
- Значение типа `double` — приближенное значение интеграла.

**Ограничения:**
- Размерности векторов `a`, `b` и `n` должны быть одинаковыми и больше нуля.
- Для каждого измерения `i` должно выполняться `a[i] < b[i]`.
- Каждый элемент `n[i]` должен быть четным положительным числом.

## 3. Базовый алгоритм (последовательный)

В классе `VlasovaASimpsonMethodSEQ` реализован следующий вычислительный процесс:

На этапе `PreProcessingImpl()` для каждого измерения вычисляется шаг как отношение разности к количеству подинтервалов.
Также формируется массив, который определяет количество узлов сетки по каждому измерению.

В методе `RunImpl()` организуется перебор всех узлов многомерной сетки. Для этого используется вспомогательный метод `Nextindex()`, 
который последовательно генерирует все возможные комбинации индексов, начиная с нулевого вектора и до тех пор, пока не будут перебраны все сочетания.
Условие завершения перебора проверяется путем анализа того, вернулись ли все индексы в нулевое положение.

Для каждого сгенерированного индекса вызывается метод `ComputeWeight()`, который определяет 
весовой коэффициент для данного узла на основе правил метода Симпсона. Одновременно метод `ComputePoint()` преобразует текущий индекс
в физические координаты точки, используя нижние пределы и вычисленные шаги.Полученная точка передается в подынтегральную 
функцию `func()`, результат умножается на соответствующий вес и накапливается в переменной `sum`.

После завершения обхода всех узлов вычисляется итоговый множитель `factor`, представляющий собой 
произведение отношений `h_[i] / 3.0` по всем измерениям. Финальное значение интеграла `result_` получается как произведение накопленной
суммы `sum` на этот множитель. Результат сохраняется через вызов `GetOutput()` и проверяется на конечность в методе `PostProcessingImpl()`.

## 4. Детали реализации

Реализация находится в файлах `tasks\vlasova_a_simpson_method\seq\include\ops_seq.hpp` и 
`tasks\vlasova_a_simpson_method\seq\src\ops_seq.cpp` и представляет собой класс `VlasovaASimpsonMethodSEQ`, унаследованный от базового класса `BaseTask`.

**Ключевые компоненты и их функции:**

1.  **`ValidationImpl()`**:
    -   Проверяет согласованность размерностей векторов `a`, `b` и `n`.
    -   Убеждается, что нижний предел меньше верхнего по каждому измерению (`a[i] < b[i]`).
    -   Проверяет, что количество шагов `n[i]` является четным и положительным.
    -   Проверяет, что указатель на функцию `func` корректен.

2.  **`PreProcessingImpl()`**:
    -   Вычисляет шаги интегрирования `h_[i] = (b[i] - a[i]) / n[i]`.
    -   Определяет размерность сетки `dimensions_[i] = n[i] + 1`.

3.  **`RunImpl()`**:
    -   Это основная вычислительная логика.
    -   Использует вспомогательный метод `Nextindex()`, который реализует перебор по всем узлам сетки. 
    Он последовательно увеличивает многомерный индекс, начиная от `(0,0,...,0)` до `(n1, n2, ..., nd)`.
    -   Для каждого индекса:
        *   `ComputeWeight()`: вычисляет произведение весов Симпсона по всем измерениям.
        *   `ComputePoint()`: вычисляет координаты точки $x_i$ по текущему индексу.
        *   Вычисляется `weight * f(x_point)` и накапливается в переменной `sum`.
    -   После завершения обхода всех узлов, вычисляется итоговый результат: `result = sum * (h1 * h2 * ... * hd) / 3^d`.

4.  **`PostProcessingImpl()`**:
    -   Проверяет, что результат `result` является конечным числом (`std::isfinite`).


**Рекомендации по использованию памяти:**

Алгоритм не требует выделения больших объемов памяти. 
Основное потребление связано с хранением входных векторов (`a`, `b`, `n`) и небольшого служебного состояния (`h_`, `dimensions_`).
Значения функции не сохраняются, а сразу добавляются в сумму, что делает этот подход экономичным с точки зрения RAM.

## 8. Приложения

```
bool VlasovaASimpsonMethodSEQ::RunImpl() {
  size_t dim = task_data_.a.size();

  std::vector<int> cur_index(dim, 0);
  std::vector<double> cur_point;
  double sum = 0.0;
  bool has_more = true;

  while (has_more) {
    double weight = 0.0;

    ComputeWeight(cur_index, weight);
    ComputePoint(cur_index, cur_point);

    sum += weight * task_data_.func(cur_point);
    Nextindex(cur_index);
    has_more = false;
    for (size_t i = 0; i < dim; ++i) {
      if (cur_index[i] != 0) {
        has_more = true;
        break;
      }
    }
  }

  // Множитель: (h1 * h2 * ... * hd) / 3^d
  double factor = 1.0;
  for (size_t i = 0; i < dim; ++i) {
    factor *= h_[i] / 3.0;
  }

  result_ = sum * factor;
  GetOutput() = result_;

  return true;
}
```
