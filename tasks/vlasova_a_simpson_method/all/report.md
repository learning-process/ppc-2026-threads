## 1. Введение

Задача — приближенно вычислить значение кратного интеграла по d-мерному гиперпараллелепипеду, 
используя составную квадратурную схему Симпсона.

## 2. Постановка задачи

Требуется вычислить приближенное значение многомерного определенного 
интеграла от заданной функции на гиперпрямоугольной области.

**Входные данные:**
- Вектор `a` типа `std::vector<double>` — нижние пределы интегрирования по каждому измерению.
- Вектор `b` типа `std::vector<double>` — верхние пределы интегрирования по каждому измерению.
- Вектор `n` типа `std::vector<int>` — количество подинтервалов разбиения по каждому измерению
(каждое значение должно быть четным и положительным).
- Объект `func` типа `std::function<double(const std::vector<double>&)>` — подынтегральная функция.

**Выходные данные:**
- Значение типа `double` — приближенное значение интеграла.

**Ограничения:**
- Размерности векторов `a`, `b` и `n` должны быть одинаковыми и больше нуля.
- Для каждого измерения `i` должно выполняться `a[i] < b[i]`.
- Каждый элемент `n[i]` должен быть четным положительным числом.

## 3. Схема распараллеливания

В гибридной реализации используется двухуровневый параллелизм: 
MPI для межпроцессного взаимодействия и TBB для внутрипроцессного распараллеливания.

**Межпроцессный уровень (MPI):**
- Получаются ранг процесса `rank` и общее количество процессов 
`size` через `MPI_Comm_rank()` и `MPI_Comm_size()`.
- Вычисляется общее количество узлов сетки `total_points` 
как произведение всех элементов `dimensions_[i]`.
- Общий диапазон индексов `[0, total_points)` равномерно распределяется между процессами. 
Для этого вычисляется размер базового блока `chunk = total_points / size` 
и остаток `rem = total_points % size`.
- Процессы с рангом меньше `rem` получают `chunk + 1` точек, остальные — `chunk` точек.
- После завершения локальных вычислений выполняется `MPI_Allreduce()` 
с оператором `MPI_SUM` для суммирования частичных результатов всех процессов.

**Внутрипроцессный уровень (TBB):**
- Каждый MPI-процесс внутри своего диапазона использует `tbb::parallel_reduce` для параллельной обработки.
- Диапазон итераций задается через `tbb::blocked_range<size_t>(begin, end)`.
- TBB автоматически разбивает диапазон на поддиапазоны, распределяет их между потоками и выполняет редукцию.
- Для преобразования глобального индекса в многомерный используется операция деления и взятия остатка.

## 4. Детали реализации

Реализация находится в файлах `tasks\vlasova_a_simpson_method\all\include\ops_all.hpp` 
и `tasks\vlasova_a_simpson_method\all\src\ops_all.cpp` и представляет собой 
класс `VlasovaASimpsonMethodALL`, унаследованный от базового класса `BaseTask`.

**Ключевые компоненты и их функции:**

1.  **`ValidationImpl()`**:
    - Проверяет согласованность размерностей векторов `a`, `b` и `n`.
    - Убеждается, что нижний предел меньше верхнего по каждому измерению (`a[i] < b[i]`).
    - Проверяет, что количество шагов `n[i]` является четным и положительным.
    - Проверяет, что указатель на функцию `func` корректен.

2.  **`PreProcessingImpl()`**:
    - Вычисляет шаги интегрирования `h_[i] = (b[i] - a[i]) / n[i]`.
    - Определяет размерность сетки `dimensions_[i] = n[i] + 1`.

3.  **`RunImpl()`**:
    - Получает ранг процесса (`rank`) и общее число процессов (`size`) через MPI.
    - Вычисляет общее количество точек `total_points` как произведение всех `dimensions_[i]`.
    - Распределяет диапазоны индексов между процессами: 
    каждый процесс получает непрерывный блок `[begin, end)`.
    - Внутри каждого процесса вызывает `tbb::parallel_reduce` 
    для параллельного суммирования по своему диапазону.
    - В лямбда-функции тела редукции для каждого индекса выполняет преобразование
     в многомерный индекс через операции деления и взятия остатка.
    - Вызывает `ComputeWeight()` и `ComputePoint()` для каждого узла.
    - После локальной редукции вызывает `MPI_Allreduce()` для объединения результатов всех процессов.
    - Вычисляет итоговый множитель `factor` как произведение `h_[i] / 3.0`.
    - Устанавливает `result_ = global_sum * factor`.

4.  **`PostProcessingImpl()`**:
    - Проверяет, что результат `result` является конечным числом (`std::isfinite`).

Методы `ComputeWeight()` и `ComputePoint()` полностью идентичны последовательной версии.

**Рекомендации по использованию памяти:**

Алгоритм не требует выделения больших объемов памяти. 
Основное потребление связано с хранением входных векторов (`a`, `b`, `n`) и 
небольшого служебного состояния (`h_`, `dimensions_`). Каждый MPI-процесс обрабатывает
только свою часть диапазона, что позволяет решать задачи большего размера, 
чем позволяет оперативная память одного узла. 
Внутри каждого процесса TBB использует собственные локальные векторы для каждого потока.

bool VlasovaASimpsonMethodALL::RunImpl() {
  int rank = 0;
  int size = 1;
  MPI_Comm_rank(MPI_COMM_WORLD, &rank);
  MPI_Comm_size(MPI_COMM_WORLD, &size);

  size_t dim = task_data_.a.size();

  size_t total_points = 1;
  for (size_t i = 0; i < dim; ++i) {
    total_points *= static_cast<size_t>(dimensions_[i]);
  }

  const auto urank = static_cast<size_t>(rank);
  const auto usize = static_cast<size_t>(size);
  const size_t chunk = total_points / usize;
  const size_t rem = total_points % usize;

  size_t begin = 0;
  size_t end = 0;
  if (urank < rem) {
    begin = urank * (chunk + 1);
    end = begin + (chunk + 1);
  } else {
    begin = (rem * (chunk + 1)) + ((urank - rem) * chunk);
    end = begin + chunk;
  }

  double local_sum = tbb::parallel_reduce(
      tbb::blocked_range<size_t>(begin, end), 0.0,
      [this, dim](const tbb::blocked_range<size_t> &range, double acc) {
        std::vector<int> cur_index(dim, 0);
        std::vector<double> cur_point;
        double local_weight = 0.0;
        for (size_t idx = range.begin(); idx != range.end(); ++idx) {
          size_t temp = idx;
          for (size_t i = 0; i < dim; ++i) {
            cur_index[i] = static_cast<int>(temp % static_cast<size_t>(dimensions_[i]));
            temp /= static_cast<size_t>(dimensions_[i]);
          }
          ComputeWeight(cur_index, local_weight);
          ComputePoint(cur_index, cur_point);
          acc += local_weight * task_data_.func(cur_point);
        }
        return acc;
      },
      [](double x, double y) { return x + y; });

  double global_sum = 0.0;
  MPI_Allreduce(&local_sum, &global_sum, 1, MPI_DOUBLE, MPI_SUM, MPI_COMM_WORLD);

  double factor = 1.0;
  for (size_t i = 0; i < dim; ++i) {
    factor *= h_[i] / 3.0;
  }
  result_ = global_sum * factor;
  GetOutput() = result_;

  return true;
}
