## 1. Контекст

Задача состоит в реализации последовательной версии алгоритма вычисления многомерного интеграла методом трапеций.
Последовательная версия служит эталоном. С её результатом сравнивается корректность параллельных версий (OMP, TBB, STL,
ALL), а с её временем - их ускорение.

## 2. Постановка задачи

Требуется приближённо вычислить определённый интеграл функции нескольких переменных по многомерному прямоугольному
параллелепипеду.

Входные данные - структура `Input`:

- `a` - вектор нижних границ интегрирования по каждому измерению;
- `b` - вектор верхних границ интегрирования по каждому измерению;
- `n` - вектор количества разбиений по каждому измерению;
- `function_choice` - номер подынтегральной функции.

Выходные данные: одно число типа `double` - приближённое значение интеграла.

Размерность интеграла равна длине вектора `a`. Поддерживаются 4 подынтегральные функции:

- 0: сумма координат $x_1 + x_2 + \ldots + x_d$;
- 1: произведение координат $x_1 \cdot x_2 \cdot \ldots \cdot x_d$;
- 2: сумма квадратов $x_1^2 + x_2^2 + \ldots + x_d^2$;
- 3: синус суммы $\sin(x_1 + x_2 + \ldots + x_d)$.

Ограничения на входные данные (проверяются в `ValidationImpl`):

- вектор `a` не пустой, длины векторов `a`, `b`, `n` совпадают;
- $n_i > 0$ для каждого измерения;
- $a_i < b_i$ для каждого измерения;
- `function_choice` лежит в диапазоне от 0 до 3.

## 3. Базовый алгоритм

### 3.1 Метод трапеций

Сначала рассмотрим одномерный случай. Отрезок интегрирования $[a, b]$ делится на $n$ равных частей с шагом
$h = (b - a) / n$. Получается $n + 1$ узловая точка $x_0, x_1, \ldots, x_n$, где $x_i = a + i \cdot h$. В каждой точке
вычисляется значение функции $f(x_i)$.

Площадь под графиком на одном отрезке $[x_i, x_{i+1}]$ приближается площадью трапеции:

$$S_i = \frac{f(x_i) + f(x_{i+1})}{2} \cdot h$$

Интеграл - это сумма площадей всех трапеций:

$$\int_a^b f(x)\,dx \approx \sum_{i=0}^{n-1} \frac{f(x_i) + f(x_{i+1})}{2} \cdot h$$

Если раскрыть эту сумму, то значение функции в каждой внутренней точке входит в неё дважды (один раз как правый край
одной трапеции, второй раз как левый край соседней), а значения на концах отрезка - только один раз. Поэтому формулу
можно переписать так:

$$\int_a^b f(x)\,dx \approx \frac{h}{2}\left(f(x_0) + 2 f(x_1) + \ldots + 2 f(x_{n-1}) + f(x_n)\right)$$

То есть у каждой точки есть весовой коэффициент: 1 для крайних точек и 2 для внутренних. Множитель $h/2$ выносится за
скобку.

### 3.2 Многомерный случай

Для нескольких измерений метод трапеций применяется по каждому измерению по очереди. Вес точки многомерной сетки - это
произведение её весов по каждому измерению. Так как вес по одному измерению равен 1 или 2, итоговый вес точки равен
$2^k$, где $k$ - число измерений, по которым точка является внутренней.

Множитель $h/2$ тоже превращается в произведение: из каждого измерения выносится свой множитель $h_i / 2$. В итоге для
$d$ измерений общий множитель равен $h_1 h_2 \cdots h_d / 2^d$, а итоговая формула выглядит так:

$$I \approx \frac{h_1 h_2 \cdots h_d}{2^d} \sum w \cdot f()$$

где $w$ - вес точки, а сумма берётся по всем точкам сетки.

### 3.3 Перебор точек сетки

Важно понимать, что под сеткой точек подразумевается порядковое расположение точек относительно начала координат. То
есть координаты сетки для конкретной точки - это лишь её порядковый номер в каждом измерении. Зная её расположение в
сетке и зная границы `a` и `b` и шаг `h`, несложно найти точные координаты точки в пространстве и посчитать значение
функции в этой точке.

Сетка многомерная, но перебрать её точки удобно одним циклом. Для этого все точки сетки последовательно нумеруются и по
номеру восстанавливаются координаты точки (порядковый номер точки по каждому измерению).

Это можно пояснить на простом примере - таблице 2 на 3 (2 строки, 3 столбца). Если выписать её клетки в один ряд, строка за строкой,
получится список из 6 клеток с номерами от 0 до 5:

| индекс | 0 | 1 | 2 | 3 | 4 | 5 |
| --- | --- | --- | --- | --- | --- | --- |
| клетка | (0,0) | (0,1) | (0,2) | (1,0) | (1,1) | (1,2) |

Чтобы по индексу узнать клетку, индекс делится на размеры измерений. Номер столбца - это остаток от деления индекса на
число столбцов, номер строки - целая часть от деления. Например, индекс 4: остаток от деления на 3 равен 1 (столбец 1),
целая часть равна 1 (строка 1) - то есть клетка (1, 1).

В коде то же самое делается для любого числа измерений: индекс точки последовательно делится на размер каждого
измерения, остаток даёт координату по этому измерению, а целая часть передаётся дальше.

Основной фрагмент `RunImpl` (`seq/src/ops_seq.cpp`):

```cpp
for (int64_t pnt = 0; pnt < points; pnt++) {
  int64_t rem_index = pnt;
  int weight = 1;

  for (int dim = 0; dim < dimensions; dim++) {
    int dim_coord = static_cast<int>(rem_index % dim_sizes[dim]);
    rem_index /= dim_sizes[dim];

    current_point[dim] = in.a[dim] + (dim_coord * h[dim]);

    if (dim_coord > 0 && dim_coord < in.n[dim]) {
      weight *= 2;
    }
  }

  sum += integral_function(current_point) * weight;
}
```

Внешний цикл проходит по всем точкам сетки. Внутренний цикл по индексу точки восстанавливает её координаты, заодно
вычисляет реальные координаты точки `current_point` и вес: вес умножается на 2 для каждого измерения, по которому точка
внутренняя. Затем значение функции с весом добавляется в общую сумму. После цикла сумма умножается на множитель
$h_1 \cdots h_d / 2^d$.

Критерий корректности: при уменьшении шага (увеличении $n_i$) результат сходится к точному значению интеграла.
Корректность проверяется сравнением с интегралом, посчитанным аналитически.

### 3.4 Сложность

Пусть $d$ - размерность интеграла, а $n$ - общее число точек сетки.
Основной цикл проходит все $n$ точек, и для каждой точки выполняет $O(d)$ операций: восстановление координат, вычисление
точки и веса, вызов подынтегральной функции. Поэтому время работы - $O(n \cdot d)$.

По памяти алгоритм хранит лишь несколько векторов длины $d$ (координаты точки, шаги $h_i$, размеры измерений), то есть
требует $O(d)$ дополнительной памяти и не зависит от числа точек.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`.

Класс `SamoylenkoIIntegralTrapezoidSEQ` наследуется от `BaseTask` и переопределяет четыре метода:

- `ValidationImpl` - проверяет ограничения на входные данные. Если данные некорректны, возвращает `false`.
- `PreProcessingImpl` - обнуляет выходное значение.
- `RunImpl` - считает шаги $h_i$, число точек, перебирает все точки сетки в одном цикле, накапливает взвешенную сумму и
  умножает её на общий множитель.
- `PostProcessingImpl` - ничего не делает, результат уже готов после `RunImpl`.

Подынтегральная функция выбирается функцией `GetIntegrationFunction` по номеру `function_choice` и возвращается как
`std::function`.
