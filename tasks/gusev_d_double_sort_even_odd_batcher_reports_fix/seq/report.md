## 1. Контекст

Последовательная версия является базовой для всей серии реализаций. Она нужна не только
как самостоятельная программа, но и как эталон корректности: параллельные OMP, STL, TBB
и ALL сравниваются с тем же числовым порядком и с тем же способом обработки `double`.

Задача хорошо подходит для отчета по параллельному программированию, потому что в ней
есть две разные части: локальная поразрядная сортировка блоков и последующее слияние
отсортированных последовательностей. В SEQ обе части выполняются без потоков, поэтому
получается понятный baseline без накладных расходов синхронизации.

---

## 2. Постановка задачи

На вход подается `std::vector<double>`. Требуется вернуть вектор той же длины,
отсортированный по неубыванию.

Критерии корректности:

1. Результат совпадает с `std::ranges::sort`.
2. Количество элементов и кратность повторяющихся значений сохраняются.
3. Корректно обрабатываются пустой вектор, один элемент, повторы, отрицательные числа,
   экстремальные значения и знаковые нули.
4. Нарушение порядка вызовов `Validation`, `PreProcessing`, `Run`, `PostProcessing`
   обнаруживается тестовой инфраструктурой.

---

## 3. Базовый алгоритм

Сортировка `double` выполняется как LSD radix sort по восьми байтам 64-битного ключа.
Прямое сравнение исходных битов для `double` некорректно: отрицательные числа в формате
IEEE 754 должны идти перед положительными, а битовый порядок без преобразования этого не
гарантирует. Поэтому значение сначала переводится в сортируемый ключ:

- для неотрицательных чисел инвертируется старший бит знака;
- для отрицательных чисел инвертируются все биты.

После локальной поразрядной сортировки две половины входного массива объединяются
четно-нечетным слиянием Бэтчера. Последняя фаза выполняет соседние compare-exchange,
чтобы восстановить полный порядок после слияния четных и нечетных подпоследовательностей.

## 4. Детали реализации

Основные файлы:

- `tasks/gusev_d_double_sort_even_odd_batcher/seq/include/ops_seq.hpp`
- `tasks/gusev_d_double_sort_even_odd_batcher/seq/src/ops_seq.cpp`

Короткий фрагмент из `seq/src/ops_seq.cpp`, задающий ключ для radix sort:

```cpp
uint64_t DoubleToSortableKey(double value) {
  const auto bits = std::bit_cast<uint64_t>(value);
  const uint64_t sign_mask = 0x8000000000000000ULL;
  return (bits & sign_mask) == 0 ? (bits ^ sign_mask) : (~bits);
}
```

Этот код является инвариантом всей работы: все версии используют ту же идею
преобразования, иначе отрицательные значения и `-0.0` могли бы упорядочиваться неверно.
В `RunImpl()` входной вектор делится пополам, обе половины сортируются функцией
`RadixSortDoubles`, а затем объединяются через `MergeBatcherEvenOdd`.

## 8. Выводы

SEQ-версия корректно задает эталон для остальных реализаций. Основной риск этой
реализации связан не с распараллеливанием, а с числовым порядком `double`; он снят через
преобразование битового представления в сортируемый ключ. На малых входах SEQ ожидаемо
стабилен, потому что не платит за создание потоков, планировщик задач или MPI-обмены.
