## 1. Контекст

В рамках задачи реализовано последовательное умножение разреженных комплексных
матриц, представленных в формате **CCS (Compressed Column Storage)**.
Последовательная версия используется как **эталонная baseline-реализация**,
относительно которой далее сравниваются параллельные backend-ы (`OMP`, `TBB`,
`STL`, `ALL`).

Использование последовательной версии необходимо для:

- проверки корректности результатов параллельных реализаций;
- получения эталонного времени выполнения (`p = 1`);
- вычисления ускорения (`speedup`) и эффективности (`efficiency`) в последующих
  отчётах.

---

## 2. Постановка задачи

### Входные данные

На вход подаются две разреженные комплексные матрицы:

- матрица **A**
- матрица **B**

в формате **CCS (Compressed Column Storage)**.

Каждая матрица представляется структурой `MatrixCCS`, содержащей:

- `rows` — число строк;
- `cols` — число столбцов;
- `col_ptr` — массив указателей на начало столбцов;
- `row_ind` — индексы строк ненулевых элементов;
- `values` — значения ненулевых комплексных элементов типа
  `std::complex<double>`.

### Выходные данные

Результатом является матрица:

\[ C = A \times B \]

также представленная в формате CCS.

### Ограничения

Для корректного умножения необходимо выполнение условия:

\[ A.cols = B.rows \]

Также предполагается корректность CCS-представления:

- согласованные размеры массивов;
- монотонность `col_ptr`;
- корректные индексы строк.

## 3. Базовый алгоритм

Последовательный алгоритм реализует умножение двух разреженных матриц в формате
CCS через обход ненулевых элементов.

Основная идея состоит в том, что вычисление выполняется **по столбцам матрицы
B**.

### Пошаговая схема алгоритма

1. Для каждого столбца `i` матрицы `B` создаётся временный dense-вектор
   `column_c`, соответствующий столбцу результирующей матрицы `C`.

2. Выполняется обход всех ненулевых элементов текущего столбца `B`.

3. Для каждого ненулевого элемента:

\[ B(k, i) \]

находится соответствующий столбец `k` матрицы `A`.

4. Выполняется накопление результата:

\[ C(:, i) += A(:, k) \cdot B(k, i) \]

5. После завершения вычислений временный dense-вектор преобразуется обратно в
   разреженный формат:
    - сохраняются только ненулевые элементы;
    - обновляются `row_ind`, `values` и `col_ptr`.

### Асимптотика

Так как матрицы являются разреженными, сложность определяется не размерностью
матрицы, а количеством ненулевых элементов.

Пусть:

- `nnz(A)` — число ненулевых элементов матрицы `A`;
- `nnz(B)` — число ненулевых элементов матрицы `B`.

Тогда трудоёмкость можно оценить как:

\[ O\left( \sum*{b \in nnz(B)} nnz(A*{column(b)}) \right) \]

где для каждого ненулевого элемента `B` обрабатывается соответствующий столбец
`A`.

В худшем случае (при плотных матрицах) сложность вырождается в:

\[ O(n^3) \]

### Асимптотика по памяти

Дополнительная память определяется временным dense-вектором:

\[ O(rows(A)) \]

который используется для накопления одного столбца результата.

### Инварианты и критерий корректности

Корректным считается результат, если:

- размерность результата совпадает с:

\[ (rows(A), cols(B)) \]

- все ненулевые элементы соответствуют математическому произведению матриц;
- CCS-представление результата остаётся корректным;
- отсутствуют лишние нулевые элементы в `values`.

## 5.Детали реализации (SEQ)

Последовательная реализация выполнена в рамках одного потока исполнения без
использования каких-либо средств параллелизма. Все вычисления производятся
детерминированно, что упрощает отладку и проверку корректности результата.

### 5.1 Структура решения

В основе реализации используется функция обработки одного столбца результата:

- для каждого столбца матрицы **B** формируется временный dense-вектор;
- затем выполняется накопление вкладов от соответствующих столбцов матрицы
  **A**;
- после завершения обработки столбца выполняется преобразование в CCS.

### 5.2 Ключевая функция вычисления

Основная логика сосредоточена в обработке одного столбца:

- обход ненулевых элементов столбца B;
- доступ к соответствующим столбцам A;
- накопление значений в временном буфере;
- фильтрация ненулевых значений.

### 5.3 Особенности реализации

- используется временный вектор размера `rows(A)`;
- отсутствуют конкурентные состояния (data races);
- порядок вычислений строго фиксирован;
- память выделяется один раз на столбец и переиспользуется.

### 5.4 Преимущества реализации

- простота отладки;
- предсказуемость результатов;
- отсутствие накладных расходов на синхронизацию;
- служит эталоном для сравнения с параллельными версиями.

## 6. Сложность реализации

### 6.1 Временная сложность

Пусть:

- `cols(B)` — число столбцов матрицы B;
- `nnz(B)` — число ненулевых элементов B;
- `nnz(A)` — число ненулевых элементов A.

Тогда вычислительная сложность определяется как:

\[ O\left(\sum*{b \in nnz(B)} nnz(A*{column(b)})\right) \]

В худшем случае:

\[ O(n^3) \]

### 6.2 Пространственная сложность

Дополнительная память:

\[ O(rows(A)) \]

Используется временный dense-вектор для накопления одного столбца результата.
