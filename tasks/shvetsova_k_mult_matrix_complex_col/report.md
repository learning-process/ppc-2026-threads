## 1. Введение

Задача умножения разреженных матриц в формате CCS (Compressed Column Storage) с
комплексными элементами является вычислительно затратной, особенно при
увеличении размерности входных данных.

Основные сложности:

- высокая стоимость обхода разреженной структуры,
- нерегулярный доступ к памяти,
- большое число независимых вычислений по столбцам,
- необходимость эффективного управления параллелизмом.

В рамках работы реализованы и исследованы следующие версии алгоритма:

- SEQ — последовательная реализация,
- STL — многопоточная реализация через std::thread,
- OMP — реализация с OpenMP,
- TBB — реализация с Intel oneTBB,
- ALL — гибридная MPI + oneTBB реализация.

---

## 2. Постановка задачи

Даны две разреженные матрицы с элементами комплексного типа:

- $A \in \mathbb{C}^{n \times n}$
- $B \in \mathbb{C}^{n \times n}$

в формате CCS.

Требуется вычислить:

$$
C = A \times B
$$

и представить результат также в формате CCS.

---

## 3. Базовый алгоритм (SEQ)

Алгоритм основан на по-столбцовом умножении:

- для каждого столбца матрицы $B$:
    - перебираются ненулевые элементы,
    - соответствующие столбцы матрицы $A$ суммируются,
    - формируется столбец результата $C$.

### Особенности SEQ

- отсутствует параллелизм,
- минимальные накладные расходы,
- используется как baseline для сравнения.

---

## 4. STL реализация (std::thread)

### Подход STL:

- разбиение столбцов $B$ между потоками,
- фиксированное число потоков = hardware_concurrency(),
- каждый поток обрабатывает свой диапазон столбцов.

### Особенности STL:

- ручное управление потоками,
- отсутствие work-stealing,
- возможный дисбаланс нагрузки при неравномерной структуре матрицы.

---

## 5. OMP реализация (OpenMP)

### Подход OMP:

- директива parallel for по столбцам матрицы $B$,
- автоматическое распределение итераций.

### Особенности OMP:

- минимальный оверхед по сравнению с STL,
- более эффективное распределение нагрузки,
- хорошая масштабируемость на CPU.

---

## 6. TBB реализация (oneTBB)

### Подход TBB:

- tbb::parallel_for с blocked_range,
- динамическое распределение задач.

### Особенности TBB:

- более гибкий баланс нагрузки,
- эффективное использование потоков,
- снижение простоев при неравномерных данных.

### Ключевая идея:

Каждый поток обрабатывает набор столбцов независимо, используя локальные буферы
для предотвращения гонок.

---

## 7. Гибридная реализация (ALL = MPI + TBB)

### Архитектура:

- MPI отвечает за распределение столбцов между процессами,
- oneTBB используется внутри каждого процесса.

### Разбиение:

- каждый MPI-ранг получает часть столбцов матрицы $B$,
- используется схема:
    - равномерное распределение,
    - остаток (remainder) распределяется по первым рангам.

### Коммуникации:

- используется MPI_Gatherv,
- собираются:
    - индексы строк,
    - действительные части,
    - мнимые части,
- итоговая сборка выполняется на rank 0.

---

## 8. Общая схема вычислений

Во всех реализациях используется единый принцип:

1. Итерация по столбцам $B$
2. Для каждого ненулевого элемента:
    - умножение соответствующего столбца $A$
3. Аккумуляция результата в столбец $C$
4. Сохранение результата в CCS формате

## 12. Выводы

1. Все параллельные реализации показывают ускорение относительно SEQ.
2. Наиболее эффективной внутри одного узла является TBB реализация.
3. OpenMP показывает близкую к TBB производительность при меньшем управлении
   потоками.
4. STL версия уступает из-за ручного распределения нагрузки.
5. Гибридная версия ALL эффективна только на больших размерах задачи (≥ 4000),
   где стоимость MPI-коммуникаций становится незначительной относительно
   вычислений.

---

## 13. Итог

Лучшее масштабирование достигается при комбинировании уровней параллелизма:

- внутри узла → oneTBB / OpenMP,
- между узлами → MPI.

Гибридная модель является наиболее универсальной, но требует достаточно больших
входных данных для раскрытия потенциала.
