---

## 1. Контекст

Вычисление произведения разреженных матриц с комплексными элементами требует
значительного объема вычислений и операций обращения к памяти. Для ускорения
обработки больших матриц применяется гибридный подход, сочетающий:

- межпроцессное распараллеливание через MPI,
- внутрипроцессное распараллеливание через oneTBB.

Такой подход позволяет масштабировать вычисления как на уровне узлов, так и на
уровне ядер одного процессора.

---

## 2. Постановка задачи

Даны две разреженные матрицы $A$ и $B$ в формате CCS, содержащие элементы типа
`std::complex<double>`.

Требуется вычислить произведение:

$$
C = A \times B
$$

Данный отчет описывает гибридную реализацию (ALL), где вычисления распределяются
между MPI-процессами, а внутри каждого процесса используются потоки oneTBB.

В качестве baseline для анализа производительности используется последовательная
реализация (SEQ).

---

## 3. Базовый алгоритм

Алгоритм основан на по-столбцовом вычислении результата:

- каждый столбец матрицы $B$ обрабатывается независимо,
- для каждого ненулевого элемента столбца $B$ выполняется умножение
  соответствующего столбца матрицы $A$,
- результаты аккумулируются в столбец результирующей матрицы $C$.

Такой подход хорошо соответствует CCS-формату и минимизирует лишние обращения к
памяти.

---

## 4. Межпроцессная схема (MPI)

В гибридной версии используется статическое разбиение по столбцам матрицы $B$.

### Разделение данных:

- каждый MPI-ранг получает набор столбцов:
    $$
    cols\_per\_rank = \frac{B.cols}{size}
    $$
- остаточные столбцы распределяются между первыми рангами:
    $$
    remainder = B.cols \bmod size
    $$

Таким образом:

- первые `remainder` рангов обрабатывают на 1 столбец больше,
- обеспечивается балансировка нагрузки.

### Коммуникации:

- все ранги выполняют вычисления локально,
- результаты собираются на rank 0 с помощью `MPI_Gatherv`, так как число
  ненулевых элементов (NNZ) различается между процессами,
- отдельно собираются:
    - индексы строк,
    - действительная часть значений,
    - мнимая часть значений.

---

## 5. Внутрипроцессная схема (oneTBB)

Внутри каждого MPI-ранга используется oneTBB для параллельной обработки
столбцов.

### Основной механизм:

- используется `tbb::parallel_for` с `tbb::blocked_range`,
- каждая итерация обрабатывает один или несколько столбцов,
- диапазон автоматически распределяется между потоками.

### Изоляция данных:

Для исключения гонок:

- каждый поток использует локальный буфер `column_c_local`,
- результат сохраняется в независимую структуру `SparseColumn`,
- отсутствует синхронизация внутри вычисления столбца.

### Локальная агрегация:

После завершения параллельной части:

- выполняется flatten всех `SparseColumn`,
- формируются линейные массивы:
    - `local_rows`,
    - `local_vals_real`,
    - `local_vals_imag`.

---

## 6. Глобальная сборка результата

На процессе rank 0 выполняется финальная сборка CCS-матрицы:

- используется `MPI_Gatherv` для переменных размеров блоков,
- дополнительно собирается информация о числе ненулевых элементов по столбцам,
- итоговая структура восстанавливается функцией `AssembleFinalMatrix`.

---

## 7. Детали реализации

Файл: `all/include/ops_all.hpp`, `all/src/ops_all.cpp`

### Ключевой участок TBB + MPI:

```cpp
tbb::parallel_for(
  tbb::blocked_range<int>(0, local_count),
  [&](const tbb::blocked_range<int> &r) {
    std::vector<std::complex<double>> column_c_local(matrix_a.rows, {0.0, 0.0});

    for (int i = r.begin(); i != r.end(); ++i) {
      int global_col = local_start + i;
      ComputeColumnTask(global_col, matrix_a, matrix_b,
                        column_c_local, local_columns[i]);
    }
  }
);
```

### Размер задачи:

---

## 9. Анализ производительности

- На малых размерах задачи ускорение ограничено накладными расходами MPI:
    - коммуникации,
    - сериализация данных,
    - сборка результата.

- Основной прирост достигается за счет:
    - параллельной обработки столбцов между MPI-рангами,
    - дополнительного распараллеливания внутри ранга через oneTBB.

- Гибридная модель наиболее эффективна при:
    - большом числе столбцов,
    - высокой плотности вычислений на столбец,
    - размере матрицы ≥ 4000.

---

## 10. Вывод

Гибридная реализация ALL обеспечивает масштабируемость за счет комбинации:

- MPI (межузловое распределение столбцов),
- oneTBB (внутриузловое распараллеливание).

Экспериментальные результаты показывают, что:

- на малых размерах ускорение ограничено накладными расходами,
- на больших размерах наблюдается устойчивый прирост производительности (~1.3x
  относительно SEQ),
- архитектура решения корректно масштабируется при увеличении размера входных
  данных.
