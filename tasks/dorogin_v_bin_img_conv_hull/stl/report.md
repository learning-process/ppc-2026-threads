## 1. Контекст

Параллелизм через **`std::thread`**: диапазон индексов компонент `[0, n)`
делится на чанки по числу потоков; каждый поток вызывает **`FillConvexHullsChunk`**
для своего поддиапазона.

## 2. Постановка задачи

Постановка **совпадает** с SEQ; отличается только распараллеливание построения оболочек.

## 3. Базовый алгоритм

Как в OMP/TBB: последовательные бинаризация и компоненты, параллельные оболочки.

## 4. Схема распараллеливания

1. `num_threads = max(1, hardware_concurrency())`.
2. `chunk = (n + num_threads - 1) / num_threads`.
3. Для каждого `thread_index` создаётся поток с `[begin, end)`.
4. `join` всех потоков перед возвратом из `RunImpl`.

Вспомогательный метод снижает когнитивную сложность `RunImpl` (требование clang-tidy).

## 5. Детали реализации

- **Файлы:** `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`, класс `DoroginVBinImgConvHullSTL`.
- Статический метод `FillConvexHullsChunk` обрабатывает диапазон компонент.
