## Введение

Алгоритмы обработки изображений характеризуются высокой вычислительной сложностью и независимостью обработки отдельных
пикселей, что делает их идеальными кандидатами для параллелизации. Рассмотренный в данном отчёте алгоритм выполняет
линейное растяжение гистограммы изображения (контрастирование) путем масштабирования значений яркости пикселей из
исходного диапазона в полный диапазон [0, 255].

В работе рассмотрены 5 реализаций: последовательная (SEQ), с использованием OpenMP, std::thread из стандартной библиотеки
C++, Intel TBB и гибридная MPI+STL (ALL).

## Постановка задачи

Входные данные: Вектор целых чисел std::vector, представляющих значения яркости пикселей полутонового изображения
(диапазон 0–255).

Выходные данные: Вектор целых чисел того же размера с преобразованными значениями яркости.

Преобразование (линейная растяжка гистограммы):

        output[i] = ((input[i] - min_val) * 255) / (max_val - min_val)
        
где min_val и max_val — минимальное и максимальное значения яркости во входном изображении.

### Ограничения

1. Размер входного вектора может варьироваться от 16 до 10⁷ элементов.

2. В случае max_val == min_val (однородное изображение) выходное изображение равно входному.

## Описание последовательного алгоритма

Алгоритм выполняет линейное растяжение гистограммы изображения за два прохода:

1. Поиск границ диапазона – находит минимальное и максимальное значения яркости среди всех пикселей с помощью std::ranges::minmax_element.

2. Проверка частного случая – если все пиксели одинаковые (min == max), возвращает копию входного изображения
(изменение контраста невозможно).

3. Преобразование пикселей – каждый пиксель пересчитывается по формуле:

        новое_значение = ((старое_значение - минимум) * 255) / (максимум - минимум)

Эта операция масштабирует исходный диапазон [min, max] в полный диапазон [0, 255].

## Описание алгоритмов параллелизации

### OpenMP-версия

Алгоритм выполняет параллельное линейное растяжение гистограммы изображения с использованием OpenMP:

1. Параллельный поиск min/max – с помощью директивы #pragma omp parallel for reduction(min:min_val) reduction(max:max_val)
 все потоки одновременно находят минимальное и максимальное значения яркости, автоматически объединяя результаты через редукцию.

2. Проверка частного случая – если min == max (однородное изображение), потоки параллельно копируют входные данные
 в выходной массив.

3. Параллельное преобразование – каждый поток обрабатывает свою часть пикселей по формуле
 ((input[i] - min) * 255) / (max - min).
 Директива default(none) явно указывает область видимости переменных.

### TBB-версия

Алгоритм выполняет параллельное линейное растяжение гистограммы с использованием библиотеки
Intel TBB (Threading Building Blocks):

1. Параллельный поиск min/max – через tbb::parallel_reduce данные разбиваются на блоки (blocked_range),
каждый поток находит локальные min/max в своем блоке, затем результаты объединяются кастомной функцией редукции,
возвращающей структуру MinMax с обоими значениями.

2. Проверка частного случая – если min == max (однородное изображение), tbb::parallel_for выполняет параллельное
копирование входных данных в выходной массив.

3. Параллельное преобразование – tbb::parallel_for автоматически распределяет блоки пикселей между потоками, каждый
пиксель преобразуется по формуле ((input[i] - min) * 255) / (max - min). TBB использует динамическую балансировку
нагрузки (воровство задач) для оптимального распределения работы.

### STL-версия

Алгоритм выполняет параллельное линейное растяжение гистограммы с использованием нативных потоков C++ (std::thread):

1. Разбиение данных – функция GetThreadRange вычисляет для каждого потока границы обрабатываемого блока:
начало b = tid * chunk и конец e (последний поток получает остаток).

2. Параллельный поиск min/max – создаются потоки, каждый находит локальные минимум и максимум в своем блоке,
результаты сохраняются в векторы local_min и local_max. После завершения всех потоков глобальные min/max определяются
через std::ranges::min_element/max_element.

3. Проверка частного случая – если global_min == global_max (однородное изображение), выполняется параллельное
копирование через std::ranges::copy.

4. Параллельное преобразование – создается новая группа потоков, каждый преобразует пиксели своего блока по формуле
((input[i] - min) * 255) / (max - min). Все потоки синхронизируются через join().

### ALL-версия

Алгоритм выполняет двухуровневое параллельное линейное растяжение гистограммы, комбинируя MPI для межпроцессного
взаимодействия и STL threads для внутрипроцессного параллелизма.

1. Распределение данных (MPI) – процесс 0 рассылает всем процессам размер данных через MPI_Bcast. Каждый процесс
вычисляет свой блок данных (равномерное распределение с учётом остатка) и получает свою порцию через MPI_Scatterv.

2. Локальный поиск min/max (STL threads) – каждый MPI процесс запускает пул потоков (std::thread), которые параллельно
 находят минимум и максимум в своём подблоке. Результаты объединяются через std::ranges::min_element/max_element.

3. Глобальная редукция (MPI) – через MPI_Allreduce все процессы обмениваются локальными min/max и вычисляют глобальные
 значения (операции MPI_MIN и MPI_MAX).

4. Локальное преобразование (STL threads) – каждый процесс параллельно преобразует свой блок данных по формуле
 ((input[i] - min) * 255) / (max - min). Для однородного изображения выполняется параллельное копирование.

5. Сбор результатов (MPI) – все процессы отправляют обработанные блоки процессу 0 через MPI_Gatherv,
который собирает их в итоговый выходной массив.
