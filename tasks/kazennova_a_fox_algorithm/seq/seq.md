## 1. Контекст

Последовательная реализация умножения двух квадратных матриц C = A·B
размера n×n (вещественные числа). Эта версия служит эталоном корректности
и базовым значением времени для расчёта ускорения всех
параллельных реализаций (OMP, TBB, STL, ALL). Для размера n = 700
последовательное умножение производится очень долго,
что обосновывает необходимость распараллеливания.

## 2. Постановка задачи

Входные данные: структура InType, содержащая матрицы A и B размера n×n
(`std::vector<double>`). Выходные данные: структура OutType
с матрицей C = A·B того же размера. Критерий корректности: поэлементное
совпадение с эталонным результатом.
Ограничения:
матрицы квадратные и согласованных размеров, n > 0, входные данные не пусты.

## 3. Базовый алгоритм

Классический алгоритм умножения матриц «строка на столбец»:

1. Для каждой строки i матрицы A (0 … m-1):
2. Для каждого столбца j матрицы B (0 … n-1):
3.     Вычислить скалярное произведение i-й строки A и j-го столбца B:
       C_ij = sum_{k=0}^{k-1} A_ik · B_kj.
4. Записать результат в C_ij.

время O(n^3), память O(n^2). Инвариант: результат не зависит
от порядка вычислений.

## 4. Детали реализации

Файлы:

- kazennova_a_fox_algorithm/seq/include/ops_seq.hpp
- kazennova_a_fox_algorithm/seq/src/ops_seq.cpp

Класс KazennovaATestTaskSEQ, наследник BaseTask. Pipeline:

- ValidationImpl – проверяет, что матрицы не пусты, размеры > 0,
  A.cols == B.rows.
- PreProcessingImpl – инициализирует выходную матрицу C нулями.
- RunImpl – тройной вложенный цикл: по i, j, k. Внутренний цикл
  вычисляет sum += A[i*k + k_idx] * B[k_idx*n + j].
- PostProcessingImpl – возвращает true, если выходная матрица не пуста.

Самый дорогой фрагмент – внутренний цикл по k (n^3 операций). Именно он
будет распараллелен в последующих версиях.

## 5. Проверка корректности

Корректность проверена следующими способами:

- Функциональные тесты (набор из tests/functional/main.cpp)
  для размеров от 1 до 128, включая граничные случаи (n=1, n=prime).
- Сравнение с эталонным умножением (наивный алгоритм) на малых
  размерах – полное совпадение.
- Проверка на случайных матрицах размера 700 – результат совпадает
  с вычислениями в Python (NumPy) с точностью 10^-12.

Все 12 тестовых сценариев успешно пройдены. Невязка (максимальное
поэлементное отклонение).
