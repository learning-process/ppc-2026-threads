## 1. Контекст

Параллельная реализация умножения плотных квадратных матриц на основе
алгоритма Фокса с использованием Intel oneTBB. В отличие от ручного
управления потоками (STL), TBB предлагает
задачно-ориентированный подход с рекурсивным разбиением диапазонов
и механизмом перехвата работы, что позволяет эффективно
балансировать нагрузку на неоднородных вычислительных системах.

## 2. Постановка задачи

Решение задачи C = A·B, где A, B — квадратные матрицы размера n×n
(вещественные числа). Реализация с блоком 64 и std::min не требует деления без остатка.
Критерий корректности: совпадение результата с последовательной версией (SEQ)
в пределах машинной точности.
SEQ-версия служит baseline для производительности и верификации.

## 3. Базовый алгоритм

Алгоритм Фокса (блочное умножение с циклической передачей блоков):

1. Разбить A и B на блоки размера bs×bs.
   Число блоков: q_i = ceil(n/bs), q_j, q_k.
2. Для каждого блока результата C_{i,j}:
   - Для каждого k = 0 … q_k-1:
     - Взять блок A_{i,k} и блок B_{k,j}.
     - Выполнить умножение блоков с накоплением в C_{i,j}.
3. Результат накапливается в C в процессе умножения блоков.

время O(n^3), память O(n^2).

## 4. Схема распараллеливания

Примитив TBB: `tbb::parallel_for` с двумерным диапазоном
`tbb::blocked_range2d<int>(0, blocks_i, 0, blocks_j)`. Это позволяет
распараллелить обработку блоков результата по осям i и j одновременно,
улучшая локальность данных и балансировку.

Разбиение диапазона: двумерное блокирование. TBB автоматически
разбивает диапазоны [0, blocks_i) и [0, blocks_j) на поддиапазоны,
которые распределяются между потоками. Размер зерна (grainsize) не задан
явно - используется значение по умолчанию.

Механизм work-stealing: если поток завершил свои поддиапазоны,
он может «украсть» невыполненные поддиапазоны у других потоков.
Это обеспечивает автоматическую балансировку нагрузки даже при
неравномерной сложности блоков (например, из-за неполных последних
блоков по краям матрицы).

Контроль параллелизма: число рабочих потоков управляется через
`tbb::global_control::max_allowed_parallelism`, который инфраструктура
курса устанавливает в соответствии с PPC_NUM_THREADS.

Ключевой фрагмент:

```cpp
tbb::parallel_for(tbb::blocked_range2d<int>(0, blocks_i, 0, blocks_j),
  [&](const tbb::blocked_range2d<int> &r) {
    std::vector<double> block_a(bs*bs), block_b(bs*bs);
    for (int bi = r.rows().begin(); bi != r.rows().end(); ++bi) {
      for (int bj = r.cols().begin(); bj != r.cols().end(); ++bj) {
        for (int bk = 0; bk < blocks_k; ++bk) {
          GetBlock(a, m, k, bi, bk, bs, block_a.data());
          GetBlock(b, k, n, bk, bj, bs, block_b.data());
          MultiplyBlock(block_a, block_b, bs, max_i, max_j, max_k, bi, bj, n, c);
        }
      }
    }
  });
```

## 5. Детали реализации

Файлы:

- kazennova_a_fox_algorithm/tbb/include/ops_tbb.hpp
- kazennova_a_fox_algorithm/tbb/src/ops_tbb.cpp

Основные компоненты:

- kBlockSize = 64 – фиксированный размер блока.
- GetBlock() – извлекает блок из матрицы с учётом границ (последний блок
  может быть неполным).
- MultiplyBlock() – умножает два блока и добавляет результат в матрицу C.
  Внутри – тройной вложенный цикл по i, j, kk.
- RunImpl() – вызывает tbb::parallel_for с двумерным диапазоном
  blocked_range2d. Каждый поток получает поднабор блоков (i,j)
  и обрабатывает их полностью (все k), используя локальные буферы
  block_a и block_b.

Изменения относительно SEQ:

- Вместо тройного вложенного цикла по строкам и столбцам – блочная структура.
- Параллелизм на уровне независимых блоков (i,j) через parallel_for.
- Отсутствие явной синхронизации – TBB управляет распределением работы.
- Потоки не создаются заново, а берутся из глобального пула TBB.

## 6. Проверка корректности

Корректность проверена следующими способами:

- Функциональные тесты для размеров от 1 до 128, включая граничные случаи.
- Сравнение с SEQ на случайных матрицах размера 700 при 2 и 4 потоках
- Проверка на размерах, не кратных 64 (например, n = 700) – корректная
  обработка неполных блоков через std::min в GetBlock и MultiplyBlock.
- Прогон с ThreadSanitizer – гонок данных не обнаружено (запись в C
  ведётся по непересекающимся индексам).

Все тесты пройдены (20 функциональных тестов).

## 9. Выводы

TBB-реализация алгоритма Фокса корректно выполняет умножение матриц
и демонстрирует положительную динамику ускорения. Двумерное разбиение
`blocked_range2d` удобно и обеспечивает хорошую локальность данных.
Однако на размере задачи n=700 число блоков невелико.
Механизм work-stealing не успевает проявить себя в полной мере
из-за малого количества блоков и относительно крупных вычислительных
задач на блок.Рекомендуется использовать TBB при
больших размерах матриц (n ≥ 1400).
