## 1. Контекст
Алгоритм Джарвиса строит выпуклую оболочку множества точек, последовательно находя вершины с минимальным полярным углом относительно текущей. Он начинается с самой нижней левой точки и повторяет выбор следующей вершины через предикат поворота, пока не вернётся к началу. Последовательная версия служит эталоном корректности и базовой производительности для всех параллельных реализаций.

## 2. Постановка задачи
- **Входные данные**: `std::vector<Point>` – набор точек на плоскости.  
  `Point` – структура с полями `double x, y`.
- **Выходные данные**: `std::vector<Point>` – вершины выпуклой оболочки в порядке обхода против часовой стрелки, начиная с самой левой нижней точки.  
- **Ограничения**: входной вектор не должен быть пустым; допускаются точки с любыми вещественными координатами.  
- **Крайние случаи**:
  - Одна точка → оболочка состоит из этой же точки.
  - Две точки → оболочка содержит обе точки (отрезок).
  - Три и более точек, лежащих на одной прямой → оболочка состоит из двух крайних точек.
  - Наличие дубликатов → дубликаты удаляются на этапе предобработки.

## 3. Базовый алгоритм
1. Удаление дубликатов через `std::set<Point>`.
2. Если после удаления точек ≤ 2, оболочка = все точки.
3. Найти самую левую нижнюю точку `p0` (минимальный x, при равном x – минимальный y).
4. Текущая вершина `current = p0`. Добавить в результат.
5. Повторять:
   - Выбрать следующую вершину `next` как любую другую точку (например, первую, не равную `current`).
   - Для каждой точки `p` из всех точек:
     - Вычислить ориентацию `CheckLeft(current, next, p)`.
     - Если `orient > 0` → `p` левее → обновить `next = p`.
     - Если `orient == 0` (коллинеарны) → выбрать более дальнюю точку (по квадрату расстояния).
   - Если `next == p0` → оболочка замкнута, выход.
   - Иначе добавить `next` в результат, `current = next`.
6. Вернуть результат.

**Асимптотика**:
- Время: O(n·h), где n – число точек после удаления дубликатов, h – размер оболочки. В худшем случае (точки на окружности) h ≈ n, т.е. O(n²).
- Память: O(n) для хранения точек и результата.

## 4. Детали реализации

```cpp
Point OrehovNJarvisPassSEQ::FindNext(Point current) const {
  Point next = current == input_[0] ? input_[1] : input_[0];
  for (auto p : input_) {
    if (current == p || next == p) {
      continue;
    }
    double orient = CheckLeft(current, next, p);
    if (orient > 0) {
      next = p;
    }
    if (orient == 0) {
      if (Distance(current, next) < Distance(current, p)) {
        next = p;
      }
    }
  }
  return next;
}

double OrehovNJarvisPassSEQ::CheckLeft(Point a, Point b, Point c) {
  return ((b.x - a.x) * (c.y - a.y)) - ((b.y - a.y) * (c.x - a.x));
}

Point OrehovNJarvisPassSEQ::FindFirstElem() const {
  Point current = input_[0];
  for (auto f : input_) {
    if (f.x < current.x || (f.y < current.y && f.x == current.x)) {
      current = f;
    }
  }
  return current;
}

double OrehovNJarvisPassSEQ::Distance(Point a, Point b) {
  return std::sqrt(pow(a.y - b.y, 2) + pow(a.x - b.x, 2));
}
```

- `ValidationImpl()` – проверяет, что входной вектор не пуст (на нулевом процессе в MPI-версии, но в SEQ – просто `!GetInput().empty()`).
- `PreProcessingImpl()` – удаляет дубликаты: создаёт `std::set<Point>` из входных точек, затем копирует в `input_`. Это гарантирует уникальность точек для алгоритма.
- `RunImpl()` – реализует описанный выше алгоритм. Результат сохраняется в `res_`.
- `PostProcessingImpl()` – копирует `res_` в выходной вектор `GetOutput()`.

## 5. Проверка корректности

- **Сравнение результатов**: выходной вектор `res_` сравнивается с эталонным вектором, заданным в функциональных тестах. Порядок точек важен: оболочка должна быть перечислена, начиная с самой левой нижней точки, против часовой стрелки.

- **Функциональные тесты**:
  - Тест 1: 7 точек, образующих выпуклый многоугольник. Ожидается 5 вершин: (0,0), (0,3), (2,4), (4,2), (3,0).
  - Тест 2: одна точка (0,0) → оболочка {(0,0)}.
  - Тест 3: две точки (0,0) и (1,0) → оболочка {(0,0), (1,0)}.
