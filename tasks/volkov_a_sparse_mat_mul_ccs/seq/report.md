## 1. Контекст

В данной работе решается задача умножения двух разреженных матриц, представленных в формате CCS (Compressed Column
Storage). Последовательная реализация необходима для того, чтобы задать корректный эталон работы алгоритма. Именно с
этой версией будут сравниваться все последующие параллельные реализации (OMP, TBB, std::thread, MPI) для оценки
ускорения и проверки правильности вычислений.

## 2. Постановка задачи

На вход подаются две матрицы `A` и `B`, хранящиеся в формате CCS. Формат подразумевает наличие трех массивов: значения
(values), индексы строк (row_indices) и указатели на начало столбцов (col_ptrs).
Выходными данными является матрица `C`, также в формате CCS, которая представляет собой результат матричного умножения
`A * B`.
Главное ограничение и условие корректности: количество столбцов матрицы `A` должно быть равно количеству строк матрицы
`B`. Крайние случаи включают пустые матрицы (все нули) и матрицы с очень высокой степенью разреженности.

## 3. Базовый алгоритм

Алгоритм вычисляет результирующую матрицу по столбцам.

1. Для каждого столбца `j` матрицы `B` мы находим его ненулевые элементы.
2. Каждый ненулевой элемент столбца `j` имеет индекс строки `b_row` и значение `b_val`. По правилам умножения матриц это
означает, что нам нужно взять столбец матрицы `A` с индексом `b_row`, умножить все его элементы на скаляр `b_val` и
прибавить к текущему вычисляемому столбцу результата.
3. Для накопления суммы используется плотный вектор `col_accumulator` размером, равным количеству строк матрицы `A`.
4. После обработки всех необходимых столбцов из `A`, плотный вектор сканируется. Значения, превышающие порог `1e-10`,
записываются в массивы результирующей матрицы `C`, а сам аккумулятор очищается нулями для следующего шага.

Асимптотика по времени в худшем случае (плотные матрицы) составляет O(N^3). Для разреженных матриц время пропорционально
`cols_B * rows_A + K`, где K - общее число операций скалярного умножения ненулевых элементов.
Асимптотика по памяти составляет O(rows_A) для хранения локального аккумулятора столбца, плюс память под итоговую
матрицу `C`.
Инварианты: отсутствие явных нулей в массиве значений результата (отсечение по `1e-10`).

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

Входная валидация в `ValidationImpl` строго проверяет совпадение размерностей (`matrix_a.cols_count ==
matrix_b.rows_count`). Основная логика сосредоточена в методе `RunImpl`.

Ниже приведен ключевой фрагмент кода, отвечающий за накопление значений одного столбца:

```cpp
// File: seq/src/ops_seq.cpp
for (int k = b_start; k < b_end; ++k) {
  int b_row = matrix_b.row_indices[k];
  double b_val = matrix_b.values[k];

  int a_start = matrix_a.col_ptrs[b_row];
  int a_end = matrix_a.col_ptrs[b_row + 1];

  for (int idx = a_start; idx < a_end; ++idx) {
    int a_row = matrix_a.row_indices[idx];
    double a_val = matrix_a.values[idx];

    col_accumulator[a_row] += a_val * b_val;
  }
}
```

Этот фрагмент показывает, как алгоритм избегает умножения нулей. Мы итерируемся только по тем элементам, которые реально
существуют в структуре CCS. Самой дорогой операцией здесь является косвенная адресация при записи в `col_accumulator`.

## 5. Проверка корректности

Корректность проверяется путем сравнения полученной матрицы `C` с ожидаемым результатом. В функциональных тестах
используются:

- Умножение на единичную матрицу (результат должен совпадать с исходной матрицей).
- Умножение нулевых матриц (на выходе должна быть пустая структура CCS).
- Умножение случайно сгенерированных матриц с заранее известной плотностью.
Все тесты детерминированы, результаты сверяются с учетом погрешности вычислений с плавающей точкой.

Самым ресурсоемким фрагментом последовательного кода является цикл очистки и упаковки аккумулятора `for (int i = 0; i <
matrix_a.rows_count; ++i)`, так как он выполняется полностью для каждого столбца матрицы `B`, независимо от степени
разреженности результата.

## 8. Выводы

Реализованная последовательная версия корректно обрабатывает формат CCS и не выполняет лишних арифметических операций
над нулями. Данный алгоритм выбран как эталон, поскольку он обладает понятной структурой циклов, которую в дальнейшем
будет удобно декомпозировать по столбцам матрицы `B` для распределения работы между потоками. Полученные метрики
производительности станут надежной базой (знаменателем) для оценки качества распараллеливания и расчета ускорения в
последующих отчетах.
