- Технологии: SEQ

## 1. Назначение

BorunovVComplexCcsSEQ - последовательный baseline: один поток. База для speedup в perf-таблицах (\(T_{\mathrm{seq,task\_run}}\)).

## 2. Распараллеливание

Параллелизма нет. Декомпозиция не применяется — обход всех столбцов j = 0 … B.num_cols - 1 в одном потоке.

Алгоритм RunImpl (seq/src/ops_seq.cpp):

1. Для столбца j: накопление в col_accumulator[i] через столбцы A[:,p] и B[p,j].
2. Учёт ненулевых: is_non_zero[i], список non_zero_indices.
3. std::ranges::sort(non_zero_indices).
4. Запись в C при |z| > 1e-9, сброс аккумулятора.
5. C.col_ptrs[j+1] = |values|.

## 3. Синхронизация

Не требуется. Запись сразу в глобальные C.values, C.row_indices, C.col_ptrs.

## 4. Отличия от других реализаций

| Аспект           | SEQ               | OMP / TBB             | STL                    |
|------------------|-------------------|-----------------------|------------------------|
| Потоки           | 1                 | PPC_NUM_THREADS       | hardware_concurrency() |
| Маркер ненулевых | vector<bool>      | marker[i] != j        | vector<bool>           |
| Сортировка       | std::ranges::sort | CustomShellSort       | std::ranges::sort      |
| Слияние          | не нужно          | merge буферов потоков | merge после join()     |

SEQ может быть медленнее OMP/TBB даже при малом workers из-за других структур в горячем цикле, а не из-за скрытого параллелизма.

## 5. Конфигурация workers

workers = 1 (не используется PPC_NUM_THREADS в RunImpl).

## 7. Pipeline задачи

| Этап | Действие |
|--------------------|-------------------------------------------------|
| ValidationImpl     | A.num_cols == B.num_rows, размеры col_ptrs      |
| PreProcessingImpl  | размеры C, обнуление col_ptrs, очистка массивов |
| RunImpl            | умножение                                |
| PostProcessingImpl | без работы                                      |

## 8. Наблюдения

- pipeline быстрее task_run у SEQ из-за учёта только RunImpl в task_run и полного конвейера в pipeline при сопоставимой нагрузке на один поток.
- Для сравнения OMP/TBB/STL/ALL baseline всегда SEQ task_run.

## 9. Вывод

SEQ - однопоточный столбцово-ориентированный SpGEMM для CCS. Корректный baseline без параллельных конструкций в RunImpl.
