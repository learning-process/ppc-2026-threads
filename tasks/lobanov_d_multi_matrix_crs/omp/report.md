## 1. Контекст

OpenMP (Open Multi-Processing) — стандарт параллельного программирования для систем с разделяемой памятью. Задача умножения разреженных матриц в формате CRS идеально подходит для этой модели, поскольку обработка каждой строки результирующей матрицы не зависит от других строк. Внедрение OpenMP позволяет получить ускорение при минимальных изменениях исходного последовательного кода — достаточно добавить несколько директив компилятора. Данная реализация демонстрирует эффективность подхода data parallelism на вычислительно интенсивной задаче.

## 2. Постановка задачи

**Входные данные:** две разреженные матрицы `A` и `B` в CRS-формате (Compressed Row Storage). Каждая описывается:
- `row_count`, `column_count` – размерности;
- `row_pointer_data` – массив смещений начала строк;
- `column_index_data` – индексы столбцов ненулевых элементов;
- `value_data` – значения элементов (тип `double`).

**Выходные данные:** матрица `C = A × B` в том же CRS-формате.

**Ограничения:** `A.column_count == B.row_count`, все размерности положительны. Элементы, по модулю меньшие `1e-12`, отбрасываются.

**Базовая линия (baseline):** последовательная реализация SEQ с временем выполнения **389.7 мс**.

## 3. Базовый алгоритм

Используется тот же алгоритм, что и в SEQ:

1. **Транспонирование** матрицы `B` → получение `B^T` в CRS-формате.
2. **Вычисление скалярных произведений** – для каждой строки `i` матрицы `A` и каждой строки `j` матрицы `B^T` методом двух указателей.
3. **Формирование выходной CRS** – накопление ненулевых элементов в результирующие массивы.

Отличие от SEQ заключается в распараллеливании внешнего цикла (по строкам `A`) между потоками.

## 4. Схема распараллеливания

Распараллеливанию подвергается внешний цикл по строкам матрицы `A`. Используется следующая директива:

```cpp
for (int i = 0; i < rows; ++i) {
    std::vector<double> local_vals;
    std::vector<int> local_cols;
    
    // Вычисление строки i
    for (int j = 0; j < colsB; ++j) {
        double sum = DotProduct(A, i, Bt, j);
        if (std::abs(sum) > 1e-12) {
            local_vals.push_back(sum);
            local_cols.push_back(j);
        }
    }
    
    // Безопасное перемещение в глобальный буфер
    temp_values[i] = std::move(local_vals);
    temp_cols[i] = std::move(local_cols);
}
```

Когда OpenMP дал выигрыш:  

Задача с независимыми итерациями (строки результата не зависят друг от друга).

Чем выигрыш ограничен:

Память – при 12 потоках пропускная способность памяти не успевает за вычислительными возможностями.  

Размер задачи – на матрицах меньше 1000×1000 накладные расходы OpenMP могут превысить выгоду от распараллеливания.
