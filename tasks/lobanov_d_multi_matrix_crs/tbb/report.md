## 1. Контекст

Библиотека oneTBB (Threading Building Blocks) реализует задачно-ориентированную модель параллелизма (task-based parallelism), в отличие от потоково-ориентированной модели OpenMP. В TBB программист оперирует не потоками, а задачами, а планировщик библиотеки самостоятельно решает, сколько потоков создать, как распределить задачи и как балансировать нагрузку с помощью механизма **кражи задач (work-stealing)**.

Для задачи умножения разреженных матриц TBB подходит идеально по нескольким причинам:
- Количество ненулевых элементов в разных строках матрицы может сильно отличаться, создавая неравномерную вычислительную нагрузку.
- TBB автоматически адаптируется к этой неравномерности, динамически перераспределяя задачи между потоками.
- Разработчику не нужно вручную указывать `schedule(dynamic)` или рассчитывать размеры чанков.

Данная реализация демонстрирует, как с помощью TBB можно получить масштабируемое решение с минимальными усилиями по сравнению с ручным управлением потоками.

## 2. Постановка задачи

**Входные данные:** две разреженные матрицы `A` и `B` в CRS-формате (Compressed Row Storage). Каждая описывается:
- `row_count`, `column_count` – размерности;
- `row_pointer_data` – массив смещений начала строк;
- `column_index_data` – индексы столбцов ненулевых элементов;
- `value_data` – значения элементов (тип `double`).

**Выходные данные:** матрица `C = A × B` в том же CRS-формате.

**Ограничения:** `A.column_count == B.row_count`, все размерности положительны. Элементы, по модулю меньшие `1e-12`, отбрасываются.

**Базовая линия (baseline):** последовательная реализация SEQ с временем выполнения **389.7 мс**.

## 3. Базовый алгоритм

Используется тот же алгоритм, что и в SEQ:

1. **Транспонирование** матрицы `B` → получение `B^T` в CRS-формате.
2. **Вычисление скалярных произведений** – для каждой строки `i` матрицы `A` и каждой строки `j` матрицы `B^T` методом двух указателей.
3. **Формирование выходной CRS** – накопление ненулевых элементов в результирующие массивы.

## 4. Схема распараллеливания

### 4.1 Выбор примитива

Для распараллеливания внешнего цикла (по строкам матрицы `A`) выбран примитив `tbb::parallel_for`. Это основной строительный блок TBB для параллельного выполнения итераций независимого цикла.

```cpp
#include <tbb/tbb.h>
#include <tbb/global_control.h>

int num_threads = ppc::util::GetNumThreads();
tbb::global_control ctrl(tbb::global_control::max_allowed_parallelism, num_threads);

tbb::parallel_for(0, rows, [&](int i) {
    // Вычисление строки i результирующей матрицы
    std::vector<double> local_vals;
    std::vector<int> local_cols;
    
    for (int j = 0; j < colsB; ++j) {
        double sum = DotProduct(A, i, Bt, j);
        if (std::abs(sum) > 1e-12) {
            local_vals.push_back(sum);
            local_cols.push_back(j);
        }
    }
    
    temp_values[i] = std::move(local_vals);
    temp_cols[i] = std::move(local_cols);
});
```
