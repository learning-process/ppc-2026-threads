## 1. Введение

Поразрядная сортировка является эффективным и популярным линейным алгоритмом сортировки.
Данная лабораторная работа направлена на реализацию последовательной и
нескольких параллельных версий этого алгоритма.

## 2. Постановка задачи

Необходимо реализовать алгоритм поразрядной сортировки для целых чисел со знаком.
На входе: размер массива `n` или сам массив для генерации и тестирования.
Ограничения:

- Обязательно использование принципов поразрядного распределения.
- Корректная обработка массивов как из положительных, так и из отрицательных чисел.
На выходе: отсортированный по возрастанию массив входных данных.
Требуется разработать базовую последовательную реализацию, а также параллельные реализации
с использованием библиотек: MPI, OpenMP, Intel Threading Building Blocks и стандартной
библиотеки потоков std::thread. Еще необходимо провести замеры и анализ эффективности.

## 3. Описание последовательного алгоритма

Базовый алгоритм реализует метод поразрядной сортировки в десятичной системе счисления
с использованием устойчивой сортировки подсчетом.
Так как массив может содержать отрицательные числа, то для них принцип сортировки
по разрядам имеет краевой случай смены порядка. Элементы первоначально разделяются
на два списка: положительные и отрицательные (сохраняются по модулю).
К каждому из этих подмассивов применяется сортировка:

1. Для каждого разряда выполняется сортировка подсчетом.
2. Вычисляется гистограмма частот встречаемости цифры в рассматриваемом разряде для каждого числа.
3. Вычисляются префиксные суммы для определения точных смещений размещения.
4. Значения перераспределяются во временный выходной массив.
После сортировки, отрицательные элементы инвертируются и записываются обратно
в порядке убывания в начало исходного массива, за которыми следуют
отсортированные положительные элементы.

Сложность базового алгоритма: $O(d \cdot n)$, где $d$ — количество разрядов, а $n$ — размер входного массива.

## 4. Схема распараллеливания

Каждая из технологий программирования предоставляет свой механизм распараллеливания, который интегрируется в логику алгоритма.

### OpenMP (OMP)

- **Декомпозиция**: Осуществляется параллельное выполнение сортировки подсчетом.
  Директивой `#pragma omp for schedule(static)` входной массив делится на равные
  смежные блоки данных для каждого потока пула.
- **Синхронизация и планирование**: Каждый поток заполняет локальную гистограмму частот.
  После неявной барьерной синхронизации локальные массивы сливаются в глобальный для
  расчета глобальных смещений. Рассчитав глобальные стартовые индексы, каждый поток вычисляет
  свои позиционные смещения и записывает свои значения в общий результирующий буфер,
  без состояния race condition.

### Intel Threading Building Blocks (TBB)

- **Декомпозиция**: В основе используются компоненты `tbb::parallel_reduce` и блокирующие
  диапазоны `tbb::blocked_range`, что дает возможность динамически делить обработку массива
  с применением алгоритма Work-stealing на подзадачи.
- **Обмены**: В рамках `parallel_reduce` потоки агрегируют локальные массивы счетчиков,
  а метод объединения сливает гистограммы двух задач. Это решение исключает гонки данных
  и обеспечивает высокую степень масштабируемости.

### std::thread (STL)

- **Декомпозиция**: Пул потоков создается статично в коде `std::thread(worker)`.
  Общий массив делится на блоки `n / thread_count`.
- **Синхронизация и планирование**: Рабочие функции выполняют операции, а основной поток
  выступает оркестратором. Логика разбивки на локальные гистограммы и построения префиксных
  сумм идентична схеме OMP.

### Message Passing Interface (MPI)

- **Топология/обмены**: Рут-процесс делит исходный массив на примерно равные блоки и
  производит рассылку данных по всем узлам кластера, используя механизм `MPI_Scatterv`.
- **Локальная сортировка**: Каждый вычислительный процесс локально производит поразрядную
  сортировку своего блока данных, используя последовательную логику.
- **Схема слияния**: Отсортированные блоки каждой ноды отправляются рут-процессу.
  Для финального объединения рут-процесс выполняет многопутевое слияние всех
  упорядоченных блоков в итоговый отсортированный массив.

## 5. Детали реализации

- **Структура классов**: Структура классов построена на базе интерфейса `ppc::core::Task`.
  Созданы классы-наследники `KamaletdinovRBitwiseIntSEQ`, `KamaletdinovRBitwiseIntOMP` и т.д.
- **Управление памятью**: При выполнении `CountingSortByDigit` требуется выделение $O(N)$
  памяти на вспомогательный массив результатов. В многопоточном варианте добавляется
  требование памяти для хранения локальных счетчиков — `O(NumThreads * 10 * sizeof(int))`.

## 8. Заключение

В ходе выполнения работы были разработаны последовательная и параллельные версии алгоритма
поразрядной сортировки. Многопоточные технологии эффективно справляются с задачей за счет
независимого вычисления локальных гистограмм фрагментов массива, что уменьшает время
обработки разряда. Опытным путем подтверждено, что инструменты с общей памятью ускоряют
алгоритм почти в 2.4 раза, при этом масштабируемость достигает предела в районе 6-9
потоков из-за накладных расходов. При этом MPI применяет модель распределенной сортировки
данных с последующим слиянием. По результатам проведенных тестов, MPI показала большие
накладные расходы на создание процессов и межпроцессные коммуникации. Для тестируемого
объема данных это привело к сильному замедлению алгоритма. Таким образом, для сортировки
целых чисел локально наилучшим решением является архитектура с общей памятью, не
требующая дорогостоящей пересылки данных.

## Приложение

```cpp
// разделение чисел по знаку
std::vector<int> neg;
std::vector<int> pos;

for (int x : arr) {
  if (x < 0) {
    neg.push_back(-x);
  } else {
    pos.push_back(x);
  }
}

RadixSortPositive(neg);
RadixSortPositive(pos);

std::size_t idx = 0;
for (int i = static_cast<int>(neg.size()) - 1; i >= 0; i--) {
  arr[idx++] = -neg[i];
}
for (int p : pos) {
  arr[idx++] = p;
}
```

// расчет глобальных префиксных сумм и смещений для потоков в OMP
std::array<int, 10> global_start = {};
for (int digit_idx = 1; digit_idx < 10; digit_idx++) {
  global_start.at(digit_idx) = global_count.at(digit_idx - 1);
}

std::vector<std::array<int, 10>> thread_offsets(thread_count);
for (int digit_idx = 0; digit_idx < 10; digit_idx++) {
  int offset = global_start.at(digit_idx);
  for (int thread_idx = 0; thread_idx < thread_count; thread_idx++) {
    thread_offsets.at(thread_idx).at(digit_idx) = offset;
    offset += local_counts.at(thread_idx).at(digit_idx);
  }
}
