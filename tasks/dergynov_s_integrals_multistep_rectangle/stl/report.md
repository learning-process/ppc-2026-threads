## 1. Введение

STL-версия использует `std::thread` для ручного управления потоками
без внешнего runtime.

## 2. Постановка задачи

- **Вход**:
  - подынтегральная функция `std::function<double(const std::vector<double>&)>`
  - границы интегрирования `std::vector<std::pair<double, double>>`
  - количество шагов разбиения `int`
- **Условие корректности входа**:
  - функция задана корректно
  - количество шагов > 0
  - границы не пусты и для каждого измерения `left < right`
- **Ограничения**: размерность произвольная, но для тестов используется 1-3
- **Выход**: приближённое значение интеграла `double`

## 3. Базовый алгоритм (последовательный)

Алгоритм совпадает с SEQ (метод средних прямоугольников).

## 4. Схема распараллеливания

- **Разбиение данных**: общее количество точек сетки `total_points = n^dim`
  равномерно распределяется между потоками с учётом остатка
- **Локальные результаты**: каждый поток накапливает сумму в своём элементе массива
- **Синхронизация**: все потоки запускаются одновременно,
  затем вызывается `join()` для ожидания завершения
- **Обработка ошибок**: используется `std::atomic<bool>` для флага ошибки

Для реального параллелизма соблюдён порядок:

1. Сначала запускаются все потоки
2. Затем вызывается `join()` для всех запущенных потоков

## 7. Результаты

### 7.1 Корректность

STL-версия проходит те же функциональные тесты, что и baseline SEQ.
Результаты совпадают с точностью до 1e-6.

### 7.2 Производительность

Определения метрик:

- `workers` — число потоков `std::thread`
- `time` — wall-clock время, секунды
- `speedup = T_seq / T_mode`
- `efficiency = speedup / workers * 100%`

Результаты:

| mode | workers | time (task_run), s | time (pipeline), s | speedup | efficiency, % |
|------|--------:|-------------------:|-------------------:|--------:|--------------:|
| seq  |       1 |            0.02584 |            0.02783 |    1.00 |           N/A |
| stl  |       2 |            0.02671 |            0.01551 |    0.97 |         48.5% |
| stl  |       4 |            0.01530 |            0.01472 |    1.69 |         42.2% |

Комментарий: STL-версия показывает лучшее ускорение среди всех технологий
при 4 потоках — 1.69x. Это объясняется эффективным ручным управлением
распределением работы между потоками.
