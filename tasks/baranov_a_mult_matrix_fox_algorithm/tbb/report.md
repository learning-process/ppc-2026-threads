## 1. Контекст

Данная реализация представляет собой параллельную версию алгоритма Фокса с использованием библиотеки Intel oneTBB. TBB предоставляет высокоуровневые абстракции для параллельного программирования, такие как `parallel_for`, которые автоматически управляют пулом потоков и распределением работы. Это позволяет сосредоточиться на логике алгоритма, а не на низкоуровневом управлении потоками.

## 2. Постановка задачи

Постановка задачи полностью идентична последовательной версии (см. `seq/report.md`). Отличие заключается в способе достижения результата: вычисления выполняются параллельно с использованием TBB-примитивов.

## 3. Базовый алгоритм

Базовый алгоритм полностью совпадает с последовательной версией (блочный алгоритм Фокса). Основное изменение – замена последовательных циклов на `tbb::parallel_for`.

## 4. Схема распараллеливания

**Стратегия:** Декомпозиция по данным. TBB автоматически разбивает итерационное пространство на задачи и распределяет их между потоками из встроенного пула.

**Два подхода к распараллеливанию:**

1. **StandardMultiplication (наивное умножение):**
   - Внешний цикл по строкам `i` распараллеливается с помощью `tbb::parallel_for`.
   - Каждый поток обрабатывает одну или несколько строк независимо.
   - TBB автоматически балансирует нагрузку между потоками.

```cpp
void BaranovAMultMatrixFoxAlgorithmTBB::StandardMultiplication(size_t n) {
  const auto &[matrix_size, matrix_a, matrix_b] = GetInput();
  auto &output = GetOutput();

  tbb::parallel_for(static_cast<size_t>(0), n, [&](size_t i) {
    for (size_t j = 0; j < n; ++j) {
      double sum = 0.0;
      for (size_t k = 0; k < n; ++k) {
        sum += matrix_a[(i * n) + k] * matrix_b[(k * n) + j];
      }
      output[(i * n) + j] = sum;
    }
  });
}
```

2. **FoxBlockMultiplication (блочный алгоритм):**
   - Внешний цикл по стадиям `bk` выполняется последовательно (итерации зависимы).
   - Внутренний цикл по блокам (линеаризованный в `linear_idx`) распараллеливается с помощью `tbb::parallel_for`.
   - Блоки обрабатываются независимо, так как на каждой стадии `bk` они не пересекаются по данным.

```cpp
void BaranovAMultMatrixFoxAlgorithmTBB::FoxBlockMultiplication(size_t n, size_t block_size) {
  const auto &[matrix_size, matrix_a, matrix_b] = GetInput();
  auto &output = GetOutput();

  size_t num_blocks = (n + block_size - 1) / block_size;

  //Инициализация(параллельная)
  tbb::parallel_for(static_cast<size_t>(0), n * n, [&](size_t idx) { output[idx] = 0.0; });

  //Основной цикл по стадиям
  for (size_t bk = 0; bk < num_blocks; ++bk) {
    tbb::parallel_for(static_cast<size_t>(0), num_blocks * num_blocks, [&](size_t linear_idx) {
      size_t bi = linear_idx / num_blocks;
      size_t bj = linear_idx % num_blocks;

      size_t broadcast_block = (bi + bk) % num_blocks;

      size_t i_start = bi * block_size;
      size_t i_end = std::min(i_start + block_size, n);
      size_t j_start = bj * block_size;
      size_t j_end = std::min(j_start + block_size, n);
      size_t k_start = broadcast_block * block_size;
      size_t k_end = std::min(k_start + block_size, n);

      //Умножение блоков(вложенные циклы)
      for (size_t i = i_start; i < i_end; ++i) {
        for (size_t j = j_start; j < j_end; ++j) {
          double sum = 0.0;
          for (size_t k = k_start; k < k_end; ++k) {
            sum += matrix_a[(i * n) + k] * matrix_b[(k * n) + j];
          }
          output[(i * n) + j] += sum;
        }
      }
    });
  }
}
```

**Ключевые особенности TBB:**

- `tbb::parallel_for(start, end, lambda)` автоматически разбивает диапазон `[start, end)` на поддиапазоны (grainsize = 1 по умолчанию).
- TBB использует пул потоков, который создаётся один раз и переиспользуется, что исключает overhead на создание/уничтожение потоков.
- Работа распределяется между потоками с помощью алгоритма work-stealing: idle-потоки "воруют" задачи у занятых.
- Отсутствует явная синхронизация: каждый блок обрабатывается одним потоком, а разные блоки работают с непересекающимися участками результирующей матрицы на каждой стадии `bk`.

**Синхронизация:**

В данной реализации **отсутствует** явная синхронизация (мьютексы, атомарные операции). Это возможно благодаря тому, что:

- В `StandardMultiplication` каждый поток записывает результат в разные строки матрицы.
- В `FoxBlockMultiplication` на каждой стадии `bk` каждый поток обрабатывает свой уникальный блок, и разные блоки не пересекаются.

**Управление потоками:** TBB автоматически управляет пулом потоков. Количество потоков можно контролировать через `tbb::global_control` или переменные окружения TBB.

**Разбиение диапазона:** В реализации используется упрощённая форма `tbb::parallel_for(start, end, lambda)`, где TBB автоматически разбивает диапазон на поддиапазоны. Эквивалентно использованию `tbb::blocked_range<size_t>(start, end)` с grainsize = 1 (значение по умолчанию).

**Partitioner:** Используется partitioner по умолчанию (`tbb::auto_partitioner`), который адаптивно выбирает стратегию разбиения в зависимости от нагрузки.

**Grain size:** Размер зёрна не задавался явно; используется значение по умолчанию (1 итерация на задачу). Для данной задачи это приемлемо, так как каждая итерация достаточно трудоёмкая (умножение n элементов).

## 5. Детали реализации

**Файлы:** `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`
**Ключевые особенности реализации:**

1. **Использование tbb::parallel_for:**
   - В `StandardMultiplication`: распараллеливание по строкам `i`.
   - В `FoxBlockMultiplication`: распараллеливание по линеаризованным индексам блоков (`linear_idx`).
   - В инициализации: параллельное обнуление результирующей матрицы.

2. **Отсутствие ручного разбиения на чанки:**
   - В отличие от STL-версии, здесь не нужно вычислять `chunk_size` и распределять работу вручную.
   - TBB автоматически определяет оптимальное разбиение.

3. **Лямбда-захват:**

   ```cpp
   tbb::parallel_for(static_cast<size_t>(0), n, [&](size_t i) { ... });
   ```

4. **Линеаризация индексов:**

  ```cpp
   size_t bi = linear_idx / num_blocks;
   size_t bj = linear_idx % num_blocks;
   ```

- Двумерный индекс блока преобразуется в одномерный для упрощения итерации.

5. **Отсутствие отдельной функции ProcessBlock:**

- В отличие от SEQ и STL, умножение блоков встроено прямо в лямбду для уменьшения overhead'а вызовов.

## 9. Выводы

- TBB-версия демонстрирует отличную производительность (ускорение ~7.7x на 12 потоках), сопоставимую с OpenMP и превосходящую STL.
- Ключевое преимущество TBB перед STL: использование пула потоков, который создаётся один раз, что исключает overhead на создание/уничтожение потоков на каждой стадии алгоритма.
- `tbb::parallel_for` обеспечивает автоматическое разбиение работы и балансировку нагрузки через механизм work-stealing, что упрощает код по сравнению с ручным управлением потоками в STL.
- Код на TBB получился более лаконичным и читаемым по сравнению со STL-версией (отсутствие ручных расчётов chunk_size, эмуляции потоков, join-ов).
- TBB, как и OpenMP, позволяет легко масштабировать производительность без изменения кода при добавлении новых ядер.
- Основной недостаток TBB: необходимость подключения внешней библиотеки (в отличие от STL, которая есть в стандартной библиотеке C++).
- Для данной задачи TBB и OpenMP показали примерно одинаковую производительность. Выбор между ними может определяться требованиями к портируемости и зависимостям проекта.
- TBB-версия является хорошим выбором для C++-проектов, где требуется высокопроизводительный параллелизм без привязки к конкретному компилятору (в отличие от OpenMP).
