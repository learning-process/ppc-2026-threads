## 1. Контекст

Данная реализация представляет собой последовательный эталон для задачи умножения двух квадратных матриц. Алгоритм Фокса использует блочное разбиение матриц для улучшения локальности данных (кэш-дружественности). Эта версия служит базовой линией (baseline) для оценки ускорения, достигаемого в параллельных реализациях (OpenMP, TBB, STD, MPI).

## 2. Постановка задачи

- **Входные данные:**
  - `n` (size_t) — размерность квадратных матриц.
  - `matrix_a` (vector<double>) — первая матрица размера n x n, хранимая в row-major порядке.
  - `matrix_b` (vector<double>) — вторая матрица размера n x n.
- **Выходные данные:**
  - `matrix_c` (vector<double>) — результирующая матрица размера n x n, где `C = A * B`.
- **Ограничения:**
  - `n > 0`.
  - Размеры входных векторов должны точно равняться `n * n`.
- **Крайние случаи:**
  - Матрица 1x1 (тривиальное умножение скаляров).
  - Размер матрицы, не кратный размеру блока (обрабатывается с помощью `std::min`).

## 3. Базовый алгоритм

Реализован блочный вариант алгоритма Фокса, адаптированный для последовательного выполнения:

1. **Разбиение:** Матрицы разбиваются на блоки (`block_size = 64`).
2. **Инициализация:** Результирующая матрица заполняется нулями.
3. **Вычисления:**
    - Тройной вложенный цикл по блокам `bi`, `bj`, `bk`.
    - Для каждой итерации `bk` определяется "широковещательный" блок A (`broadcast_block = (bi + bk) % num_blocks`).
    - Вызывается `ProcessBlock`, которая выполняет умножение блока A на блок B и накапливает результат в блоке C.
4. **Асимптотика:**
    - **Время:** O(n^3). Блочная структура не меняет общее количество операций (2*n^3 умножений и сложений), но улучшает использование кэша.
    - **Память:** O(n^2) для хранения трёх матриц.

**Инварианты:** Для любого `i, j`, элемент `C[i][j]` является суммой произведений `A[i][k] * B[k][j]` для всех `k` от `0` до `n-1`.

## 4. Детали реализации

- **Файлы:**
  - `seq/include/ops_seq.hpp`
  - `seq/src/ops_seq.cpp`
- **Класс:** `BaranovAMultMatrixFoxAlgorithmSEQ` наследник `BaseTask`.
- **Методы:**
  - `ValidationImpl`: Проверяет, что размер матриц корректен и они не пусты.
  - `PreProcessingImpl`: Выделяет память под результирующую матрицу и заполняет её нулями.
  - `RunImpl`: Основная логика. Выбирает между `StandardMultiplication` (для маленьких матриц `n < 64`) и `FoxBlockMultiplication` (для больших).
  - `PostProcessingImpl`: В данной реализации не выполняет действий, всегда возвращает `true`.
- **Особенности:**
  - Использование `ProcessBlock` для инкапсуляции вычислений на уровне блоков.
  - `StandardMultiplication` реализует классический алгоритм O(n^3) для небольших размеров, чтобы избежать overhead от блочной логики.

## 5. Проверка корректности

- **Метод:** Результат последовательной реализации сравнивается с эталонным умножением, реализованным наивным способом (или проверенным математическим пакетом) на случайных матрицах.
- **Функциональные тесты:** Покрывают матрицы размеров 1, 2, 3, 5, 8, 16, 32, 50, 64, 100, 127, 256.
- **Примеры:**
    1. `A = [[1, 0], [0, 1]]`, `B = [[5, 6], [7, 8]]`. Ожидаемый результат: `C = [[5, 6], [7, 8]]`.
    2. `A = [[2, 0], [0, 2]]`, `B = [[1, 1], [1, 1]]`. Ожидаемый результат: `C = [[2, 2], [2, 2]]`.
    3. `A = ones(3,3)`, `B = ones(3,3)`. Ожидаемый результат: `C` — матрица 3x3, заполненная `3.0`.
