**Student:** Баранов А.

**Variant:** Умножение квадратных матриц с использованием блочного алгоритма Фокса

## 1. Контекст

Данная реализация представляет собой гибридную параллельную версию алгоритма Фокса, использующую два уровня параллелизма:

- **Межпроцессный уровень (MPI):** Распределение блоков матрицы между процессами.
- **Внутрипроцессный уровень (потоки):** Распараллеливание вычислений внутри каждого процесса с использованием OpenMP, TBB или STL.

Гибридный подход позволяет эффективно использовать как распределённую память (кластеры, узлы), так и общую память внутри узла (многоядерные процессоры).

## 2. Постановка задачи

Постановка задачи полностью идентична последовательной версии (см. `seq/report.md`). Отличие заключается в распределении вычислений между процессами MPI и потоками внутри каждого процесса.

## 3. Базовый алгоритм

Базовый алгоритм совпадает с последовательной версией (блочный алгоритм Фокса). Основное изменение – добавление двух уровней параллелизма:

1. MPI распределяет блоки между процессами.
2. Внутри каждого процесса потоки параллельно обрабатывают назначенные блоки.

## 4. Схема распараллеливания

**Два уровня параллелизма:**

1. **Межпроцессный уровень (MPI):**
   - Матрицы A, B, C распределяются по процессам блоками.
   - Каждый процесс отвечает за вычисление своей части результирующей матрицы.
   - Процессы обмениваются блоками матрицы A на каждой стадии алгоритма.
   - Используются коллективные операции MPI (Broadcast, Barrier).

2. **Внутрипроцессный уровень (потоки):**
   - Внутри каждого процесса задачи по блокам распараллеливаются на потоки.
   - Используемая технология: OpenMP/TBB/STL (выбирается при компиляции).

**Распределение работы между процессами:**

- Размерность матрицы `n` делится на `P` процессов.
- Каждый процесс получает приблизительно `n/P` строк результирующей матрицы.
- Блоки матрицы A широковещательно рассылаются всем процессам на каждой стадии `bk`.
- Процессы работают независимо, обмениваясь данными только через MPI.

**Алгоритм (псевдокод):**

```cpp
int rank, size;
MPI_Comm_rank(MPI_COMM_WORLD, &rank);
MPI_Comm_size(MPI_COMM_WORLD, &size);

// Разбиение строк между процессами
size_t rows_per_proc = n / size;
size_t start_row = rank * rows_per_proc;
size_t end_row = (rank == size - 1) ? n : start_row + rows_per_proc;

for (size_t bk = 0; bk < num_blocks; ++bk) {
    // Широковещательная рассылка текущего блока A
    MPI_Bcast(block_A_data, block_size * block_size, MPI_DOUBLE, root, MPI_COMM_WORLD);
    
    // Параллельное умножение блоков (потоки внутри процесса)
    #pragma omp parallel for
    for (size_t block = 0; block < local_blocks; ++block) {
        // Вычисление блока C
    }
    
    MPI_Barrier(MPI_COMM_WORLD);
}
```

**Синхронизация:**

- **MPI_Barrier** используется для синхронизации процессов в конце каждой стадии `bk`.
- **MPI_Bcast** обеспечивает рассылку блоков матрицы A от корневого процесса всем остальным.
- Внутрипроцессная синхронизация отсутствует, так как потоки работают с непересекающимися данными.

**Управление процессами и потоками:**

- **MPI:** Количество процессов задаётся через `mpiexec -np <N>`.
- **Потоки:** Количество потоков внутри процесса задаётся через:
  - OpenMP: `OMP_NUM_THREADS`
  - TBB: `TBB_NUM_THREADS`
  - STL: `std::thread::hardware_concurrency()`

**Масштабирование:** Общее количество работников = `процессы × потоки_на_процесс`.

## 5. Детали реализации

**Файлы:** `all/include/ops_all.hpp`, `all/src/ops_all.cpp`

**Ключевые особенности реализации:**

1. **Инициализация MPI:**

```cpp
MPI_Init(&argc, &argv);
MPI_Comm_rank(MPI_COMM_WORLD, &rank);
MPI_Comm_size(MPI_COMM_WORLD, &size);
```

2. **Распределение строк по процессам:**

```cpp
size_t rows_per_proc = n / size;
size_t start_row = rank * rows_per_proc;
size_t end_row = (rank == size - 1) ? n : start_row + rows_per_proc;
```

3. **Широковещательная рассылка:**

```cpp
MPI_Bcast(block_a.data(), block_size * block_size, MPI_DOUBLE, broadcast_root, MPI_COMM_WORLD);
```

4. **Внутрипроцессное распараллеливание:**
Используется одна из технологий (OpenMP/TBB/STL) для обработки локальных блоков.

5. **Сбор результатов:**
Каждый процесс хранит свою часть результирующей матрицы; сборка происходит через MPI_Gather (при необходимости).

## 6. Проверка корректности

**Метод:** Результат выполнения гибридной ALL-версии сравнивается с результатом SEQ-версии на идентичных входных данных.

**Функциональные тесты:** Используется тот же набор тестов, что и для `seq`, но запускаемый с разным количеством процессов (1, 2, 4) и потоков (1, 2, 4, 6).

**Результат:** Для всех протестированных конфигураций и размеров матриц результаты совпадают с эталонными с точностью до `1e-9`. MPI-коммуникации и многопоточные вычисления не вносят ошибок.

**Особенности проверки:**

- При `size=1` (один процесс) гибридная версия эквивалентна чисто потоковой версии.
- При `size>1` необходимо убедиться, что распределение строк между процессами корректно покрывает всю матрицу без пропусков и дублирования.

## 9. Выводы

- Гибридная ALL-версия (MPI + потоки) работоспособна и демонстрирует корректные результаты.
- На одном узле (многоядерный процессор) чисто потоковые версии (OpenMP/TBB) показывают лучшую производительность, чем гибридная с MPI.
- Добавление MPI вносит дополнительный overhead на коммуникации, который не окупается на одном узле.
- **Потенциальные сценарии использования гибридной версии:**
  - Запуск на кластере из нескольких узлов (распределённая память).
  - Сверхбольшие матрицы, которые не помещаются в память одного узла.
  - Задачи, где пропускная способность сети (InfiniBand) выше, чем overhead на коммуникации.
- **На одном узле** рекомендуется использовать чистые потоковые версии (OpenMP или TBB).
- **Масштабируемость:** При увеличении числа процессов ускорение растёт медленнее из-за коммуникационных затрат.
- **Рекомендация:** Гибридный подход целесообразен только при запуске на кластере с числом узлов > 1.

**Итоговое сравнение всех технологий (n=2048, максимальное ускорение):**

| Технология | Ускорение | Сложность реализации | Рекомендуемый сценарий |
|------------|-----------|---------------------|-----------------------|
| SEQ | 1.00x | Низкая | Базовая линия, отладка |
| OpenMP | 7.59x | Низкая | Многоядерные CPU |
| TBB | 7.67x | Средняя | Кроссплатформенные C++ проекты |

| ALL (MPI+OMP) | ~7.45x | Очень высокая | Кластеры, распределённая память |
