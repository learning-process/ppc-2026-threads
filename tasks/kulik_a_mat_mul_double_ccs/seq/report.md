## 1. Введение

Разреженные матрицы встречаются в физическом моделировании, машинном обучении
и алгоритмах на графах. Их особенность: подавляющее число элементов равно нулю,
поэтому хранить их полностью нецелесообразно.

Формат CCS (Compressed Column Storage) хранит только ненулевые элементы,
упорядоченные по столбцам.

Данная работа описывает последовательную реализацию умножения разреженных матриц
в формате CCS с элементами типа `double`. Реализация служит базовым алгоритмом
для замера производительности параллельных версий (OpenMP, TBB, STL Threads,
MPI+OpenMP).

- - -

## 2. Постановка задачи

Требуется вычислить произведение **C = A × B**, где:

- **A** — разреженная матрица размером `n × k` в формате CCS;
- **B** — разреженная матрица размером `k × m` в формате CCS;
- **C** — результирующая разреженная матрица размером `n × m` в формате CCS.

Требования к реализации:

- Входные данные передаются как `std::tuple<CCS, CCS>`, результат записывается
  в `OutType` (`CCS`).
- Строки внутри каждого столбца результата упорядочены по возрастанию.
- Валидация входных данных: `a.m == b.n` (число столбцов A равно числу строк B).

## 3. Алгоритм

### 3.1. Структура данных CCS

    struct CCS {
      size_t n, m, nz;         // строки, столбцы, число ненулевых элементов
      vector<size_t> col_ind;  // col_ind[j]..col_ind[j+1]-1 — диапазон j-го столбца
      vector<size_t> row;      // номера строк ненулевых элементов
      vector<double> value;    // значения ненулевых элементов
    };

Элементы j-го столбца расположены в `row` и `value` на позициях
`col_ind[j] … col_ind[j+1]−1`.

### 3.2. Реализация

Для каждого столбца `j` матрицы B:

1. Перебираются ненулевые элементы `B[ind][j] = b_val` из j-го столбца B.
2. Для каждого такого элемента выполняется проход по `ind`-му столбцу A:
   `accum[i] += A[i][ind] * b_val` для каждого ненулевого `A[i][ind]`.
3. Строка `i` регистрируется в `nnz_rows` при первом посещении (флаг `nz_elem_rows[i]`).
4. После прохода `nnz_rows` сортируется; ненулевые `accum[i]` копируются в `c.row` и `c.value`.
5. Аккумулятор и флаги сбрасываются точечно — только для затронутых строк.

Псевдокод:

    для j = 0..b.m-1:
        c.col_ind[j] = |c.value|
        для k в [b.col_ind[j], b.col_ind[j+1]):
            ind = b.row[k];  b_val = b.value[k]
            для zc в [a.col_ind[ind], a.col_ind[ind+1]):
                i = a.row[zc]
                accum[i] += a.value[zc] * b_val
                если !nz_elem_rows[i]: nnz_rows.push(i); nz_elem_rows[i] = true
        sort(nnz_rows)
        для i в nnz_rows:
            если accum[i] != 0.0: c.row.push(i); c.value.push(accum[i])
            accum[i] = 0.0;  nz_elem_rows[i] = false
        nnz_rows.clear()
    c.nz = |c.value|;  c.col_ind[b.m] = c.nz

### 3.3. Сложность

Сильно зависит от степени разреженности входных матриц.

| Характеристика | Оценка |
| --- | --- |
| Вычислительная | O(nnz(B) × avg\_col\_nnz(A)) |
| Память | O(n + nnz(C)) |

## 4. Детали реализации

### 4.1. Архитектура

Файлы:

- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

Ход выполнения программы:

#### ValidationImpl

Проверка условия `a.m == b.n`.

#### PreProcessingImpl

Все входные данные уже готовы.

#### RunImpl

Вычислительная работа алгоритма.

#### PostProcessingImpl

Результат уже в готовом виде; корректность проверяется в тестовых файлах.

### 4.2. Вспомогательные структуры

Три вектора выделяются **один раз** перед главным циклом и переиспользуются
для каждого столбца:

| Вектор | Размер | Назначение |
| --- | --- | --- |
| `accum` | `a.n` | Вещественный аккумулятор, инициализирован `0.0` |
| `nz_elem_rows` | `a.n` | Маска «строка уже посещена», инициализирована `false` |
| `nnz_rows` | динамический | Список затронутых строк текущего столбца |

Точечный сброс аккумулятора (только затронутых позиций) исключает `O(n)` `memset`
на каждой итерации внешнего цикла.

### 4.3. Запись результата

Массивы `c.value` и `c.row` наращиваются через `push_back` по мере обработки
столбцов. Смещение начала j-го столбца фиксируется до обработки:
`c.col_ind[j] = c.value.size()`. После цикла закрывающая запись
`c.col_ind[b.m] = c.nz` завершает формирование CCS-матрицы.

### 4.4. Обработка нулевых элементов

Условие `if (accum[i] != 0.0)` при записи в `c.value` гарантирует, что элементы,
обнулившиеся вследствие суммирования с плавающей точкой, не попадают в результат.
Это обеспечивает минимально возможное заполнение матрицы C.

## 5. Тесты

Корректность работы алгоритма проверялась функциональными тестами,
а производительность — перфоманс-тестами.

- **Функциональные тесты:** матрица 6×6 с 12 ненулевыми элементами умножается
  сама на себя. Результат вычислен вручную и переведён в формат CCS — это
  эталонное значение матрицы C.
- **Перфоманс-тесты:** создаются две ленточные матрицы A и B. Для проверки
  корректности используется метод умножения справа на произвольный вектор:
  `(A × B) × x = A × (B × x)`. Если результирующие векторы совпадают с учётом
  погрешности машинной арифметики, матрицы равны.

## Приложение

Основная часть алгоритма:

    for (size_t j = 0; j < b.m; ++j) {
      c.col_ind[j] = c.value.size();
      for (size_t k = b.col_ind[j]; k < b.col_ind[j + 1]; ++k) {
        size_t ind = b.row[k];
        double b_val = b.value[k];
        for (size_t zc = a.col_ind[ind]; zc < a.col_ind[ind + 1]; ++zc) {
          size_t i = a.row[zc];
          double a_val = a.value[zc];
          accum[i] += a_val * b_val;
          if (!nz_elem_rows[i]) {
            nz_elem_rows[i] = true;
            nnz_rows.push_back(i);
          }
        }
      }
      std::ranges::sort(nnz_rows);
      for (size_t i : nnz_rows) {
        if (accum[i] != 0.0) {
          c.row.push_back(i);
          c.value.push_back(accum[i]);
        }
        accum[i] = 0.0;
        nz_elem_rows[i] = false;
      }
      nnz_rows.clear();
    }
    c.nz = c.value.size();
    c.col_ind[b.m] = c.nz;
