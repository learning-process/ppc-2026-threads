## 1. Введение

Разреженные матрицы встречаются в физическом моделировании, машинном обучении
и алгоритмах на графах. Их особенность: подавляющее число элементов равно нулю,
поэтому хранить их полностью нецелесообразно.

Формат CCS (Compressed Column Storage) хранит только ненулевые элементы,
упорядоченные по столбцам.

Последовательный алгоритм умножения разреженных матриц в формате CCS обрабатывает столбцы результирующей матрицы один за
другим. При больших размерах матриц это начинает занимать много времени, причем столбцы можно обрабатывать независимо.

Данная работа описывает параллельную реализацию умножения разреженных матриц
в формате CCS с элементами типа `double` с использованием стандартной библиотеки
потоков C++ (`std::thread`). В отличие от более высокоуровневых библиотек (например OpenMP),
управление потоками, декомпозиция задачи и распределение нагрузки выполняются
вручную.

## 2. Постановка задачи

Требуется вычислить произведение **C = A × B**, где:

- **A** — разреженная матрица размером `n × k` в формате CCS;
- **B** — разреженная матрица размером `k × m` в формате CCS;
- **C** — результирующая разреженная матрица размером `n × m` в формате CCS.

Требования к реализации:

- Параллельное вычисление с использованием `std::thread`.
- Входные данные передаются как `std::tuple<CCS, CCS>`, результат записывается в `OutType` (`CCS`).
- Строки внутри каждого столбца результата упорядочены по возрастанию.
- Валидация входных данных: `a.m == b.n` (число столбцов A равно числу строк B).

## 3. Алгоритм

### 3.1. Схема параллелизма

Столбцы матрицы B разбиваются на `threads_count` непрерывных диапазонов — по одному на поток.
Граница диапазона потока `tid`:

    jstart = (tid * b.m) / threads_count
    jend   = ((tid + 1) * b.m) / threads_count

Диапазоны не пересекаются и покрывают все `b.m` столбцов. Каждый поток работает
с независимым набором столбцов, что полностью исключает гонки при формировании
промежуточных результатов.

### 3.2. Реализация: фаза накопления

Каждый поток выполняет `ProcessColumnsRange`, в которой:

1. Выделяются **локальные** вспомогательные структуры — `accum`, `nz_elem_rows`,
   `nnz_rows` — один раз на поток, переиспользуются для каждого столбца.
2. Для каждого столбца `j` из своего диапазона вызывается `ProcessColumn`:
   - Перебираются ненулевые элементы `B[ind][j] = b_val` из j-го столбца B.
   - Для каждого такого элемента выполняется проход по `ind`-му столбцу A:
     `accum[i] += A[i][ind] * b_val`.
   - Строка `i` регистрируется в `nnz_rows` при первом посещении (флаг `nz_elem_rows[i]`).
   - После прохода `nnz_rows` сортируется; ненулевые `accum[i]` записываются в
     `local_values[j]` и `local_rows[j]`.
   - Аккумулятор и флаги сбрасываются точечно.

Результаты хранятся в `local_values[b.m]` и `local_rows[b.m]` — по одному вектору
на столбец, независимо от потока-владельца.

### 3.3. Сборка CCS

После `join` всех потоков фазы накопления серийно строится `col_ind` результирующей
матрицы через prefix-sum по размерам `local_values[j]`, и выделяется память под
`c.value` и `c.row`.

### 3.4. Реализация: фаза копирования

Вторая волна потоков (`CopyColumnsRange`) копирует данные из `local_values[j]` /
`local_rows[j]` непосредственно в `c.value` / `c.row` по известным смещениям
`c.col_ind[j]`. Каждый поток работает с тем же диапазоном столбцов, что и в фазе
накопления.

### 3.5. Сложность

| Характеристика | Оценка |
| --- | --- |
| Вычислительная | O(nnz(B) × avg\_col\_nnz(A) / threads_count) |
| Дополнительная память | O(threads_count × n + nnz(C)) |

## 4. Схема распараллеливания

STL-версия создаёт потоки явно и вручную задаёт декомпозицию задачи.

### 4.1. Этапы выполнения алгоритма

Алгоритм состоит из двух последовательных этапов работы потоков, разделённых серийной фазой построения `col_ind`:

    ┌──────────────────────────────────────────────────────────────┐
    │  Волна 1: threads_count × std::thread                        │
    │  ProcessColumnsRange — накопление в local_values/local_rows  │
    │                                                              │
    │  ┌──────────┬──────────┬──────────┬──────────┐              │
    │  │  tid=0   │  tid=1   │   ...    │ tid=N-1  │              │
    │  │[j0, j1)  │[j1, j2)  │   ...    │[jN, b.m) │              │
    │  └──────────┴──────────┴──────────┴──────────┘              │
    └──────────────────────────────────────────────────────────────┘
               │  join всех потоков
               ▼
    ┌──────────────────────────────────────────────────────────────┐
    │  Последовательно: prefix-sum local_values[j].size() → col_ind│
    │  Выделение c.value, c.row                                    │
    └──────────────────────────────────────────────────────────────┘
               │
               ▼
    ┌──────────────────────────────────────────────────────────────┐
    │  Волна 2: threads_count × std::thread                        │
    │  CopyColumnsRange — копирование в c.value/c.row              │
    │                                                              │
    │  ┌──────────┬──────────┬──────────┬──────────┐              │
    │  │  tid=0   │  tid=1   │   ...    │ tid=N-1  │              │
    │  │[j0, j1)  │[j1, j2)  │   ...    │[jN, b.m) │              │
    │  └──────────┴──────────┴──────────┴──────────┘              │
    └──────────────────────────────────────────────────────────────┘

**Этап 1 — накопление (`ProcessColumnsRange`).** Каждый поток получает непрерывный диапазон
столбцов B и вычисляет ненулевые элементы результирующих столбцов C.
Результаты складываются в `local_values[j]` и `local_rows[j]` — отдельные векторы на
каждый столбец. Потоки не взаимодействуют: запись идёт в непересекающиеся области памяти.

**Серийная фаза.** После `join` всех потоков первой волны строится массив `col_ind`
через prefix-sum по `local_values[j].size()`, выделяется память под `c.value` и `c.row`.

**Этап 2 — копирование (`CopyColumnsRange`).** Те же диапазоны столбцов, те же потоки.
Каждый поток копирует данные из `local_values[j]` / `local_rows[j]` в `c.value` / `c.row`
по известному смещению `c.col_ind[j]`. Запись не пересекается.

## 5. Детали реализации

### 5.1. Архитектура

Файлы:

- `stl/include/ops_stl.hpp`
- `stl/src/ops_stl.cpp`

Ход выполнения программы:

#### ValidationImpl

Проверка условия `a.m == b.n`.

#### PreProcessingImpl

Все входные данные уже готовы.

#### RunImpl

Управление потоками и вычислительная работа алгоритма.

#### PostProcessingImpl

Результат уже в готовом виде; корректность проверяется в тестовых файлах.

### 5.2. Определение числа потоков

    const size_t num_threads_raw = std::thread::hardware_concurrency();
    const size_t num_threads     = std::max<size_t>(1, num_threads_raw == 0 ? 1 : num_threads_raw);
    const size_t threads_count   = std::min(num_threads, b.m == 0 ? 1 : b.m);

`hardware_concurrency()` может вернуть 0, если платформа не поддерживает
определение числа логических ядер; такой случай явно обрабатывается.
Число потоков ограничивается сверху числом столбцов `b.m`, чтобы не создавать
пустые потоки.

### 5.3. Вспомогательные структуры

Три вектора выделяются **один раз на поток** в `ProcessColumnsRange`
и переиспользуются для каждого столбца диапазона:

| Вектор | Размер | Назначение |
| --- | --- | --- |
| `accum` | `a.n` | Вещественный аккумулятор, инициализирован `0.0` |
| `nz_elem_rows` | `a.n` | Маска «строка уже посещена», инициализирована `false` |
| `nnz_rows` | динамический | Список затронутых строк текущего столбца |

Для `nnz_rows` вызывается `reserve(a.n)`, что исключает реаллокации при
обработке плотных столбцов.

Корректность работы алгоритма проверялась функциональными тестами, а производительность — перфоманс-тестами.

### 6.1. Функциональные тесты

Матрица 6×6 с 12 ненулевыми элементами умножается сама на себя. Результат вычислен вручную и переведён в CCS — это
эталонное значение матрицы C.

### 6.2. Перфоманс-тесты

Создаются две ленточные матрицы A и B. Для проверки корректности используется метод умножения справа на произвольный
вектор:

`(A × B) × x = A × (B × x)`

Если результирующие векторы совпадают с учётом погрешности машинной арифметики, матрицы равны.

## Приложение

Фаза накопления (один поток, диапазон `[jstart, jend)`):

    std::vector<double> accum(a.n, 0.0);
    std::vector<bool>   nz_elem_rows(a.n, false);
    std::vector<size_t> nnz_rows;
    nnz_rows.reserve(a.n);

    for (size_t j = jstart; j < jend; ++j) {
      for (size_t k = b.col_ind[j]; k < b.col_ind[j + 1]; ++k) {
        size_t ind   = b.row[k];
        double b_val = b.value[k];
        for (size_t zc = a.col_ind[ind]; zc < a.col_ind[ind + 1]; ++zc) {
          size_t i     = a.row[zc];
          double a_val = a.value[zc];
          accum[i] += a_val * b_val;
          if (!nz_elem_rows[i]) {
            nz_elem_rows[i] = true;
            nnz_rows.push_back(i);
          }
        }
      }
      std::ranges::sort(nnz_rows);
      for (size_t i : nnz_rows) {
        if (accum[i] != 0.0) {
          local_rows[j].push_back(i);
          local_values[j].push_back(accum[i]);
        }
        accum[i]       = 0.0;
        nz_elem_rows[i] = false;
      }
      nnz_rows.clear();
    }

Фаза копирования (один поток, диапазон `[jstart, jend)`):

    for (size_t j = jstart; j < jend; ++j) {
      size_t offset = c.col_ind[j];
      size_t col_nz = local_values[j].size();
      for (size_t k = 0; k < col_nz; ++k) {
        c.value[offset + k] = local_values[j][k];
        c.row[offset + k]   = local_rows[j][k];
      }
    }
