## 1. Контекст

TBB-реализация блочной фильтрации изображения ядром Гаусса 3×3.
Используется task-based параллелизм с автоматической балансировкой нагрузки
(work stealing). Цель — оценить эффективность TBB на задаче с регулярной
вычислительной нагрузкой.

## 2. Постановка задачи

Полностью совпадает с последовательной версией (см. seq/report.md).
Вход — изображение, выход — отфильтрованное изображение.
Корректность проверяется сравнением с SEQ.

## 3. Базовый алгоритм

Тот же блочный алгоритм, что в SEQ: разбиение на блоки 64×64,
копирование с отступами, свёртка с ядром Гаусса, запись результата.

## 4. Схема распараллеливания

**Какой примитив выбран:**

- `tbb::parallel_for` для фильтрации внутри блока
- `tbb::parallel_for` для копирования с отступами
- `tbb::parallel_for` для внешнего цикла по блокам

**Как определяется диапазон работы (blocked_range):**
В коде используется вызов `tbb::parallel_for(0, inner_height, [&](int row) {...})`.
При передаче двух целых чисел TBB внутри неявно создаёт объект
`tbb::blocked_range<int>(0, inner_height)`,
который представляет диапазон `[0, inner_height)`
и умеет рекурсивно разбивать себя на поддиапазоны.

**Какой grainsize используется и почему:**
Явно не задан. TBB выбирает размер зёрен автоматически.

**Какой partitioner используется:**
`auto_partitioner` (по умолчанию).
Адаптивный механизм, который пытается подобрать оптимальную глубину разбиения, минимизируя оверхед.

**Способ контроля конкуренции:**
Конкуренция отсутствует.
Каждый поток обрабатывает свой поддиапазон строк и записывает результат в независимые позиции выходного буфера (разные пиксели).
Гонок данных нет, средства синхронизации (mutex, atomic, critical section) не требуются.
Количество потоков TBB по умолчанию равно числу аппаратных ядер и программно не ограничивалось.

## 5. Детали реализации

**Файлы:** tbb/include/ops_tbb.hpp, tbb/src/ops_tbb.cpp

**Фрагмент кода фильтрации:**

```cpp
tbb::parallel_for(0, inner_height, [&](int row) {
    for (int col = 0; col < inner_width; ++col) {
        for (int channel = 0; channel < channels; ++channel) {
            float sum = 0.0f;
            for (int ky = -1; ky <= 1; ++ky) {
                for (int kx = -1; kx <= 1; ++kx) {
                    sum += input_block[...] * kGaussianKernel[...];
                }
            }
            output_block[...] = static_cast<uint8_t>(std::round(sum));
        }
    }
});
```

**Фрагмент кода копирования с отступами:**

```cpp
tbb::parallel_for(0, padded_height, [&](int row) {
    for (int col = 0; col < padded_width; ++col) {
        int src_y = std::clamp(block_y + row - 1, 0, height - 1);
        int src_x = std::clamp(block_x + col - 1, 0, width - 1);
        for (int channel = 0; channel < channels; ++channel) {
            // копирование пикселя
        }
    }
});
```

**Фрагмент кода внешнего цикла по блокам:**

```cpp
tbb::parallel_for(0, blocks_y, [&](int by) {
    for (int bx = 0; bx < blocks_x; ++bx) {
        // обработка одного блока
    }
});
```

**Особенности реализации:**

- Нет ручного управления потоками.
- TBB сам выбирает число worker-потоков.
- Используется автоматическое разбиение диапазонов.

## 6. Проверка корректности

Сравнение с SEQ. Функциональные тесты (из tests/functional/main.cpp):

- Тест 1: 2×2 серое, вход [100,150,200,250] → выход [138,163,188,213]
- Тест 2: 3×3 серое, вход [1,2,3,4,5,6,7,8,9] → выход [2,3,4,4,5,6,7,7,8]
- Тест 3: 2×2 RGB, вход 12 чисел → выход 12 чисел
- Тест 4: 1×1 серое, вход [255] → выход [255]

Все тесты пройдены для 1, 2, 4, 8 потоков.

## 8. Результаты

| Потоков | Время, с | Speedup | Efficiency |
|---------|----------|---------|------------|
| 1       | 0.288    | 0.74    | 74%        |
| 2       | 0.148    | 1.45    | 72%        |
| 4       | 0.080    | 2.68    | 67%        |
| 8       | 0.052    | 4.12    | 51%        |

**Комментарий о масштабируемости и узких местах:**

- TBB показывает лучшее ускорение, чем OpenMP (4.12× против 1.86× на 8 потоках).
- При 1 потоке TBB медленнее SEQ на 26% из-за оверхеда.
- Эффективность остаётся высокой до 4 потоков (67%), затем снижается.
- Work stealing помогает балансировать нагрузку при неравномерных блоках.

## 9. Выводы

TBB обеспечивает ускорение до 4.12× на 8 потоках.
TBB рекомендуется использовать при числе потоков 4 и более.
