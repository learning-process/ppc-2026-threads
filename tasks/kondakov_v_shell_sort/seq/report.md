## 1. Контекст

Последовательная версия (SEQ) нужна как отправная точка проекта. Она решает задачу упорядочивания массива и выступает в роли "эталона" для проверки корректности остальных параллельных реализаций. Кроме того, время ее работы служит базой для расчета ускорения и эффективности всех параллельных версий.

## 2. Постановка задачи

На вход подается одномерный вектор целых чисел (`std::vector<int>`). На выходе требуется получить тот же вектор, отсортированный по возрастанию. Ограничения: решение использует O(N) дополнительной памяти. Отрабатываются крайние случаи: пустые векторы, векторы из 1-2 элементов, уже отсортированные.

## 3. Базовый алгоритм

Используется алгоритм сортировки Шелла c простым слиянием. Сначала вектор делится на две равные половины. Каждая половина сортируется методом Шелла с классическим выбором зазоров (начиная с N/2 и деля на 2 на каждом шаге). На каждом шаге элементы, отстоящие друг от друга на величину зазора, сортируются вставками. После того как обе половины отсортированы, они объединяются в единый отсортированный массив с помощью функции простого линейного слияния.
Асимптотика по времени: в среднем $O(N \log^2 N)$, в худшем - $O(N^2)$.
Асимптотика по памяти: $O(N)$ из-за выделения памяти под половинки массива.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`
Метод `ValidationImpl` выступает первичным фильтром: он проверяет через метод `empty()`, что входной вектор содержит хотя бы один элемент, тем самым отсекая некорректные наборы данных до начала вычислений. Метод `PreProcessingImpl` отвечает за подготовку рабочего пространства: он напрямую копирует исходные данные из входа во внутренний выходной буфер, который в дальнейшем будет мутировать.

В самом начале метода `RunImpl` стоит защита от лишней работы: если размер массива меньше двух (0 или 1 элемент), алгоритм мгновенно завершается. В основном сценарии вычисляется средний индекс (`mid`), и массив физически разбивается на два временных вектора (`left_part` и `right_part`). К каждому из этих векторов последовательно применяется функция `ShellSort`. Затем отсортированные половинки сливаются обратно в общий выходной буфер с помощью вспомогательной функции `SimpleMerge`. Завершается метод проверкой корректности сортировки с помощью стандартного алгоритма `std::ranges::is_sorted`. Метод `PostProcessingImpl` выполняет финальную формальную проверку, удостоверяясь, что выходной вектор после всех манипуляций не оказался пустым.

Листинг функции сортировки Шелла:

```cpp
// File: seq/src/ops_seq.cpp
void ShellSort(std::vector<int> &data) {
  const size_t n = data.size();
  for (size_t gap = n / 2; gap > 0; gap /= 2) {
    for (size_t i = gap; i < n; ++i) {
      int value = data[i];
      size_t j = i;
      while (j >= gap && data[j - gap] > value) {
        data[j] = data[j - gap];
        j -= gap;
      }
      data[j] = value;
    }
  }
}
```

## 5. Проверка корректности

Результат сравнивается побитово со стандартной функцией `std::sort`. Для проверки используются 12 функциональных тестов, которые покрывают как обычные случайные массивы, так и особые случаи.
Примеры входа и ожидаемого выхода:

- Тест с одним элементом: вход `[1]`, ожидается `[1]`.
- Обратный порядок: вход `[5, 4, 3, 2, 1]`, ожидается `[1, 2, 3, 4, 5]`.
- Смешанные числа: вход `[42, -42, 17, 0, -1]`, ожидается `[-42, -1, 0, 17, 42]`.
