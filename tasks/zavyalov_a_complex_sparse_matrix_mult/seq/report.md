## 1. Контекст

Задача — перемножение двух разреженных матриц, элементы которых являются комплексными числами. Последовательная (SEQ)
версия служит эталоном корректности: именно её результаты будут использоваться как опорные при проверке всех
параллельных реализаций (OMP, TBB, STL, ALL), а время её
выполнения на `p=1` станет знаменателем при расчёте ускорения.

## 2. Постановка задачи

**Входные данные (`InType`):** пара разреженных матриц `(A, B)
` типа `SparseMatrix`, упакованных в `std::tuple`. Каждая матрица хранится в формате CCS (столбцовый формат): массив
значений `val`, массив индексов строк `row_ind` и массив индексов столбцов `col_ind`, а также поля `height` и `width`.

**Выходные данные (`OutType`):** разреженная матрица `C = A × B` в том же формате CCS.

**Ограничения и крайние случаи:**

- Умножение определено только если ширина матрицы `A` совпадает с высотой матрицы `B` (`A.width == B.height`).
Нарушение этого условия является единственным блокирующим ограничением и обрабатывается в `ValidationImpl`.
- Если одна или обе матрицы пусты (`height == 0` или `width == 0`), конструктор возвращает пустую структуру без входа в
циклы.
- Нулевые элементы не хранятся: в конструкторе `SparseMatrix` ячейки с `val == Complex(0.0)` пропускаются.
- Результирующие ячейки, ставшие нулевыми после сложения вкладов, формально попадут в `mp` с нулевым значением;
вынесение такой фильтрации за рамки текущей реализации является осознанным упрощением.

## 3. Базовый алгоритм

Алгоритм реализован через перебор всех ненулевых элементов матрицы `A` по внешнему циклу и всех ненулевых элементов
матрицы `B` по внутреннему. Вклад пары `(i, j)` добавляется в
результат тогда и только тогда, когда столбец элемента `A` совпадает со строкой элемента `B` (`col_a == row_b`).
Промежуточные суммы накапливаются в `std::map<pair<size_t,size_t>, Complex>` с ключом `(строка, столбец)` результата.

**Пошаговое описание:**

1. Проверить совместимость размеров.
2. Для каждого ненулевого элемента `val_a` из `A` (строка `row_a`, столбец `col_a`):
   - Для каждого ненулевого элемента `val_b` из `B` (строка `row_b`, столбец `col_b`):
     - Если `col_a == row_b`, добавить `val_a * val_b` к `mp[{row_a, col_b}]`.
3. Собрать результирующую матрицу `C` из накопленного `mp`.

**Асимптотика по времени:** O(nnz_A · nnz_B), где `nnz_A` и `nnz_B` — число ненулевых элементов матриц.

**Асимптотика по памяти:** O(nnz_A + nnz_B + nnz_C), где `nnz_C` — число ненулевых в результате.

**Инварианты и критерий корректности:**

- `res.height == A.height`, `res.width == B.width`.
- Каждая запись в `res` имеет уникальную пару `(row, col)`.
- При умножении комплексных чисел используется правило: `(a + bi)(c + di) = (ac−bd) + (ad+bc)i`.

## 4. Детали реализации

**Файлы:**

- `common/include/common.hpp` — типы `Complex`, `SparseMatrix`, `InType`, `OutType`, `BaseTask`.
- `seq/include/ops_seq.hpp` — объявление класса `ZavyalovAComplSparseMatrMultSEQ`.
- `seq/src/ops_seq.cpp` — реализация четырёх методов пайплайна.

**Структура пайплайна:**

```cpp
// File: seq/src/ops_seq.cpp

bool ZavyalovAComplSparseMatrMultSEQ::ValidationImpl() {
  const auto &matr_a = std::get<0>(GetInput());
  const auto &matr_b = std::get<1>(GetInput());
  return matr_a.width == matr_b.height;  // единственная проверка совместимости
}

bool ZavyalovAComplSparseMatrMultSEQ::RunImpl() {
  const auto &matr_a = std::get<0>(GetInput());
  const auto &matr_b = std::get<1>(GetInput());
  GetOutput() = matr_a * matr_b;  // делегирование оператору SparseMatrix::operator*
  return true;
}
```

- **`ValidationImpl`** — проверяет, что `A.width == B.height`. При нарушении пайплайн прерывается без вычислений.
- **`PreProcessingImpl`** — возвращает `true` без действий: данные уже получены в конструкторе через `GetInput() = in`.
- **`RunImpl`** — вызывает `operator*` структуры `SparseMatrix`. Всё вычислительное ядро сосредоточено именно здесь.
- **`PostProcessingImpl`** — возвращает `true` без действий: результат уже записан в `GetOutput()`.

**Ключевой фрагмент вычислительного ядра** (`common/include/common.hpp`):

```cpp
// File: common/include/common.hpp
// SparseMatrix::operator*(const SparseMatrix &matr_b) — вычислительное ядро

for (size_t i = 0; i < Count(); ++i) {
  size_t row_a = row_ind[i];
  size_t col_a = col_ind[i];
  Complex val_a = val[i];

  for (size_t j = 0; j < matr_b.Count(); ++j) {
    size_t row_b = matr_b.row_ind[j];
    if (col_a == row_b) {
      mp[{row_a, matr_b.col_ind[j]}] += val_a * matr_b.val[j];
    }
  }
}
```

Вложенный цикл является «горячей точкой» алгоритма. При плотных матрицах сложность растёт квадратично по `nnz`.
Именно эти два цикла будут распараллеливаться в параллельных версиях.
