## 1. Контекст

Последовательная версия алгоритма линейной фильтрации изображения с использованием ядра Гаусса 3x3 является
фундаментальной базой (baseline) всего проекта.
Она формирует эталонный конвейер обработки данных и устанавливает критерии корректности для последующих
параллельных реализаций. Оценка производительности однопоточной
версии критически важна, так как её время выполнения служит единственным знаменателем при расчете реального
ускорения (Speedup) и эффективности (Efficiency) многопоточных
конфигураций.

## 2. Постановка задачи

- **Входные данные (InType)**: Целое положительное число N, задающее размер квадратного изображения (ширина W = N,
высота H = N). Само изображение передается в виде одномерного
массива байт в трехканальном формате RGB (3 байта на один пиксель).
- **Выходные данные (OutType)**: Целое число типа int в диапазоне [0, 255], представляющее собой среднее
арифметическое значение интенсивности всех байт результирующего изображения.
Используется для компактной проверки тестов во фреймворке.
- **Ограничения**: Размер кадра N должен быть строго больше или равен 3 (N >= 3), что продиктовано размером
апертуры фильтра (3x3). Меньшие значения отсекаются
на этапе валидации.
- **Крайние случаи**: Пиксели на границах кадра, для которых окрестность 3x3 выходит за рамки матрицы,
обрабатываются по стратегии зажима (clamp) — индекс ограничивается
диапазонами [0, W-1] и [0, H-1], возвращая значение ближайшего крайнего пикселя.

## 3. Базовый алгоритм

Пошаговое описание последовательного алгоритма:

1. Выделяется непрерывный участок памяти под входной вектор input_image_ и выходной
вектор output_image_ размером N *N* 3 байт в рамках
единого адресного пространства.
2. Метод RunImpl запускает последовательный обход матрицы пикселей во вложенных циклах по строкам py
(от 0 до height_) и столбцам px (от 0 до width_).
3. Для каждого пикселя поочередно обрабатываются три цветовых канала RGB (ch от 0 до 2) внутри метода
ApplyGaussianToPixel.
4. Вокруг целевого пикселя собирается локальная окрестность 3x3. Координаты смещения фильтра clamp-ятся под
физические границы кадра
с помощью std::clamp. Значения пикселей умножаются на соответствующие веса целочисленной матрицы Гаусса, после чего
накопленная сумма
делится на нормировочный коэффициент kKernelSum = 16.
5. Полученное значение типа uint8_t записывается в выходной буфер по вычисленному смещению.

- **Асимптотика по времени**: O(N^2) — линейный обход всех пикселей матрицы, где для каждого элемента выполняется
фиксированное число операций свёртки (9 итераций ядра на канал).
- **Асимптотика по памяти**: O(N^2) — для хранения исходного и отфильтрованного изображений в памяти требуются два
независимых
буфера, масштабирующихся пропорционально общему числу пикселей кадра.
- **Инварианты и критерий корректности**: Значения пикселей строго лежат в диапазоне [0, 255]. Итоговый результат
 выполнения
алгоритма полностью детерминирован и не зависит от запусков.

## 4. Детали реализации

Файлы: seq/include/ops_seq.hpp, seq/src/ops_seq.cpp

Конвейер обработки данных жестко регламентирован базовыми методами каркаса BaseTask:

- **ValidationImpl**: Выполняет строгую проверку входных параметров. Проверяет, что размер
кадра N >= 3 (kKernelSize). Если условие нарушено, возвращает false, блокируя запуск.
- **PreProcessingImpl**: Извлекает размеры кадра, аллоцирует память под внутренние векторы input_image_ и
output_image_ с
помощью .assign() и инициализирует входной кадр тестовым заполнением со значением интенсивности 100.
- **RunImpl**: Выполняет основную вычислительную работу, вызывая вложенные последовательные циклы фильтрации и
метод ApplyGaussianToPixel
для каждой координатной пары.
- **PostProcessingImpl**: Агрегирует результаты. Суммирует значения всех элементов результирующего массива
output_image_ в переменную
типа int64_t для предотвращения переполнения, делит её на общий размер вектора и записывает полученное среднее
значение (OutType) в область памяти GetOutput().

### Листинг ключевого фрагмента (базовое вычисление)

```cpp
void BuzulukskiDGausGorizontalSEQ::ApplyGaussianToPixel(int py, int px) {
  const auto u_width = static_cast<std::size_t>(width_);
  const auto u_height = static_cast<std::size_t>(height_);
  const auto u_channels = static_cast<std::size_t>(kChannels);

  for (int ch = 0; ch < kChannels; ++ch) {
    int sum = 0;
    for (int ky = -1; ky <= 1; ++ky) {
      for (int kx = -1; kx <= 1; ++kx) {
        const int ny = std::clamp(py + ky, 0, static_cast<int>(u_height) - 1);
        const int nx = std::clamp(px + kx, 0, static_cast<int>(u_width) - 1);

        const auto idx = (((static_cast<std::size_t>(ny) * u_width) + static_cast<std::size_t>(nx)) * u_channels) + static_cast<std::size_t>(ch);
        const auto row_idx = static_cast<std::size_t>(ky) + 1;
        const auto col_idx = static_cast<std::size_t>(kx) + 1;

        sum += static_cast<int>(input_image_.at(idx)) * kKernel.at(row_idx).at(col_idx);
      }
    }
    const auto out_idx = (((static_cast<std::size_t>(py) * u_width) + static_cast<std::size_t>(px)) * u_channels) + static_cast<std::size_t>(ch);
    output_image_.at(out_idx) = static_cast<uint8_t>(sum / kKernelSum);
  }
}
```
