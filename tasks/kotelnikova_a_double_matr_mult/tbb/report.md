## 1. Контекст

В отличие от OpenMP, где управление потоками происходит напрямую, TBB абстрагирует низкоуровневые детали и предоставляет
высокоуровневые алгоритмы (parallel_for, parallel_reduce, parallel_scan) и контейнеры. Библиотека автоматически
балансирует нагрузку между доступными ядрами и масштабируется при изменении числа аппаратных потоков.
Данная работа реализует умножение разреженных матриц с использованием oneTBB.
Последовательная версия (seq/report.md) служит эталоном корректности, а OpenMP-версия (omp/report.md) — для
сравнения с конкурентной технологией.

## 2. Постановка задачи

### Входные данные

Две разреженные матрицы A и B в формате CCS.
Структура SparseMatrixCCS:

- `values` — массив ненулевых значений (тип double);
- `row_indices` — массив индексов строк для каждого значения;
- `col_ptrs` — массив указателей на начало каждого столбца (размер `cols + 1`);
- `rows` — количество строк;
- `cols` — количество столбцов.

### Выходные данные

Результирующая матрица C = A × B в формате CCS.

### Ограничения

- Количество столбцов матрицы A должно равняться количеству строк матрицы B
(причем количество строк и столбцов матрицы должно быть неотрицательно);
- Все индексы строк находятся в диапазоне `[0, rows-1]`;
- Указатели `col_ptrs` являются неубывающей последовательностью
- Первый указатель `col_ptrs[0] = 0`;
- Последний указатель `col_ptrs[cols] = общему количеству ненулевых элементов`.

## 3. Базовый алгоритм

Последовательная версия вычисляет произведение матриц по следующей схеме. Для каждого столбца j матрицы B:

- Обход ненулевых элементов столбца j, получение пар (k, `B[k][j]`);
- Для каждого k обход ненулевых элементов столбца k матрицы A;
- Накопление в `temp[i]: A[i][k] × B[k][j]`;
- Сохранение ненулевых значений из temp в результирующую матрицу.
Детальное описание последовательного алгоритма приведено в `seq/report.md`.

## 4. Схема распараллеливания

В отличие от OpenMP, где используется `#pragma omp parallel for`, в TBB применяется `tbb::parallel_for`:

```cpp
tbb::parallel_for(tbb::blocked_range<int>(0, b.cols, grain_size),
                  [&](const tbb::blocked_range<int> &range) {
    for (int j = range.begin(); j < range.end(); ++j) {
      // обработка столбца j
    }
  },
                  tbb::simple_partitioner());
```

### Основные параметры

`blocked_range<int>(0, b.cols, grain_size)` — определяет диапазон итераций (столбцы) и размер гранулы (chunk).
`grain_size = 8` — каждый поток получает блок из 8 столбцов. Выбран  как оптимальный для балансировки нагрузки
и минимизации overhead.
`simple_partitioner()` — статическое разбиение без рекурсивного деления. Используется для минимизации overhead
при неравномерной нагрузке.

### Двухпроходный алгоритм

Как и в OpenMP-версии, алгоритм разделён на два прохода:

1. Первый проход (подсчёт) — вычисление количества ненулевых элементов в каждом столбце результата.
2. Второй проход (заполнение) — запись значений в заранее выделенные массивы.

## 5. Детали реализации

Файлы: `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`

### Основные методы

`ValidationImpl()` — проверка корректности входных данных:

- Валидность структуры CCS обеих матриц
- Совместимость размерностей (A.cols == B.rows)

`PreProcessingImpl()` — подготовка к вычислениям:

- Создание результирующей матрицы C с размерами A.rows × B.cols

`RunImpl()` — выполнение умножения:

- Вызов `MultiplyMatrices(A, B)`

`PostProcessingImpl()` — завершающие действия:

- В данной реализации не требуется (возвращает true)

### Вспомогательные методы

`IsMatrixValid()` — проверка валидности CCS формата (реализация ограничений из 2):

- Размеры матрицы неотрицательны;
- Размер `col_ptrs` равен `cols + 1`;
- Количество значений равно количеству индексов строк;
- Первый указатель `col_ptrs[0] = 0`;
- Последний указатель `col_ptrs[cols] = nnz` (где `nnz(X)` — количество ненулевых элементов в матрице X);
- Указатели неубывают и неотрицательны;
- Индексы строк в допустимом диапазоне.

`ComputeColumn()` — вычисляет значения одного столбца результирующей матрицы
(листинг представлен в файле `omp/report.md`).

`CountNonZero()` — подсчитывает количество ненулевых элементов в столбце с заданной погрешностью
(листинг представлен в файле `omp/report.md`).

`FillColumn()` — заполняет результирующие массивы значениями и индексами строк для одного столбца
(листинг представлен в файле `omp/report.md`).

`MultiplyMatrices()` — основной алгоритм с TBB распараллеливанием:

- Реализует двухпроходную схему (описанную в разделе 4);
- Использует `tbb::parallel_for` с `blocked_range` и `grain_size = 8`;
- Первый проход: параллельное вычисление количества ненулевых элементов для каждого столбца;
- Второй проход: параллельное заполнение результирующих массивов.

```cpp
// File: kotelnikova_a_double_matr_mult/tbb/src/ops_tbb.cpp
SparseMatrixCCS KotelnikovaATaskTBB::MultiplyMatrices(const SparseMatrixCCS &a, const SparseMatrixCCS &b) {
  SparseMatrixCCS result(a.rows, b.cols);

  const double epsilon = 1e-10;
  const int grain_size = 8;
  std::vector<int> col_start(b.cols, 0);

  tbb::parallel_for(tbb::blocked_range<int>(0, b.cols, grain_size),
                    [&](const tbb::blocked_range<int> &range) {
    for (int j = range.begin(); j < range.end(); ++j) {
      std::vector<double> column = ComputeColumn(a, b, j);
      col_start[j] = CountNonZero(column, epsilon);
    }
  }, tbb::simple_partitioner());

  std::vector<int> col_ptr(b.cols + 1, 0);
  for (int j = 0; j < b.cols; ++j) {
    col_ptr[j + 1] = col_ptr[j] + col_start[j];
  }

  const int total_nnz = col_ptr[b.cols];
  result.values.resize(total_nnz);
  result.row_indices.resize(total_nnz);
  result.col_ptrs = col_ptr;

  tbb::parallel_for(tbb::blocked_range<int>(0, b.cols, grain_size),
                    [&](const tbb::blocked_range<int> &range) {
    for (int j = range.begin(); j < range.end(); ++j) {
      std::vector<double> column = ComputeColumn(a, b, j);
      FillColumn(column, epsilon, result.row_indices, result.values, col_ptr[j]);
    }
  }, tbb::simple_partitioner());

  return result;
}
```

## 9. Выводы

### TBB дает выигрыш

На больших матрицах достигается ускорение, сопоставимое с OpenMP.
При равномерной нагрузке TBB эффективно балансирует задачи

### Недостатки TBB

- Больший overhead на малых задачах — создание объектов blocked_range и лямбда-функций дороже,
чем директива #pragma omp
- Сложность отладки — модель задач менее прозрачна, чем модель потоков OpenMP
- Дополнительная зависимость — требуется наличие библиотеки oneTBB

### Сравнение с OpenMP

- TBB-версия демонстрирует примерно одинаковое ускорение с OpenMP (разница в пределах погрешности измерений).
Выбор между технологиями определяется:
- TBB предпочтительнее, если важна переносимость и интеграция с современным C++
- OpenMP предпочтительнее, если нужны минимальные изменения последовательного кода и поддержка директив
