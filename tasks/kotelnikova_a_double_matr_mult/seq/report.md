## 1. Контекст

Умножение разреженных матриц является одной из ключевых операций в вычислительной линейной алгебре.
Использование данного формата хранения (в данном случае CCS — Compressed Column Storage) позволяет:

- Экономить память за счёт хранения только ненулевых элементов;
- Сокращать количество арифметических операций при умножении;
- Эффективно работать с матрицами, где плотность ненулевых элементов не превышает 10-20%.
Последовательная версия является эталоном корректности и базой для измерения ускорения параллельных реализаций.

## 2. Постановка задачи

### Входные данные

Две разреженные матрицы A и B в формате CCS.
Структура SparseMatrixCCS:

- `values` — массив ненулевых значений (тип double);
- `row_indices` — массив индексов строк для каждого значения;
- `col_ptrs` — массив указателей на начало каждого столбца (размер `cols + 1`);
- `rows` — количество строк;
- `cols` — количество столбцов.

### Выходные данные

Результирующая матрица C = A × B в формате CCS.

### Ограничения

- Количество столбцов матрицы A должно равняться количеству строк матрицы B
(причем количество строк и столбцов матрицы должно быть неотрицательно);
- Все индексы строк находятся в диапазоне `[0, rows-1]`;
- Указатели `col_ptrs` являются неубывающей последовательностью
- Первый указатель `col_ptrs[0] = 0`;
- Последний указатель `col_ptrs[cols] = общему количеству ненулевых элементов`.

### Крайние случаи

- Нулевая матрица (отсутствие ненулевых элементов);
- Матрицы с размерностью 0×0;
- Единичная матрица;
- Диагональные матрицы;
- Прямоугольные матрицы (когда A.cols != A.rows).

## 3. Базовый алгоритм

### Принцип работы

Алгоритм выполняет умножение матриц `C[i][j] = A[i][k] × B[k][j]`, но обрабатывает
только ненулевые элементы благодаря CCS формату.

### Пошаговое описание

1. Инициализация

- Создаётся результирующая матрица C с размерами `A.rows × B.cols`.
- Выделяется временный вектор `temp` размером `A.rows` для накопления значений текущего столбца.

2. Обход столбцов матрицы B
Для каждого столбца j матрицы B (от 0 до B.cols-1):

- Запоминается текущая позиция в массиве значений C как начало столбца j.
- Обрабатываются все ненулевые элементы столбца j матрицы B.

3. Внутреннее умножение
Для каждого ненулевого элемента `B[k][j]`:

- Находятся все ненулевые элементы в столбце k матрицы A.
- Для каждого элемента `A[i][k]` выполняется: `temp[i] += A[i][k] × B[k][j]`

4. Сохранение результатов столбца

- Все ненулевые значения из `temp` (`|temp[i]| > 1e-10`) сохраняются в C
- Сохраняются соответствующие индексы строк
- `temp` обнуляется для следующего столбца

5. Завершение

- Устанавливается финальный указатель `C.col_ptrs[B.cols]`

```cpp
// File: kotelnikova_a_double_matr_mult/seq/src/ops_seq.cpp
SparseMatrixCCS KotelnikovaATaskSEQ::MultiplyMatrices(const SparseMatrixCCS &a, const SparseMatrixCCS &b) {
  SparseMatrixCCS result(a.rows, b.cols);
  std::vector<double> temp(a.rows, 0.0);

  for (int j = 0; j < b.cols; ++j) {
    result.col_ptrs[j] = static_cast<int>(result.values.size());
    ProcessColumn(a, b, j, temp);
    SaveColumnResults(a.rows, temp, result);
  }

  result.col_ptrs[b.cols] = static_cast<int>(result.values.size());
  return result;
}
```

### Критерий корректности

Результат умножения должен совпадать с результатом умножения плотных матриц, преобразованных из разреженного формата.
Допустимая погрешность: 1e-10 для сравнения значений с плавающей точкой.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

### Основные методы

`ValidationImpl()` — проверка корректности входных данных:

- Валидность структуры CCS обеих матриц
- Совместимость размерностей (A.cols == B.rows)

`PreProcessingImpl()` — подготовка к вычислениям:

- Создание результирующей матрицы C с размерами A.rows × B.cols

`RunImpl()` — выполнение умножения:

- Вызов `MultiplyMatrices(A, B)`

`PostProcessingImpl()` — завершающие действия:

- В данной реализации не требуется (возвращает true)

### Вспомогательные методы

`IsMatrixValid()` — проверка валидности CCS формата (реализация ограничений из 2):

- Размеры матрицы неотрицательны;
- Размер `col_ptrs` равен `cols + 1`;
- Количество значений равно количеству индексов строк;
- Первый указатель `col_ptrs[0] = 0`;
- Последний указатель `col_ptrs[cols] = nnz` (где `nnz(X)` — количество ненулевых элементов в матрице X);
- Указатели неубывают и неотрицательны;
- Индексы строк в допустимом диапазоне.

`MultiplyMatrices()` — основной алгоритм умножения:

- Реализует описанную выше схему (в пункте 3);
- Так же разбит на вспомогательные функции для снижения когнитивной сложности:
`ProcessColumn()` - обрабатывает один столбец матрицы B.

```cpp
// File: kotelnikova_a_double_matr_mult/seq/src/ops_seq.cpp
void ProcessColumn(const SparseMatrixCCS &a, const SparseMatrixCCS &b, int j, std::vector<double> &temp) {
  for (int k = 0; k < a.cols; ++k) {
    double b_val = 0.0;
    for (int b_idx = b.col_ptrs[j]; b_idx < b.col_ptrs[j + 1]; ++b_idx) {
      if (b.row_indices[b_idx] == k) {
        b_val = b.values[b_idx];
        break;
      }
    }

    if (b_val == 0.0) continue;

    for (int a_idx = a.col_ptrs[k]; a_idx < a.col_ptrs[k + 1]; ++a_idx) {
      const int i = a.row_indices[a_idx];
      const double a_val = a.values[a_idx];
      temp[i] += a_val * b_val;
    }
  }
}
```

`SaveColumnResults()` — сохраняет накопленные значения в результирующую матрицу.

```cpp
// File: kotelnikova_a_double_matr_mult/seq/src/ops_seq.cpp
void SaveColumnResults(int rows, std::vector<double> &temp, SparseMatrixCCS &result) {
  for (int i = 0; i < rows; ++i) {
    if (std::abs(temp[i]) > 1e-10) {
      result.values.push_back(temp[i]);
      result.row_indices.push_back(i);
      temp[i] = 0.0;
    }
  }
}
```

### Обработка граничных случаев

- Пустые матрицы (zero_result): корректно обрабатываются, результат нулевая матрица.
- Единичная матрица: A × I = A.
- Прямоугольные матрицы: поддерживаются любые размеры при условии A.cols == B.rows.

## 5. Проверка корректности

Результат умножения разреженных матриц сравнивается с эталоном — умножением плотных
матриц, полученных преобразованием из CCS формата.

Функциональные тесты (5 тестовых случаев):

| Тест                | Размеры   | Описание                        | Проверяемые элементы    |
|---------------------|-----------|---------------------------------|-------------------------|
| simple_3x3          | 3×3 × 3×3 | Умножение с диагональной B      | 5 ненулевых элементов   |
| rectangular_2x3_3x2 | 2×3 × 3×2 | Прямоугольные матрицы           | 3 ненулевых элемента    |
| zero_result         | 2×2 × 2×2 | Умножение с нулевым результатом | Нулевая матрица         |
| identity_matrix     | 3×3 × 3×3 | Умножение на единичную матрицу  | A × I = A               |
| diagonal_matrices   | 4×4 × 4×4 | Умножение диагональных матриц   | 4 диагональных элемента |

Пример тестового случая (simple_3x3):

Матрица A (3×3):

```cpp
[1.5, 0,   2.8]
[0,   3.7, 0  ]
[4.2, 0,   5.1]
```

Матрица B (3×3):

```cpp
[1.2, 0,   0  ]
[0,   2.3, 0  ]
[0,   0,   3.4]
```

Ожидаемый результат:

C = A × B:

```cpp
[1.8, 0,    9.52]
[0,   8.51, 0   ]
[5.04, 0,   17.34]
```

Допустимая погрешность для сравнения значений с плавающей точкой: `ε = 1e-10`
