## 1. Контекст

Данная работа переносит последовательный алгоритм умножения разреженных матриц на платформу OpenMP.
Основная цель — достичь ускорения за счёт параллельной обработки независимых столбцов результирующей матрицы.

## 2. Постановка задачи

### Входные данные

Две разреженные матрицы A и B в формате CCS.
Структура SparseMatrixCCS:

- `values` — массив ненулевых значений (тип double);
- `row_indices` — массив индексов строк для каждого значения;
- `col_ptrs` — массив указателей на начало каждого столбца (размер `cols + 1`);
- `rows` — количество строк;
- `cols` — количество столбцов.

### Выходные данные

Результирующая матрица C = A × B в формате CCS.

### Ограничения

- Количество столбцов матрицы A должно равняться количеству строк матрицы B
(причем количество строк и столбцов матрицы должно быть неотрицательно);
- Все индексы строк находятся в диапазоне `[0, rows-1]`;
- Указатели `col_ptrs` являются неубывающей последовательностью
- Первый указатель `col_ptrs[0] = 0`;
- Последний указатель `col_ptrs[cols] = общему количеству ненулевых элементов`.

## 3. Базовый алгоритм

Последовательная версия вычисляет произведение матриц по следующей схеме. Для каждого столбца j матрицы B:

- Обход ненулевых элементов столбца j, получение пар (k, `B[k][j]`);
- Для каждого k обход ненулевых элементов столбца k матрицы A;
- Накопление в `temp[i]: A[i][k] × B[k][j]`;
- Сохранение ненулевых значений из temp в результирующую матрицу.
Детальное описание последовательного алгоритма приведено в `seq/report.md`.

## 4. Схема распараллеливания

### Основная идея

Алгоритм разделён на два независимых прохода:

1. Первый проход (подсчёт) — для каждого столбца j вычисляется количество ненулевых
элементов в этом столбце результата.
Это позволяет выделить память единым блоком и избежать повторных перераспределений.
2. Второй проход (заполнение) — для каждого столбца j вычисляются ненулевые значения и
записываются в заранее выделенные позиции.

### Распределение работы

- Область распараллеливания: внешний цикл по столбцам `j = 0..b.cols-1`;
- Каждый поток обрабатывает несколько столбцов;
- Число потоков задаётся через переменную окружения `OMP_NUM_THREADS`;
- Переменная цикла j является приватной для каждого потока;
- Общий доступ к матрицам A и B (только чтение) — потокобезопасен;
- Нет гонок, так как каждый поток пишет в разные позиции результирующих массивов;

### Выбор schedule

```cpp
#pragma omp parallel for default(none) shared(a, b, col_start, epsilon) schedule(dynamic, 8)
```

Используется `schedule(dynamic, 8)` по следующим причинам:

- Разреженные матрицы имеют неравномерное распределение ненулевых элементов;
- Время обработки разных столбцов может сильно различаться (от 1 до сотен операций);
- Динамическое планирование с `chunk size = 8` обеспечивает лучшую балансировку нагрузки;

### Память и синхронизация

В распараллеленной версии отсутствуют:

- Критические секции;
- Атомарные операции;
- Явные барьеры (неявный барьер в конце parallel for).
Это достигнуто за счёт:
- Независимой обработки столбцов;
- Двухпроходного алгоритма;
- Раздельной записи в разные сегменты массивов `values` и `row_indices`;

## 5. Детали реализации

### Основные методы

`ValidationImpl()` — проверка корректности входных данных:

- Валидность структуры CCS обеих матриц
- Совместимость размерностей (`A.cols == B.rows`)

`PreProcessingImpl()` — подготовка к вычислениям:

- Создание результирующей матрицы C с размерами `A.rows × B.cols`

`RunImpl()` — выполнение умножения:

- Вызов `MultiplyMatrices(A, B)`

`PostProcessingImpl()` — завершающие действия:

- В данной реализации не требуется (возвращает true)

### Вспомогательные методы

`IsMatrixValid()` — проверка валидности CCS формата (реализация ограничений из 2):

- Размеры матрицы неотрицательны;
- Размер `col_ptrs` равен `cols + 1`;
- Количество значений равно количеству индексов строк;
- Первый указатель `col_ptrs[0] = 0`;
- Последний указатель `col_ptrs[cols] = nnz` (где `nnz(X)` — количество ненулевых элементов в матрице X);
- Указатели неубывают и неотрицательны;
- Индексы строк в допустимом диапазоне.

`ComputeColumn()` — вычисляет значения одного столбца результирующей матрицы:

```cpp
// File: kotelnikova_a_double_matr_mult/omp/src/ops_omp.cpp
std::vector<double> ComputeColumn(const SparseMatrixCCS &a, const SparseMatrixCCS &b, int col_idx) {
  std::vector<double> temp(a.rows, 0.0);

  for (int b_idx = b.col_ptrs[col_idx]; b_idx < b.col_ptrs[col_idx + 1]; ++b_idx) {
    const int k = b.row_indices[b_idx];
    const double b_val = b.values[b_idx];

    for (int a_idx = a.col_ptrs[k]; a_idx < a.col_ptrs[k + 1]; ++a_idx) {
      const int i = a.row_indices[a_idx];
      temp[i] += a.values[a_idx] * b_val;
    }
  }

  return temp;
}
```

`CountNonZero()` — подсчитывает количество ненулевых элементов в столбце с заданной погрешностью:

```cpp
// File: kotelnikova_a_double_matr_mult/omp/src/ops_omp.cpp
int CountNonZero(const std::vector<double> &column, double epsilon) {
  int count = 0;
  for (double val : column) {
    if (std::abs(val) > epsilon) {
      ++count;
    }
  }
  return count;
}
```

`FillColumn()` — заполняет результирующие массивы значениями и индексами строк для одного столбца:

```cpp
// File: kotelnikova_a_double_matr_mult/omp/src/ops_omp.cpp
void FillColumn(const std::vector<double> &column, double epsilon, 
                std::vector<int> &row_indices, std::vector<double> &values, int start_pos) {
  int pos = start_pos;
  for (size_t i = 0; i < column.size(); ++i) {
    if (std::abs(column[i]) > epsilon) {
      row_indices[pos] = static_cast<int>(i);
      values[pos] = column[i];
      ++pos;
    }
  }
}
```

`MultiplyMatrices()` — основной алгоритм с OpenMP распараллеливанием:

- Реализует двухпроходную схему (описанную в разделе 4);
- Использует schedule(dynamic, 8) для балансировки нагрузки;
- Первый проход: параллельное вычисление количества ненулевых элементов для каждого столбца;
- Второй проход: параллельное заполнение результирующих массивов.

```cpp
// File: kotelnikova_a_double_matr_mult/omp/src/ops_omp.cpp
SparseMatrixCCS KotelnikovaATaskOMP::MultiplyMatrices(const SparseMatrixCCS &a, const SparseMatrixCCS &b) {
  SparseMatrixCCS result(a.rows, b.cols);

  const double epsilon = 1e-10;
  std::vector<int> col_start(b.cols, 0);

#pragma omp parallel for default(none) shared(a, b, col_start, epsilon) schedule(dynamic, 8)
  for (int j = 0; j < b.cols; ++j) {
    std::vector<double> column = ComputeColumn(a, b, j);
    col_start[j] = CountNonZero(column, epsilon);
  }

  std::vector<int> col_ptr(b.cols + 1, 0);
  for (int j = 0; j < b.cols; ++j) {
    col_ptr[j + 1] = col_ptr[j] + col_start[j];
  }

  const int total_nnz = col_ptr[b.cols];
  result.values.resize(total_nnz);
  result.row_indices.resize(total_nnz);
  result.col_ptrs = col_ptr;

#pragma omp parallel for default(none) shared(a, b, result, col_ptr, epsilon) schedule(dynamic, 8)
  for (int j = 0; j < b.cols; ++j) {
    std::vector<double> column = ComputeColumn(a, b, j);
    FillColumn(column, epsilon, result.row_indices, result.values, col_ptr[j]);
  }

  return result;
}
```

## 9. Выводы

OpenMP-версия алгоритма умножения разреженных матриц:

1. Полностью корректна — результаты совпадают с корректной последовательной версией для всех
функциональных тестов,
включая граничные случаи (нулевые матрицы, единичные матрицы, прямоугольные матрицы);
2. Даёт сверхлинейное ускорение для данной задачи — на матрицах размером 700×700 с плотностью
10% достигается ускорение до 10.45-11.47× на 4 потоках.
3. Причины сверхлинейного ускорения:

- Эффект кэша — при параллельной обработке данные активнее используют кэш-память каждого ядра;
- Уменьшение накладных расходов памяти — временные векторы распределяются по потокам, снижая
общее количество обращений к памяти;
- Лучшая локальность данных — каждый поток работает с непрерывными диапазонами столбцов;
- Неравномерность разреженности — динамическое планирование (schedule(dynamic, 8)) эффективно
распределяет нагрузку.
