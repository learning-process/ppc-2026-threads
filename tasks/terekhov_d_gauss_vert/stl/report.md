## 1. Контекст

Данная реализация использует стандартную библиотеку потоков C++ (`std::thread`) для ручного
управления параллелизмом. Подход позволяет исключить зависимости от сторонних библиотек
(OpenMP, TBB) и обеспечивает полный контроль над жизненным циклом потоков.

## 2. Постановка задачи

Фильтрация изображения размером 10 млн пикселей. Сравнение производится с последовательной
версией (SEQ), время выполнения которой составляет **0.0809 сек**.

## 3. Базовый алгоритм

Алгоритм соответствует последовательной схеме: создание дополненного изображения с зеркальными
границами и применение маски Гаусса. Обработка изображения разделяется на вертикальные полосы,
количество которых определяется числом доступных аппаратных потоков.

## 4. Схема распараллеливания

- **Декомпозиция**: Изображение разбивается на `num_bands`, значение которого
определяется через `ppc::util::GetNumThreads()`.
- **Управление потоками**: Создаётся вектор `std::vector<std::thread>`,
в который помещаются объекты потоков, выполняющие лямбда-функцию с вызовом `ProcessBand`.
- **Синхронизация**: Используется стратегия `fork-join`. Основной
поток дожидается завершения всех рабочих потоков с помощью вызова `join()` в отдельном цикле.
- **Безопасность данных**: Потокобезопасность обеспечивается логическим разделением
выходного массива данных на независимые вертикальные полосы. Каждый поток записывает результат
в строго определённый диапазон индексов, что предотвращает состояние гонки.

## 5. Детали реализации

Файлы: `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`
Основные компоненты:

- `PreProcessingImpl`: Формирование зеркальной рамки (padding).
- `RunImpl`: Вызывает `SolveSTL`, где происходит создание и запуск потоков.
- `PostProcessingImpl`: Проверка выходного контейнера.

Листинг ключевого фрагмента:

```cpp
for (int band = 0; band < num_bands; ++band) {
  threads.emplace_back([&output, &padded_image, padded_width, width, height, band, band_width, num_bands]() {
    ProcessBand(output, padded_image, padded_width, width, height, band, band_width, num_bands);
  });
}

for (auto &thread : threads) {
  thread.join();
}
```

## 6. Проверка корректности

Корректность работы подтверждена функциональными тестами `GaussFilterTestsSTL`.
 Результаты обработки идентичны последовательному эталону на всех тестовых наборах данных.

## 9. Выводы

Реализация на базе `std::thread` показала результаты, схожие с TBB. Основное время выполнения на
одном потоке близко к SEQ-версии, однако масштабируемость отсутствует. Рост времени при
увеличении числа потоков обусловлен накладными расходами на создание и уничтожение объектов
потоков ОС, а также возможным негативным влиянием эффекта ложного разделения кэш-линий
(false sharing) на границах вертикальных полос при интенсивной записи в общую память.
