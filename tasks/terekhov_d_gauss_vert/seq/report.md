## 1. Контекст

Последовательная версия реализует базовый алгоритм свёртки изображения с
использованием ядра Гаусса. Данная реализация является эталоном (baseline)
для проверки корректности параллельных версий и расчёта ускорения.

## 2. Постановка задачи

- **Вход**: Объект `Image` (ширина, высота, одномерный вектор `int`).
- **Выход**: Объект `Image` того же размера с применённым фильтром.
- **Ограничения**: Ядро фильтра фиксировано (3x3). При обработке границ
используется зеркальное отражение пикселей (padding).
- **Критерий корректности**: Значение каждого пикселя результата должно
соответствовать взвешенной сумме соседних пикселей в окне 3x3 с округлением до ближайшего целого.

## 3. Базовый алгоритм

1. **Зеркалирование (Padding)**: Создаётся временный буфер размером $(W+2) \times (H+2)$,
куда копируется изображение с отражением крайних пикселей. Это позволяет унифицировать обработку границ.
2. **Свёртка**: Для каждого пикселя исходного изображения вычисляется значение:
$Result(x,y) = \sum_{i=-1}^{1} \sum_{j=-1}^{1} Input(x+i, y+j) \times Kernel(i+1, j+1)$
3. **Вертикальное разбиение**: Логика разделена на «полосы» (bands), что подготавливает структуру
кода к параллельной обработке.

Асимптотика по времени: $O(W \times H)$.
Асимптотика по памяти: $O(W \times H)$ для хранения padded-копии.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

- `ValidationImpl`: Проверяет, что $W, H > 0$ и размер вектора `data` равен $W \times H$.
- `PreProcessingImpl`: Выделяет память под выход и формирует `padded_image_` с зеркалированием.
- `RunImpl`: Запускает `ProcessBands`, который последовательно обрабатывает 4 вертикальные полосы изображения.
