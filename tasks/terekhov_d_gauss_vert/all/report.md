## 1. Контекст

Данная реализация использует гибридную модель параллелизма, сочетающую межпроцессное взаимодействие (MPI)
и многопоточность внутри процесса (OpenMP). Такой подход позволяет эффективно масштабировать вычисления
как на распределённых системах, так и на многоядерных узлах.

## 2. Постановка задачи

Фильтрация изображения размером 10 млн пикселей. Результаты гибридной версии сравниваются с последовательным
эталоном (SEQ) — **0.0809 сек**.

## 3. Базовый алгоритм

Алгоритм адаптирован для распределённых вычислений. Вместо вертикальных полос на уровне MPI используется
горизонтальное разделение изображения (по строкам). Каждый процесс отвечает за свой блок строк, формирует
для него локальный padded-буфер и применяет фильтр.

## 4. Межпроцессная схема (MPI)

- **Роли**: Все процессы (ранги) участвуют в вычислениях.
- **Распределение данных**: Исходное изображение делится между процессами по строкам. Каждый процесс
вычисляет свои индексы `start_row` и `end_row`.
- **Коммуникации**: Используется `MPI_Allgatherv` для сбора локально обработанных фрагментов изображения
со всех узлов в единый выходной массив. Это гарантирует, что каждый процесс в итоге обладает полным результатом.
- **Синхронизация**: Межпроцессная синхронизация осуществляется неявно в момент коллективного сбора данных.

## 5. Внутрипроцессная схема (OpenMP)

- **Локальный параллелизм**: Внутри каждого MPI-процесса цикл обработки строк распараллелен с помощью OpenMP.
- **Директива**: `#pragma omp parallel for schedule(static)`. Позволяет задействовать все доступные ядра
процессора в рамках одного вычислительного узла (ранга).

## 6. Детали реализации

Файлы: `all/include/ops_all.hpp`, `all/src/ops_all.cpp`
Ключевые функции:

- `FillLocalPadded`: Формирует локальный буфер с учётом необходимых граничных строк для свертки.
- `GatherResults`: Инкапсулирует логику `MPI_Allgather` и `MPI_Allgatherv` для объединения данных.

Листинг ключевого фрагмента (OpenMP внутри MPI):

```cpp
#pragma omp parallel for schedule(static) default(none) \
    shared(local_result, local_padded, padded_width, width, local_height) shared(kGaussKernel)
for (int row = 0; row < local_height; ++row) {
    // Вычисление свертки для локальной строки
}
```

## 7. Проверка корректности

Корректность подтверждена тестами `GaussFilterTestsALL`. Проверяется согласованность данных
между рангами и отсутствие искажений на границах зон ответственности процессов.
