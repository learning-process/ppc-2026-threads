## 1. Контекст

Версия на Intel TBB разрабатывалась с целью применить высокоуровневые параллельные примитивы
библиотеки — `parallel_reduce`, `parallel_for`, `parallel_sort` — к фазам алгоритма Грэхема.
В отличие от OMP, TBB предоставляет параллельную сортировку, что позволяет распараллелить
доминирующую `O(n log n)` фазу. Базовый эталон — последовательная реализация (`≈ 0.510 с`).

---

## 2. Постановка задачи

Та же, что и для SEQ: по множеству `n` точек на плоскости построить выпуклую оболочку методом Грэхема.
В тестах производительности используется сетка `1500 × 1500 = 2 250 000` точек.

---

## 3. Базовый алгоритм

Алгоритм идентичен последовательной версии и состоит из пяти этапов:

1. **RemoveDuplicates** — дедупликация точек с допуском `kKEps`: `O(n log n)`.
2. **FindAnchorIndex** — линейный поиск точки с минимальной `y` (при равенстве — минимальной `x`): `O(n)`.
3. **CollectAndSortIndices** — сборка индексов и сортировка по полярному углу от опорной точки: `O(n log n)`.
4. **AllCollinearWithAnchor** — проверка вырожденного случая: `O(n)`.
5. **BuildHullFromSorted** — построение оболочки стеком: `O(n)`.

---

## 4. Схема распараллеливания

### 4.1. Поиск опорной точки (`FindAnchorIndex`)

Реализован через `tbb::parallel_reduce` — шаблон «разделяй и властвуй» с последующим слиянием локальных результатов.
Вспомогательная функция `IsLowerAnchor` определяет, является ли точка `lhs` «более нижней» чем `rhs`.

```cpp
return tbb::parallel_reduce(
    tbb::blocked_range<size_t>(1, xs.size()),
    size_t{0},
    [&xs, &ys](const tbb::blocked_range<size_t> &range, size_t local_idx) {
        for (size_t i = range.begin(); i < range.end(); ++i) {
            if (IsLowerAnchor(xs, ys, i, local_idx)) local_idx = i;
        }
        return local_idx;
    },
    [&xs, &ys](size_t left, size_t right) {
        return IsLowerAnchor(xs, ys, left, right) ? left : right;
    });
```

Второй лямбда-аргумент — функция слияния двух локальных результатов.
Такая схема безопасна по данным: каждый поток работает только со своим `local_idx`.

### 4.2. Заполнение массива индексов (`CollectAndSortIndices`)

Инициализация вектора `idxs` выполняется через `tbb::parallel_for`:

```cpp
tbb::parallel_for(
    tbb::blocked_range<size_t>(0, xs.size()),
    [&idxs, anchor_idx](const tbb::blocked_range<size_t> &r) {
        for (size_t i = r.begin(); i < r.end(); ++i) {
            if (i < anchor_idx)      idxs[i]     = i;
            else if (i > anchor_idx) idxs[i - 1] = i;
        }
    });
```

После заполнения применяется `tbb::parallel_sort` — параллельная сортировка слиянием, встроенная в TBB.
Сортировка по полярному углу выполняется параллельно:

```cpp
tbb::parallel_sort(idxs.begin(), idxs.end(),
    [&xs, &ys, anchor_idx](size_t lhs, size_t rhs) {
        const double cross = CrossVal(xs, ys, anchor_idx, lhs, rhs);
        if (std::abs(cross) < kKEps)
            return Dist2(xs, ys, anchor_idx, lhs) < Dist2(xs, ys, anchor_idx, rhs);
        return cross > 0;
    });
```

### 4.3. Проверка коллинеарности (`AllCollinearWithAnchor`)

Реализована через `tbb::parallel_reduce` с булевым аккумулятором.
Поддерживается ранняя остановка внутри диапазона: если локальный флаг `local_ok` становится `false`, цикл диапазона прерывается.

```cpp
return tbb::parallel_reduce(
    tbb::blocked_range<size_t>(1, sorted_idxs.size()),
    true,
    [&](const tbb::blocked_range<size_t> &r, bool local_ok) {
        for (size_t i = r.begin(); i < r.end() && local_ok; ++i) {
            if (std::abs(CrossVal(xs, ys, anchor_idx,
                                  sorted_idxs[0], sorted_idxs[i])) > kKEps)
                local_ok = false;
        }
        return local_ok;
    },
    [](bool lhs, bool rhs) { return lhs && rhs; });
```

### 4.4. Последовательные фазы

`RemoveDuplicates` и `BuildHullFromSorted` оставлены последовательными: первая выполняется до начала
основной обработки и не является узким местом по времени в общей картине,
вторая принципиально последовательна из-за зависимости каждого шага от предыдущего состояния стека.

TBB-версия показывает наилучшую производительность среди однопроцессных
параллельных реализаций благодаря `tbb::parallel_sort`. Ускорение остаётся близким к единице: закон Амдала ограничен
