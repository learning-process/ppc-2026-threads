## 1. Контекст

Последовательная реализация алгоритма построения выпуклой оболочки методом Грэхема служит эталонной базой для сравнения
с параллельными версиями (OMP, TBB, STL, MPI+TBB). Реализация не использует никаких средств параллелизма и выполняется
в одном потоке.

---

## 2. Постановка задачи

Дано множество из `n` точек на плоскости, заданных двумя векторами `xs` и `ys`. Требуется найти минимальную выпуклую
оболочку — наименьший выпуклый многоугольник, содержащий все исходные точки.
Результат — упорядоченная последовательность вершин оболочки.

В тестах производительности используется сетка `1500 × 1500 = 2 250 000` точек. Ожидаемый ответ — четыре угловые точки прямоугольника.

## 3. Базовый алгоритм

Алгоритм Грэхема строит выпуклую оболочку за `O(n log n)` и состоит из следующих этапов.

### 3.1. Удаление дубликатов (`RemoveDuplicates`)

Точки объединяются в пары `(x, y)`, лексикографически сортируются и
дедуплицируются с учётом числовой погрешности `kKEps`.
Это гарантирует корректность при наличии совпадающих точек во входных данных.

```cpp
std::ranges::sort(pts, [](const auto &a, const auto &b) {
    if (std::abs(a.first - b.first) > kKEps) return a.first < b.first;
    return a.second < b.second;
});
auto new_end = std::ranges::unique(pts, [](const auto &a, const auto &b) {
    return std::abs(a.first - b.first) < kKEps &&
           std::abs(a.second - b.second) < kKEps;
});
pts.erase(new_end.begin(), pts.end());
```

### 3.2. Поиск опорной точки (`FindAnchorIndex`)

Линейным проходом `O(n)` находится точка с наименьшей координатой `y` (при равенстве — с наименьшей `x`).
Она становится начальной точкой сортировки по полярному углу.

```cpp
for (size_t i = 1; i < xs.size(); ++i) {
    if (ys[i] < ys[idx] - kKEps ||
        (std::abs(ys[i] - ys[idx]) < kKEps && xs[i] < xs[idx] - kKEps))
        idx = i;
}
```

### 3.3. Сортировка по полярному углу (`CollectAndSortIndices`)

Собираются индексы всех точек, кроме опорной. Они сортируются по полярному углу относительно опорной точки с помощью
векторного произведения (`CrossVal`). При коллинеарности — по расстоянию (`Dist2`). Сложность: `O(n log n)`.

```cpp
std::ranges::sort(idxs, [&](size_t a, size_t b) {
    double cr = CrossVal(xs, ys, anchor_idx, a, b);
    if (std::abs(cr) < kKEps)
        return Dist2(xs, ys, anchor_idx, a) < Dist2(xs, ys, anchor_idx, b);
    return cr > 0;
});
```

### 3.4. Обработка вырожденных случаев

Перед построением стека проверяются граничные ситуации:

- `n = 0` — пустой результат;
- `n < 3` — все точки возвращаются как есть;
- все точки коллинеарны — возвращаются только крайние две.

Коллинеарность проверяется функцией `AllCollinearWithAnchor` за `O(n)`.

### 3.5. Построение оболочки стеком (`BuildHullFromSorted`)

Классический алгоритм Грэхема: точки добавляются на стек по одной в порядке сортировки.
Если три последние точки не образуют левый поворот (векторное произведение ≤ `kKEps`), верхушка стека выталкивается.
Сложность: `O(n)`.

```cpp
while (stack.size() >= 2) {
    size_t q = stack.back();
    size_t p = stack[stack.size() - 2];
    double cr = CrossVal(xs, ys, p, q, cur);
    if (cr <= kKEps) stack.pop_back();
    else break;
}
stack.push_back(cur);
```

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

Вся обработка ведётся через две независимые копии векторов `xs` и `ys`
(координаты хранятся раздельно `InType = pair<vector<double>, vector<double>>`).
Точность сравнений регулируется константой `kKEps`, определённой в `common/include/common.hpp`.

## 5. Проверка корректности

Функциональные тесты (`func_tests.cpp`) покрывают:

- пустой входной набор;
- одну точку;
- две точки;
- четыре коллинеарные точки на одной прямой;
- набор с дублирующимися точками (дублирующиеся `(0,0)` в пятом тесте);
- невыпуклый многоугольник из 7 точек;
- точки с вещественными координатами, в том числе отрицательными;
- точки с координатами порядка `9×10⁹` (проверка числовой устойчивости);
- сетку с внутренними точками (ожидается 4 угловых вершины).

Для всех 10 тест-кейсов последовательная реализация выдаёт корректный результат.
