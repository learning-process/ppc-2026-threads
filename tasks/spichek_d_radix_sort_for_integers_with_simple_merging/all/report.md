## 1. Контекст

Данная реализация (ALL) представляет собой гибридную модель распараллеливания, сочетающую
межпроцессное взаимодействие (MPI) и внутрипроцессную многопоточность (oneTBB).

## 2. Постановка задачи

Задача состоит в сортировке массива `std::vector<int>` по неубыванию. Последовательная
версия (SEQ) служит эталоном корректности и базой для вычисления ускорения.

## 3. Базовый алгоритм

В основе лежит поразрядная сортировка (LSD Radix Sort). Внутри параллельных веток
применена оптимизация: основание системы счисления увеличено до 256.

## 4. Межпроцессная схема

В текущей реализации выбрана архитектура с явным выделением главного процесса:

1. **Роли рангов:** Ветвление осуществляется по `rank == 0`. Остальные в ожидании.
2. **Отсутствие обменов:** Так как вся нагрузка на 0 ранге, коммуникации не используются.
3. **Синхронизация:** В конце метода установлен общий барьер `MPI_Barrier(MPI_COMM_WORLD)`.

## 5. Внутрипроцессная схема

Вся фактическая работа выполняется внутри процесса `rank == 0` с использованием TBB:

1. Массив делится на `num_threads` независимых блоков.
2. Запускается пул потоков `tbb::parallel_for`, где каждый поток выполняет сортировку.
3. Отсортированные блоки последовательно сливаются в единый массив.

## 6. Детали реализации

```cpp
int rank = 0;
MPI_Comm_rank(MPI_COMM_WORLD, &rank);
if (rank == 0 && !GetOutput().empty()) {
  int num_threads = ppc::util::GetNumThreads();
  std::vector<std::vector<int>> local_data = SplitData(GetOutput(), num_threads);
  tbb::parallel_for(0, num_threads, [&](int i) { RadixSort(local_data[i]); });
  GetOutput() = MergeData(local_data);
}
MPI_Barrier(MPI_COMM_WORLD);
```

*Пояснение к реализации:* Данный код изолирует вычисления строго на нулевом узле.
Это исключает риски состояния гонки, но означает, что мощность узлов `rank > 0`
не утилизируется, они просто блокируются на барьере до окончания сортировки.
