## 1. Контекст

Данная реализация представляет собой базовую (последовательную) версию поразрядной сортировки
для вектора целых чисел. Она служит эталоном корректности и базовой линией производительности,
относительно которой будет измеряться ускорение параллельных реализаций (OMP, TBB, STL, ALL).

## 2. Постановка задачи

Необходимо отсортировать вектор целых чисел (`std::vector<int>`). Алгоритм должен
корректно обрабатывать: положительные и отрицательные числа, дублирующиеся значения,
уже отсортированные массивы, а также пустые векторы.

Входные и выходные данные представляют собой `std::vector<int>`.

## 3. Базовый алгоритм

В качестве базового алгоритма выбрана поразрядная сортировка (LSD Radix Sort) по основанию 10
с использованием сортировки подсчетом (Counting Sort) для каждого разряда.
Пошаговое описание:

1. Поиск минимального элемента. Если присутствуют отрицательные числа, массив сдвигается
на величину модуля минимального элемента.
2. Поиск максимального элемента для определения количества разрядов.
3. Итерация по каждому десятичному разряду с выполнением устойчивой сортировки подсчетом.
4. Выполняется обратный сдвиг всех элементов для восстановления исходных значений.

Асимптотика по времени: $O(d \cdot (n + b))$, где $n$ — размер массива, $b$ — основание
системы счисления ($b=10$). Фактически алгоритм работает за линейное время $O(n)$.

## 4. Детали реализации

Ключевой фрагмент обработки отрицательных чисел в `RadixSort`:

```cpp
int min_val = *std::ranges::min_element(data);
if (min_val < 0) {
  for (auto &x : data) {
    x -= min_val; // Сдвиг значений в положительную область
  }
}
```

*Пояснение к реализации:* Сдвиг значений позволяет корректно использовать LSD Radix Sort
без риска получения отрицательных индексов. В конце выполняется обратный сдвиг на `min_val`.
