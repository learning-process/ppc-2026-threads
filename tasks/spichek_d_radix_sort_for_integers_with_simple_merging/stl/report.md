## 1. Контекст

Данная реализация использует встроенные средства стандартной библиотеки C++ (`std::thread`).
Здесь применяется полностью ручное управление жизненным циклом потоков и разбиение данных.

## 2. Постановка задачи

Задача состоит в сортировке массива `std::vector<int>` по неубыванию. Последовательная
версия (SEQ) служит эталоном корректности и базой для вычисления ускорения.

## 3. Базовый алгоритм

В основе лежит поразрядная сортировка (LSD Radix Sort). Внутри параллельных потоков
основание системы счисления увеличено с 10 до 256, что снижает количество проходов.

## 4. Схема распараллеливания

В реализации STL применяется ручная декомпозиция данных:

1. **Ручное разбиение:** Входной массив явно делится на `num_threads` независимых блоков.
2. **Запуск потоков:** Создается вектор `std::thread`. В цикле запускаются потоки.
3. **Синхронизация:** Выполняется явное ожидание завершения работы потоков (`join`).
4. **Слияние:** Главный поток объединяет отсортированные блоки.

## 5. Детали реализации

```cpp
std::vector<std::thread> threads(num_threads);
for (int i = 0; i < num_threads; i++) {
  threads[i] = std::thread([&, i]() { RadixSort(local_data[i]); });
}
for (int i = 0; i < num_threads; i++) {
  threads[i].join();
}
```

*Пояснение к реализации:* Крайне важно, что запуск потоков и вызов метода `.join()`
разнесены в два разных цикла. Если бы `join()` вызывался сразу, это привело бы к полной
сериализации. Состояний гонки не возникает, примитивы синхронизации не требуются.
