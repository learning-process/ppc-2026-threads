## 1. Контекст
STL-версия реализована для изучения низкоуровневого подхода к распараллеливанию с использованием `std::async` и `std::future`. В отличие от OpenMP и TBB, где управление потоками автоматизировано, здесь программист самостоятельно решает, как делить диапазоны столбцов, запускать асинхронные задачи и ожидать их завершения через `future.get()`.

Последовательная версия служит эталоном корректности, а OpenMP и TBB - для сравнения с автоматизированными подходами.


## 2. Постановка задачи
Даны две разреженные матрицы A и B в формате CCS (Compressed Column Storage) с комплексными элементами. Требуется вычислить их произведение C = A × B и представить результат также в формате CCS.
Условие корректности: количество столбцов A должно равняться количеству строк B

Подробная постановка уже дана в `seq/report.md`


## 3. Базовый алгоритм
Последовательная версия (SEQ) реализует следующий алгоритм:
1. Транспонирование матрицы A -> Aт (также в формате CCS). Это позволяет эффективно обходить строки A при вычислении скалярных произведений.
2. Для каждого столбца j матрицы B:
 - Для каждой строки i матрицы A:
      Вычислить скалярное произведение i-й строки A и j-го столбца B
      Если результат не ноль - сохранить в результат
3. Формирование выходной матрицы C в формате CCS.

Асимптотика:
По времени: O(nnz_A × nnz_B) в худшем случае (при плотных матрицах)
По памяти: O(nnz_A + nnz_B + nnz_C)


## 4. Схема распараллеливания
### Разбиение данных на диапазоны
Столбцы результирующей матрицы C делятся между потоками на непрерывные диапазоны. Количество потоков определяется через `std::thread::hardware_concurrency()`. Размер базовой части вычисляется как total_cols / num_threads, остаток распределяется между первыми потоками по одному столбцу.

### Что делает каждый поток
Каждый поток выполняет функцию `ProcessColumnRange()`, которая обрабатывает все столбцы из полученного диапазона. Для каждого столбца вычисляются ненулевые элементы (через `ComputeScalarProduct`), которые сохраняются в локальные векторы rows и vals, а затем перемещаются в col_rows[col] и col_vals[col].

### Где хранятся локальные результаты
Для каждого столбца j заранее созданы векторы col_rows[j] и col_vals[j] (размер total_cols). Разные потоки работают с разными индексами j, поэтому конфликтов нет.

### Как и когда выполняется join
Задачи запускаются через `std::async` с политикой `std::launch::async`, которая гарантирует выполнение в отдельном потоке. Возвращённые объекты `std::future` сохраняются в вектор. После запуска всех задач в цикле вызывается `future.get()` для ожидания завершения каждой.

### Синхронизация
Синхронизация не используется. Каждый поток работает только со своим диапазоном столбцов, и разные потоки никогда не пишут в один и тот же индекс j. Критические секции, мьютексы и атомарные операции не требуются.


## 5. Детали реализации
Файлы
`stl/include/ops_stl.hpp` - объявление класса BarkalovaMMultMatrixCcsSTL
`stl/src/ops_stl.cpp` - реализация всех методов

Количество потоков определяется через std::thread::hardware_concurrency(). Каждый поток получает непрерывный диапазон столбцов. Размер диапазона вычисляется как total_cols / num_threads, остаток распределяется между первыми потоками.
```cpp
// File: `stl/src/ops_stl.cpp`
int cols_per_thread = total_cols / static_cast<int>(num_threads);
int remainder = total_cols % static_cast<int>(num_threads);
int current_start = 0;

for (unsigned int thread_idx = 0; thread_idx < num_threads; ++thread_idx) {
    int cols_for_thread = cols_per_thread + (thread_idx < remainder ? 1 : 0);
    int start = current_start;
    int end = current_start + cols_for_thread;
    current_start = end;
    
    futures.push_back(std::async(std::launch::async, ProcessColumnRange, 
                     start, end, std::cref(at), std::cref(b),
                     std::ref(col_rows), std::ref(col_vals)));
}
```

После завершения всех потоков (вызов `future.get()` для каждой задачи) результаты из col_rows и col_vals последовательно собираются в финальные векторы.


## 6. Проверка корректности
Результаты STL-версии сравниваются с эталонной последовательной реализацией. Совпадение считается достигнутым, если совпадают размеры матриц, количество ненулевых элементов, массивы col_ptrs и row_indices, а значения элементов отличаются не более чем на eps = 10^-10

Используется 6 предопределённых тестовых случаев, покрывающих различные ситуации
Эти тесты проверяют корректность на матрицах разных размеров (включая прямоугольные), с разной разреженностью и с комплексными числами.
Все тесты пройдены успешно.
