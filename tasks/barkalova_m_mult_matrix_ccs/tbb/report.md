## 1. Контекст

Данная реализация использует библиотеку Intel oneTBB (Threading Building Blocks) для распараллеливания. TBB обеспечивает
задачно-ориентированное распараллеливание, которое автоматически распределяет работу между потоками с учётом балансировки
нагрузки и эффективного использования кэша.

## 2. Постановка задачи

Даны две разреженные матрицы A и B в формате CCS (Compressed Column Storage) с комплексными элементами. 
Требуется вычислить их произведение C = A × B и представить результат также в формате CCS.
Условие корректности: количество столбцов A должно равняться количеству строк B

Подробная постановка уже дана в `seq/report.md`

## 3. Базовый алгоритм

Последовательная версия (SEQ) реализует следующий алгоритм:

1. Транспонирование матрицы A -> Aт (также в формате CCS). Это позволяет эффективно обходить строки A при вычислении скалярных
произведений.
2. Для каждого столбца j матрицы B:

- Для каждой строки i матрицы A:
      Вычислить скалярное произведение i-й строки A и j-го столбца B
      Если результат не ноль - сохранить в результат

3. Формирование выходной матрицы C в формате CCS.

Асимптотика:
По времени: O(nnz_A × nnz_B) в худшем случае (при плотных матрицах)
По памяти: O(nnz_A + nnz_B + nnz_C)

## 4. Схема распараллеливания

Распараллелен внешний цикл по столбцам результирующей матрицы C с использованием `tbb::parallel_for`:

```cpp
// File: 
tbb::parallel_for(tbb::blocked_range<int>(0, c.cols), [&](const tbb::blocked_range<int> &range) {
        for (int j = range.begin(); j < range.end(); ++j) {
            ProcessColumn(j, at, b, col_rows[j], col_vals[j]);
        }
    });
```

Выбран `tbb::parallel_for`, который автоматически распределяет независимые итерации между потоками.

`tbb::blocked_range<int>(0, c.cols)` определяет диапазон итераций от 0 до c.cols. TBB самостоятельно
разбивает этот диапазон на блоки. Явный grainsize не указан, поэтому используется значение по умолчанию - TBB 
автоматически подбирает размер блока для баланса между параллелизмом и накладными расходами.

Лямбда-функция захватывает все необходимые данные по ссылке ([&]), что даёт доступ к at, b, col_rows, col_vals 
внутри тела цикла.

Отдельного ограничения конкуренции не применялось. TBB использует количество потоков, заданное через PPC_NUM_THREADS, и
автоматически распределяет работу между ними.

## 5. Детали реализации

Файлы
`tbb/include/ops_tbb.hpp` - объявление класса BarkalovaMMultMatrixCcsTBB
`tbb/src/ops_tbb.cpp` - реализация всех методов

Локальные аккумуляторы:
Вместо общих векторов values и row_indices, как в SEQ, в TBB-версии для каждого столбца j создаются отдельные векторы col_rows[j]
и col_vals[j]. Это позволило потокам писать результаты независимо друг от друга.
После завершения parallel_for результаты из col_rows и col_vals последовательно объединяются в финальные векторы row_indices,
values и col_ptrs.

Работа с памятью:
В `ProcessColumn()` для векторов rows и vals вызывается reserve(100). Это позволяет заранее выделить память и избежать
многократных переаллокаций при добавлении элементов. Значение 100 выбрано на основе экспериментальных данных.
После заполнения локальные векторы перемещаются в col_rows[j] и col_vals[j] через std::move, что исключает копирование данных.
При формировании выходной матрицы векторы values, row_indices и col_ptrs также присваиваются с помощью std::move.

## 6. Проверка корректности

Результаты TBB-версии сравниваются с эталонной последовательной реализацией. Совпадение считается достигнутым, 
если совпадают размеры матриц, количество ненулевых элементов, массивы col_ptrs и row_indices, а значения элементов 
отличаются не более чем на eps = 10^-10

Используется 6 предопределённых тестовых случаев, покрывающих различные ситуации
Эти тесты проверяют корректность на матрицах разных размеров (включая прямоугольные), с разной разреженностью и с комплексными
числами.
Все тесты пройдены успешно.

## 9. Выводы

TBB удобен при неравномерной нагрузке на столбцы, так как автоматически балансирует работу между потоками. 
Код получается проще, чем с OpenMP, потому что не нужно выбирать schedule.
TBB подходит для средних и больших разреженных матриц. На маленьких задачах лучше использовать последовательную версию (т.к.
затраты на создание и распределение задач превышают выигрыш).
