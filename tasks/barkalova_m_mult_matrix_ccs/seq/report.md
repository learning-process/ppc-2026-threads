## 1. Контекст

В данной работе реализовано последовательное умножение двух разреженных матриц в формате CCS (Compressed Column Storage) с комплексными элементами.
Данная версия используется в качестве эталона корректности и базового значения для оценки производительности параллельных реализаций (OMP, TBB, STL, ALL).

## 2. Постановка задачи

### Входные данные

Матрица A в формате CCS размера rows_A × cols_A
Матрица B в формате CCS размера rows_B × cols_B
(cols_A == rows_B)

### Выходные данные

Матрица C = A × B в формате CCS размера rows_A × cols_B

### Формат CCS

values - ненулевые элементы (комплексные числа)
row_indices - номера строк для каждого элемента
col_ptrs - указатели на начало каждого столбца (размер cols + 1)
nnz - количество ненулевых элементов

### Ограничения

- Количество столбцов A равно количеству строк B
- Размеры матриц - положительные целые числа
- Вектор указателей на начало каждого столбца (col_ptrs) по размеру на единицу больше количества столбцов матрицы
- Первый элемент вектора указателей (col_ptrs[0]) равен нулю (означает, что первый столбец начинается с нулевой позиции)
- Количество ненулевых элементов (nnz) совпадает с размером вектора значений (values)

Крайние случаи:

- Нулевые или отрицательные размеры матриц (отбрасываются на валидации)

## 3. Базовый алгоритм

Алгоритм умножения реализован в функции MultMatrix, которая последовательно вызывает вспомогательные функции TransponirMatr, ProcessColumn и ComputeScalarProduct.

### 1. Транспонирование матрицы A

Исходная матрица A хранится в формате CCS по столбцам. Для удобного доступа к строкам A выполняется её транспонирование.
Алгоритм транспонирования:

1. Записываются размеры: at.rows = a.cols, at.cols = a.rows, at.nnz = a.nnz
2. Если a.nnz == 0, создаётся пустая матрица At с col_ptrs размера at.cols + 1, заполненным нулями
3. Создаётся вектор row_count размера at.cols, в котором подсчитывается, сколько ненулевых элементов содержится в каждой строке A (будущем столбце At)
4. На основе row_count строится col_ptrs для At: col_ptrs[0] = 0, col_ptrs[i+1] = col_ptrs[i] + row_count[i]
5. Создаётся вектор current_pos для отслеживания текущей позиции в каждом столбце At
6. Проход по каждому столбцу A и каждому ненулевому элементу в нём: определяется строка row этого элемента, вычисляется позиция pos = at.col_ptrs[row] + current_pos[row], в At по этой позиции записываются значение элемента и номер столбца col, счётчик current_pos[row] увеличивается

### 2. Инициализация результирующей матрицы

В функции MultMatrix после получения транспонированной матрицы At:

1. Создаются пустые векторы values (значения), rows (индексы строк), col_ptrs (указатели на столбцы)
2. В col_ptrs добавляется начальный элемент 0
3. Переменная nz = 0 отслеживает общее количество ненулевых элементов
4. Задаются размеры выходной матрицы C: c.rows = a.rows, c.cols = b.cols

### 3. Вычисление столбцов результирующей матрицы

Запускается цикл по всем столбцам результата for `(int j = 0; j < c.cols; j++)`. На каждой итерации:

- Вызывается функция `ProcessColumn(at, b, j, values, rows, nz)`, которая заполняет ненулевые элементы для столбца j
- В конец вектора col_ptrs добавляется текущее значение nz, которое указывает на начало следующего столбца

### 4. Обработка одного столбца (ProcessColumn)

Функция `ProcessColumn` получает номер столбца col_idx и ссылки на накапливаемые векторы. Внутри неё запускается цикл `for (int i = 0; i < at.cols; i++)`, где at.cols равно количеству строк исходной матрицы A. Для каждого i:

- Вычисляется скалярное произведение строки A[i] и столбца B[col_idx] через вызов `ComputeScalarProduct(at, b, i, col_idx)`
- Полученное значение sum проверяется на неравенство нулю
- Если условие выполняется, sum добавляется в вектор values, индекс i - в вектор rows, а счётчик nz увеличивается на 1

### 5. Вычисление скалярного произведения (ComputeScalarProduct)

Функция `ComputeScalarProduct` получает параметры row_a (столбец в At, соответствующий строке A) и col_b (столбец в B). Её работа организована следующим образом:
ks = at.col_ptrs[row_a] - индекс начала столбца row_a в At
ls = b.col_ptrs[col_b] - индекс начала столбца col_b в B
kf = at.col_ptrs[row_a + 1] - 1 - последний индекс столбца row_a в At
lf = b.col_ptrs[col_b + 1] - 1 - последний индекс столбца col_b в B

- Запускается цикл `while ((ks <= kf) && (ls <= lf))`, который выполняет слияние двух отсортированных списков индексов:
  - Если `at.row_indices[ks] < b.row_indices[ls]`, указатель ks сдвигается вправо
  - Если `at.row_indices[ks] > b.row_indices[ls]`, указатель ls сдвигается вправо
  - Если индексы равны, соответствующие значения перемножаются и прибавляются к сумме, после чего оба указателя сдвигаются
- Цикл завершается, когда один из указателей превышает последний индекс своего столбца
- Функция возвращает накопленную сумму sum.

### 6. Формирование выходной матрицы

Выходной матрице C присваиваются заполненные векторы: c.values = values, c.row_indices = rows, c.col_ptrs = col_ptrs, устанавливается c.nnz = nz.

## 4. Детали реализации

Файлы:
`seq/include/ops_seq.hpp` - интерфейс класса BarkalovaMMultMatrixCcsSEQ
`seq/src/ops_seq.cpp` - реализация

Методы класса:
ValidationImpl       Проверка корректности входных матриц
PreProcessingImpl  Подготовка данных (пустая)
RunImpl              Основное вычисление
PostProcessingImpl  Проверка выходной матрицы

Формат хранения данных:
Для представления разреженных матриц используется формат CCS (Compressed Column Storage), который включает следующие компоненты:

- values - вектор ненулевых элементов матрицы, записанных по столбцам
- row_indices - вектор номеров строк соответствующих элементов
- col_ptrs - вектор указателей на начало каждого столбца в values (размер cols + 1)
- rows, cols - размерность матрицы
- nnz - количество ненулевых элементов

Ключевые функции: TransponirMatr, ComputeScalarProduct

Обработанные исключительные и граничные случаи:

- Нулевая матрица (nnz = 0) - при транспонировании создаётся пустая матрица с корректным col_ptrs
- Несовпадение размеров A.cols != B.rows - отклоняется на валидации
- Отрицательные или нулевые размеры матриц - отклоняются на валидации
- Некорректный формат CCS (неверный размер col_ptrs, col_ptrs[0] != 0, несоответствие nnz и values.size()) - отклоняется на валидации
- Нулевой результат скалярного произведения - элемент не включается в выходную матрицу (eps = 10^-10)

## 5. Проверка корректности

Результат сравнивается с эталонной матрицей по размерам (rows, cols), количеству ненулевых элементов (nnz), векторам col_ptrs, row_indices и значениям values с точностью eps = 10^-10.
