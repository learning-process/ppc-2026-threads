# Умножение разреженных матриц в формате CRS (комплексные числа) - TBB

**Студент:** Видерман Александра Геннадьевна, группа 3823Б1ПР2.  
**Технология:** TBB.  
**Вариант:** 6.

## 1. Контекст

Версия на oneTBB показывает, как ту же CRS-задачу можно выразить через
библиотеку задач. Здесь параллелизм строится не вручную, а через
runtime, который сам разбивает диапазон работы и балансирует нагрузку.

## 2. Постановка задачи

Вход и выход совпадают с остальными backend-ами: две CRS-матрицы
комплексных чисел нужно перемножить и вернуть CRS-результат.
Корректность проверяется сравнением с `SEQ` и общим набором
функциональных тестов.

## 3. Схема распараллеливания

Для TBB естественной единицей работы являются строки результата. Каждому
поддиапазону строк соответствует отдельная задача `parallel_for`, а
промежуточные результаты складываются в локальные буферы. Такая схема
хорошо соответствует задаче, потому что строки независимы и не требуют
блокировок между собой.

oneTBB удобен тем, что снимает с кода часть ручной работы по управлению
потоками. Вместо явного `join` и ручного разбиения диапазона runtime сам
подбирает распределение чанков между worker-ами.

## 4. Детали реализации

В реализации используется `oneapi::tbb::parallel_for` по диапазону
строк. Для каждого чанка создаётся локальное накопление, после чего
ненулевые значения фильтруются по `EPSILON` и собираются в CRS-результат.
Это сохраняет ту же математическую схему, что и в `SEQ`, но переносит
её исполнение на task-based runtime.

## 5. Проверка корректности

Функциональные тесты TBB-версии совпадают с общим набором: проверяются
некорректные размеры, пустые и граничные матрицы, корректность
CRS-структуры, отсутствие явных нулей и совпадение с `SEQ`. Тесты
проходят без ошибок.

## 6. Экспериментальная среда

Замеры выполнялись на Intel(R) Core(TM) i5-9300H CPU @ 2.40GHz,
16 GB RAM, Windows 10, в `Release`-сборке Visual Studio Community 2022
Release - amd64. Для сравнения использовалась серия из 5 внешних
запусков `BandMatrixPerf` при `PPC_NUM_THREADS=4`.

## 7. Результаты

На `BandMatrixPerf` получена серия:

| Серия из 5 запусков, s | Медиана, s | Speedup относительно `SEQ` |
| --- | ---: | ---: |
| 0.00023368, 0.00025544, 0.00028702, 0.00029672, 0.00024854 | 0.00025544 | 0.49 |

Из параллельных single-node вариантов TBB показывает лучший результат
на этой задаче. Тем не менее он всё равно уступает последовательному
базовому варианту, потому что полезной работы на одну строку
недостаточно, чтобы полностью окупить overhead runtime.

## 8. Выводы

TBB-версия получилась самой удачной среди параллельных single-node
backend-ов. Она сохраняет ясную структуру алгоритма и при этом даёт
более мягкое управление задачами, чем ручные потоки или более тяжёлые
гибридные схемы.
