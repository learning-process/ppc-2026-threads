## 1. Контекст

Алгоритм четно-нечетного слияния Бэтчера, комбинированный с поразрядной сортировкой образует мощный конвейер.
Последовательная версия служит эталоном корректности и задает базовое время, относительно которого
будет измеряться ускорение параллельных реализаций.

## 2. Постановка задачи

**Вход:** кортеж `std::tuple<int, std::vector<int>>`, где `int` — размер массива, а вектор — сами элементы.
**Выход:** отсортированный по возрастанию `std::vector<int>`.
**Ограничения:** сеть сортировки Бэтчера строго требует, чтобы размер массива был степенью двойки.
Для произвольного `N` требуется дополнение массива фиктивными элементами (padding).

## 3. Базовый алгоритм

Последовательный алгоритм состоит из следующих этапов:

1. **Препроцессинг:** нахождение минимального элемента (сдвига) для работы с отрицательными числами,
приведение всех элементов к неотрицательным. Вычисление ближайшей степени двойки `power_`
и дополнение массива фиктивными элементами `INT_MAX`.
2. **Сортировка:** массив условно делится пополам. К каждой половине применяется
поразрядная сортировка с основанием 256.
3. **Слияние:** применяется сеть четно-нечетного слияния Бэтчера `OddEvenMerge(...)` для полного массива.
4. **Постпроцессинг:** обрезка фиктивных элементов до исходного размера `size_` и обратный сдвиг значений.

Асимптотика по времени: поразрядная сортировка работает за линейное время,
а слияние Бэтчера работает за $O(N \log^2 N)$ операций.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

```cpp
bool ShemetovDRadixOddEvenMergeSortSEQ::RunImpl() {
  if (power_ <= 1) return true;

  size_t middle = power_ / 2;
  RadixSort(array_, 0, middle - 1);
  RadixSort(array_, middle, power_ - 1);

  OddEvenMerge(array_, 0, power_);
  return true;
}
```

В функции RunImpl происходит четкое разделение массива на две независимые половины перед поразрядной сортировкой,
т.о. мы формируем основу для будущей декомпозиции в параллельных версиях.
Функция `PostProcessingImpl(...)` отсекает фиктивные элементы и возвращает исходный сдвиг.

## 5. Проверка корректности

SEQ-версия успешно проходит все функциональные тесты, включая массивы с отрицательными числами,
обратно отсортированные массивы, неотсортированные массивы и массивы не кратные степени двойки.
Итоговая валидация `std::ranges::is_sorted` подтверждает правильность работы алгоритма..
