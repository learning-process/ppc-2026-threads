## 1. Контекст

Технология `Intel Threading Building Blocks (TBB)` переводит алгоритм на задачно-ориентированную модель,
где диспетчеризацией пула потоков и балансировкой нагрузки (work-stealing) занимается внутренний runtime.

## 2. Схема распараллеливания

1. **Поразрядная сортировка:** Вместо ручного разбиения на множество чанков (как в OMP),
TBB-реализация использует `tbb::parallel_invoke`, параллельно запуская `RadixSort` для левой и правой половин массива.
2. **Слияние Бэтчера:** Вложенные циклы вынесены в функции `ApplyFirstPass(...)` и `ApplyMainPass(...)`.
Обе функции используют `tbb::parallel_for` поверх `tbb::blocked_range<size_t>`. TBB самостоятельно решает,
как нарезать диапазон `num_blocks` на задачи.

## 3. Детали реализации

Файлы: `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`

```cpp
  size_t step = padding * 2;
  size_t num_blocks = (segment - padding) / step;

  tbb::parallel_for(tbb::blocked_range<size_t>(0, num_blocks), [&](const tbb::blocked_range<size_t> &range) {
    for (size_t block = range.begin(); block != range.end(); ++block) {
      // Логика сети Бэтчера
    }
  });
}
```

Такой подход избавляет код от жесткой привязки к количеству потоков и позволяет планировщику TBB оптимально загружать ядра.

## 4. Проверка корректности

Отсутствие гонок обеспечивается тем, что при каждом шаге `ApplyMainPass(...)` разные блоки обращаются к строго
непересекающимся индексам массива. Функциональные тесты пройдены успешно.
