# Поразрядная сортировка для целых чисел с четно-нечетным слиянием Бэтчера (ALL)

- Студент: Шеметов Даниил Олегович

- Технология: ALL (MPI + OpenMP)
- Вариант: 18

## 1. Контекст

Гибридная `ALL` версия демонстрирует возможность масштабирования алгоритма на распределенные вычислительные системы
(кластеры), объединяя межпроцессный обмен `MPI` с внутриузловым параллелизмом OpenMP.

## 2. Межпроцессная схема

1. **Вычисление конфигурации:** Определяется количество MPI-рангов `ranks`, которая тоже является степенью двойки.
Высчитывается размер чанка: `chunk = power_ / ranks`.
2. **Scatter:** Нулевой процесс рассылает чанки по всем процессам с помощью `MPI_Send`,
локальные узлы получают их через `MPI_Recv`.
3. **Обработка:** Каждый ранг локально вызывает `MPISort(...)` для своего фрагмента.
4. **Gather:** Отсортированные чанки возвращаются на нулевой ранг.
5. **Глобальное слияние:** Нулевой ранг вызывает `MPIMerge(...)` (сеть Бэтчера)
для объединения всех чанков в единый отсортированный массив.
6. **Широковещание:** Готовый результат транслируется всем рангам через `MPI_Bcast`.

## 3. Внутрипроцессная схема

Внутри каждого узла функции `MPISort(...)` и `MPIMerge(...)` используют `#pragma omp parallel` и `#pragma omp for`.
Это позволяет узлу полностью утилизировать свои локальные процессорные ядра без дополнительных сетевых издержек.

## 4. Детали реализации

Файлы: `all/include/ops_all.hpp`, `all/src/ops_all.cpp`

Реализация `ScatterData(...)` и `GatherData(...)` использует ветвление по переменной `mpi_init`,
позволяя алгоритму корректно деградировать до последовательного OpenMP, если среда `MPI` не была инициализирована.

```c++
  if (rank < ranks) {
    size_t chunk = power_ / ranks;
    std::vector<int> segment_array(chunk);

    ScatterData(array_, segment_array, chunk, rank, ranks, mpi_init);

    MPISort(segment_array, chunk);

    GatherData(array_, segment_array, chunk, rank, ranks, mpi_init);

    if (rank == 0) {
      MPIMerge(chunk);
    }
  }
```

## 5. Проверка корректности

Вызов `MPI_Bcast` в конце играет роль важнейшей точки синхронизации: он гарантирует, что нулевой ранг транслирует
итоговый отсортированный массив всем остальным процессам. Благодаря этому `PostProcessingImpl(...)` на всех узлах
работает с согласованным массивом, и каждый ранг может успешно пройти проверку `std::ranges::is_sorted`.
