## 1. Контекст
Решается задача повышения контраста полутонового (grayscale) изображения.
Метод – линейная растяжка гистограммы (contrast stretching): преобразование исходных яркостей пикселей так, чтобы они заняли весь доступный диапазон [0, 255].
Последовательная версия нужна как эталон для сравнения с параллельными реализациями (OpenMP, TBB, MPI, STL), чтобы оценить ускорение и эффективность распараллеливания.

## 2. Постановка задачи

#### Входные данные
- Вектор `std::vector<uint8_t>` – пиксели полутонового изображения (значения 0…255).
- Размер вектора = ширина × высота изображения.

#### Выходные данные
- Вектор `std::vector<uint8_t>` той же длины – обработанное изображение.

#### Ограничения
- Только полутоновые изображения (один канал).
- Значения пикселей – целые беззнаковые 8 бит.

#### Крайние случаи
- Пустой входной вектор.
- Однородное изображение.
- Уже полностью контрастное изображение (`min == 0`, `max == 255`) – выход должен совпадать со входом.

## 3. Базовый алгоритм
### 3.1. Пошаговое описание
1. Найти минимальное (`min_val`) и максимальное (`max_val`) значение пикселя во входном векторе.
2. Если `min_val == max_val` – изображение однородное. Скопировать входные данные в выходной вектор.
3. Иначе для каждого пикселя p вычислить новое значение по формуле:
   - $output[i]=\frac{(input[i] - min_I) * (255 - 0)}{max_I-min_I}$, где
     - $output[i]$ – новое значение яркости пикселя после растяжения;
     - $input[i]$ – исходное значение яркости текущего пикселя исходного изображения;
     - $min_I$ – минимальное значение яркости во всём исходном изображении;
     - $max_I$ – максимальное значение яркости во всём исходном изображении.
4. Результат привести к диапазону [0, 255] (формула уже даёт значения в этом диапазоне, но для страховки используется `std::clamp` для защиты от ошибок округления).
5. Записать результат в выходной вектор.

### 3.2. Асимптотика по времени
- Один проход для поиска min/max (`≈1.5*(N − 1)` сравнений).
- Один проход для преобразования (`n` вычислений нового пикселя).

### 3.3. Асимптотика по памяти
- `O(n)` – выделяется выходной вектор того же размера.

### 3.4. Инварианты
- Значения выходного вектора всегда лежат в [0, 255].
- Для однородного изображения выход = вход.
- Для неоднородного изображения минимум выходного вектора = 0, максимум = 255.

### 3.5. Критерий корректности
- Если исходное изображение однородно, то выходное изображение совпадает со входным.
- Иначе выходное изображение имеет `min_out == 0`, `max_out == 255`.

## 4. Детали реализации
### 4.1. Файлы
- `seq/include/ops_seq.hpp` – объявление класса `OtcheskovSContrastLinStretchSEQ`.
- `seq/src/ops_seq.cpp` – реализация методов.

### 4.2. Этапы задачи (методы базового класса)
- Валидация (`ValidationImpl`) – проверка на пустоту входных данных.
- Препроцессинг (`PreProcessingImpl`) – подготовка выходного массива, путём выделения памяти размером, равным размеру входного вектора.
- Вычисления (`RunImpl`) – последовательный обход элементов массива с применением формулы линейного растяжения гистограммы.
- Постпроцессинг (`PostProcessingImpl`) – не используется.

### 4.3. Детали вычисления (`RunImpl`)
1. Поиск минимума и максимума – `std::ranges::minmax_element`.
2. Проверка на однородность.
3. Преобразование в цикле:
    ```cpp
    int pixel = static_cast<int>(input[i]);
    int value = (pixel - min_i) * 255 / (range);
    value = std::clamp(value, 0, 255);
    output[i] = static_cast<uint8_t>(value);
    ```

### 4.4. Исключительные и граничные случаи
- **Пустое изображение** – `ValidationImpl` возвращает `false`, задача не выполняется (функция возвращает `false`).

- **Однородное изображение** – в `RunImpl` после нахождения min/max проверяется их равенство. Если равны, выполняется присваивание векторов `output = input`.

- **Диапазон значений** – целочисленное деление и `std::clamp` гарантируют результат в [0,255].

## 5. Проверка корректности

### 5.1. Метод сравнения
Результат работы последовательной версии считается корректным, если выходное изображение удовлетворяет условию, заданному функцией `CheckRange`:
```cpp
bool CheckRange(const OutType &data) {
  if (data.empty()) return false;
  auto [min_it, max_it] = std::ranges::minmax_element(data);
  return (*min_it == 0 && *max_it == 255) || (*min_it == *max_it);
}
```
То есть изображение либо **полностью растянуто** (минимальное значение 0, максимальное 255), либо **однородно** (все пиксели равны между собой).
Валидационный тест дополнительно проверяет, что при пустом входе выходной вектор пуст.

#### Запуск тестов:
```bash
./bin/ppc_func_tests --gtest_filter="*Otcheskov*seq*"
```

### 5.2. Тесты
| Категория      | Входные данные | Ожидаемый выход (проверка с помощью `CheckRange`) |
| -------------- | -------------- | ------------------------------------ |
| Валидационный  | Пустой вектор  | Выходной вектор пуст (`data.empty()`)|
| Функциональные | Синтетические изображения разного размера с низкой контрастностью (значения от 100 до 150) | Повышение контраста (`min_out == 0` и `max_out == 255`) |
| Однородное изображение | Изображение, заполненное нулями | Выходное изображение однородно (`min_out == max_out`) |
| Реальное изображение   | Файл `data/grayimg.jpg`, загружаемый через `stbi_load` | Повышение контраста (`min_out == 0` и `max_out == 255`) |

#### Характерные примеры входа и ожидаемого выхода:
1. Функциональный тест (2x2)
   - Вход: `[10, 50, 100, 200]`.
   - Выход: `[0, 53, 120, 255]`.
2. Однородное изображение
   - Вход: все пиксели = 0.
   - Выход: все пиксели = 0, т.е. однородное (min==max).
3. Реальное изображение
   - Вход: изображение `data/grayimg.jpg` с исходным диапазоном, не покрывающим весь [0,255].
   - Выход: после обработки минимальное значение становится 0, максимальное – 255.

Все тесты подтверждают, что последовательная реализация корректно выполняет линейную растяжку гистограммы.
