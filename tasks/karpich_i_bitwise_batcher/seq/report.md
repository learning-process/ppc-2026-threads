## 1. Контекст

Поразрядная сортировка является алгоритмом сортировки без сравнений, который
позволяет сортировать числа за линейное время, что выгодно отличает его от традиционных
сортировок со временем выполнения `O(N log N)`. Последовательная реализация необходима
для установления базовой линии производительности и проверки корректности
всех последующих параллельных версий алгоритма.

## 2. Постановка задачи

Задан большой вектор целых 32-битных знаковых чисел (обозначение его длины как `N`).
Требуется отсортировать этот вектор по возрастанию.
Входные и выходные данные представлены массивами (векторами).
Для сортировки элементов должен применяться алгоритм поразрядной сортировки,
проходящий по каждому байту числа от младшего к старшему.
В качестве устойчивой сортировки внутри должна использоваться сортировка подсчетом.

## 3. Базовый алгоритм

Последовательный алгоритм представляет собой классическую поразрядную сортировку целых чисел по
основанию 256. Так как размер 32-битного числа составляет 4 байта, алгоритм включает 4 итерации:

1. Заводится буферный массив такого же размера для промежуточной записи результата.
2. Выполняется цикл по байтам: от 0 до 3.
3. На каждой итерации:
    - Создается массив счетчиков частот (размера 256), заполняемый нулями.
    - Выполняется проход по исходному массиву; для каждого числа вычисляется значение текущего
      байта, и соответствующий счетчик инкрементируется.
    - Массив частот преобразуется в массив префиксных сумм, чтобы индекс счетчика указывал на
      стартовую позицию для вставки элементов с данным значением байта.
    - Выполняется обратный проход по исходному массиву для расстановки элементов в буферный массив.
    - На последней итерации учитывается знак числа для корректной сортировки отрицательных чисел.
    - Буферный массив становится исходным для следующей итерации.

Так как последовательная сортировка полностью сортирует весь массив, слияние по алгоритму Бэтчера
для нее не требуется.
Асимптотика по времени: `O(B * N)`, где `B` - количество байт (в данном случае `B = 4`).
Асимптотика по памяти: `O(N)` для хранения буферного массива.
Инварианты и критерий корректности: на каждом `i`-ом шаге числа отсортированы по `i` младшим
байтам; итоговый массив должен быть полностью отсортирован по возрастанию с сохранением всех
исходных элементов.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`
В рамках инфраструктуры курса переопределены базовые методы задачи:

- `ValidationImpl`: проверяет корректность входных данных.
- `PreProcessingImpl`: копирует входные данные во внутренние структуры задачи для последующей
  обработки.
- `RunImpl`: выполняет подготовку данных (дополнение массива до степени двойки), разбиение
  на две половины для сортировки и их последующее слияние Бэтчера.
- `PostProcessingImpl`: проверяет отсортированность и копирует результат.

Ниже представлен листинг основной функции поразрядной сортировки,
которая последовательно обрабатывает байты числа и распределяет их по корзинам:

```cpp
void RadixSortPositive(std::vector<int> &arr) {
  int n = static_cast<int>(arr.size());
  if (n <= 1) return;
  int max_val = *std::ranges::max_element(arr);
  if (max_val == 0) return;

  std::vector<int> buffer(n);
  for (int shift = 0; shift < 32 && (max_val >> shift) > 0; shift += 8) {
    std::vector<int> count(256, 0);
    for (int i = 0; i < n; i++) count[(arr[i] >> shift) & 0xFF]++;
    for (int i = 1; i < 256; i++) count[i] += count[i - 1];
    for (int i = n - 1; i >= 0; i--) buffer[--count[(arr[i] >> shift) & 0xFF]] = arr[i];
    arr = buffer;
  }
}
```

Здесь видно, что алгоритм устойчив за счет прохода по массиву с конца при расстановке элементов.

## 5. Проверка корректности

Корректность проверялась с помощью юнит-тестов на базе `Google Test`. Исходные массивы
инициализировались псевдослучайными числами, после чего результаты последовательной версии
сравнивались с эталонной сортировкой из стандартной библиотеки (`std::sort`).
Дополнительно проверялось сохранение корректного количества элементов.
Характерные примеры:

- Сортировка массива с отрицательными числами: `[-5, 2, -1, 0, 5]` -> `[-5, -1, 0, 2, 5]`.
- Сортировка массива одинаковых элементов: `[1, 1, 1]` -> `[1, 1, 1]`.
