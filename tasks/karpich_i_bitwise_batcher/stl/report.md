## 1. Контекст

В данном отчете рассматривается распараллеливание алгоритма с помощью стандартных потоков C++
(`std::thread`). Ручная thread-based реализация позволяет получить полный контроль над пулом
потоков, распределением задач и синхронизацией.

## 2. Постановка задачи

Задан вектор целых 32-битных знаковых чисел, который необходимо отсортировать.
Эталонной считается последовательная версия. Необходимо реализовать локальную поразрядную
сортировку блоков и слияние сетью Бэтчера с помощью `std::thread`.

## 3. Базовый алгоритм

Последовательная версия выполняет 4 побайтовых прохода по массиву с подсчетом частот.

## 4. Схема распараллеливания

- **Разбиение данных:** Массив дополняется до степени двойки, после чего делится ровно пополам.
- Создаются два системных потока `std::thread`, один из которых сортирует левую половину,
  а другой — правую. Хранение локальных результатов выполняется в стеке
  каждого потока, что исключает гонки данных.
- **Синхронизация:** Вызывается метод `join()` для обоих потоков, заставляющий главный поток
  дождаться завершения локальных сортировок.
- **Слияние Бэтчера:** Выполняется последовательно в главном потоке для слияния двух
  уже отсортированных половин. Это решение минимизирует накладные расходы на создание тысяч
  потоков и барьерную синхронизацию на каждом слое компараторов.

## 5. Детали реализации

Файлы: `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`
Ниже представлен фрагмент из `RunImpl`, демонстрирующий ручное создание и синхронизацию
потоков `std::thread` перед запуском сети Бэтчера:

```cpp
  int half = padded / 2;
  std::vector<int> left(data_.begin(), data_.begin() + half);
  std::vector<int> right(data_.begin() + half, data_.end());

  std::thread left_thread([&left]() { RadixSort(left); });
  std::thread right_thread([&right]() { RadixSort(right); });
  left_thread.join();
  right_thread.join();

  std::ranges::copy(left, data_.begin());
  std::ranges::copy(right, data_.begin() + half);

  BatcherMerge(data_, 0, padded - 1);
```

При подходе с `std::thread` мы избежали накладных расходов на создание пула потоков
внутри самой сети слияния, вынеся распараллеливание на самый высокий уровень.
