## 1. Контекст

В рамках данной задачи используется Intel Threading Building Blocks для
распараллеливания алгоритма сортировки. TBB предоставляет высокоуровневые алгоритмы
с балансировкой нагрузки и отлично подходит для задач с независимыми
итерациями.

## 2. Постановка задачи

Требуется отсортировать массив целых 32-битных знаковых чисел, используя локальную поразрядную
сортировку и глобальное слияние сетью Бэтчера. В качестве эталона выступает последовательная
версия.

## 3. Базовый алгоритм

Последовательная версия выполняет 4 побайтовых прохода сортировки над всем массивом за `O(N)`.

## 4. Схема распараллеливания

В качестве основного примитива распараллеливания выбран `tbb::parallel_for`.

- Для локальной поразрядной сортировки используется `tbb::parallel_for` по
  индексам блоков данных.
- Для применения сети компараторов на стадии слияния Бэтчера: каждый слой сети представляет
  набор независимых пар. Обход этих пар также распараллеливается через `tbb::parallel_for`.
- Диапазон работы определяется с помощью `tbb::blocked_range<int>`. Grainsize подбирается
  встроенным планировщиком `auto_partitioner`, который динамически дробит диапазоны на
  подзадачи для достижения оптимального баланса между накладными расходами и равномерной загрузкой.
- Конкуренция контролируется классом `tbb::global_control`, ограничивающим максимальное
  число потоков (`max_allowed_parallelism`).
- Синхронизация между слоями сети реализуется неявно через ожидание завершения вызова
  `tbb::parallel_for` для теку слоя перед переходом к следующему.

## 5. Детали реализации

Файлы: `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`
В реализации TBB используются высокоуровневые алгоритмы параллелизма. Вместо ручного создания
потоков используется `tbb::parallel_invoke` для параллельного запуска двух крупных независимых
подзадач (сортировка левой и правой половин), а затем применяется `tbb::parallel_for` для
применения сети компараторов Бэтчера.

Листинг параллельного обхода компараторов Бэтчера на заданном слое:

```cpp
void BatcherMergeParallel(std::vector<int> &arr, int lo, int hi) {
  auto levels = BuildMergeNetwork(lo, hi);
  for (int lvl = static_cast<int>(levels.size()) - 1; lvl >= 0; lvl--) {
    const auto &level = levels[lvl];
    int level_size = static_cast<int>(level.size());
    tbb::parallel_for(0, level_size, [&arr, &level](int idx) {
      auto [a, b] = level[idx];
      if (arr[a] > arr[b]) {
        std::swap(arr[a], arr[b]);
      }
    });
  }
}
```

`tbb::parallel_for` принимает лямбду, которая безопасно выполняет операцию `swap` над парами
элементов. Синхронизация между слоями сети `levels` происходит неявно: следующий шаг цикла
`for` по слоям не начнется, пока `tbb::parallel_for` текущего слоя полностью не завершится.

## 6. Проверка корректности

Корректность TBB-реализации проверена на различных размерах массивов, включая некратные числу
потоков и содержащие отрицательные числа. Результаты полностью совпадают с SEQ-версией.
Граничные тесты также успешно пройдены.

## 7. Экспериментальная среда

- **CPU**: Apple M2 (8 cores)
- **Compiler**: GCC 13.3.0 (g++), CMake Release

## 9. Выводы

Применение TBB удобно за счет использования `parallel_for` и `blocked_range`, которые снимают
с программиста задачу ручного распределения индексов. Тем не менее, overhead на
диспетчеризацию задач и постоянную синхронизацию в конце каждого слоя сети Бэтчера заметно
мешает масштабируемости на задачах с мелкими независимыми операциями (как `std::swap`).
