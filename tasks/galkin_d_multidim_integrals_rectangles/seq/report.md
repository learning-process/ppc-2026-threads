## 1. Контекст

Последовательная реализация задаёт эталон результата и базовое время вычисления (p = 1), относительно которых
сравниваются все параллельные реализации. Алгоритм простой и регулярный, что делает его удобным кандидатом для
распараллеливания по данным.

## 2. Постановка задачи

**Вход (`InType`):** кортеж из трёх полей:

- функция `f : ℝᵈ → ℝ`,
- вектор пар границ `{(a₀,b₀), …, (aₐ₋₁,bₐ₋₁)}` (d-мерный прямоугольник),
- целое `n > 0` — число разбиений по каждой оси.

**Выход (`OutType`):** вещественное число — численное значение интеграла.

**Ограничения (ValidationImpl):**

- `borders` не пустой;
- все `aᵢ`, `bᵢ` конечны и `aᵢ < bᵢ`;
- `func` не null, `n > 0`.

**Крайние случаи:** одномерные функции с минимальным n, разные области и функции с известным аналитическим
интегралом (полиномы, тригонометрические, экспоненциальные).

## 3. Базовый алгоритм

По каждой оси строятся `n` равных ячеек шириной `hᵢ = (bᵢ − aᵢ) / n`. Объём элементарной ячейки:
`V_cell = h₀ × h₁ × … × hₐ₋₁`. Для каждой ячейки с многомерным индексом `(i₀, …, iₐ₋₁)` вычисляется
центральная точка `xⱼ = aⱼ + (iⱼ + 0.5) × hⱼ` и значение функции в этой точке добавляется к сумме.
Итоговый интеграл: `I ≈ (sum всех f(x)) × V_cell`.

Обход ячеек реализован через вспомогательную функцию `NextIndex()`, которая инкрементирует d-мерный
индекс с младшей оси (аналог инкремента с переносом в позиционной системе).

**Временна́я сложность:** O(nᵈ) вычислений f — доминирует объём сетки.  
**Память:** O(d) — два вектора длиной d (индексы и координаты точки).  
**Инвариант:** после полного обхода `sum` содержит сумму значений f во всех центрах ячеек; итог умножается на
`V_cell`.

## 4. Детали реализации

**Файлы:** `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

- `ValidationImpl` — проверяет корректность входа;
- `PreProcessingImpl` — сбрасывает выход в 0;
- `RunImpl` — вычисляет шаги, объём ячейки, обходит все ячейки через `NextIndex`, проверяет `isfinite(fx)` на каждом шаге;
- `PostProcessingImpl` — проверяет `isfinite(result_)`.

Ключевой фрагмент — тело цикла в `RunImpl`:

```cpp
// File: seq/src/ops_seq.cpp
while (true) {
  for (std::size_t i = 0; i < dim; ++i) {
    x[i] = borders[i].first + ((static_cast<double>(idx[i]) + 0.5) * h[i]);
  }
  const double fx = func(x);
  if (!std::isfinite(fx)) { return false; }
  sum += fx;
  if (!NextIndex(idx, dim, n)) { break; }
}
GetOutput() = sum * cell_v;
```

Вектор `x` выделяется один раз до цикла и переиспользуется — без лишних аллокаций внутри итерации.

## 5. Проверка корректности

SEQ — эталон для всех параллельных реализаций: каждый функциональный тест задаёт функцию и ожидаемое
значение, результат сравнивается с допуском 1e-4.

Набор из 23 параметрических тестов (`tests/functional/main.cpp`):

| Группа | Примеры функций |
| ------ | --------------- |
| 1D | `1`, `x`, `x²`, `x³`, `sin(x)`, `exp(x)` |
| 2D | `1`, `x·y`, `x²+y`, `exp(x+y)`, `sin(x)·cos(y)` |
| 3D | `1`, `x·y·z`, `x²+y²+z²`, `sin(x)·cos(y)·exp(z)` |
| 4D | `1`, произведения координат |

Дополнительно: тесты малых интервалов (границы 1e-3, 0.01) для проверки численной точности.

## 7. Результаты

Тест производительности: 2D интеграл sin(x₀)·sin(x₁) на [0,π]², n=10 000; total_cells = 10 000² = 10⁸.

| Режим | Время, с |
| ----- | -------- |
| task\_run | 0.516 |

Время task\_run ≈ 0.516 с используется как знаменатель ускорения во всех параллельных отчётах.

Узкое место — вычисление `func(x)` в каждой из 10⁸ ячеек; оба вычисления sin занимают большую часть времени.

## 8. Выводы

SEQ — простой и полностью предсказуемый эталон. Он работает без аллокаций внутри основного цикла,
не зависит от числа потоков и служит знаменателем ускорения для всех параллельных реализаций.
