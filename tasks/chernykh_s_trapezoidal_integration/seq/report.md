## 1. Контекст

Задача состоит в вычислении многомерных интегралов с использованием метода трапеций.
В одномерном случае для функции f(x) на отрезке [a, b] с шагом h = (b - a) / n интеграл приближённо вычисляется по формуле трапеций: I = h * ((f(x_0) + f(x_n)) / 2 + (f(x_1) + f(x_2) + ... + f(x_n-1)))

Для многомерного интеграла размерности d по области D = [a_1, b_1] x [a_2, b_2] x ... x [a_d, b_d] формула обобщается на многомерную координатную сетку. 
Шаг по каждому измерению h_k = (b_k - a_k) / n_k. Формула многомерных трапеций принимает вид произведения шагов по всем осям на сумму значений функции в узлах сетки, умноженных на весовые коэффициенты каждой точки.

Где весовые коэффициенты w для каждого измерения зависят от положения узла на сетке:
- w = 1/2, если узел является граничным по данному измерению;
- w = 1, если узел находится внутри интервала.

## 2. Постановка задачи
### Входные данные:
- `limits`: Вектор пар [a_i, b_i] - границы интегрирования по каждому измерению. 
- `steps`: Вектор целых положительных чисел `n_i`, задающий количество шагов разбиения по каждому измерению.
- `func`: Функция, интеграл которой нужно посчитать, передается в виде лямбда выражения.
### Выходные данные:
Значение типа double - результат счета.

### Ограничения и крайние случаи:
- Входные вектора `limits` и `steps` должны совпадать по размеру и не должны быть пустыми.
- Входной вектор `steps` должен содержать числа строго больше нуля.

## 3. Базовый алгоритм

Алгоритм представляет собой обобщение формулы трапеций на многомерный случай. Область интегрирования разбивается на сетку узлов. В каждом узле вычисляется значение функции, умноженное на коэффициент, зависящий от того, внутренний ли это узел или граничный.

### Вычислительная фаза:
- По каждому измерению рассчитывается шаг сетки: `h = (b - a) / n`
- Совершается обход всех узлов с помощью вектора `counters`
- Для каждого узла вычисляется координата точки и весовой коэффициент `weight`
- Коэффициент `weight` инициализируется единицей и умножается на 0.5 за каждое попадание координаты узла на границу отрезка
- Значения функции, умноженные на вес, добавляются в общую сумму
- Итоговый результат умножается на произведение шагов по всем осям

### Асимптотика:
- Время: O(d * N), где d — размерность пространства, N — общее число узлов сетки
- Память: O(d) для векторов координат

### Инварианты:
- Каждый узел сетки обрабатывается строго один раз

## 4. Детали реализации

### Файлы:
- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

### Pipeline реализации:
- **ValidationImpl**: 
	- Проверка на пустые входные данные, соответствие размеров `limits` и `steps`, а также `steps[i] > 0`.
- **PreProcessingImpl**: 
	- Входные данные уже готовы.
- **RunImpl**: 
	- Выполнение последовательного цикла `while (!done)`, внутри которого на каждой итерации вызывается метод `CalculatePointAndWeight` для расчета координат и веса узла.
	- Накопление взвешенной суммы значений лямбда-функции в переменной `total_sum`.
	- Умножение полученной суммы на произведение шагов сетки по всем осям `h_prod`.
- **PostProcessingImpl**: 
	- Результат сформирован и записан в структуру вывода `GetOutput()`.

### Главный вычислительный цикл:  
``` cpp
while (!done) {
    double weight = CalculatePointAndWeight(input, counters, current_point);
    total_sum += input.func(current_point) * weight;

    for (std::size_t i = 0; i < dims; ++i) {
        if (std::cmp_less(++counters[i], input.steps[i] + 1)) {
            break;
        }
        if (std::cmp_equal(i, dims - 1)) {
            done = true;
        } else {
            counters[i] = 0;
        }
    }
}
```

### Обработанные исключительные случаи:  
Корректно обрабатываются вырожденные интервалы (когда левая граница равна правой: a = b). В этом случае шаг h становится равным 0, алгоритм выдает результат 0.0 без деления на ноль или падения программы.  
## 5. Проверка корректности

Для функциональных тестов была выбрана погрешность 10^-3, поскольку метод трапеций имеет второй порядок сходимости O(h^2). На тесте Sin_1D при количестве участков разбиения N = 1000 и границах интегрирования [0, pi] теоретическая погрешность метода вычисляется по формуле:
  Δ = ((b - a) * h^2 / 12) * max|f''(x)|
Где:
  * b - a = 3.141592 - 0 = 3.141592
  * h = 3.141592 / 1000 ≈ 0.003141592
  * f''(x) = (sin(x))'' = -sin(x), на нашем отрезке max|-sin(x)| = 1.

Подставляя данные значения в формулу, получаем:
  Δ ≈ 2.58e-6
Таким образом, погрешность самого метода на данном тесте составляет примерно 2.58 * 10^-6. 
В конечном итоге для функциональных тестов была выбрана погрешность 10^-3, чтобы они не падали, при изменении размеров сеток. Для перфоманс тестов была выбрана погрешность 10^-1, по тем же причинам.
