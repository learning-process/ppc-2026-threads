## 1. Контекст

В данной работе реализован параллельный алгоритм вычисления многомерных
интегралов методом Монте-Карлос использованием гибридной технологии ALL,
объединяющей MPI и OpenMP.

MPI применяется для распределения вычислений между процессами,
а OpenMP — для параллельной обработки внутри каждого процесса.

## 2. Постановка задачи

Требуется реализовать алгоритм численного интегрирования многомерной
функции методом Монте-Карло.

Интеграл вычисляется путём генерации большого количества
случайных точек внутри заданной области
и последующего усреднения значений функции в этих точках.

Необходимо:

обеспечить корректность вычислений;
реализовать параллельную версию алгоритма;
сравнить результаты с последовательной реализацией;
оценить производительность гибридного подхода.

## 3. Базовый алгоритм

В основе решения лежит классический метод Монте-Карло.

Основные этапы алгоритма:

1.Генерация случайных точек в пределах области интегрирования.
2.Вычисление значения функции в каждой точке.
3.Накопление суммы значений функции.
4.Усреднение результатов и умножение на объём области интегрирования.

Точность результата зависит от количества случайных точек.

## 4. Межпроцессная схема

Для распределения вычислений между процессами используется MPI.
Общее количество случайных точек делится между MPI-процессами:

каждый процесс получает собственную часть выборки;
остаток распределяется между первыми процессами;
процессы вычисляют локальные суммы независимо друг от друга.

После завершения вычислений выполняется:

суммирование накопленных локальных сумм на нулевом процессе
с помощью MPI_Reduce;
передача итогового результата всем процессам от нулевого
 через MPI_Bcast.

Схема распределения нагрузки:

uint64_t n_local = n_total / size;
uint64_t remainder = n_total % size;

if (rank < static_cast\<int\>(remainder)) {
  n_local++;
}

Данный подход обеспечивает более равномерную загрузку процессов.

## 5. Внутрипроцессная схема

Внутри каждого MPI-процесса используется OpenMP.

Параллельная область организована с помощью директивы:

\# pragma omp parallel reduction(+ : local_sum)

Каждый поток:

генерирует собственные случайные точки;
использует независимый генератор случайных чисел;
вычисляет локальную сумму значений функции.

Для генерации случайных чисел применяется:

std::minstd_rand;
std::uniform_real_distribution.

Итоговая сумма внутри процесса объединяется через механизм reduction.

## 6. Детали реализации

В реализации были учтены следующие моменты:

независимые генераторы случайных чисел для потоков;
распределение нагрузки между MPI-процессами;
использование OpenMP reduction вместо ручной синхронизации;
отсутствие гонок данных;
передача итогового результата всем процессам через MPI_Bcast.

Фрагмент основной вычислительной части:

\# pragma omp parallel default(none) shared(input, func, dim, n_local, rank) \

reduction(+ : local_sum)
{
  uint32_t seed = static_cast<uint32_t>(
      rank * omp_get_max_threads() + omp_get_thread_num());

  std::minstd_rand gen(seed ^ 0x9e3779b9U);

  std::vector\<double\> point(dim);

\# pragma omp for schedule(static)

  for (int64_t i = 0; i < static_cast<int64_t>(n_local); ++i) {
    ...
    local_sum += func(point);
  }
}
