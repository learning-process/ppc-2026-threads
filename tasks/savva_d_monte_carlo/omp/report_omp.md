## 1. Контекст

В данной работе реализован параллельный алгоритм вычисления
многомерных интегралов методом Монте-Карло с использованием технологии OpenMP.

OpenMP позволяет организовать многопоточное выполнение вычислений
внутри одного процесса за счёт директив компилятора.
Основная идея реализации заключается в распределении вычислений между
потоками с последующим объединением частичных результатов.

## 2. Постановка задачи

Требуется реализовать алгоритм численного интегрирования многомерной
функции методом Монте-Карло.

Интеграл вычисляется путём генерации большого количества
случайных точек внутри заданной области
и последующего усреднения значений функции в этих точках.

Необходимо:

- обеспечить корректность вычислений;
- реализовать параллельную версию алгоритма;
- сравнить результаты с последовательной реализацией;
- оценить производительность OMP-реализации.

## 3. Базовый алгоритм

В основе решения лежит классический метод Монте-Карло.

Основные этапы алгоритма:

1.Генерация случайных точек в пределах области интегрирования.
2.Вычисление значения функции в каждой точке.
3.Накопление суммы значений функции.
4.Усреднение результатов и умножение на объём области интегрирования.

Точность результата зависит от количества случайных точек.

## 4. Схема распараллеливания

Основная вычислительная часть помещена внутрь параллельной области:

\# pragma omp parallel default(none) reduction(+ : sum)

После создания параллельной области OpenMP формирует группу потоков,
каждый из которых выполняет часть вычислений независимо от остальных.

Для распределения итераций между потоками используется директива:

\# pragma omp for schedule(static)

Статическое распределение (schedule(static)) делит диапазон итераций на
блоки приблизительно одинакового размера и заранее назначает их потокам.
Это уменьшает накладные расходы на планирование и подходит для равномерной
вычислительной нагрузки.

Каждый поток:

создаёт собственный генератор случайных чисел;
генерирует независимые случайные точки;
вычисляет локальную сумму значений функции.

Для генерации случайных чисел используются:

std::minstd_rand;
std::uniform_real_distribution.

Инициализация генератора зависит от номера потока:

std::minstd_rand gen(1337 + omp_get_thread_num());

Это позволяет избежать использования общего генератора случайных чисел
и исключает возникновение состояния гонки.

Для хранения координат случайной точки каждый поток использует собственный локальный вектор:

std::vector\<double\> point(dim);

Объединение частичных сумм выполняется автоматически через механизм reduction:

reduction(+ : sum)

OpenMP создаёт приватную копию переменной sum для каждого потока, после чего
локальные результаты суммируются в общую переменную после завершения параллельной области.

Таким образом:

вычисления выполняются параллельно;
потоки не конфликтуют между собой;
отсутствуют гонки данных.

## 5. Детали реализации

В реализации были учтены следующие моменты:

независимые генераторы случайных чисел для потоков;
статическое распределение нагрузки между потоками;
использование OpenMP reduction;
отсутствие гонок данных;
локальное хранение промежуточных данных внутри потоков.

Фрагмент основной вычислительной части:

\# pragma omp parallel default(none) shared(input, func, dim, n) \
    reduction(+ : sum)
{
  std::minstd_rand gen(1337 + omp_get_thread_num());

  std::vector<std::uniform_real_distribution\<double\>> dists(dim);

  for (size_t i = 0; i < dim; ++i) {
    dists[i] = std::uniform_real_distribution\<double\>(
        input.lower_bounds[i],
        input.upper_bounds[i]);
  }

  std::vector\<double\> point(dim);

\# pragma omp for schedule(static)
  for (int64_t i = 0; i < n; ++i) {
    ...
    sum += func(point);
  }
}
