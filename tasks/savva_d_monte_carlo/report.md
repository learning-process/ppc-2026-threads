## 1. Введение

    Методами Монте-Карло называют численные методы решения математических
задач при помощи моделирования случайных величин. Однако,
решать методами Монте-Карло можно любые математические задачи, а не
только задачи вероятностного происхождения.
    Преимущества подобных недетерминированных методов особенно ярко проявляются
при решении задач большой размерности, когда применение традиционных
детерминированных методов затруднено или совсем невозможно.
    С развитием технологий программирования моделирование случайных чисел стало
легко реализуемым, следовательно метод Монте-Карло получил широкое
распространение в задачах, где традиционные численные методы трудно или
невозможно реализовать. Так как совершенствование ЭВМ продолжается,
можно предположить что методы Монте-Карло продолжат своё развитие,
а также расширится область их применения.
    В данной работе рассматривается вычисление многомерных интегралов
на основе метода Монте-Карло. Исследуется возможность увеличения эффективности
работы алгоритма при использовании параллельных технологий на потоках.
Для этого реализуются последовательная SEQ версия алгоритма,  а также версии
с использованием технологий OMP, TBB, STL, ALL. Измеряются и сравниваются
ускорения параллельных реализаций в сравнении с последовательной.
Это позволяет сделать выводы об оптимальной реализации алгоритма на основе
полученных эмпирических данных.

## 2. Постановка задачи

Требуется реализовать алгоритм численного вычисления многомерных
интегралов методом Монте-Карло.

Входными данными являются:

- интегрируемая функция;
- границы интегрирования по каждому измерению;
- количество случайных точек, используемых для моделирования.

Пусть требуется вычислить интеграл вида:

$$
I = \int_{a_1}^{b_1} \int_{a_2}^{b_2} \cdots \int_{a_n}^{b_n}
    f(x_1, x_2, \ldots, x_n) \, dx_1 dx_2 \ldots dx_n
$$

Метод Монте-Карло основан на случайной генерации точек внутри
области интегрирования и последующем вычислении среднего
значения функции:

$$
I \approx V \cdot \frac{1}{N} \sum_{i=1}^{N} f(X_i)
$$
где:

- $V$ — объём области интегрирования;
- $N$ — количество случайных точек;
- $X_i$ — случайная точка внутри области интегрирования.

Необходимо:

- реализовать последовательную версию алгоритма (SEQ);
- разработать параллельные версии с использованием технологий:
  - OpenMP (OMP);
  - Intel oneTBB (TBB);
  - STL (std::thread);
  - гибридной схемы ALL;
- обеспечить корректность вычислений;
- выполнить сравнение производительности различных реализаций;
- исследовать влияние числа потоков и процессов на ускорение алгоритма.

Результатом работы программы является приближённое значение
многомерного интеграла, вычисленное методом Монте-Карло.

## 3. Проверка корректности

Корректность реализаций проверена с помощью 17 функциональных тестов,
 включающих:

- **2D функции**: константа, линейная, произведение, сумма квадратов,
 тригонометрическая;
- **3D и 4D функции**: константа, линейная, произведение, сумма квадратов;
- **Граничные случаи**: малые выборки, нулевой объём, отрицательные границы,
 большие области.

**Критерии проверки:**

- Относительная погрешность ≤ 5% для стандартных тестов
- Абсолютная погрешность ≤ 0.5 для малых выборок
- Абсолютная погрешность ≤ 0.1 для нулевого объёма

**Результат:** Все тесты успешно пройдены для всех реализаций
(SEQ, OMP, TBB, STL, ALL),
что подтверждает корректность разработанных параллельных алгоритмов.

## 4. Сравнение производительности

Для оценки производительности использовалась тестовая функция

$$
f(x, y, z) = \sum_{i=1}^{3} \left( \sin(x_i) \cdot \cos(x_i) + e^{-|x_i|} \right).
$$

Область интегрирования была выбрана $[-10, 10]^3$.

В алгоритме Монте-Карло использовалось 5 000 000 случайных точек.

Тестовая функция была выбрана не случайно: для оценки производительности
на многопотоковых программах необходимо выполнение тяжёлой вычислительной
работы, поэтому функция является комбинацией нескольких тяжеловесных
математических операций.

- **Процессор:** Intel Pentium Silver N5030;
- **Оперативная память:** 8,0 ГБ
- **Операционная система:** Windows 10 Pro;
- **Архитектура:** x64;
- **Компилятор:** Microsoft Visual C++ (MSVC);
- **Тип сборки:** Release


**Сборка:**

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release --parallel
```

**Конфигурация** задаётся парой:

- `mpiexec -n P` — число MPI-процессов (`P`);
- `OMP_NUM_THREADS=T` — потоков OpenMP на процесс;
- **total workers** = `P × T`.

**Функциональные тесты:**

```bash
mpiexec -n P ./build/bin/ppc_func_tests.exe --gtest_filter="*Savva*"
```

**Тесты производительности:**

```powershell
$env:OMP_NUM_THREADS="T"
mpiexec -n P ./build/bin/ppc_perf_tests.exe --gtest_filter="*Savva*"
```

Из используемых многопоточных технологий наилучший результат
показала технология OpenMP. Это объясняется тем, что в
данной задаче вычислительная работа, выполянемая потоками
на разных итерациях алгоритма, сравнима по объёму. OpenMP
предоставляет возможность эффективно реализовать
статическое распределение, что подходит для таких
задач. TBB, известный хорошим динамическим планировщиком,
в данной задаче не нашёл возможности
такого же эффективного применения.

В версии ALL, сочетающей параллелизм потоков и процессов,
в качестве многопоточной технологии была выбрана OMP,
как наиболее эффективная. На двух процессах ALL
показала наилучшее ускорение. При дальнейшем увеличении
числа процессов значительного прироста ускорения добиться
не удалось, что связано с ограниченным числом ядер процессора
(в моём случае равным 4).
