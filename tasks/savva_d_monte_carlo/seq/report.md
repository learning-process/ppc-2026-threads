## 1. Контекст

В данной работе реализован последовательный алгоритм вычисления
многомерных интегралов методом Монте-Карло.

Последовательная версия используется в качестве базовой реализации
для проверки корректности результатов и сравнения производительности
с параллельными версиями алгоритма.

Вычисления выполняются в одном потоке без использования
средств многопоточности и межпроцессного взаимодействия.

## 2. Постановка задачи

Требуется реализовать алгоритм численного интегрирования
многомерной функции методом Монте-Карло.

Интеграл вычисляется путём генерации большого количества
случайных точек внутри заданной области
и последующего усреднения значений функции в этих точках.

Необходимо:

обеспечить корректность вычислений;
реализовать последовательную версию алгоритма;
использовать случайную генерацию точек;
вычислить итоговое значение интеграла.

## 3. Базовый алгоритм

В основе решения лежит классический метод Монте-Карло.

Основные этапы алгоритма:

1.Генерация случайных точек в пределах области интегрирования.
2.Вычисление значения функции в каждой точке.
3.Накопление суммы значений функции.
4.Усреднение результатов и умножение на объём области интегрирования.

Точность результата зависит от количества случайных точек.

## 4. Схема выполнения

Вычисления выполняются последовательно в одном потоке.

Для генерации случайных чисел используется генератор:

std::minstd_rand

Генератор объявлен как:

static thread_local std::minstd_rand generator(std::random_device{}());

Это позволяет сохранить состояние генератора между вызовами функции
и избежать повторной инициализации.

Для генерации координат случайных точек используются распределения:

std::uniform_real_distribution\<double\>

Для каждой координаты создаётся отдельное распределение
в соответствии с границами области интегрирования.

В реализации используется частичная ручная оптимизация вычислений:
обработка точек выполняется блоками по четыре точки за одну итерацию цикла.

Основной цикл имеет вид:

for (; i + 3 < n; i += 4)

Внутри одной итерации одновременно генерируются четыре случайные точки
и вычисляются четыре значения функции:

sum += func(p1) + func(p2) + func(p3) + func(p4);

После завершения основного цикла отдельно обрабатываются оставшиеся точки,
если количество точек не кратно четырём.

## 5. Детали реализации

В реализации были учтены следующие моменты:

проверка корректности входных данных;
использование независимого генератора случайных чисел;
предварительное создание распределений для каждой координаты;
уменьшение накладных расходов за счёт обработки четырёх точек за итерацию;
вычисление среднего значения функции с последующим умножением на объём области интегрирования.

Фрагмент основной вычислительной части:

for (; i + 3 < n; i += 4) {
  for (size_t j = 0; j < dim; ++j) {
    p1[j] = distributions[j](generator);
    p2[j] = distributions[j](generator);
    p3[j] = distributions[j](generator);
    p4[j] = distributions[j](generator);
  }

  sum += func(p1) + func(p2) + func(p3) + func(p4);
}
