## 1. Контекст

В данной работе реализован параллельный алгоритм вычисления
многомерных интегралов методом Монте-Карло
с использованием библиотеки oneTBB.

oneTBB предоставляет высокоуровневые средства для организации
параллельных вычислений и автоматического распределения нагрузки
между потоками. Основная идея реализации заключается в разбиении
диапазона вычислений на независимые блоки с последующим
параллельным вычислением локальных сумм и их объединением.

## 2. Постановка задачи

Требуется реализовать алгоритм численного интегрирования многомерной
функции методом Монте-Карло.

Интеграл вычисляется путём генерации большого количества
случайных точек внутри заданной области
и последующего усреднения значений функции в этих точках.

Необходимо:

- обеспечить корректность вычислений;
- реализовать параллельную версию алгоритма;
- сравнить результаты с последовательной реализацией;
- оценить производительность TBB-реализации.

## 3. Базовый алгоритм

В основе решения лежит классический метод Монте-Карло.

Основные этапы алгоритма:

1.Генерация случайных точек в пределах области интегрирования.
2.Вычисление значения функции в каждой точке.
3.Накопление суммы значений функции.
4.Усреднение результатов и умножение на объём области интегрирования.

Точность результата зависит от количества случайных точек.

## 4. Схема распараллеливания

Параллельное выполнение реализовано с использованием алгоритма tbb::parallel_reduce.

Диапазон вычислений разбивается на блоки с помощью:

tbb::blocked_range<int64_t>(0, n)

Для контроля гранулярности можно указать `grainsize`:
`tbb::blocked_range<int64_t>(0, n, 1000)`
Планировщик по умолчанию — `auto_partitioner`, который адаптивно
балансирует нагрузку между потоками.

Каждый блок содержит часть итераций генерации случайных точек.
Библиотека oneTBB автоматически распределяет эти блоки между рабочими потоками.

Основная вычислительная часть выполняется внутри лямбда-функции:

[&](const tbb::blocked_range<int64_t> &r, double local_sum)

Для каждого блока:

создаётся независимый генератор случайных чисел;
генерируются случайные точки;
вычисляются значения функции;
накапливается локальная сумма.

Для генерации случайных чисел используются:

std::minstd_rand;
std::uniform_real_distribution.

Начальное значение генератора зависит от начала диапазона:

std::minstd_rand gen(
    static_cast<uint32_t>(r.begin()) ^ 0x9e3779b9U);

Это позволяет обеспечить независимость последовательностей случайных чисел в различных задачах.

Для хранения координат точки используется локальный вектор:

std::vector\<double\> point(dim);

После завершения вычислений локальные суммы автоматически объединяются с помощью функции редукции:

[](double a, double b) {
  return a + b;
}

Таким образом:

- вычисления выполняются параллельно;
- библиотека автоматически балансирует нагрузку между потоками;
- отсутствуют гонки данных;
- объединение результатов выполняется без ручной синхронизации.

## 5. Детали реализации

В реализации были учтены следующие моменты:

использование tbb::parallel_reduce для параллельного накопления суммы;
автоматическое распределение задач между потоками;
независимые генераторы случайных чисел для различных блоков;
объединение частичных результатов через механизм редукции.

Фрагмент основной вычислительной части:

double sum = tbb::parallel_reduce(
    tbb::blocked_range<int64_t>(0, n),
    0.0,

    [&](const tbb::blocked_range<int64_t> &r,
        double local_sum) {

      std::minstd_rand gen(
          static_cast<uint32_t>(r.begin())
          ^ 0x9e3779b9U);

      std::vector\<double\> point(dim);

      for (int64_t i = r.begin(); i < r.end(); ++i) {

        for (size_t j = 0; j < dim; ++j) {
          point[j] = dists[j](gen);
        }

        local_sum += func(point);
      }

      return local_sum;
    },

    [](double a, double b) {
      return a + b;
    });
