## 1. Контекст

В данной работе реализован параллельный алгоритм вычисления
многомерных интегралов методом Монте-Карло с использованием средств
стандартной библиотеки C++ (std::thread).

std::thread позволяет организовать многопоточное выполнение вычислений
внутри одного процесса за счёт ручного управления потоками.
Основная идея реализации заключается в распределении вычислений между
потоками с последующим объединением частичных результатов.

## 2. Постановка задачи

Требуется реализовать алгоритм численного интегрирования многомерной
функции методом Монте-Карло.

Интеграл вычисляется путём генерации большого количества
случайных точек внутри заданной области
и последующего усреднения значений функции в этих точках.

Необходимо:

- обеспечить корректность вычислений;
- реализовать параллельную версию алгоритма;
- сравнить результаты с последовательной реализацией;
- оценить производительность STL-реализации.

## 3. Базовый алгоритм

В основе решения лежит классический метод Монте-Карло.

Основные этапы алгоритма:

1.Генерация случайных точек в пределах области интегрирования.
2.Вычисление значения функции в каждой точке.
3.Накопление суммы значений функции.
4.Усреднение результатов и умножение на объём области интегрирования.

Точность результата зависит от количества случайных точек.

## 4. Схема распараллеливания

Параллельное выполнение реализовано с использованием std::thread.

Количество потоков определяется через:

std::thread::hardware_concurrency()

Полученное значение ограничивается числом генерируемых точек, чтобы избежать создания избыточного количества потоков.

Общий объём работы делится между потоками статически. Для каждого потока вычисляется количество точек:

const int64_t points_per_thread = n / num_threads;
const int64_t tail = n % num_threads;

Оставшиеся точки равномерно распределяются между первыми потоками.

Каждый поток выполняет функцию worker, внутри которой:

создаётся собственный генератор случайных чисел;
генерируются случайные точки;
вычисляются значения функции;
накапливается локальная сумма.

Для генерации случайных чисел используются:

std::minstd_rand;
std::uniform_real_distribution.

Инициализация генератора зависит от номера потока:

std::minstd_rand gen(1337 + thread_id);

Для хранения координат точки используется локальный вектор:

std::vector\<double\> point(dim);

Каждый поток записывает результат только в собственную ячейку массива:

partial_sums[thread_id]

Это позволяет избежать гонок данных без использования мьютексов.

После запуска всех потоков основной поток ожидает их завершения через:

t.join();

Затем выполняется объединение частичных сумм и вычисление итогового значения интеграла.

Таким образом:

вычисления выполняются параллельно;
нагрузка равномерно распределяется между потоками;
отсутствуют гонки данных;
синхронизация выполняется только после завершения вычислений.

## 5. Детали реализации

В реализации были учтены следующие моменты:

ручное управление потоками через std::thread;
статическое распределение нагрузки;
независимые генераторы случайных чисел для потоков;
объединение результатов после завершения вычислений c помощью std::thread::join();.

Фрагмент основной вычислительной части:

auto worker = [&](int thread_id, int64_t pts) {
  std::minstd_rand gen(1337 + thread_id);

  std::vector<std::uniform_real_distribution\<double\>> dists;

  for (size_t i = 0; i < dim; ++i) {
    dists.emplace_back(
        input.lower_bounds[i],
        input.upper_bounds[i]);
  }

  std::vector\<double\> point(dim);

  double local_sum = 0.0;

  for (int64_t i = 0; i < pts; ++i) {
    for (size_t j = 0; j < dim; ++j) {
      point[j] = dists[j](gen);
    }

    local_sum += func(point);
  }

  partial_sums[thread_id] = local_sum;
};
