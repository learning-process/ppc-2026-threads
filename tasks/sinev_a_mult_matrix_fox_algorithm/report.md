## 1. Введение

Умножение плотных матриц — одна из ключевых задач вычислительной математики. В работе реализован алгоритм Фокса (блочная схема) с использованием пяти технологий: последовательная версия (SEQ), OpenMP, STL (`std::thread`), Intel TBB и гибридная MPI+OpenMP (ALL).  
Цель — сравнить эффективность этих подходов для умножения квадратных матриц `double` размера `N × N`.

## 2. Единая постановка задачи

**Входные данные:**
- `matrix_size` — размер `N` (положительное целое)
- `matrix_a`, `matrix_b` — векторы длины `N×N` с элементами `double`

**Выходные данные:**
- `output` — результирующая матрица `C = A × B`

**Критерий корректности:**  
Результат параллельной реализации считается верным, если максимальная абсолютная погрешность относительно SEQ не превышает `1e-10`.

## 6. Интерпретация различий
### 6.1 SEQ (базовый эталон)

Последовательная версия с тройным вложенным циклом – основа для сравнения. Время 0.026 с для N=1000 отражает базовую вычислительную сложность `O(N³)`.

### 6.2 OMP – лучший результат

OpenMP даёт ускорение 10.77× при эффективности 67%. Причины:
- Низкие накладные расходы на создание потоков.
- Эффективная работа с кэш-памятью благодаря collapse(2).
- Отсутствие лишних абстракций.

### 6.3 STL (std::thread) – средний результат

Ускорение 2.10×, эффективность 13%.
Причины:
- Ручное управление потоками через атомарный счётчик.
- Нет встроенной редукции или оптимизации кэша.
- Динамическое распределение блоков добавляет накладные расходы.

### 6.4 TBB – худший результат среди многоядерных

Ускорение всего 1.58× при эффективности 9.9%.
Причины:
- Высокие накладные расходы task scheduler’а.
- Для задачи с мелкими блоками динамическое планирование (work-stealing) не окупается.
 -TBB эффективен на более крупных гранулах вычислений.

### 6.5 ALL (MPI+OpenMP) – лучший гибрид

- При 4 процессах × 4 потока ускорение 5.52×, эффективность 34.5%.
- При оптимальной конфигурации (квадратная решётка, размер матрицы кратен q) гибридная версия работает хорошо.
- При неоптимальных конфигурациях (5 или 12 процессов) происходит fallback на OpenMP-умножение, производительность падает.
- Основное преимущество ALL – возможность работы на нескольких узлах (хотя в данном тесте использовался один узел).

## 8. Заключение

В ходе работы реализованы и протестированы пять версий алгоритма Фокса для умножения плотных матриц.
### 8.1 Основные выводы

- OpenMP – безусловный лидер для одноузловых вычислений (10.77× ускорение). Рекомендуется для всех случаев, когда задача помещается в память одного узла.
- Гибридная MPI+OpenMP показывает хороший результат (5.52×) при правильной конфигурации (квадратное число процессов, размер матрицы кратен q). Это единственный вариант для масштабирования на несколько узлов.
- STL даёт умеренное ускорение (2.10×) и может быть полезна, если нельзя использовать OpenMP или TBB, но требует ручной оптимизации.
- TBB не подходит для задач с очень мелкими блоками – накладные расходы превышают выигрыш.

### 8.2 Ограничения сравнения

- Измерения проведены на одном узле (ALL тоже запускалась на одном узле, поэтому коммуникационные затраты минимальны).
- Размер матрицы фиксирован (1000). При N > 2000 соотношение может измениться.
- Fallback-режим для ALL значительно медленнее.

### 8.3 Рекомендации

- Для одноузловых вычислений на матрицах размером до нескольких тысяч – OpenMP.
- Для многоузловых вычислений или очень больших матриц (>5000) – MPI+OpenMP с правильным разбиением.
- STL – только при жёстких ограничениях на внешние зависимости.
- TBB не рекомендуется для данной задачи.
