## 1. Контекст

Решается задача умножения двух квадратных плотных матриц `C = A × B` с элементами типа `double`.
Последовательная версия реализует классический алгоритм за O(n³) и служит единственным эталоном корректности
и производительности для всех параллельных реализаций (OpenMP, TBB, std::thread, гибридная MPI+threads).
Без фиксированного baseline невозможно измерить ускорение, а без строгой проверки корректности — гарантировать,
что распараллеливание не внесло ошибок.

## 2. Постановка задачи

**Входные данные (предметная задача):**

- Квадратная матрица `A` размера `n × n` (элементы типа `double`).
- Квадратная матрица `B` размера `n × n` (элементы типа `double`).

**Выходные данные:**

- Квадратная матрица `C` размера `n × n`, где `C[i][j] = Σ_{k=0}^{n-1} A[i][k] · B[k][j]`.

**Ограничения (предметные):**

- `n ≥ 1` (размерность матрицы — натуральное число).
- Матрицы плотные (заполнены ненулевыми элементами в общем случае).
- Элементы — числа двойной точности IEEE 754.

**Крайние случаи (предметные):**

- `n = 1`: произведение двух скаляров.
- Нулевые матрицы: `A = 0` или `B = 0` → `C = 0`.
- Единичная матрица: `A = I` → `C = B`.
- Матрицы с отрицательными элементами — допустимо.

**Границы, блокируемые в ValidationImpl (код):**

- `n ≤ 0` — отвергается.
- Несоответствие размера переданных векторов `n²` — отвергается (защита от выхода за границы памяти).

## 3. Базовый алгоритм

**Пошаговое описание:**

1. Получить размер `n` из входных данных.
2. Выделить память под результирующую матрицу `C` размером `n×n`, заполнить нулями.
3. Для каждой строки `i` от `0` до `n-1`:
   - Для каждого столбца `j` от `0` до `n-1`:
     - Вычислить скалярное произведение строки `A[i][*]` и столбца `B[*][j]`:
       - Инициализировать `sum = 0.0`.
       - Для каждого `k` от `0` до `n-1`:
         - `sum += A[i][k] * B[k][j]`
     - Записать `sum` в `C[i][j]`.
4. Вернуть `C`.

**Асимптотика по времени:** O(n³) — три вложенных цикла, каждый до `n` (ровно `n³` умножений и `n³` сложений).

**Асимптотика по памяти:** O(n²) — хранение двух входных матриц и одной выходной (всего 3·n²·sizeof(double)).

**Инварианты и критерий корректности:**

- Инвариант цикла по `i,j`: после каждой итерации `C[i][j] = Σ_{k=0}^{n-1} A[i][k]·B[k][j]`.
- Критерий корректности: для всех `i,j` выполняется математическое определение произведения матриц с точностью
  до погрешности `double` (относительная погрешность < 1e-10 для тестов с целыми числами).

## 4. Детали реализации

**Файлы:**

- `seq/include/ops_seq.hpp` — объявление класса `SinevAMultMatrixFoxAlgorithmSEQ`.
- `seq/src/ops_seq.cpp` — реализация.

**Роль методов pipeline:**

- `ValidationImpl`: проверяет `matrix_size > 0` и что векторы `matrix_a`, `matrix_b` имеют ровно
  `matrix_size * matrix_size` элементов. Возвращает `false` при нарушении — задача не запускается.
- `PreProcessingImpl`: выделяет `std::vector<double>` для выходной матрицы, заполняет нулями.
- `RunImpl`: выполняет тройной вложенный цикл умножения. Если выходной вектор имеет неправильный размер
  (защита от ошибок в PreProcessing), перевыделяет память.
- `PostProcessingImpl`: возвращает `true` (постобработка не требуется).

**Иллюстративный листинг (ядро алгоритма):**

```cpp
// File: seq/src/ops_seq.cpp
// Функция RunImpl — критическая секция последовательного вычисления
bool SinevAMultMatrixFoxAlgorithmSEQ::RunImpl() {
  const auto &[matrix_size, matrix_a, matrix_b] = GetInput();
  size_t n = matrix_size;
  auto &output = GetOutput();

  for (size_t i = 0; i < n; ++i) {
    for (size_t j = 0; j < n; ++j) {
      double sum = 0.0;
      for (size_t k = 0; k < n; ++k) {
        sum += matrix_a[(i * n) + k] * matrix_b[(k * n) + j];
      }
      output[(i * n) + j] = sum;
    }
  }
  return true;
}
```

**Обработка исключительных и граничных случаев:**

- `n = 1`: циклы работают корректно, выполняется одно умножение.
- Некорректные входные данные (размер `n=0` или неполные векторы) отлавливаются в `ValidationImpl`,
  задача не выполняется.
- При `n > 0` и корректных векторах исключения в `RunImpl` не возникают (нет деления на ноль,
   нет выхода за границы благодаря проверенным размерам).

## 5. Проверка корректности

**Метод сравнения:** результат последовательной версии сравнивается с эталонной реализацией `ReferenceMultiply`
из тестового кода. Используется поэлементное сравнение с абсолютной погрешностью `epsilon = 1e-10`.

**Функциональные тесты:**  
13 тестов, охватывающих размеры: 1, 2, 4, 6, 8, 9, 10, 16, 18, 25, 50, 75, 100.
Генерация данных: арифметическая прогрессия в `A` (1,2,3,…) и обратная прогрессия в `B` (n², n²-1, …, 1).
Такой выбор обеспечивает детерминированность и покрытие различных паттернов доступа к памяти.

**Характерные примеры:**

1. **Размер 1×1**  
Вход: `A = [1]`, `B = [1]` (т.к. size=1, start=1, обратная прогрессия даёт 1)
Ожидаемый выход: `C = [1]`
Проверяет: базовое умножение скаляров.

2. **Размер 2×2**  
Вход: `A = [1, 2; 3, 4]`, `B = [4, 3; 2, 1]` (обратная прогрессия: size=4, элементы 4,3,2,1)
Ожидаемый выход: `C = [1*4+2*2, 1*3+2*1; 3*4+4*2, 3*3+4*1] = [8, 5; 20, 13]`
Проверяет: правильность суммирования и индексации.

3. **Размер 3×3 с единичной матрицей**  
Вход: `A = [1,2,3; 4,5,6; 7,8,9]`, `B = [9,8,7; 6,5,4; 3,2,1]`
Ожидаемый выход вычисляется по формуле суммы произведений.
Проверяет: работу для нечётного размера, отсутствие ошибок округления.
