## 2. Постановка задачи

Даны две квадратные плотные матрицы `A` и `B` размера `N × N`.

Требуется вычислить результирующую матрицу:

```math
C = A \times B
```

где:

```math
C_{ij} = \sum_{k=0}^{N-1} A_{ik} \cdot B_{kj}
```

### Входные данные

- `matrix_size` — размер матрицы `N`
- `matrix_a` — первая матрица размера `N × N`
- `matrix_b` — вторая матрица размера `N × N`

### Выходные данные

- `output` — результирующая матрица размера `N × N`

### Ограничения

- Размер матрицы должен быть больше нуля
- Матрицы должны быть квадратными
- Размер входных массивов должен быть равен `N × N`
- Тип элементов — `double`

### Пример

Входные данные:

```text
A =
1 2
3 4

B =
5 6
7 8
```

Выходные данные:

```text
C =
19 22
43 50
```

## 3. Описание базового алгоритма (последовательная версия)

В последовательной реализации используется классический алгоритм умножения матриц с тремя вложенными циклами.

Алгоритм выполняет следующие шаги:

1. Итерация по строкам матрицы `A`
2. Итерация по столбцам матрицы `B`
3. Вычисление скалярного произведения строки и столбца

Основной алгоритм:

```cpp
for (size_t i = 0; i < n; ++i) {
  for (size_t j = 0; j < n; ++j) {
    double sum = 0.0;

    for (size_t k = 0; k < n; ++k) {
      sum += matrix_a[(i * n) + k] *
             matrix_b[(k * n) + j];
    }

    output[(i * n) + j] = sum;
  }
}
```

### Сложность алгоритма

#### Временная сложность

```math
O(N^3)
```

Используются три вложенных цикла.

#### Пространственная сложность

```math
O(N^2)
```

Память требуется для хранения:
- матрицы `A`
- матрицы `B`
- результирующей матрицы `C`

## 4. Схема распараллеливания

В данной работе реализована последовательная версия алгоритма, однако структура проекта подготовлена для дальнейшего распараллеливания по блочной схеме алгоритма Фокса.

Алгоритм Фокса предполагает:
- разбиение матриц на блоки
- распределение блоков между потоками или процессами
- локальное вычисление произведений блоков
- последовательное смещение блоков матрицы `B`

Преимущества блочного подхода:
- улучшение локальности данных
- уменьшение числа обращений к памяти
- эффективная масштабируемость
- снижение накладных расходов синхронизации

В последовательной версии все вычисления выполняются в одном потоке без распределения данных.
