## 1. Введение

Метод Монте-Карло — это вычислительный алгоритм, использующий случайную выборку для аппроксимации значений многомерных интегралов. Данная реализация использует OpenMP для распараллеливания основного цикла генерации точек и суммирования значений функции. Ожидаемый результат — численное приближение интеграла заданной функции по гиперпрямоугольной области, полученное с ускорением относительно последовательной версии.

## 2. Постановка задачи

Вычисление определённого интеграла вида:

$$I = \int_{a_1}^{b_1} \int_{a_2}^{b_2} ... \int_{a_d}^{b_d} f(x_1, x_2, ..., x_d) \, dx_1 dx_2 ... dx_d$$


**Входные данные:**
- lower_bounds — вектор нижних пределов интегрирования по каждому измерению
- upper_bounds — вектор верхних пределов интегрирования по каждому измерению
- num_points — количество точек выборки
- func_type — тип интегрируемой функции (константа, линейная, произведение, квадратичная, экспонента)

**Выходные данные:**
- result — приближённое значение интеграла (вещественное число)

**Ограничения:**
- Размерности lower_bounds и upper_bounds совпадают и не равны нулю
- Для каждого измерения lower_bounds[i] < upper_bounds[i]
- num_points > 0

## 3. Базовый алгоритм (последовательная версия)

Алгоритм использует метод Монте-Карло с квазислучайной последовательностью Кронекера:

1. Вычисляется объём области интегрирования: V = prod_{i=1}^{d} (b_i - a_i)
2. Генерируется num_points точек в единичном гиперкубе [0,1)^d с помощью последовательности Кронекера
3. Каждая точка масштабируется в реальную область интегрирования: x_j = a_j + u_j * (b_j - a_j)
4. Для каждой точки вычисляется значение подынтегральной функции f(x)
5. Вычисляется среднее арифметическое значений функции
6. Результат интеграла: I ≈ V * (1/n) * sum_{i=1}^{n} f(x_i)

**Сложность по времени:** O(n * d), где n — количество точек, d — размерность

**Сложность по памяти:** O(d)

**Инварианты:** Результат сходится к точному значению интеграла при увеличении num_points.

## 4. Схема распараллеливания

Распараллеливание реализовано с помощью директивы OpenMP `#pragma omp parallel for`. Основной цикл по точкам выборки разделяется между потоками.

**Область распараллеливания:** Цикл генерации точек и суммирования значений функции.

**Атрибуты переменных:**
- lower_bounds, upper_bounds, dim, num_points, func_type — shared (общие для всех потоков)
- sum — reduction(+) (каждый поток имеет локальную копию, в конце выполняется суммирование)

**Параметры scheduling:**
- schedule(static) — итерации цикла статически распределяются между потоками блоками примерно равного размера

**Синхронизация:** Неявный барьер в конце параллельной области. Редукция переменной sum обеспечивает потокобезопасное накопление результата.

## 5. Детали реализации

**Структура кода:**
- ops_omp.hpp — объявление класса NikitinAMonteCarloOMP, наследующего BaseTask
- ops_omp.cpp — реализация методов класса с использованием OpenMP

**Ключевые функции:**
- EvaluateFunction — вычисляет значение тестовой функции заданного типа
- KroneckerSequence — генерирует квазислучайную последовательность

**Методы класса:**
- ValidationImpl — проверяет корректность входных данных
- RunImpl — выполняет распараллеленный вычислительный цикл

**Граничные случаи:**
- При num_points = 0 валидация возвращает false
- При несовпадении размерностей границ валидация возвращает false
- Для функций, требующих двух измерений (Product, Quadratic), при недостаточной размерности возвращается 0.0

**Использование памяти:** Точка хранится как локальный вектор размера dim, выделяемый в каждой итерации каждым потоком.

Корректность проверяется на наборе тестов с известными аналитическими значениями интегралов. Результат параллельной версии сравнивается с результатом последовательной версии.
