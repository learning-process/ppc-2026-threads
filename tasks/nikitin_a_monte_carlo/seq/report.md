## 1. Введение

Метод Монте-Карло — это вычислительный алгоритм, использующий случайную выборку
для аппроксимации значений многомерных интегралов.
Данная последовательная реализация служит эталоном корректности
и базой для сравнения производительности параллельных версий.
Ожидаемый результат — численное приближение интеграла заданной функции
по гиперпрямоугольной области.

## 2. Постановка задачи

Вычисление определённого интеграла вида:

$$I = \int_{a_1}^{b_1} \int_{a_2}^{b_2} ... \int_{a_d}^{b_d} f(x_1, x_2, ..., x_d) \, dx_1 dx_2 ... dx_d$$

**Входные данные:**

- lower_bounds — вектор нижних пределов интегрирования по каждому измерению
- upper_bounds — вектор верхних пределов интегрирования по каждому измерению
- num_points — количество точек выборки
- func_type — тип интегрируемой функции
  (константа, линейная, произведение, квадратичная, экспонента)

**Выходные данные:**

- result — приближённое значение интеграла (вещественное число)

**Ограничения:**

- Размерности lower_bounds и upper_bounds совпадают и не равны нулю
- Для каждого измерения lower_bounds[i] < upper_bounds[i]
- num_points > 0

## 3. Базовый алгоритм (последовательная версия)

Алгоритм использует метод Монте-Карло с квазислучайной последовательностью Кронекера:

1. Вычисляется объём области интегрирования: V = prod_{i=1}^{d} (b_i - a_i)
2. Генерируется num_points точек в единичном гиперкубе [0,1)^d
   с помощью последовательности Кронекера
3. Каждая точка масштабируется в реальную область интегрирования:
   x_j = a_j + u_j * (b_j - a_j)
4. Для каждой точки вычисляется значение подынтегральной функции f(x)
5. Вычисляется среднее арифметическое значений функции
6. Результат интеграла: I ≈ V *(1/n)* sum_{i=1}^{n} f(x_i)

**Сложность по времени:** O(n * d), где n — количество точек, d — размерность

**Сложность по памяти:** O(d)

**Инварианты:** Результат сходится к точному значению интеграла при увеличении num_points.

## 4. Схема распараллеливания

Для последовательной версии распараллеливание не применяется.
Все вычисления выполняются в одном потоке.

## 5. Детали реализации

**Структура кода:**

- ops_seq.hpp — объявление класса NikitinAMonteCarloSEQ, наследующего BaseTask
- ops_seq.cpp — реализация методов класса

**Ключевые функции:**

- EvaluateFunction — вычисляет значение тестовой функции заданного типа
- KroneckerSequence — генерирует квазислучайную последовательность
  для равномерного покрытия области

**Методы класса:**

- ValidationImpl — проверяет корректность входных данных
- RunImpl — выполняет основной вычислительный цикл

**Граничные случаи:**

- При num_points = 0 валидация возвращает false
- При несовпадении размерностей границ валидация возвращает false
- Для функций, требующих двух измерений (Product, Quadratic),
  при недостаточной размерности возвращается 0.0

**Использование памяти:** Точка хранится как временный вектор размера d,
выделяемый на каждой итерации.

## 7. Результаты и обсуждение

### 7.1 Корректность

Корректность проверяется на наборе тестов с известными аналитическими значениями интегралов:

- Константная функция f=1 на единичном квадрате: интеграл = объёму
- Линейная функция f=x на отрезке [0,1]: интеграл = 0.5
- Функция-произведение f=x*y на единичном квадрате: интеграл = 0.25
- Квадратичная функция f=x^2+y^2 на единичном квадрате: интеграл ≈ 0.6667
- Экспонента f=e^x на отрезке [0,1]: интеграл = e - 1 ≈ 1.71828

Для каждого типа функции выполняется сравнение полученного приближения
с точным значением с заданной относительной погрешностью.
