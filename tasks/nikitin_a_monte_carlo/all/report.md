## 1. Введение

Метод Монте-Карло — это вычислительный алгоритм, использующий случайную выборку
для аппроксимации значений многомерных интегралов. Данная реализация является гибридной
и использует два уровня параллелизма: межпроцессный (MPI) для распределения точек выборки
между процессами и внутрипроцессный (OpenMP) для распараллеливания вычислений внутри
каждого процесса. Ожидаемый результат — численное приближение интеграла заданной функции
по гиперпрямоугольной области, полученное с использованием иерархической параллельной схемы.

## 2. Постановка задачи

Вычисление определённого интеграла вида:

$$I = \int_{a_1}^{b_1} \int_{a_2}^{b_2} ... \int_{a_d}^{b_d} f(x_1, x_2, ..., x_d) \, dx_1 dx_2 ... dx_d$$

**Входные данные:**

- lower_bounds — вектор нижних пределов интегрирования по каждому измерению
- upper_bounds — вектор верхних пределов интегрирования по каждому измерению
- num_points — количество точек выборки
- func_type — тип интегрируемой функции (константа, линейная, произведение, квадратичная, экспонента)

**Выходные данные:**

- result — приближённое значение интеграла (вещественное число)

**Ограничения:**

- Размерности lower_bounds и upper_bounds совпадают и не равны нулю
- Для каждого измерения lower_bounds[i] < upper_bounds[i]
- num_points > 0

## 3. Базовый алгоритм (последовательная версия)

Алгоритм использует метод Монте-Карло с квазислучайной последовательностью Кронекера:

1. Вычисляется объём области интегрирования: V = prod_{i=1}^{d} (b_i - a_i)
2. Генерируется num_points точек в единичном гиперкубе [0,1)^d с помощью последовательности Кронекера
3. Каждая точка масштабируется в реальную область интегрирования: x_j = a_j + u_j * (b_j - a_j)
4. Для каждой точки вычисляется значение подынтегральной функции f(x)
5. Вычисляется среднее арифметическое значений функции
6. Результат интеграла: I ≈ V *(1/n)* sum_{i=1}^{n} f(x_i)

**Сложность по времени:** O(n * d), где n — количество точек, d — размерность

**Сложность по памяти:** O(d)

**Инварианты:** Результат сходится к точному значению интеграла при увеличении num_points.

## 4. Схема распараллеливания

Реализация использует гибридную модель параллелизма: MPI + OpenMP.

### 4.1 Межпроцессная схема (MPI)

**Распределение данных:** Диапазон индексов [0, num_points) распределяется между MPI-процессами.

**Формула разбиения:**

- points_per_rank = num_points / size (целая часть)
- remainder = num_points % size (остаток)
- Первые remainder процессов получают на одну точку больше

**Роли процессов:**

- Все процессы участвуют в вычислениях (каждый обрабатывает свой поддиапазон)
- Процесс с rank = 0 выполняет сбор результатов и финальное вычисление

**Коммуникации:**

- MPI_Reduce — сбор локальных сумм от всех процессов на процесс 0 с операцией суммирования
- MPI_Bcast — рассылка финального результата от процесса 0 всем процессам
- MPI_Barrier — синхронизация всех процессов в конце выполнения

### 4.2 Внутрипроцессная схема (OpenMP)

**Распараллеливание:** Внутри каждого MPI-процесса цикл по точкам поддиапазона
распараллеливается с помощью OpenMP.

**Атрибуты переменных:**

- my_start, my_end, dim, lower_bounds, upper_bounds, func_type — shared (общие для всех потоков)
- local_sum — reduction(+) (каждый поток имеет локальную копию, в конце выполняется суммирование)

**Параметры scheduling:**

- schedule(static) — итерации цикла статически распределяются между потоками

**Синхронизация:** Неявный барьер в конце параллельной области. Редукция переменной local_sum
обеспечивает потокобезопасное накопление.

### 4.3 Иерархия параллелизма

Общая конфигурация: size (MPI-процессов) × threads_per_process (OpenMP-потоков)

## 5. Детали реализации

**Структура кода:**

- ops_all.hpp — объявление класса NikitinAMonteCarloALL, наследующего BaseTask
- ops_all.cpp — реализация методов класса с использованием MPI и OpenMP

**Ключевые функции:**

- EvaluateFunction — вычисляет значение тестовой функции заданного типа
- KroneckerSequence — генерирует квазислучайную последовательность

**Методы класса:**

- ValidationImpl — проверяет корректность входных данных
- RunImpl — выполняет гибридный вычислительный алгоритм

**MPI-вызовы:**

- MPI_Comm_rank / MPI_Comm_size — определение ранга и числа процессов
- MPI_Reduce — сбор и суммирование локальных результатов
- MPI_Bcast — рассылка результата всем процессам
- MPI_Barrier — глобальная синхронизация

**OpenMP-директива:**

- #pragma omp parallel for reduction(+ : local_sum) schedule(static) default(none)

**Граничные случаи:**

- При num_points = 0 валидация возвращает false
- При несовпадении размерностей границ валидация возвращает false
- Для функций, требующих двух измерений (Product, Quadratic), при недостаточной размерности возвращается 0.0
- При remainder > 0 первые процессы получают дополнительную точку для равномерного распределения нагрузки

**Использование памяти:** Точка хранится как локальный вектор размера dim,
выделяемый в каждой итерации каждого потока каждого процесса.

## 8. Выводы

Гибридная реализация ALL объединяет два уровня параллелизма: MPI для распределения работы
между процессами и OpenMP для распараллеливания внутри каждого процесса.
Процесс с рангом 0 выполняет роль сборщика результатов через MPI_Reduce,
после чего результат рассылается всем процессам через MPI_Bcast.
MPI_Barrier в конце обеспечивает глобальную синхронизацию.
Иерархическая схема позволяет эффективно использовать как многоузловые кластеры (через MPI),
так и многоядерные процессоры (через OpenMP).
