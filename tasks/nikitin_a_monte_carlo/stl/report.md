## 1. Введение

Метод Монте-Карло — это вычислительный алгоритм, использующий случайную выборку для аппроксимации значений многомерных интегралов. Данная реализация использует средства стандартной библиотеки C++ (std::thread, std::async, std::future) для ручного управления потоками и распараллеливания основного цикла генерации точек и суммирования значений функции. Ожидаемый результат — численное приближение интеграла заданной функции по гиперпрямоугольной области, полученное с ускорением относительно последовательной версии.

## 2. Постановка задачи

Вычисление определённого интеграла вида:

$$I = \int_{a_1}^{b_1} \int_{a_2}^{b_2} ... \int_{a_d}^{b_d} f(x_1, x_2, ..., x_d) \, dx_1 dx_2 ... dx_d$$


**Входные данные:**
- lower_bounds — вектор нижних пределов интегрирования по каждому измерению
- upper_bounds — вектор верхних пределов интегрирования по каждому измерению
- num_points — количество точек выборки
- func_type — тип интегрируемой функции (константа, линейная, произведение, квадратичная, экспонента)

**Выходные данные:**
- result — приближённое значение интеграла (вещественное число)

**Ограничения:**
- Размерности lower_bounds и upper_bounds совпадают и не равны нулю
- Для каждого измерения lower_bounds[i] < upper_bounds[i]
- num_points > 0

## 3. Базовый алгоритм (последовательная версия)

Алгоритм использует метод Монте-Карло с квазислучайной последовательностью Кронекера:

1. Вычисляется объём области интегрирования: V = prod_{i=1}^{d} (b_i - a_i)
2. Генерируется num_points точек в единичном гиперкубе [0,1)^d с помощью последовательности Кронекера
3. Каждая точка масштабируется в реальную область интегрирования: x_j = a_j + u_j * (b_j - a_j)
4. Для каждой точки вычисляется значение подынтегральной функции f(x)
5. Вычисляется среднее арифметическое значений функции
6. Результат интеграла: I ≈ V * (1/n) * sum_{i=1}^{n} f(x_i)

**Сложность по времени:** O(n * d), где n — количество точек, d — размерность

**Сложность по памяти:** O(d)

**Инварианты:** Результат сходится к точному значению интеграла при увеличении num_points.

## 4. Схема распараллеливания

Распараллеливание реализовано с помощью ручного разбиения диапазона и асинхронных задач.

**Декомпозиция данных:** Диапазон индексов [0, num_points) разбивается на непрерывные поддиапазоны, количество которых равно числу потоков.

**Формула разбиения:**
- points_per_thread = num_points / num_threads (целая часть)
- remainder = num_points % num_threads (остаток)
- Первые remainder потоков получают на одну точку больше

**Что делает каждый поток:** Вычисляет локальную сумму значений функции для своего поддиапазона точек.

**Хранение локальных результатов:** Каждая асинхронная задача (std::async) возвращает double через std::future.

**Синхронизация:**
- std::async с политикой std::launch::async гарантирует асинхронное выполнение
- std::future::get() блокирует поток до получения результата и объединяет локальные суммы

**Порядок join:** Не требуется явного join, так как std::future::get() дожидается завершения задачи.

## 5. Детали реализации

**Структура кода:**
- ops_stl.hpp — объявление класса NikitinAMonteCarloSTL, наследующего BaseTask
- ops_stl.cpp — реализация методов класса с использованием std::async/std::future

**Ключевые функции:**
- EvaluateFunction — вычисляет значение тестовой функции заданного типа
- KroneckerSequence — генерирует квазислучайную последовательность
- ComputePartialSum — вычисляет сумму на заданном поддиапазоне [start, end)

**Методы класса:**
- ValidationImpl — проверяет корректность входных данных
- RunImpl — выполняет распараллеленный вычислительный цикл

**Определение числа потоков:**
- std::thread::hardware_concurrency() — аппаратное число потоков
- При недоступности — значение по умолчанию 2
- Число потоков ограничено количеством точек (не больше num_points)

**Граничные случаи:**
- При num_points = 0 валидация возвращает false
- При несовпадении размерностей границ валидация возвращает false
- Для функций, требующих двух измерений (Product, Quadratic), при недостаточной размерности возвращается 0.0

**Использование памяти:** Точка хранится как локальный вектор размера dim, выделяемый в каждой задаче каждого потока.

## 8. Выводы

Реализация STL с использованием std::async и std::future обеспечивает ручное управление потоками для распараллеливания метода Монте-Карло. Разбиение диапазона на непрерывные поддиапазоны гарантирует отсутствие конфликтов при записи. Использование std::future::get() для объединения результатов упрощает код по сравнению с явным созданием std::thread и вызовом join(). Ускорение растёт с увеличением числа потоков, но эффективность снижается из-за накладных расходов на создание и синхронизацию асинхронных задач.
