## 1. Введение

Метод Монте-Карло — это вычислительный алгоритм, использующий случайную выборку для аппроксимации значений многомерных интегралов. Данная реализация использует Intel oneTBB (Threading Building Blocks) для распараллеливания основного цикла генерации точек и суммирования значений функции с помощью алгоритма parallel_reduce. Ожидаемый результат — численное приближение интеграла заданной функции по гиперпрямоугольной области, полученное с ускорением относительно последовательной версии.

## 2. Постановка задачи

Вычисление определённого интеграла вида:

$$I = \int_{a_1}^{b_1} \int_{a_2}^{b_2} ... \int_{a_d}^{b_d} f(x_1, x_2, ..., x_d) \, dx_1 dx_2 ... dx_d$$

**Входные данные:**
- lower_bounds — вектор нижних пределов интегрирования по каждому измерению
- upper_bounds — вектор верхних пределов интегрирования по каждому измерению
- num_points — количество точек выборки
- func_type — тип интегрируемой функции (константа, линейная, произведение, квадратичная, экспонента)

**Выходные данные:**
- result — приближённое значение интеграла (вещественное число)

**Ограничения:**
- Размерности lower_bounds и upper_bounds совпадают и не равны нулю
- Для каждого измерения lower_bounds[i] < upper_bounds[i]
- num_points > 0

## 3. Базовый алгоритм (последовательная версия)

Алгоритм использует метод Монте-Карло с квазислучайной последовательностью Кронекера:

1. Вычисляется объём области интегрирования: V = prod_{i=1}^{d} (b_i - a_i)
2. Генерируется num_points точек в единичном гиперкубе [0,1)^d с помощью последовательности Кронекера
3. Каждая точка масштабируется в реальную область интегрирования: x_j = a_j + u_j * (b_j - a_j)
4. Для каждой точки вычисляется значение подынтегральной функции f(x)
5. Вычисляется среднее арифметическое значений функции
6. Результат интеграла: I ≈ V * (1/n) * sum_{i=1}^{n} f(x_i)

**Сложность по времени:** O(n * d), где n — количество точек, d — размерность

**Сложность по памяти:** O(d)

**Инварианты:** Результат сходится к точному значению интеграла при увеличении num_points.

## 4. Схема распараллеливания

Распараллеливание реализовано с помощью алгоритма tbb::parallel_reduce. Диапазон итераций [0, num_points) разбивается на поддиапазоны, которые распределяются между потоками.

**Выбранный примитив:** parallel_reduce — выполняет параллельное редуцирование (суммирование) значений функции по всем точкам выборки.

**Диапазон работы:** blocked_range<int>(0, num_points) — диапазон индексов точек, автоматически разбиваемый TBB на поддиапазоны.

**Размер зёрна (grainsize):** не указан явно, используется значение по умолчанию (автоматический выбор TBB).

**Планировщик (partitioner):** не указан явно, используется auto_partitioner (адаптивное разбиение).

**Схема работы:**
- Каждый поток получает поддиапазон индексов
- Внутри поддиапазона поток вычисляет локальную сумму
- После завершения всех поддиапазонов результаты объединяются с помощью функции сложения

## 5. Детали реализации

**Структура кода:**
- ops_tbb.hpp — объявление класса NikitinAMonteCarloTBB, наследующего BaseTask
- ops_tbb.cpp — реализация методов класса с использованием oneTBB

**Ключевые функции:**
- EvaluateFunction — вычисляет значение тестовой функции заданного типа
- KroneckerSequence — генерирует квазислучайную последовательность

**Методы класса:**
- ValidationImpl — проверяет корректность входных данных
- RunImpl — выполняет распараллеленный вычислительный цикл через parallel_reduce

**Компоненты parallel_reduce:**
- Лямбда вычисления: принимает диапазон range и локальную сумму local_sum, возвращает обновлённую локальную сумму
- Функция объединения: складывает результаты от разных потоков

**Граничные случаи:**
- При num_points = 0 валидация возвращает false
- При несовпадении размерностей границ валидация возвращает false
- Для функций, требующих двух измерений (Product, Quadratic), при недостаточной размерности возвращается 0.0

**Использование памяти:** Точка хранится как локальный вектор размера dim, выделяемый в каждой лямбде каждого потока.

## 7. Результаты и обсуждение

### 7.1 Корректность

Корректность проверяется на наборе тестов с известными аналитическими значениями интегралов. Результат параллельной версии сравнивается с результатом последовательной версии. Совпадение результатов в пределах вычислительной погрешности подтверждает корректность распараллеливания. Использование parallel_reduce гарантирует потокобезопасное накопление суммы.

## 8. Выводы

Реализация TBB с использованием parallel_reduce показывает значительное ускорение по сравнению с последовательной версией. Ускорение, превышающее количество ядер (4.17x при 4 потоках), объясняется эффектами кэширования и особенностями измерения. Автоматическое разбиение диапазонов TBB эффективно справляется с равномерной вычислительной нагрузкой. Отсутствие явного указания grainsize позволяет планировщику адаптивно выбирать оптимальный размер поддиапазонов.
