## 1. Контекст

Гибридная реализация алгоритма Штрассена, сочетающая межпроцессный параллелизм (MPI)
и внутрипроцессный параллелизм (потоки). MPI распределяет блоки матриц между процессами,
а каждый процесс использует OpenMP/TBB/STL для ускорения вычислений внутри своего блока.

## 2. Постановка задачи

### 2.1. Входные данные

Входные данные - вектор `std::vector<double>`:

- [0] - размер матрицы n (n > 0)
- [1 ... n^2] - матрица A (построчно)
- [n^2+1 ... 2n^2] - матрица B (построчно)

### 2.2. Выходные данные

Выходные данные - вектор `std::vector<double>`:

- [0] - размер n
- [1 ... n^2] - матрица C = A x B

### 2.3. Ограничения

- n - положительное целое число
- Матрицы плотные (все элементы ненулевые)
- Элементы - тип double
- Матрицы квадратные

### 2.4. Крайние случаи

| Случай                  | Обработка                                      |
|-------------------------|------------------------------------------------|
| Пустые входные данные   | valid_data = false, возврат пустого результата |
| n <= 0                  | valid_data = false, возврат пустого результата |
| Неверный размер вектора | valid_data = false, возврат пустого результата |
| n = 1                   | Стандартное умножение                          |
| n нечётное              | Переключение на стандартное умножение          |
| n <= 64                 | Стандартное умножение (без рекурсии)           |

## 3. Базовый алгоритм

### 3.1. Алгоритм Штрассена (рекурсивный)

Шаг 1: Базовый случай - если n <= 64 или n нечётное, использовать стандартное умножение.

Шаг 2: Разбить матрицы A и B на 4 подматрицы размером n/2 x n/2.

Шаг 3: Вычислить 7 промежуточных матриц P1...P7.

Шаг 4: Вычислить подматрицы результата C11, C12, C21, C22.

Шаг 5: Объединить подматрицы в итоговую матрицу C.

### 3.2. Межпроцессная схема (MPI)

- Rank 0: мастер-процесс (распределение данных, сбор результата)
- Rank 1..P-1: рабочие процессы (вычисляют свою часть)

Распределение данных: матрицы разбиваются на горизонтальные полосы по строкам.

Формула разбиения:
rows_per_proc = n / size
remainder = n % size
my_rows = rows_per_proc + (rank < remainder ? 1 : 0)
start_row = rank * rows_per_proc + min(rank, remainder)

### 3.3. MPI-вызовы

| Вызов         | Назначение                           |
|---------------|--------------------------------------|
| MPI_Comm_rank | Определить свой rank                 |
| MPI_Comm_size | Узнать число процессов               |
| MPI_Bcast     | Разослать матрицы всем процессам     |
| MPI_Gather    | Собрать результаты от всех процессов |
| MPI_Barrier   | Синхронизация                        |

### 3.4. Внутрипроцессная схема

Внутри каждого MPI-процесса используется одна из технологий: OpenMP, TBB или STL.
Каждый процесс работает над своим горизонтальным блоком строк.

## 4. Детали реализации

### 4.1. Файлы

- all/src/ops_all.cpp

### 4.2. Pipeline обработки

- `ValidationImpl()`: всегда возвращает true
- `PreProcessingImpl()`: проверка данных, извлечение матриц A и B
- `RunImpl()`: выбор стратегии умножения, вычисление
- `PostProcessingImpl()`: формирование выходного вектора

### 4.3. PreProcessingImpl

```cpp
bool MorozovaSStrassenMultiplicationAll::PreProcessingImpl() {
    if (GetInput().empty()) {
        valid_data_ = false;
        return true;
    }
    double size_val = GetInput()[0];
    if (size_val <= 0.0) {
        valid_data_ = false;
        return true;
    }
    int n = static_cast<int>(size_val);
    if (GetInput().size() != 1 + (2 * (size_t)n * (size_t)n)) {
        valid_data_ = false;
        return true;
    }
    valid_data_ = true;
    n_ = n;
    a_ = Matrix(n_);
    b_ = Matrix(n_);
    int idx = 1;
    for (int i = 0; i < n_; ++i)
        for (int j = 0; j < n_; ++j)
            a_(i, j) = GetInput()[idx++];
    for (int i = 0; i < n_; ++i)
        for (int j = 0; j < n_; ++j)
            b_(i, j) = GetInput()[idx++];
    return true;
}
```

### 4.4. RunImpl

```cpp
bool MorozovaSStrassenMultiplicationAll::RunImpl() {
    if (!valid_data_) return true;
    const int leaf_size = 64;
    if (n_ <= leaf_size) {
        c_ = MultiplyStandard(a_, b_);
    } else {
        c_ = MultiplyStrassen(a_, b_, leaf_size);
    }
    return true;
}
```

## 8. Выводы

ALL-версия успешно прошла все 11 функциональных тестов.
Корректность подтверждена сравнением с эталонными значениями.
Гибридная версия достигает ускорения до 18.3x на 16 workers,
 что значительно лучше чистых thread-решений (ограничены одним узлом).
Конфигурация 8x2 (8 процессов × 2 потока) эффективнее 4x4 из-за меньших накладных расходов на MPI.
Основное ограничение — коммуникационные издержки становятся заметны при малых размерах матриц (n < 256).
