## 1. Контекст

Данная реализация является эталонной последовательной версией алгоритма поразрядной сортировки с последующим применением четно-нечетного слияния Бэтчера. Она служит базовой линией для проверки корректности и расчета ускорения в параллельных версиях.

## 2. Постановка задачи

На вход подается вектор целых чисел `std::vector<int>` произвольного размера (в том числе содержащий отрицательные элементы). Необходимо отсортировать его по возрастанию.

## 3. Базовый алгоритм

1. **Дополнение:** Вектор дополняется максимальными значениями `std::numeric_limits<int>::max()` до ближайшей степени двойки, так как сеть слияния Бэтчера работает с массивами размером $2^k$.
2. **Разделение и сортировка:** Массив делится на две равные части. Каждая часть сортируется с помощью LSD Radix Sort (по основанию 256, 4 прохода по 8 бит). Для корректной работы с отрицательными числами старший бит инвертируется.
3. **Слияние Бэтчера:** Две отсортированные половины объединяются с помощью алгоритма четно-нечетного слияния Бэтчера.
4. **Обрезка:** Дополнительные элементы удаляются, размер возвращается к исходному.

Асимптотика по времени: $O(N + N \log N)$ (поразрядная сортировка линейна, слияние двух отсортированных массивов сетью Бэтчера требует $O(N \log N)$ операций).
Асимптотика по памяти: $O(N)$ на временные буферы.

## 4. Детали реализации

Файлы: `seq/include/ops_seq.hpp`, `seq/src/ops_seq.cpp`

```cpp
// File: seq/src/ops_seq.cpp
  size_t pow2 = 1;
  while (pow2 < original_size) pow2 <<= 1;
  data.resize(pow2, std::numeric_limits<int>::max());

  size_t half = pow2 / 2;
  auto half_dist = static_cast<std::ptrdiff_t>(half);
  std::vector<int> left(data.begin(), data.begin() + half_dist);
  std::vector<int> right(data.begin() + half_dist, data.end());

  RadixSort(left);
  RadixSort(right);
```

Фрагмент демонстрирует логику подготовки данных. `ValidationImpl` проверяет, что входной вектор не пуст. `PreProcessingImpl` выделяет память под результат. `RunImpl` производит основную вычислительную работу.

## 5. Проверка корректности

Корректность алгоритма подтверждается функциональными тестами. Тестируются крайние случаи: массив из 1 элемента, уже отсортированный массив, обратно отсортированный, массив с одинаковыми элементами и массив с отрицательными числами.
