## 1. Контекст

Ручная реализация многопоточности средствами стандартной библиотеки C++ (`std::thread`), требующая явного распределения диапазонов данных и синхронизации потоков.

## 2. Постановка задачи

Исключить высокоуровневые абстракции (OpenMP, TBB) и реализовать параллелизм, вручную управляя жизненным циклом потоков.

## 3. Базовый алгоритм

Аналогичен эталону: дополнение массива, поразрядная сортировка половин, слияние Бэтчера.

## 4. Схема распараллеливания

1. При сортировке половин создается один `std::thread` для вызова `RadixSort` на левой половине, в то время как главная нить обрабатывает правую. Вызов `join()` блокирует продолжение до завершения побочного потока.
2. В алгоритме слияния используется вспомогательная функция `RunInThreads`. Общий объем задач делится на равные блоки (chunks) пропорционально числу потоков.

## 5. Детали реализации

Файлы: `stl/include/ops_stl.hpp`, `stl/src/ops_stl.cpp`

```cpp
// File: stl/src/ops_stl.cpp (из анонимного пространства имен)
for (size_t thread_idx = 0; thread_idx < num_threads; ++thread_idx) {
  // Вычисление start_idx и end_idx
  if (start_idx < end_idx) {
    threads.emplace_back([start_idx, end_idx, &task_func]() {
      for (size_t task_idx = start_idx; task_idx < end_idx; ++task_idx) task_func(task_idx);
    });
  }
}
for (auto &thread_obj : threads) thread_obj.join();
```

Код демонстрирует правильный шаблон работы с `std::thread`: сначала все потоки помещаются в std::vector (цикл emplace_back), и только потом, в отдельном цикле, вызывается join(). Это предотвращает принудительную сериализацию, гарантируя реальный параллелизм. Отсутствие мьютексов обосновано тем, что потоки работают со строго непересекающимися индексами массива.
