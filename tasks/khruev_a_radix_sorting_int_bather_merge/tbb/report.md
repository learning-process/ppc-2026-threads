## 1. Контекст

Реализация алгоритма с использованием библиотеки TBB, которая предлагает
задачно-ориентированную парадигму и балансировку нагрузки через work-stealing.

## 2. Постановка задачи

Адаптация базового SEQ-алгоритма к примитивам TBB с учетом концепции разделения
данных на диапазоны (blocked ranges).

## 3. Базовый алгоритм

Массив добивается до $2^k$, бьётся пополам, каждая половина сортируется Radix
Sort, а затем сливается четно-нечетным слиянием.

## 4. Схема распараллеливания

1. Для независимой сортировки двух половин используется `tbb::parallel_invoke`.
2. Для распараллеливания циклов слияния Бэтчера используется `tbb::parallel_for`
в комбинации с `tbb::blocked_range`. Разбиение (partitioner) и размер зерна
(grainsize) оставлены по умолчанию (auto partitioner), что позволяет
планировщику TBB самостоятельно принимать решения об оптимальном разделении
задач в зависимости от доступных ресурсов.

## 5. Детали реализации

Файлы: `tbb/include/ops_tbb.hpp`, `tbb/src/ops_tbb.cpp`

```cpp
// File: tbb/src/ops_tbb.cpp
tbb::parallel_for(tbb::blocked_range<size_t>(0, num_blocks), [&](const tbb::blocked_range<size_t> &r) {
  for (size_t block_idx = r.begin(); block_idx != r.end(); ++block_idx) {
    size_t i = po + (block_idx * 2 * po);
    for (size_t j = 0; j < po; ++j) {
      CompareExchange(a, i + j, i + j + po);
    }
  }
});
```

Данный фрагмент иллюстрирует распараллеливание основного шага сети Бэтчера.
Использование лямбда-функции и объекта `blocked_range` обеспечивает прозрачное
разделение итераций между рабочими потоками `TBB`.

## 6. Проверка корректности

Сравнение выполняется относительно SEQ. TBB-версия проходит тот же набор
функциональных тестов, подтверждая детерминированность и потокобезопасность
операций CompareExchange.

## 9. Выводы

Оптимальной конфигурацией для TBB оказались 2 потока, где достигнута
максимальная эффективность ~64–67% и ускорение ~1.34x. Увеличение размера пула
TBB до 4 и 8 потоков не дало прироста (время стагнирует или слегка деградирует).
Это объясняется тем, что алгоритм жестко разделен на 2 крупные независимые
подзадачи (поразрядная сортировка двух половин), а накладные расходы
планировщика TBB на распределение мелких итераций слияния превышают выгоду от
использования дополнительных логических потоков.
