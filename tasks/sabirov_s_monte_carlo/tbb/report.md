## 1. Контекст

Версия на Intel oneTBB переводит метод Монте-Карло на задачно-ориентированный подход. Вместо явного
управления потоками используется высокоуровневый алгоритм `tbb::parallel_reduce`, который сам разбивает
диапазон индексов на чанки и выполняет редукцию частичных сумм.

## 2. Постановка задачи

Та же задача, что и в SEQ: оценка интеграла

> `I ≈ V · (1 / N) · Σ f(x^(k))`,

на гиперпрямоугольнике `Ω = [a_i, b_i]` с поддержкой восьми типов подынтегральных функций из `FuncType`.

## 3. Базовый алгоритм

Сохраняется: суммирование значений `f(x^(k))` по равномерно распределённым в `Ω` точкам с делением
на `N` и умножением на объём `V`.

## 4. Схема распараллеливания

Распределение работы делегировано планировщику TBB:

1. **Диапазон индексов:** `tbb::blocked_range<int>(0, N)` представляет множество выборок, которое
   автоматически режется `auto_partitioner` на блоки подходящего размера.
2. **Локальное накопление:** лямбда-«рабочий» получает блок `[r.begin(), r.end())`. Для блока
   создаётся локальный `std::mt19937`, инициализируемый `std::seed_seq` от `(r.begin(), r.end(), N)`.
   Такой посев обеспечивает воспроизводимость распределения нагрузки между чанками и независимость
   генераторов потоков. В цикле блока генерируются точки и накапливается локальная сумма.
3. **Слияние:** комбайнер `(a, b) -> a + b` объединяет частичные суммы блоков попарно, что
   соответствует ассоциативной редукции и не зависит от порядка обхода.
4. **Контроль числа потоков:** в начале `RunImpl` создаётся
   `oneapi::tbb::global_control(max_allowed_parallelism, max(1, ppc::util::GetNumThreads()))`. Объект
   живёт всё время выполнения `RunImpl` и ограничивает параллелизм арены TBB значением
   `ppc::util::GetNumThreads()`. Это синхронизирует политику потоков с другими реализациями курса.

## 5. Детали реализации

Файлы: [tbb/include/ops_tbb.hpp](tbb/include/ops_tbb.hpp), [tbb/src/ops_tbb.cpp](tbb/src/ops_tbb.cpp).

Точечная функция вычисления `detail::EvaluateAt` вынесена в одноимённое пространство имён в заголовке
TBB-варианта, чтобы её можно было обозревать и переиспользовать из лямбды-редуктора.

```cpp
const size_t max_threads = static_cast<size_t>(std::max(1, ppc::util::GetNumThreads()));
oneapi::tbb::global_control gc(oneapi::tbb::global_control::max_allowed_parallelism, max_threads);

const double sum = tbb::parallel_reduce(
    tbb::blocked_range<int>(0, n_samples), 0.0,
    [&](const tbb::blocked_range<int> &r, double init) {
      double local = init;
      std::vector<double> point(static_cast<size_t>(dims));
      std::seed_seq seed{static_cast<uint32_t>(r.begin()),
                         static_cast<uint32_t>(r.end()),
                         static_cast<uint32_t>(n_samples)};
      std::mt19937 gen(seed);
      for (int i = r.begin(); i != r.end(); ++i) {
        for (int j = 0; j < dims; ++j) {
          point[j] = dists[j](gen);
        }
        local += detail::EvaluateAt(ftype, point);
      }
      return local;
    },
    [](double a, double b) { return a + b; });
```

Вектор `dists` собирается один раз до `parallel_reduce` и используется задачами только на чтение
(вызов `operator()` на копии распределения внутри лямбды не модифицирует общий объект). Это
устраняет необходимость дополнительной синхронизации.

## 6. Проверка корректности

TBB-вариант входит в общий набор тестов `ppc_func_tests` вместе с SEQ, OMP, STL и ALL и проходит
13 сценариев с допуском `5.0 / sqrt(N)`. Использование детерминированного посева через `seed_seq` от
границ блока облегчает воспроизведение поведения при отладке.

`parallel_reduce` даёт плавное масштабирование: 1.76× при двух потоках, 6.6× при восьми, эффективность
держится в коридоре 78–91%. Лёгкий «налог» в `T(1) > T_seq` (≈ 11%) — это стоимость собственного
рантайма oneTBB: запуск арены, постановка/выборка задач, инициализация `global_control`; в SEQ ничего
из этого нет. Разница между `pipeline` и `task_run` укладывается в единицы миллисекунд — для нашего
алгоритма `Validation/PreProcessing/PostProcessing` ничтожны на фоне 10 млн бросков точек.

## 9. Выводы

`parallel_reduce` хорошо ложится на задачу численного интегрирования: независимые броски точек и
коммутативно-ассоциативная редукция. После выравнивания политики потоков через `global_control` TBB
реагирует на `PPC_NUM_THREADS` так же, как остальные варианты курса, что делает сравнение корректным.
