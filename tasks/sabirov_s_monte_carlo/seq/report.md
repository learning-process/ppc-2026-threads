## 1. Контекст

Метод Монте-Карло — стохастический способ численного интегрирования, особенно удобный для функций большого числа
переменных. На фиксированной области интегрирования сходимость метода имеет порядок `O(1 / sqrt(N))` и не зависит от
размерности, что выгодно отличает его от классических квадратурных формул.

Последовательная реализация задаёт эталонный пайплайн `Validation → PreProcessing → Run → PostProcessing` и формирует
базовое время, относительно которого оцениваются параллельные варианты.

## 2. Постановка задачи

**Вход:** структура `MCInput` со следующими полями:

- `lower`, `upper` — векторы границ интегрирования (`lower[i] < upper[i]`),
- `num_samples` — количество выборок `N > 0`,
- `func_type` — тип подынтегральной функции `FuncType`.

**Выход:** действительное число — оценка интеграла.

**Допустимые типы функций** (`enum class FuncType`):

| Значение | Подынтегральная функция |
| --- | --- |
| `kLinear` | `f(x) = Σ x_i` |
| `kSumCubes` | `f(x) = Σ x_i^3` |
| `kCosProduct` | `f(x) = Π cos(x_i)` |
| `kExpNeg` | `f(x) = exp(-Σ x_i)` |
| `kMixedPoly` | `f(x) = Σ (x_i^2 + x_i)` |
| `kSinSum` | `f(x) = Σ sin(x_i)` |
| `kSqrtSum` | `f(x) = Σ sqrt(x_i)` |
| `kQuarticSum` | `f(x) = Σ x_i^4` |

**Ограничения и крайние случаи:** размерность `1 ≤ dim ≤ 10`, `lower.size() == upper.size()`,
все компоненты строго удовлетворяют `lower[i] < upper[i]`, число выборок положительно, `func_type` лежит в
указанном диапазоне.

## 3. Базовый алгоритм

Пусть `Ω = [a_1, b_1] × ... × [a_d, b_d]` — область интегрирования объёма `V = Π (b_i − a_i)`.
Тогда

> `I ≈ V · (1 / N) · Σ f(x^(k))`,

где `x^(k)` — независимые точки, равномерно распределённые в `Ω`. Стандартная оценка погрешности
имеет вид `σ_I ≈ V · σ_f / sqrt(N)`. В тестах используется допуск `5.0 / sqrt(N)`.

Асимптотика по времени: `O(N · d)` арифметических операций (`d` — размерность).
Асимптотика по памяти: `O(d)` — хранение одной точки, набора распределений и границ.

## 4. Детали реализации

Файлы: [seq/include/ops_seq.hpp](seq/include/ops_seq.hpp), [seq/src/ops_seq.cpp](seq/src/ops_seq.cpp).

- В `ValidationImpl` проверяются размерности, положительность `num_samples`, корректность интервалов и
  диапазон `func_type`, а также ограничение по числу измерений `dim ≤ 10`.
- В `PreProcessingImpl` входные параметры копируются в члены задачи (`lower_`, `upper_`, `num_samples_`,
  `func_type_`), `GetOutput()` обнуляется.
- В `RunImpl` инициализируется генератор `std::mt19937`, по каждому измерению строится
  `std::uniform_real_distribution<double>([a_i, b_i])`, в цикле генерируются точки и накапливается сумма
  значений подынтегральной функции, после чего результат масштабируется на объём `V` и делится на `N`.
- Вычисление значения функции вынесено в анонимный `namespace` (`EvaluateAt` и набор `EvalLinear`, `EvalSumCubes`,
  ...). Такой подход снижает когнитивную сложность `RunImpl` и единообразно используется во всех вариантах.
- В `PostProcessingImpl` дополнительной работы не требуется — результат уже находится в `GetOutput()`.

```cpp
std::vector<double> point(dims);
double sum = 0.0;
for (int i = 0; i < num_samples_; ++i) {
  for (int j = 0; j < dims; ++j) {
    point[j] = dists[j](gen);
  }
  sum += EvaluateAt(func_type_, point);
}
GetOutput() = volume * sum / static_cast<double>(num_samples_);
```

Этот фрагмент представляет ядро алгоритма: генерация очередной точки и накопление вклада в выборочную сумму.

## 5. Проверка корректности

Реализация SEQ принимается за эталон. Корректность подтверждается функциональными тестами
`ppc_func_tests` — 13 сценариев покрывают все восемь подынтегральных функций в 1D/2D/3D с допуском
`5.0 / sqrt(N)`. Дополнительно отрабатываются граничные случаи `ValidationImpl`: вырожденные интервалы,
несовпадение размерностей, `num_samples ≤ 0`, недопустимый `func_type`.

Основную долю времени занимает цикл по выборкам: на каждом шаге выполняется `d` обращений к
`uniform_real_distribution` и одно вычисление значения функции. При `dim = 3` и `N = 10^7` цикл выполняется
30 миллионов раз, и его линейная часть полностью доминирует. Разница `pipeline − task_run` ≈ 3 мс — это
суммарная стоимость `Validation + PreProcessing + PostProcessing`, которая в Monte Carlo сводится к
короткой валидации входа и копированию двух векторов длины `d ≤ 10`. На фоне 30 млн арифметических
операций цикла эта величина естественно тонет в шуме.
