## 1. Контекст

Реализация на `std::thread` отличается от OMP/TBB тем, что разработчик полностью контролирует
разбиение работы и явно управляет жизненным циклом потоков. Это даёт прозрачную картину
накладных расходов и удобно для аккуратной демонстрации идеи распараллеливания метода Монте-Карло.

## 2. Постановка задачи

Та же задача: оценка интеграла

> `I ≈ V · (1 / N) · Σ f(x^(k))`,

на гиперпрямоугольнике `Ω = [a_i, b_i]` с поддержкой восьми типов функций из `FuncType`.

## 3. Базовый алгоритм

Совпадает с SEQ: суммирование значений `f(x^(k))` на точках, равномерно распределённых в `Ω`,
с делением на `N` и умножением на объём `V`.

## 4. Схема распараллеливания

Подход «разбей и склей» вручную:

1. **Число потоков:** берётся из `ppc::util::GetNumThreads()` (нижняя граница — 1), это та же
   единая точка управления, что и в OMP/TBB.
2. **Разбиение диапазона:** интервал индексов `[0, N)` режется на `num_threads` непересекающихся
   полуинтервалов с границами `start = (t · N) / num_threads`, `end = ((t + 1) · N) / num_threads`.
   Это даёт сбалансированное и детерминированное распределение даже при `N`, не кратном числу потоков.
3. **Локальные ресурсы:** у каждого потока собственный `std::mt19937`, инициализируемый
   `std::seed_seq` от `(ti, start, end, N)`, локальный буфер `point` и аккумулятор `local`. Записи в
   общий массив `partial_sums` идут строго по уникальному индексу потока — это исключает гонки и
   позволяет обойтись без атомиков и мьютексов.
4. **Сборка:** после `join()` всех потоков главный поток складывает `partial_sums` в общий `sum`
   и завершает формулой `V · sum / N`. Шаг редукции дешёвый — это просто сумма по числу потоков.

## 5. Детали реализации

Файлы: [stl/include/ops_stl.hpp](stl/include/ops_stl.hpp), [stl/src/ops_stl.cpp](stl/src/ops_stl.cpp).

```cpp
const int num_threads = std::max(1, ppc::util::GetNumThreads());
std::vector<double> partial_sums(static_cast<size_t>(num_threads), 0.0);
std::vector<std::thread> threads;
threads.reserve(static_cast<size_t>(num_threads));

for (int ti = 0; ti < num_threads; ++ti) {
  const int start = (ti * n_samples) / num_threads;
  const int end = ((ti + 1) * n_samples) / num_threads;
  threads.emplace_back([&, ti, start, end, dims, ftype]() {
    if (start >= end) {
      return;
    }
    std::vector<double> point(static_cast<size_t>(dims));
    std::seed_seq seed{static_cast<uint32_t>(ti), static_cast<uint32_t>(start),
                       static_cast<uint32_t>(end), static_cast<uint32_t>(n_samples)};
    std::mt19937 gen(seed);
    double local = 0.0;
    for (int i = start; i < end; ++i) {
      for (int j = 0; j < dims; ++j) {
        point[j] = dists[j](gen);
      }
      local += EvaluateAt(ftype, point);
    }
    partial_sums[ti] = local;
  });
}
for (auto &th : threads) {
  th.join();
}
```

Вычисление подынтегральной функции — те же `EvaluateAt` и набор `Eval*` в анонимном `namespace`,
что обеспечивает консистентность с SEQ и параллельными вариантами.

## 6. Проверка корректности

STL-вариант входит в общий набор тестов `ppc_func_tests` и проходит 13 сценариев с допуском
`5.0 / sqrt(N)`. Граничный случай `start >= end` (когда `num_threads > N`) обрабатывается ранним
возвратом потока без обращения к `partial_sums`.

На 1 потоке STL даже немного быстрее SEQ за счёт того, что генератор и буфер `point` живут в локальном
кадре потока, а не разделяются с управляющим кодом. До 4 потоков масштабирование почти линейное;
с 6–8 потоков на лёгком теле итерации проявляются накладные расходы на создание `std::thread` и
насыщение L3 — эффективность опускается в коридор 55–65%, ускорение продолжает расти, но медленнее.
Разница `pipeline − task_run` остаётся в единицах миллисекунд — `PreProcessing/PostProcessing` для
Monte Carlo дешевле собственно цикла на пять порядков.

## 9. Выводы

Ручное разбиение диапазона и собственный генератор у каждого потока дают предсказуемое поведение
и хорошее ускорение на основной задаче Монте-Карло. Привязка числа потоков к
`ppc::util::GetNumThreads()` делает результаты STL-варианта сопоставимыми с OMP и TBB по тому же
протоколу запуска.
