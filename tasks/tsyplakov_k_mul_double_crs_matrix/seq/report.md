## 1. Контекст

Умножение разреженных матриц — базовая операция в вычислительной линейной алгебре, широко применяемая в решении СЛАУ.
Также в конечно-элементных методах, графовых алгоритмах и машинном обучении. Использование разреженных форматов хранения
позволяет:

- Существенно экономить память за счёт хранения только ненулевых элементов (типичная плотность 1–10%);

- Сокращать количество арифметических операций — обрабатываются только ненулевые элементы, а не вся матрица целиком;

- Эффективно работать с матрицами, где плотность ненулевых элементов мала.

Последовательная версия является базой для измерения ускорения параллельных реализаций (OMP, TBB, STL, ALL).

## 2. Постановка задачи

### Входные данные

Две разреженные матрицы `A` и `B` в формате CRS. Структура `SparseMatrixCRS`:

- `values` (тип: `std::vector<double>`) - массив ненулевых значений (построчно)
- `col_index` (тип: `std::vector<int>`) - массив индексов столбцов для каждого значения
- `row_ptr` (тип: `std::vector<int>`) - указатели на начало каждой строки (размер `rows + 1`)
- `rows` (тип: `int`) - количество строк
- `cols` (тип: `int`) - количество столбцов

### Выходные данные

Результирующая матрица `C = A * B` в формате CRS.

### Ограничения

- количество столбцов матрицы A должно равняться количеству строк матрицы B (`A.cols == B.rows`).
- все индексы находятся в диапазоне `[0, cols - 1]`.
- Указатели `row_ptr` образуют неубывающую последовательность.
- `row_ptr[0]`, `row_ptr[rows] = nnz`- общее количество ненулевых элементов.
- размерности матриц положительны: `rows > 0`, `cols > 0`.

### Крайние случаи

- при нулевой матрице `B` результатом будет нулевая матрица.
- если `B` - единичная матрица, то `A * B = A`.
- если умножение диагональных матриц, то результат - диагональная матрица.

## 3. Базовый алгоритм

### Принцип работы

Алгоритм использует классическую схему умножения разреженных матриц в CRS-формате. Для каждой строки `i` матрицы `A`:

1. Берутся ненулевые элементы строки `i` (пары `(k, A_ik)`);
2. Для каждого такого элемента `A_ik` извлекается строка k матрицы B;
3. Выполняется накопление: `C[i][j] += A_ik * B[k][j]` для всех ненулевых `B[k][j]`.

### Пошаговое описание

1. **Инициализация:** Создаётся результирующая матрица `C` с размерами `A.rows × B.cols`. `row_ptr` заполняется нулями
   (позже будут вычислены смещения).

2. **Обход строк матрицы A:** Для каждой строки `i` от `0` до `A.rows-1`: Извлекаются ненулевые элементы строки `i` из
   матрицы `A`.

Создаётся ассоциативный контейнер `std::map<int, double>` для накопления результатов по столбцам `C`.

3. **Внутреннее умножение:** Для каждого ненулевого элемента `(k, A_ik)` из строки `i`: Из матрицы B извлекается строка
   `k` (все ненулевые элементы в этой строке).

Для каждого элемента `(j, B_kj)` из строки `k`: `temp_result[j] += A_ik * B_kj`

4. **Сохранение результатов строки:** Из `temp_result` удаляются значения, близкие к нулю `(|val| < eps)`. Оставшиеся
   значения и индексы столбцов добавляются в `values` и `col_index` матрицы `C`. Обновляется
   `row_ptr[i+1] = row_ptr[i] + (число ненулевых в строке)`.

5. **Завершение:** Устанавливается финальный указатель `row_ptr[A.rows] = values.size()`.

### Асимптотическая сложность

`nnz(A)` - количество ненулевых элементов в A. `nnz(B)` - количество ненулевых элементов в B. `nnz(C)` - количество
ненулевых элементов в C. `n` - размерность матриц.

**ВРЕМЯ:** `O(nnz(A) * avg_nnz_per_row(B) + nnz(C))`. В худшем случае: `O(n^3)`. В разреженном случае значительно
меньше.

**ПАМЯТЬ:** `O(nnz(A) + nnz(B) + nnz(C) + A.rows)`.

### Критерий корректности

Результат умножения должен совпадать с результатом умножения плотных матриц, преобразованных из CRS-формата. Допустимая
погрешность: `e = 100 × DBL_EPSILON = 2.22e-14`.

## 4. Детали реализации

### Файлы

- `seq/include/ops_seq.hpp` - заголовочный файл с объявлением класса.
- `seq/src/ops_seq.cpp` - реализация.

### Основные методы (наследуются от BaseTask)

- Метод `ValidationImpl()` - Проверка корректности входных данных (совместимость размерностей, положительность
  размеров).
- Метод `PreProcessingImpl()` - Инициализация выходной матрицы C с размерами A.rows × B.cols.
- Метод `RunImpl()` - Выполнение умножения матриц (основной алгоритм).
- Метод `PostProccessingImpl()` - Завершающие действия (в данной реализации не требуются).

### Ключевые фрагменты кода

Основной цикл умножения (`RunImpl`):

```cpp
for (int i = 0; i < a.rows; ++i) {
    int start_a = a.row_ptr[i];
    int end_a = a.row_ptr[i + 1];

    // Ассоциативный массив для накопления: столбец -> значение
    std::map<int, double> temp_result;

    // Для каждого ненулевого элемента в строке i матрицы A
    for (int k_idx = start_a; k_idx < end_a; ++k_idx) {
        int k = a.col_index[k_idx];      // столбец A = строка B
        double a_ik = a.values[k_idx];

        // Извлекаем строку k из матрицы B
        int start_b = b.row_ptr[k];
        int end_b = b.row_ptr[k + 1];

        for (int j_idx = start_b; j_idx < end_b; ++j_idx) {
            int j = b.col_index[j_idx];
            double b_kj = b.values[j_idx];
            temp_result[j] += a_ik * b_kj;
        }
    }

    // Сохранение ненулевых результатов строки i
    const double eps = std::numeric_limits<double>::epsilon() * 100;
    c.row_ptr[i] = static_cast<int>(c.values.size());

    for (const auto &[col, val] : temp_result) {
        if (std::abs(val) > eps) {
            c.col_index.push_back(col);
            c.values.push_back(val);
        }
    }
}
c.row_ptr[a.rows] = static_cast<int>(c.values.size());
```

Валидация входных данных (`ValidationImpl`):

```cpp
bool TsyplakovKTestTaskSEQ::ValidationImpl() {
    return (GetInput().a.cols == GetInput().b.rows) &&
           (GetInput().a.rows > 0) &&
           (GetInput().a.cols > 0) &&
           (GetInput().b.rows > 0) &&
           (GetInput().b.cols > 0);
}
```

### Особенности реализации

1. **Использование `std::map` для накопления**: `std::map<int, double>` автоматически сортирует по ключу (столбцу) и
   обеспечивает уникальность столбцов. Это упрощает код, но добавляет логарифмический фактор `O(log M)` на каждую
   операцию вставки.

2. **Отфильтровка малых значений**: Значения, по модулю меньшие `e = 100 * DBL_EPSILON`, не включаются в результат. Это
   защищает от накопления численного шума.

3. **Два прохода по A (в закомментированной части)**: В коде видна альтернативная реализация через
   `MultiplyRowByMatrix`. В текущей версии используется прямой проход с `std::map`, что более эффективно.

## 5. Проверка корректности

### Метод верификации

Результат умножения разреженных матриц сравнивается с эталоном — умножением плотных матриц. Для всех тестов выполняется
проверка:

1. Размерности выходной матрицы соответствуют `A.rows * B.cols`;
2. `row_ptr` имеет размер `rows + 1` и является неубывающим;
3. `values` и `col_index` имеют одинаковый размер.

### Функциональные тесты (4 тестовых случая)

| №   | Название    | Размеры   | Описание                               | Ожидаемый nnz |
| --- | ----------- | --------- | -------------------------------------- | ------------- |
| 1   | identity    | 2×2 × 2×2 | A = [[1,0],[0,1]], B = [[2,0],[0,3]]   | 2             |
| 2   | simple_2x2  | 2×2 × 2×2 | Разреженные матрицы с 3 ненулевыми в A | 3             |
| 3   | zero_matrix | 2×2 × 2×2 | B — нулевая матрица                    | 0             |
| 4   | sparse_3x3  | 3×3 × 3×3 | Разреженные матрицы сложной структуры  | 4             |

### Пример тестового случая (`simple_2x2`)

**Матрица А**:

```cpp
[1.0, 2.0]
[3.0, 0.0]
```

CRS: `values = [1.0, 2.0, 3.0]`, `col_index = [0, 0, 1]`, `row_ptr = [0, 1, 3]`.

**Матрица В**:

```cpp
[4.0, 0.0],
[5.0, 6.0]
```

CRS: `values = [4.0, 5.0, 6.0]`, `col_index = [0, 0, 1]`, `row_ptr = [0, 1, 3]`.

**Ожидаемый результат `C = A * B`**:

```cpp
[1*4 + 2*5, 1*0 + 2*6] = [4+10, 0+12] = [14.0, 12.0]
[3*4 + 0*5, 3*0 + 0*6] = [12.0, 0.0]
```

CRS: `values = [14.0, 12.0, 12.0]`, `col_index = [0, 1, 0]`, `row_ptr = [0, 2, 3]`.

Допустимая погрешность: `e = 1e-10` для сравнения значений с плавающей точкой.

1. **Размер задачи:** 100000×100000

3. **Baseline для параллельных версий:**

4. **Масштабируемость:**
    - При `n = 100000` вычислительная работа достаточна для эффективной параллелизации

## 8. Выводы

**Почему SEQ используется как эталон для ускорения:**

1. **Детерминированность**: При одинаковых входных данных результат всегда одинаков (с точностью до машинной
   погрешности);
2. **Простота верификации**: Отсутствие гонок данных и проблем синхронизации упрощает проверку корректности;
3. **Прозрачность**: Алгоритм напрямую отражает математическую формулу умножения матриц;
4. **Отсутствие оверхэда**: Нет затрат на создание потоков, распределение данных и синхронизацию;
5. **Измеримость**: Время выполнения служит честным знаменателем для расчёта ускорения параллельных версий.
