## 1. Контекст

Intel TBB (Threading Building Blocks) — библиотека параллельного программирования на C++, основанная на концепции задач
(tasks), а не потоков. TBB автоматически управляет пулом потоков и балансировкой нагрузки, предоставляя высокоуровневые
алгоритмические шаблоны, такие как parallel_for, parallel_reduce, parallel_scan и другие.

Ключевые особенности TBB:

- Work-stealing планировщик — потоки, завершившие свою работу, «воруют» задачи у других потоков
- Автоматическая балансировка нагрузки — не требует явного указания schedule
- Масштабируемость — код адаптируется к разному количеству ядер

В данной реализации TBB применяется для распараллеливания вычислений при умножении разреженных матриц.

## 2. Постановка задачи

### Входные данные

Две разреженные матрицы `A` и `B` в формате CRS. Структура `SparseMatrixCRS`:

- `values` (тип: `std::vector<double>`) - массив ненулевых значений (построчно)
- `col_index` (тип: `std::vector<int>`) - массив индексов столбцов для каждого значения
- `row_ptr` (тип: `std::vector<int>`) - указатели на начало каждой строки (размер `rows + 1`)
- `rows` (тип: `int`) - количество строк
- `cols` (тип: `int`) - количество столбцов

### Выходные данные

Результирующая матрица `C = A * B` в формате CRS.

### Ограничения

- количество столбцов матрицы A должно равняться количеству строк матрицы B (`A.cols == B.rows`).
- все индексы находятся в диапазоне `[0, cols - 1]`.
- Указатели `row_ptr` образуют неубывающую последовательность.
- `row_ptr[0]`, `row_ptr[rows] = nnz`- общее количество ненулевых элементов.
- размерности матриц положительны: `rows > 0`, `cols > 0`.

### Крайние случаи

- при нулевой матрице `B` результатом будет нулевая матрица.
- если `B` - единичная матрица, то `A * B = A`.
- если умножение диагональных матриц, то результат - диагональная матрица.

### Особенности TBB-версии

- Распараллеливание через `tbb::parallel_for` с `tbb::blocked_range`
- Work-stealing планировщик для автоматической балансировки
- Вычислительное ядро вынесено в отдельную функцию `ComputeRow`
- Используется `std::unordered_map` для накопления результатов

## 3. Базовый алгоритм

### Принцип работы

Алгоритм использует классическую схему умножения разреженных матриц в CRS-формате. Для каждой строки `i` матрицы `A`:

1. Берутся ненулевые элементы строки `i` (пары `(k, A_ik)`);
2. Для каждого такого элемента `A_ik` извлекается строка k матрицы B;
3. Выполняется накопление: `C[i][j] += A_ik * B[k][j]` для всех ненулевых `B[k][j]`.

### Пошаговое описание

1. **Инициализация:** Создаются временные структуры для хранения результатов каждой строки:

`row_values` — вектор векторов значений для каждой строки `row_cols` — вектор векторов индексов столбцов для каждой
строки

2. **Параллельный обход строк матрицы A:** TBB разбивает диапазон строк `[0, rows)` на поддиапазоны
   `tbb::blocked_range`. Каждый поддиапазон обрабатывается одним потоком.

3. **Внутреннее умножение (ComputeRow):** Для строки `i` создаётся локальный аккумулятор
   `std::unordered_map<int, double>`, куда накапливаются произведения.

4. **Фильтрация малых значений:** Из аккумулятора удаляются значения, близкие к нулю `(|val| < 1e-12)`. Это защищает от
   накопления численного шума.

5. **Сохранение результатов строки:** Оставшиеся значения и индексы столбцов сохраняются в `row_values[i]` и
   `row_cols[i]`.

6. **Формирование выходной матрицы:** После завершения всех параллельных итераций вычисляются смещения `row_ptr` и
   данные копируются в выходную матрицу `C`.

### Асимптотическая сложность

| Метрика                   | Формула                                             | Примечание                                                          |
| ------------------------- | --------------------------------------------------- | ------------------------------------------------------------------- |
| **Время (параллельное)**  | `T_parallel = T_compute / P + T_overhead`           | T_overhead — создание задач, разбиение, синхронизация               |
| **Время (вычисления)**    | `T_compute ≈ nnz(A) × avg_nnz_per_row(B) + nnz(C)`  | Для диагональных матриц: = 100,000 × 1 + 100,000 = 200,000 операций |
| **Время (худший случай)** | `O(n^3 / P)`                                        | Для плотных матриц размером n×n                                     |
| **Память**                | `O(nnz(A) + nnz(B) + nnz(C) + A.rows + P × buffer)` | buffer — память под локальные unordered_map                         |

### Критерий корректности

Результат умножения должен совпадать с результатом умножения плотных матриц, преобразованных из CRS-формата. Допустимая
погрешность: `e = 1e-12`.

## 4. Схема распараллеливания

### Распараллеливаемая область

Внешний цикл по строкам матрицы `A` является идеальным кандидатом для распараллеливания:

- Вычисления для каждой строки полностью независимы

- Отсутствуют гонки данных при записи в разные строки

- Нет необходимости в синхронизации между итерациями

### TBB-примитивы

| Примитив                  | Использование               | Назначение                                      |
| ------------------------- | --------------------------- | ----------------------------------------------- |
| `tbb::parallel_for`       | `parallel_for(range, body)` | Параллельное выполнение итераций над диапазоном |
| `tbb::blocked_range<int>` | `blocked_range(0, rows)`    | Разбивает диапазон на поддиапазоны              |
| **Grainsize**             | По умолчанию                | Автоматический выбор для минимизации оверхэда   |
| **Планировщик**           | Work-stealing               | Кража задач для балансировки нагрузки           |

### Переменные и их атрибуты

| Переменная           | Доступ                              | Обоснование                      |
| -------------------- | ----------------------------------- | -------------------------------- |
| `a`                  | Только чтение (shared)              | Все потоки читают одну матрицу A |
| `b`                  | Только чтение (shared)              | Все потоки читают одну матрицу B |
| `row_values`         | Запись по индексу i (разные строки) | Гонок нет — разные строки        |
| `row_cols`           | Запись по индексу i (разные строки) | Аналогично row_values            |
| `acc` (в ComputeRow) | Локальная                           | Уникальна для каждой строки      |

### Синхронизация

- `parallel_for` неявно синхронизирует все задачи

- Дополнительных барьеров или мьютексов не требуется

### Ключевые фрагменты кода

Основной параллельный цикл:

```cpp
tbb::parallel_for(tbb::blocked_range<int>(0, rows), [&](const tbb::blocked_range<int> &range) {
    for (int i = range.begin(); i < range.end(); ++i) {
        ComputeRow(a, b, i, row_values[i], row_cols[i]);
    }
});
```

Функция ComputeRow (вычислительное ядро):

```cpp
void ComputeRow(const SparseMatrixCRS &a, const SparseMatrixCRS &b,
                int row, std::vector<double> &values, std::vector<int> &cols) {
    std::unordered_map<int, double> acc;

    for (int idx_a = a.row_ptr[row]; idx_a < a.row_ptr[row + 1]; ++idx_a) {
        const int k = a.col_index[idx_a];
        const double val_a = a.values[idx_a];

        for (int idx_b = b.row_ptr[k]; idx_b < b.row_ptr[k + 1]; ++idx_b) {
            const int j = b.col_index[idx_b];
            acc[j] += val_a * b.values[idx_b];
        }
    }

    values.reserve(acc.size());
    cols.reserve(acc.size());

    for (const auto &[col, val] : acc) {
        if (std::fabs(val) > 1e-12) {
            cols.push_back(col);
            values.push_back(val);
        }
    }
}
```

### Особенности реализации

- Вынос ComputeRow в отдельную функцию — улучшает читаемость и тестируемость

- Использование `std::unordered_map` — средняя сложность вставки `O(1)` против `O(log n)` у map

- Промежуточное хранение результатов — каждая строка накапливается локально, затем объединяется

- Резервирование памяти — `reserve(acc.size())` уменьшает количество переаллокаций

- Автоматический выбор `grainsize — TBB` сам определяет оптимальный размер задач

### Анализ производительности

Ключевое наблюдение: TBB-версия работает медленнее последовательной в 1.6-1.8 раза. Однако это значительно лучше, чем
OpenMP (который был медленнее в 6.1 раза).

**Почему TBB медленнее SEQ на данной задаче?**

**Микроскопическая вычислительная работа на строку**:

- Матрицы диагональные → 1 ненулевой элемент на строку

- 1 умножение + 1 сложение на строку (2 операции)

- Вычислительная работа крайне мала

**Оверхэд TBB**:

- Создание задач и их распределение

- Work-stealing (кража задач) — полезен для балансировки, но добавляет накладные расходы

- Синхронизация после `parallel_for`

- Для 100,000 строк оверхэд становится доминирующим

**Вывод**: На микроскопической нагрузке ни одна технология не даёт ускорения, но TBB показывает наименьшее замедление
(всего 1.6× против 6.1× у OpenMP). Это подтверждает эффективность work-stealing планировщика TBB.

### Что нужно для получения реального ускорения

Для достижения ускорения необходимо изменить плотность матрицы с 0.0001% хотя бы до 0.1%. Тогда в тесты необходимо
добавить случайное заполнение матрицы:

```cpp
...хэдеры...

namespace tsyplakov_k_mul_double_crs_matrix {

class TsyplakovKRunPerfTestsThreads : public ppc::util::BaseRunPerfTests<InType, OutType> {
  const int kSize_ = 10000;           // матрица размером 10000 на 10000
  const double kDensity_ = 0.001;      // 0.1% плотности (100 ненулевых на строку)
  const double kMinValue_ = -10.0;     // минимальное значение ненулевого элемента
  const double kMaxValue_ = 10.0;      // максимальное значение ненулевого элемента

  InType input_data_{};

  // Генерация случайной разреженной матрицы в формате CRS
  SparseMatrixCRS GenerateRandomSparseMatrix(int size, double density, double min_val, double max_val) {
    SparseMatrixCRS matrix(size, size);

    std::random_device rd;
    std::mt19937 gen(rd());
    std::uniform_real_distribution<double> density_dist(0.0, 1.0);
    std::uniform_real_distribution<double> value_dist(min_val, max_val);
    std::uniform_int_distribution<int> col_dist(0, size - 1);

    std::vector<std::vector<std::pair<int, double>>> rows(size);

    for (int i = 0; i < size; ++i) {
      // Ожидаемое количество ненулевых элементов в строке i
      int expected_nnz_per_row = static_cast<int>(size * density);
      if (expected_nnz_per_row < 1) expected_nnz_per_row = 1;

      // Используем оба подхода для более реалистичного распределения:
      // 1) Часть элементов — случайные позиции
      // 2) Часть элементов — гарантированные (чтобы не было пустых строк)

      // Добавляем хотя бы 1 гарантированный элемент на строку (диагональный)
      rows[i].emplace_back(i, value_dist(gen));

      // Добавляем случайные элементы для достижения желаемой плотности
      int current_nnz = 1;
      while (current_nnz < expected_nnz_per_row) {
        int col = col_dist(gen);
        // Избегаем дублирования столбцов в одной строке
        bool exists = false;
        for (const auto& [c, _] : rows[i]) {
          if (c == col) {
            exists = true;
            break;
          }
        }
        if (!exists) {
          rows[i].emplace_back(col, value_dist(gen));
          current_nnz++;
        }
      }

      // Сортируем по индексу столбца (требование CRS формата)
      std::sort(rows[i].begin(), rows[i].end());
    }

    // Заполняем структуру CRS
    matrix.row_ptr[0] = 0;
    for (int i = 0; i < size; ++i) {
      matrix.row_ptr[i + 1] = matrix.row_ptr[i] + static_cast<int>(rows[i].size());
      for (const auto& [col, val] : rows[i]) {
        matrix.col_index.push_back(col);
        matrix.values.push_back(val);
      }
    }

    return matrix;
  }

  void SetUp() override {
    // Генерируем две случайные разреженные матрицы с одинаковой плотностью
    SparseMatrixCRS a = GenerateRandomSparseMatrix(kSize_, kDensity_, kMinValue_, kMaxValue_);
    SparseMatrixCRS b = GenerateRandomSparseMatrix(kSize_, kDensity_, kMinValue_, kMaxValue_);

    input_data_ = {.a = a, .b = b};
  }

  bool CheckTestOutputData(OutType &output_data) final {
    // Проверяем, что размерность выходной матрицы корректна
    if (output_data.rows != kSize_ || output_data.cols != kSize_) {
      return false;
    }

    // Проверяем, что row_ptr имеет правильный размер
    if (static_cast<int>(output_data.row_ptr.size()) != kSize_ + 1) {
      return false;
    }

    // Проверяем, что row_ptr является неубывающей последовательностью
    for (int i = 0; i < kSize_; ++i) {
      if (output_data.row_ptr[i] > output_data.row_ptr[i + 1]) {
        return false;
      }
    }

    // Проверяем, что values и col_index имеют одинаковый размер
    if (output_data.values.size() != output_data.col_index.size()) {
      return false;
    }

    // Пустой результат допустим, если произведение дало нулевую матрицу
    return true;
  }

  InType GetTestInputData() final {
    return input_data_;
  }
};

TEST_P(TsyplakovKRunPerfTestsThreads, RunPerfModes) {
  ExecuteTest(GetParam());
}

}  // namespace

}  // namespace tsyplakov_k_mul_double_crs_matrix
```

## 9. Выводы

Что показала TBB-реализация на разреженных матрицах:

1. TBB — эффективная библиотека для задач с нетривиальной вычислительной нагрузкой. При достаточном объёме работы на
   итерацию work-stealing планировщик обеспечивает хорошую балансировку.

2. На микроскопических задачах TBB проигрывает последовательной версии, но проигрыш значительно меньше, чем у OpenMP:
   `OpenMP: медленнее в 6.1× TBB: медленнее в 1.6×`
3. Work-stealing планировщик TBB более эффективен на нерегулярной нагрузке, чем статические или динамические схемы
   OpenMP.

4. TBB предоставляет более высокоуровневый API, чем ручное управление потоками, при сохранении хорошей
   производительности.

Что касается с матрицами со **случайным заполнением**, то на них технология TBB демонстрирует **ускорение >3×**, что
подтверждает её эффективность для практических задач. Диагональные матрицы являются "частным" случаем, не отражающим
реальную производительность.
