# Метод трапеций для многомерных интегралов — SEQ

## 1. Контекст

Последовательная реализация используется как baseline-версия для всех дальнейших сравнений.

Алгоритм вычисляет двумерный определённый интеграл методом трапеций на равномерной прямоугольной сетке.

---

## 2. Постановка задачи

Входные данные:

- границы интегрирования;
- размеры сетки;
- тип функции;
- параметр `epsilon`.

Выходные данные:

- численное приближение интеграла.

---

## 3. Последовательный алгоритм

Область интегрирования разбивается на равномерную декартову сетку.

Для каждой точки сетки:

- вычисляются координаты;
- определяются весовые коэффициенты;
- вычисляется значение функции;
- результат добавляется к общей сумме.

Вычислительная сложность:

```math
O(n_x n_y)
```

Память:

```math
O(1)
```

---

## 4. Детали реализации

Основные файлы:

- `seq/include/ops_seq.hpp`
- `seq/src/ops_seq.cpp`

Основная вычислительная логика находится в `ComputeIntegral()`.

Поддерживаются следующие функции:

```cpp
x^2 + y^2
sin(x) * cos(y)
sin(x) + cos(y)
exp(x + y)
```

Механизм уточнения сетки через `epsilon` структурно присутствует, однако в performance-экспериментах использовалась фиксированная сетка с `epsilon = 0.0`.

---

## 5. Проверка корректности

Корректность проверялась общими функциональными тестами.

Проверялись:

- конечность результата;
- детерминированность выполнения;
- корректность вычисляемых значений.

---

## 6. Экспериментальная среда

- ОС: Linux
- Компилятор: GCC
- Тип сборки: Release

Размер сетки:

```cpp
5000 × 5000
```
