## 1. Контекст

Данная реализация представляет собой последовательную версию алгоритма вычисления двумерного интеграла методом трапеций.

SEQ-версия используется как baseline для сравнения производительности всех параллельных backend-ов.

Реализация не содержит средств распараллеливания и выполняет вычисления в одном потоке.

---

## 2. Постановка задачи

Требуется вычислить двумерный определённый интеграл на прямоугольной области методом трапеций.

Входные параметры:

- границы интегрирования;
- размеры сетки;
- тип функции;
- параметр `epsilon`.

Размер сетки performance-тестов:

```cpp
step_n_size = {5000, 5000}
```

Интегрируемая функция:

```math
f(x, y) = x^2 + y^2
```

---

## 3. Базовый алгоритм

Алгоритм использует классическую двумерную схему метода трапеций.

Для каждой точки сетки:

1. вычисляются координаты `(x, y)`;
2. вычисляются весовые коэффициенты;
3. вычисляется значение функции;
4. вклад добавляется к общей сумме.

Итоговое значение интеграла вычисляется как:

```math
I \approx h_x h_y \sum_{i=0}^{n_x}\sum_{j=0}^{n_y} w_i w_j f(x_i, y_j)
```

Асимптотика алгоритма:

```math
O(n_x \cdot n_y)
```

Память:

```math
O(1)
```

---

## 4. Схема вычислений

Последовательная версия выполняет полный обход двумерной сетки без распараллеливания.

Внешний цикл отвечает за координату `x`, внутренний — за координату `y`.

Вычисление интеграла выполняется в одном accumulator.

Дополнительная синхронизация отсутствует.

---

## 5. Детали реализации

```cpp
for (int i = 0; i <= steps[0]; i++) {
  for (int j = 0; j <= steps[1]; j++) {
    result += wx * wy * FunctionTypeChoose(...);
  }
}
```

*Пояснение к реализации:*  
SEQ-версия использует единственный поток выполнения.

Накопление результата производится последовательно,
что гарантирует полную детерминированность вычислений.

Механизм уточнения сетки через `epsilon`
структурно присутствует в реализации,
однако в performance-тестах использовалась фиксированная сетка.
