# Метод трапеций для многомерных интегралов — ALL

## 1. Контекст

ALL-реализация сочетает distributed-memory и shared-memory параллелизм.

Используются:

- MPI между процессами;
- OpenMP внутри процесса.

---

## 2. Гибридная схема распараллеливания

Строки вычислительной сетки распределяются между MPI rank-ами.

Внутри каждого процесса локальные вычисления распараллеливаются OpenMP.

Глобальное объединение результатов выполняется через:

```cpp
MPI_Allreduce
```

OpenMP использует:

```cpp
reduction(+ : local_sum)
```

---

## 3. Детали реализации

Файлы:

- `all/include/ops_all.hpp`
- `all/src/ops_all.cpp`

Распределение строк между процессами вычисляется как:

```cpp
rows_per_proc = total_rows / size
```

Оставшиеся строки распределяются между первыми rank-ами.

Локальные вычисления используют статическое OpenMP schedule.

---

## 4. Проверка корректности

Корректность проверялась сравнением с:

- SEQ;
- OMP;
- TBB.

Результаты оставались детерминированными при различных конфигурациях MPI.
