# Метод трапеций для многомерных интегралов — STL

## 1. Контекст

STL-реализация использует асинхронное выполнение задач через `std::async`.

---

## 2. Схема распараллеливания

Область интегрирования делится на независимые диапазоны.

Каждая асинхронная задача вычисляет:

- локальную сумму;
- частичный результат интегрирования.

Используется:

```cpp
std::async(std::launch::async, ...)
```

После завершения задач результаты объединяются.

---

## 3. Детали реализации

Файлы:

- `stl/include/ops_stl.hpp`
- `stl/src/ops_stl.cpp`

Количество workers фиксировано:

```cpp
int num_threads = 4;
```

Каждый worker обрабатывает собственный диапазон строк.

Синхронизация выполняется через:

```cpp
future.get()
```

Общий accumulator во время вычислений отсутствует.

---

## 4. Проверка корректности

Корректность проверялась сравнением с SEQ-реализацией.

Независимая обработка диапазонов исключает гонки данных.

## 6. Выводы

По сравнению с OpenMP и TBB наблюдаются более высокие runtime-overhead.

Масштабируемость ограничивается затратами на управление асинхронными задачами.
