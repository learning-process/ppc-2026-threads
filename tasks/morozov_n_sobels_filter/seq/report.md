## 1. Контекст

В задачах связанных с обработкой изображений нередко требуется выделять конкретные объекты. Многие алгоритмы нахождения
объектов на изображении требуют выделенных границ объекта. Основной вопрос как же выделить сами границы. Данную задачу
решает оператор Собеля.

В данном отчете описывается последовательный алгоритм оператора (фильтра) Собеля.

Задачи обработки изображения - работа с большими массивами чисел, следовательно параллельная обработка поможет увеличить
эффективность вычислений. Последовательная версия - эталон корректности для будущих параллельных алгоритмов и точка
отсчета времени выполнения операции над изображением.

## 2. Постановка задачи

Требуется выделить ребра на изображении с использованием оператора Собеля.

Входные / выходные данные: изображение в оттенках серого с высотой `H` и шириной `W`.

## 3. Базовый алгоритм

Пошаговое описание.

Алгоритм представляет собой проход по всем пикселям изображения (кроме крайних - граница в 1 пиксель) и вычисления для
них нового значения равного градиенту яркости изображения. Градиент вычисляется по следующей формуле
`sqrt(gradX^2 + gradY^2)`. Градиенты по осям `X` и `Y` получаются путем свертки ядра Собеля для конкретного направления.

```text
Ядро для оси X:
-1   0   +1
-2   0   +2
-1   0   +1

Ядро для оси Y:
-1  -2  -1
 0   0   0
+1  +2  +1
```

Вычисленные значения свертки для пикселя записываются в новое изображение.

Операции свертки для каждого ядра = (9 операций на пиксель) _ 2 = 18; Свертка выполняется для каждого пикселя => `W _
H \* 18` операций занимает фильтрация изображения.

Асимптотика по времени: O(`H` \* `W`)

Так как размеры матриц свертки малы относительно изображения, то их размер можно не учитывать.

Асимптотика по памяти: O(`H` \* `W`)

Инварианты и критерий корректности:

1. Цвет пикселя находится в диапазоне от 0-255 до и после вычисления сверткиж
2. Результирующее значение пикселя `i,j` зависит от его окрестных (матрица 3х3) пикселей исходного изоражения.

## 7. Результаты

Последовательная версия используется как базовая точка отсчета для всех последующих параллельных реализаций. Поэтому
время выполнения SEQ-версии далее используется как знаменатель при расчете ускорения.

| Backend | Конфигурация | Время, с | Ускорение | Эффективность |
| ------- | ------------ | -------- | --------- | ------------- |
| SEQ     | 1 поток      | 0.2336   | 1.00x     | 100%          |

Формулы, которые используются в последующих отчетах:

`Speedup = T_seq / T_parallel`

`Efficiency = Speedup / Workers`

где:

- `T_seq` — время последовательной реализации;
- `T_parallel` — время параллельной реализации;
- `Workers` — количество рабочих потоков или worker-единиц.

Самым дорогим фрагментом последовательной реализации является двойной цикл по внутренним пикселям изображения. Для
каждого пикселя выполняются две свертки ядрами Собеля по осям `X` и `Y`, затем вычисляется итоговая величина градиента
через `sqrt(gradX^2 + gradY^2)` и выполняется ограничение результата диапазоном `[0; 255]`.

Так как каждый пиксель обрабатывается независимо, последовательная версия хорошо подходит в качестве baseline для оценки
ускорения: параллельные версии должны выполнять те же вычисления, но распределять независимые пиксели или блоки пикселей
между рабочими потоками.

## 8. Выводы

Последовательная реализация корректно решает задачу выделения ребер на изображении с использованием оператора Собеля и
используется как эталон для всех параллельных backend-ов.

Именно SEQ-версия выбрана в качестве baseline, потому что:

- она имеет наиболее простую структуру выполнения;
- не содержит параллельного выполнения и, следовательно, не зависит от порядка работы потоков;
- ее результат удобно использовать для попиксельного сравнения с OMP, TBB и STL-реализациями;
- ее время выполнения используется для расчета ускорения и эффективности параллельных версий.

Асимптотика алгоритма составляет `O(H * W)` по времени и `O(H * W)` по памяти. Это означает, что при увеличении размера
изображения основная вычислительная нагрузка растет пропорционально количеству пикселей. За счет независимости обработки
пикселей задача хорошо подходит для распараллеливания.
